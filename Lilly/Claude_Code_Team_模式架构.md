# Claude Code Team 模式架构笔记

## 核心概念

Claude Code Team 模式是一种多智能体协作方案，与旧的 SAG（Single Agent Group）模式对比：

| 特性 | SAG 模式 | Team 模式 |
|------|---------|-----------|
| 上下文持久性 | 每次启动重置 | 子智能体持续待命 |
| 上下文管理 | 200K 上下文易溢出 | 任务拆分到各子智能体分摊 |
| 性能衰减 | 压缩后急剧下降 | 整体稳定 |
| Token 消耗 | 基准 | 约为 SAG 的 2 倍及以上 |

## Team 模式核心优势

1. **上下文不再容易溢出**：每个子智能体独立维护上下文，总 Token 消耗增加但无一被突破
2. **任务自动模块化拆分**：Leader 自动将任务拆分为小模块，分配给对应子智能体

## 推荐的 6 子智能体团队

| 角色 | 职责 |
|------|------|
| 后端开发 (Backend Dev) | 后端功能开发 |
| 前端开发 (Frontend Dev) | 前端界面开发 |
| 研究员 (Researcher) | 需求调研、技术调研 |
| 端到端测试 (E2E Tester) | 测试用例编写与执行 |
| 审查员 (Reviewer) | 代码审查 |
| 重构清洁器 (Refactorer) | 代码重构优化 |

## 关键工程教训

- **必须让 Opus 全量读取 skills**：探索型智能体若未完整了解 skills 就去创建项目，结果不理想
- **文件持久化**：所有进度、Plan、发现写入文档，而非存放在子智能体全局状态
- **信息损失**：子智能体交互存在信息损失问题，因此任务必须拆分为小模块
