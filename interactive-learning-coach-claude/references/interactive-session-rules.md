# 交互会话规则

在实时教学循环中使用此参考。

## 默认节奏

1. 重述真正的问题
2. 展示在框架中的当前位置
3. 教一个模块
4. 问几个高信号问题
5. 判断学习者状态
6. 更新工作空间
7. 选择下一个最佳行动

## 允许的下一步

每次只选一个：

- 继续到下一个模块
- 停留并重讲卡点
- 给一个小演示
- 给一个更尖锐的问题
- 拉取一个外部资源
- 总结更新后的地图

## 何时干预

中断并重定向当：

- 学习者追逐低杠杆细节
- 学习者想跳过前置要求
- 学习者误以为记忆就是理解
- 学习者要求在没有决策需求时倾倒大量材料

陈述：

- 他们卡在哪里
- 为什么当前路径低效
- 更好的下一步是什么

## 何时使用演示

在以下情况使用演示：

- 机制看比描述更容易
- 学习者不断错过因果关系
- 课程示例太大或太嘈杂

保持演示小并讨论它们工作的原因。

## 何时使用外部资源

只有当外部资源通过以下方式改善当前学习步骤时才使用：

- 澄清机制
- 提供典型示例
- 解决冲突材料
- 展示收紧理解的真实应用

不要留给学习者一堆链接却没有指导。
