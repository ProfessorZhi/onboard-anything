## 核心问题
- 如何用Claude Code处理Jupyter笔记本而非普通代码文件
- 如何将混乱的笔记本通过职责分离重构为可维护的代码结构
- 如何将重构成果无缝转换为可部署的Web仪表盘

## 最小知识骨架

1. **read notebook**
Claude Code专用工具，专门处理.ipynb文件，可探索单元格结构并创建修改

2. **职责分离**
将数据加载、指标计算、可视化分离为独立Python模块，提高可测试性和可维护性

3. **Streamlit**
Python仪表盘库，将数据处理代码与可交互Web界面无缝衔接

4. **迭代优化工作流**
生成→检查→发现问题→提出具体需求→修改→验证的循环模式

5. **提示工程策略**
主动询问AI最佳提示方式、提前规划布局、给出具体而非模糊的需求

6. **配置灵活性**
通过参数化（年份、月份筛选）替代硬编码，增强代码复用性

## 关系与依赖

```
read notebook
    ↓ 替代read工具用于笔记本场景
职责分离
    ↓ 模块化基础
    ├── 数据加载模块
    ├── 指标计算模块
    └── 可视化模块
    ↓ 衔接层
Streamlit仪表盘
    ↓ 驱动
迭代优化工作流
    ↑ 反馈
    └── 提示工程策略
```

- read notebook 是 read 的场景专门化
- 职责分离支撑模块化测试和复用
- Streamlit 是数据处理到Web界面的桥梁
- 迭代工作流是整个开发过程的主循环
- 提示工程贯穿所有交互环节

## 易错点

- **长提示显示截断**：粘贴长提示词时Claude Code不显示全部内容，属于终端显示特性非问题
- **默认年份配置**：2024年数据不完整，需手动设置默认年份为2023年
- **AI生成内容需验证**：生成的README等文件可能需要手动调整，不必全盘接受
- **依赖安装遗漏**：Claude Code生成代码后需手动执行pip install，运行环境配置不自动完成
- **README创建策略**：有时修改现有文件比创建新文件更高效，需根据上下文调整指令
