# 资源清单

## 1. Figma MCP服务器

- **名称**：Figma MCP服务器
- **类型**：MCP服务器 / AI扩展工具
- **在本P中的作用**：读取Figma设计文件，提取底层代码（CSS/HTML）和图像，获取设计规范，为Claude Code生成代码提供原始输入
- **来源**：字幕直接提到
- **后续可替代练习方向**：探索Figma API的直接调用方式，或尝试其他设计工具（如Sketch、Adobe XD）的MCP服务器实现

---

## 2. Playwright MCP服务器

- **名称**：Playwright MCP服务器
- **类型**：MCP服务器 / 自动化测试工具
- **在本P中的作用**：自动化浏览器操作、导航到本地开发服务器、截取界面截图验证生成效果，实现"生成→测试→可视化验证"的闭环
- **来源**：字幕直接提到
- **后续可替代练习方向**：使用传统方式（手动打开浏览器）、Puppeteer、或Cypress等端到端测试框架进行效果验证

---

## 3. Figma开发者模式

- **名称**：Figma开发者模式
- **类型**：设计工具功能
- **在本P中的作用**：在Figma中启用开发者模式，允许MCP服务器访问设计文件的底层代码和图层信息；无需Figma Pro订阅即可使用
- **来源**：字幕直接提到
- **后续可替代练习方向**：了解Figma插件开发、Export功能导出设计资源、或使用Figma REST API获取设计数据

---

## 4. Next.js

- **名称**：Next.js
- **类型**：前端框架
- **在本P中的作用**：作为React框架承载整个Web应用，使用create-next-app快速初始化项目，app文件夹下组织组件和页面
- **来源**：字幕直接提到
- **后续可替代练习方向**：使用Vite + React、Gatsby、Remix、或纯HTML/CSS快速原型等其他技术栈替代Next.js

---

## 5. Recharts

- **名称**：Recharts
- **类型**：数据可视化库
- **在本P中的作用**：在React应用中渲染条形图等数据可视化图表，展示联邦储备经济数据
- **来源**：字幕直接提到
- **后续可替代练习方向**：使用Chart.js、Apache ECharts、D3.js、或Visx等图表库替代

---

## 6. 联邦储备经济数据（FRED）API

- **名称**：FRED API
- **类型**：数据API
- **在本P中的作用**：获取真实的经济数据（失业率、联邦基金利率、国债收益率、CPI等），替换Mock数据使应用具有实际价值
- **来源**：字幕直接提到
- **后续可替代练习方向**：集成其他公开API（世界银行API、IMF数据、中国国家统计局数据、Yahoo Finance等），或使用本地JSON/CSV数据文件

---

## 7. create-next-app

- **名称**：create-next-app
- **类型**：脚手架工具 / 命令
- **在本P中的作用**：通过npx执行，快速创建Next.js项目并自动安装依赖
- **来源**：字幕直接提到
- **后续可替代练习方向**：手动搭建Next.js项目结构，或使用T3 Stack、Next.js Bootstrap等预设项目模板

---

## 8. Node.js与npm

- **名称**：Node.js + npm
- **类型**：运行环境 / 包管理器
- **在本P中的作用**：运行Next.js开发服务器，安装recharts等npm依赖
- **来源**：字幕直接提到（未明确命名但贯穿全程）
- **后续可替代练习方向**：使用pnpm或yarn替代npm作为包管理器

---

## 9. Figma账号

- **名称**：Figma账号
- **类型**：平台账号
- **在本P中的作用**：登录Figma访问设计文件，启用开发者模式，使用MCP服务器连接设计
- **来源**：字幕直接提到
- **后续可替代练习方向**：使用其他协作设计平台（如Figma之外的设计工具），或使用开源替代方案（如Penpot）

---

## 10. localhost:3000

- **名称**：localhost:3000
- **类型**：开发服务器地址
- **在本P中的作用**：Next.js开发服务器默认端口，Claude通过Playwright访问该地址进行截图验证
- **来源**：字幕直接提到
- **后续可替代练习方向**：配置不同的端口号，或使用ngrok将本地服务暴露到公网进行远程测试
