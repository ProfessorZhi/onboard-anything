# P9 9-基于Figma原型创建Web应用 知识框架

## 本P位置

- **课程定位**：Claude Code教程第9P，从基础操作进入高级集成的转折点
- **前置内容**：P1-P8完成Claude Code基础操作学习
- **后续铺垫**：为数据可视化、真实API集成等进阶功能奠定基础
- **核心角色**：展示AI与外部设计工具深度集成的核心实操环节

## 核心问题

如何将Figma设计原型快速转化为可运行的Web应用，并通过MCP服务器实现设计到代码的无缝转化

## 最小知识骨架

1. **MCP服务器** — AI与外部工具之间的通信协议桥梁
2. **Figma MCP** — 读取设计文件、提取底层代码和图像的工具
3. **Playwright MCP** — 自动化浏览器测试和截图验证的工具
4. **Next.js + Recharts** — 现代Web应用与数据可视化的技术栈组合
5. **Figma开发者模式** — 启用MCP访问设计文件的权限开关
6. **FRED API** — 联邦储备经济数据的真实数据源
7. **设计→开发→验证工作流** — 完整的AI驱动开发闭环

## 输入 / 输出 / 前置 / 支撑

**输入**

- Figma中的仪表盘原型文件
- 开发者模式下复制的特定图层标识
- 给Claude的自然语言任务指令

**输出**

- 可运行的Next.js Web应用
- 基于Recharts的数据可视化界面
- 真实联邦储备经济数据（失业率、利率、收益率等）

**前置条件**

- Node.js环境和npm
- Figma账号并启用开发者模式
- FRED API密钥（用于获取真实数据）

**支撑工具**

- Figma MCP服务器
- Playwright MCP服务器
- create-next-app脚手架工具

## 易错点

1. **开发者模式≠订阅** — 误以为需要Figma Pro才能使用开发者模式，实际上免费即可启用

2. **MCP配置不跨项目** — 每次新建Claude Code项目都需要重新配置MCP服务器，不会自动继承

3. **重复启动服务器** — 如果开发服务器已在运行，不要再次启动npm命令，避免端口冲突

4. **忽略依赖安装提示** — Claude会自动提示并安装必要依赖（如recharts），学生常忽略这些提示

5. **设计质量预期偏差** — 原型过于简陋时，生成的代码需要更多手动调整，要理解设计质量决定生成效果下限

6. **模型选择不当** — Sonnet可完成大多数任务，但复杂任务建议切换到Opus模型以获得更好表现

7. **API密钥遗漏** — FRED API需要单独获取和配置密钥，代码中若未正确配置会导致数据无法加载
