## 工具类

### Claude Code
- **类型**：AI 编程工具 / CLI 工具
- **在本P中的作用**：核心开发工具，贯穿全P。用于生成计划、修改代码、接受文件上下文、执行工具调用、与 MCP 服务器交互。
- **来源**：字幕直接提到
- **后续可替代练习方向**：可尝试用 Cursor、Copilot Chat、Continue 等 AI 编辑器执行相同的"计划-审批-执行"工作流，对比体验差异。

---

### Playwright MCP 服务器
- **类型**：MCP（Model Context Protocol）服务器 / 浏览器自动化工具
- **在本P中的作用**：扩展 Claude Code 的能力，使其能自动打开浏览器、截屏、分析 UI 问题、程序化执行修复，替代手动截图反馈流程。
- **来源**：字幕直接提到
- **后续可替代练习方向**：可尝试接入其他 MCP 服务器（如文件系统 MCP、数据库 MCP），练习"工具注册 → 验证连接 → 调用工具"完整流程；也可练习用 Puppeteer 替代 Playwright 完成类似自动化测试任务。

---

### Playwright（npx 启动）
- **类型**：Node.js 浏览器自动化框架
- **在本P中的作用**：作为 Playwright MCP 服务器的底层运行时，通过 `npx playwright` 命令启动，为 Claude Code 提供浏览器操控能力。
- **来源**：字幕直接提到（`npx playwright` 命令）
- **后续可替代练习方向**：可独立练习 Playwright 脚本编写，完成页面截图、元素定位、点击操作等，理解 MCP 服务器封装前的底层能力。

## 文件与配置类

### Claude.md
- **类型**：项目内存文件 / 配置文件
- **在本P中的作用**：持久化项目偏好（如"不要用 `./run.sh` 启动服务器"），供 Claude Code 在任意会话中读取，避免重复告知；同时可作为团队内部文档参考，但会被 git 忽略。
- **来源**：字幕直接提到
- **后续可替代练习方向**：在自己的项目中创建 Claude.md，记录项目结构说明、禁止操作、常用偏好，验证不同会话中 Claude 能否正确读取并遵守。

---

### run.sh
- **类型**：Shell 脚本 / 服务器启动脚本
- **在本P中的作用**：项目原有的服务器启动脚本。本P中老师明确告知 Claude "服务器已在运行，无需执行此脚本"，并将该偏好写入 Claude.md。
- **来源**：字幕直接提到（`./run.sh`）
- **后续可替代练习方向**：练习将项目启动约束写入 Claude.md，观察 Claude 在多个会话中是否能正确规避冲突操作。

---

### Python 工具文件（py文件）
- **类型**：后端源代码文件
- **在本P中的作用**：后端开发任务的修改目标。在此文件中新增"获取课程详情"工具函数，实现按课程 ID 查询课次列表的能力。字幕原文为"Pi文件"，疑为语音识别错误，推断为 `.py` 文件。
- **来源**：基于上下文推断（字幕识别存在误差）
- **后续可替代练习方向**：在自己的 Python 项目中，用 Claude Code 计划模式添加一个新函数，体验"计划 → 审批 → 自动执行"的后端开发闭环。

## 协议与机制类

### MCP（Model Context Protocol，模型上下文协议）
- **类型**：扩展协议 / 集成规范
- **在本P中的作用**：作为 Claude Code 的能力扩展机制，允许接入外部数据源和工具系统（本P以 Playwright MCP 为例）。老师强调所有 MCP 服务器遵循统一的连接命令规范，可查阅文档获取最新版本命令。
- **来源**：字幕直接提到
- **后续可替代练习方向**：查阅 MCP 官方文档，尝试手动配置一个非 Playwright 的 MCP 服务器（如 SQLite MCP），熟悉"注册 → 验证 → 调用"的通用流程。

---

### RAG 系统（检索增强生成系统）
- **类型**：AI 应用架构组件
- **在本P中的作用**：项目内部的工具注册与调用中枢。新开发的后端工具需在 RAG 系统中注册，才能被 LLM 在对话中识别并调用。构成"工具开发三步"的最后一环。
- **来源**：字幕直接提到
- **后续可替代练习方向**：阅读 LangChain 或 LlamaIndex 的 Tool/RetrievalQA 文档，对比不同 RAG 框架中工具注册方式的异同；尝试在现有 RAG 项目中手动注册自定义工具。

## 命令与操作类

### `claude mcp` 命令
- **类型**：CLI 命令
- **在本P中的作用**：在 Claude Code 中添加 MCP 服务器，本P用于注册 Playwright MCP 服务器（指定服务器名称为 `playwright`）。
- **来源**：字幕直接提到
- **后续可替代练习方向**：练习用此命令注册、列出、移除 MCP 服务器，理解 Claude Code 的插件管理机制。

### `/mcp` 斜杠命令
- **类型**：Claude Code 内置命令
- **在本P中的作用**：在 Claude Code 会话中查看当前已连接的 MCP 服务器列表及其可用工具。
- **来源**：字幕直接提到
- **后续可替代练习方向**：在接入 MCP 服务器后，用 `/mcp` 验证连接状态，核对工具列表是否与文档一致。

### `npx playwright`
- **类型**：Node.js 命令 / 包执行命令
- **在本P中的作用**：启动 Playwright MCP 服务器的具体命令。
- **来源**：字幕直接提到
- **后续可替代练习方向**：在终端直接运行 `npx playwright --help` 熟悉 Playwright CLI，了解 MCP 服务器封装之下的原始能力边界。

### Shift+Tab（连按两次）
- **类型**：快捷键操作
- **在本P中的作用**：在 Claude Code 中开启计划模式（plan mode），强制 Claude 先生成修改方案、待审批后再执行；同一快捷键也用于启用自动接受修改（Enable auto-accept edits）。
- **来源**：字幕直接提到
- **后续可替代练习方向**：在实际任务中对比"直接执行"与"先进计划模式"的输出质量，建立对计划模式适用场景的判断。

### @标签（文件引用）
- **类型**：Claude Code 内置语法
- **在本P中的作用**：在对话中直接引用文件或文件夹路径，为 Claude 提供精确的代码上下文，配合 Tab 键补全文件名。
- **来源**：字幕直接提到
- **后续可替代练习方向**：练习在不同粒度下使用 @标签（引用单个文件 vs 整个文件夹），观察上下文质量对代码输出的影响。

## 系统提示词（System Prompt）

- **来源**：字幕直接提到
- **后续可替代练习方向**：练习为不同工具撰写系统提示词描述，对比不同描述精度下 LLM 调用工具的准确率。
