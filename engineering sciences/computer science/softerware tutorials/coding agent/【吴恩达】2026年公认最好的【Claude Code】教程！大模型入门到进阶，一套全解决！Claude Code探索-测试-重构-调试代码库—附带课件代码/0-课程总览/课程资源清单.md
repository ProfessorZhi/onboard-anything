- **课程名称**：Claude Code 实战教程——从入门到进阶

| mock | 模拟外部依赖，隔离测试 | P5 |

### 5.2 调试方法论

- **测试驱动调试循环**：编写测试 → 运行测试定位Bug → 修复 → 验证
- **避免无效循环**：不要复制粘贴错误信息给AI，要先编写测试定位根因
- **扩展思考模式**：Claude Code处理复杂任务时会分配更多令牌进行深度分析

## 十、后续可替代练习方向

### 10.1 探索与理解方向

**替代项目选择**：

| 项目类型 | 推荐练习目标 | 预期收获 |
|----------|--------------|----------|
| 开源Python项目 | 用Claude Code探索Django/Flask源码结构 | 练习高层次提问替代逐文件浏览 |
| JavaScript/TypeScript项目 | 探索React/Vue源码，理解组件架构 | 练习非Python生态的代码库理解 |
| 陌生技术栈项目 | 选择完全未接触过的技术栈（如Rust、Go） | 测试Claude Code的泛化理解能力 |

**练习要点**：

- 用@标签精准引用相关文件
- 通过CLAUDE.md持久化项目理解成果
- 验证Claude Code生成的架构理解是否与文档一致

### 10.2 功能开发方向

**替代项目选择**：

| 项目类型 | 推荐练习目标 | 预期收获 |
|----------|--------------|----------|
| CLI工具开发 | 开发一个命令行效率工具（如文件批量重命名） | 练习从需求到可运行工具的完整流程 |
| REST API开发 | 用FastAPI/Express构建一个小而完整的API服务 | 练习前后端分离场景下的功能开发 |
| Chrome扩展开发 | 开发一个浏览器扩展解决实际问题 | 练习MCP扩展能力的替代方案探索 |

**练习要点**：

- 使用Shift+Tab计划模式审批修改
- 截图反馈法用于视觉验证
- 通过Claude.md记录项目偏好

### 10.3 测试与调试方向

**替代项目选择**：

| 项目类型 | 推荐练习目标 | 预期收获 |
|----------|--------------|----------|
| 有已知Bug的开源项目 | 定位并修复issue中描述的Bug | 练习测试驱动调试方法论 |
| 添加新功能到遗留代码 | 为老旧项目添加现代特性（如类型注解、异步支持） | 练习复杂修改前的计划模式 |
| 性能优化项目 | 定位并解决性能瓶颈 | 练习Claude Code的多轮迭代分析 |

**练习要点**：

- 先编写测试，再定位Bug
- 使用子代理并行探索多种解决方案
- 记录调试过程中发现的有价值上下文

### 10.4 重构方向

**练习要点**：

- 通过/compact保留重构过程中的关键决策
- 用计划模式审批重大架构变更
- 通过CLAUDE.md固化重构后的代码规范

## 十一、关键技能路线图

```
阶段一：入门（第1-2P）
├── 理解Claude Code定位与智能代理架构
├── 区分CLI与Sonar操作入口
└── 建立"清晰上下文"方法论认知

├── CLAUDE.md持久化记忆机制

├── 子代理并行探索方案
```
