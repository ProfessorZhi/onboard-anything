## 核心问题

如何利用Claude Code将一个混乱的Jupyter笔记本重构为结构清晰、职责分离的代码库，并最终转换为可部署的Streamlit数据仪表盘，实现从数据分析到Web展示的完整工作流。

## 最小知识骨架

1. **Read notebook工具**：Claude Code读取和解析Jupyter笔记本单元格结构的核心工具
2. **笔记本重构策略**：职责分离原则（数据加载层、业务逻辑层、表现层）
3. **提示词工程实践**：如何撰写高质量重构提示词以获得理想输出
4. **Python模块化拆分**：将笔记本拆分为data_loader.py和metrics.py等专业文件
5. **Streamlit仪表盘构建**：基于Python的数据展示界面开发
6. **迭代式代码优化**：通过多轮提示逐步完善代码质量
7. **部署与环境配置**：requirements.txt管理及依赖安装流程

## 易错点

**提示词模糊**：直接说“重构这个”而不描述具体需求和预期效果，导致Claude无法准确理解意图

**忽视职责分离**：未明确要求代码模块化，可能导致生成代码仍然堆叠在一起，失去重构价值

**数据配置错误**：忽略年份配置和月份筛选功能，直接导致生成的仪表盘出现空值、冗余卡片或不合理的数据展示

**急于求成**：一次性在提示词中要求过多功能，应采用迭代式方法逐步完善

**放弃优化机会**：未意识到可以主动询问Claude获取最佳提示词建议，错失提升输出质量的机会
