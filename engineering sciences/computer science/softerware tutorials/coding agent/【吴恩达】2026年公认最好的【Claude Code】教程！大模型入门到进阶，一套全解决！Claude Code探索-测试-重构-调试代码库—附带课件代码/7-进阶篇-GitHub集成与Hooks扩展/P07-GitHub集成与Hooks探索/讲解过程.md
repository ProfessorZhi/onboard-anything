## 本P定位

本P是整个Claude Code教程的第七部分，起到承上启下的作用。首先，它作为“终端外使用Claude Code”的开篇，介绍GitHub集成功能，让Claude从命令行助手扩展为团队协作工具。同时，本P后半部分引入了Hooks这一高级功能，为后续在Jupyter笔记本等不同环境中操作埋下伏笔。

## 本P主线

本P有两条并行的教学主线：

**主线一：GitHub集成**（00:02.5 - 00:272.9）
从终端内的对话扩展到GitHub平台，让Claude参与代码审查、Issue处理、拉取请求管理等团队协作场景。核心逻辑是“终端能做→GitHub也能做，且能做得更多”。

**主线二：Hooks钩子机制**（00:469.4 - 00:657.8）
介绍Claude Code的操作生命周期钩子，允许用户在工具执行前后注入自定义逻辑，实现测试触发、权限控制、自我审查等高级功能。这是对Claude行为控制的底层能力介绍。

## 易错点

**GitHub集成相关：**

- 安装GitHub应用后需要额外身份验证，容易忽略CLI提示的额外步骤
- 创建拉取请求时不要忘记检查自动生成的Claude操作YAML文件内容
- 合并拉取请求前应确认Claude的审查意见，即使是自己生成的PR也需要审查

**Hooks相关：**

- Hook配置写在 .claude/settings.local.json 中，不是settings.json
- 匹配器默认匹配所有内容，需要明确指定要触发的工具类型
- 编写Hook中的shell命令时必须注意安全性，避免执行危险操作
- Hook配置是Git跟踪的文件，修改后会被版本控制

**工作流程相关：**

- 使用resume标志可以恢复之前的对话上下文，适合完成遗留任务
- Claude生成的代码会自动进行自我审查，这是一种质量保障机制
