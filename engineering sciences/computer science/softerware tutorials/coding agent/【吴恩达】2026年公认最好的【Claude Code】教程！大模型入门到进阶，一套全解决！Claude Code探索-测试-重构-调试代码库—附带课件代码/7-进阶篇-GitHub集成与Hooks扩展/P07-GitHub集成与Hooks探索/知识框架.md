### 骨架3：Hooks生命周期钩子类型
pre-tool-use / post-tool-use / on-notify / on-user-input / on-task-stop / on-subagent-response

### 骨架4：Hook配置结构
匹配器规则（matcher）+ 执行动作（shell命令）→ 写入.claude/settings.local.json

### 骨架5：resume恢复机制
使用--resume标志恢复历史对话上下文，继续未完成任务

### 骨架6：Git协作基础操作
git push origin main / 分支删除 / 工作树清理

## 易错点

- **身份验证遗漏**：安装GitHub App后忽略CLI额外验证步骤，导致集成不完整
- **配置文件位置**：Hook配置必须写入.claude/settings.local.json，而非settings.json
- **匹配器未指定**：matcher默认匹配所有事件，未明确指定工具类型会导致意外触发
- **Shell安全性**：Hook中执行shell命令时未检查危险操作，可能产生安全风险
- **版本控制冲突**：Hook配置被Git跟踪，多人协作时配置变更可能冲突
- **PR合并前审查**：Claude自己生成的PR也需审查意见确认，不能跳过复核
- **YAML文件检查**：创建PR时未检查自动生成的Claude操作YAML文件，可能包含非预期配置
