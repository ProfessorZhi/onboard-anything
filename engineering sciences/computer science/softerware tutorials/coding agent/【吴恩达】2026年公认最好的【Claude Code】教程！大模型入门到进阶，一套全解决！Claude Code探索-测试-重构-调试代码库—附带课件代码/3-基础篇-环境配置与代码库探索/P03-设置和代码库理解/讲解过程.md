## 本P定位

本P是课程进入实操前的"启动阶段"。第一章（探索与理解）和第二章（构建功能）之间的一座桥——在真正让 Claude Code 写代码、改文件之前，先确保学习者掌握如何用 Claude Code 高效地读懂一个陌生的代码库。核心论点是：**Claude Code 是更好的解释者，在让它写代码之前，先让它帮你理解代码**。

## 本P主线

1. **引入端到端 RAG 应用示例**，作为接下来所有演示的载体
2. **展示用 Claude Code 提问来了解代码库整体架构**——智能搜索而非逐文件浏览
3. **追踪处理流程**：从前端请求 → API 端点 → RAG 检索 → 响应生成，提供详细路径
4. **要求生成可视化图表**：演示 Claude Code 可以创建 D3.js / React 可视化
5. **介绍 /init 命令与 CLAUDE.md 机制**：Claude Code 的"记忆系统"，这是后续高效使用的基础
6. **介绍内置辅助命令**：/help、/clear、/compact、Escape 键
7. **演示 Git 协作能力**：Claude Code 自动生成提交消息

### 06:30 - 08:15 /init 命令与 CLAUDE.md 文件系统

**讲了什么：** 这是本 P 的**核心知识点**。老师介绍 Claude Code 推荐的第一件事是运行 `/init` 命令（在 Claude Code 中输入斜杠可以看到内置命令列表）。/init 的作用：用代码库文档初始化一个 **CLAUDE.md** 文件。

**CLAUDE.md 的作用：**

- 为 Claude Code 引入"记忆"，让它知道如何在当前代码库中最佳工作
- 指定运行方式、测试方式、链接、长期记忆内容

**三种 CLAUDE.md 文件的作用域（核心概念）：**

| 文件名 | 位置 | Git 追踪 | 用途 |
|---|---|---|---|
| CLAUDE.md | 项目根目录或子目录 | 是（会提交到版本控制） | 与团队共享的项目记忆 |
| CLAUDE.md.local | 项目根目录或子目录 | 否（被 Git 忽略） | 个人编辑环境和终端环境的自定义说明 |
| CLAUDE.md（全局） | ~/.claude/ 目录 | 否 | 跨所有项目的全局指令 |

老师演示 /init 自动分析代码库，生成了包含项目概述、关键技术、架构、核心组件信息的 CLAUDE.md 文件。老师还展示了如何手动编辑 CLAUDE.md，添加自定义指令，例如"总是使用 UV 来运行服务器，不要直接使用 pip"，验证了编辑确实被写入文件。

**为什么这里讲：** CLAUDE.md 是后续高效使用 Claude Code 的基础设施——没有它，Claude Code 每次对话都是"无记忆的陌生人"；有了它，Claude Code 才真正成为了解你项目的长期助手。这是本 P 最重要的新概念。

**与前后内容的连接：** CLAUDE.md 讲完后，自然引出其他辅助命令的学习。

### 08:15 - 09:10 辅助命令与上下文管理

**讲了什么：** 老师介绍了几个内置辅助命令：

- **/**（或 /help）：显示所有命令的快速描述和总结，帮助开始使用 Claude Code 时快速了解可用能力
- **/clear**：清除对话历史，从头开始。在切换不同年份、不同功能分支时非常有用——清理上下文窗口
- **/compact**：清除历史但**保留摘要**，允许在之前工作的基础上继续构建，而不是完全从头开始
- **Escape 键**：退出正在执行的任何命令（如 /compact 正在进行时）。老师强调"不要觉得如果 Claude Code 没有给你需要的东西，你必须等待它"——Escape 让你主动控制对话节奏

**为什么这里讲：** 这些是日常使用 Claude Code 的基础操作工具，帮助学习者建立"Claude Code 是受你控制的工具，而不是你需要等待的权威"的认知。

## 本P结论

1. **Claude Code 是更好的解释者**：在用它写代码之前，先用它理解代码库，尤其是当你对底层技术不熟悉时
2. **智能搜索优先于逐文件浏览**：用高层次问题快速获得架构、组件和流程的整体视图
3. **Claude Code 可以生成可视化**：支持 D3.js、React 等工具来创建图表和 Web 可视化
4. **CLAUDE.md 是 Claude Code 的记忆系统**：
   - 项目级（提交到 Git，团队共享）
   - 本地级（Git 忽略，仅自己使用）
   - 全局级（跨所有项目的通用指令）
5. **/init 是启动新项目时的第一步**：自动分析代码库并生成 CLAUDE.md
6. **Claude Code 内置命令提供上下文管理**：/clear、/compact、Escape 键让你控制对话节奏
7. **Claude Code 可以自动生成有意义的 Git 提交消息**

## 易错点

1. **混淆三种 CLAUDE.md 的作用域**：将本地配置提交到 Git，或期望全局配置在某项目内生效。需要明确：项目级随项目走，本地级不提交，全局级跨所有项目
2. **不知道 Escape 可以中断**：等待 Claude Code 完成当前任务，即使已经得到了不需要的答案。正确做法：随时按 Escape 终止
3. **忘记 /init 可以自动生成**：手动从头编写 CLAUDE.md，效率低；应该先用 /init 生成框架再补充
4. **用 Claude Code 写代码前不先理解代码**：老师反复强调"先解释再编写"，否则无法判断 Claude Code 给出的代码是否正确
5. **混淆 /clear 和 /compact**：/clear 完全清除历史，/compact 保留摘要——需要保留上下文时应该用 /compact
6. **忽略 CLAUDE.md 中的自定义指令价值**：CLAUDE.md 不只是存项目说明，还可以存特定工具偏好（如"总是用 UV 而非 pip"），对团队协作有实际约束作用
