## 结构识别题

**题目：以下是关于 CLAUDE.md 配置的三条描述，请判断正误并说明理由**

1. 项目根目录下的 `CLAUDE.md` 会被 Git 追踪，适合存放团队共享的代码库说明
2. `CLAUDE.local.md` 存放在项目根目录，用于存储本地调试偏好
3. `~/.claude/` 目录下的配置可以跨项目生效，且所有协作成员共享

## 最小实现题

1. 使用 Claude Code 的 `/init` 命令初始化项目
2. 用一条高层次自然语言提问，获取该代码库的整体架构视图
3. 选择合适的辅助命令，清空当前对话但保留项目上下文
4. 追踪一条用户查询从发送到返回响应的完整流程
5. 假设你需要本地偏好"使用 UV 而非 pip 管理依赖"，说明如何通过 CLAUDE.md 实现

## 最小替代练习

**题目：Claude Code 的 CLI 交互模式并非唯一选择，GitHub Copilot 的 VS Code 扩展也支持类似的对话式代码理解。请查阅官方文档，对比两者在以下维度的差异**

- 上下文持久化机制（是否支持类似 CLAUDE.md 的文件系统）
- 内置命令覆盖度（如 `/init`、`/compact` 是否有对应功能）
- 对代码库整体架构的探索能力
