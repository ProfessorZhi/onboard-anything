## 核心问题

如何用Claude Code高效理解陌生代码库？

## 最小知识骨架

1. **RAG应用探索模式** —— 用高层次自然语言提问替代逐文件浏览，快速获得架构视图
2. **智能流程追踪** —— 前端请求→API端点→RAG检索→响应生成的完整生命周期
3. **CLAUDE.md文件系统** —— 三种作用域（项目/本地/全局）的持久化记忆机制
4. **/init命令** —— 自动生成CLAUDE.md的标准入口，分析代码库并建立上下文
5. **内置辅助命令** —— /clear（清空）、/compact（压缩保留摘要）、Escape（中断）
6. **Git协作集成** —— 自动生成描述性提交消息

## 关系与依赖

```
/init命令 → 生成 CLAUDE.md
RAG探索 → 流程追踪（递进深化）
可视化图表 → 增强流程追踪的理解效果
辅助命令 → 控制对话节奏，贯穿所有操作
CLAUDE.md → 为后续所有操作提供上下文基础
Git协作 → 本P收尾，下一课构建功能的起点
```

## 易错点

1. **混淆探索与构建阶段** —— 用Claude Code写代码前不先理解代码，无法判断代码是否正确
2. **混淆/clear与/compact** —— /clear完全清空，/compact保留摘要；需要上下文时必须用/compact
3. **忽视Escape键中断能力** —— 等待Claude Code完成不需要的任务，正确的做法是随时按Escape终止
4. **混淆三种CLAUDE.md作用域** —— 将本地配置提交到Git，或期望全局配置在单个项目内生效
5. **手动编写CLAUDE.md** —— 应该先用/init自动生成框架再补充，而非从头编写
6. **忽视CLAUDE.md自定义指令** —— 以为CLAUDE.md只存项目说明，可以约束工具偏好（如"总是用UV而非pip"）
