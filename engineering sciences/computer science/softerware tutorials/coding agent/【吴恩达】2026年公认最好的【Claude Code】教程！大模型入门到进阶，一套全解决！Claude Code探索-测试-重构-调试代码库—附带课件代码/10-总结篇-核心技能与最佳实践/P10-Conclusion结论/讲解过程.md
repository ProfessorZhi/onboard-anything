## 讲解过程

### 00:00 - 00:55 课程回顾与最佳实践建议

**开场定位（00:00-00:10）**

老师以“恭喜大家”直接开场，确认学生已经学完了全部课程内容，建立学习完成的仪式感和成就感。

**核心技能回顾（00:10-00:25）**

老师明确列举了学生在课程中习得的四大核心能力：

- **探索（Explore）**：理解代码库结构
- **测试（Test）**：编写和运行测试验证功能
- **重构（Refactor）**：改进代码质量
- **调试（Debug）**：定位和修复问题

这一回顾与课程标题中的四个关键词形成对应，强化学生对课程核心目标的认知。

**最佳实践原则（00:25-00:40）**

老师提出Claude Code发挥最佳效果的前提条件，这些原则贯穿整个课程的技术演示：

1. **给出清晰的指令**：明确告诉Claude Code要完成什么任务
2. **提供明确的上下文**：让Claude Code理解项目的背景和约束
3. **指向相关文件**：在对话中明确引用需要操作的具体文件路径

这三点是老师在整个课程中反复强调的方法论，本P作为最终总结将其提炼为使用Claude Code的通用原则。

**具体配置建议（00:40-00:50）**

老师给出两个可立即执行的行动项：

- **将代码库规范添加到Claude Code MD配置中**：指CLAUDE.md文件，这是Claude Code在项目启动时读取的配置文件，老师在课程中已经演示过这一实践
- **添加项目相关信息**：将任何希望Claude Code在对话中记住的项目上下文写入配置
- **扩展功能，连接MCP服务**：老师列举了具体的服务名称作为示例：Playwright（浏览器自动化测试）和Figma（设计工具集成）

**收尾寄语（00:50-00:55）**

老师表达感谢并以期待作结，呼应课程开头建立的学习社群感。

## 本P结论

1. **四大核心技能**：探索、测试、重构、调试代码库
2. **最佳使用原则**：清晰指令 + 明确上下文 + 指向相关文件
3. **配置建议**：在CLAUDE.md中添加代码库规范和项目信息

5. **实践导向**：期待学生将所学技能应用到实际项目中

## 易错点

1. **仅使用模糊指令**：不给出清晰的任务描述，导致Claude Code无法理解真实意图
2. **忽略上下文提供**：不说明项目背景、约束条件或已有的实现逻辑
3. **不指定相关文件**：让Claude Code在庞大的代码库中猜测应该操作哪些文件
4. **不配置CLAUDE.md**：每次对话都需要重复说明项目规范，降低效率
5. **不扩展MCP能力**：仅使用Claude Code的默认功能，错过与外部工具链集成的机会
