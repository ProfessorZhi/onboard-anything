## 本P位置

## 核心问题

如何将Claude Code四大核心技能（探索、测试、重构、调试）内化为可复用的工程方法论，并落地到实际项目中？

## 最小知识骨架

1. **Explore** - 理解代码库结构，快速定位关键代码位置
2. **Test** - 编写和运行测试，验证功能正确性
3. **Refactor** - 改进代码质量，提升可维护性
4. **Debug** - 定位和修复问题，确保系统稳定
5. **清晰指令原则** - 明确告诉AI要完成什么任务，而非模糊描述
6. **明确上下文原则** - 提供项目背景、约束条件和已有的实现逻辑
7. **指向相关文件原则** - 在对话中明确引用需要操作的具体文件路径
8. **CLAUDE.md配置** - 将代码库规范持久化为Claude Code启动时读取的配置
9. **MCP服务扩展** - 通过Model Context Protocol连接Playwright、Figma等外部工具

## 关系与依赖

四大操作技能（Explore/Test/Refactor/Debug）构成Claude Code的核心能力域，三条使用原则（清晰指令+明确上下文+指向文件）是释放这些能力的前提条件，CLAUDE.md配置是将原则固化为可复用配置的实现机制，MCP扩展则是能力边界的向外延伸。

核心链条：明确上下文 → 给出清晰指令 → 指向相关文件 → 执行操作（Explore/Test/Refactor/Debug）

## 输入 / 输出 / 前置 / 支撑

**输入**：待处理的代码库 + 用户的问题描述和操作意图

**输出**：修复后的代码、可运行的测试用例、重构后的代码模块

**前置**：代码库基础操作能力 + Claude Code CLI工具使用

**支撑**：Claude Code CLI、CLAUDE.md配置文件、MCP服务框架

## 资源与外部对象

Claude Code CLI、CLAUDE.md（配置文件）、MCP服务框架、Playwright（浏览器自动化测试）、Figma（设计工具集成）

## 易错点

仅使用模糊指令导致Claude Code无法理解真实意图。忽略上下文提供，不说明项目背景和约束条件。不指定相关文件，让AI在庞大代码库中盲目猜测。每次对话重复说明项目规范，未配置CLAUDE.md实现持久化。忽视MCP能力扩展，仅使用默认功能而错过与外部工具链集成的机会。
