## 本P资源清单

### Claude Code MD (CLAUDE.md)

- 类型：配置文件
- 在本P中的作用：Claude Code在项目启动时读取的配置文件，用于将代码库规范持久化，避免每次对话重复说明项目背景和约束条件
- 来源：字幕直接提到
- 后续可替代练习方向：研究不同类型项目（前端/后端/全栈）的CLAUDE.md规范写法，对比最佳实践差异

### MCP (Model Context Protocol)

- 类型：服务框架 / 协议
- 在本P中的作用：连接Claude Code与外部工具的协议层，通过MCP服务可扩展Claude Code的能力边界，实现与测试工具、设计工具等的深度集成
- 来源：字幕直接提到
- 后续可替代练习方向：阅读MCP官方文档，了解如何搭建自定义MCP服务器

### Playwright

- 类型：测试工具（MCP服务）
- 在本P中的作用：浏览器自动化测试框架，作为MCP服务示例被提及，展示Claude Code可通过MCP连接外部测试工具
- 来源：字幕直接提到
- 后续可替代练习方向：使用Playwright编写端到端测试用例，集成到CI/CD流程

### Figma

- 类型：设计工具（MCP服务）
- 在本P中的作用：设计工具集成，作为MCP服务示例被提及，展示Claude Code与设计工具链路的连接可能
- 来源：字幕直接提到
- 后续可替代练习方向：探索Figma MCP集成实践，实现设计稿到代码的协作流程

### Claude Code CLI

- 类型：命令行工具
- 在本P中的作用：Claude Code的命令行界面工具，是执行探索、测试、重构、调试四大核心操作的基础运行环境
- 来源：基于上下文推断（课程贯穿使用该工具，P2-P9所有演示均依赖此工具）
- 后续可替代练习方向：结合Shell脚本和Makefile实现Claude Code工作流自动化

### Claude Code

- 类型：AI编程助手 / 开发工具
- 在本P中的作用：课程核心工具，通过清晰指令、明确上下文、指向相关文件三项原则驱动，探索代码库、编写运行测试、重构代码、定位修复问题
- 来源：基于上下文推断（课程主题和标题明确指向该工具，P2-P9持续使用）
- 后续可替代练习方向：在真实开源项目中应用四大技能组合，完成从问题发现到解决的全流程
