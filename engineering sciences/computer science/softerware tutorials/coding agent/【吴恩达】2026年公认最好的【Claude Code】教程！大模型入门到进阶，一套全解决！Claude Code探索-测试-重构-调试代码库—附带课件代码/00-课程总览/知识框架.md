## 第一部分：认知基础层

### 第1章：入门与基础认知

**包含分P**：P1-Introduction介绍、P2-什么是Claude Code？、P3-设置和代码库理解

**内容定位**：
- 建立对 Claude Code 的基础认知
- 理解工具定位与适用场景
- 完成开发环境配置
- 学习如何阅读和理解现有代码库

**能力边界**：本阶段不涉及实际功能开发，专注于工具认知和环境准备

**前置模块**：无（本模块为课程起点）

**后置模块**：→ 第2章（核心开发能力）

---

## 第二部分：核心技能层

### 第2章：核心开发能力——添加功能

**包含分P**：P4-添加功能

**核心问题**：如何运用 Claude Code 完成一个功能的完整开发？

**主干知识骨架**：

| 编号 | 知识模块 | 关键内容 |
|------|----------|----------|
| 1 | 工作方法论三件套 | @标签引用文件、计划模式（Shift+Tab×2）、自动接受修改（Shift+Tab×1） |
| 2 | 前端功能开发流程 | 需求分析 → 引用文件 → 计划模式 → 审查计划 → 自动执行 → 验证 |
| 3 | 视觉迭代方法 | 截图分析方法——直接粘贴截图 + 描述期望效果，Claude 自主分析与修改 |
| 4 | 对话上下文管理 | 何时清空对话、如何清空（新对话按钮） |
| 5 | MCP 服务器基础 | Model Context Protocol 协议，连接外部数据源和系统的协议标准 |
| 6 | Playwright MCP | 浏览器自动化测试，支持截图、按键、导航、截图等能力 |
| 7 | 后端工具开发 | 理解架构 → 制定计划 → 创建工具代码 → 更新系统提示 → 注册 RAG → 测试 |

**能力边界**：
- 掌握单功能开发的完整工作流
- 能够使用截图进行视觉迭代
- 了解 MCP 服务器的基本概念和 Playwright MCP 的使用

**前置模块**：第1章（入门与基础认知）

**后置模块**：→ 第3章（测试驱动开发与代码重构）

---

### 第3章：测试驱动开发与代码重构

**包含分P**：P5-测试、错误调试和代码重构

**核心问题**：如何用测试驱动的系统方法调试错误，并重构代码以支持更复杂的工具调用场景？

**主干知识骨架**：

| 编号 | 知识模块 | 关键内容 |
|------|----------|----------|
| 1 | 测试先行调试法 | 不直接复制错误给AI，先编写测试建立验证基础，通过测试定位根因 |
| 2 | 多文件协作架构 | 理解 ai_generator.py、rag.py、tools.py 的交互关系 |
| 3 | Claude Code 配置能力 | think（扩展思考）、计划模式（执行前确认）、task（调度子代理） |
| 4 | 单轮→多轮工具调用重构 | 支持迭代调用所有必要工具，积累信息后生成响应 |
| 5 | 子代理并行探索策略 | 调度多个子代理同时头脑风暴不同重构方案 |
| 6 | 向后兼容设计 | 重构核心逻辑时确保不破坏现有系统 |

**能力边界**：
- 建立系统化的错误调试方法论
- 能够将单轮工具调用重构为多轮架构
- 掌握使用测试验证外部行为的方法

**前置模块**：第2章（核心开发能力——添加功能）

**后置模块**：→ 第4章（并行开发与自定义命令）

## 第三部分：效率扩展层

### 第4章：并行开发与自定义命令

**包含分P**：P6-同时添加多个功能

**核心问题**：如何让多个 Claude Code 实例在同一项目中并行工作而不互相覆盖文件？

**主干知识骨架**：

| 编号 | 知识模块 | 关键内容 |
|------|----------|----------|
| 1 | 自定义命令创建 | Claude/.claude/commands/ 文件夹，Markdown 格式，$ARGUMENTS 变量引用 |
| 2 | commands vs Claude.md | commands 需手动触发，Claude.md 全局自动加载 |
| 3 | Git 工作树概念 | 代码库的独立工作副本，共享仓库历史，独立文件系统操作 |
| 4 | Git 工作树命令 | git worktree add / list / remove |
| 5 | 多实例并行开发 | 每个工作树启动独立终端，每个终端运行独立 Claude Code 会话 |
| 6 | 合并与冲突处理 | Claude Code 可辅助分析冲突并解决 |
| 7 | 命令预授权机制 | .settings.json 配置常用命令自动批准 |

**能力边界**：
- 解决多功能同时开发时的文件覆盖问题
- 掌握自定义命令的创建与参数传递
- 能够使用 Git 工作树实现真正的并行开发

**前置模块**：第3章（测试驱动开发与代码重构）

**后置模块**：→ 第5章（平台集成与自动化）

---

### 第5章：平台集成与自动化

**包含分P**：P7-GitHub集成与Hooks探索

**核心问题**：如何将 Claude Code 能力从终端扩展到 GitHub 平台，以及如何通过 Hooks 机制在工具执行过程中注入自定义自动化逻辑？

**主干知识骨架**：

| 编号 | 知识模块 | 关键内容 |
|------|----------|----------|
| 1 | GitHub App 安装与身份验证 | Claude Code 接入 GitHub 的入口 |
| 2 | GitHub Actions 工作流配置 | YAML 配置文件定义自动审查等行为的触发条件和执行逻辑 |
| 3 | PR 自动代码审查 | GitHub Actions 驱动的 Claude 分析 PR 变更、检查质量、识别安全问题 |
| 4 | Issue 驱动开发 | 在 Issue 中 @提到 Claude，触发分析问题、生成 PR 的完整闭环 |
| 5 | Claude Code Hooks 事件模型 | pre_tool_use / post_tool_use / on_tool_use / on_user_message / on_task_stop / on_sub_agent_response |
| 6 | Hooks 配置与 matcher 规则 | settings.json 中定义触发条件和执行命令的对应关系 |

**能力边界**：
- 将开发能力从终端扩展到 GitHub 平台
- 实现自动代码审查工作流
- 掌握自定义钩子逻辑的配置方法

**前置模块**：第4章（并行开发与自定义命令）

**后置模块**：→ 第6章（多场景应用）

## 第四部分：场景应用层

### 第6章：多场景应用——数据处理

**包含分P**：P8-重构Jupyter笔记本并创建仪表盘

**核心问题**：如何利用 Claude Code 将混乱的 Jupyter 笔记本重构为结构清晰的代码库，并转换为可部署的 Streamlit 数据仪表盘？

**主干知识骨架**：

| 编号 | 知识模块 | 关键内容 |
|------|----------|----------|
| 1 | Read notebook 工具 | Claude Code 读取和解析 Jupyter 笔记本单元格结构的核心工具 |
| 2 | 笔记本重构策略 | 职责分离原则（数据加载层、业务逻辑层、表现层） |
| 3 | 提示词工程实践 | 如何撰写高质量重构提示词以获得理想输出 |
| 4 | Python 模块化拆分 | 拆分为 data_loader.py、metrics.py 等专业文件 |
| 5 | Streamlit 仪表盘构建 | 基于 Python 的数据展示界面开发 |
| 6 | 迭代式代码优化 | 通过多轮提示逐步完善代码质量 |

**能力边界**：
- 从 Web 应用开发转向数据科学场景
- 实现 Jupyter 笔记本的职责分离重构
- 生成可部署的 Streamlit 仪表盘

**前置模块**：第5章（平台集成与自动化）

**后置模块**：→ 第7章（多场景应用——设计工具集成）

### 第7章：多场景应用——设计工具集成

**包含分P**：P9-基于Figma原型创建Web应用

**核心问题**：如何将 Figma 设计原型快速转化为可运行的 Web 应用，并通过 MCP 服务器实现设计到代码的无缝转化？

**主干知识骨架**：

| 编号 | 知识模块 | 关键内容 |
|------|----------|----------|
| 1 | MCP 服务器 | AI 与外部工具之间的通信协议桥梁 |
| 2 | Figma MCP | 读取设计文件、提取底层代码和图像的工具 |
| 3 | Playwright MCP | 自动化浏览器测试和截图验证的工具 |
| 4 | Next.js + Recharts | 现代 Web 应用与数据可视化的技术栈组合 |
| 5 | Figma 开发者模式 | 启用 MCP 访问设计文件的权限开关（免费可用） |
| 6 | FRED API | 联邦储备经济数据的真实数据源 |
| 7 | 设计→开发→验证工作流 | 完整的 AI 驱动开发闭环 |

**能力边界**：
- 使用 MCP 服务器连接外部工具
- 实现从 Figma 设计原型到可运行 Web 应用的无缝转化
- 整合真实 API 数据到应用中

## 跨章节核心概念地图

### MCP 服务器体系

```
MCP（Model Context Protocol）
    │
    ├── Figma MCP → P9（Figma 设计读取）
    │
    ├── Playwright MCP → P4、P9（浏览器自动化测试）
    │
    └── 其他 MCP → P8 数据处理场景扩展
```

### 配置体系

```
配置文件层级
    │
    ├── Claude.md（全局自动加载）
    │
    ├── .claude/commands/（需手动触发）
    │
    └── .settings.json（命令预授权）
```

### 工作流模式

```
计划模式（Plan Mode）
    │
    ├── Shift+Tab×2：激活计划模式
    │
    ├── 适用：大修改、多文件、前后端配合
    │
    └── 输出：待办清单式计划
```
