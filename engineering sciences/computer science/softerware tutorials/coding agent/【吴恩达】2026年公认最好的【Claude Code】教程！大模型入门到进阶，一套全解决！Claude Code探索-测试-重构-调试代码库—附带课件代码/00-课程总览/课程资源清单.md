## 常见易错点汇总

| 易错点 | 说明 | 正确做法 |
|--------|------|----------|
| 混淆commands和Claude.md作用域 | 自定义命令需手动触发，Claude.md才全局生效 | 区分使用场景，配置用Claude.md，指令用commands |
| 忽略工作树隔离保护 | 多终端操作导致文件覆盖 | 必须通过git worktree创建隔离环境 |
| MCP配置不跨项目 | 每次新建项目需重新配置MCP | 将MCP配置步骤文档化，便于复用 |
| 开发者模式≠订阅 | 误以为需Figma Pro才能使用开发者模式 | 免费版即可启用开发者模式 |
| 合并前未规范提交 | 工作树内随意操作导致合并困难 | 保持良好提交习惯，分支清晰 |
| 忽略依赖安装提示 | Claude自动提示安装依赖时被忽略 | 关注Claude的依赖安装建议 |
| API密钥遗漏 | FRED等API需要单独配置密钥 | 记录所有API密钥配置步骤 |
| 盲目信任AI生成结果 | PR审查后直接合并 | Claude生成的代码需自我审查 |
