## 本P位置

课程第1P，课程入口与认知奠基环节。前置：无。后续：技术架构详解（Eli）+ 三个实战案例分P展开。

## 核心问题

Claude Code 是什么、为什么现在需要系统学习它、课程将覆盖什么、如何正确理解其工作原理。

## 最小知识骨架

1. **Claude Code工具定位** — Andrew Ng个人背书为"目前最喜欢的编码工具"，Anthropic开发，代表AI辅助编码的飞跃
2. **AI编码工具演进阶段** — 简单问答→自动补全→自主Agent，Claude Code显著延长独立工作时间，支持多实例并行
3. **核心方法论：清晰上下文** — 指向相关文件+描述功能特性+正确使用MCP生态，三层结构
4. **三个实战案例地图** — RAG完整开发（全栈重构测试）/ Jupyter数据分析（探索可视化）/ Figma前端应用（MCP集成）
5. **架构原理：非语义索引** — 不依赖向量化/图结构，仅依赖grep/ls等简单文件操作，靠Agent"做笔记"驱动决策
6. **课程价值双向承诺** — 非用户学方法论，现有用户学最佳实践，通用性框架
7. **安全预告** — Eli预告后续讨论架构相关安全问题

## 关系与依赖

                                              ↓

## 资源与外部对象

- **EdAI** — 教育平台，与Anthropic联合推出
- **Anthropic** — Claude Code开发商，合作方
- **Claude Code** — 核心工具，AI编码助手
- **Colab** — 演示用云端Jupyter环境
- **MCP服务器（Model Context Protocol）** — 扩展生态，连接外部工具

## 易错点

- **误认为Claude Code依赖语义搜索** — 本P明确澄清其基于简单文件操作而非语义索引
- **将本课当作功能介绍课** — 核心是"最佳实践"方法论，非工具罗列
- **低估上下文的关键性** — 架构不持有语义理解，Agent完全依赖显式提供的上下文
- **忽略安全维度** — Eli预告的安全问题涉及Agent自主执行风险
