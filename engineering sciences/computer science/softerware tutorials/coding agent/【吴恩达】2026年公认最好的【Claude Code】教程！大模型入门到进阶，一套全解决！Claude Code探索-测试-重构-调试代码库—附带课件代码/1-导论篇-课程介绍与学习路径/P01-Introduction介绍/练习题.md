## 结构识别题

### 题目 1

Claude Code 的架构原理是"非语义索引"，但其使用方法论强调"清晰上下文"。

请绘制：从"用户输入问题"到"Claude Code给出正确回应"，这两层机制（架构原理 ↔ 使用方法论）之间的协作流程图。

> **考察点**：架构（文件操作+做笔记）与方法论（上下文三要素）的因果关系

---

### 题目 2

课程预告了三个实战案例：RAG完整开发、Jupyter数据分析、Figma前端应用。

请问这三个案例分别对应"清晰上下文"三要素（指向文件、描述功能、使用MCP）中的哪些？为什么 Figma 案例必须依赖 MCP，而 RAG 案例不需要？

> **考察点**：MCP 在上下文构建中的角色；案例与方法的映射

---

## 机制理解题

### 题目 3

Claude Code 明确**不依赖**语义嵌入、向量化、图结构等语义搜索技术。

如果让你向一位初学者解释"它如何在不知道代码语义的情况下找到相关文件"，你会用什么比喻？请结合"做笔记驱动决策"这个机制来描述。

> **考察点**：非语义索引的内在逻辑；Agent 做笔记的工作方式

---

### 题目 4

课程指出 Claude Code 代表"AI编码工具演进"的第三阶段（自主Agent），其关键能力是"显著延长独立工作时间，支持多实例并行"。

对比第二阶段（自动补全），第三阶段在**上下文管理**上有什么本质区别？请从上下文来源、上下文持有者、上下文更新频率三个维度回答。

> **考察点**：演进阶段的本质差异；上下文管理范式转变

---

## 最小实现题

### 题目 5

假设 Claude Code 的核心逻辑可以简化为以下步骤：
① 读取目标文件列表 → ② 执行文件内容摘要（"做笔记"）→ ③ 基于笔记推理下一步行动 → ④ 执行操作（编辑/测试/探索）

请用自然语言描述：一个用户想要"重构项目中所有涉及数据库连接的函数"，Claude Code 会如何依次执行上述四步？

> **考察点**：对 Agent 工作流的整体把握；"做笔记"机制的具体表现
