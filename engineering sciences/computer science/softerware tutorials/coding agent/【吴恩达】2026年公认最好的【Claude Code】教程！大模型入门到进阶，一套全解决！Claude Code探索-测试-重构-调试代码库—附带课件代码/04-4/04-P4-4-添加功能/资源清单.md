# P4 4-添加功能 资源清单

## Claude Code 内置功能

### @标签引用文件

- **类型**：功能/交互方式
- **在本P中的作用**：指定Claude需要阅读的文件/文件夹，提供上下文让Claude准确理解项目结构，是提高代码质量的关键操作
- **来源**：字幕直接提到
- **后续可替代练习方向**：在自己的项目中练习使用@标签引用不同类型文件（源码、配置、文档），对比有上下文和无上下文时Claude的响应质量差异

### 计划模式（Plan Mode）

- **类型**：功能/交互方式
- **在本P中的作用**：通过Shift+Tab激活，先让Claude制定修改计划，批准后再执行，适合较大修改，降低出错风险
- **来源**：字幕直接提到
- **后续可替代练习方向**：在涉及多文件修改时练习使用计划模式，对比计划执行与直接执行的差异

### 自动接受修改（--approve-all）

- **类型**：功能/设置
- **在本P中的作用**：启用后避免重复授权信任的操作，提升效率，配合CLAUDE.md可记录偏好
- **来源**：字幕直接提到
- **后续可替代练习方向**：在信任的操作序列中启用此功能，测试其对效率的影响

### 截图粘贴功能

- **类型**：功能/交互方式
- **在本P中的作用**：允许粘贴截图让Claude分析视觉问题，比语言描述更准确，减少来回沟通
- **来源**：字幕直接提到
- **后续可替代练习方向**：在前端项目中用截图方式描述样式问题，对比文字描述与截图描述的效果差异

### CLAUDE.md文件

- **类型**：文件/项目配置
- **在本P中的作用**：记录项目级偏好和规则，帮助Claude理解团队习惯（如“不启动服务器”），避免重复错误
- **来源**：字幕直接提到
- **后续可替代练习方向**：为自己的项目创建CLAUDE.md，尝试记录不同的偏好设置，观察Claude的遵循程度

### /mcp命令

- **类型**：命令/工具
- **在本P中的作用**：用于查看已连接的MCP服务器状态，确认外部工具连接情况
- **来源**：字幕直接提到
- **后续可替代练习方向**：添加不同的MCP服务器后练习使用此命令查看状态
