## 核心问题
**如何在多个Claude Code实例并行工作时避免文件覆盖冲突？**

## 最小知识骨架

1. **git worktree**：在独立目录创建代码库副本的Git机制
2. **自定义命令**：在`Claude/commands/`下创建Markdown文件定义可复用指令
3. **$arguments**：自定义命令中引用传入参数的变量
4. **CLAUDE.md作用域**：全局配置 vs 自定义命令的跨对话复用
5. **git worktree add/list/remove**：工作树的创建/查看/删除命令
6. **git merge + 冲突解决**：多分支合并流程及Claude Code辅助解决冲突
7. **权限命令机制**：通过`.claude/settings/local.json`预授权高频命令

## 关系与依赖

```
自定义命令铺垫（2-6min）
    ↓
git worktree引入（9-10min）→ 核心解决方案
    ↓
创建工作树（10-15min）
    ↓
并行执行任务（18-28min）→ 调用自定义命令
    ↓
分别提交（30-33min）
    ↓
git merge合并（33-37min）→ 冲突处理（37-39min）
    ↓
删除工作树（39-41min）→ 生命周期完成
```

## 输入 / 输出 / 前置 / 支撑

| 类型 | 内容 |
|------|------|
| **输入** | 多会话并行需求；自定义命令的Markdown定义 |
| **输出** | 多工作树并行开发能力；合并后完整代码库 |
| **前置** | 单会话Claude Code操作（P1-P5）；Git基础 |
| **支撑对象** | `Claude/commands/`目录；`trees/`文件夹；`.claude/settings/local.json` |

## 资源与外部对象

| 对象 | 作用 |
|------|------|
| `Claude/commands/` | 自定义命令存放位置 |
| `implement feature.md` | 示例自定义命令文件 |
| `$arguments` | 自定义命令参数变量 |
| `CLAUDE.md` | 全局配置文件（对比对象） |
| `.claude/settings/local.json` | 本地权限配置 |
| `trees/` | 工作树根目录 |
| `ui功能/测试功能/质量功能` | 三个并行工作树 |
| `piproject.toml` | 冲突发生地（跨工作树修改） |

## 易错点

1. **自定义命令内容不自动加入上下文**——需手动调用，不同于CLAUDE.md
2. **$arguments变量缺失**——带参自定义命令必须使用此变量
3. **无工作树直接多终端并行**——同一文件必然覆盖冲突
4. **合并前未分别提交**——工作树更改无法正确合并
5. **冲突解决跳过测试验证**——Claude Code辅助仍需测试保证正确性
6. **高频命令每次手动授权**——应配置local.json预授权
