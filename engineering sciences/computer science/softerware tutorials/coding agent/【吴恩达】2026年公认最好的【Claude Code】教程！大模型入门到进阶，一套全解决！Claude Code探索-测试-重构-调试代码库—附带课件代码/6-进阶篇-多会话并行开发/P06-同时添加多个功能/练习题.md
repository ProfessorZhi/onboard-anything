对比以下两种配置的自动性：

| 配置对象 | 放置位置 | 新会话时是否**自动**生效 | 调用方式 |
|----------|----------|--------------------------|----------|
| 全局指令 | `CLAUDE.md` | ______ | ______ |
| 自定义命令 | `Claude/commands/xxx.md` | ______ | ______ |

请解释：为什么 Claude Code 设计为"自定义命令需要手动调用"而非"自动加载"？

**参考答案要点**：
- 自定义命令可能包含敏感操作（文件修改、删除、shell命令）
- 自动加载可能导致新会话意外执行危险指令
- 手动调用提供显式确认，增加安全性

### 练习 3：并行冲突原理

**问题**：如果不使用 git worktree，而是打开两个终端窗口同时对同一仓库执行 Claude Code 操作，可能发生什么？请描述具体场景。

**参考答案要点**：
- 两个 Claude Code 实例可能同时修改 `piproject.toml` 等共享文件
- 后保存的版本会覆盖先保存的版本，导致数据丢失
- 原因：两个终端共享同一个 `.git` 目录和工作目录，无隔离机制

## 最小实现题

### 练习 4：带参数自定义命令

**需求**：创建一个自定义命令 `Claude/commands/refactor.md`，用于重构指定模块。

**完成以下步骤**：

1. 在 `Claude/commands/` 目录下创建 `refactor.md`
2. 文件内容需包含：
   - 命令描述（`description` 字段）
   - 使用 `$arguments` 接收传入的模块名参数
   - 调用 Claude Code 对该模块进行重构的指令

**参考结构**：

```markdown
---
description: 重构指定模块的代码结构和命名
---
请对 `$arguments` 模块执行以下重构操作：
1. 分析当前模块的依赖关系
2. 提取重复代码为共享函数
3. 优化命名使其更清晰
4. 确保重构后所有测试通过
```

**验证方法**：在 Claude Code 中输入 `/refactor auth` 应触发对该模块的重构流程。

### 练习 5：高频命令预授权

**场景**：你需要在 Claude Code 中频繁执行 `npm test` 和 `git status`，但每次都弹出授权提示。

**任务**：在 `.claude/settings/local.json` 中配置预授权，使得这两个命令在当前工作目录下无需确认。

**参考答案**：

```json
{
  "permissions": {
    "allow": [
      "npm test",
      "git status"
    ]
  }
}
```

## 最小替代练习

### 练习 6：工作树隔离机制替代方案

**背景**：git worktree 需要 Git 2.5+ 版本，你的旧系统无法升级。

**任务**：设计一个轻量替代方案，实现多会话并行开发时的目录隔离（不要求版本控制集成）。

**提示**：

- 核心需求是什么？（目录隔离）
- 什么命令可以创建目录副本？
- 手动管理时需要同步哪些文件？

**参考答案要点**：

```bash
# 替代方案：使用 rsync 或 cp 创建目录副本
rsync -av --exclude='.git' /path/to/project/ /path/to/project-copy/
```

**局限性**：

- 无法自动合并代码变更
- 需要手动同步分支间的差异
- 不适合长期并行开发

**适用场景**：临时并行测试、快速对比、演示环境隔离
