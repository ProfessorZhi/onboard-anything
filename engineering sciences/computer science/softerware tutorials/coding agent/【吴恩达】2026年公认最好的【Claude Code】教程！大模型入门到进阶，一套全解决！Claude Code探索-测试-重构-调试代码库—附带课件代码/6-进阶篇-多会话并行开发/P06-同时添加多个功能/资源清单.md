## 命令行工具

**git worktree add**
- 类型：Git命令
- 在本P中的作用：创建独立的代码库工作副本，用于多会话并行开发时避免文件覆盖冲突
- 来源：字幕直接提到
- 后续可替代练习方向：在已有Git仓库中实际创建工作树，并验证各工作树之间的独立性

**git worktree list**
- 类型：Git命令
- 在本P中的作用：查看当前仓库中所有工作树的列表，确认工作树创建状态
- 来源：基于上下文推断
- 后续可替代练习方向：结合add命令使用，验证工作树创建和删除后的列表变化

**git worktree remove**
- 类型：Git命令
- 在本P中的作用：删除不再需要的工作树，完成工作树生命周期管理的最后一步
- 来源：基于上下文推断
- 后续可替代练习方向：删除已完成合并的工作树，保持仓库整洁

**git merge**
- 类型：Git命令
- 在本P中的作用：将各工作树中独立开发的更改合并回主分支
- 来源：基于上下文推断
- 后续可替代练习方向：模拟多人协作场景，在有冲突和无冲突两种情况下练习合并流程

## 配置与文件

**Claude文件夹（Claude/）**
- 类型：配置目录
- 在本P中的作用：Claude Code的根配置目录，用于存放自定义命令和本地设置
- 来源：字幕直接提到
- 后续可替代练习方向：查看自己的Claude Code配置目录结构，了解其组织方式

**Claude/commands/ 目录**
- 类型：配置目录
- 在本P中的作用：存放自定义命令的专用目录，在该目录下创建Markdown文件即可定义新命令
- 来源：字幕直接提到
- 后续可替代练习方向：在该目录下创建自己的第一个自定义命令

**implement feature.md**
- 类型：自定义命令文件
- 在本P中的作用：示例自定义命令，用于快速实现新功能；文件名为命令名称，Markdown内容为命令描述
- 来源：字幕直接提到
- 后续可替代练习方向：仿照该文件格式创建其他自定义命令，如代码审查、部署发布等

**$arguments**
- 类型：变量
- 在本P中的作用：自定义命令中的参数引用变量，用于传递用户输入的参数给Claude Code执行
- 来源：字幕直接提到
- 后续可替代练习方向：在自定义命令中使用该变量实现带参数的命令，如指定文件名、功能描述等

**front_end_changes.md**
- 类型：输出文件
- 在本P中的作用：示例输出文件，用于记录前端修改的具体内容；演示如何将Claude Code的输出定向到特定文件
- 来源：字幕直接提到
- 后续可替代练习方向：创建类似的任务记录文件，追踪Claude Code对特定模块的修改

**CLAUDE.md**
- 类型：全局配置文件
- 在本P中的作用：Claude Code的全局配置文件，与自定义命令形成对比——它会自动加载到上下文，适合全局设置
- 来源：字幕直接提到
- 后续可替代练习方向：编辑CLAUDE.md文件，添加项目特定的全局上下文和约束

**.claude/settings/local.json**
- 类型：本地配置文件
- 在本P中的作用：存储本地权限配置，用于预授权高频使用的命令（如Playwright、Git操作），避免每次重复确认
- 来源：字幕直接提到
- 后续可替代练习方向：查看并编辑该文件，学习配置允许的命令以提升工作效率

**piproject.toml**
- 类型：Python项目配置文件
- 在本P中的作用：本P中演示冲突发生的文件——三个工作树同时修改了该文件，导致合并时产生冲突
- 来源：字幕直接提到
- 后续可替代练习方向：在使用工作树时注意区分各工作树的职责范围，避免多个工作树修改同一配置文件
