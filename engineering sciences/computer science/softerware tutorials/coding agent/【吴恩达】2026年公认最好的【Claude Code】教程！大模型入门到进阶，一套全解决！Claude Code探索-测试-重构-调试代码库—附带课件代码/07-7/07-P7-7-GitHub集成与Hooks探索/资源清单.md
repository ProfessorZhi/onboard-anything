资源清单

## 基础设施

**Claude Code**
- 类型：AI编程助手（主体工具）
- 在本P中的作用：贯穿全P的核心工具，本P展示了将其能力从终端扩展到GitHub平台的完整路径
- 来源：字幕直接提到
- 后续可替代练习方向：使用其他支持GitHub集成的AI编码工具（如GitHub Copilot）对比体验差异

**Claude Code GitHub App**
- 类型：GitHub应用（集成入口）
- 在本P中的作用：Claude Code接入GitHub的身份验证载体，安装后可在PR和Issue中触发Claude响应反馈
- 来源：字幕直接提到
- 后续可替代练习方向：尝试为其他AI工具配置类似的GitHub集成应用

**Claude Code SDK**
- 类型：软件开发工具包
- 在本P中的作用：Claude Code GitHub集成的底层技术基础，允许在终端界面外使用核心功能
- 来源：字幕直接提到
- 后续可替代练习方向：查阅官方文档了解SDK的完整API能力，探索自定义集成场景

**GitHub Actions**
- 类型：CI/CD自动化平台
- 在本P中的作用：驱动自动代码审查的执行引擎，Claude在此环境中运行并分析PR变更
- 来源：字幕直接提到
- 后续可替代练习方向：自定义GitHub Actions工作流，添加更多自动化检查步骤

## 配置文件

**code_review.yml**
- 类型：代码审查配置文件
- 在本P中的作用：定义Claude Code审查PR的行为规则，包括按作者筛选、运行环境、审查提示内容等
- 来源：字幕直接提到
- 后续可替代练习方向：自定义审查配置，调整审查深度、增加特定检查规则、添加团队定制化提示

**GitHub Actions YAML文件**
- 类型：CI/CD流程定义文件
- 在本P中的作用：Claude创建PR时自动生成，定义GitHub Actions的触发条件和执行步骤
- 来源：字幕直接提到
- 后续可替代练习方向：手动编写或修改YAML文件，实现自定义的CI/CD流程

**本地settings.json**
- 类型：项目级配置文件
- 在本P中的作用：Claude Code的Hooks配置位置，与权限配置同级，用于定义工具执行前后的自定义逻辑
- 来源：字幕直接提到
- 后续可替代练习方向：在settings.json中尝试配置不同类型的Hook事件，实现多样化的自动化场景

## 系统命令

**git push origin main**
- 类型：Git命令
- 在本P中的作用：将本地代码变更推送到GitHub主分支，完成清理上节遗留问题后的同步操作
- 来源：字幕直接提到
- 后续可替代练习方向：练习使用git命令管理多分支场景下的代码推送策略

**say命令（macOS）**
- 类型：系统命令（TTS语音合成）
- 在本P中的作用：作为post-tool-use hook的执行示例，当读取或grep操作完成时触发朗读反馈
- 来源：字幕直接提到
- 后续可替代练习方向：Linux用户可使用espeak或festival替代，Windows用户可探索PowerShell语音合成方案

**shell命令**
- 类型：系统命令类别
- 在本P中的作用：Hooks中可以执行的命令类型，可用于运行测试、linters、发送通知等自动化任务
- 来源：字幕直接提到
- 后续可替代练习方向：编写更复杂的shell脚本作为Hook执行体，如自动格式化代码、生成变更摘要等
