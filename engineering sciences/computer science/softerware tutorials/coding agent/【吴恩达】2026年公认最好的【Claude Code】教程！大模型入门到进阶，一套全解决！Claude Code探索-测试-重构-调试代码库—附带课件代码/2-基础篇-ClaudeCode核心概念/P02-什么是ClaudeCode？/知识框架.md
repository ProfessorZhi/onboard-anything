# P02 知识框架 - 什么是Claude Code？

- 所属章节：`2-1-什么是Claude Code？`

## 本P位置

- 课程：第一章·基础认知模块（第1课）
- 前置：课程入口，无先修分P
- 后置：P3（更大代码库场景）
- 定位：建立对Claude Code的整体认知框架，锚定“智能代理”概念

## 核心问题

- Claude Code是什么（what）
- 为什么需要它（why）
- 它能做什么（what can it do）

## 最小知识骨架

1. **智能代理架构**：模型+工具链+运行环境，三要素缺一不可
2. **工具链设计**：不是直接给模型任务，而是提供工具集让模型主动行动
3. **两种操作入口**：CLI vs Sonar，按任务复杂度和订阅计划选择
4. **内置工具集**：文件读写、编辑、搜索、子代理、bash命令
5. **MCP可扩展性**：Model Context Protocol，连接外部服务的标准协议
6. **智能搜索机制**：不索引代码库，数据留在本地
7. **记忆系统**：CLAUDE.md配置文件 + 本地对话存储

## 关系与依赖

```
智能代理架构（顶层）
    ↓
工具链设计（实现机制）
    ├── 内置工具集（工具链内容）
    ├── MCP可扩展性（工具链延伸）
    └── 智能搜索（搜索工具实现）
    ↓
记忆系统（独立支撑）
    ├── CLAUDE.md（持久配置）
    └── 本地对话存储（会话上下文）
```

- 两种操作入口（CLI/Sonar）是进入智能代理架构的门户
- 演示→展示开箱即用能力，验证工具链有效性
- 核心教学取向：从“发现、解释、设计”开始，而非“写代码”

## 输入 / 输出 / 前置 / 支撑

- **输入**：极简自然语言提示（例：“制作一个很酷的可视化效果”）
- **输出**：可运行代码、执行结果、可视化效果
- **前置**：无特定先修知识，本P即为入口
- **支撑**：
  - VSCode（演示环境）
  - CLI / Sonar（操作入口）
  - 本地机器（对话存储、数据安全边界）

## 资源与外部对象

- **CLI**：命令行界面操作入口
- **Sonar**：另一操作入口
- **MCP服务器**：扩展功能的数据源
- **CLAUDE.md**：用户自定义配置文件
- **GitHub**：可集成的外部平台
- **bash/shell**：内置命令执行能力
- **子代理**：可创建的任务执行单元
- **demo文件夹**：演示用空项目目录

## 易错点

- 把Claude Code当成纯代码生成工具，忽视“发现、解释、设计”能力
- 认为CLI和Sonar可以随意切换（实际取决于任务复杂度和订阅计划）
- 误以为代码库会被索引或上传（实际使用智能搜索，数据留本地）
- 忽略CLAUDE.md的配置文件作用
- 低估MCP的扩展价值
- 把演示速度当作实际工作流标准
