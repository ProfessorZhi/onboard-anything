# P02 资源清单 - 什么是Claude Code？

- 所属章节：`2-1-什么是Claude Code？`
- 分P标题：`P02 什么是Claude Code？`

资源清单

Claude Code

- 类型：AI编程助手 / 智能代理开发工具
- 在本P中的作用：核心介绍对象。本P系统讲解其定位（不只是代码生成工具，而是智能代理）、架构（模型+工具链+运行环境）、两种操作入口（CLI/Sonar）、核心能力模块（工具、记忆）
- 来源：字幕直接提到
- 后续可替代练习方向：安装并启动Claude Code，熟悉基本交互流程

CLI（命令行界面）

- 类型：操作入口
- 在本P中的作用：Claude Code的两种操作入口之一。适合通过终端直接与Claude Code交互的场景
- 来源：字幕直接提到
- 后续可替代练习方向：在终端中安装并启动Claude Code，体验命令行交互模式

Sonar

- 类型：操作入口
- 在本P中的作用：Claude Code的两种操作入口之一。适用于特定任务复杂性和订阅计划场景
- 来源：字幕直接提到
- 后续可替代练习方向：了解Sonar与CLI的适用场景差异，尝试不同入口的适用条件

MCP（Model Context Protocol，模型上下文协议）

- 类型：协议 / 扩展标准
- 在本P中的作用：开源、模型无关的协议，用于扩展Claude Code能力。允许MCP服务器为Claude Code添加各种任务的功能，是Claude Code可扩展性的核心机制
- 来源：字幕直接提到
- 后续可替代练习方向：研究现有MCP服务器生态，了解如何通过MCP连接外部服务和数据源

VSCode（Visual Studio Code）

- 类型：代码编辑器 / 演示环境
- 在本P中的作用：演示环节使用的编辑器环境，展示Claude Code与VSCode的集成效果，让学习者实时看到代码更改
- 来源：字幕直接提到
- 后续可替代练习方向：在VSCode中配置Claude Code扩展，体验集成开发环境中的AI辅助编程

GitHub

- 类型：代码托管平台 / 可集成服务
- 在本P中的作用：Claude Code可集成的外部平台之一，属于应用场景介绍
- 来源：字幕直接提到
- 后续可替代练习方向：探索Claude Code与GitHub的集成方式，如代码拉取、提交等操作

CLAUDE.md

- 类型：配置文件 / 记忆系统组件
- 在本P中的作用：记忆功能的实现载体。用户可在其中定义通用配置或样式指南，在启动Claude Code会话时自动加载到上下文中，实现持久化配置
- 来源：字幕直接提到
- 后续可替代练习方向：创建个人CLAUDE.md文件，定义项目规范和偏好设置

智能搜索（Agentic Search）

- 类型：搜索机制 / 安全特性
- 在本P中的作用：Claude Code的搜索实现方式。与传统索引不同，采用智能搜索策略，代码不必发送到服务器，数据留在本地生态系统，确保安全边界
- 来源：字幕直接提到
- 后续可替代练习方向：对比传统代码索引方案，理解智能搜索的设计权衡

子代理（subagent）

- 类型：功能模块 / 任务执行单元
- 在本P中的作用：Claude Code可创建或运行的子任务执行单元，属于内置工具集中的高级能力
- 来源：字幕直接提到
- 后续可替代练习方向：尝试在复杂任务中拆分子代理，体验并行或分层任务执行

bash/shell

- 类型：命令行工具 / 内置命令执行能力
- 在本P中的作用：Claude Code内置的bash命令执行工具，因为Claude Code运行在命令行环境中
- 来源：字幕直接提到
- 后续可替代练习方向：结合Claude Code执行系统命令，体验AI与命令行工具的协同

MCP服务器

- 类型：扩展服务 / 外部功能源
- 在本P中的作用：为Claude Code提供扩展功能的服务器，是MCP协议的具体实现形式
- 来源：字幕直接提到
- 后续可替代练习方向：部署一个简单的MCP服务器，测试自定义工具接入Claude Code

demo文件夹

- 类型：演示项目 / 临时目录
- 在本P中的作用：演示环节使用的空文件夹，用于展示Claude Code的开箱即用能力，从零创建可视化效果
- 来源：基于上下文推断
- 后续可替代练习方向：从空项目开始，用极简提示词让Claude Code创建完整功能
