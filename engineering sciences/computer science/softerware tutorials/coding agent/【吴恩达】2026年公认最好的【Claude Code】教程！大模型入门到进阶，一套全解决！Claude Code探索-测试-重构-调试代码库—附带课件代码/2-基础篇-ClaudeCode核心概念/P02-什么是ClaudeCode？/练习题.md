## 结构识别题

### 题目 1：架构要素识别

以下三个描述，分别对应Claude Code智能代理架构的哪个要素？

A. 大模型本身，负责理解指令、规划行动、生成响应  
B. Claude Code提供的文件读写、搜索、执行命令等能力  
C. Claude Code运行的实际计算环境（本地机器）

请按顺序写出对应关系，并说明三者如何协作完成一个任务。

## 机制理解题

### 题目 2：工具链设计原理

课程提到Claude Code的设计理念是"不是直接给模型任务，而是提供工具集让模型主动行动"。

请思考并回答：

- 这种设计与传统的"模型直接生成代码"有何本质区别？
- 为什么说这种设计对复杂任务更有优势？

---

### 题目 3：数据安全与智能搜索

Claude Code使用"智能搜索"而非索引代码库，且数据留在本地。

请解释：

- 这种设计在数据安全层面解决了什么问题？
- 智能搜索可能的技术实现思路是什么？

## 最小实现题

### 题目 4：提示词驱动生成

假设你有一个空的`demo`文件夹（参考课程演示），你需要用一句话提示词让Claude Code完成以下任务：

请设计一个最小提示词，使其能驱动Claude Code：

1. 在当前目录创建一个简单的Python文件（如`hello.py`）
2. 该文件包含一个打印"Hello, Claude"的功能

（不需要实际执行，只需写出你会在Claude Code中输入的提示词，并说明你为什么这样设计。）

---

### 题目 5：配置记忆系统

Claude Code通过`CLAUDE.md`配置文件实现持久化配置。

请设计一个最小的`CLAUDE.md`文件内容，包含：

- 项目的基本描述
- 你希望Claude Code每次对话时默认使用的编程语言偏好
- 一个你希望它在执行代码前始终遵守的规则

## 最小替代练习

### 题目 6：同类工具对比

如果在你的学习环境中无法使用Claude Code，可以尝试使用**GitHub Copilot Chat**（VSCode插件）作为替代练习。

请设计一个对比练习：

1. 在同一个简单项目场景下，分别用Claude Code和GitHub Copilot Chat完成"读取项目文件并列出目录结构"这个任务
2. 记录两者在以下维度的差异：

- 交互方式
- 响应速度（主观感受）
- 对话上下文保持能力

（注意：这是练习替代，不是要求两者完全等价，重点是体验AI辅助编程工具的通用设计模式。）
