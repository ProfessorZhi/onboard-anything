## 本P位置
课程第9分P/第9章末节。从Claude Code基础操作（安装、启动、编辑、Git、测试）转向**多工具集成与实际应用**的转折点。首次引入MCP协议串联外部服务，形成完整工作流示范。

## 核心问题
如何让AI直接"看"Figma设计稿并自动生成可运行的Web应用，同时用真实数据替换模拟数据？

## 最小知识骨架
1. **MCP协议** - Claude Code与外部工具通信的中间层标准
2. **Figma MCP Server** - 将Figma设计稿转为可读代码/图像
3. **Playwright MCP Server** - 自动化浏览器测试与截图验证
4. **Next.js App Router** - 目标应用框架，代码组织在app文件夹
5. **FRED API** - 联邦储备经济数据库，获取真实数据
6. **Figma开发者模式** - MCP正常工作的必要前置设置
7. **recharts** - 指定使用的React图表库

## 关系与依赖
```
Figma原型 → Figma MCP Server → Claude Code → Next.js应用
                                                 ↓
                                          Playwright验证
                                                 ↓
                                          FRED API真实数据
```
- MCP Server是双向通道：读取设计数据 + 返回执行结果
- Figma开发者模式未开 → MCP Server无法工作
- 原型质量决定生成效果下限

## 输入 / 输出 / 前置 / 支撑
**输入**：Figma设计原型（含开发者模式启用的图层标识符）

**输出**：可运行的Next.js Web应用（含真实经济数据图表）

**前置**：
- Claude Code已安装
- Figma账户 + 开发者模式启用
- FRED API密钥（可选，真实数据必需）

**支撑**：
- npm/包管理器
- 本地主机3030端口
- Opus/Sonnet模型（复杂任务建议Opus）

## 资源与外部对象
- Figma：设计工具
- Figma MCP Server：中间件，5个工具（获取代码/图像/设计规则）
- Playwright MCP Server：中间件，浏览器自动化
- Next.js：目标框架
- recharts：图表库
- FRED API：真实经济数据源
- 联邦储备经济数据：失业率/CPI/国债收益率等

## 易错点
- MCP配置不跨项目：每个独立项目需单独配置
- Figma开发者模式未开启：MCP无法工作
- 原型过于简陋：AI生成效果受限
- 缺少FRED API密钥：无法获取真实数据
- Sonnet处理复杂多步骤任务可能不如Opus
- 开发服务器已在运行时不重复启动
