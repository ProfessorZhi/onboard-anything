## 工具 / 平台

### Figma

- 类型：设计工具 / 在线平台
- 在本P中的作用：承载 Web 应用仪表盘原型，作为 AI 代码生成的设计输入源；MCP 服务器通过其开发者模式读取图层结构、代码和图像
- 来源：字幕直接提到
- 后续可替代练习方向：可尝试用自己已有的 Figma 文件（如个人作品集、数据看板）复现本P流程；也可探索 Penpot（开源设计工具）是否具备类似 MCP 接口能力

---

### Figma MCP 服务器（Figma Dev Mode MCP Server）

- 类型：MCP 服务器 / 中间件
- 在本P中的作用：作为 Claude Code 与 Figma 之间的通信桥梁；提供至少五个工具，包括获取原型底层代码、获取图像、获取设计规则，使 Claude 能直接读取 Figma 设计稿数据
- 来源：字幕直接提到
- 后续可替代练习方向：研究 Figma 官方 MCP Server 文档，了解五个工具的具体参数；尝试在自己的 Claude Code 项目中独立配置此服务器并验证连接

---

### Playwright MCP 服务器

- 类型：MCP 服务器 / 中间件
- 在本P中的作用：作为 Claude Code 与 Playwright 之间的通信桥梁；在本P中用于导航到本地开发服务器（localhost:3030）、截图并将结果返回给 Claude，实现"生成 → 视觉验证"闭环
- 来源：字幕直接提到
- 后续可替代练习方向：单独练习 Playwright 截图功能，了解其与 Puppeteer 的差异；尝试在 MCP 配置中仅保留 Playwright 服务器并让 Claude 完成独立的 UI 验证任务

---

### Claude Code

- 类型：AI 编程助手 / 命令行工具
- 在本P中的作用：整个工作流的核心执行引擎；接收用户指令后，依次调用 Figma MCP 工具（读取设计）、生成 Next.js 代码、安装依赖、接入 FRED API、调用 Playwright MCP 工具（截图验证）
- 来源：字幕直接提到
- 后续可替代练习方向：尝试在不同项目中重新配置 MCP 服务器，验证"MCP 配置不跨项目"的规则；对比有无 MCP 时 Claude Code 的输出差异

---

## 框架 / 库

### Next.js

- 类型：前端框架（React 生态）
- 在本P中的作用：目标应用框架，用于承载从 Figma 原型生成的 Web 应用；采用 App Router 架构，核心代码组织在 app 文件夹；开发服务器运行于 localhost:3030
- 来源：字幕直接提到
- 后续可替代练习方向：若不熟悉 Next.js，可先用 Vite + React 重现相同流程，对比两种框架下 Claude 生成代码的结构差异；或直接跟随 Next.js 官方文档补充 App Router 基础知识

---

### recharts

- 类型：React 图表库
- 在本P中的作用：老师在提示词中明确要求 Claude 使用此库创建经济数据图表（条形图等），是指令的一部分而非 Claude 自主选择
- 来源：字幕直接提到
- 后续可替代练习方向：将 recharts 替换为 Chart.js、Tremor 或 Victory，观察 Claude 在接到不同库指定时生成代码的差异；练习如何在提示词中精确指定技术栈

---

## API / 数据源

### FRED API（Federal Reserve Economic Data API）

- 类型：REST API / 公开经济数据接口
- 在本P中的作用：提供真实经济数据（CPI、失业率、国债收益率、劳工工资等），用于替换应用中的模拟假数据；Claude 在执行过程中发现需要 API 密钥，老师前往账户获取后继续流程；Claude 随后编写了数据获取服务和代理请求逻辑
- 来源：字幕直接提到"联邦储备经济数据"和"API密钥"；FRED 具体名称基于上下文推断
- 后续可替代练习方向：注册 FRED 账户获取免费 API 密钥，自行调用 /series/observations 端点获取数据并渲染；或替换为世界银行开放数据 API、国家统计局数据接口，练习数据接入流程

## 命令 / 操作

### npx create-next-app（Next.js 初始化命令）

- 类型：命令行指令
- 在本P中的作用：在当前文件夹创建 Next.js 应用，使用最新版本，遵循大部分默认设置；是整个工作流的第一步操作
- 来源：字幕直接提到执行了"几个命令行指令"创建 Next.js 应用，使用最新版本并遵循默认设置；具体命令字符串字幕未完整给出（基于 Next.js 文档推断为 npx create-next-app）
- 后续可替代练习方向：独立执行此命令，熟悉 App Router / Pages Router 的选择提示；对比 TypeScript 与 JavaScript 模板的目录结构差异

---

### mcp（MCP 连接验证命令）

- 类型：Claude Code 内置命令
- 在本P中的作用：用于检查当前项目中已配置的 MCP 服务器是否正常连接，以及列出各服务器提供的工具数量；老师用此命令确认 Figma MCP Server 和 Playwright MCP Server 均已连接
- 来源：字幕中提到使用命令检查连接状态（原文疑似"mscp"为口误或笔误，基于上下文推断为 Claude Code 的 mcp 命令）
- 后续可替代练习方向：在配置好 MCP 服务器后，主动用此命令验证连接，养成"配置后即验证"的习惯；记录工具列表并理解每个工具的用途

---

## 配置 / 模式

### Figma 开发者模式（Developer Mode）

- 类型：Figma 内部设置选项
- 在本P中的作用：Figma MCP 服务器正常工作的必要前置条件；启用后可获取图层标识符，用于 MCP 服务器配置；老师明确提示此步骤不可遗漏
- 来源：字幕直接提到
- 后续可替代练习方向：在 Figma 免费账户中确认开发者模式的启用范围和限制；对照老师说明的"无需订阅 Figma 开发者模式付费计划"，确认自己账户的实际可用功能边界

---

### MCP 项目级配置（非全局配置）

- 类型：配置机制 / 架构约定
- 在本P中的作用：老师明确指出 MCP 服务器配置在独立项目中，不会自动应用到所有 Claude Code 项目，需要为每个项目单独添加；这是本P的易错知识点
- 来源：字幕直接提到
- 后续可替代练习方向：创建两个不同的 Claude Code 项目，只在其中一个中配置 MCP 服务器，验证隔离效果；查阅 Claude Code 文档中关于全局配置与项目级配置的说明

## 模型

### Claude Opus 模型

- 类型：大语言模型（Anthropic Claude 系列）
- 在本P中的作用：老师建议在执行本P的复杂多步骤任务时优先使用 Opus；若有 Pro Plan 可直接选用，若无则 Sonnet 仍可胜任但效果稍弱
- 来源：字幕直接提到
- 后续可替代练习方向：分别用 Opus 和 Sonnet 执行相同的"Figma → 代码"任务，记录输出质量和步骤完整性的差异，形成自己的模型选择判断依据

### Claude Sonnet 模型

- 类型：大语言模型（Anthropic Claude 系列）
- 在本P中的作用：作为 Opus 的备选方案；老师表示 Sonnet 能完成大多数任务，适合无 Pro Plan 的学习者
- 来源：字幕直接提到
- 后续可替代练习方向：同上；重点观察 Sonnet 在哪类子步骤（如多工具链式调用、API 文档理解）上开始出现质量下降
