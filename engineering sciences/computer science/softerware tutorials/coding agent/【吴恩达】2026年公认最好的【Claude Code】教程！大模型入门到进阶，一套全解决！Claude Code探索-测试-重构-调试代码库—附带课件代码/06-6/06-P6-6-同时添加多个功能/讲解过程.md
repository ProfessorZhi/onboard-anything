# P6 6-同时添加多个功能

## 本P定位

本P是课程第六章的第6个子章节，承接P5关于Claude Code权限管理的内容，同时为后续P7（GitHub集成、拉取请求审查）做铺垫。本P的核心主题是**使用Git工作树实现多Claude Code实例并行工作**，解决多个功能同时开发时的文件覆盖问题。

## 本P主线

**问题引出** → **自定义命令铺垫** → **Git工作树解决方案讲解** → **三工作树并行演示** → **合并与冲突处理** → **效果展示与总结**

## 本P结论

1. **自定义命令**：通过在Claude/.claude/commands/文件夹中创建Markdown文件，可以定义可复用的命令，支持$ARGUMENTS参数传递，适合跨对话使用的标准化操作。

2. **Git工作树并行开发**：使用git worktree命令创建代码库副本，多个Claude Code实例可以独立操作同一项目的不同部分，避免文件覆盖问题。

3. **工作树工作流**：创建工作树 → 独立开发 → 提交更改 → 合并回主分支 → 删除工作树。

4. **Claude Code辅助合并**：可以利用Claude Code自动分析并解决合并冲突，配合测试确保合并后代码正常。

5. **应用场景**：适合同时开发多个功能、进行代码重构、运行测试、添加质量工具等需要并行处理的场景。

## 易错点

1. **混淆commands和Claude.md的作用域**：自定义命令不会自动添加到上下文，Claude.md才是全局配置。

2. **忽略工作树隔离机制**：以为多终端直接操作代码库可以避免冲突，忽视Git工作树的隔离保护。

3. **合并前未确保提交规范**：跨工作树开发时，每个工作树都应有清晰的提交信息，便于后续合并和代码审查。

4. **忘记预授权常用命令**：每次使用都要确认会降低效率，应在.settings.json中提前配置。
