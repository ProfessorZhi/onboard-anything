# 资源清单

---

## 1. Git 工作树（git worktree）

- **类型**：Git 内置命令
- **在本P中的作用**：创建代码库的隔离副本，解决多个 Claude Code 实例并行工作时同一文件被覆盖的问题；通过 `git worktree add/list/remove` 命令实现工作树的生命周期管理
- **来源**：字幕直接提到
- **后续可替代练习方向**：使用 git worktree 管理 feature 分支并行开发，在不同工作树中独立实现多个功能后再合并回主分支

---

## 2. implement feature.md

- **类型**：自定义命令文件
- **在本P中的作用**：存储在 `.claude/commands/` 目录下的可复用命令模板，通过 Markdown 文件定义，支持 `$ARGUMENTS` 变量传递参数；老师使用此命令在三个工作树中分别添加不同功能
- **来源**：字幕直接提到
- **后续可替代练习方向**：创建其他类型的自定义命令（如重构命令、审查命令、测试命令），探索自定义命令的参数组合使用方式

---

## 3. front end changes.md

- **类型**：变更记录文档
- **在本P中的作用**：在 UI 功能工作树中创建的变更日志文档，用于记录前端功能的实现进度和变更内容（如深色/浅色主题切换）
- **来源**：字幕直接提到
- **后续可替代练习方向**：为每个工作树建立系统化的变更日志机制，探索用文档管理并行开发进度的最佳实践

---

## 4. .settings.json

- **类型**：Claude Code 配置文件
- **在本P中的作用**：位于 `.claude/` 目录下，用于预授权常用命令（如 playwright），避免每次执行时的确认提示，提升并行开发效率
- **来源**：字幕直接提到
- **后续可替代练习方向**：根据实际项目需求扩展预授权命令列表，探索团队配置共享机制

---

## 5. Claude.md

- **类型**：Claude Code 配置文件
- **在本P中的作用**：全局配置文件，使所有 Claude Code 实例自动加载相同配置；与 commands 文件夹形成互补，Claude.md 全局生效而自定义命令需手动触发
- **来源**：字幕直接提到
- **后续可替代练习方向**：对比学习 Claude.md 与 commands 文件夹的不同适用场景，探索团队级全局配置管理方案

---

## 6. Playwright

- **类型**：测试框架
- **在本P中的作用**：端到端测试工具示例，老师在 .settings.json 中授予其执行权限；用于 UI 功能验证（如主题切换功能测试）
- **来源**：字幕直接提到
- **后续可替代练习方向**：在实际项目中集成 Playwright 编写端到端测试用例，或替换为其他测试框架进行对比练习

---

## 7. FastAPI

- **类型**：Python Web 框架
- **在本P中的作用**：测试工作树中的 API 端点测试对象，用于演示如何为 Web 应用编写测试用例
- **来源**：字幕直接提到
- **后续可替代练习方向**：为现有 FastAPI 项目配置测试环境，探索 API 端点测试的最佳实践

---

## 8. pyproject.toml

- **类型**：Python 项目配置文件
- **在本P中的作用**：Python 项目依赖管理文件，测试功能工作树在此文件中添加代码质量工具依赖
- **来源**：字幕直接提到
- **后续可替代练习方向**：通过 pyproject.toml 管理项目依赖，探索 Python 项目的规范化依赖管理方式

---

## 9. .claude/commands/ 目录

- **类型**：目录结构
- **在本P中的作用**：Claude Code 自定义命令的存放目录，目录下的 Markdown 文件会被识别为可复用命令；支持跨对话使用
- **来源**：基于上下文推断（老师提到在 Claude 文件夹内新建 commands 文件夹）
- **后续可替代练习方向**：规范化自定义命令的文件结构，建立团队共享命令库的最佳实践

---

## 10. .trees/ 目录

- **类型**：目录结构
- **在本P中的作用**：Git 工作树的集中存储目录，老师在此目录下创建 ui功能、测试功能、质量功能 三个工作树
- **来源**：字幕直接提到
- **后续可替代练习方向**：设计工作树目录的命名规范和存储策略，优化多工作树项目的文件组织
