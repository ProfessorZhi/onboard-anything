---

## 最小知识骨架

1. **测试驱动调试循环**：编写测试→运行测试定位Bug→修复→验证
2. **Bug根因发现**：max_results=0导致向量搜索返回0个结果
3. **pytest + mock测试框架**：单元测试验证程序行为，mock模拟外部依赖
4. **扩展思考模式**：Claude Code处理复杂任务时分配更多令牌进行深度分析
5. **计划模式**：修改前先展示计划，用户确认后再执行
6. **子代理并行探索**：task工具调度多个思考者同时探索方案
7. **单次调用→多轮迭代**：AI生成器架构重构，支持复杂查询的多轮工具调用

## 易错点

1. **无序调试**：直接复制错误到AI而不先编写测试，导致无意义循环
2. **跳过验证**：修复后不运行测试验证，无法防止回归
3. **忽视测试价值**：只把测试当“这次验证”，忽视“基础设施”价值
4. **单次调用局限**：用单次工具调用处理复杂查询（如跨课程比较）
5. **破坏性重构**：修改生成器时不验证向后兼容，破坏现有API
6. **盲目自动接受**：复杂重构时跳过计划模式，执行破坏性修改
7. **单一方案依赖**：不探索多种重构方案，由AI单方面决定
