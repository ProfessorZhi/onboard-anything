# P46 知识框架 - 【进阶篇】闭包
## 本P位置
- 课程：Python入门零基础教程
- 阶段：进阶篇
- 顺序：第46P，进阶篇第1小节第1个知识点
- 前置：Python基础（函数定义、作用域）
- 后置：P47装饰器（直接依赖）、P49迭代器与生成器

作用：闭包是装饰器的实现原理，学透闭包才能理解装饰器。

## 核心问题
- 闭包是什么：带记忆的嵌套函数
- 闭包解决什么问题：让函数记住状态，避免重复传参
- 内层函数为何能记住外层局部变量：闭包机制使然

## 知识骨架
1. **嵌套函数**：外层函数内部定义内层函数，内层函数仅在外层函数内部可见
2. **闭包定义**：外层函数执行完毕后，内层函数仍能访问外层函数的局部变量
3. **闭包三条件**：
   - 函数嵌套
   - 内层函数引用外层局部变量（非全局、非内层自定）
   - 外层函数返回内层函数引用（不加括号）
4. **基础版闭包**：外层函数定义固定局部变量
5. **升级版闭包**：外层函数接收参数，实现动态记忆
6. **闭包三用途**：保存状态、数据封装、为装饰器奠基

## 关系与依赖
- 依赖链：函数基础 → 嵌套函数 → 闭包 → 装饰器
- 闭包是装饰器的底层实现
- 嵌套函数是闭包的空间结构基础

## 输入 / 输出 / 前置 / 支撑
- **前置**：嵌套函数、局部变量、作用域概念
- **输入**：外层函数的参数或固定值
- **输出**：内层函数的引用（函数对象）
- **支撑**：IDE（PyCharm）用于演示和调试闭包函数

## 易错点
- 返回内层函数时误加括号：`return inner()`会立即执行，应`return inner`
- 用全局变量替代外层局部变量：闭包记忆的是局部变量
- 误以为内层函数可在外层外部直接调用：内层函数仅在外层函数内部可用

## 外部资源与对象
- **IDE**（PyCharm或类似）：本P全程演示环境，用于编写和运行闭包代码
- **闭包函数对象**：IDE显示形式为`Function outer.<locals>.inner at 0x...`，标识返回的是本地函数引用
