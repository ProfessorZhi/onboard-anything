# P46 讲解过程 - 【进阶篇】闭包
## 本P定位

本P是Python进阶篇的第一个小节——闭包（Closure）与装饰器——的开篇。闭包被定位为装饰器的基础，老师先讲解闭包本身的概念、构成条件和代码演示，为下一P装饰器的学习做铺垫。

## 讲解过程

### 一、进阶内容概述（0:00–1:02）

老师先从宏观上介绍了进阶篇的整体框架：本进阶篇分为三个小节，分别是"闭包和装饰器"、"单例模式和魔术方法"、"迭代器和生成器"。老师强调这些是"更好用、更实用、更能提升代码效率的高级特性"，能让代码"从能用升级到好用优雅"。

具体到本小节，老师解释了两个核心概念：**闭包是用来给函数装记忆的，装饰器是用来给函数加插件的**。

### 二、引入闭包的前置知识：嵌套函数（1:02–7:05）

老师指出，闭包的学习需要先掌握一个前提知识——**嵌套函数**。嵌套函数的定义是：在一个函数里面再定义另一个函数，内层函数只能在外层函数内部使用。

老师打开IDE现场演示：

```python
def outer():
    print("我是外层函数")
    def inner():
        print("我是内层函数")
    inner()
outer()
```

演示要点包括：

1. **内层函数的灰色提示**：老师指出IDE中inner函数显示灰色，表示没有被调用，这是正常状态，因为inner定义在outer函数内部。
2. **直接调用内层函数会报错**：老师尝试在outer函数外部直接写`inner()`，运行后得到`NameError: name 'inner' is not defined`。老师解释原因——inner是一个变量，放在函数里面就只能在那个函数里面用，超出作用域会报错。
3. **正确调用内层函数的方式**：在outer函数内部、inner函数定义之后的位置调用`inner()`。运行后输出两条语句：先是"我是外层函数"，然后是"我是内层函数"。
4. **空间类比解释**：老师用"大房间套小房间"来比喻嵌套结构——想要进入小房间，必须先进入大房间；如果没有进入outer函数，就无法调用inner。

老师还配合了画图说明，画了一个outer大框，里面包含一个用蓝色标注的inner小框，表示函数的嵌套关系。

### 三、闭包的概念引入（7:05–8:35）

老师正式介绍闭包的定义：

> 闭包就像一个带记忆的工具。外层函数定义了一个局部变量，内层函数使用了这个变量，而且外层函数最后把内层函数给打包返回。哪怕外层函数执行完了，内层函数依然能够记住那个变量的值，不会随着外层函数的结束而消失，下一次调用的时候还能够用上。

老师举了一个生活化的**奶茶店案例**：你去奶茶店点了三分糖的奶茶，店员记住了你的甜度偏好，下次再来点单时不用重复说，店员会按照之前的记忆去做。这里的"参数"是用户，"局部变量"是你的甜度偏好，"调用内层函数"就是你去点单。

### 四、闭包的三个必要条件（8:35–9:48）

老师明确强调，形成闭包必须满足**三个条件，少一个都不可以**：

1. **函数嵌套**：外层函数里面必须定义了内层函数。
2. **变量引用**：内层函数要使用外层函数的局部变量。这个局部变量不能是全局变量，也不能是内层函数自己定义的变量，只能是外层函数的局部变量。
3. **返回内层函数**：外层函数的返回值是内层函数的引用——即返回内层函数的函数名，**不加括号**。老师特别解释：小括号代表调用函数、执行函数内容；不加括号则代表返回函数本身（即函数的引用）。

### 五、基础版闭包代码演示（9:48–16:30）

老师按照三个条件，在IDE中逐行编写基础版闭包：

```python
def outer():
    sugar = "三分糖"
    def inner():
        print(f"制作一杯{sugar}的奶茶")
    return inner

make_milk_tea = outer()
make_milk_tea()
make_milk_tea()
make_milk_tea()
make_milk_tea()
```

演示要点：

1. 确认三个条件都满足：函数嵌套（inner在outer内）、内层函数使用外层函数的局部变量sugar、外层函数返回inner（不加括号）。
2. 调用外层函数`outer()`：得到返回值inner（一个函数对象），IDE打印显示为`Function outer.<locals>.inner at 0x...`，说明返回的是inner函数的内存地址。
3. 用变量`make_milk_tea`接收返回值：`make_milk_tea`实际上就是inner函数本身。
4. 调用`make_milk_tea()`：多次调用，每次都输出"制作一杯三分糖的奶茶"，说明inner记住了sugar变量的值，没有因为outer函数执行完毕而消失。
5. 老师强调这就是闭包的**"记忆魔法"**——虽然outer函数已经执行完，但inner函数依然能访问到outer函数里的局部变量。

### 六、升级版闭包代码演示（16:30–23:05）

老师指出基础版只记住固定的变量，不够灵活，因此升级为**让外层函数也接收参数**，从而能动态记忆不同的值：

```python
def outer(base_num):
    def inner(add_num):
        print(f"{base_num} + {add_num} = {base_num + add_num}")
    return inner

# 第一个闭包：记住 base_num = 10
add = outer(10)
add(3)    # 输出 10 + 3 = 13
add(5)    # 输出 10 + 5 = 15
add(8)    # 输出 10 + 8 = 18

# 第二个闭包：记住 base_num = 15
add15 = outer(15)
add15(3)  # 输出 15 + 3 = 18
add15(10) # 输出 15 + 10 = 25
```

演示要点：

1. `outer(10)`传入参数10，得到inner函数的引用赋值给变量add。此时base_num=10这个局部变量被记住了。
2. 多次调用`add()`，每次只传add_num参数，结果都是在10的基础上相加。
3. 创建另一个闭包`outer(15)`，得到的add15记住的是base_num=15，两者**互不影响、各自独立**。
4. 老师强调：形参（如base_num）本质上也是局部变量，因此可以被内层函数使用和记住。

### 七、闭包的三个核心用途总结（23:05–25:02）

老师总结了闭包的三大核心应用场景：

1. **保存状态**：让函数记住关键变量，避免重复传入相同参数。例如记住用户偏好（如奶茶甜度）、计算基准值等。
2. **数据封装**：外层函数的局部变量只能被内层函数访问，外部无法直接修改，保证数据安全。老师演示在outer函数外部尝试`print(base_num)`会报错，因为base_num是局部变量而非全局变量。
3. **基础铺垫**：闭包是装饰器的核心实现原理，把闭包吃透，后面学装饰器就会很轻松。

## 本P结论

1. 闭包是"带记忆的嵌套函数"，外层函数执行完毕后，内层函数仍能记住外层函数的局部变量。
2. 形成闭包必须同时满足三个条件：**函数嵌套、内层函数引用外层局部变量、外层函数返回内层函数引用（不加括号）**。
3. 闭包的三个核心用途：**保存状态避免重复传参、数据封装保证安全、为装饰器奠定基础**。
4. 闭包的核心价值在于让代码"从单纯实现功能升级到优雅地管理功能"，在复杂项目中能大幅提升复用性和可维护性。
5. 下一P将学习装饰器，它是闭包的直接应用。
