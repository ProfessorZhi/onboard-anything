# P49 知识框架 - 【进阶篇】魔术方法
## 本P位置

进阶篇第49P，承接P48单例模式，聚焦Python面向对象两个高级特性：魔术方法与魔术属性。位于OOP进阶链条：基础语法 → 类与对象 → 继承 → 单例模式 → 魔术方法。后续可能衔接属性访问控制、元类等更高级主题。

## 核心问题

- 什么是魔术方法/魔术属性，命名规范是什么
- 如何利用Python内置的这些特殊方法增强对象行为
- 不同魔术方法的触发时机是什么
- 如何判断一个对象是否可调用

## 知识骨架

```
魔术方法/属性
├── 定义：__名称__格式，Python解释器自动触发
├── 魔术属性（只读信息）
│   ├── __doc__：类的文档字符串
│   ├── __module__：所属模块名（主程序返回__main__）
│   ├── __class__：所属类
│   └── __dict__：实例属性字典
└── 魔术方法（可重写）
    ├── __str__：print/str()时调用，必须返回str
    ├── __del__：对象销毁时调用，用于释放资源
    └── __call__：对象()时调用，使实例可调用
```

## 关系与依赖

| 关系类型 | 说明 |
|---------|------|
| 依赖前置 | 必须先理解类与实例的关系、__init__的作用 |
| 关联上一P | 与单例模式同属OOP高级特性，常结合使用 |
| 内部关联 | __call__与callable()函数形成判断-实现闭环 |
| 依赖解释器 | 所有魔术方法的触发时机由Python解释器控制 |

## 输入 / 输出 / 前置 / 支撑

| 类型 | 内容 |
|-----|------|
| 输入 | 类定义、实例化参数 |
| 前置知识 | 类与对象基础、__init__方法、self概念 |
| 输出 | 自定义打印信息、返回值（__call__）、资源释放 |
| 支撑工具 | callable()用于判断对象是否实现__call__ |
| 触发时机 | 解释器在特定场景自动调用，无需手动 |

## 易错点

| 易错点 | 正确做法 |
|-------|---------|
| __str__返回非字符串 | 必须返回str类型，否则TypeError |
| __del__手动调用 | 禁止手动调用，交给垃圾回收机制 |
| 对象默认可调用 | 未实现__call__时，对象()会报TypeError |
| __module__混淆 | 主程序执行返回__main__，非模块名 |
| __call__参数随意 | 虽可自定义，但需与实际调用匹配 |
