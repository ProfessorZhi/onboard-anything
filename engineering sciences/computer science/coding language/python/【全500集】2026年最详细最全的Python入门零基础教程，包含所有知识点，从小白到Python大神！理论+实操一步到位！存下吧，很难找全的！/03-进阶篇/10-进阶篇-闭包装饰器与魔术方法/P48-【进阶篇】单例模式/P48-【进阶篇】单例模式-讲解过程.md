### 2. 铺垫知识：理解 `__new__` 和 `__init__` 的区别与执行顺序

**时间区间：114.0 - 817.1**

老师指出，在正式学习单例模式之前，必须先搞懂一个关键知识点：**对象实例化的秘密**，也就是 `__new__` 和 `__init__` 的区别。

#### 2.1 核心概念澄清

- 大多数学生以为 `__init__` 是创建对象的第一步，但实际上**最先执行的是 `__new__`**。
- 两者是对象创建时的"黄金搭档"，分工明确，缺一不可。

#### 2.2 造房子比喻

老师用造房子的比喻帮助理解：

- `__new__`：负责**打地基、造房子框架**，核心任务是在内存中为对象分配一块空间，然后返回对象引用——相当于给房子编一个门牌号。
- `__init__`：负责**装修房子、摆放家具**，接收 `__new__` 返回的对象引用（即 `self`），给对象初始化属性。

简单总结：`__new__` 用来**创建对象**，`__init__` 用来**初始化对象**。

#### 2.3 对象实例化的三步执行流程

当通过类名加小括号 `类名()` 创建对象时，Python 解释器按以下三步执行：

1. **执行 `__new__`**：分配内存空间，返回对象引用。
2. **自动传参**：Python 解释器把 `__new__` 返回的对象引用作为 `self` 参数，传递给 `__init__`。
3. **执行 `__init__`**：给对象设置属性，完成最终初始化动作。

#### 2.4 代码演示：House 类

老师编写一个 `House` 类，直观展示两者的执行顺序：

```python
class House:
    def __new__(cls, *args, **kwargs):
        print("__new__执行：分配内存，造房子框架")
        obj = object.__new__(cls)  # 调用父类的__new__分配内存
        return obj  # 必须返回对象引用，否则__init__不会执行
    
    def __init__(self, address, area):
        print("__init__执行：装修房子，摆放家具")
        self.address = address
        self.area = area
```

然后实例化：
```python
my_house = House("湖南长沙", 120)
```

运行后观察到打印顺序：**先输出 `__new__` 的打印，再输出 `__init__` 的打印**，证明 `__new__` 先执行。

老师还打印 `obj` 本身，说明 `obj` 就是 `__init__` 中的 `self`，即对象本身。

#### 2.5 关键注意点：`__new__` 必须返回对象引用

老师强调：如果重写 `__new__` 但没有返回对象引用，`__init__` 不会执行。

演示错误写法：
```python
class House:
    def __new__(cls, *args, **kwargs):
        print("只造框架，不返回门牌号")
        # 没有 return 语句
    
    def __init__(self, address, area):
        print("装修房子，摆好家具")
```

实例化后发现：**只执行了 `__new__`，`__init__` 未触发**。

老师解释：`__new__` 必须调用父类（顶级父类 `object`）的 `__new__` 方法来创建对象引用并返回。如果 `return 123` 之类的非对象引用是不行的，`__init__` 会因为找不到对象而无法执行。

#### 2.6 对比表格

老师整理了一张表格，对比 `__new__` 和 `__init__` 的核心区别：

| 对比维度 | `__new__` | `__init__` |
|---|---|---|
| 核心作用 | 创建对象、分配内存 | 初始化对象、设置属性 |
| 第一个参数 | `cls` 代表当前类 | `self` 代表对象本身 |
| 返回值 | 必须返回对象引用（调用父类 `__new__`） | 无返回值，默认返回 `None` |
| 执行时机 | 实例化时**先执行** | `__new__` 执行完后**后执行** |

### 3. 正式学习单例模式

**时间区间：820.7 - 959.7**

#### 3.1 什么是单例模式

单例模式是一种设计模式，核心思想一句话：**让一个类在整个程序中只能创建一个实例对象**。

#### 3.2 生活实例扩展

除了前面提到的回收站，老师还补充了多个生活中的单例例子：

- **公司打印机**：全公司员工共用一台打印机，不会每人配一台，避免资源浪费。
- **手机短信应用**：无论从哪个入口打开，都是同一个界面，数据保持一致。
- **音乐播放器**：同一时间只播放一首歌，播放器实例只有一个，避免同时播放多首歌的冲突。

#### 3.3 单例模式的核心优势

1. **节省内存资源**：避免创建多个重复的对象。尤其是创建成本高的对象（如数据库连接、网络连接），创建一次供全程序使用，不必每次操作数据库都新建连接。
2. **统一管理，保证数据一致性**：全程序使用同一个对象，对象的属性和状态是统一的，不会出现多个对象数据不同的情况，避免冲突。
   - 举例：游戏场景管理器，所有角色由同一个管理器控制，不会状态混乱。
   - 举例：配置管理器，所有模块读取同一个配置，不会读不同的配置文件导致冲突。

### 4. 单例模式的代码实现

**时间区间：959.7 - 1639.4**

#### 4.1 实现思路

重写 `__new__` 方法是最常用且最直观的方式，设计思路如下：

1. **定义类属性** `_instance`，初始值设为 `None`，专门用来记录唯一的对象。
2. **重写 `__new__`**：
   - 如果 `_instance` 为 `None`，说明还没有创建对象，调用父类的 `__new__` 创建对象，并存入 `_instance`。
   - 如果 `_instance` 不为 `None`，说明已经有对象了，直接返回 `_instance` 中记录的对象。
3. 这样无论实例化多少次，都只会返回同一个对象。

#### 4.2 基础版本代码

```python
class Singleton:
    _instance = None  # 类属性，初始值None
    
    def __new__(cls, *args, **kwargs):
        if cls._instance is None:  # 如果没有对象
            obj = object.__new__(cls)  # 创建对象
            cls._instance = obj  # 存入类属性
        return cls._instance  # 直接返回
```

#### 4.3 代码调试过程

老师编写代码时出现一个小错误，演示了调试思路：

- 初次实现发现 `Singleton` 为 `None`。
- 检查发现 `if` 分支里忘记写 `return` 语句，而 `else` 里也有 `return`，两处代码重复。
- 优化：直接去掉 `else`，在 `if` 判断后面直接 `return cls._instance`，无论是否创建新对象都返回同一个引用。

#### 4.4 验证单例效果

```python
obj1 = Singleton()
obj2 = Singleton()
print(id(obj1) == id(obj2))  # True，说明是同一个对象
```

老师用 `id()` 方法查看对象的内存地址（返回十进制），对比发现 `obj1` 和 `obj2` 的 `id` 相同，证明是同一个对象。

### 5. 解决重复初始化问题

**时间区间：1343.0 - 1625.7**

#### 5.1 问题描述

基础版本的单例模式存在一个问题：如果类有 `__init__` 方法，每次实例化时都会执行 `__init__`，后面的值会覆盖前面的值。

```python
class Singleton:
    _instance = None
    
    def __new__(cls, *args, **kwargs):
        if cls._instance is None:
            obj = object.__new__(cls)
            cls._instance = obj
        return cls._instance
    
    def __init__(self, name):
        self.name = name  # 每次实例化都会重新赋值

obj1 = Singleton("冰冰")
obj2 = Singleton("苏苏")
obj3 = Singleton("小红")
print(obj1.name)  # 输出：小红，后面的值覆盖了前面的
```

#### 5.2 解决方案：使用 `hasattr` 避免重复初始化

```python
def __init__(self, name):
    if not hasattr(self, "name"):  # 如果对象没有name属性
        self.name = name  # 才初始化
```

这样：
- 第一次实例化时，对象没有 `name` 属性，执行赋值。
- 后续实例化时，对象已有 `name` 属性，跳过赋值，保留第一次的值。

老师强调：`hasattr(obj, "属性名")` 用于检查对象是否有指定的属性或方法，**属性名需要以字符串形式传入**。这样就避免了重复初始化问题，使单例模式更加完善。

### 6. 另一种实现方式：利用 Python 模块天然单例特性

**时间区间：1639.4 - 1796.0**

#### 6.1 原理

Python 的模块是天然的单例模式：

- 模块第一次被导入时，会生成一个 `.pyc` 编译文件。
- 后续再导入该模块时，不会重新执行模块代码，只会加载已生成的 `.pyc` 文件。
- 因此模块中的对象也是唯一的。

#### 6.2 代码演示

**步骤一**：新建模块文件 `singleton.py`，定义类并实例化对象：

```python
class TestSingleton:
    def show(self):
        print("我是唯一的对象")

unique_obj = TestSingleton()
```

**步骤二**：回到主文件，多次导入并验证：

```python
from singleton import unique_obj as obj1
from singleton import unique_obj as obj2

print(id(obj1) == id(obj2))  # True
obj1.show()  # 可正常调用
```

运行结果显示 `obj1` 和 `obj2` 的 `id` 相同，证明是同一个对象。

#### 6.3 适用场景

老师指出，这种模块方式适合简单场景，不需要写复杂的 `__new__` 重写逻辑，直接导入模块对象即可。

### 7. 单例模式的应用场景总结

**时间区间：1800.8 - 1843.9**

老师总结了单例模式的三类典型应用场景：

1. **系统工具类**：回收站、打印机、音乐播放器、日志管理器（同一时间只有一个日志写入对象）。
2. **资源密集型对象**：数据库连接池、网络连接、文件读写句柄。这类对象创建成本高，复用对象更高效。
3. **全局的管理器**：游戏场景管理器、配置管理器、用户状态管理器。需要统一管理数据或状态，避免冲突。

## 本P结论

1. **对象实例化的完整流程**：通过 `类名()` 创建对象时，Python 先执行 `__new__` 分配内存并返回对象引用，再将引用作为 `self` 传递给 `__init__` 进行属性初始化。`__new__` 必须返回对象引用，否则 `__init__` 不会执行。

2. **单例模式的核心机制**：通过重写 `__new__` 方法，配合类属性 `_instance` 记录唯一对象，确保无论实例化多少次都返回同一对象。

3. **避免重复初始化的方法**：在 `__init__` 中使用 `hasattr(self, "属性名")` 判断对象是否已有该属性，只有首次初始化时才赋值。

4. **模块天然单例**：Python 模块第一次导入后生成 `.pyc`，后续导入只加载已编译文件，模块内对象天然唯一，适合简单场景。

5. **单例模式的三大应用场景**：系统工具类、资源密集型对象、全局管理器。
