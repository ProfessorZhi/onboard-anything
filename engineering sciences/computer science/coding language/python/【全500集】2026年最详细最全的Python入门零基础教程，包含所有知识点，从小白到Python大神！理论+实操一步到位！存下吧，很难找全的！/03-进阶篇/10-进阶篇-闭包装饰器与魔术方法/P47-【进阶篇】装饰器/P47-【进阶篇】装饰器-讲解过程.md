# P47 讲解过程 - 【进阶篇】装饰器
## 本P定位

本P是Python入门课程的进阶篇内容，主题为「装饰器」。承接前几P对闭包（闭包）的讲解，进一步讲解Python中装饰器这一核心特性的原理与使用方法。装饰器是Python中用于在不修改原函数代码、也不改变函数调用方式的前提下，给函数添加额外功能的语法机制。

## 讲解过程

### 一、问题引入：为什么需要装饰器

**场景设定**：假设有三个函数——发消息（send_message）、转账（send_money）、看动态（look_news），产品要求只有在登录之后才能执行这些功能。

**直接修改的两种问题**：
- **代码冗余**：三个函数都要写一遍相同的登录验证逻辑，后续修改任何一个都要同步修改其他两个，可维护性差。实际场景中可能还有看朋友圈、看视频号、发朋友圈等功能，函数数量更多。
- **无法修改第三方函数**：如果这些函数是第三方库提供的，根本改不了源代码。自己的核心代码也可能不允许他人修改。

**装饰器的核心价值**：像给函数「穿衣服」一样，给原函数套一层新的功能。原函数本身完全不变，既不需要修改代码，也不需要改变调用方式，新增功能还能复用。

### 二、装饰器的手动实现方式

**装饰器的本质**：一个接收函数作为参数、返回新函数的闭包。输入是原函数，输出是原函数加新功能的组合函数。

**闭包实现装饰器的三大条件**：

**1. 函数嵌套**：定义外层函数和内层函数

```python
def login_decorator(func):
    def wrapper():
        pass
    return wrapper
```

**2. 内层函数使用外层函数的局部变量**：把原函数作为外层函数的参数，内层函数调用原函数

```python
def login_decorator(func):
    def wrapper():
        # 新增功能写在原函数调用之前——先登录验证，再执行原功能
        print("登录验证")
        # 原函数的功能
        func()
    return wrapper
```

**3. 外层函数的返回值是内层函数的引用**：return wrapper

**手动调用装饰器的方式**：

```python
# 第一步：调用外层函数，传入原函数作为参数
send_msg_function = login_decorator(send_message)
# 第二步：调用内层函数 wrapper()
send_msg_function()
```

**执行流程解析**：
- 调用 login_decorator(send_message) 时，传入的 func 就是 send_message 函数
- 函数定义部分被忽略（因为只是定义，没有调用）
- return wrapper 把内层函数 wrapper 作为返回值
- send_msg_function 接收到的就是 wrapper 函数引用，可以调用
- 调用 send_msg_function() 时，实际调用的是 wrapper()
- wrapper() 先执行新增的登录验证功能，再调用原函数 func()

同样的装饰器可以复用给其他函数，只需要传入不同的函数名即可解决代码冗余问题。

### 三、装饰器的语法糖（@语法）

**语法糖写法**：在原函数上方使用 @装饰器名，等效于手动包装的代码

```python
@login_decorator
def send_message():
    print("发送消息给冰冰")
```

这行 @login_decorator 等价于：

```python
send_message = login_decorator(send_message)
```

**语法糖的优势**：
- 代码更简洁、更优雅
- 一眼就能看出函数被添加了登录验证功能
- 这是实际开发中最常用的方式，推荐使用

### 四、带参数的被装饰函数

**问题**：前面的示例中原函数都没有参数，但实际开发中函数往往需要接收参数。例如 send_message(name) 发送消息给谁，send_money(name, money) 转账给谁多少元。

**解决方案**：给装饰器的内层函数加上 *args 和 **kwargs 两个参数，能够接收任意数量、任意类型的参数，然后再把参数传给原函数。

**标准写法**：

```python
def login_test(func):
    def wrapper(*args, **kwargs):
        print("登录验证")  # 新增功能
        result = func(*args, **kwargs)  # 原功能，接收返回值
        print("退出登录")  # 也可以在原功能之后新增功能
        return result
    return wrapper
```

**为什么不使用固定参数**：如果装饰器内层函数写死为 name 参数，那么只能装饰接收 name 参数的函数，无法装饰 send_money 这样接收多个不同参数（如 name 和 money）的函数。使用 *args 和 **kwargs 可以装饰任意参数组合的函数，更加灵活。

**验证示例**：

```python
@login_test
def send_message(name):
    print(f"发送消息给{name}")

send_message("冰冰")
# 输出：
# 登录验证
# 发送消息给冰冰

@login_test
def send_money(name, money):
    print(f"转账给{name}金额{money}元")

send_money("冰冰", 500)
# 输出：
# 登录验证
# 转账给冰冰金额500元
```

### 五、多个装饰器的叠加使用

**叠加规则**：一个函数可以套多个装饰器。执行顺序是「由内到外」——先执行离函数最近的装饰器（最后写的 @装饰器），再执行离函数远的装饰器（最先写的 @装饰器）。就像穿衣服，先穿里面的衣服，再穿外套。

**示例**：

```python
# 定义加前缀的装饰器
def decorator1(func):
    def wrapper(*args, **kwargs):
        result = func(*args, **kwargs)
        return "前缀哈哈" + result
    return wrapper

# 定义加后缀的装饰器
def decorator2(func):
    def wrapper(*args, **kwargs):
        result = func(*args, **kwargs)
        return result + "后缀嘻嘻"
    return wrapper

@decorator2
@decorator1
def hello():
    return "hello"
```

**执行顺序解析**（decorator2 在上、decorator1 在下的情况）：
- 先执行 decorator1(func) ——离函数更近，先执行
  - func 是 hello()，返回值是 "hello"
  - 返回 "前缀哈哈" + "hello" = "前缀哈哈hello"
- 再执行 decorator2(func) ——离函数更远，后执行
  - func 是 "前缀哈哈hello"
  - 返回 "前缀哈哈hello" + "后缀嘻嘻" = "前缀哈哈hello后缀嘻嘻"

**如果交换装饰器顺序**（decorator1 在上、decorator2 在下）：
- 先执行 decorator2(hello)
- 再执行 decorator1(结果)
- 最终得到 "前缀哈哈hello后缀嘻嘻"，结果相同
- 但注意：前缀和后缀添加的时机不同，最终字符串内容顺序可能不同（取决于装饰器内部逻辑如何拼接）

**核心结论**：多个装饰器的执行顺序是「由内到外」——离被装饰函数越近，越先执行。

## 本P结论

1. **装饰器的本质是闭包**：装饰器是一个接收函数作为参数、返回新函数的闭包，满足函数嵌套、内层函数使用外层函数局部变量、外层函数返回内层函数引用三大条件。

2. **装饰器的核心价值**：在不修改原函数代码、不改变函数调用方式的前提下，给函数添加额外的功能，同时新增功能可以被多个函数复用。

3. **装饰器的两种调用方式**：手动包装（调用外层函数接收返回值，再调用返回值）和语法糖（@装饰器名，推荐使用）。

4. **带参数函数的装饰器标准写法**：内层函数使用 *args 和 **kwargs 接收任意参数，再传给原函数 func(*args, **kwargs)，保证装饰器能复用给任意参数组合的函数。

5. **多个装饰器的执行顺序**：由内到外——离被装饰函数越近的装饰器越先执行，后写的装饰器先执行。
