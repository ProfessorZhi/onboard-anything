## 一、结构识别

**题目 1.1**  
以下代码定义了三个类，请指出它们的继承关系，并说明哪个是父类、哪个是子类。

```python
class Vehicle:
    def move(self):
        print("行驶中")

class Car(Vehicle):
    pass

class Bike(Vehicle):
    pass
```

<details>
<summary>参考答案</summary>

- `Vehicle` 是父类（基类）
- `Car` 和 `Bike` 是子类，继承自 `Vehicle`
- 继承关系：`Car → Vehicle → object`，`Bike → Vehicle → object`

</details>

**题目 1.2**  
以下三层结构中，哪些类之间存在直接继承关系？

```
LivingThing
    ↓
Animal
    ↓
Dog
```

<details>
<summary>参考答案</summary>

- `Animal` 直接继承 `LivingThing`
- `Dog` 直接继承 `Animal`
- `Dog` 与 `LivingThing` 是间接继承关系（通过 `Animal`）
- 注意：`Dog` 不是 `LivingThing` 的直接子类，但通过继承链仍然拥有 `LivingThing` 的成员

</details>

## 二、机制理解

**题目 2.1**  
阅读以下代码，说出调用 `dog.bark()` 时的输出顺序，并解释原因。

```python
class Animal:
    def greet(self):
        print("Animal greet")

class Dog(Animal):
    def greet(self):
        print("Dog greet")

class Husky(Dog):
    def greet(self):
        print("Husky greet")
        super().greet()

h = Husky()
h.greet()
```

<details>
<summary>参考答案</summary>

输出顺序：
```
Husky greet
Dog greet
```

原因：子类 `Husky` 重写了 `greet` 方法，内部通过 `super().greet()` 调用父类 `Dog` 的实现。由于 `Husky` 的 `greet` 方法中没有调用 `Animal` 的 `greet`，因此 `Animal` 的实现不会被执行。

</details>

**题目 2.2**  
关于私有属性，以下代码的运行结果是什么？

```python
class Parent:
    def __init__(self):
        self.public = "公开"
        self.__private = "私有"

class Child(Parent):
    def show(self):
        print(self.public)
        print(self.__private)

c = Child()
c.show()
```

<details>
<summary>参考答案</summary>

- `print(self.public)` 输出：`公开`
- `print(self.__private)` 报错：`AttributeError: 'Child' object has no attribute '_Child__private'`

原因：以 `__` 开头的私有属性不会被子类继承，Python 会在编译时将其名称改写为 `_类名__属性名`，而子类找不到改写后的名称。

</details>

**题目 2.3**  
多继承时，方法解析顺序（MRO）遵循什么规则？

```python
class A:
    def test(self):
        print("A")

class B:
    def test(self):
        print("B")

class C(A, B):
    pass

c = C()
c.test()
```

<details>
<summary>参考答案</summary>

输出：`A`

规则：多继承时，Python 按照「从左到右」的顺序在父类列表中查找方法。因此 `C(A, B)` 会先在 `A` 中查找，找到则停止搜索。

</details>

## 三、最小实践

**题目 3.1**  
编写代码：定义一个 `Shape` 基类，包含 `area()` 方法返回 0。然后定义 `Square` 子类，继承 `Shape`，在 `__init__` 中接收边长，并重写 `area()` 返回边长的平方。实例化并调用。

```python
class Shape:
    def area(self):
        return 0

class Square(Shape):
    def __init__(self, side):
        self.side = side
    
    def area(self):
        return self.side ** 2

s = Square(5)
print(s.area())
```

<details>
<summary>参考答案</summary>

输出：`25`

关键点：子类通过重写方法覆盖了父类的 `area()` 实现。

</details>

**题目 3.2**  
编写代码：定义 `Base` 类包含 `greet()` 方法输出 `"Hello"`，`Derived` 类继承 `Base` 并在 `greet()` 中**先调用父类方法**，再输出 `"World"`。使用 `super()` 实现。

```python
class Base:
    def greet(self):
        print("Hello")

class Derived(Base):
    def greet(self):
        super().greet()
        print("World")

d = Derived()
d.greet()
```

<details>
<summary>参考答案</summary>

输出：
```
Hello
World
```

关键点：`super().greet()` 调用父类实现，保留父类逻辑后再扩展子类逻辑。

</details>

**题目 3.3**  
使用**组合**替代继承，重写以下继承结构：

```python
# 原继承方式
class Engine:
    def start(self):
        print("引擎启动")

class Car(Engine):
    pass
```

请用组合方式实现，使 `Car` 能够使用 `Engine` 的功能。

```python
# 组合方式
class Engine:
    def start(self):
        print("引擎启动")

class Car:
    def __init__(self):
        self.engine = Engine()
    
    def start(self):
        self.engine.start()

c = Car()
c.start()
```

<details>
<summary>参考答案</summary>

关键区别：

- 继承：`Car(Engine)` 表示 "Car **是** Engine"
- 组合：`Car` 持有 `Engine` 实例，表示 "Car **有** Engine"

组合避免了多继承的方法冲突问题，是更灵活的代码复用方式。

</details>

## 四、扩展思考

**题目 4.1**  
在以下场景中，你倾向于使用多继承还是单继承+组合？为什么？

> 场景：一个 `Smartphone` 需要同时具备 `Camera` 的拍照功能和 `Phone` 的通话功能。

<details>
<summary>参考答案</summary>

**推荐：单继承 + 组合**

```python
class Phone:
    def call(self):
        print("通话中")

class Camera:
    def take_photo(self):
        print("拍照中")

class Smartphone(Phone):
    def __init__(self):
        self.camera = Camera()
```

理由：

- `Smartphone` 不是 `Camera`，也不是 `Phone`（is-a 关系不成立）
- 多继承可能导致方法冲突，如两个父类都有 `start()` 方法
- 组合更灵活，可以随时替换 `Camera` 的实现

</details>

**题目 4.2**  
以下两种调用父类方法的写法，哪种更灵活？为什么？

```python
# 写法 A
class Child(Parent):
    def greet(self):
        super().greet()
        print("child")

# 写法 B
class Child(Parent):
    def greet(self):
        Parent.greet(self)
        print("child")
```

<details>
<summary>参考答案</summary>

**写法 A（super）更灵活**，原因：

| 方面 | super() | 类名.method(self) |
|------|---------|-------------------|
| 父类重命名 | 不受影响 | 必须修改代码 |
| 多继承 | 自动按 MRO 顺序查找 | 固定调用某个父类 |
| 代码可维护性 | 高 | 低 |

```python
# 如果 Parent 更名为 Base
# 写法 B 必须同时修改 Parent.greet(self) → Base.greet(self)
# 写法 A 无需任何修改
```

</details>

**题目 4.3**  
如果子类没有定义 `__init__` 方法，实例化时需要注意什么？请举例说明。

<details>
<summary>参考答案</summary>

**注意：子类会继承父类的 `__init__`，必须按父类签名传参。**

```python
class Parent:
    def __init__(self, name):
        self.name = name

class Child(Parent):
    pass

# 错误：缺少 name 参数
# c = Child()  → TypeError

# 正确：必须传递 name
c = Child("小明")
print(c.name)  # 小明
```

核心原因：子类未定义 `__init__` 时，直接使用父类的 `__init__`，父类需要什么参数，子类就得传什么参数。

</details>
