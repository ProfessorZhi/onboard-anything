# P35 讲解过程 - 【进阶篇】面向对象之继承
## 本P定位

本P属于“进阶篇”的核心知识点，承接面向对象基础，引入类与类之间的层级复用机制。通过生活类比、代码演示、错误排查三种方式，依次讲解继承概念、单继承语法、继承传递性、方法重写、多继承语法与坑点，最后给出替代方案和总结。

---

## 讲解过程

### 一、引入继承概念（0:01-0:40）

**核心目标**：用生活例子建立直观认知

老师首先说明继承的**核心作用是代码重用**。用生活类比引入：儿子会继承父亲的身高、发色；对应到Python中，子类能够直接继承父类的属性和方法，不需要重复编写相同代码。

**关键结论**：
- 学会继承能大幅减少冗余代码
- 让类和类之间形成清晰的家族关系
- 代码更简洁、更好维护

### 二、什么是继承（0:40-1:04）

**概念定义**：
- 继承是类与类之间的**父子关系**
- 被继承的类叫**父类**，也叫**基类**。举例：动物类
- 继承父类的类叫**子类**，也叫**派生类**。举例：狗类、猫类

**核心逻辑**：
1. 子类能直接使用父类的**非私有**属性和方法
2. 子类能根据需求**新增**自己的属性或方法
3. 子类甚至可以**修改**父类已有的方法

### 三、生活实例巩固概念（1:11-2:12）

通过三个生活类比让概念落地：

| 父类 | 子类 | 继承的内容 | 子类特有 |
|------|------|-----------|---------|
| 动物 | 狗 | 吃东西、睡觉 | 汪汪叫 |
| 电子设备 | 手机 | 通电开机 | 打电话、发消息 |
| 人 | 学生 | 姓名、年龄 | 成绩 |

**提炼公式**：继承 = 复用已有的 + 新增特有的

### 四、为什么要学习继承（2:12-3:25）

老师给出继承的**三个核心作用**：

1. **减少重复代码**
   - 举例：狗、猫、猪三个类都有姓名、年龄属性和吃东西、睡觉方法
   - 如果每个类都写一遍，代码冗余
   - 把共性内容统一放到动物父类，子类直接继承，代码量大幅减少

2. **代码结构清晰**
   - 通过父类-子类层级关系，代码逻辑更直观
   - 举例：生物类 → 动物类 → 狗类，一眼就能看出关联

3. **便于扩展子类**
   - 在父类基础上新增功能，不影响父类本身
   - 给狗类添加“叼东西”方法，完全不影响父类动物和其他子类

### 五、单继承语法与演示（3:25-8:17）

**语法结构**：
```python
class 子类名(父类名):
    pass
```

**代码演示：Animal类与Dog类**

第一步：定义父类Animal
```python
class Animal:
    def __init__(self, name, age):
        self.name = name
        self.age = age
    
    def eat(self):
        print(f"{self.name} 在吃东西")
    
    def sleep(self):
        print(f"{self.name} 在睡觉")
```

第二步：定义子类Dog，继承Animal
```python
class Dog(Animal):
    def bark(self):
        print("小狗汪汪叫")
```

第三步：实例化子类并调用方法
```python
dog1 = Dog("旺财", 1)  # 需要传参，因为继承了父类的__init__
print(dog1.name)       # 输出：旺财（继承自父类的属性）
dog1.eat()             # 输出：旺财 在吃东西（继承自父类的方法）
dog1.sleep()           # 输出：旺财 在睡觉（继承自父类的方法）
dog1.bark()            # 输出：小狗汪汪叫（子类自己的方法）
```

**错误排查演示**：
- 如果直接 `Dog()` 不传参数会**报错**
- 原因：子类自己没有 `__init__`，但继承了父类的 `__init__`，所以需要传参
- 老师强调：子类可以访问父类的**公开属性和方法**，**私有属性不会继承**

**关键结论**：子类对象既能有父类的属性和方法，也能有自己的属性和方法，完全不用重复写父类的代码。

### 六、继承的传递性（8:18-10:58）

**概念**：如果类A继承类B，类B又继承类C，那么类A会**间接**继承类C的非私有属性和方法。

**代码演示：三层继承关系**

```python
class LivingThing:
    feature = "有生命"  # 类属性
    
    def breathe(self):
        print("呼吸")

class Animal(LivingThing):  # Animal继承LivingThing
    pass

class Dog(Animal):  # Dog继承Animal
    pass
```

验证传递性：
```python
dog1 = Dog("旺财", 1)
print(dog1.feature)  # 输出：有生命（间接继承自LivingThing）
dog1.breathe()       # 输出：呼吸（间接继承自LivingThing）
```

**生活类比**：爷爷传家宝给爸爸，爸爸传给你，你传给儿子……一代一代传承下来。

**关键结论**：继承具有传递性，子类不仅能继承直接父类，还能继承间接上层父类的非私有成员。

### 七、方法的重写（11:02-15:56）

**背景**：子类继承父类后，可能觉得父类的某个方法不符合自己的需求。

**定义**：子类定义一个和父类**同名**的方法，覆盖或扩展父类的功能。

**两种情况**：

#### 情况1：覆盖父类方法

子类的方法实现和父类**完全不同**。

```python
class Animal:
    def make_sound(self):
        print("动物发出声音")

class Cat(Animal):
    def make_sound(self):  # 与父类同名，完全覆盖
        print("猫咪：喵喵喵")
```

验证：
```python
animal = Animal()
animal.make_sound()  # 输出：动物发出声音

cat = Cat()
cat.make_sound()     # 输出：猫咪：喵喵喵
```

#### 情况2：扩展父类方法

保留父类方法的核心功能，再新增自己的逻辑。使用 `super()` 语法：

```python
class Animal:
    def make_sound(self):
        print("动物发出声音")

class Cat(Animal):
    def make_sound(self):
        super().make_sound()  # 先调用父类的方法
        print("猫咪：喵喵喵") # 再新增自己的逻辑
```

**`super()` 解释**：
- `super()` 是一个特殊的类
- `super()` 创建的对象代表**父类对象**
- 可以调用父类的方法
- 推荐使用 `super()`，因为它**不依赖父类名**，更灵活、可维护性更好

**关键结论**：扩展比完全覆盖更灵活，既保留父类功能，又扩展子类特性。

### 八、多继承语法（16:00-20:58）

**定义**：子类同时继承**多个**父类，整合多个父类的功能。

**语法**：
```python
class 子类名(父类1, 父类2, ...):
    pass
```

**代码演示：智能手表同时继承手表类和健康设备类**

```python
class Watch:
    def show_time(self):
        print("当前时间：16:25")

class HealthDevice:
    def check_heart_rate(self):
        print("心率监测：75次/每秒")

class SmartWatch(Watch, HealthDevice):  # 同时继承两个父类
    pass
```

验证：
```python
sw = SmartWatch()
sw.show_time()          # 输出：当前时间：16:25（来自Watch）
sw.check_heart_rate()   # 输出：心率监测：75次/每秒（来自HealthDevice）
```

### 九、多继承的坑：同名方法冲突（20:58-25:48）

**问题**：如果多个父类有同名方法，子类调用时会遵循继承顺序，**优先继承左边的父类方法**。

**代码演示：拍照手机类**

```python
class Phone:
    def take_photo(self):
        print("手机拍照：普通画质")

class Camera:
    def take_photo(self):
        print("相机拍照：高清画质")

class CameraPhone(Phone, Camera):  # 左边是Phone
    pass
```

验证：
```python
cp = CameraPhone()
cp.take_photo()  # 输出：手机拍照：普通画质（优先使用左边父类Phone的方法）
```

**两种解决方案**：

**方案1：直接指定父类名调用**
```python
Camera.take_photo(cp)  # 传入子类对象作为self参数，输出：相机拍照：高清画质
```

注意：通过类名调用实例方法时，需要手动传入对象作为 `self` 参数。

**方案2：在子类中重写方法，整合逻辑**
```python
class CameraPhone(Phone, Camera):
    def take_photo(self):
        print("手机拍照：普通画质")
        print("相机拍照：高清画质")
```

### 十、多继承的弊端与替代方案（25:48-27:48）

**多继承的三大弊端**：

1. **冲突风险高**：除了方法同名，属性同名也可能出问题，且依赖继承顺序，改顺序容易出错

2. **代码复杂度高**：父类太多时，层级关系混乱，难以理解和维护

3. **耦合性强**：牵一发而动全身，任意一个父类修改了属性或方法，此类很可能受影响

**推荐替代方案：单继承 + 组合**

```python
class Watch:
    def show_time(self):
        print("当前时间：16:25")

class HealthDevice:
    def check_heart_rate(self):
        print("心率监测")

class SmartWatch(Watch):  # 只继承一个父类
    def __init__(self):
        self.hd = HealthDevice()  # 在init中创建其他类的实例
```

调用方式：
```python
sw = SmartWatch()
sw.show_time()           # 直接调用继承的方法
sw.hd.check_heart_rate() # 通过实例调用其他类的功能
```

**关键结论**：单继承 + 组合既避免了多继承冲突，又能实现多个功能复用，代码更稳定、更好维护。

### 十一、核心总结（27:55-30:15）

| 序号 | 知识点 | 核心内容 |
|------|--------|---------|
| 1 | 继承的本质 | 子类复用父类的非私有属性和方法，核心目的是减少重复代码 |
| 2 | 单继承 | 括号内只指定一个父类名，支持传递性，是实际开发中最常用的方式 |
| 3 | 方法重写 | 覆盖（同名方法完全替换）vs 扩展（用super()调用父类再新增逻辑） |
| 4 | super()推荐 | 不依赖父类名，更灵活、可维护性更好，除非迫不得已才用类名调用 |
| 5 | 多继承 | 括号内指定多个父类名，但存在同名冲突、耦合性高的问题，**尽量避免使用** |
| 6 | 继承顺序 | 单继承：子类→父类→更上层父类；多继承：子类→左侧父类→右侧父类→object（所有类的顶级父类） |

**课后练习建议**：自己定义一个父类到子类的层级关系，如：电子产品 → 电脑 → 笔记本电脑。给父类写通用属性和方法，子类通过继承复用，再重写或新增自己特有的方法。

## 本P结论

1. **继承**是面向对象的重要特性，核心是子类复用父类的非私有属性和方法，实现代码重用、减少冗余。

2. **单继承**（`class 子类(父类)`）是最常用的继承方式，具有传递性，支持多层继承链。

3. **方法重写**分为覆盖（直接替换）和扩展（`super()`调用父类再新增）两种，推荐使用 `super()` 方式。

4. **多继承**（`class 子类(父类1, 父类2)`）虽然能整合多个父类功能，但存在同名冲突、耦合性高的风险，实际开发中**尽量避免**，推荐用单继承加组合替代。
