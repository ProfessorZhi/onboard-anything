# P35 知识框架 - 【进阶篇】面向对象之继承
## 本P位置
进阶篇 · 面向对象核心机制。承接类与对象基础（P33-P34），为后续多态、Mixin模式、框架设计打基础。

---

## 核心问题
如何让类与类之间实现代码复用，避免重复定义？

---

## 知识骨架

**继承 = is-a 关系**
父类定义公共属性和方法，子类直接继承使用

**三层结构**
```
LivingThing（顶层）
    ↓
Animal（中间层）
    ↓
Dog（具体类）
```

**语法**
```python
class 子类(父类):
    pass
```

**方法重写两种方式**
| 方式 | 语法 | 结果 |
|------|------|------|
| 覆盖 | 直接定义同名方法 | 完全替换父类实现 |
| 扩展 | super().method() + 新逻辑 | 保留父类 + 新增子类逻辑 |

**多继承语法**
```python
class 子类(父类1, 父类2):
    pass
```

**组合替代方案**
```python
class 子类(父类):
    def __init__(self):
        self.other = 其他类()
```

---

## 关系与依赖

**继承链**
单继承：子类 → 父类 → object（隐式顶级父类）
多继承：子类 → 左侧父类 → 右侧父类 → object

**方法解析顺序（MRO）**
多继承时，优先继承左边父类的同名方法

**替代关系**
多继承（有冲突风险）→ 单继承 + 组合（推荐）

**super() 依赖**
super() 是内置类，不依赖父类名，比 类名.method(self) 更灵活

---

## 易错点

**子类实例化必须传参**
子类未定义 __init__ 时继承父类 __init__，不传参则 TypeError

**私有成员不继承**
父类以 __ 开头的属性和方法不会被子类继承

**多继承同名冲突**
左侧父类优先级高于右侧，调错方法导致逻辑错误

**通过类名调用实例方法需手动传 self**
父类名.method(子类实例)
