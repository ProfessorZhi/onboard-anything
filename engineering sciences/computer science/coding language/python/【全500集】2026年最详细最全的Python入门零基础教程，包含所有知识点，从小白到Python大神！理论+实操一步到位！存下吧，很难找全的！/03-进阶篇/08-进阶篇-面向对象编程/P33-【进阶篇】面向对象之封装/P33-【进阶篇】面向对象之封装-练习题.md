## 参考答案（简要）

### 1. 结构识别

| 标记 | 属性/方法     | 类型         |
|------|---------------|--------------|
| A    | name          | 公开属性     |
| B    | __age         | 私有属性     |
| C    | _grade        | 受保护属性   |
| D    | show_info     | 公开接口     |
| E    | __update_age  | 私有方法     |

### 2. 机制理解

**2.1** 运行结果：`AttributeError: 'Safe' object has no attribute '__password'`

原因：`__password` 被 Python 执行了 Name Mangling，改名为 `_Safe__password`，原名称不存在。

**2.2** 强制访问方式：
```python
print(s._Safe__password)  # 可以访问，但不符合规范
```

### 3. 最小实践（参考答案）

```python
class Wallet:
    def __init__(self, initial=0):
        self.__balance = initial
    
    def deposit(self, amount):
        if amount > 0:
            self.__balance += amount
            print(f"存款成功: +{amount}")
        else:
            print("存款金额必须大于0")
    
    def withdraw(self, amount):
        if amount > 0 and amount <= self.__balance:
            self.__balance -= amount
            print(f"取款成功: -{amount}")
        else:
            print("取款失败：金额无效或余额不足")
    
    def show_balance(self):
        print(f"当前余额: {self.__balance}")
```

### 4. 扩展思考（要点）

**4.1** Python 选用约定而非强制，是"成人哲学"——相信开发者会遵守规范。好处是灵活、便于调试；代价是无法完全防止误操作。这体现了 Python 的设计哲学：**实用优于严格**。

**4.2** 用户名和密码都应使用私有属性，通过公开接口操作。原因：密码属于敏感数据，直接暴露可被任意读取和修改；封装后可在 setter 中添加加密、校验等逻辑，保证数据安全。
