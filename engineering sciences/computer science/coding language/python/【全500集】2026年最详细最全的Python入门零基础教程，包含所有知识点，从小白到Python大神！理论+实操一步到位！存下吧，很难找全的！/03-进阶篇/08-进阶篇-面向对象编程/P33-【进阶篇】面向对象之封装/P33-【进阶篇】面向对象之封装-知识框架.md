1. 什么是封装？封装要解决什么问题？

3. 私有属性的访问边界在哪里？

## 知识骨架

**封装 = 数据打包 + 逻辑隐藏 + 接口暴露**

- 核心思想：隐藏复杂，暴露简单
- 实现手段：
  - 类：属性和方法的容器（本身就是封装的体现）
  - 私有属性（`self.__属性名`）：限制外部直接访问
  - 公开接口（普通方法）：替代直接访问，可在内部添加验证逻辑
- Name Mangling：Python将`__属性名`改写为`_类名__属性名`，属约定而非强制
- 四大意义：数据安全、隐藏复杂、便于维护、代码清晰

## 关系与依赖

```
类（封装载体）
  │
  ├── 公开属性（可直接访问）→ 基础封装
  │     ↑ 存在问题：数据可被随意修改
  │
  └── 私有属性（__前缀）→ 进阶封装
        ↑ 被保护：外部直接访问报AttributeError
        │
        └── 公开接口（方法）→ 操作私有属性
              ↑ 可添加验证逻辑，保证数据安全
```

三大特性关系：
- 封装：打包 + 保护数据
- 继承：复用（下一P）
- 多态：灵活（再下一P）

## 易错点

- 误以为`__属性名`是绝对私有：Python只做Name Mangling，不做真正禁止
- 混淆公开属性和私有属性的使用场景：敏感数据用私有，普通数据用公开
- 在公开接口中忘记验证逻辑：封装的价值在于可校验
- 强行访问私有属性：`_类名__属性名`可以访问但不推荐
