## 知识骨架

**概念层**

- 类（Class） = 抽象设计图 / 模板，规定对象的特征和行为
- 对象（Object） = 按照类这个模板造出来的具体成品
- 先有类，后有对象

**组成层**

- 类 = 属性（静态特征）+ 行为（动态操作，通过方法实现）
- 实例方法：定义在类中的函数，第一个参数必须是 self
- self：代表调用该方法的对象，由 Python 解释器自动传递

**属性层**

- 类属性：定义在类中方法体外，所有对象共享，通过类或对象访问
- 实例属性：定义在构造函数 __init__ 中，每个对象独有

**生命周期**

- 创建：类名() 实例化 → 自动调用 __init__
- 销毁：对象无引用时 → Python 垃圾回收 → 自动调用 __del__

**语法要素**

- class 关键字 + 大驼峰类名 + 冒号
- pass 空语句占位
- __init__ 前后各两个下划线，构造函数
- __del__ 前后各两个下划线，析构函数

## 关系与依赖

```
函数基础
    ↓
类定义 → class 关键字 + 大驼峰命名法
    ↓
对象创建 → 类名() 实例化
    ↓
实例方法定义 → 第一个参数 self
    ↓
属性设计 → 类属性 vs 实例属性
    ↓
初始化 → __init__ 构造函数自动调用
    ↓
清理 → __del__ 析构函数
```

**依赖链**：

- 实例方法依赖 self → self 依赖对象调用
- 实例属性依赖 __init__ → __init__ 依赖类实例化
- 类属性独立于 __init__ → 定义在类体方法外即可

## 输入 / 输出 / 前置 / 支撑

**输入**

- 类名（大驼峰字符串）

**输出**

- 类定义：创建模板，不执行逻辑
- 对象：类名() 返回一个独立实例，带唯一内存地址
- 方法调用：执行类中定义的函数逻辑

**前置知识**

- 变量与数据类型
- 函数定义与调用
- 缩进与代码块

**支撑工具**

## 易错点

1. **self 误解**：误以为需要手动传参 → 实际上 Python 解释器自动传递
2. **命名混淆**：类名未使用大驼峰 → 与变量/函数命名无法区分
3. **属性归属**：实例属性在 __init__ 外定义 → 变成类属性，所有对象共享
4. **实例化遗漏参数**：__init__ 有参数但实例化时未传 → TypeError
5. **类属性 vs 实例属性 访问**：实例属性试图用类名访问 → AttributeError
6. **pass 滥用**：以为 pass 有特殊功能 → 仅仅是空语句占位
