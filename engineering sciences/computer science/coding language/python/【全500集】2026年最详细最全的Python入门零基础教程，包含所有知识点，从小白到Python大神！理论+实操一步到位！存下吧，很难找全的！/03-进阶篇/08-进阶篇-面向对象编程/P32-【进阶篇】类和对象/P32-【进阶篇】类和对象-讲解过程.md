# P32 讲解过程 - 【进阶篇】类和对象
## 本P定位

本P是Python面向对象编程的入门篇章，核心任务是帮助学习者建立“类和对象”这一核心概念。主要内容包括：面向对象编程思想的引入、类与对象的定义与关系、类的语法与命名规范、对象的创建与实例化、实例方法的定义与self参数、类属性与实例属性的区分、以及`__init__`构造函数的用法。

本P重在夯实概念基础，为后续三个P的面向对象特性（封装、继承、多态）做准备。

## 讲解过程

### 一、课程引入：为什么要学面向对象（0.1-83.4）

**开篇强调重要性（0.1-25.4）**

课程以“面向对象编程是Python中极具核心价值的知识点”开篇。指出它能够把复杂功能拆解成独立对象，让代码更具有结构性、可复用性更好，是后续开发复杂项目或框架的重要基础。

**预告五个小节（25.4-83.4）**

老师提前预告了本章节的完整学习路径，分为五个小节：
1. 类和对象：搞懂核心概念、学会定义类、创建对象、搭建编程基础
2. 封装：掌握隐藏对象内部细节、统一对外接口的方法，让代码更加安全
3. 继承：减少重复编码、提升开发效率
4. 多态：理解同一接口、多种实现的核心思想，让程序更加灵活、可扩展
5. 静态方法和类方法：掌握类的特殊方法，适配更多场景

### 二、核心概念建立：类与对象的关系（87.2-146.8）

**设计图与成品的比喻（87.2-136.9）**

老师使用通俗的比喻解释类和对象的关系：

- **类（Class）**：就像设计图，规定汽车的样子和功能，是抽象的模板，不是真实存在的事物
- **对象（Object）**：就是真正能开的汽车，是按照设计图造出来的成品

老师通过例子说明类的价值：如果同一型号造了100辆汽车，想要给所有汽车添加新功能，只需要修改设计图纸（类），所有对象会自动拥有新功能。这体现了类的复用性和可维护性。

**核心规则强调（423.7-437.3）**

老师明确指出一个关键规则：**先有类，后有对象**。必须先写好设计图纸，才能造出成品，对象是根据类创建出来的具体事物。

### 三、编程思想对比：面向过程 vs 面向对象（151.1-306.3）

**为什么需要面向对象（154.5-165.3）**

在学习类和对象之前，老师先帮助学生分清楚两种编程思想，这样能够更清楚为什么需要使用面向对象。

**洗衣服案例对比（169.5-244.7）**

老师用洗衣服这个生活案例通俗对比两种编程思想：

| 维度 | 面向过程 | 面向对象 |
|------|----------|----------|
| 核心 | 函数 | 对象 |
| 注重 | 步骤 | 谁来做 |
| 特点 | 每一步亲力亲为 | 找对象帮你做事情 |
| 适用场景 | 简单任务 | 复杂、可复用的任务 |
| 比喻 | 手洗衣服：找桶→装水→加洗衣粉→搓洗→漂洗→拧干→晾晒 | 洗衣机洗衣服：放衣服→按启动键→洗衣机自动完成中间步骤→只需晾晒 |

**洗衣机比喻详解（208.2-244.7）**

老师强调面向对象编程的核心是找对象帮你做事情。洗衣机就是一个对象，自带洗涤功能，用户不用关心内部细节，只需要调用它就可以。这种思想以对象为核心，注重的是“谁来做”。

**面向对象的四个核心优势（244.7-306.3）**

1. **代码复用性高**：洗衣机对象可以反复使用，不用每次都重新写洗涤步骤
2. **可维护性强**：想给洗衣机加除菌模式，只需修改洗衣机类的代码，不影响其他部分
3. **可扩展性好**：基于现有对象能够轻松添加新功能（如在洗衣机基础上加烘干功能）
4. **更符合人类思维**：像现实世界一样用对象组织代码，更容易理解

### 四、类的核心组成：属性和行为（311.3-390.9）

**属性（Attributes）**

- 定义：事物的特征，是固定的数据
- 举例：汽车的品牌、颜色、轮子数量
- 作用：描述事物是什么样子的

**行为（Behaviors）**

- 定义：事物能够做的事情，是动态的概念
- 举例：汽车能够启动、加速、刹车
- 实现：通过函数（方法）进行实现

老师强调：设计图纸（类）本身是不能启动的，它只是一个抽象概念、理论，真正实际的是造出来的汽车（对象）。

### 五、类的定义语法（444.1-523.3）

**基本语法（448.5-485.3）**

```python
class Car:  # class关键字 + 类名 + 冒号
    # 类体（自动缩进）
```

**类名命名规范（459.8-517.4）**

老师详细讲解了类名的命名规范：

1. **使用大驼峰命名法（CapWords）**：每个单词的首字母大写
   - 正确示例：`Car`、`Student`、`PhoneNumber`
   - 目的：区分类和变量/函数，能够清晰知道它是一个类

2. **遵循变量命名规则**：
   - 不能是关键字
   - 不能以数字开头
   - 只能由数字、字母、下划线组成

**类体的内容（523.3-536.4）**

类体中可以包含：
- 类的描述信息（可选）：说明类是干什么的、有什么作用、注意事项
- 属性和方法

**pass空语句（607.1-637.8）**

老师演示了最简类定义：

```python
class Car:
    """这是汽车类"""
    pass
```

解释`pass`的作用：
- 表示站位、空语句
- 暂时不写内容，提高可读性
- 好比斗地主时说pass跳过

### 六、对象的创建与实例化（646.6-835.8）

**创建对象语法（656.6-668.4）**

```python
类名()  # 类名 + 小括号
```

- 如果对象需要使用多次，定义变量接收
- 如果对象只需使用一次，直接使用

**实例化演示（673.2-745.3）**

老师演示创建两个汽车对象：

```python
car1 = Car()
print(car1)  # 输出类似：<__main__.Car object at 0x000001EC...>
car2 = Car()
print(car2)  # 输出类似：<__main__.Car object at 0x000008F9...>
```

**内存地址验证（782.2-821.2）**

老师强调打印结果中的关键信息：
- `__main__`：主程序，当前正在执行的程序
- `Car`：类名
- `object`：对象
- `at 0x...`：16进制内存地址

通过对比两个对象的内存地址（`0xEC1C0` vs `0x8F9D0`），验证它们是两个独立的对象。就像不同地址的两个人，是不同的对象。

### 七、实例方法的定义与调用（854.8-1224.3）

**方法的概念（871.3-877.3）**

方法就是定义在类里面的函数，是对象能够执行的操作。今天学习的是**实例方法**，必须由对象调用。

**实例方法的定义（887.5-900.8）**

关键要求：**第一个参数必须是self**

```python
class Car:
    def start(self):
        print("汽车启动了")
    
    def run(self):
        print("汽车正在行驶")
```

**self参数详解（892.7-1117.0）**

老师通过打印`self`参数来解释其含义：

```python
def start(self):
    print(self)  # 打印self参数
```

运行结果显示：
- `self`打印出来的内容类似`<__main__.Car object at 0x...>`
- 与调用它的对象（如`car1`）的打印结果完全相同
- 内存地址也一致

**结论**：
- `self`代表调用该方法的对象
- 不需要手动传参，Python解释器会自动传递
- `self`可以改成其他名称，但只是一种约定，建议使用`self`

**调用方法语法（910.1-918.1）**

```python
car1.start()  # 通过对象名.方法名() 调用
car2.start()
```

**多对象验证（1125.0-1168.9）**

老师演示用不同对象调用同一方法：

```python
car1.start()  # self = car1
car2.start()  # self = car2
```

两次运行的`self`内存地址不同，证明`self`代表的是调用它的那个对象。

**自动传参机制（1168.9-1188.2）**

老师总结：Python解释器会自动把调用实例方法的对象传递给第一个参数`self`，不需要我们手动传参。

### 八、类属性与实例属性（1232.4-1989.8）

**属性的分类**

1. **类属性**：所有对象共享的特征，在类中方法体外定义
2. **实例属性**：每个对象独有的特征，通常在构造函数中定义

#### 8.1 类属性（1241.6-1411.9）

**定义方式**

```python
class Car:
    wheel_count = 4  # 在类中方法体外定义
    
    def start(self):
        print("汽车启动了")
```

**访问方式（1322.5-1397.4）**

类属性可以通过对象和类两种方式访问：

```python
car1 = Car()
car2 = Car()
print(car1.wheel_count)  # 通过对象访问：4
print(car2.wheel_count)  # 通过对象访问：4
print(Car.wheel_count)   # 通过类访问：4
```

**共享特性（1401.8-1408.4）**

老师强调类属性的“共享”含义：
- 所有对象访问类属性，结果都是相同的值
- 通过类名修改类属性，所有对象的属性值都会跟着改变
- 这是共性，不是个性

#### 8.2 实例属性（1411.9-1713.1）

**定义方式一：外部赋值（1450.3-1698.2）**

```python
car1 = Car()
car1.color = "红色"      # 通过对象名.属性名赋值
car1.brand = "保时捷"

car2 = Car()
car2.color = "蓝色"
car2.brand = "宾利"
```

**存在问题（1698.4-1713.1）**

老师指出这种方式的问题：
- 每次创建对象调用方法前都要单独赋值
- 对象多了会多很多行代码
- 麻烦且容易出错

**定义方式二：构造函数`__init__`（1713.1-1989.8）**

老师引入`__init__`方法来解决上述问题：

```python
class Car:
    def __init__(self, color, brand):
        self.color = color
        self.brand = brand
    
    def start(self):
        print(f"{self.color}的{self.brand}启动了")
```

**`__init__`方法特点（1714.5-1827.6）**

1. **方法名固定**：`__init__`前后各两个下划线
2. **第一个参数是self**
3. **后续参数**：需要给实例属性传的参数（如品牌、型号、颜色）
4. **自动调用**：类实例化时会自动调用

老师演示验证自动调用：

```python
class Car:
    def __init__(self, color, brand):
        print("我是__init__方法")
        self.color = color
        self.brand = brand

car1 = Car("红色", "保时捷")
car2 = Car("蓝色", "宾利")
```

运行结果显示两次都打印了“我是__init__方法”，证明类实例化时会自动调用。

**构造函数创建对象（1890.6-1978.9）**

```python
car1 = Car("红色", "保时捷")  # 实例化时自动调用__init__
car1.start()  # 输出：红色的保时捷启动了

car2 = Car("蓝色", "宾利")
car2.start()  # 输出：蓝色的宾利启动了
```

**参数传递流程（1925.0-1978.9）**

老师详细解释参数传递过程：

对于`car1 = Car("红色", "保时捷")`：
1. 类实例化，自动调用`__init__`
2. `"红色"`传递给`color`形参
3. `"保时捷"`传递给`brand`形参
4. `self.color = color`将`"红色"`赋值给实例属性`car1.color`
5. `self.brand = brand`将`"保时捷"`赋值给实例属性`car1.brand`

对于`car2 = Car("蓝色", "宾利")`：
- 同样流程，区别是参数值不同
- 最终`car2.color = "蓝色"`，`car2.brand = "宾利"`

**构造函数的优势（1990.5-2006.5）**

- 创建对象时直接传入品牌和颜色，就能自动初始化实例属性
- 代码更加简洁
- 逻辑更加清晰

### 九、析构函数`__del__`简介（2014.1-2033.9）

老师简要介绍析构函数：

```python
class Car:
    def __init__(self, brand):
        self.brand = brand
    
    def __del__(self):
        print("对象被销毁")
```

**特点**：
- `__del__`是Python的特殊方法
- 当对象被销毁、内存即将回收之前，Python解释器会自动调用
- 一般用于释放资源，如关闭文件、断开网络连接等

老师说明这只是简单介绍，让大家了解有这个方法存在。

## 本P结论

1. **类和对象的关系**：类是抽象的设计图/模板，对象是按照类这个设计图造出来的具体成品。类用于批量创建具有相同特征的对象，先有类后有对象。

2. **面向对象vs面向过程**：面向过程注重步骤、以函数为核心；面向对象注重谁来做、以对象为核心。面向对象具有代码复用性高、可维护性强、可扩展性好、更符合人类思维四大优势。

3. **类的定义语法**：`class 关键字 + 类名（遵循大驼峰命名法）+ 冒号`，类体包含描述信息和属性/方法。可用`pass`作为空语句占位。

4. **对象的创建**：`类名()` 创建对象，也叫实例化。通过打印对象的内存地址可以验证不同对象是独立的实例。

5. **实例方法**：定义在类中的函数，必须由对象调用。第一个参数必须是`self`，代表调用该方法的对象，由Python解释器自动传递。

6. **类属性vs实例属性**：
   - 类属性在类中方法体外定义，所有对象共享
   - 实例属性在构造函数`__init__`中定义，每个对象独有

7. **`__init__`构造函数**：`__init__`前后各两个下划线，在类实例化时自动调用，用于初始化实例属性。创建对象时直接传参即可自动完成属性初始化。

8. **`__del__`析构函数**：了解即可，用于对象销毁前释放资源。
