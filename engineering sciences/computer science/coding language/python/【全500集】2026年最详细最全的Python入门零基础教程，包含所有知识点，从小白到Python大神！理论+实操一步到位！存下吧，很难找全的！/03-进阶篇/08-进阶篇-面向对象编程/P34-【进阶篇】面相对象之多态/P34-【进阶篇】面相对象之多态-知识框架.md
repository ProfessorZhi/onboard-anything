# P34 知识框架 - 【进阶篇】面相对象之多态
## 本P位置

- 课程阶段：进阶篇·面向对象编程
- 承接关系：封装 → 继承 → **多态**（三大特性最后一站）
- 前置依赖：P32封装、P33继承已掌握
- 后续应用：P35异常处理等场景会用到多态思维

## 核心问题

**同一动作（方法名），不同对象执行不同逻辑，如何实现？**

- 动作是接口，实现是私有逻辑
- 调用方只认接口，不问对象类型
- 结果由对象自己的实现决定

## 知识骨架

```
多态（Polymorphism）
├── 定义：同一方法名 + 不同对象的不同实现 = 不同结果
├── 核心思想：统一接口、多种实现
└── 两种实现路径
    ├── 路径A：鸭子类型（Duck Typing）—— Python特有
    │   ├── 约束：有同名方法即可，无需继承
    │   ├── 机制：运行时检查方法是否存在
    │   └── 典型场景：无关类共享同一接口（如支付类群）
    └── 路径B：继承 + 方法重写
        ├── 约束：子类必须重写父类同名方法
        ├── 机制：父类定义规范，子类提供实现
        └── 典型场景：类有明确层级关系（如动物体系）
```

## 关系与依赖

| 概念 | 与多态的关系 |
|------|-------------|
| **封装** | 提供对外统一接口，是多态调用方依赖的界面 |
| **继承** | 路径B的实现基础，路径A不需要继承 |
| **统一方法名** | 多态成立的核心前提，无同名方法则无法统一调用 |
| **方法重写** | 路径B的必要条件，子类覆盖父类同名方法 |

```
封装（隐藏实现）
        ↓ 提供统一接口
继承（建立层级，可选）
        ↓ 支持方法重写
多态（统一调用 + 差异化结果）
```

## 输入 / 输出 / 前置 / 支撑

**前置知识**
- 类与对象基本概念
- 实例方法定义与调用
- 继承语法（路径B需要）

**输入**
- 多个对象实例（WechatPay、AliPay、BankCardPay 等）
- 统一方法名（pay、make_sound、eat 等）
- 参数值（money、数额等）

**输出**
- 各对象执行自身实现后的差异化结果
- 同一调用语句产生不同输出

**支撑要素**
- 统一调用函数（pay_order、animal_speak、master.feed 等）
- 统一的形参引用（obj、animal、pet）

## 易错点

1. **传入对象没有同名方法** → AttributeError，运行时才暴露
   - 例：pay_order("支付宝", 50) → 字符串无pay方法

2. **用 isinstance 做类型判断** → 破坏多态灵活性，违反统一接口原则

3. **父类方法写成 pass** → 子类可能忘记重写，导致调用父类空逻辑

4. **Duck typing 缺少显式约束** → 传错对象类型不易发现，无编译期检查

5. **混淆两种实现路径的适用场景** → 无层级关系时强行用继承，增大耦合
