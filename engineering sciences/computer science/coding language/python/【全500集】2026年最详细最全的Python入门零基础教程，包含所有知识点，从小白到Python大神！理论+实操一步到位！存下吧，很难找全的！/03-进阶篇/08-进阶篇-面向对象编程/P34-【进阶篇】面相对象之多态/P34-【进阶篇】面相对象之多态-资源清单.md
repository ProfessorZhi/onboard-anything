# P34 资源清单 - 【进阶篇】面相对象之多态
## 资源汇总说明

本P为Python面向对象编程的进阶概念教学，主要依赖Python语言本身和IDE环境进行代码演示。未涉及外部工具、第三方库或在线平台。以下为基于字幕和上下文推断可能涉及的资源。

---

## 1. Python 语言

- **类型**：编程语言 / 运行时环境
- **在本P中的作用**：全程用于演示多态的两种实现方式（鸭子类型、继承+方法重写），是所有示例代码的运行环境
- **来源**：字幕直接提到（课程全程使用Python进行演示）
- **建议练习方向**：使用Python 3.8+版本自行实现多态场景，尝试不同的类设计组合

---

## 2. Python 官方文档

- **类型**：文档 / 官方资料
- **在本P中的作用**：关于类定义、方法重写、继承机制的权威参考。当学员对某些语法细节有疑问时可查阅
- **来源**：基于上下文推断（Python教学必然涉及的官方参考资料）
- **建议练习方向**：查阅官方文档中关于类（Classes）和继承（Inheritance）的章节，加深对多态底层的理解

---

## 3. 本地 IDE / 代码编辑器

- **类型**：开发工具 / 软件
- **在本P中的作用**：老师全程在IDE中进行代码演示，学员需使用IDE编写、运行和调试多态相关代码
- **来源**：字幕直接提到（“全程在IDE中进行代码演示”）
- **建议练习方向**：配置好Python运行环境的IDE（如VS Code、PyCharm等），亲手敲入并运行课堂示例代码

---

## 4. 课堂示例代码（代码模板）

- **类型**：代码片段 / 示例集合
- **在本P中的作用**：提供三个可直接运行的模板案例，分别是支付系统（鸭子类型）、动物叫声系统（继承+重写）、宠物喂养系统（继承+重写实战）
- **来源**：字幕直接提到（老师全程手写演示）
- **建议练习方向**：将示例代码迁移到自己的项目中，或基于模板改编出新的多态场景（如交通工具系统、电器开关系统）

---

## 5. 常见多态设计模式参考

- **类型**：代码模式 / 设计思路
- **在本P中的作用**：提供两种多态实现路径的典型应用场景：路径A（鸭子类型）适合无层级关系的独立类集合，路径B（继承+重写）适合有层级关系的类族
- **来源**：基于上下文推断（课程核心知识点，属于OOP经典设计模式范畴）
- **建议练习方向**：对比两种实现路径的优劣，尝试在同一个项目中同时使用两种方式，理解各自的适用场景
