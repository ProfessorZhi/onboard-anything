## 一、结构识别

### 题目 1.1：路径判断

以下代码体现了哪种多态实现路径？请说明判断依据。

```python
class Payment:
    pass

class WechatPay:
    def pay(self, money):
        print(f"微信支付 {money} 元")

class AliPay:
    def pay(self, money):
        print(f"支付宝支付 {money} 元")

def checkout(obj, money):
    obj.pay(money)

checkout(WechatPay(), 100)
checkout(AliPay(), 100)
```

**考察点**：鸭子类型 vs 继承+重写的识别

## 五、最小替代练习

### 练习：支付网关多态实现

本 P 无外部依赖，此练习以**纯 Python** 实现一个接近生产场景的支付网关。

**需求**：
- 模拟三种支付方式：微信支付、支付宝、银行卡
- 统一接口：`pay(amount: float) -> bool`
- 实现一个函数 `process_payment(payment_obj, amount)`，对任何实现了 `pay` 方法的对象都能正常工作
- 至少验证两种支付方式的调用

**验收标准**：
- 同一行代码 `process_payment(pay_obj, 99.9)` 对不同支付对象产生不同内部行为
- 不使用 isinstance 进行类型判断
- 代码结构清晰，便于扩展第四种支付方式
