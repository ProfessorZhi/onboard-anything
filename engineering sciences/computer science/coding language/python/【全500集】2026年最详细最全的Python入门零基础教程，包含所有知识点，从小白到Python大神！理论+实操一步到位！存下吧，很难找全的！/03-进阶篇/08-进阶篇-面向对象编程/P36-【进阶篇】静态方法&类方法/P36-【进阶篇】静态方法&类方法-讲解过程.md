# P36 讲解过程 - 【进阶篇】静态方法&类方法
## 本P定位

本P位于Python面向对象进阶篇，承接实例方法的基础概念，引入两类特殊方法：静态方法（`@staticmethod`）和类方法（`@classmethod`）。教学目标是通过对比讲解，让学习者理解为什么需要这两种方法、如何定义、以及何时该用哪种方法。本P时长约28分钟，是从「实例方法」向「类级别方法」的概念升级。

---

## 讲解过程

### 一、引入背景：实例方法的局限

**时间区间：约1分-4分**

课程开头，老师先回顾面向对象三大特性（封装、继承、多态），指出这些特性围绕类和实例展开，用到的方法基本是实例方法。实例方法的核心特征：

- 第一个参数必须是 `self`
- `self` 代表调用该方法的对象本身
- 调用方式固定：必须先创建对象，再通过对象调用
- 核心用途：操作**实例属性**（例如银行账户类的取款、存款方法通过 `self` 访问余额）

老师以 `Person` 类为例演示实例方法的调用：

```python
class Person:
    def __init__(self, name):
        self.name = name
    
    def greet(self):
        return f"你好呀，我是{self.name}"
```

然后演示直接用类名调用会报错（`Person.greet()` 不可行），说明实例方法依赖对象。

**核心问题引出**：如果一个功能根本不需要访问实例属性，只是一个简单工具函数（如计算圆的面积、验证手机号格式），却还要创建对象才能调用，就很麻烦。静态方法和类方法就是为了解决这个问题而生。

### 二、静态方法讲解

**时间区间：约4分-13分**

#### 2.1 概念引入

老师用**工具箱里的扳手**作比喻：

- 扳手是工具箱的一部分
- 使用扳手时不需要关心工具箱本身的状态（颜色、大小）
- 只需要扳手能拧螺丝（传入参数）就能工作

静态方法就是这种「属于类，但不依赖类或实例任何状态」的独立工具函数。

#### 2.2 定义规则（两个关键）

1. **必须加装饰器** `@staticmethod` —— 这是标记静态方法的标志，不能省略
2. **方法内没有 `self` 或 `cls` 参数** —— 完全靠传入的参数工作

#### 2.3 调用方式

- **推荐**：直接通过类名调用 `类名.方法名()`，无需创建对象
- 次选：通过对象调用（不推荐，容易让人误以为方法与对象有关）

#### 2.4 代码演示：几何工具类

老师定义了一个 `GeometryTool`（几何工具）类来计算圆的面积：

```python
class GeometryTool:
    @staticmethod
    def circle_area(r):
        return 3.14 * (r ** 2)
```

演示两种调用方式：

```python
# 方式一：通过类名调用（推荐）
result = GeometryTool.circle_area(2)  # 返回 12.56
print(result)

# 方式二：通过对象调用（不推荐但可行）
geo = GeometryTool()
result2 = geo.circle_area(3)  # 返回 28.26
print(result2)
```

老师特别强调：方法体内没有用到 `self`，说明这个方法本身不需要对象状态，加 `@staticmethod` 装饰器后就可以通过类名直接调用，不必为调用一个简单工具函数而专门创建对象。

#### 2.5 适用场景总结

静态方法适合以下三种情况：

1. **实现通用工具功能**：数学计算（圆面积、矩形面积）、格式验证（手机号、邮箱）、数据转换（字符串转日期）、单位换算
2. **不需要访问类属性和实例属性**：功能完全依赖传入的参数，与类/实例状态无关
3. **归类独立函数**：把相关的工具函数放在同一个类里，让代码结构更清晰（例如把所有几何计算函数放进 `GeometryTool` 类）

### 三、类方法讲解

**时间区间：约13分-22分**

#### 3.1 概念引入

老师用**班级管理函数**作比喻：

- 类方法负责处理所有学生共享的公共信息（如全班人数、班级名称、班主任姓名）
- 而不是某个学生的个人信息（姓名、成绩、身高——这些属于实例属性）

类方法是和类本身绑定的方法，专门用来操作**类属性**（所有对象共享的属性）。

#### 3.2 定义规则（两个关键）

1. **必须加装饰器** `@classmethod`
2. **第一个参数通常是 `cls`** —— 代表类本身，通过 `cls` 可以访问或修改类属性

#### 3.3 `cls` 的特性

- 不需要手动传参，Python 自动把类作为参数传递进去
- 参数名虽然可以用其他名字，但建议用 `cls`（class 的简写，可读性更好）

#### 3.4 调用方式

- **推荐**：直接通过类名调用 `类名.方法名()`
- 次选：通过对象调用（但 `cls` 依然代表类本身，与对象无关）

#### 3.5 代码演示：Person 类操作类属性

老师定义人类的平均寿命为类属性 `average_age = 75`：

```python
class Person:
    average_age = 75  # 类属性，所有人共享
    
    @classmethod
    def get_average(cls):
        return cls.average_age
    
    @classmethod
    def set_average(cls, age):
        cls.average_age = age
```

演示调用：

```python
# 通过类名调用
print(Person.get_average())  # 75
Person.set_average(100)     # 修改为 100
print(Person.get_average())  # 100

# 验证修改对所有对象生效
p1 = Person()
p2 = Person()
print(p1.average_age)  # 100
print(p2.average_age)  # 100
```

#### 3.6 为什么不用实例方法操作类属性？

老师先演示了用实例方法 `self` 访问/修改类属性的错误方式：

```python
# 错误示范：self.set_average(100) 只是创建了实例属性，没有修改类属性
self.average_age = age  # 这是在创建实例属性，不是修改类属性
```

验证方式：创建第二个对象时发现其 `average_age` 仍是原来的值（75），说明类属性没有被修改成功。

然后老师演示用 `Person.average_age = age` 直接修改，但指出弊端：代码依赖类名，如果类名变化需要多处修改，可维护性差。

**结论**：操作类属性应该用类方法，通过 `cls` 访问/修改类属性，可维护性更好。

#### 3.7 核心用途

1. **访问或修改类属性**：获取全班人数、更新产品默认价格、重置统计数据
2. **工厂方法**：封装实例化逻辑，提供多种创建对象的方式（例如根据手机号、身份证号创建不同类型的用户对象）
3. **类级别的初始化**：需要用到类本身信息的功能，例如从配置文件读取数据初始化类属性

### 四、三种方法对比表格

**时间区间：约22分-26分**

老师用表格对比三种方法的维度：

| 维度 | 静态方法 | 类方法 | 实例方法 |
|------|---------|--------|---------|
| **第一个参数** | 无参数限制（不需要 `self`/`cls`） | `cls`，自动接收类本身 | `self`，自动接收对象本身 |
| **装饰器** | `@staticmethod` | `@classmethod` | 不需要装饰器 |
| **能否访问类属性** | 可以，但不推荐（无必要） | 可以（通过 `cls.属性名`） | 可以，但不推荐 |
| **能否访问实例属性** | 不能 | 不能 | 能 |
| **推荐调用方式** | `类名.方法名()` | `类名.方法名()` | 必须 `对象名.方法名()` |
| **核心用途** | 独立工具功能（计算、验证、转换） | 操作类属性、工厂方法 | 操作实例属性 |

老师特别说明：类属性（`average_age`）三种方法其实都可以访问，但类方法通过 `cls` 访问可维护性最好。实例属性只有实例方法可以访问。

### 五、记忆口诀

**时间区间：约26分-27分**

老师给出了一个覆盖90%以上场景的记忆口诀：

> - 要操作**实例属性** → 用**实例方法**（带 `self` 参数）
> - 要操作**类属性** → 用**类方法**（带 `cls` 参数）
> - 什么属性都不用操作，只靠参数就能干活 → 用**静态方法**

写代码时先判断：要不要访问属性？访问的是实例属性还是类属性？就能快速决定用哪种方法。

### 六、总结与练习建议

**时间区间：约27分-28分**

老师总结本节核心：

1. **静态方法**：本质是类里的独立工具函数，和类、实例都无关。加 `@staticmethod`，没有 `self`/`cls`，优先类名调用，适合通用工具功能（计算、格式验证等）
2. **类方法**：和类绑定，只和类有关。加 `@classmethod`，第一个参数是 `cls`，优先类名调用，适合操作类属性、实现工厂方法
3. **调用优先级**：静态方法和类方法优先通过类调用，不用创建对象；实例方法必须创建对象，通过对象调用
4. **核心区别**：静态方法啥都不依赖，类方法只依赖类，实例方法只依赖实例
5. **设计意义**：让类的功能划分更清晰——独立工具函数归静态方法，类相关操作归类方法，实例相关操作归实例方法，代码结构更规整

**课后练习建议**：
- 定义一个日期工具类，用静态方法实现判断闰年、计算两个日期天数差
- 定义一个学生类，用类方法统计全校学生总数、重置学生编号计数器

## 本P结论

1. **静态方法**（`@staticmethod`）是独立于类和实例的纯工具函数，不需要 `self`/`cls` 参数，通过类名直接调用，适合数学计算、格式验证、数据转换等通用工具功能
2. **类方法**（`@classmethod`）与类绑定，第一个参数 `cls` 自动接收类本身，用于操作所有对象共享的类属性，也适合实现工厂方法（封装多种实例化逻辑）
3. **实例方法**（`self`）与对象绑定，用于操作实例属性，这是它与前两者的根本区别
4. 三种方法的核心作用是**让类的功能划分更清晰**，避免为调用简单工具函数而被迫创建对象，提升代码可维护性
