---

## 知识骨架

三种方法构成类内方法的完整光谱：

| 方法 | 装饰器 | 第一个参数 | 访问能力 | 推荐调用 |
|------|--------|-----------|----------|----------|
| 静态方法 | `@staticmethod` | 无 | 不能访问任何属性 | `类名.方法()` |
| 类方法 | `@classmethod` | `cls` | 类属性 | `类名.方法()` |
| 实例方法 | 无 | `self` | 实例属性 + 类属性 | `对象.方法()` |

核心记忆口诀：
- 动实例属性 → 实例方法（`self`）
- 动类属性 → 类方法（`cls`）
- 什么属性都不动 → 静态方法

---

## 关系与依赖

```
实例方法 (self) ──依赖──▶ 实例对象
    │
    └──可访问──▶ 类属性

类方法 (cls) ──依赖──▶ 类本身
    │
    └──可修改──▶ 类属性

静态方法 ──不依赖──▶ 类 / 实例
    │
    └──纯参数驱动
```

静态方法是类的「外接工具箱」；类方法是类的「自我管理层」；实例方法是对象的「行为能力」。

## 易错点

1. **忘记加装饰器**：静态方法和类方法必须分别加 `@staticmethod` 和 `@classmethod`，不写则变成普通实例方法
2. **静态方法误用 `self`**：静态方法内没有任何 `self` 或 `cls`，只有传入参数
3. **类方法用 `self` 改类属性**：`self.average_age = age` 创建的是实例属性，不是类属性，类属性未被修改
4. **用类名调用实例方法**：`Person.method()` 报错，实例方法必须通过对象调用
5. **`cls` 参数名可改但不推荐**：可用其他名字，但可读性差，社区约定用 `cls`
