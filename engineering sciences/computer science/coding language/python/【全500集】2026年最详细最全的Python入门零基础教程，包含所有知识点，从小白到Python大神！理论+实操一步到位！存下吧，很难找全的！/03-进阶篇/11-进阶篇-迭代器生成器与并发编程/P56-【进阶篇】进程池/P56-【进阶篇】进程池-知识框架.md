# P56 知识框架 - 【进阶篇】进程池
## 本P位置

Python并发编程进阶篇。承接进程基础（Process类、进程属性、进程通信），引入进程池解决批量任务处理效率问题。为后续多进程并发应用打基础。

## 核心问题

如何高效处理大量任务？逐个创建进程开销大，进程池通过复用固定数量进程，避免频繁创建销毁，实现批量任务高效并行处理。

## 知识骨架

进程池核心概念：复用进程、固定数量、任务排队分发。

Pool类四个关键方法：apply_async异步提交任务、close关闭池、join等待完成、get获取结果。

CPU核数决定最优进程数：multiprocessing.cpu_count()获取。

同步vs异步：apply_async异步提交并行执行，apply同步提交串行执行。

进程vs线程选型：CPU密集型用进程/进程池，IO密集型用线程。

## 关系与依赖

multiprocessing模块（标准库）→ 提供Pool类、cpu_count()、Queue类

Process类基础 → 进程池构建在其之上

Pool实例化 → 控制进程复用

apply_async → 异步提交任务到进程池

get() → 从apply_async返回对象中提取结果

close() → 在join()之前调用，停止接受新任务

join() → 阻塞主进程直到所有任务完成

CPU核数 → 决定Pool参数N的最优值

## 输入 / 输出 / 前置 / 支撑

前置知识：Process类使用、进程属性、进程间通信Queue

输入：任务函数、可迭代参数列表、进程数N

输出：各任务返回值列表

进程数建议：N=cpu_count()，通常等于CPU核数

## 易错点

main函数必须加if __name__ == '__main__'保护，否则子进程可能重复导入。

apply_async的args参数单个元素必须加逗号：args=(i,)。

close()和join()必须成对使用且顺序固定：先close再join。

get()会阻塞等待该任务完成，获取结果前确保任务已执行。

apply无下划线是同步提交，会阻塞等待，失去进程池并行意义。

进程数过多导致频繁切换反而降低效率。

## 外部资源与对象

multiprocessing：Python标准库，进程编程核心模块，提供Process、Pool、Queue、cpu_count等

multiprocessing.Pool：进程池核心类，管理固定数量进程的复用

multiprocessing.cpu_count()：查看本机CPU核数，用于设置Pool参数

multiprocessing.Queue：进程间通信队列，本P未详述但作为进程通信方式存在
