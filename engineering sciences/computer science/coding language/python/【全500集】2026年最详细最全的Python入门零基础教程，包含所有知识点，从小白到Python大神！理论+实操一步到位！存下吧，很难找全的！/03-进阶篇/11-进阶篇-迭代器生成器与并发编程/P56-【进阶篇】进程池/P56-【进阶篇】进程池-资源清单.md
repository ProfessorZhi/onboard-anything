# P56 资源清单 - 【进阶篇】进程池
## 一、标准库与模块

### multiprocessing
- **类型**：Python 标准库
- **在本 P 中的作用**：进程编程的核心模块，提供 Process 类、Pool 类、Queue 类、cpu_count() 等进程相关功能。本 P 所有代码都基于该模块展开。
- **来源**：字幕直接提到
- **建议方向**：阅读官方文档中关于进程池的完整 API 说明；对比不同操作系统下的进程行为差异；探索模块中的其他工具类（如 Manager、Pipe 等）

### time
- **类型**：Python 标准库
- **在本 P 中的作用**：代码演示中用于 time.sleep(2) 模拟计算密集型任务的耗时，帮助直观理解进程池分组处理的效果。
- **来源**：基于上下文推断（代码演示中使用）
- **建议方向**：使用 time.time() 或 time.perf_counter() 精确测量代码执行时间；对 sleep 参数进行对比实验（如 0.5 秒、1 秒、2 秒）

## 二、类与对象

### multiprocessing.Pool
- **类型**：类（multiprocessing 模块核心类）
- **在本 P 中的作用**：进程池的核心类，通过实例化创建进程池并管理固定数量进程的复用。实例化时传入的参数 N 决定最大并发进程数。
- **来源**：字幕直接提到
- **建议方向**：尝试使用不同的进程数（N=1、N=CPU核数、N=2倍CPU核数）观察性能变化；查阅官方文档了解 Pool 的其他方法（如 map、imap、map_async 等）

### multiprocessing.Queue
- **类型**：类（multiprocessing 模块）
- **在本 P 中的作用**：进程间通信的队列类。本 P 在知识框架中作为进程通信方式提及，用于实现多个生产进程和消费进程之间的通信。
- **来源**：字幕直接提到（在知识框架和课后练习中提及）
- **建议方向**：结合进程池实现生产者-消费者模式；对比 multiprocessing.Queue 与 queue.Queue（线程队列）的区别

## 三、方法与函数

### multiprocessing.cpu_count()
- **类型**：函数
- **在本 P 中的作用**：获取当前 CPU 核心数，用于设置 Pool 实例化时的参数 N。老师建议将参数设为 cpu_count() 的返回值以充分利用多核。
- **来源**：字幕直接提到
- **建议方向**：编写跨平台代码，自动适配不同机器的 CPU 核数；在不同设备上运行该函数，观察输出差异

### apply_async()
- **类型**：Pool 实例方法
- **在本 P 中的作用**：异步提交任务的核心方法。主进程不等待任务完成就继续执行后续代码，效率高。参数一为目标函数，参数二为元组形式的参数列表。
- **来源**：字幕直接提到
- **建议方向**：结合回调函数（callback 参数）实现任务完成后的自动处理；尝试传递多个参数或关键字参数

### apply()（无下划线）
- **类型**：Pool 实例方法
- **在本 P 中的作用**：同步提交任务的方法，与 apply_async 相对。老师强调其会阻塞等待每个任务完成，失去进程池并行处理的意义。
- **来源**：字幕直接提到（作为对比项）
- **建议方向**：对比 apply 和 apply_async 的执行时间差异；理解同步与异步在并发编程中的权衡

### close()
- **类型**：Pool 实例方法
- **在本 P 中的作用**：关闭进程池，不再接受新的任务提交。必须先于 join() 调用。
- **来源**：字幕直接提到
- **建议方向**：了解未调用 close() 直接退出的后果；在异常处理场景下使用 try-finally 确保 close() 被执行

### join()
- **类型**：Pool 实例方法
- **在本 P 中的作用**：阻塞主进程，等待进程池中所有任务执行完毕后再继续。必须在 close() 之后调用。
- **来源**：字幕直接提到
- **建议方向**：使用 join(timeout) 尝试超时控制；结合异常处理理解阻塞等待的场景

### get()
- **类型**：apply_async 返回对象的方法
- **在本 P 中的作用**：获取异步任务的返回结果。会阻塞等待该任务完成，直到拿到结果才返回。
- **来源**：字幕直接提到
- **建议方向**：处理 get() 可能抛出的异常；在调用 get() 前通过 wait() 或 ready() 检查任务状态

## 四、Python 语法与惯用法

### if __name__ == '__main__'
- **类型**：Python 入口保护惯用法
- **在本 P 中的作用**：防止 Windows/macOS 等平台上的子进程重复导入模块导致问题。老师在易错点中强调必须使用此保护。
- **来源**：字幕直接提到（易错点中强调）
- **建议方向**：在不同操作系统上测试不加保护的后果；养成多进程编程中使用该惯用法的习惯

### args=(i,) 单元素元组逗号规则
- **类型**：Python 语法规则
- **在本 P 中的作用**：apply_async 的 args 参数必须以元组形式传递，单个元素时末尾必须加逗号，否则会被当作非元组对象处理。
- **来源**：字幕直接提到（易错点中强调）
- **建议方向**：对比 args=(i) 和 args=(i,) 的不同行为；练习传递多个参数的情况
