# P56 讲解过程 - 【进阶篇】进程池
## 本P定位

本P是Python并发编程的进阶内容，承接前面关于进程的基础知识（Process类、进程属性、进程通信），重点讲解进程池（Pool类）的概念、优势和使用方法。通过实例演示帮助学生理解进程池如何高效处理大量任务，并明确进程与线程的选型场景。

## 讲解过程

### 1. 引入进程池的必要性

**问题背景铺垫（3.5-48.9秒）**

老师先指出如果需要创建几十个甚至上百个进程，逐个创建会非常繁琐，而且每个进程都要占用独立的内存和CPU资源，造成浪费。

**核心概念引入（48.9-118.7秒）**

- 进程池的核心思想是"复用进程"，预先创建固定数量的进程，这些进程会循环使用来处理不同的任务。
- 老师用了两个生动的比喻：
  - 工厂生产线：预先开三条生产线（即三个进程），十个任务来时三条线同时处理，处理完一个任务后立即接下一个，不用为每个任务新建生产线。
  - 接水例子：三个水龙头接十桶水，每次可以同时接三桶，其中一桶接完就补上新的，直到全部完成。

**关键机制说明（78.9-118.7秒）**

- Pool实例化时传入的参数N通常等于CPU核数，以充分利用多核。
- 任务提交后，进程池会分配空闲进程处理。
- 任务处理完成后，进程不会销毁，而是返回进程池等待下一个任务。
- 所有任务执行完后才关闭进程池。
- 这样避免了频繁创建和销毁进程的开销，大幅提升处理大量任务的效率。

### 2. Pool类的API介绍

**类的实例化（118.7-139.1秒）**

从multiprocessing模块导入Pool类，通过小括号实例化创建进程池。参数N表示最大进程数，即同时执行的任务数，建议设置为CPU核数。

**核心方法讲解（153.2-190.8秒）**

老师逐一介绍四个常用方法：

- apply_async()：异步提交任务的方法，主进程不等待继续执行，效率高。参数一是target即要执行的任务函数，参数二是args以元组形式传参（单个元素时末尾必须加逗号）。
- close()：关闭进程池，不再接受新的任务。
- join()：阻塞主进程，等待进程池所有任务完成，必须在close方法之后调用。
- get()：用于获取异步任务的返回结果，通过apply_async的返回值调用。

### 3. 代码演示：处理七个计算任务

**需求说明（196.2-227.4秒）**

用进程池处理七个计算任务，每个任务耗时约两秒。进程池大小设为三，模拟三核CPU环境。

**任务函数定义（244.6-299.0秒）**

```python
def calculate(n):
    print(f"处理任务{num}，乘以三后变成了{num*3}")
    time.sleep(2)
    return num * 3
```

函数功能：接收一个数字，打印处理信息，延时两秒模拟计算密集型任务耗时，然后返回计算结果。

**进程池创建与任务提交（318.3-476.3秒）**

```python
if __name__ == '__main__':
    from multiprocessing import Pool
    
    p = Pool(3)  # 最大进程数为3
    result = []
    
    for i in range(7):
        res = p.apply_async(calculate, args=(i,))
        result.append(res)
```

老师强调必须从multiprocessing模块导入Pool类。循环中通过apply_async异步提交七个任务，传入calculate函数和参数i，apply_async返回结果对象存入列表保存。

**问题演示与排查（482.2-573.4秒）**

提交完任务后调用p.close()关闭进程池。

第一次运行（无join时）：
```python
print("\n所有任务的返回结果：")
for res in result:
    print(res)
```

发现问题：打印出的result列表每个元素是Pool类的apply_async result对象，不是具体数值。老师指出需要通过get方法获取具体数据。

第二次运行后：
```python
for res in result:
    print(res.get())
```

可以获取数据，但结果仍然混乱——"所有任务的返回结果"先打印出来，而处理任务的输出反而在后。这是因为没有等待所有任务完成就执行了后面的代码。

**关键步骤：join方法（587.9-598.0秒）**

```python
p.close()
p.join()  # 等待所有任务完成后才继续
print("\n所有任务的返回结果：")
for res in result:
    print(res.get())
```

老师强调join方法必须在close方法之后调用，用于阻塞主进程等待所有任务完成。之后再遍历获取结果，才能得到正确的输出。

**运行结果分析（598.0-810.1秒）**

- 三个进程同时处理三个任务，每个任务耗时2秒。
- 第一轮处理任务0、1、2（2秒）。
- 第二轮处理任务3、4、5（2秒）。
- 第三轮处理任务6（2秒）。
- 总耗时约6秒。

对比单进程串行处理：需要7×2=14秒。进程池效率提升一倍多。

老师补充：如果任务耗时设为5秒，三轮分别是5秒、5秒、5秒，总计15秒，验证了分组处理的逻辑。

### 4. CPU核数查看方法

**multiprocessing.cpu_count()（848.8-877.2秒）**

```python
print(multiprocessing.cpu_count())
```

运行输出18，说明该电脑CPU核数为18。老师建议将Pool参数设为cpu_count的返回值。同时提醒进程数过多会导致进程切换频繁，反而降低效率。

### 5. 同步与异步的区别

**概念讲解（892.7-971.5秒）**

老师指出两个注意点：

apply_async是异步提交：主进程不会等待单个任务完成，而是继续提交所有任务，效率高。

apply_async是异步提交：主进程不会等待单个任务完成，而是继续提交所有任务，效率高。

apply（无下划线）是同步提交：主进程会等待每一个任务完成后再提交下一个，相当于串行执行，失去进程池意义。

老师用生活实例区分：
- 同步：先烧开水再泡面，有先后顺序。
- 异步：边烧水边追剧，可以同时进行。

### 6. get方法的阻塞特性

**（975.8-1002.9秒）**

```python
res.get()
```

get方法会阻塞等待任务完成，直到拿到结果才返回。

## 本P结论

1. **进程池的核心价值**：预先创建固定数量的进程，通过复用机制避免频繁创建销毁进程的开销，适合处理大量CPU密集型任务。
2. **Pool类的使用规范**：实例化时设置进程数N（建议等于CPU核数），通过apply_async异步提交任务，close关闭进程池，join等待所有任务完成，get获取返回结果。
3. **同步与异步的区别**：apply_async是异步提交（推荐），主进程不等待继续执行；apply是同步提交，会阻塞等待每个任务完成，失去进程池并行处理的意义。
4. **IO密集型选线程，CPU密集型选进程**：爬虫、文件处理、网络服务等IO密集型任务用线程；大量计算、数据分析、机器学习等CPU密集型任务用进程或进程池，可充分利用多核CPU算力。
