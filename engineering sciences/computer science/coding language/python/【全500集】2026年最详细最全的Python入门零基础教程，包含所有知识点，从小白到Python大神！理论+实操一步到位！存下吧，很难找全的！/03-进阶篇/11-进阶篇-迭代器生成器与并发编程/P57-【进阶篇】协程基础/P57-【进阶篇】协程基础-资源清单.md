### 1. `yield` 关键字

- **类型**：Python 内置语法关键字
- **在本P中的作用**：
  - 将普通函数转变为生成器函数，使其返回生成器对象（即协程对象）
  - 在函数执行过程中标记「暂停点」：执行到 `yield` 时，保存当前局部状态（变量值、执行位置），让出 CPU 执行权，暂停函数运行
  - 配合 `next()` 实现任务状态的保存与恢复，是本P演示协程底层原理的核心语法
- **来源**：字幕直接提到
- **建议方向**：
  - 梳理 `yield` 与普通 `return` 的区别（单次返回 vs 多次暂停/恢复）
  - 练习带 `yield` 的生成器函数，观察调用函数体与调用 `next()` 的执行时机差异
  - 进一步了解 `yield` 的值传递用法（`yield value` 和 `send(value)`），为理解 async/await 打基础
  - 阅读 Python 官方文档：[Generator Types](https://docs.python.org/3/reference/expressions.html#yield-expressions)

---

### 2. `next()` 函数

- **类型**：Python 内置函数
- **在本P中的作用**：
  - 驱动生成器对象（协程对象）从上一次 `yield` 暂停处恢复执行，直至遇到下一个 `yield` 再次暂停
  - 在本P的演示中承担「任务切换器」的角色：手动调用 `next(g1)` / `next(g2)` 实现两个任务的交替执行
  - 揭示协程「主动切换」的底层逻辑：切换完全由程序员在代码中显式控制，无操作系统介入
- **来源**：字幕直接提到
- **建议方向**：
  - 在 `for` 循环场景中练习多个生成器的手动轮换调度，感受任务切换顺序对输出的影响
  - 研究 `StopIteration` 异常的触发时机（生成器耗尽时），及其在实际使用中的处理方式
  - 对比 `next()` 与 `send()` 的区别，理解向协程传值的机制

---

### 3. `time.sleep()` 函数

- **类型**：Python 标准库函数（`time` 模块）
- **在本P中的作用**：
  - 在生成器函数内部模拟 IO 耗时操作（如网络请求、文件读写等的等待阶段）
  - 演示协程在「等待」期间并不真正释放控制权（因为此处用的是同步阻塞的 `time.sleep`，而非真正的异步 IO），为后续协程库引入异步非阻塞 IO 做概念对比铺垫
  - 让输出结果有间隔，便于观察任务交替切换的过程
- **来源**：字幕直接提到
- **建议方向**：
  - 理解 `time.sleep()` 是同步阻塞调用，在真实协程场景中应替换为 `asyncio.sleep()`（异步非阻塞）
  - 对比两者在协程框架中的行为差异，验证阻塞调用会「卡住」整个线程的现象

---

### 4. `time` 模块

- **类型**：Python 标准库模块
- **在本P中的作用**：
  - 作为 `time.sleep()` 的载体被 `import time` 引入，提供时间相关工具函数
  - 本P仅使用了其 `sleep()` 函数，其余功能未涉及
- **来源**：字幕直接提到（`import time`）
- **建议方向**：
  - 了解 `time.time()` 和 `time.perf_counter()` 的用途，可用于对比协程、线程、进程在 IO 密集型任务中的实际耗时差异

---

### 5. 生成器函数（Generator Function）

- **类型**：Python 语言机制（带 `yield` 的函数构成的特殊函数类型）
- **在本P中的作用**：
  - 作为协程底层原理的实现载体：包含 `yield` 的函数即为生成器函数，调用后返回生成器对象，生成器对象本质上就是一个协程对象
  - 课程通过定义 `task1()`（唱歌）和 `task2()`（跳舞）两个生成器函数，直观演示了协程状态保存（`yield` 暂停时局部变量保留）与任务切换（`next()` 恢复执行）的核心逻辑
  - 揭示「单线程内多任务交替执行」的实现方式，为后续协程库的使用提供概念基础
- **来源**：字幕直接提到
- **建议方向**：
  - 手写多个生成器函数，模拟不同数量的并发任务（如 3 个以上），练习手动调度
  - 尝试将手动 `next()` 调度改写为循环或调度器函数，理解协程框架自动调度的雏形
  - 了解生成器函数与普通函数在内存中的存储方式差异（帧对象的保存）

---

### 6. 协程库（async/await 体系）

- **类型**：Python 语言特性 + 标准库（`asyncio`）+ 第三方生态
- **在本P中的作用**：
  - 本P仅作预告，未展开讲解
  - 课程明确指出：用生成器手动调用 `next()` 切换任务不够便捷，实际开发中会使用专门的协程库，实现任务切换的自动化
  - 作为本P内容的后续承接，点明了学习路径的走向
- **来源**：基于上下文推断（课程明确提到「专门的协程库」，结合 Python 生态，指向 `asyncio` 及 async/await 语法）
- **建议方向**：
  - 学习 `async def` 定义协程函数、`await` 挂起协程的基本语法，与本P的 `yield` / `next()` 做对比映射
  - 了解 `asyncio` 标准库的核心组件：事件循环（`event loop`）、协程对象、`Task`、`Future`
  - 在掌握 asyncio 基础后，可进一步了解第三方协程框架：`aiohttp`（异步 HTTP 客户端/服务端）、`httpx`（支持异步的 HTTP 库）
  - 参考资源：[Python 官方 asyncio 文档](https://docs.python.org/3/library/asyncio.html)
