# P57 知识框架 - 【进阶篇】协程基础
## 本P位置
- **篇章**：进阶篇 → Python并发编程体系
- **前序**：P55进程、P56线程
- **后续**：专门的协程库（async/await语法）
- **定位**：协程概念入门 + 生成器实现原理铺垫

## 核心问题
1. 协程是什么？为什么需要协程？
2. 协程如何在单线程内实现多任务切换？
3. 协程与进程、线程的核心差异是什么？

## 知识骨架

```
协程（Coroutine）
├── 定义：单线程内的"微线程"多任务方案
├── 核心机制
│   ├── 主动切换：程序员控制，非OS调度
│   └── 状态保存：yield保存状态，next恢复执行
├── 实现载体
│   └── 生成器函数（带yield的函数）
└── 适用场景：IO密集型高并发
```

## 关系与依赖

```
依赖链：进程/线程基础 → 协程概念 → 协程库（async/await）

进程 ──突破GIL→ 协程
线程 ──轻量替代→ 协程
生成器 ──本质就是→ 协程
```

## 输入 / 输出 / 前置 / 支撑

| 类型 | 内容 |
|------|------|
| **前置知识** | 进程、线程概念，GIL限制，IO密集型与CPU密集型区别 |
| **核心输入** | yield关键字、next()函数 |
| **核心输出** | 协程切换逻辑理解、状态保存机制理解 |
| **支撑工具** | time.sleep()（模拟IO等待） |
| **后续支撑** | 专门的协程库（自动切换替代手动next调用） |

## 易错点
- **混淆调度方**：协程由程序员控制，进程/线程由操作系统调度
- **误用场景**：协程不适合CPU密集型任务，只适合IO密集型
- **生成器误区**：调用`task1()`返回生成器对象，不执行函数体，需用`next()`驱动
- **单线程误解**：协程在单线程内运行，无资源竞争，但需注意自身代码逻辑

## 外部资源与对象

| 资源/对象 | 角色 | 关联 |
|----------|------|------|
| `yield` | 协程状态保存语法 | 暂停执行、让出CPU、保存状态 |
| `next()` | 协程恢复执行函数 | 从yield处继续、触发任务切换 |
| `time.sleep()` | IO模拟工具 | 演示等待时切换逻辑 |
| 协程库 | 后续学习对象 | 本P铺垫，实现自动切换 |
| 生成器函数 | 协程底层实现 | 本质即协程对象 |
