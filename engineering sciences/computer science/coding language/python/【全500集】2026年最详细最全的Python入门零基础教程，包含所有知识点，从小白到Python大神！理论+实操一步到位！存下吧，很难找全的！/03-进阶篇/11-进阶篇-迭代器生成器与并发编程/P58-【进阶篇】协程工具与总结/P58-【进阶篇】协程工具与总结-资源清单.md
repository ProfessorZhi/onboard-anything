## 一、库与框架

---

### 1. greenlet

| 字段 | 内容 |
|------|------|
| **名称** | greenlet |
| **类型** | Python 第三方库 |
| **在本 P 中的作用** | 协程手动切换工具。提供 `greenlet` 类和 `switch()` 方法，用于创建协程对象并在任务之间显式跳转。本 P 将其定位为「比生成器更直观、但比 gevent 更原始」的中间层，主要用于理解协程切换机制。 |
| **来源** | 字幕直接提到 |
| **后续建议方向** | 用 `switch()` 手写一个三任务轮转示例，对比与生成器 `yield`+`next()` 的写法差异；阅读 greenlet 官方文档了解其内存模型与 C 扩展实现原理 |

---

### 2. gevent

| 字段 | 内容 |
|------|------|
| **名称** | gevent |
| **类型** | Python 第三方库 |
| **在本 P 中的作用** | 协程自动切换核心工具，本 P 重点演示对象。基于 greenlet 实现，遇到 IO 操作时自动挂起当前协程并切换到其他协程，无需手动调用 switch。本 P 演示了 `spawn()`、`sleep()`、`joinall()` 的基本用法，并演示了打补丁前后任务执行顺序的对比。 |
| **来源** | 字幕直接提到 |
| **后续建议方向** | 用 gevent 写一个并发抓取多个 URL 响应时间的小脚本；对比打补丁前后的总耗时，直观验证自动切换的性能提升；阅读 gevent 官方文档的 Greenlet 生命周期章节 |

---

### 3. requests

| 字段 | 内容 |
|------|------|
| **名称** | requests |
| **类型** | Python 第三方库 |
| **在本 P 中的作用** | 未单独演示，作为 `monkey.patch_all()` 的典型受益库被提及。打补丁后，requests 的底层 socket IO 会被替换为 gevent 可识别版本，从而在网络等待期间触发自动切换，适用于高并发爬虫场景。 |
| **来源** | 字幕直接提到 |
| **后续建议方向** | 结合 `monkey.patch_all()` + `gevent.spawn()` + `requests.get()` 写一个多 URL 并发爬取示例，测量与串行爬取的耗时差距 |

### 4. time.sleep

| 字段 | 内容 |
|------|------|
| **名称** | time.sleep |
| **类型** | Python 标准库函数（time 模块） |
| **在本 P 中的作用** | 作为「gevent 默认无法识别的系统 IO 阻塞函数」的典型代表被演示。本 P 通过对比「未打补丁时 time.sleep 导致任务串行」与「打补丁后任务并发切换」，说明 `monkey.patch_all()` 的必要性。 |
| **来源** | 字幕直接提到 |
| **后续建议方向** | 在同一脚本中分别用 `time.sleep` 和 `gevent.sleep` 做 IO 模拟，对比两种情况下任务的执行顺序；验证打补丁后 `time.sleep` 的行为是否与 `gevent.sleep` 等效 |

## 四、关键 API 与方法速查

以下方法在本 P 均有代码演示，单独列出便于后续查阅与练习。

| 方法 / API | 所属库 | 本 P 作用 | 来源 |
|------------|--------|-----------|------|
| `greenlet(func)` | greenlet | 创建协程对象，参数为函数引用（不加括号） | 字幕直接提到 |
| `协程对象.switch()` | greenlet | 手动切换到指定协程，或从当前协程跳出 | 字幕直接提到 |
| `gevent.spawn(func, *args)` | gevent | 创建并立即启动协程，支持传参 | 字幕直接提到 |
| `gevent.sleep(n)` | gevent | 模拟 IO 阻塞，gevent 原生可识别，触发自动切换 | 字幕直接提到 |
| `协程对象.join()` | gevent | 阻塞主程序直到单个协程执行完毕 | 字幕直接提到 |
| `gevent.joinall([...])` | gevent | 阻塞主程序直到列表内所有协程执行完毕 | 字幕直接提到 |
| `gevent.monkey.patch_all()` | gevent | 将系统 IO 函数（time.sleep、socket 等）替换为 gevent 可识别版本，须置于文件最开头 | 字幕直接提到 |

## 五、资源依赖层级总览

以下结构说明本 P 各资源之间的依赖与演进关系，便于理解选型逻辑：

    Python 生成器（yield / next）
        └── greenlet（基于生成器封装，提供 switch 手动切换）
                └── gevent（基于 greenlet，提供 IO 自动切换）
                        └── monkey.patch_all()（让系统 IO 函数纳入 gevent 管控）
                                ├── time.sleep  → 打补丁后可识别
                                ├── urllib      → 打补丁后可识别
                                └── requests    → 打补丁后可识别

## 六、补充说明

- **monkey.patch_all() 的位置要求**：必须置于脚本最顶部，早于其他所有 import 语句，否则补丁对已加载模块无效。此约束在本 P 被明确强调，属于高频易错点。
- **gevent 与 async/await 的关系**：本 P 知识框架标注「后续将进入 async/await 语法糖」，gevent 是协程工具链的实践层，async/await 是语言层原生支持，两者定位不同，后续章节可对比学习。
- **requests 的线程安全说明**：本 P 未涉及，但在实际使用 gevent + requests 时，Session 对象的复用方式值得额外研究（属于建议方向，非本 P 内容）。
