## 知识骨架

```
协程工具链
├── GREENLET（手动切换）
│   ├── 基于：生成器
│   ├── 方法：switch()
│   └── 局限：IO阻塞不会自动切
│
└── GEVENT（自动切换）
    ├── 基于：greenlet
    ├── 核心方法：
    │   ├── spawn() 创建并启动
    │   ├── sleep() 触发自动切换
    │   └── joinall() 等待全部完成
    │
    └── monkey.patch_all() 
        └── 替换系统IO为GEVENT可识别版本
```

## 关系与依赖

| 层级 | 工具 | 依赖关系 |
|------|------|----------|
| 底层 | 生成器 | GREENLET的底层实现 |
| 中层 | GREENLET | GEVENT的底层实现 |
| 上层 | GEVENT | 实际项目选型 |
| 补丁 | monkey.patch_all | 让GEVENT识别系统IO |

**切换逻辑对比**：

| 工具 | 切换触发 | IO阻塞处理 |
|------|----------|------------|
| 生成器 | next()手动 | 不会自动切 |
| GREENLET | switch()手动 | 不会自动切 |
| GEVENT | 遇到IO自动 | 全自动 |

## 易错点

1. **GREENLET不会自动切IO**
   - time.sleep会让整个线程卡住
   - 必须在任务之间手动switch

2. **monkey.patch_all()位置错误**
   - 必须放在文件最开头、所有import之前
   - 否则补丁无效，原代码不生效

3. **greenlet()参数带括号**
   - 错误：`g1 = greenlet(eat_fruit())`
   - 正确：`g1 = greenlet(eat_fruit)`

4. **混淆spawn和手动创建**
   - gevent.spawn()同时创建并启动
   - GREENLET需要先创建再switch启动

5. **GEVENT.sleep vs time.sleep**
   - gevent.sleep()：GEVENT可识别，触发切换
   - time.sleep()：系统函数，需要patch_all才识别

## 外部资源与对象

| 资源 | 类型 | 作用 |
|------|------|------|
| **greenlet** | 第三方库 | 手动切换协程，pip install greenlet |
| **gevent** | 第三方库 | 自动切换协程，pip install gevent |
| **monkey.patch_all()** | GEVENT内置方法 | 打补丁，替换系统IO为GEVENT版本 |
| **time.sleep** | Python标准库 | 系统IO阻塞函数，需打补丁才能被GEVENT识别 |
| **requests** | 第三方库 | 网络请求库，打补丁后IO可被GEVENT识别 |
| **urllib** | Python标准库 | 网络请求模块，打补丁后IO可被GEVENT识别 |
| **pip** | 工具 | 安装greenlet和gevent |
