# P51 知识框架 - 【进阶篇】生成器
## 本P位置

Python进阶篇 · 迭代与生成模块 · P50迭代器的后继

上接P50迭代器知识，下承协程/异步编程基础。生成器是连接"手动迭代协议"与"自动内存优化"的桥梁。

## 核心问题

如何用最少的代码实现按需计算、避免大数据一次性加载导致的内存爆炸？

## 知识骨架

```
惰性计算（核心思想）
    │
    ├─ 形式一：生成器表达式
    │       语法糖，中括号→小括号
    │       内存占用：几乎为零（只存生成器对象）
    │
    └─ 形式二：生成器函数
            关键词：yield
            特性：调用不执行、暂停恢复、保存状态
```

**三者包含链**：生成器 ⊂ 迭代器 ⊂ 可迭代对象

**可一句话区分**：
- 可迭代对象 = 能被for循环（列表/元组/字符串/range）
- 迭代器 = 能被next()调用（iter()包装后的对象）
- 生成器 = 自带__iter__和__next__、内存几乎为零的特殊迭代器

## 关系与依赖

```
列表推导式 [i for i in range(10)]
    │
    └── 改括号 ──→ 生成器表达式 (i for i in range(10))
                            │
                            └── 改结构 ──→ 生成器函数 def f(): yield i

可迭代对象 + iter() ──→ 迭代器 ──→ 生成器（简化实现）
                              ↑
                     自定义迭代器类（手动写__iter__/__next__）
```

**从属关系**：
- 生成器一定是迭代器，迭代器一定是可迭代对象
- 反向不成立：可迭代对象（列表）不一定是迭代器，迭代器不一定是生成器

## 输入 / 输出 / 前置 / 支撑

| 概念 | 输入 | 输出 | 前置依赖 | 支撑技术 |
|------|------|------|----------|----------|
| 生成器表达式 | 可迭代对象 | generator对象 | 列表推导式语法 | Python解释器内置惰性求值 |
| 生成器函数 | 函数调用（不触发执行） | generator对象 | yield语法、函数概念 | 调用栈状态保存机制 |
| 迭代器协议 | 可迭代对象 | 迭代器 | __iter__/__next__ | for循环底层实现 |
| next()内置函数 | 迭代器/生成器 | 下一个元素 | 迭代器未耗尽 | StopIteration异常机制 |

**使用决策树**：
```
数据量 < 10000，需要重复遍历？ → 列表
数据量 > 10000，不需要重复遍历？ → 生成器表达式
需要自定义遍历逻辑？ → 自定义迭代器类
需要分步生成、流式处理、暂停恢复？ → 生成器函数
```

## 易错点

- **生成器函数调用不执行**：generator()返回对象，不执行函数体，只有next()才触发
- **一次性耗尽**：遍历完后生成器/迭代器为空，需重新创建
- **yield与return不共存**：return直接终止迭代，后续yield不执行，最终结果只能通过异常捕获传递
- **混淆括号类型**：中括号=列表推导式（立即加载），小括号=生成器表达式（惰性）
- **for循环自动处理StopIteration**：不要误以为for循环不会抛异常

## 外部资源与对象

本P未引用任何外部程序、平台、仓库或工具。所有演示基于Python内置机制。

## 核心结论

1. 生成器 = 惰性计算 + 自动实现迭代协议 + 几乎零内存占用
2. 生成器表达式 = 列表推导式改括号，最简方式
3. 生成器函数 = yield关键字，调用不执行、next触发、暂停恢复
4. 内存优化本质：用多少取多少，不提前占用
