# P51 资源清单 - 【进阶篇】生成器
## 说明

本P聚焦于Python生成器的概念、语法和应用场景，讲解内容全部基于Python内置机制，未引用任何外部网站、仓库、工具或第三方库。以下资源均为Python语言内置组件。

---

## 内置函数

### next()

- **类型**：内置函数
- **在本P中的作用**：手动从生成器或迭代器中获取下一个元素，是触发生成器函数执行的关键手段；通过逐次调用观察yield的暂停恢复行为
- **来源**：字幕直接提到
- **后续可练习方向**：用next()逐步调试生成器，对比next()与for循环的执行差异；捕获StopIteration异常处理迭代终止

### iter()

- **类型**：内置函数
- **在本P中的作用**：将可迭代对象（如列表、字符串）转换为迭代器，是理解迭代器协议的入口；知识框架中作为关联词条列出
- **来源**：基于上下文推断（与生成器/迭代器体系紧密相关）
- **后续可练习方向**：对不同数据类型使用iter()，观察转换前后的差异；结合next()练习手动遍历可迭代对象

### range()

- **类型**：内置函数
- **在本P中的作用**：生成数字序列，是演示生成器表达式的常用输入源；课程中用`range(1000000)`对比列表与生成器的内存占用差异
- **来源**：字幕直接提到
- **后续可练习方向**：用range()生成大范围序列，对比列表推导式与生成器表达式的性能；结合filter练习复杂筛选场景

### enumerate()

- **类型**：内置函数
- **在本P中的作用**：为可迭代对象添加索引编号，返回枚举对象（本质上也是一种迭代器）；知识框架关联词条中列出
- **来源**：知识框架推断
- **后续可练习方向**：在遍历生成器时同时获取索引；理解enumerate内部实现机制

## 语法与数据类型

### 列表推导式（List Comprehension）

- **类型**：语法结构
- **在本P中的作用**：生成器表达式的直接前置知识；通过对比列表推导式（中括号）与生成器表达式（小括号）的内存占用，突显惰性计算的优势
- **来源**：字幕直接提到（作为铺垫知识点详细讲解）
- **后续可练习方向**：熟练掌握带if条件的列表推导式；对比同一逻辑用循环、列表推导式、生成器表达式三种写法的性能差异

### generator类型

- **类型**：内置类型
- **在本P中的作用**：生成器表达式和生成器函数返回对象的类型标识；课程演示了`type(gn)`输出`<generator object <genexpr> at 0x...>`
- **来源**：字幕直接提到
- **后续可练习方向**：使用`isinstance()`判断对象是否为生成器；结合`inspect`模块查看生成器状态

### StopIteration异常

- **类型**：内置异常
- **在本P中的作用**：迭代器/生成器耗尽时next()抛出的标准异常；for循环内部自动捕获该异常以终止遍历
- **来源**：字幕直接提到
- **后续可练习方向**：手动捕获StopIteration以传递生成器的最终结果值；理解for循环的底层异常处理机制

## 魔术方法

### __iter__()

- **类型**：魔术方法
- **在本P中的作用**：使对象可被for循环遍历，所有可迭代对象和迭代器都实现此方法；生成器自带此方法，无需手动实现
- **来源**：知识框架推断
- **后续可练习方向**：在自定义类中实现`__iter__`，构造可迭代对象；对比手动实现迭代器与使用生成器的代码复杂度

### __next__()

- **类型**：魔术方法
- **在本P中的作用**：迭代器的核心方法，实现逐个取值逻辑；生成器由Python解释器自动提供此方法，开发者无需手动编写
- **来源**：知识框架推断
- **后续可练习方向**：手动实现一个自定义迭代器类，加深对迭代协议的理解；分析生成器与自定义迭代器的性能差异

## 关键字

### yield

- **类型**：关键字
- **在本P中的作用**：定义生成器函数的核心关键字；具有返回值、暂停函数运行、保存状态的三大功能；与return的区别在于暂停而非终止
- **来源**：字幕直接提到（生成器函数部分重点讲解）
- **后续可练习方向**：编写多yield的生成器函数实现复杂分步逻辑；练习用yield实现斐波那契数列等经典算法；探索yield from实现生成器委托

### Python迭代器协议

- **类型**：语言特性/协议
- **在本P中的作用**：定义了可迭代对象与迭代器之间的契约关系；生成器是迭代器协议的最佳实现方式；是理解for循环底层机制的理论基础
- **来源**：知识框架推断（P50迭代器内容延续）
- **后续可练习方向**：阅读官方文档中关于迭代器协议的说明；对比实现迭代器协议的多种方式
