# P52 讲解过程 - 【进阶篇】多任务与线程基础
## 本P定位

本P是Python进阶篇的起点，正式引入**多任务编程**概念。教学目标是让学习者理解多任务的思想，并掌握使用 `threading` 模块实现**多线程**的基本方法。本P不涉及进程与协程的代码实现（将在后续小节中单独讲解），而是围绕线程这一最轻量、最常用的多任务方式，从概念到实操展开系统教学。

---

## 讲解过程

### 一、课程引入与内容预告

**（0:00 - 0:22）**

课程开场，老师直接点明本P的核心价值：多任务编程是提升程序性能的核心技巧，也是处理并发场景的必备能力。随后预告本次课程的结构将分为三个小节，分别对应实现多任务的三种核心方式：线程、进程、协程。老师用通俗的类比解释三者定位：

- **线程**：轻量级多任务，像人边听歌边刷手机，同一时间执行多个任务，适合 IO 密集型场景（文件读写、网络请求）。
- **进程**：重量级多任务，每个进程有独立工作空间，可调用多核 CPU，适合计算密集型任务。
- **协程**：超轻量微线程，比线程更节省资源，能在单线程内灵活切换任务，高频切换场景下效率极高。

### 二、单任务模式的效率问题

**（1:19 - 2:13）**

老师先回顾学习者之前写的程序都是**单任务模式**，用"先唱完一首歌再去跳舞"来类比——全程只能专注一件事，必须等上一件彻底结束才能开始下一件，效率偏低。而线程可以让程序拥有一心多用的能力，让多个操作并行进行，充分调动 CPU 资源，轻松应对多 IO 操作（如同时下载多个文件、处理多个请求）。

### 三、用代码直观对比单任务与多任务

**（2:54 - 6:49）**

老师开始用代码演示单任务与多任务的实际差异。首先定义两个任务函数：

- `sing()`：打印"我在唱歌"，用 `time.sleep(2)` 模拟唱歌耗时两秒，然后打印"唱完歌了"。
- `dance()`：打印"我在跳舞"，同样 `time.sleep(2)` 模拟跳舞耗时两秒，然后打印"跳完舞了"。

老师强调 `time.sleep()` 是模拟 IO 操作（等待音频播放）的典型方式，以秒为单位延时。

单任务执行方式：先调用 `sing()`，再调用 `dance()`。运行结果显示：

- 先输出"我在唱歌"
- 等待两秒后输出"唱完歌了"
- 再输出"我在跳舞"
- 再等待两秒后输出"跳完舞了"
- **总耗时约 4 秒**（两个任务时间之和）

老师随后演示如何用 `time.time()` 记录程序执行耗时，进一步验证单任务模式的总耗时等于各任务耗时之和，因为任务是顺序执行的。

### 四、引入多任务的核心价值

**（6:49 - 7:54）**

老师指出，唱歌和跳舞本质上可以同时进行。如果使用多任务模式，这两个任务可以并行执行，**总耗时只相当于耗时最长的那个任务**，约等于 2 秒，效率直接翻倍。这就是多任务编程的核心价值。

### 五、进程与线程的核心概念

**（7:54 - 9:58）**

在正式进入线程实操之前，老师先帮学习者理清两个核心概念：**进程与线程的关系**。

- **进程**：打开一个程序就会创建一个进程（如打开微信或浏览器），是操作系统分配资源的基本单位，每个进程有独立的内存空间，互不干扰。
- **线程**：进程内的最小执行单元，是 CPU 调度的基本单位。例如微信进程中，聊天、发文件、语音通话各自对应一个线程，它们都处于同一进程内。

**核心关系**：
- 一个进程默认有一个**主线程**（程序自带的）。
- 开发者可以创建多个**子线程**（通过代码创建的）。
- 线程依附于进程而存在，没有进程就没有线程——就像员工必须在公司里才能工作。
- 同一进程内的所有线程**共享进程的资源**（如全局变量、文件句柄），线程间通信方便，不需要额外处理数据传递。

### 六、线程的三大核心优势

**（9:58 - 10:38）**

老师总结线程常用的三个优势：

1. **轻量级**：创建和切换线程的成本很低，比进程更高效，适合频繁创建和销毁的场景。
2. **共享资源**：同一进程内的线程共享全局变量、文件等资源，通信方便。
3. **并发执行**：多个线程可以同时执行，充分利用 CPU 多核资源，提升程序运行效率。

### 七、实操：threading 模块的基本使用

**（10:38 - 16:40）**

老师正式进入实操环节。Python 通过内置的 `threading` 模块实现多线程，核心是 `threading.Thread` 类。创建和启动线程只需三步：

1. 导入 `threading` 模块。
2. 通过 `threading.Thread()` 创建线程对象。
3. 调用线程对象的 `start()` 方法启动线程。

#### 7.1 无参数线程

老师演示创建两个子线程分别执行 `sing` 和 `dance` 任务：

```python
import threading

def sing():
    print("我在唱歌")
    time.sleep(2)
    print("唱完歌了")

def dance():
    print("我在跳舞")
    time.sleep(2)
    print("跳完舞了")

if __name__ == '__main__':
    t1 = threading.Thread(target=sing)
    t2 = threading.Thread(target=dance)
    t1.start()
    t2.start()
```

运行结果显示"我在唱歌"和"我在跳舞"同时输出，两秒后同时输出"唱完歌了"和"跳完舞了"，验证了多任务的并行执行。

#### 7.2 使用 join() 确保主线程等待子线程

运行后老师发现一个问题：用于计算总耗时的 `print` 语句会打印在子线程的输出中间，而非最后。老师解释这是因为主线程（执行 `print` 的代码）和子线程是同时运行的。为了让主线程等待子线程执行完再继续，需要使用 `join()` 方法：

```python
t1.join()
t2.join()
print(f"多任务总耗时: {time.time() - start:.2f} 秒")
```

加入 `join()` 后，程序会先执行完两个子线程，再计算并打印总耗时（约 2 秒）。

### 八、创建线程的三个注意事项

**（16:40 - 18:21）**

老师强调三个必须养成的习惯：

1. **target 参数只写函数名，不加括号**：加括号表示直接调用函数，而不是创建线程对象。
2. **必须用 start() 方法启动线程**：直接调用线程对象（如 `t1()`）是不会运行的，必须通过 `start()` 启动。
3. **必须使用 `if __name__ == '__main__'` 保护**：这是 Windows 系统的要求，能避免线程创建时出现重复执行的问题。如果文件被当作模块导入，没有 `if __name__ == '__main__'` 保护的话，导入时所有代码都会执行。

### 九、实操：带参数的线程

**（18:21 - 25:05）**

如果线程要执行的任务函数需要接收参数，可以使用 `args`（位置参数）或 `kwargs`（关键字参数）传参。

#### 9.1 使用 args 传参（位置参数）

```python
def sing(song_name):
    print(f"我在唱 {song_name}")
    time.sleep(2)
    print("唱完了")

def dance(dance_type, duration):
    print(f"我在跳 {dance_type}，跳 {duration} 秒")
    time.sleep(duration)
    print("跳完了")

if __name__ == '__main__':
    t1 = threading.Thread(target=sing, args=("七里香",))  # 注意末尾逗号
    t1.start()
```

老师特别提醒：**如果 args 只传一个参数，元组末尾必须加逗号**，否则 Python 会把它当作普通字符串而不是元组。

#### 9.2 使用 kwargs 传参（关键字参数）

```python
if __name__ == '__main__':
    t2 = threading.Thread(
        target=dance,
        kwargs={"dance_type": "街舞", "duration": 2}
    )
    t2.start()
```

kwargs 传递字典，字典的键名必须与函数参数名一致。

### 十、线程执行顺序的不确定性

**（25:05 - 29:03）**

老师演示一个重要特性：**线程的执行顺序由 CPU 调度决定，不按创建顺序**。

老师创建一个 `print_num(numb)` 函数，接收一个数字参数，打印该数字后 `sleep` 0.1 秒。然后用循环创建 3 个线程，分别传入 1、2、3。

多次运行程序，结果每次输出的数字顺序都不同：可能是 1-3-2、2-3-1、3-1-2……每次运行都不一样。老师解释这是因为 CPU 会根据资源情况随机调度线程，不依赖创建时的顺序。这体现了线程的**无序性**特点，是后续学习线程同步（如锁机制）的重要背景知识。

## 提到的外部资源与对象

| 名称 | 类型 | 在本P中的角色 |
|------|------|---------------|
| `time` | 标准库模块 | 提供 `sleep()` 模拟 IO 耗时，提供 `time()` 获取当前时间戳用于计时 |
| `threading` | 标准库模块 | Python 实现多线程的核心模块 |
| `threading.Thread` | 类 | 用于创建线程对象的类，核心参数是 `target`（指定任务函数） |
| `args` | 参数名 | Thread 构造函数的参数，用于传递位置参数（元组） |
| `kwargs` | 参数名 | Thread 构造函数的参数，用于传递关键字参数（字典） |
| `join()` | 实例方法 | 等待线程执行结束的方法，常用于主线程等待子线程 |
| `__name__ == '__main__'` | Python 惯用法 | 程序入口保护，避免多线程代码被导入时重复执行 |

## 本P结论

1. **多任务的核心价值**：让多个任务并行执行，总耗时由最长任务决定而非所有任务之和，效率可大幅提升（如本P例子：4秒 → 2秒）。

2. **进程与线程的关系**：进程是资源分配单位，线程是 CPU 调度的最小单位；线程依附进程而存在，同一进程的线程共享资源。

3. **线程的三大优势**：轻量级（创建/切换成本低）、共享资源（通信方便）、并发执行（利用多核）。

4. **threading 模块三步走**：导入模块 → `threading.Thread(target=函数名)` 创建对象 → `start()` 启动线程。

5. **带参数线程**：使用 `args`（元组，末尾逗号）或 `kwargs`（字典，键名匹配形参）传参。

6. **线程无序性**：CPU 随机调度线程，执行顺序不可预测，这是后续线程同步知识的铺垫。

7. **养成习惯**：使用 `if __name__ == '__main__'` 保护多线程代码；`target` 不加括号；必须用 `start()` 启动。
