## 知识骨架

```
多任务编程
├── 为什么需要多任务
│   └── 单任务顺序执行，总耗时 = 各任务耗时之和
│   └── 多任务并行执行，总耗时 ≈ 最长任务耗时
│
├── 进程 vs 线程
│   ├── 进程：资源分配单位，独立内存空间
│   └── 线程：CPU调度最小单位，依存于进程
│
└── threading模块
    ├── Thread类
    │   ├── target：指定任务函数（不加括号）
    │   ├── args：位置参数元组（单参数末尾逗号）
    │   └── kwargs：关键字参数字典
    ├── start()：启动线程
    └── join()：主线程等待子线程
```

---

## 关系与依赖

- **进程是线程的容器**：线程依附进程存在，同进程线程共享资源（全局变量、文件句柄）
- **主线程与子线程**：程序入口自动创建主线程，代码创建的为子线程
- **target与函数**：target参数接收函数引用而非函数调用，加括号会直接执行
- **join()阻塞主线程**：确保子线程执行完毕后再继续，常用于计时统计

## 易错点

- **target不加括号**：`target=sing`而非`target=sing()`
- **args单参数逗号**：`args=("七里香",)`而非`args=("七里香")`
- **kwargs键名匹配**：字典键必须与函数形参名一致
- **必须用start()启动**：直接调用线程对象不会执行
- **if __name__ == '__main__'保护**：Windows系统要求，防止导入时重复执行
- **线程执行无序**：CPU随机调度，顺序不可预测，依赖创建顺序是错误假设

## 外部资源与对象

| 名称 | 类型 | 作用 |
|------|------|------|
| time | 标准库模块 | sleep()模拟IO耗时，time()获取时间戳计时 |
| threading | 标准库模块 | Python多线程实现的核心模块 |
| threading.Thread | 类 | 线程对象构造函数，接收target/args/kwargs参数 |
| args | 参数名 | Thread的位置参数（元组类型） |
| kwargs | 参数名 | Thread的关键字参数（字典类型） |
| join() | 实例方法 | 阻塞调用线程直至目标线程执行完毕 |
| if __name__ == '__main__' | 惯用法 | 程序入口保护，Windows多线程必需 |
