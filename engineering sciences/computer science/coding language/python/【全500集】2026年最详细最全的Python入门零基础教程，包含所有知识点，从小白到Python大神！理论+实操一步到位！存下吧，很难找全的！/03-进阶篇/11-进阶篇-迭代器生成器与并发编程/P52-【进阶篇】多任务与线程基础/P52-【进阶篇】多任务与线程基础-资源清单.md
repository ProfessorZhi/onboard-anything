## 核心模块与类

### time（标准库模块）

- **类型**：Python 标准库模块
- **在本P中的作用**：提供 `time.sleep()` 模拟 IO 耗时操作，提供 `time.time()` 获取当前时间戳用于计算程序执行耗时
- **来源**：字幕直接提到
- **建议方向**：学习时间模块的其他功能如 `time.strftime()` 格式化时间输出，以及 `datetime` 模块的进阶用法

### threading（标准库模块）

- **类型**：Python 标准库模块
- **在本P中的作用**：Python 实现多线程编程的核心模块，无需安装，Python 内置提供
- **来源**：字幕直接提到
- **建议方向**：深入阅读官方文档中 threading 模块的其他内容，包括 `Lock`（线程锁）、`Condition`（条件变量）、`Event`（事件）等同步原语

### threading.Thread（类）

- **类型**：Python 标准库类
- **在本P中的作用**：用于创建线程对象的核心类，通过 `target`、`args`、`kwargs` 等参数指定线程要执行的任务
- **来源**：字幕直接提到
- **建议方向**：了解 Thread 类的其他构造参数如 `name`（线程命名）、`daemon`（守护线程），以及 `current_thread()` 获取当前线程对象

## 核心方法与函数

### time.sleep()

- **类型**：函数（time 模块）
- **在本P中的作用**：以秒为单位阻塞当前线程，常用于模拟 IO 操作（如音频播放、网络请求）的等待时间
- **来源**：字幕直接提到
- **建议方向**：对比 `asyncio.sleep()` 实现异步延时，了解 `threading.Event.wait()` 等其他线程同步延时方式

### time.time()

- **类型**：函数（time 模块）
- **在本P中的作用**：返回当前时间戳（Unix 时间），通过差值计算程序执行耗时
- **来源**：字幕直接提到
- **建议方向**：学习 `time.perf_counter()` 获取更高精度计时，了解 `timeit` 模块进行基准测试

### Thread.start()

- **类型**：实例方法（Thread 类）
- **在本P中的作用**：启动线程，使其开始执行目标函数。必须通过此方法启动，直接调用线程对象不会执行
- **来源**：字幕直接提到
- **建议方向**：了解 `run()` 方法与 `start()` 的区别（直接调用 `run()` 会在当前线程执行，而 `start()` 会创建新线程）

### Thread.join()

- **类型**：实例方法（Thread 类）
- **在本P中的作用**：阻塞调用线程（通常是主线程），直到被调用的线程执行完毕。用于确保子线程完成后再继续主线程逻辑
- **来源**：字幕直接提到
- **建议方向**：在多线程场景中实践 join() 的超时参数 `timeout`，以及多个线程的 join() 顺序对性能的影响

## 参数与惯用法

### target（参数名）

- **类型**：Thread 构造函数的参数名
- **在本P中的作用**：指定线程执行的目标函数，只写函数名不加括号（加括号会直接调用函数）
- **来源**：字幕直接提到
- **建议方向**：理解函数引用与函数调用的区别，为后续学习回调函数、装饰器等高级用法打基础

### args（参数名）

- **类型**：Thread 构造函数的参数名
- **在本P中的作用**：以元组形式传递位置参数给目标函数。注意单参数元组末尾必须加逗号
- **来源**：字幕直接提到
- **建议方向**：对比其他语言的多线程参数传递方式，如 Java 的构造函数参数、C++ 的 lambda 封装

### kwargs（参数名）

- **类型**：Thread 构造函数的参数名
- **在本P中的作用**：以字典形式传递关键字参数给目标函数，字典键名必须与函数形参名一致
- **来源**：字幕直接提到
- **建议方向**：了解 Python 的解包机制 `**kwargs` 在函数调用中的应用

### if __name__ == '__main__'（惯用法）

- **类型**：Python 程序入口保护惯用法
- **在本P中的作用**：确保多线程代码在被导入为模块时不会重复执行，Windows 系统多线程编程的必备习惯
- **来源**：字幕直接提到
- **建议方向**：深入理解 Python 模块导入机制与 `__name__` 变量的作用，阅读 Python 官方编码规范中关于入口保护的建议

## 外部概念与平台

### CPU 多核（硬件概念）

- **类型**：硬件概念
- **在本P中的作用**：多线程能够同时利用多个 CPU 核心实现真正的并行计算，提升程序执行效率
- **来源**：基于上下文推断
- **建议方向**：使用 `psutil` 库查看进程在不同核心上的分布，使用 `multiprocessing` 模块实现多进程以调用多核 CPU

### 操作系统线程调度

- **类型**：操作系统概念
- **在本P中的作用**：线程的执行顺序由操作系统 CPU 调度器随机决定，不依赖线程创建顺序，这是理解线程无序性的基础
- **来源**：基于上下文推断
- **建议方向**：了解 GIL（全局解释器锁）对 Python 多线程的影响，在 I/O 密集型与计算密集型场景中选择合适的多任务方案

### 进程（操作系统概念）

- **类型**：操作系统概念
- **在本P中的作用**：资源分配的基本单位，拥有独立内存空间；与线程关系密切（进程包含线程，线程依附进程）
- **来源**：字幕直接提到
- **建议方向**：对比学习 P53 的进程编程，理解多进程与多线程的适用场景差异
