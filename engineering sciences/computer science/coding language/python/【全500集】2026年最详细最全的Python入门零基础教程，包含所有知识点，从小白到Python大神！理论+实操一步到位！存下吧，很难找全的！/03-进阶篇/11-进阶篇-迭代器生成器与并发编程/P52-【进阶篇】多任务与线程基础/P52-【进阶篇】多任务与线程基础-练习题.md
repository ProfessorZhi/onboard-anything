## 最小实践

1. 定义函数 `download(filename, seconds)`，用 `time.sleep(seconds)` 模拟下载，完成后打印 `{filename} 下载完毕`

## 扩展思考

**Q6：以下代码连续启动 5 个线程，每个线程打印自己的编号。多次运行后观察输出顺序。**

```python
import threading

def show(n):
    print(f"线程 {n} 执行")

for i in range(5):
    threading.Thread(target=show, args=(i,)).start()
```

- 输出顺序每次是否相同？
- 这说明了线程调度的什么本质特性？
- 如果业务逻辑依赖"线程按创建顺序执行"，会带来什么隐患？应该用什么机制来保证顺序（本P不要求实现，说出机制名称即可）？
