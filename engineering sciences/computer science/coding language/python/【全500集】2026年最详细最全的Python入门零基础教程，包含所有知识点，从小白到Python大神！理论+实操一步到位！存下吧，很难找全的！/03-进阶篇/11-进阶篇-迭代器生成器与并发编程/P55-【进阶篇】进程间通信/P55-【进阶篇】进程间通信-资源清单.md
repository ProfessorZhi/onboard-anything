# P55 资源清单 - 【进阶篇】进程间通信

> 本P涉及资源均来自 Python 标准库，无需额外安装。来源标注说明："字幕直接提到"指老师在讲解中明确命名；"上下文推断"指由演示代码或语义隐含可确认的资源。

---

## 1. multiprocessing.Queue

| 字段 | 内容 |
|------|------|
| **名称** | `multiprocessing.Queue` |
| **类型** | Python 标准库类 |
| **在本P中的作用** | 本P核心教学对象。提供进程安全的 FIFO 队列，作为生产进程与消费进程之间的数据通道。演示了 `put`、`get`、`empty`、`full`、`qsize` 五个核心方法，并通过 `maxsize` 参数控制队列容量上限。 |
| **来源** | 字幕直接提到 |
| **官方文档** | https://docs.python.org/zh-cn/3/library/multiprocessing.html#multiprocessing.Queue |

**本P涉及的关键用法**

| 方法/参数 | 签名 | 阻塞行为 |
|-----------|------|----------|
| 构造 | `Queue(maxsize=0)` | `maxsize=0` 表示无上限 |
| `put` | `q.put(item)` | 队列已满时阻塞等待 |
| `get` | `q.get()` | 队列为空时阻塞等待 |
| `empty` | `q.empty()` → `bool` | 非阻塞，返回瞬时状态 |
| `full` | `q.full()` → `bool` | 非阻塞，返回瞬时状态 |
| `qsize` | `q.qsize()` → `int` | 非阻塞，返回当前元素数量 |

**建议方向**

- 练习用 `maxsize` 设置不同容量，观察 `put` 阻塞触发时机
- 单独验证 `empty()`/`full()` 的"瞬时性"缺陷：在判断与使用之间插入 `time.sleep()`，观察结果是否已过期
- 阅读官方文档中关于 `put_nowait`、`get_nowait` 的非阻塞变体，与本P的阻塞版本对比
- 探索 `JoinableQueue`，了解 `task_done()` 与 `join()` 在队列层面的协作方式

## 2. multiprocessing.Process

| 字段 | 内容 |
|------|------|
| **名称** | `multiprocessing.Process` |
| **类型** | Python 标准库类 |
| **在本P中的作用** | 前置依赖对象（承接 P54）。在本P实战部分用于创建生产进程（`p1`）和消费进程（`p2`），通过 `args=(q,)` 将共享队列实例传入子进程函数，并通过 `start()` 启动、`join()` 等待完成。 |
| **来源** | 字幕直接提到 |
| **官方文档** | https://docs.python.org/zh-cn/3/library/multiprocessing.html#multiprocessing.Process |

**本P涉及的关键用法**

| 方法/参数 | 在本P中的作用 |
|-----------|--------------|
| `Process(target=f, args=(q,))` | 绑定目标函数并传入队列对象 |
| `p.start()` | 启动子进程 |
| `p.join()` | 主进程阻塞，等待指定子进程结束后再继续 |

**建议方向**

- 练习将 `p1.join()` 去掉，对比有无 `join` 时生产/消费交错的输出差异
- 在子进程中不通过参数传递队列，而是尝试直接在子进程内新建队列，观察通信失败的现象，理解"共享同一实例"的必要性
- 阅读 `Process` 的 `daemon` 属性，了解守护进程与普通子进程的退出行为差异

| 字段 | 内容 |

| **在本P中的作用** | `Queue` 和 `Process` 的父模块，通过 `from multiprocessing import Queue, Process` 导入。本P使用的所有多进程工具均来自此模块。 |

- 查阅 `multiprocessing.Manager`，了解跨进程共享复杂数据结构（如 `dict`、`list`）的方法

## 4. time 模块

| 字段 | 内容 |
|------|------|
| **名称** | `time` |
| **类型** | Python 标准库模块 |
| **在本P中的作用** | 辅助工具。在生产函数中 `time.sleep(0.5)` 模拟生产耗时，在消费函数中 `time.sleep(1)` 模拟消费耗时，人为制造生产速度（0.5s）快于消费速度（1s）的场景，使进程并发交错输出更直观可见。 |
| **来源** | 字幕直接提到（演示代码中明确使用） |
| **官方文档** | https://docs.python.org/zh-cn/3/library/time.html |

**建议方向**

- 调整 `sleep` 参数比例（如生产1秒、消费0.2秒），观察消费速度超过生产时的输出变化
- 将 `sleep` 全部去掉，观察进程调度完全由操作系统决定时的输出顺序是否稳定

## 6. 共享内存——后续扩展资源

| 字段 | 内容 |
|------|------|
| **名称** | 共享内存（`multiprocessing.Value` / `multiprocessing.Array` / `multiprocessing.shared_memory`） |
| **类型** | Python 标准库进程间通信机制 |
| **在本P中的作用** | 本P末尾提及的另一种进程间通信方式，与 `Queue`、`Pipe` 并列列举，本P未展开。共享内存允许多个进程直接读写同一块内存区域，性能高但需要手动同步。 |
| **来源** | 字幕直接提到（扩展提示中提及"共享内存"） |
| **官方文档** | https://docs.python.org/zh-cn/3/library/multiprocessing.html#shared-ctypes-objects |

**建议方向**

- 与 `Queue` 对比，理解"自动同步（队列）"与"手动同步（共享内存+锁）"的取舍关系
- 探索 Python 3.8 引入的 `multiprocessing.shared_memory`，了解跨进程共享大块数据（如 numpy 数组）的现代做法
