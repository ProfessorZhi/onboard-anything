# P53 练习题 - 【进阶篇】线程核心与线程同步
## 1. 结构识别

**以下哪种情况属于"资源竞争"（race condition）？**

A. 线程A读取共享变量x=0，线程B也读取x=0，两者各自+1后分别写回，最终x=1

B. 线程A完整执行完毕后，线程B才开始执行，最终x=2

C. 两个线程各自操作自己的局部变量，互不影响

D. 线程A在持有锁的情况下调用sleep，其他线程等待锁

---

## 2. 结构识别

**`count += 1` 在多线程环境下是不安全的，根本原因是？**

A. count变量是全局变量，所有线程都能访问

B. 该操作底层分为"读取→计算→写回"三个原子指令，而非单一原子操作

C. Python的整数是不可变对象，修改时会创建新对象

D. 线程调度器在执行完+号运算后就会切换线程

---

## 3. 机制理解

**使用 `join()` 和使用 `Lock()` 解决资源竞争的本质区别是什么？**

A. join比Lock更高效，因为join不需要获取锁

B. join通过让线程串行执行牺牲并发性换取安全；Lock保留并发性但引入锁管理复杂度

C. join只能用于两个线程，Lock可以用于任意数量线程

D. 两者没有本质区别，只是语法不同

## 4. 机制理解

**以下哪种情况最可能导致死锁（Deadlock）？**

A. 所有线程都正确使用 `with lock:` 语句获取和释放锁

B. 线程A先获取锁1再获取锁2，线程B先获取锁2再获取锁1

C. 只有一个线程在运行，不存在锁竞争

D. 所有读操作都加了锁
