# P54 讲解过程 - 【进阶篇】进程基础
## 本P定位

本P是「进阶篇」的第二小节，承接上一P的线程内容。核心目标是帮助学习者理解进程与线程的根本差异，掌握 `multiprocessing` 模块创建和管理进程的方法，并为后续进程间通信（Queue）做铺垫。

## 讲解过程

### 一、从线程短板引出进程

课程首先回顾了线程的两个明显短板：

- **资源共享问题**：线程之间共享资源，容易引发资源竞争。
- **GIL限制**：受 Python 全局解释器锁的约束，即使电脑是多核 CPU，也无法让所有核心同时发力。

老师用比喻形象说明："好比你一群人挤着用同一套工具，那又慢对吧，又容易引发冲突。"

接着点出进程的作用——弥补线程短板的重量级多任务方案。进程的两大核心优势：

- 每个进程拥有独立的资源空间，彼此互不干扰。
- 能够彻底突破 GIL 束缚，让多核 CPU 各自全力运转。

老师总结学习进程的意义：能够摆脱线程在多核利用上的局限，轻松应对大规模数值计算、复杂算法运算等 CPU 密集型任务，让程序在多核环境下发挥最大算力，大幅提升计算效率。

### 二、进程的定义

老师给出进程的定义：**正在运行的程序**。

举例说明：

- 打开微信、浏览器，或运行一个 Python 脚本，操作系统都会自动创建进程。
- 每一个进程都会分配独立的内存空间、CPU 等资源，进程之间互不干扰。

用房子做比喻："你有多个独立的房子对吧，每一个房子里面都有自己的家具，就是有它自己的资源，房子之间是各自独立的，不会互相影响到。"

### 三、进程与线程的核心区别对比

老师强调"学好进程首先得分清楚它和线程的核心区别"，用一张表格列举了六项核心区别：

| 维度 | 进程 | 线程 |
|------|------|------|
| 资源分配 | 独立资源空间，有独立的内存、CPU等资源，占用较多 | 共享所属进程的资源，占用较少 |
| 通信难度 | 资源不共享、各自独立，通信需要通过专门机制（如队列），难度较高 | 资源共享，通信方便 |
| 执行效率 | 创建和切换成本高，效率较低 | 创建或切换成本低，效率较高 |
| 多核利用 | 支持真正并行，充分利用多核 CPU | 受 GIL 限制，多核利用率低，看似并行实则串行 |
| 稳定性 | 一个进程崩溃不影响其它进程，稳定性高 | 一个线程崩溃可能导致整个进程崩溃，稳定性低 |
| 适用场景 | CPU 密集型任务（如大量计算） | IO 密集型任务（如下载文件、网络请求） |

老师用微信和浏览器的例子说明稳定性差异：同时打开微信和浏览器，微信崩溃不会导致浏览器关闭，因为是两个独立进程；但如果微信里的聊天线程崩溃，可能导致整个微信进程关闭。

最后给出黄金选型法则一句话总结：

- 处理 IO 密集型任务（下载文件、网络请求）→ 用线程，效率更高、更灵活。
- 处理 CPU 密集型任务（复杂计算、数据分析）→ 用进程，能够充分利用多核。

### 四、进程的五种状态

老师介绍进程在运行过程中会经历的五种状态，由操作系统统一调度，无需深入研究，知道大概流程即可。

**新建态**：

- 进程刚被创建，操作系统正在为其分配资源（如内存），还没有进入就绪队列。
- 举例：双击桌面上的浏览器图标后，系统正在加载浏览器程序到内存里。

**就绪态**：

- 进程已完全准备好运行，具备所有运行条件，正在等待 CPU 调度。
- 举例：同时打开浏览器、代码编辑器和音乐播放器，音乐播放器正在播放（运行态），浏览器和代码编辑器就处于就绪态，它们已准备好随时运行，只要操作系统决定把 CPU 从音乐播放器切换给它们，它们就能立即执行。

**运行态**：

- CPU 正在执行进程的功能，执行它的指令。
- 举例：微信播放语音、浏览器下载图片并显示、视频播放器解码视频等。

**等待态 / 阻塞态**：

- 进程因等待某个外部事件的发生而暂停运行，即使有 CPU 空闲也无法运行。
- 外部事件包括：输入、网络响应、文件读写的完成等。
- 举例：微信打开着但没有执行任何操作，正在等待点击鼠标或输入文字；下载软件正在等待网络数据从服务器传输过来。

**终止态**：

- 进程执行完毕或被强制关闭，操作系统回收它所占用的所有资源。
- 举例：手动关闭软件（手动关闭微信）、Python 脚本运行结束并退出。

### 五、创建和启动进程——基础示例

老师指出本小节重点：如何创建和启动进程。Python 中通过 `multiprocessing` 模块的 `Process` 类创建进程，步骤和线程类似，但要注意进程的独立资源特性，不能完全照搬线程的用法。

**示例：创建两个无参数的进程，分别执行唱歌和跳舞任务。**

首先定义任务函数：

```python
def sing():
    print("我在唱歌")
    time.sleep(2)
    print("唱完了")
    
def dance():
    print("我在跳舞")
    time.sleep(2)
    print("跳完舞了")
```

然后创建子进程：

```python
if __name__ == '__main__':
    import multiprocessing
    p1 = multiprocessing.Process(target=sing)  # 注意不要加小括号，加小括号表示调用，不加表示函数本身
    p2 = multiprocessing.Process(target=dance)
    p1.start()
    p2.start()
```

运行结果：两个任务同时进行，总耗时约两秒。

**三个关键注意点：**

1. **建议加上 `if __name__ == '__main__'` 判断**：Windows 系统当中创建进程时会重新导入脚本，如果不加这个判断，可能会导致无限创建进程，从而报错甚至崩溃。

2. **进程创建成本比线程高**：虽然这个示例的耗时看起来和线程一样，但底层创建进程时，操作系统是要为进程分配独立的内存、分配资源的，比线程要复杂得多。

3. **启动子进程用 `start` 方法**：不能直接调用进程对象（如 `p1()`），也不能直接调用函数，那样就不是启动进程，而是执行函数，会有先后顺序。

### 六、带参数的进程创建

老师演示如何给进程传参，与线程完全一样，用 `args`（元组形式）或 `kwargs`（字典形式）传参。

**示例：定义带参数的任务函数。**

```python
def sing(name, age):
    print(f"{age}岁的{name}在唱歌")
    time.sleep(2)
    os.getpid()  # 查看当前进程的id
    os.getppid()  # 查看父进程的id

def dance(name):
    print(f"{name}在跳舞")
    time.sleep(2)
```

**传参方式一：`args` 以元组形式传参**（顺序和个数要保持一致）：

```python
p1 = Process(target=sing, args=("冰冰", 18))
```

**传参方式二：`kwargs` 以字典形式传参**（键名必须和参数名保持一致）：

```python
p2 = Process(target=dance, kwargs={"name": "小明"})
```

运行结果：18岁的冰冰在唱歌，小明在跳舞。唱歌进程和跳舞进程的 ID 不一样，但父进程 ID 相同（因为处在同一个 Py 文件里）。

### 七、进程属性

老师介绍进程的几个实用属性，用于查看关键信息。

**导入方式简化**：

```python
from multiprocessing import Process
```

**查看主进程 ID**：

```python
import os
print(f"主进程PID: {os.getpid()}")
```

**查看进程名称**：

- 默认名称是 `Process-1`、`Process-2`。
- 可以在创建进程时通过 `name` 参数指定名称：

```python
p1 = Process(target=sing, args=("冰冰", 18), name="唱歌进程")
p2 = Process(target=dance, kwargs={"name": "小明"}, name="跳舞进程")
print(p1.name)  # 输出：唱歌进程
print(p2.name)  # 输出：跳舞进程
```

**查看进程 ID**：

```python
print(f"p2的PID: {p2.pid}")
```

**查看进程存活状态**：

```python
print(f"p2的存活状态: {p2.is_alive()}")
```

- 进程启动后、执行完毕前，`is_alive()` 返回 `True`。
- 进程执行结束后，返回 `False`。

**`join` 方法**：让主进程等待子进程执行完毕后再继续执行主进程的代码。

```python
p1.start()
p2.start()
p1.join()
p2.join()
# 此时才执行主进程的代码
```

老师演示了 `join` 放在不同位置的效果，放在 `start` 后面会先执行子进程再执行主进程。

### 八、进程不共享全局资源

老师强调这是进程和线程最核心的区别之一。

**核心结论**：

- 线程会共享全局变量。
- 进程有独立的资源空间，全局变量不会被多个进程共享。
- 每个进程都会复制一份全局变量的副本，修改之后只会影响自己的副本，不会影响其它进程。
- 好处是不会出现资源竞争，不需要加锁；坏处是无法直接通过全局变量进行通信，需要通过专门的机制。

**代码验证示例**：

```python
count = 0

def add_count():
    global count
    count += 100
    print(f"子进程执行后count的值: {count}")

if __name__ == '__main__':
    print(f"主进程初始count: {count}")
    p1 = Process(target=add_count)
    p2 = Process(target=add_count)
    p1.start()
    p2.start()
    p1.join()
    p2.join()
    print(f"主进程最终count: {count}")
```

**运行结果分析**：

- 主进程初始 count 是 0，最终 count 还是 0。
- 两个子进程内的 count 都是 100（0 + 100），而不是 200（100 + 100）。
- 说明每个进程都复制了 count 的副本，有自己的独立副本，修改后互不影响。

### 九、引出进程间通信——Queue

老师指出既然进程之间不共享资源，如果需要让进程与进程之间实现通信，就必须使用专门的通信机制。

其中 `multiprocessing` 模块里的 **Queue 队列**是最常用最安全的方式。

**Queue 的特点**：

- 遵循先进先出（FIFO）规则。
- 先进去的先出来，后进去的后出来。
- 支持多进程安全的入队（存数据）和出队（取数据），不会出现冲突问题。
- 比喻：像一个共享的信箱，一个进程往信箱里放信，另一个进程从信箱里取信。

## 提到的外部资源与对象

| 资源/对象 | 扮演角色 | 说明 |
|----------|----------|------|
| `multiprocessing` 模块 | Python 标准库，用于创建和管理进程 | 核心工具模块 |
| `Process` 类 | 创建进程的类 | 通过实例化创建子进程 |
| `time` 模块 | 标准库，用于 `sleep` 延时 | 模拟任务耗时 |
| `os` 模块 | 标准库，用于获取进程 ID | `os.getpid()` 获取当前进程 ID，`os.getppid()` 获取父进程 ID |
| `Queue`（multiprocessing.Queue） | 进程间通信机制 | 先进先出的安全队列，课程后续会详细讲解 |

## 本P结论

1. **进程 vs 线程**：进程是正在运行的程序，拥有独立资源空间，能突破 GIL 限制充分利用多核 CPU；线程共享资源但受 GIL 限制。CPU 密集型任务用进程，IO 密集型任务用线程。

2. **进程五种状态**：新建态 → 就绪态 → 运行态 → 等待态/阻塞态 → 终止态。

3. **创建进程基本语法**：
   - `Process(target=函数名, args=(参数,))` 或 `kwargs={"参数名": 值}`
   - `p.start()` 启动
   - `p.join()` 等待
   - Windows 系统必须加 `if __name__ == '__main__'` 保护。

4. **进程属性**：`name`（名称）、`pid`（ID）、`is_alive()`（存活状态）。

5. **进程不共享全局资源**：每个进程持有全局变量的独立副本，修改互不影响，好处是避免资源竞争，坏处是不能用全局变量通信。
