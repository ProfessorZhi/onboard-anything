### 2. `os`

| 字段 | 内容 |
|------|------|
| **名称** | `os` |
| **类型** | Python 标准库模块 |
| **在本 P 中的作用** | 用于在运行时查询进程信息。`os.getpid()` 返回当前进程的 PID，`os.getppid()` 返回父进程的 PID。本 P 用于演示子进程与主进程 PID 的差异，验证进程的独立性。 |
| **来源** | 字幕直接提到 |
| **建议方向** | 练习在多进程程序中分别在主进程和子进程内打印 `os.getpid()` 与 `os.getppid()`，对照系统任务管理器（Windows）或 `ps` 命令（Linux/macOS）验证 PID 对应关系；进一步了解 `os.cpu_count()` 查询逻辑核心数，为 `Pool` 的 `processes` 参数做铺垫。 |

### 3. `time`

| 字段 | 内容 |
|------|------|
| **名称** | `time` |
| **类型** | Python 标准库模块 |
| **在本 P 中的作用** | 配合演示用途。`time.sleep(2)` 在任务函数中模拟耗时操作，使并发效果（两个进程同时运行总耗时约 2 秒而非 4 秒）在输出中可被观察到。 |
| **来源** | 字幕直接提到 |
| **建议方向** | 用 `time.time()` 或 `time.perf_counter()` 在进程启动前后记录时间戳，量化并发带来的加速比；对比单进程顺序执行、多线程、多进程三种方式下同一 CPU 密集型任务的耗时差异。 |

## 二、类

### 4. `multiprocessing.Process`

| 字段 | 内容 |
|------|------|
| **名称** | `multiprocessing.Process` |
| **类型** | Python 标准库类 |
| **在本 P 中的作用** | 本 P 的核心操作对象。通过实例化该类创建子进程，核心参数包括：`target`（任务函数，不加括号）、`args`（元组形式传参）、`kwargs`（字典形式传参）、`name`（自定义进程名称）。启动用 `.start()`，等待用 `.join()`，查存活状态用 `.is_alive()`，查 PID 用 `.pid`，查名称用 `.name`。 |
| **来源** | 字幕直接提到 |
| **建议方向** | 尝试继承 `Process` 类并重写 `run()` 方法来创建进程（面向对象风格），对比与 `target` 参数方式的区别；练习 `daemon` 属性的使用，观察守护进程在主进程退出后的行为；练习 `p.terminate()` 强制终止进程并观察 `is_alive()` 的状态变化。 |

---

### 5. `multiprocessing.Queue`

| 字段 | 内容 |
|------|------|
| **名称** | `multiprocessing.Queue` |
| **类型** | Python 标准库类（进程安全队列） |
| **在本 P 中的作用** | 本 P 末尾作为「进程间通信（IPC）」的引出对象。因进程之间不共享全局资源，需要专门的通信机制，`Queue` 是其中最常用、最安全的方式。遵循先进先出（FIFO）规则，支持多进程安全的入队（`put`）和出队（`get`）。详细使用留待后续 P 讲解。 |
| **来源** | 字幕直接提到 |
| **建议方向** | 预习阶段可先了解 `Queue.put()` 与 `Queue.get()` 的基本签名；后续课程讲解后，练习经典生产者-消费者模型：一个进程负责生产数据并 `put` 入队，另一个进程从队列 `get` 取数据处理，观察两者之间的协作流程。 |

## 三、函数 / 方法

---

### 6. `os.getpid()`

| 字段 | 内容 |
|------|------|
| **名称** | `os.getpid()` |
| **类型** | 标准库函数 |
| **在本 P 中的作用** | 在任务函数内部调用，返回当前正在执行该函数的进程 PID。本 P 用于演示两个子进程拥有不同 PID，直观验证进程独立性。 |
| **来源** | 字幕直接提到 |
| **建议方向** | 在主进程和多个子进程中同时打印 `os.getpid()`，制作一张「主进程 → 子进程」的 PID 树状图，与操作系统进程监视工具的输出互相印证。 |

---

### 7. `os.getppid()`

| 字段 | 内容 |
|------|------|
| **名称** | `os.getppid()` |
| **类型** | 标准库函数 |
| **在本 P 中的作用** | 在任务函数内部调用，返回当前进程的父进程 PID。本 P 用于演示同一个 Python 脚本内的两个子进程，父进程 PID 相同（均为主进程），说明它们共属一个父进程。 |
| **来源** | 字幕直接提到 |
| **建议方向** | 结合 `os.getpid()` 同时打印，对比子进程 PID 与父进程 PID，加深对进程父子关系的理解；在 Linux/macOS 上用 `pstree` 命令验证进程树结构。 |

---

|------|------|
| **名称** | `time.sleep()` |

| **来源** | 字幕直接提到（基于代码示例推断具体用法） |

## 四、关键语法 / 惯用写法

---

### 9. `if __name__ == '__main__'` 保护块

| 字段 | 内容 |
|------|------|
| **名称** | `if __name__ == '__main__'` |
| **类型** | Python 惯用语法 / 入口保护写法 |
| **在本 P 中的作用** | Windows 系统创建进程时会重新导入（`spawn` 方式）当前脚本，若不加此判断，进程创建代码会在每次导入时重复执行，导致无限递归创建进程，最终崩溃。本 P 明确要求所有进程创建代码必须写在此块内部。 |
| **来源** | 字幕直接提到 |
| **建议方向** | 故意删去此保护块，在 Windows 上运行并观察报错信息；在 Linux/macOS 上运行（fork 机制）观察行为差异，理解两种平台进程启动方式（`fork` vs `spawn`）的根本区别；查阅 `multiprocessing` 文档中关于 `start_method` 的说明。 |

## 五、概念性对象（无需安装，理解即可操作）

---

### 10. GIL（全局解释器锁）

| 字段 | 内容 |
|------|------|
| **名称** | GIL（Global Interpreter Lock，全局解释器锁） |
| **类型** | CPython 解释器机制（概念） |
| **在本 P 中的作用** | 引出进程必要性的核心理由之一。GIL 使得同一时刻只有一个线程能执行 Python 字节码，导致多线程在 CPU 密集型场景下无法真正并行利用多核；多进程因每个进程拥有独立的解释器实例，彻底绕开 GIL 限制。 |
| **来源** | 字幕直接提到 |
| **建议方向** | 编写一个 CPU 密集型函数（如计算大量素数），分别用单线程、多线程、多进程运行并计时，用数据直观感受 GIL 对多线程的压制效果和多进程对 GIL 的绕过效果；进一步了解 Python 3.12 引入的 per-interpreter GIL 方向（扩展阅读，无需深入）。 |
