# P50 资源清单 - 【进阶篇】迭代器
---

## 一、核心资源（代码层面的内置对象与函数）

| 名称 | 类型 | 在本 P 中的作用 | 来源 | 后续可练习方向 |
|------|------|----------------|------|----------------|
| `collections.abc.Iterable` | 内置模块中的抽象基类 | 作为 `isinstance()` 的第二个参数，用于判断任意对象是否可迭代 | 字幕直接提到 | 使用该类编写一个工具函数，批量检测列表、字典、字符串、数值等不同类型对象的可迭代性 |
| `isinstance()` | 内置函数 | 判断给定对象是否为 `Iterable` 实例，返回布尔值，是判断可迭代性的核心工具 | 字幕直接提到 | 设计测试脚本，对自定义类实例调用 `isinstance(obj, Iterable)`，探索用户自定义对象是否默认可迭代 |
| `iter()` | 内置函数 | 接收可迭代对象，返回对应的迭代器对象，是获取迭代器的唯一入口 | 字幕直接提到 | 用 `iter()` 对比处理 range、文件对象、自定义列表等不同数据源，观察返回的迭代器对象类型差异 |
| `next()` | 内置函数 | 从迭代器中逐个取出下一个元素，驱动迭代器前进，是手动遍历的核心工具 | 字幕直接提到 | 结合异常处理，编写循环用 `next()` 手动遍历直到捕获 `StopIteration`，模拟 for 循环的底层行为 |
| `__iter__` | 魔术方法 | 可迭代对象的核心方法，由 `iter()` 自动调用，返回一个迭代器 | 字幕直接提到 | 在自定义类中仅实现 `__iter__` 而不实现 `__next__`，观察该类的实例是否仅能用于 for 循环而无法调用 `next()` |
| `__next__` | 魔术方法 | 迭代器的核心方法，由 `next()` 自动调用，返回下一个元素，超界时抛出 `StopIteration` | 字幕直接提到 | 在自定义类的 `__next__` 中添加打印语句，追踪每次调用时 `self` 内部状态的变化，理解迭代器状态的维护机制 |
| `StopIteration` | 内置异常 | 迭代器遍历完毕时由 `__next__` 抛出，通知调用方停止遍历，for 循环自动捕获该异常以优雅结束 | 字幕直接提到 | 手动在一个迭代器上连续调用 `next()` 直到触发该异常，打印异常信息，观察迭代器的"一次性"特性 |

## 二、概念对照表（知识点层面的概念实体）

| 名称 | 类型 | 在本 P 中的作用 | 来源 | 后续可练习方向 |
|------|------|----------------|------|----------------|
| 可迭代对象（Iterable） | 概念/数据类型 | 所有能被 for 循环遍历的对象的统称，内部必须实现 `__iter__` 方法，是迭代器的数据来源 | 字幕直接提到 | 整理 Python 中所有内置可迭代对象的类型，测试每种类型的 `__iter__` 方法返回值，理解"可迭代"与"迭代器"的层级关系 |
| 迭代器（Iterator） | 概念/数据类型 | 实现 `__iter__` 和 `__next__` 的对象，支持惰性计算和一次性遍历，是本 P 的核心讲解对象 | 字幕直接提到 | 对比 listiterator、dictiterator、setiterator 等不同迭代器对象的行为差异，观察它们在调用 `next()` 后的内存占用变化 |
| 惰性计算（Lazy Evaluation） | 概念/设计模式 | 迭代器和生成器的核心设计理念——按需生成数据，用到时才创建，用完即释放，无需一次性加载全部数据 | 字幕直接提到 | 设计对比实验：创建一个包含百万个元素的列表和一个等效迭代器，分别测量两者的初始化内存占用和遍历时间，量化惰性计算的优势 |
| for 循环（语法糖） | 语言特性 | 本 P 揭示 for 循环的底层原理——自动调用 `iter()`、`next()` 并捕获 `StopIteration`，本质上是对迭代器操作的简化语法 | 字幕直接提到 | 在自定义类中实现 `__iter__` 和 `__next__`，观察同一个对象同时用于 for 循环和手动 `next()` 时的行为差异，验证两者等价性 |

## 三、可迭代数据类型汇总

|------|------|----------------|------|----------------|
| 字符串（str） | 内置数据类型 | 作为可迭代对象的典型示例，演示字符串可被 for 循环逐字符遍历 | 字幕直接提到 | 用 `iter()` 获取字符串迭代器，用 `next()` 逐字符取出，理解字符串作为字符序列的底层表示 |
| 列表（list） | 内置数据类型 | 作为最常用的可迭代对象，用于演示迭代器获取和手动遍历流程 | 字幕直接提到 | 创建包含不同类型元素的混合列表，测试 `next()` 是否能按原顺序逐个返回各种类型的元素 |
| 字典（dict） | 内置数据类型 | 作为可迭代对象，演示字典的迭代行为（默认迭代键） | 字幕直接提到 | 分别获取字典的键迭代器、值迭代器和键值对迭代器，对比三者的 `next()` 输出差异 |
| 集合（set） | 内置数据类型 | 作为可迭代对象，演示无序集合的遍历行为 | 字幕直接提到 | 创建大集合后多次获取其迭代器，观察多次遍历的顺序是否一致，理解集合的哈希无序特性 |
| 元组（tuple） | 内置数据类型 | 作为可迭代对象，与列表对比，体现不可变性 | 字幕直接提到 | 对比列表迭代器和元组迭代器的内存占用和遍历性能 |
| range 对象 | 内置数据类型/生成器 | 作为可迭代对象，演示惰性生成整数序列的特性（本身不是迭代器） | 字幕直接提到 | 对比 `range(1000000)` 与等差列表的内存占用，验证 range 对象的惰性特性 |
| 文件对象（file object） | 内置数据类型 | 作为可迭代对象，演示按行惰性读取文件的机制，是迭代器实用价值的典型体现 | 字幕直接提到 | 用 `with open()` 打开大文件，使用文件迭代器逐行读取，观察内存占用远低于一次性读取全部内容的情况 |

## 七、备注与使用说明

1. 本 P 为概念导入性质，未涉及外部第三方库、在线工具或文档资源，所有资源均属于 Python 内置组件，学习门槛为零安装。

2. `collections.abc` 模块是 Python 标准库的一部分，位于安装目录 `Lib/collections/abc.py`，其中的抽象基类用于类型检查和协议验证，是 Python 面向对象设计中的重要组成部分。

3. 建议在学习完本 P 后，立即对比阅读 Python 官方文档中关于迭代器协议（Iterator Protocol）的章节（位置：The Python Standard Library → Built-in Types → Iterator Types），将视频中的概念与官方定义相互印证。

4. 本 P 的所有练习均可直接在 Python 交互式解释器（REPL）或 Jupyter Notebook 中完成，无需额外配置环境。
