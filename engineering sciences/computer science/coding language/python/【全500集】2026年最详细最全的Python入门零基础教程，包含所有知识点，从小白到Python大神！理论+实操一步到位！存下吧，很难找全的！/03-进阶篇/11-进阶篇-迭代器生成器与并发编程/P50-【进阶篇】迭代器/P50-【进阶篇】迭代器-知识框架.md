# P50 知识框架 - 【进阶篇】迭代器
## 本P位置

- 课程阶段：Python入门→进阶篇→迭代器和生成器章节
- 章节序号：第50P（共500P）
- 定位：章节导入，为P51生成器做概念铺垫
- 前置知识：Python基础语法、数据类型（字符串/列表/字典/集合等）

## 核心问题

- 什么样的对象能被for循环遍历
- for循环的底层工作原理是什么
- 迭代器如何实现"惰性计算"
- 自定义迭代器需要实现哪些方法

## 知识骨架

```
可迭代对象（Iterable）
├── 定义：实现__iter__方法的对象
├── 判断：isinstance(obj, Iterable)
├── 常见类型：字符串、列表、元组、字典、集合、range、文件对象
│
迭代器（Iterator）
├── 定义：实现__iter__和__next__方法的对象
├── 获取：iter(可迭代对象)
├── 前进：next(迭代器)
├── 终止：抛出StopIteration
├── 特性：惰性计算、一次性使用
│
for循环
├── 本质：迭代器的语法糖
├── 底层：自动调用iter()→循环调用next()→捕获StopIteration
│
自定义迭代器
├── 必须实现__iter__（返回self）和__next__
├── __next__内部需检测边界并raise StopIteration
```

## 关系与依赖

```
可迭代对象 --__iter__--> 迭代器
     ↑                      │
     │                      ↓
  for循环 <----__next__-- next()
                          ↑
                          │
              StopIteration异常
```

- 可迭代对象是迭代器的来源
- 迭代器一定是可迭代对象（实现__iter__返回自身）
- for循环依赖迭代器工作
- StopIteration是迭代器通知遍历结束的机制

## 输入 / 输出 / 前置 / 支撑

| 类型 | 内容 |
|------|------|
| 输入 | 任意Python对象（用于判断或遍历） |
| 输出 | 迭代器对象、元素值、布尔判断 |
| 前置依赖 | 基础语法、数据类型、for循环语法 |
| 支撑知识 | __iter__、__next__魔术方法概念 |
| 后续衔接 | 生成器（更简单的惰性计算实现） |

## 易错点

- 混淆可迭代对象与迭代器：可迭代对象只有__iter__，迭代器同时有__iter__和__next__
- 自定义迭代器__next__中先修改self.current再return导致少返回一个元素：必须用result临时保存原值
- 迭代器一次性使用：遍历结束后需重新调用iter()获取新迭代器
- 数值类型不可迭代：用isinstance判断时数值返回False
- for循环自动捕获StopIteration：手动调用next()时需自行处理异常

## 外部资源与对象

| 对象 | 角色 | 关系 |
|------|------|------|
| `collections.abc.Iterable` | 可迭代判断基类 | isinstance第二个参数 |
| `isinstance()` | 内置判断函数 | 判断obj是否为Iterable实例 |
| `iter()` | 内置函数 | 可迭代对象→迭代器 |
| `next()` | 内置函数 | 迭代器→下一元素 |
| `StopIteration` | 内置异常 | 迭代器终止信号 |
| `__iter__` | 魔术方法 | 可迭代对象核心，返回迭代器 |
| `__next__` | 魔术方法 | 迭代器核心，返回下一元素 |
