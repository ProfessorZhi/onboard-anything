# P43 讲解过程 - 【进阶篇】正则进阶规则
## 本P定位

本P是正则表达式基础语法体系中的进阶部分，承接前面单字符匹配、数量限定、位置限定等基础规则，讲解**分组（Group）**这一更高级的匹配机制。分组的作用是把正则表达式的一部分括起来形成独立的匹配单元，从而支持**多选一**和**引用已匹配内容**两种更复杂的匹配场景。

---

## 讲解过程

### 1. 引出分组概念

老师先点明本节主题是"正则进阶规则"，即**匹配分组**。分组的本质是用**括号**将部分规则括起来，主要用来实现两个功能：多选一、引用分组结果，适合更复杂的匹配场景。

### 2. 第一种用法：多选一（竖杠 `|` 连接）

老师先用语言描述规则逻辑：使用竖杠符号连接不同的子规则，表示匹配任意一个符合的规则。比如定义了三个选项 `cat`、`dog`、`pig`，那么字符串只要匹配上其中任意一个就算成功。

#### 演示多选一成功的情况

老师写了一段代码演示：

```python
import re
res = re.match(r'(cat|dog|pig)', 'cat')
print(res.group())
```

依次将目标字符串换成 `cat`、`dog`、`pig`，均能匹配成功，证明竖杠实现了"或"的关系。

#### 演示多选一失败的情况

接着将目标字符串换成 `bird`：

```python
res = re.match(r'(cat|dog|pig)', 'bird')
print(res.group())
```

运行结果匹配失败。老师解释原因：三个选项都不符合，自然匹配失败。这里等效于给正则加了**可选选项**的功能。

老师特别提醒：匹配失败时通过 `group` 方法提取结果会报错，必须先做判断再调用。

### 3. 第二种用法：引用分组结果（`\1` 语法）

老师介绍 `\1` 的含义：`\1` 表示引用前面第 N 个分组匹配到的内容（语法上 `\` 在代码中需要转义，实际写为 `\\1` 或在 raw string 中写为 `\1`）。

#### 演示场景：匹配成对的 HTML 标签

老师选择用 HTML 标签作为演示案例，因为这类标签天然是**成对出现**的，引用分组结果可以确保首尾标签一致。

目标字符串示例：

```
<html>loading</html>
```

老师逐步构建正则：

| 部分 | 正则片段 | 含义 |
|------|---------|------|
| 开始标签的尖括号 | `<` | 字面匹配 |
| 标签名（需要被引用） | `\w+` | 至少一个单词字符 |
| 开始标签的反尖括号 | `>` | 字面匹配 |
| 中间内容 | `.*` | 任意除换行符以外的任意字符 |
| 结束标签的尖括号 | `</` | 字面匹配 |
| 结束标签名（必须和前面一致） | `\1` | 引用第1个分组的内容 |
| 结束标签的反尖括号 | `>` | 字面匹配 |

关键一步：把 `\w+` 那一部分用**括号包起来**作为第一个分组，然后用 `\1` 在结尾引用它。完整正则：

```python
r'<(\w+)><.*</\1>'
```

#### 第一次运行遇到问题

老师第一次运行时报错 `AttributeError: 'NoneType' object has no attribute 'group'`，匹配失败了。老师排查后发现原因：`\1` 在 Python 普通字符串中会被解释为转义字符，需要使用 **raw string（前缀 `r`）** 来避免转义问题。

#### 第二次运行成功

添加 `r` 前缀后：

```python
import re
res = re.match(r'<(\w+)><.*</\1>', '<html>loading</html>')
print(res.group())
```

匹配成功。老师验证：如果首尾标签名不一致（如 `<html>loading</div>`），匹配就会失败，因为 `\1` 引用的内容和实际的结束标签名对不上。

#### 引用分组结果的适用场景

老师总结：这种规则在处理 **HTML、XML 等成对标签**，或者需要匹配**重复内容**的场景时非常有用。

### 4. 章节总结

老师用一段完整的总结回顾了正则表达式的核心知识点，按模块梳理：

**正则表达式基础认知**
- 正则表达式是描述字符串模式的规则
- 核心用途：匹配验证、提取、替换字符串
- Python 使用正则的流程：导入 re 模块 → 定义规则 → 执行匹配 → 提取结果

**基础规则回顾**

单字符匹配：

| 符号 | 含义 |
|------|------|
| `.` | 任意单个字符（除换行符外） |
| `\d` | 任意一个数字字符 |
| `\w` | 任意一个单词字符（字母、下划线、数字、汉字） |
| `[a-z]` | 任意一个小写字母 |
| `[A-Z]` | 任意一个大写字母 |

数量限定符：

| 符号 | 含义 |
|------|------|
| `*` | 可以匹配任意数量（零次或多次） |
| `+` | 至少匹配一次 |
| `?` | 最多只匹配一次（零次或一次） |
| `{m,n}` | 指定匹配 m 到 n 次 |

位置限定：

| 符号 | 含义 |
|------|------|
| `^` | 匹配字符串开头 |
| `$` | 匹配字符串结尾 |

分组相关：

| 符号 | 含义 |
|------|------|
| `(规则)` | 创建一个分组 |
| `\|` | 多选一，表示"或者" |
| `\1` | 引用第 N 个分组的内容 |

**两个关键提醒**

1. 匹配失败返回 `None`，必须先判断再调用 `group`，否则程序报错导致后续代码无法执行。
2. 带有反斜杠的规则（如 `\d`、`\w`、`\1`）一定要使用 **raw string（`r''`）**，避免转义冲突问题。

老师最后鼓励学员：正则表达式的核心是**用规则描述模式**的逻辑，掌握这些基础规则和用法就能应对大部分字符串处理场景。课后建议通过三个实战练习巩固：验证身份证号码、提取文本中的网址、过滤评论中的敏感字符。

## 本P结论

1. **分组（Group）**是用括号将正则的某部分括起来形成的独立匹配单元，支持两种进阶用法。
2. **多选一**：`cat|dog|pig` 表示匹配 cat、dog、pig 任一选项，竖杠实现"或"的关系。
3. **引用分组结果**：`\1` 表示引用第 1 个分组已匹配到的内容，常用于匹配成对的 HTML 标签或重复内容，确保首尾一致。
4. 使用带反斜杠的正则规则时，**必须配合 raw string（`r''`）** 前缀，防止转义冲突。
5. 调用 `group()` 前**必须先判断匹配结果是否为 None**，否则程序会因 AttributeError 中断。
6. 正则表达式的核心是**用规则描述模式**，理解这一逻辑后，多敲多练即可熟练掌握。
