# P43 知识框架 - 【进阶篇】正则进阶规则
## 本P位置

正则表达式语法体系 → 进阶篇 → 分组（Group）
前置：单字符匹配、数量限定、位置限定
本P作用：引入分组机制，赋予正则表达式多选一与内容引用的能力

## 核心问题

如何用括号将正则的一部分独立为分组，实现"匹配多个候选"和"引用已匹配内容"？

## 知识骨架

```
分组（Group）
├── 语法：(规则)
├── 用途一：多选一 → 使用 | 连接候选
│   └── 示例：(cat|dog|pig)
└── 用途二：引用分组结果 → 使用 \N 引用第N个分组
    └── 示例：<(\w+)><.*</\1>  确保首尾标签名一致
```

## 关系与依赖

```
单字符匹配（\w）→ 作为分组内容 → 被 \1 引用
数量限定（+）   → 修饰分组内规则
位置限定（^/$） → 决定整个正则匹配范围
```

分组依赖前置的单字符匹配和数量限定；引用（\1）依赖前置的分组定义。

## 输入 / 输出 / 前置 / 支撑

| 节点 | 类型 | 说明 |
|------|------|------|
| `(规则)` | 输入 | 括号包裹的子表达式 |
| `\|` | 输入 | 多选一连接符 |
| `\1` | 输出/工具 | 引用第N个已匹配分组 |
| re.match() | 函数 | 执行正则匹配，返回 Match 对象或 None |
| raw string（r''） | 依赖 | 所有含反斜杠的正则必须用 raw string |
| group() | 方法 | 从 Match 对象提取匹配结果 |
| None 检查 | 前置 | 调用 group() 前必须判断 res is not None |

## 易错点

1. `\1` 在普通字符串中被 Python 转义解释为 SOH 控制字符，必须使用 raw string（`r''`）
2. 匹配失败时 re.match() 返回 None，此时调用 group() 会抛 AttributeError
3. 多选一只匹配第一个符合的候选，而非全部
4. 分组编号按左括号从左到右从 1 开始计数

## 外部资源与对象

无外部依赖。本P完全基于 Python 内置 `re` 模块进行演示。
