# P41 资源清单 - 【进阶篇】正则表达式使用
---

## 1. re 模块

- **名称**：re 模块
- **类型**：标准库
- **在本 P 中的作用**：Python 内置的正则表达式标准库，regular expression 的缩写。本 P 所有正则操作（match、后续的 search/findall/sub 等）的底层依赖，无需安装，直接 `import re` 导入即可使用
- **来源**：字幕直接提到
- **建议方向**：阅读官方文档中 re 模块的章节，了解该模块提供的全部函数及其适用场景

---

## 2. re.match()

- **名称**：re.match()
- **类型**：函数
- **在本 P 中的作用**：re 模块的入门级匹配函数，从字符串开头进行匹配，成功返回 match 对象，失败返回 None。本 P 实战演示的核心函数，用于帮助学习者理解正则三步流程
- **来源**：字幕直接提到
- **建议方向**：编写测试脚本，分别用满足规则和不满足规则的字符串调用该函数，观察返回值差异

---

## 3. match.group()

- **名称**：match.group()
- **类型**：方法
- **在本 P 中的作用**：match 对象的成员方法，用于提取匹配到的内容。本 P 强调必须在确认 match 对象非空后才能调用，否则触发 AttributeError
- **来源**：字幕直接提到
- **建议方向**：对比 .group() 与 .group(0) 的返回值，尝试查看 match 对象的其他方法（如 .start()、.end()、.span()）

---

## 4. 原始字符串语法 (r'')

- **名称**：r'' / r"" 原始字符串
- **类型**：语法特性
- **在本 P 中的作用**：正则规则的标准写法前缀，加 r 后反斜杠 \ 不会被 Python 解释器当作转义字符处理，消除语法警告。本 P 演示的 `r'H\w+'` 即使用该语法
- **来源**：字幕直接提到
- **建议方向**：对比有无 r 前缀时同一正则字符串的行为差异，例如 `r'\w+'` 与 `'\w+'` 在实际匹配中是否等价

---

## 5. 正则表达式规则字符串

- **名称**：正则表达式规则字符串
- **类型**：模式描述
- **在本 P 中的作用**：作为 re.match() 的第一个参数传入，定义匹配规则。本 P 演示了 `\d{11}`（11位数字）、`\w+@\w+\.\w+`（邮箱格式）、`H\w+`（以H开头的单词字符序列）等示例
- **来源**：字幕直接提到
- **建议方向**：学习原字符（`.`、`\d`、`\w`、`\s` 等）和量词（`*`、`+`、`?`、`{n}` 等）的基础语法，为后续小节做准备

## 10. None 判断模式

- **名称**：if result: ... else: ... None 判断模式
- **类型**：代码模式
- **在本 P 中的作用**：本 P 强调的安全写法，先判断匹配结果是否为 None，再决定是否调用 .group()，防止程序崩溃
- **来源**：字幕直接提到
- **建议方向**：在所有正则匹配代码中统一使用该模式，形成防御性编程习惯
