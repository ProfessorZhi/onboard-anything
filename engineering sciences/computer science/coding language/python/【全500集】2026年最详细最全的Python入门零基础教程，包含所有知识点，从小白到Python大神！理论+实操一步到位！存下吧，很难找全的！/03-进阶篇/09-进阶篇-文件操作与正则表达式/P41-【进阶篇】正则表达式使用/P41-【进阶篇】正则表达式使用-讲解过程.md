# P41 讲解过程 - 【进阶篇】正则表达式使用
## 本P定位

本 P 是 Python 正则表达式进阶篇的第一个小节，聚焦于正则表达式的核心匹配规则与 re.match() 的基本用法。课程以「什么是正则表达式 → 能做什么 → 怎么在 Python 中使用 → match 函数的实战演示」为推进线索，配套三个注意事项，帮助学习者建立正则表达式的基础认知框架与初步动手能力。本 P 为后续两节（常用函数、贪婪/非贪婪匹配）铺垫基础。

## 讲解过程

### 一、正则表达式的概念引入（0:00–2:30）

**教学推进：** 开门见山提出正则表达式是 Python 中「非常实用的字符串处理工具」，紧接着用四个具体场景说明它能解决什么问题——验证手机号格式、批量提取邮箱、过滤敏感字符。强调这些场景「不用写复杂的逻辑」就能搞定字符串处理，是「数据清洗、文本分析的必备技能」。

**关键话术：** 「它能够快速地去查找验证、提取符合特定规则的字符串」「繁琐的字符串处理，效率直接拉满」

### 二、课程结构预告（2:30–4:50）

**教学推进：** 将本次课程拆为三个小节，逐个说明学习目标：

1. 正则表达式的使用 → 学习核心匹配规则（原字符、量词等），掌握用规则描述字符串模式的方法
2. 常用函数 → 熟练运用 re 模块中的 search、findall、sub 等高频函数落地实际匹配需求
3. 贪婪匹配和非贪婪匹配 → 搞懂两种匹配模式的核心区别，避开匹配结果偏差的坑

**教学意图：** 让学习者对整个进阶篇的内容版图有预期，便于后续有针对性地消化。

### 三、正则表达式到底是什么（4:50–7:00）

**教学推进：** 给出一个简明定义——「正则表达式本质就是一套描述字符串模式的规则，就像一个智能的过滤器」。用类比说明：我们通过规则告诉程序要找的内容「长什么样子」，程序就能自动匹配出符合要求的内容。

**举例演示：**  
- 找 11 位数字手机号 → 写一个 `\d{11}` 规则，程序就知道找 11 位连续数字  
- 匹配邮箱 → 写 `\w+@\w+\.\w+` 规则，可匹配 `abc123@qq.com` 这样的邮箱格式

**话术特点：** 「后面会详细讲解」作为悬念钩子，暗示语法细节留待后续展开。

### 四、正则表达式的四大核心用途（7:00–11:30）

**教学推进：** 将正则的核心功能归纳为四类，配合生活化场景逐一说明：

| 用途 | 场景举例 |
|------|----------|
| 验证/判断 | 验证手机号、邮箱、身份证格式是否正确 |
| 查找 | 从新闻中提取所有日期，从评论中提取所有用户名 |
| 提取 | 从邮箱地址中单独提取用户名，从手机号 `130131111222` 中提取号段 `130` |
| 替换 | 过滤文本中的特殊字符，批量修改符合规则的内容 |

**教学总结：** 「有了它就再也不用写繁琐的代码逐个判断字符，效率能够大大提升」

### 五、Python 中使用正则：re 模块导入（11:30–13:50）

**教学推进：** 强调 Python 内置了 `re` 模块（regular expression 的缩写），无需额外安装，直接 `import re` 导入即可使用。「这是所有正则操作的基础，一定不要忘记」。

**代码演示：** `import re`

### 六、正则表达式的三步使用流程（13:50–15:30）

**教学推进：** 提炼出正则操作的通用流程框架：

1. **定义规则** → 用字符串表示匹配模式  
2. **执行匹配** → 调用 re 模块的匹配函数  
3. **提取结果** → 通过 `group()` 方法获取匹配内容

**教学意图：** 建立一个抽象的方法论，让学习者在接触具体函数前先有一个全局流程认知。

### 七、re.match() 函数详解（15:30–24:00）

#### 7.1 函数定位

**教学推进：** 选 `re.match()` 作为入门级函数，因为它「从字符串的开头开始匹配，很适合入门练习」。

#### 7.2 函数参数说明

- 第一个参数：正则表达式（规则本身）
- 第二个参数 `string`：待匹配的源字符串

#### 7.3 返回值特性

**教学推进：** 区分两种返回值：

- **匹配成功** → 返回一个 `match` 对象 → 可调用 `.group()` 提取匹配内容  
- **匹配失败** → 返回 `None` → 若对 `None` 调用 `.group()` 会报错

**代码框架演示（未写完整代码但展示了思路）：**

```
result = re.match(正则规则, 待匹配字符串)
if result:
    print(result.group())
else:
    print("匹配失败")
```

**强调：** 「因为 None 没有任何方法，直接调用 group 肯定是会报错的」

#### 7.4 实战演示：以 H 开头的字符串匹配（20:00–24:00）

**教学推进：** 演示一个完整的匹配流程——

1. 导入模块：`import re`
2. 直接在 `re.match()` 中写规则：匹配「以 H 开头、后面是多个单词字符」的字符串  
   - 规则写为 `r'H\w+'`（`\w` 匹配单个单词字符，`+` 表示一个或多个）
   - 说明：正则字符串前加 `r` 是为了取消转义，消除语法警告
3. 待匹配字符串：`"hello world"`
4. 运行结果：返回 `<re.Match object>`，因为 `hello` 满足以 H 开头

**对比演示：**  
- 改待匹配字符串为 `"world hello"` → 返回 `None`（因为开头是 `w` 不满足规则）  
- 说明：`re.match()` 只看字符串开头，开头不符合规则就直接失败返回 `None`

#### 7.5 结果提取与 None 判断（24:00–26:30）

**教学推进：** 在匹配结果外层包一层变量接收和条件判断：

```
res = re.match(r'H\w+', 'hello world')
if res:
    print(res.group())
else:
    print("匹配失败")
```

**原理解释：**  
- `res` 非空（match 对象）作为条件 → 为 `True` → 执行 `res.group()` 提取内容  
- `res` 为 `None` 作为条件 → 为 `False` → 执行 else 分支 → 避免报错导致程序崩溃

### 八、两个核心注意事项（26:30–结束）

**教学推进：** 汇总本 P 的关键注意点，帮助学习者避坑：

1. **`re.match()` 只看字符串开头**  
   - 开头不符合规则就直接失败返回 `None`，不会继续往后匹配

2. **匹配失败返回 `None`，不能直接调用 `group()`**  
   - 在提取数据前必须判断结果是否为 `None`，否则程序会报错，后续代码全部无法执行

**教学收尾话术：** 「我们要防止程序崩溃嘛」

## 本P结论

1. **正则表达式本质是一套描述字符串模式的规则**，通过它告诉程序要找的内容「长什么样子」，程序自动完成匹配，无需逐字符判断逻辑

2. **正则的四大核心用途**：验证（判断格式是否符合）、查找（从文本中找出所有匹配项）、提取（从匹配结果中取出关键片段）、替换（修改或过滤符合规则的内容）

3. **Python 中使用正则需先导入 re 模块**：`import re`，该模块为内置模块，无需额外安装

4. **正则使用三步流程**：定义规则（写正则表达式字符串）→ 执行匹配（调用 re.match/search/findall 等）→ 提取结果（match 对象调用 .group()）

5. **re.match() 从字符串开头匹配**，开头不符合规则直接返回 `None`，后续不再尝试匹配

6. **匹配结果必须做 None 判断后再提取**：匹配失败返回 `None`，直接对 `None` 调用 `.group()` 会触发 `AttributeError`，导致程序崩溃

7. **正则字符串建议使用原始字符串 `r''`**：避免反斜杠 `\` 被 Python 解释器当作转义字符，消除语法警告
