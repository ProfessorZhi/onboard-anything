# P41 知识框架 - 【进阶篇】正则表达式使用
## 本P位置

Python 正则表达式进阶篇第 1 小节。位于「基础语法→进阶技能」过渡区。承接字符串操作基础，为后续「常用函数」和「贪婪/非贪婪匹配」两个小节提供概念框架与 re.match() 入门技能。

---

## 核心问题

如何用一套规则精确描述字符串模式，让程序自动完成验证、查找、提取、替换操作。

---

## 知识骨架

```
正则表达式 = 描述字符串模式的规则
    ↓ 作用对象
Python re 模块
    ↓ 提供匹配函数
re.match() — 从开头匹配，返回 match 对象或 None
    ↓ 提取方法
match.group() — 提取匹配内容
```

---

## 关系与依赖

```
re 模块（内置）
    │
    ├─ re.match(正则规则, 字符串) → match 对象 | None
    │
    └─ match.group() → 提取内容（依赖 match 非空）
              ↓
         None 判断（前置保护）
```

三层依赖链：re 模块 → 匹配函数 → 结果提取。正则规则是字符串形式的输入，决定匹配行为。

---

## 输入 / 输出 / 前置 / 支撑

| 节点 | 类型 | 说明 |
|------|------|------|
| 正则规则 | 输入 | 字符串形式的模式描述，如 r'H\w+' |
| 待匹配字符串 | 输入 | 源文本 |
| match 对象 | 输出 | 匹配成功时返回 |
| None | 输出 | 匹配失败时返回 |
| .group() | 提取方法 | 从 match 对象提取内容 |
| import re | 前置 | 所有正则操作的基础 |

---

## 易错点

**re.match() 只看开头**  
开头不符合规则直接返回 None，不会继续往后匹配。

**对 None 调用 .group() 必崩**  
None 无任何方法，直接调用触发 AttributeError，导致后续代码全部中断。必须先做 None 判断。

**正则字符串推荐使用 r'' 前缀**  
原始字符串避免反斜杠被 Python 当转义符，消除警告。

---

## 外部资源与对象

| 资源 | 角色 |
|------|------|
| **re 模块** | Python 内置标准库，所有正则操作的底层依赖，导入方式为 import re |
| **r'' 原始字符串** | 正则规则的标准写法，防止转义干扰 |
| **match.group()** | match 对象的成员方法，作用为提取匹配内容 |

---

## 核心结论

正则表达式是一套描述字符串模式的规则，用于告诉程序「要找的内容长什么样」。Python 中通过 re 模块执行正则操作，标准流程为：定义规则（正则字符串）→ 执行匹配（调用 re 函数）→ 提取结果（调用 .group()）。re.match() 从字符串开头匹配，成功返回 match 对象，失败返回 None，None 必须先判断再提取。
