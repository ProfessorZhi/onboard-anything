# P45 讲解过程 - 【进阶篇】贪婪与非贪婪
## 本P定位

本P属于“进阶篇”，在前序课程已讲解正则核心规则和 `re` 模块常用函数的基础上，进一步深入两个容易踩坑的匹配模式——**贪婪匹配（Greedy）** 与 **非贪婪匹配（Non-Greedy / Lazy）**。两者的核心差异在于量词对匹配次数的处理策略不同，使用场景的选择会直接影响正则匹配结果的精确程度。

---

## 讲解过程

### 1. 概念引入：为什么需要区分这两种模式

**（0:00 – 0:55）**

老师开门见山指出：同样的正则规则，在不同模式下进行匹配，**结果可能完全不同**。这是很多新手容易踩的坑。区分两者的目的有两个：
- 避免匹配结果范围过宽（过度匹配）
- 避免漏抓关键内容（匹配不足）

核心口诀快速给出：
- **贪婪模式**：能够多拿就多拿
- **非贪婪模式**：能够少拿就少拿

---

### 2. 生活类比：装苹果的比喻

**（0:55 – 1:40）**

用装苹果的生活场景来类比两种模式的运行逻辑：

| 模式 | 行为描述 |
|------|----------|
| 贪婪模式 | 拿一个袋子装苹果，**能装多少装多少**，直到袋子装不下才停，核心是尽可能多拿 |
| 非贪婪模式 | 只要**装够最低要求**就停，比如要求至少装两个，装完两个就不再多拿，核心是尽可能少拿 |

这个“装苹果的要求”对应到正则中，就是**量词**——`?`、`*`、`+`、`{m,n}` 等表示匹配次数的符号。

---

### 3. 核心概念精讲：模式的定义与触发方式

**（1:40 – 2:52）**

#### 3.1 两种模式的定义

- **贪婪模式**：正则的默认模式，**不需要额外设置**，量词会自动匹配尽可能多的字符，直到无法继续。
- **非贪婪模式**：需要**手动开启**，在量词后面**加上一个问号 `?`**，量词就会匹配尽可能少的字符，只要满足最低要求就停止。

#### 3.2 关键前提（必须记住）

两种模式**只影响带数量限定的量词**，不影响匹配单个字符的规则。

**不受影响的单个字符匹配规则**包括：
- `.`（匹配任意单个字符）
- `\d`、`\D`（数字/非数字）
- `[a-z]`（字符类）
- 等等

**受影响的量词**只有以下几个：

| 量词 | 含义 | 匹配次数范围 |
|------|------|--------------|
| `*` | 任意次数 | 0 到 ∞ |
| `+` | 至少一次 | 1 到 ∞ |
| `{m,n}` | m 到 n 次 | m 到 n |
| `?` | 最多一次 | 0 到 1 |

---

### 4. 贪婪模式代码演示

**（2:58 – 6:24）**

#### 演示环境

```python
import re
```

#### 4.1 `re.match()` 方法的基本使用

- 使用 `\d` 匹配单个数字字符作为基础规则
- 目标字符串：`"12345678ABC"`

#### 4.2 逐个演示各量词的贪婪行为

**（1）`*` 星号量词**

```python
# 星号范围：0 到 正无穷
res = re.match(r'\d*', "12345678ABC")
print(res.group())  # 输出: 12345678（所有数字全部匹配）
```

结论：有多少数字就匹配多少数字，能多拿就多拿。

**（2）`+` 加号量词**

```python
# 加号范围：1 到 正无穷
res = re.match(r'\d+', "12345678ABC")
print(res.group())  # 输出: 12345678（同样是所有数字全部匹配）
```

结论：和星号表现一致，有多少要多少。

**（3）`{m,n}` 范围量词**

```python
# {2,5} 范围：2 到 5 次
res = re.match(r'\d{2,5}', "123456")
print(res.group())  # 输出: 12345（按最多次数匹配，拿满5个）
```

当字符串有6个数字时，贪婪模式会取5个（最多次数）；有4个就取4个，有3个就取3个。

#### 4.3 贪婪模式匹配优先级总结

1. **先尽可能多拿**符合量词的字符
2. **再看整体是否符合规则**
3. 只要后面没有冲突，就一直往下匹配
4. 直到遇到不匹配的字符，或达到量词上限才停止

### 5. 非贪婪模式代码演示

**（6:32 – 8:45）**

#### 开启方式

在量词后面**加上问号 `?`**

```python
量词 + ?
# 例如：\d*?  \d+?  \d{2,5}?
```

#### 逐个演示各量词的非贪婪行为

**（1）`*?` 星号加问号**

```python
res = re.match(r'\d*?', "12345678ABC")
print(res.group())  # 输出: ''（空字符，按最少次数0次匹配）
```

因为星号的最低次数是0，满足最低要求就立刻停止。

**（2）`+?` 加号加问号**

```python
res = re.match(r'\d+?', "12345678ABC")
print(res.group())  # 输出: '1'（按最少次数1次匹配）
```

加号的最低次数是1，所以只匹配到1个数字就停止。

**（3）`{2,5}?` 范围加问号**

```python
res = re.match(r'\d{2,5}?', "12345678ABC")
print(res.group())  # 输出: '12'（按最少次数2次匹配）
```

最低次数是2，所以只匹配到2个数字就停止。

#### 非贪婪模式匹配优先级总结

1. **先拿满足量词最低要求的字符**
2. 只要能够达到最低标准，**立刻停止匹配**
3. 不管后面还有多少符合规则的字符，都不再继续

### 6. 对比表格：两种模式在同规则下的差异

**（9:00 – 10:38）**

以字符串 `"123456ABC"` 和 `"A1B2C3B"` 为测试数据，逐一对比：

| 正则规则 | 贪婪模式（默认） | 非贪婪模式（加 `?`） |
|----------|------------------|----------------------|
| `\d*` | `"123456"`（全部数字） | `""`（空，按最少0次） |
| `\d+` | `"123456"`（全部数字） | `"1"`（按最少1次） |
| `\d{2,5}` | `"12345"`（拿满5个） | `"12"`（按最少2次） |
| `.*` | `"123456ABC"`（全部内容） | `""`（空） |
| `A.*B` | `"A1B2C3B"`（A到最远的B） | `"A1B"`（A到最近的B） |

#### 重点案例解析：`A.*B` 的两种模式表现

**贪婪模式 `A.*B`**

```python
res = re.search(r'A.*B', "A1B2C3B")
print(res.group())  # 输出: 'A1B2C3B'
```

匹配逻辑：从第一个 A 开始，往后一直匹配到**最远的** B，把整个字符串都吞进去了。

**非贪婪模式 `A.*?B`**

```python
res = re.search(r'A.*?B', "A1B2C3B")
print(res.group())  # 输出: 'A1B'
```

匹配逻辑：从第一个 A 开始，只匹配**最近的**那个 B，精确匹配到 `A1B`。

### 7. 实战场景：HTML 标签的精准提取

**（11:47 – 16:45）**

这是本 P 的**重点实战案例**，充分说明非贪婪模式的实用价值。

#### 目标场景

从包含多个 `<div>` 标签的 HTML 字符串中，**只提取第一个标签**。

#### 测试数据

```html
<div>第一部分</div><div>第二部分</div><div>第三部分</div>
```

#### 贪婪模式的问题

```python
res = re.search(r'<div>.*</div>', html_string)
print(res.group())
# 输出: <div>第一部分</div><div>第二部分</div><div>第三部分</div>
```

问题：**从第一个 `<div>` 一路匹配到最后一个 `</div>`**，把整个字符串都匹配上了，属于**过度匹配**。

#### 非贪婪模式的正确解法

```python
res = re.search(r'<div>.*?</div>', html_string)
print(res.group())
# 输出: <div>第一部分</div>
```

正确原因：匹配到**最近的**那对 `<div></div>` 标签，精准提取出第一部分。

#### 实战意义

- 爬虫场景中，爬取下来的 HTML 数据往往包含**很多个**成对标签
- 如果只需要获取**最近的一个**或单个标签内容，**非贪婪模式几乎是必须的**
- 非贪婪模式能够**避免匹配范围过度扩大**，是实战中不可或缺的技巧

---

### 8. 使用建议与场景选择

**（16:45 – 18:15）**

| 场景 | 推荐模式 | 说明 |
|------|----------|------|
| 提取**最近**匹配的单片段内容 | 非贪婪 `*?`、`+?`、`{m,n}?` | 如 `A.*?B` 找最近的 B |
| 提取 HTML/XML 标签时防止吞掉多个标签 | 非贪婪 `.*?` | 精准匹配单个成对标签 |
| 明确需要**最少匹配**（如只需前两个数字） | 非贪婪 `{2,5}?` | 匹配到最低次数就停止 |
| 提取**完整**的、所有符合的内容 | 贪婪（默认） | 不需要额外设置 |

**如果不确定用哪种**——两种模式都试一下，对比结果就能快速判断哪种更符合需求。

## 本P结论

1. **模式定义**
   - **贪婪模式**：默认模式，量词**尽可能多**匹配字符，不用加额外符号
   - **非贪婪模式**：在量词后面加 **`?`**，量词**尽可能少**匹配字符

2. **影响范围**
   - 两种模式**只针对量词**（`*`、`+`、`{m,n}`、`?`）生效
   - 单个字符匹配规则（`.`、`\d`、`[a-z]` 等）**完全不受影响**

3. **核心区别**
   - **贪婪**：能多拿就多拿，直到无法继续
   - **非贪婪**：能少拿就少拿，满足最低要求就停止

4. **使用技巧**
   - 需要提取**最近匹配的单片段** → 用非贪婪
   - 需要提取**完整的全部符合内容** → 用贪婪（默认）
   - 不确定时两种模式都试一下，对比结果即可
