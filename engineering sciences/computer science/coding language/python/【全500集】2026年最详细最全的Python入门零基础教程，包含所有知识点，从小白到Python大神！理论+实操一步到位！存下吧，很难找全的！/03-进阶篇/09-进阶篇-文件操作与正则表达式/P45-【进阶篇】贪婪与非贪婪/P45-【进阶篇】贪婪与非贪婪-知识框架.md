# P45 知识框架 - 【进阶篇】贪婪与非贪婪
## 本P位置

进阶篇第45课，位于“正则基础→re模块函数→贪婪/非贪婪”的学习路径上。前置知识是正则量词（`*`、`+`、`?`、`{m,n}`）和`re.match()`/`re.search()`的基本使用。本P为后续复杂正则应用（HTML解析、爬虫数据提取）提供匹配策略选择能力。

## 核心问题

- 同一个量词，为什么匹配结果范围不同
- 何时该选贪婪、何时该选非贪婪
- 如何避免`.*`过度吞掉多个目标片段

## 知识骨架

```
匹配模式
├── 贪婪模式（默认）
│   ├── 触发：无需额外符号，量词默认行为
│   ├── 策略：能多拿就多拿，直到无法继续
│   └── 结果：匹配尽可能长的内容
│
└── 非贪婪模式（Lazy）
    ├── 触发：量词后加 ?
    ├── 策略：能少拿就少拿，满足最低要求即停
    ├── 结果：匹配尽可能短的内容
    └── 适用：提取单个/最近片段
```

**核心口诀**：贪婪→多拿；非贪婪→少拿。

## 关系与依赖

```
正则基础（量词）
    │
    ├── * → {0,∞}  受模式影响
    ├── + → {1,∞}  受模式影响
    ├── {m,n}      受模式影响
    ├── ? → {0,1}  受模式影响
    │
    └── 单字符规则（.、\d、[a-z]） 不受模式影响

          ↓
     re 模块
    ├── match()  演示从头匹配
    └── search() 演示任意位置搜索

          ↓
     匹配模式（二选一）
    ├── 贪婪（默认）
    └── 非贪婪（+?）
```

依赖链路：正则量词 → 匹配模式选择 → re函数执行 → 得到匹配结果。

## 输入 / 输出 / 前置 / 支撑

| 类型 | 内容 |
|------|------|
| **输入** | 正则表达式（带/不带`?`）、待匹配字符串 |
| **输出** | 匹配结果字符串 |
| **前置** | `import re`、量词`* + ? {m,n}`含义、`re.match()`/`re.search()`用法 |
| **支撑** | `re.match()`（从头匹配）、`re.search()`（全局搜索） |

## 易错点

- **误以为非贪婪是默认**：非贪婪必须加`?`，贪婪才是默认行为
- **忽略单个字符规则不受影响**：`.`、`\d`、`[a-z]`等不受量词模式影响
- **`.*`贪婪过度匹配**：从第一个开始一直吞到最后一个，吞掉整个字符串
- **模式选择错误**：需要单个片段时用了贪婪，需要全部内容时用了非贪婪
- **不确定时未对比测试**：两种模式都试一下，对比结果即可判断

## 外部资源与对象

```
re 模块（Python 标准库）
├── match(pattern, string)  从字符串开头匹配
└── search(pattern, string) 在字符串任意位置搜索

两个函数都是本P演示匹配行为的工具载体。
```
