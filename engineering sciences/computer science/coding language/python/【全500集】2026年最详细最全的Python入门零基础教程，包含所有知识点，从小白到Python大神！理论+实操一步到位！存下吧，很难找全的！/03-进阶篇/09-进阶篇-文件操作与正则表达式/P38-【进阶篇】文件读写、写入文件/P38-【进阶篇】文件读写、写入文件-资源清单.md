# P38 资源清单 - 【进阶篇】文件读写、写入文件
## 一、核心内置函数 / 方法

| 名称 | 类型 | 在本 P 中的作用 | 来源 | 后续可练习方向 |
|------|------|----------------|------|----------------|
| open() | 内置函数 | 打开文件，支持路径、模式、编码三个核心参数 | 字幕直接提到 | 练习不同参数组合，理解各参数默认值 |
| read() | 文件对象方法 | 按字符/字节读取内容，不传参读取全部 | 字幕直接提到 | 练习 read(n) 分段读取大文件 |
| readline() | 文件对象方法 | 逐行读取，返回字符串，指针自动移动 | 字幕直接提到 | 用 readline() 配合循环处理大日志文件 |
| readlines() | 文件对象方法 | 读取所有行返回列表，便于批量处理 | 字幕直接提到 | 结合列表操作练习行筛选、统计功能 |
| write() | 文件对象方法 | 写入文本字符串或二进制字节串 | 字幕直接提到 | 练习数据持久化，将变量内容写入文件 |
| tell() | 文件对象方法 | 返回当前文件指针位置（字节数） | 字幕直接提到 | 结合 seek() 调试指针位置变化 |
| seek(offset, whence) | 文件对象方法 | 移动文件指针到指定位置 | 字幕直接提到 | 在读写切换前使用 seek(0) 定位指针 |
| close() | 文件对象方法 | 手动关闭文件释放资源 | 字幕直接提到 | 对比 with 语句的自动关闭机制 |
| print() 的 end 参数 | 内置函数参数 | 配合 readline() 消除多余空行 | 基于上下文推断 | 练习 print() 其他参数如 sep 的用法 |

## 二、核心概念 / 参数

| 名称 | 类型 | 在本 P 中的作用 | 来源 | 后续可练习方向 |
|------|------|----------------|------|----------------|
| 访问模式 | 参数体系 | 决定文件读写权限与创建行为，r/w/a/r+/w+/a+ 六种基础模式 | 字幕直接提到 | 用决策树练习根据需求选择正确模式 |
| 二进制模式 (b) | 参数后缀 | rb/wb/ab/r+b/w+b/a+b，处理图片等非文本文件 | 字幕直接提到 | 备份各种格式的二进制文件 |
| encoding='utf-8' | 参数 | 指定文件编码，解决 Windows GBK 与 UTF-8 冲突 | 字幕直接提到 | 练习不同编码文件的读写，验证乱码原因 |
| 文件指针 | 概念 | 标记读写起始位置，不同模式默认值不同 | 字幕直接提到 | 用 tell() 和 seek() 追踪指针移动轨迹 |
| CP936 / GBK | 系统编码 | Windows 系统默认编码，与 UTF-8 不兼容导致乱码 | 字幕直接提到 | 了解其他常见编码（GB2312、ASCII、BIG5） |

## 五、常见错误对照

| 错误类型 | 触发场景 | 错误信息 | 来源 | 后续可练习方向 |
|----------|----------|----------|------|----------------|
| 文件不存在 | r/r+ 模式下路径错误或文件缺失 | FileNotFoundError | 字幕直接提到 | 编写脚本前先检查文件是否存在 |
| 权限不匹配 | r 模式执行写入操作 | UnsupportedOperation | 字幕直接提到 | 养成先确认模式再操作的习惯 |
| 编码错误 | 未指定 encoding 且文件编码不匹配 | UnicodeDecodeError | 字幕直接提到 | 用 try-except 捕获编码错误并处理 |
| 读不到内容 | w+/a+ 模式写后立即读，未使用 seek(0) | 空字符串 | 字幕直接提到 | 在所有读写切换处添加 seek(0) |
