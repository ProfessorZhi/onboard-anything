# P38 讲解过程 - 【进阶篇】文件读写、写入文件
## 本P定位

本P是 Python 进阶篇关于文件操作的核心章节，承接上一 P 的文件基础概念，系统讲解文件的读写操作。重点围绕三个维度展开：访问模式的选择、读取方法的适用场景差异、中文编码问题的规避。同时引入了二进制文件处理（以图片备份为例）和文件指针操作作为进阶知识点。

## 讲解过程

### 一、文件访问模式体系

开篇即强调访问模式是文件操作的重点内容，指出"权限不匹配会导致报错"，需要先搞明白模式再去学习具体读写方法。

#### 模式对比表格讲解

以表格形式列举六种核心模式的特性差异，便于对照查阅：

| 模式 | 特性描述 |
|------|----------|
| **r** | 只读模式（默认），文件不存在报错，只能读不能写 |
| **w** | 只写模式，文件不存在则创建，文件存在则覆盖，只能写不能读 |
| **a** | 追加模式，文件不存在创建，文件存在追加到末尾，只能写不能读 |
| **r+** | 读写模式，文件不存在报错，以 r 为主，可同时读写，不覆盖原内容 |
| **w+** | 读写模式，文件不存在创建，文件存在覆盖，可同时读写 |
| **a+** | 读写模式，文件不存在创建，文件存在追加，可同时读写，写在末尾 |

#### 二进制模式扩展

在文本模式基础上添加 b 后缀表示二进制模式：rb、wb、ab、r+b、w+b、a+b。二进制模式不需要指定 encoding 参数。

#### 演示验证模式特性

以 test.txt 文件演示 r 模式：文件不存在时报错 "No such file or directory"，引出默认模式为 r 的事实。然后切换为 w 模式验证：文件不存在时会自动创建，文件存在时覆盖原内容，但无法读取（只写不能读）。再用 a 模式演示：文件不存在创建，文件存在追加新内容到末尾，同样只能写不能读，读取会报 "IO on supported operation" 错误。

### 二、读取文本文件的三种方法

声明针对文本文件的读取，Python 提供三种常用方法，适用于不同场景。

#### read() 方法

语法为文件对象.read(n)，n 为读取的字符数（文本模式）或字节数（二进制模式），省略 n 则读取整个文件。演示代码：

```python
f = open('test.txt', 'r', encoding='utf-8')
content = f.read()
print(content)
```

通过传入参数控制读取字符数：read(4) 读取 4 个字符，read(5) 读取 5 个字符，不传参数则全部读取。指出 read() 方法适合小文件，一次性加载到内存没有问题，但大文件会导致内存压力过大，不推荐使用。

#### readline() 方法

逐行读取，每次调用只读取一行内容，读完后文件指针自动移动到下一行，读到文件末尾返回空字符串。演示代码：

```python
f = open('test.txt', 'r', encoding='utf-8')
line1 = f.readline()
print(line1)
line2 = f.readline()
print(line2)
```

演示结果验证：第二行读取正常，第三行不存在时返回空字符串。注意 print() 函数的 end 参数默认带换行符，如果想去除因换行符产生的空行，可设置 end=''。强调 readline() 方法适合读取大文件（如数 GB 的日志文件），逐行读取不会占用过多内存。

#### readlines() 方法

读取所有行，以列表形式返回，列表中每个元素是文件中的一行字符串。演示代码：

```python
f = open('test.txt', 'r', encoding='utf-8')
lines = f.readlines()
print(lines)
for line in lines:
    print(line)
```

验证返回结果：列表元素包含换行符 \n，每行内容作为独立字符串存储。由于返回列表，可以直接用 for 循环遍历获取每行内容。指出该方法适合小文件，能够一次性读取所有内容，方便后续处理（如统计行数、筛选包含特定内容的行）。

### 三、写入文件操作

声明无论是文本文件还是二进制文件，写入都使用 write() 方法，语法为文件对象.write(内容)。强调文本文件写入必须是字符串，二进制文件写入必须是字节串（带 b 前缀的字符串）。

#### 文本文件写入：覆盖 vs 追加

##### w 模式演示

以 test2.txt 演示（初始文件不存在）。传入 w 模式，文件不存在自动创建。写入内容：

```python
with open('test2.txt', 'w', encoding='utf-8') as f:
    f.write('冰冰')
    f.write('\n')
    f.write('冰冰二')
```

第一次运行写入成功。第二次运行写入新内容时，发现原内容被覆盖，验证 w 模式会清空原文件重新写入的特性。指出这种行为会导致永久化保存数据丢失的问题。

##### a 模式演示

以 test3.txt 演示追加模式。文件不存在时自动创建，文件存在时新内容追加到末尾，不会覆盖旧内容：

```python
with open('test3.txt', 'a', encoding='utf-8') as f:
    f.write('冰冰')
    f.write('\n')
    f.write('冰冰二')
```

运行两次验证：第一次写入冰冰和冰冰二，第二次追加到文件末尾，原内容保留。

### 四、二进制文件读写（图片备份实例）

以图片备份为场景演示二进制文件操作。

#### 操作流程

1. **准备图片**：将一张图片 image.png 复制到当前目录
2. **读取原图**：使用 rb 模式读取二进制数据，如果用默认 r 模式会报错
3. **写入新图**：使用 wb 模式写入二进制数据到新文件

```python
# 读取原图片
with open('image.png', 'rb') as f:
    image_data = f.read()

# 写入备份图片
with open('image_backup.png', 'wb') as f2:
    f2.write(image_data)
```

运行验证成功生成 image_backup.png，内容与原图一致。总结其原理类似复制粘贴，但通过程序实现。

### 五、文件指针定位（进阶知识点）

指出在使用 w+、r+、a+ 等可读写模式时，可能遇到读写结果与预期不符的问题。

#### 问题演示

使用 a+ 模式写入数据后立即读取，发现读取结果为空。原因分析：文件指针的存在导致读写起始位置不同。

#### 概念解释

文件指针类似标记，表示下次读写的起始位置。不同模式的默认指针位置不同：
- **r、r+**：指针在文件开头，可以正常读取
- **w、w+、a、a+**：指针在文件末尾，从末尾开始读取读不到内容

#### tell() 和 seek() 方法

- **tell()**：返回当前指针位置，以字节为单位
- **seek(offset, whence)**：移动指针
  - offset：偏移量，正数向后移，负数向左移
  - whence：参考点，0=文件开头（默认），1=当前位置，2=文件末尾

演示代码：

```python
with open('test.txt', 'w+', encoding='utf-8') as f:
    f.write('hello冰冰')
    print(f.tell())  # 输出 29（字节数）
    f.seek(0)  # 将指针移到开头
    print(f.tell())  # 输出 0
    print(f.read())  # 能读取到内容
```

验证英文一个字节、中文的 UTF-8 编码字节数计算方式。总结使用读写模式时，读取前需要先将指针移到开头。

### 六、中文乱码与常见错误

#### 中文乱码问题

**问题原因**：Windows 系统下 Python 读取文件默认编码为 GBK（CP936），而文件通常以 UTF-8 编码保存，两者不匹配导致乱码或 UnicodeDecodeError。

**解决方案**：在 open() 函数中指定 encoding='utf-8'。

演示代码展示未指定编码时写入中文会出现乱码：

```python
# 未指定编码（默认 CP936/GBK）
with open('test.txt', 'w') as f:
    f.write('你好呀冰冰')
# 结果：乱码

# 指定编码
with open('test.txt', 'w', encoding='utf-8') as f:
    f.write('你好呀冰冰')
# 结果：正常
```

读取时同样需要指定编码，否则可能乱码或报错。强调处理中文文件时，encoding='utf-8' 不能省略，二进制模式不需要指定编码。

#### 三种常见错误

| 错误类型 | 错误信息 | 原因 | 解决方法 |
|----------|----------|------|----------|
| 文件不存在 | FileNotFoundError | 路径错误或文件确实不存在 | 检查路径；改用 w 或 a 模式自动创建 |
| 权限不匹配 | UnsupportedOperation | 读写模式不匹配（如 r 模式写入） | 改为 r+、w+、a+ 等读写模式 |
| 编码错误 | UnicodeDecodeError | 未指定 UTF-8 编码且编码不匹配 | 添加 encoding='utf-8' 参数 |

### 七、本节重点总结

#### 流程固定

文件操作流程：打开（open()）→ 读写（read/write）→ 关闭（close/with 自动关闭）。强调 with 语句更安全，能自动关闭文件，避免资源泄漏；没有用 with 语句则需手动调用 close()。

#### 关键参数

- **路径**：必须正确
- **访问模式**：选对模式避免权限不匹配，r 只读、w 只写覆盖、a 追加、rb/wb/ab 二进制
- **encoding**：读写中文文件必须指定为 utf-8，二进制文件不需要

#### 方法选择

- **read()**：读取指定字符/字节，适合小文件
- **readline()**：逐行读取，适合大文件
- **readlines()**：返回列表，适合小文件
- **write()**：写入字符串（文本）或字节串（二进制）

## 本P结论

1. **文件操作核心流程**：打开（优先 with 语句）→ 读写（选择合适方法）→ 关闭（with 自动关闭，需牢记打开必须关闭）
2. **访问模式选择**：r 只读、w 只写覆盖、a 追加；二进制模式加 b 后缀；需要读写用 r+、w+、a+
3. **读取方法适用场景**：read() 适合小文件指定字符读取，readline() 适合大文件逐行读取，readlines() 适合小文件返回列表批量处理
4. **写入方法**：write() 写入文本字符串或二进制字节串；w 模式覆盖、a 模式追加
5. **中文文件处理**：必须指定 encoding='utf-8'，避免 GBK 与 UTF-8 编码不匹配导致的乱码或 UnicodeDecodeError
6. **文件指针**：使用读写模式后，指针位置影响读写结果，读前需用 seek(0) 移到开头
