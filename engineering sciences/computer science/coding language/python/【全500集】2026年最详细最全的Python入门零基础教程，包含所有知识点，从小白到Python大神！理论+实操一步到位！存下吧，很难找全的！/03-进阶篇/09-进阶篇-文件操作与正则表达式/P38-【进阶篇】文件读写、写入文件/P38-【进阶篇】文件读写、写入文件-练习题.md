# P38 练习题 - 【进阶篇】文件读写、写入文件
---

## 一、结构识别

### 1.1 访问模式选择

以下场景应该使用哪种访问模式？请在 r / w / a / r+ / w+ / a+ 中选择。

| 场景 | 答案 |
|------|------|
| 读取配置文件 config.ini | r |
| 向日志文件追加新记录 | a |
| 创建新文件并写入初始内容 | w |
| 读写同一个文件，保留原有内容 | r+ |

> **解析**：r 只读不存在则报错；w 写覆盖不存在则创建；a 追加不存在则创建；r+ 读写但原有内容会被覆盖。

---

### 1.2 读取方法决策

针对以下文件特征，最合适的读取方法是 read() / readline() / readlines() 中的哪一个？

| 场景 | 最佳方法 | 原因 |
|------|----------|------|
| 文件共3行，总大小200字节 | 任意均可 | 小文件影响小 |
| 文件共50万行，总大小800MB | readline() | 避免一次性加载全部内容到内存 |
| 只需读取第1行 | readline() | 读完第一行后指针自动下移 |

## 二、机制理解

### 2.1 编码问题

在 Windows 系统下执行以下代码，结果是什么？为什么？如何修正？

```python
f = open('中文文件.txt', 'r')
content = f.read()
print(content)
f.close()
```

> **答案**：可能出现乱码或 UnicodeDecodeError。Windows 默认使用 GBK/CP936 编码，而中文文件通常为 UTF-8，编码不匹配导致解析失败。修正方法：显式指定 `encoding='utf-8'`。

---

### 2.2 文件指针行为

使用 a+ 模式打开文件后立即调用 read() 会读到什么？原因是什么？如何解决？

> **答案**：读到的内容为空字符串。a+ 模式打开时文件指针位于文件末尾，需要先用 `seek(0)` 将指针移到文件开头再读取。

---

### 2.3 w 模式与 a 模式的本质区别

w 模式和 a 模式在「文件不存在时」行为相同，都会创建新文件。但它们在「文件已存在时」有什么本质区别？

> **答案**：w 模式会先清空文件内容（覆盖），然后从文件开头写入；a 模式保留原有内容，指针移到末尾，追加新内容。

---

## 三、最小实践

### 3.1 基础读取练习

编写代码，读取 input.txt 并输出其总行数。

> **参考答案**：
```python
with open('input.txt', 'r', encoding='utf-8') as f:
    lines = f.readlines()
print(f"共 {len(lines)} 行")
```

---

### 3.2 大文件逐行处理

编写代码，逐行读取 large_file.txt，只打印第51-60行（不将整个文件加载到内存）。

> **参考答案**：
```python
with open('large_file.txt', 'r', encoding='utf-8') as f:
    for _ in range(50):
        f.readline()
    for _ in range(10):
        line = f.readline()
        if not line:
            break
        print(line, end='')
```

---

### 3.3 二进制文件备份

编写代码，将 photo.jpg 复制为 photo_backup.jpg。

> **参考答案**：
```python
with open('photo.jpg', 'rb') as src, open('photo_backup.jpg', 'wb') as dst:
    content = src.read()
    dst.write(content)
```

## 四、扩展思考

### 4.1 设计合理性

Python 的文件访问模式设计强制区分「只读」和「可写」——r 模式调用 write() 会直接抛出 UnsupportedOperation 异常。从软件工程角度分析：这种设计是好是坏？请从「防御性」「灵活性」「学习成本」三个维度展开。

> **参考思路**：
> - **防御性**：高。防止开发者意外修改只读文件，避免数据损坏。
> - **灵活性**：低。无法在运行时动态改变访问意图，需要重新打开文件。
> - **学习成本**：中等。需要记忆各模式的行为差异，但出错时错误信息明确。

---

### 4.2 综合应用

你需要编写一个函数 `process_large_log(log_path, output_path)`，功能如下：
1. 读取行数 > 100000 的日志文件
2. 过滤掉包含 "ERROR" 的行
3. 将剩余内容写入新文件

请考虑：如何避免内存溢出？如果读写过程中磁盘满了怎么办？

> **参考思路**：
> - 使用 readline() 逐行读取，配合 `with` 语句确保资源正确释放
> - 写入时检查返回值判断是否写入成功
> - 可捕获 OSError（磁盘满时抛出）并向上传递
