# P38 知识框架 - 【进阶篇】文件读写、写入文件
## 本P位置

- **所属阶段**：Python 进阶篇
- **前置内容**：P37 文件基础概念（文件路径、基础打开方式）
- **本P定位**：文件操作核心能力——读写与访问模式
- **后续衔接**：文件管理、上下文管理器、异常处理的实战应用

本P是文件操作从「概念认知」到「工程实践」的关键跨越，承上启下。

## 核心问题

1. **如何选对访问模式**：权限不匹配直接报错，读写需求决定模式选择
2. **如何根据场景选读取方法**：文件大小决定 read / readline / readlines 的取舍
3. **中文文件为何乱码**：Windows 默认 GBK 与文件 UTF-8 的编码冲突
4. **二进制文件怎么读**：图片、视频等非文本文件需要 rb / wb 模式
5. **读写模式为何读不到内容**：文件指针默认位置影响读写起点

## 知识骨架

```
文件操作
├── 访问模式体系
│   ├── 基础三模式：r（读）、w（写覆盖）、a（追加）
│   ├── 读写三模式：r+（读优先）、w+（写覆盖）、a+（追加读写）

│   ├── read(n) → 读取n字符/字节，不填则全部
│   ├── readline() → 逐行返回字符串，指针自动下移
│   └── readlines() → 返回行列表
├── 写入方法
│   └── write(内容) → 文本写字符串，二进制写字节串
├── 文件指针
│   ├── tell() → 返回当前位置（字节）
│   └── seek(offset, whence) → 移动指针
├── 编码处理
│   └── encoding='utf-8'（中文文件必须）
└── 资源管理
    ├── with 语句 → 自动关闭，推荐
    └── close() → 手动关闭
```

| 模式 | 读 | 写 | 不存在 | 存在时 |
|------|----|----|--------|--------|
| r | ✓ | ✗ | 报错 | 读取 |
| w | ✗ | ✓ | 创建 | 覆盖 |

| r+ | ✓ | ✓ | 报错 | 读写混合 |
| w+ | ✓ | ✓ | 创建 | 覆盖 |
| a+ | ✓ | ✓ | 创建 | 追加末尾 |

## 易错点

1. **乱码根因**：Windows 下 open() 默认 encoding='cp936'（GBK），中文文件必须显式指定 utf-8
2. **读前不 seek**：使用 w+、a+ 等模式写后立即读，指针在末尾，读不到内容；需先 seek(0)
3. **w 模式误用**：以为 w 是追加，实际会覆盖原内容，导致数据永久丢失
4. **权限不匹配报错**：r 模式写入 → UnsupportedOperation，Python 不允许越权操作
5. **二进制文件用文本模式**：图片用 rb 读取，否则报错；二进制模式无需 encoding 参数
6. **大文件用 read()**：一次性加载全部内容到内存，数 GB 文件会导致 OOM

**避错口诀**：读中文加 utf-8，写大文件用 readline，读写模式写完记得 seek(0)，图片视频必须加 b。
