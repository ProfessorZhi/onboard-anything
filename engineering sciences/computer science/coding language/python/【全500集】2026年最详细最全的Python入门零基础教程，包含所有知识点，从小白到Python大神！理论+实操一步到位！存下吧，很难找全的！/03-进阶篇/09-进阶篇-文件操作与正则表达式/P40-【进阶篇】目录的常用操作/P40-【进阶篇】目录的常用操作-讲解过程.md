# P40 讲解过程 - 【进阶篇】目录的常用操作
## 本P定位

本 P 是"进阶篇"的目录常用操作章节，承接前两 P 的文件读写和路径知识，进一步讲解如何用 Python 的 OS 模块自动化管理文件和目录。老师强调这些操作"既省时间，又能减少手动操作的误删漏改"，并指出"新手也能轻松搞定文件目录的自动化管理"。

## 讲解过程

### 1. 开篇引入：为什么要学目录操作

老师先说明学习背景：前面学了文件读写和路径，现在要升级技能，掌握文件和目录的管理操作。具体场景包括给文件重命名、删除无用文件、创建新文件夹、查看文件夹内容等。老师指出，以前这些任务需要手动打开文件夹点击操作，现在用 Python 内置的 OS 模块就能一键自动化完成。

### 2. 核心前提：必须导入 OS 模块

老师明确所有目录和文件的管理操作都必须先导入 OS 模块。这个模块是 Python 自带的，不需要额外安装，代码就是 `import os`。导入之后就能用 OS 模块提供的各种方法实现文件和目录的管理。

### 3. 文件操作一：重命名与移动

#### 基本方法

老师介绍 `os.rename()` 方法，这是文件操作最核心的功能之一。`os.rename` 的功能很灵活，不仅能给文件或目录改名字，还能实现同一磁盘内的文件移动或目录移动操作，本质是通过修改路径来移动的。如果只改名字，路径要和原来一致，只改文件或目录名本身；如果要移动，比如从 tools 目录移动到 image 目录，可以改路径但文件名和目录名保持不变。

#### 基本语法

`os.rename(原路径, 目标路径)`。两个参数很关键：
- 原路径：文件或目录原来的路径，可以是相对路径也可以是绝对路径，必须真实存在，否则会报错。
- 目标路径：修改之后的新路径。

#### 演示：文件重命名

老师演示将当前目录下的 `test3.txt` 改成 `冰冰.txt`，运行后确认 `test3.txt` 没有了，`冰冰.txt` 出现。文件内容"你好呀冰冰"也确认保留。接着又演示将 `test.txt` 改成 `冰冰二.txt`。

#### 演示：目录重命名

演示 `os.rename("python_base", "冰冰目录")`，确认目录名成功修改。

#### 演示：文件移动

演示将 `冰冰二.txt` 移动到 D 盘的 tools 目录下（文件名保持不变），运行后确认冰冰二.txt 从当前目录消失，然后在 D:\tools 目录下找到了它，内容完整保留。

#### 避坑点（重命名与移动）

老师结合刚才演示中出现的错误，逐一讲解避坑点：

**坑一：原路径必须存在。** 如果原路径写错了或者文件、目录根本不存在，就会直接报错。比如把 `冰冰二.txt` 移动之后，它在当前目录下已经不存在了，如果再运行同样的代码就会报"系统找不到指定的文件"。

**坑二：不支持跨盘移动。** 比如想把当前目录下的 `冰冰.txt` 移动到 C 盘，会报权限错误；如果换成 D 盘或其他盘，会报"系统无法将文件移动不同的磁盘"，说明 `os.rename` 只支持同磁盘内的移动，不支持跨盘。

**坑三：目标路径已存在同名文件或目录会直接覆盖，不会提示。** 老师演示时因为路径少写了一个反斜杠导致报错，后来修正路径后再次演示，说明当目标路径已存在同名文件时，会报"当文件已存在时，无法创建该文件"，而不是覆盖。

### 4. 文件操作二：删除文件

#### 基本方法

老师介绍 `os.remove()` 方法，功能单一，就是用来删除指定路径的文件。注意它只能删文件，不能删目录，删目录有专用的 `os.rmdir()` 方法（后面会讲）。

#### 基本语法

`os.remove(文件路径)`，路径可以是相对路径或绝对路径。

#### 演示：删除文件

老师演示删除当前目录下的 `冰冰.txt`，运行后确认删除成功。接着又把 D:\tools 目录下的 `冰冰二.txt` 删除，确认也可以用绝对路径删除。

#### 避坑点（删除文件）

**坑一：文件必须存在，否则会报错。** 比如删除已经删过的 `冰冰.txt`，再删一次就会报"系统找不到指定的文件"。

**坑二：只能删除文件，不能删除目录。** 比如用 `os.remove` 去删 `冰冰目录`（它是目录不是文件），会报"拒绝访问"权限错误。

**坑三：删除操作不可逆。** 老师强调 Python 程序删除的文件不会进入电脑回收站，是真正删除、无法恢复的。老师特意打开回收站查看，确认前面删除的 `冰冰.txt` 并不在里面，所以删除前一定要确认文件没有用了。

### 5. 目录操作

老师说明目录就是文件夹，核心操作包括四个：创建目录、删除目录、查看目录内容、查看当前目录。这四个操作覆盖了文件夹管理的大部分场景。

#### 5.1 创建目录

**基本方法：** `os.mkdir()`，只能创建一个新的空目录，不能创建已存在的目录。

**基本语法：** `os.mkdir(目录路径)`

**演示：** 在当前目录下新建"美女"目录，运行后确认目录创建成功。老师指出也可以用绝对路径。

**避坑点：**
- 目录不能已经存在，否则报 `FileExistsError`（文件存在错误）。
- 父目录必须存在。比如要在 `E:\A` 目录下创建 `B` 目录，必须先有 `E:\A`，不然会报错。

#### 5.2 删除目录

**基本方法：** `os.rmdir()`，只能删除指定的空目录，如果目录里有任何文件或子目录就不能删除，会报错。

**演示：** 删除刚才创建的"美女"目录，运行后确认删除成功。

**避坑点：**
- 目录必须存在，否则报"找不到指定的文件"。
- 目录必须为空，否则会报 `OSError` 提示"目录不是空的"。
- 删除操作不可逆，即使误删也找不回来。

#### 5.3 查看当前工作目录

**基本方法：** `os.getcwd()`。

老师讲解使用场景：写相对路径时容易忘了脚本当前在哪个目录下运行，导致路径报错，这时就可以用 `os.getcwd()` 解决。它能够获取 Python 脚本当前运行的目录，返回的是绝对路径，相当于让程序知道自己现在在哪个文件夹里，可以用来确认相对路径的起点和参考点。

#### 5.4 查看目录内容

**基本方法：** `os.listdir()`，返回指定目录下的所有文件和子目录名称组成的列表。

**基本语法：** `os.listdir(目录路径)`，如果省略参数，默认查看当前目录。

**演示：** 不传参打印 `os.listdir()`，运行后列出当前目录下的所有文件和文件夹（冰冰、data、image.png、backup.png、my_package、my_tools 等）。接着指定路径查看 D:\tools 目录，运行后列出该目录下的所有文件（7zip、向日葵、百度网盘、dict、钉钉等）。

### 6. 路径检验方法

老师讲解为什么要用路径检验：执行创建、删除、重命名操作之前，如果不确定目标路径是否存在，容易报错。比如想删除 `test.txt` 但不确定文件是否已被删除，或者想创建 `new_folder` 但怕已有同名文件夹导致报错，这时就需要路径检验方法提前判断路径状态，避免报错。

OS 模块提供了三个路径检验方法，返回值都是 `True` 或 `False`。

#### 6.1 检验路径是否存在：`os.path.exists()`

能判断指定路径（不管是文件还是目录）是否存在，存在返回 `True`，不存在返回 `False`。支持相对路径和绝对路径。

**演示：** 检查 `test.txt`（不存在）返回 `False`；检查 `冰冰目录`（存在）返回 `True`；检查 `D:\tools`（存在）返回 `True`；检查 `冰冰.txt`（不存在）返回 `False`。

#### 6.2 检验是否为文件：`os.path.isfile()`

用于判断指定路径是否存在且是一个文件，两个条件都满足才返回 `True`，否则返回 `False`。

**演示：** 检查 `py_dict.py`（存在且是文件）返回 `True`；检查 `test.txt`（不存在）返回 `False`；检查 `冰冰目录`（路径存在但是目录不是文件）返回 `False`。

#### 6.3 检验是否为目录：`os.path.isdir()`

与 `os.path.isfile` 相反，判断指定路径是否存在且是一个目录（文件夹），两个条件都满足才返回 `True`，否则返回 `False`。

**演示：** 检查 `D:\tools`（路径存在且为目录）返回 `True`；检查一个已存在但是文件的路径返回 `False`；检查当前目录不存在的 `哈哈` 路径直接返回 `False`。

### 7. 实战组合：路径检验 + 文件操作

老师演示如何把路径检验方法和前面的操作组合起来，实现安全操作，避免报错。

#### 案例一：安全删除文件

思路：删除之前先判断文件是否存在且是文件，满足条件才执行删除，否则打印"文件不存在"。

老师写出代码框架，把路径保存到变量 `file_path` 中，然后 `if os.path.isfile(file_path): os.remove(file_path)`，否则 `print("文件不存在")`。运行后，对于不存在的 `冰冰.txt` 打印"文件不存在"，对于存在的 `test2.txt` 则成功删除。强调这样写程序不管文件是否存在都能正常运行不会报错。

#### 案例二：安全创建目录

思路：创建之前先判断目录是否存在且是目录，不存在才创建，存在就打印"目录已存在"。

老师写出代码，把目录路径保存到 `dir_path` 变量中，然后 `if os.path.isdir(dir_path): print("目录已存在")` 否则 `os.mkdir(dir_path)`。运行后第一次不存在，创建了"美女"目录；再运行一次，因为目录已存在，打印"目录已存在"而不报错。强调这样写更安全。

### 8. 课堂总结

老师梳理本节重点：

**核心前提：** 所有操作前都必须先导入 OS 模块（`import os`），不需要安装可直接使用。

**文件操作：**
- 重命名或移动用 `os.rename(原路径, 目标路径)`，同磁盘可用，原路径必须存在，不支持跨磁盘移动。
- 删除文件用 `os.remove`，只能删文件，删除操作不可逆，需谨慎。

**目录操作：**
- 创建目录用 `os.mkdir`，只能创建空目录，父目录必须存在，不能重复创建。
- 删除空目录用 `os.rmdir`，只能删除空目录，删除不可逆需谨慎。
- 查看当前目录用 `os.getcwd`，返回绝对路径，帮助确认相对路径的参考点。
- 查看目录内容用 `os.listdir`，返回文件或目录的列表，不传参默认查看当前目录。

**路径检验：**
- 路径是否存在用 `os.path.exists`
- 是否为文件用 `os.path.isfile`
- 是否为目录用 `os.path.isdir`
- 建议在执行创建、删除、重命名操作之前先执行路径检验，避免报错。

**关键提醒：** 删除操作（`os.remove` 和 `os.rmdir`）不可逆，一定要先确认文件或目录没有用了再结合路径检验执行操作，最好提前备份重要数据避免误删。

**课后练习方向：** 批量重命名当前目录下所有 `.txt` 文件为"文档一.txt"、"文档二.txt"等格式；自动创建包含数据文件夹、日志文件夹等多级目录结构的项目文件夹，注意父目录要先存在。老师还提到可以用 `os.makedirs` 方法（支持创建多级目录），让大家自行了解。

## 本P结论

本 P 系统讲解了 Python OS 模块的文件和目录管理操作：
- 文件操作核心是 `os.rename`（重命名/同磁盘移动）和 `os.remove`（删除文件），各有明确的避坑规则。
- 目录操作围绕创建（`os.mkdir`）、删除（`os.rmdir`）、查看当前目录（`os.getcwd`）、查看内容（`os.listdir`）四个方法展开。
- 路径检验三件套 `os.path.exists / isfile / isdir` 是保障操作安全的关键，建议在任何增删改操作前先执行。
- 所有删除操作不可逆，务必结合路径检验并提前备份。
