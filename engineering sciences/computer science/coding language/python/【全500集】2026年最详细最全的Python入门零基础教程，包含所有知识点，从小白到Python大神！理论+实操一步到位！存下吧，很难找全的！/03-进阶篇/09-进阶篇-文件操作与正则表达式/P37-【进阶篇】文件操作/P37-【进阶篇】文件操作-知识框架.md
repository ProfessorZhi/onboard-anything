# P37 知识框架 - 【进阶篇】文件操作
## 本P位置

【进阶篇】文件操作章节第1讲，基础铺垫课。承接基础语法学习，引出数据持久化核心能力，支撑后续所有文件处理场景。

## 核心问题

如何让程序与本地文件交互，实现数据持久化存储？

具体拆解：
- 文件是什么，为什么需要文件操作
- 如何安全地打开和关闭文件
- 文本文件与二进制文件的操作差异在哪里

---

## 知识骨架

```
文件操作
├── 文件基础
│   ├── 定义：存储在外部设备的二进制数据，目的为长期保存
│   └── 类型
│       ├── 文本文件：有编码，字符→二进制，可用文本编辑器打开（.txt/.py/.md）
│       └── 二进制文件：无编码，原始字节，专用软件打开（.jpg/.mp3/.exe）
├── 操作流程（三步骤，固定顺序）
│   ├── 1. 打开文件 → 获得文件对象
│   ├── 2. 读写操作 → 数据在内存与文件间流动
│   └── 3. 关闭文件 → 释放资源，持久化写入
└── 打开/关闭实现
    ├── open() 函数
    │   ├── 参数1：文件路径（绝对路径/相对路径）
    │   ├── 参数2：访问模式（r/w/a）
    │   └── 参数3：encoding（文本文件必须，二进制文件禁用）
    └── with 语句（推荐用法，自动关闭）
```

---

## 关系与依赖

| 层级 | 内容 | 上下游关系 |
|---|---|---|
| 上游依赖 | 字符串、编码概念 | 不理解编码则无法正确操作文本文件 |
| 核心概念 | 文件对象 | open()返回此对象，后续所有操作都通过它 |
| 类型区分 | 文本 vs 二进制 | 决定是否需要encoding参数，决定用什么软件打开 |
| 流程约束 | 三步骤顺序固定 | 第一步失败则后续无法执行，最后一步不可省略 |

---

## 输入 / 输出 / 前置 / 支撑

| 类型 | 内容 |
|---|---|
| 输入 | 文件路径（字符串），访问模式（字符），encoding（字符串，仅文本） |
| 输出 | 文件对象（用于后续读写） |
| 前置知识 | 路径概念、字符编码基础 |
| 支撑工具 | with语句（安全关闭），UTF-8（标准编码） |

---

## 易错点

1. **忘记关闭文件** → 资源泄漏，文件被占用无法修改
2. **文本文件不指定encoding** → 中文内容乱码或UnicodeDecodeError
3. **二进制文件加了encoding** → 参数冲突，报错
4. **路径写错** → FileNotFoundError
5. **访问模式选错** → PermissionError或写覆盖丢失数据

---

## 关键结论（可引用）

- 文件操作使程序实现数据持久化，关机后数据不丢失
- 文本文件必须指定encoding为UTF-8，二进制文件禁止加encoding参数
- 文件操作三步骤：打开→读写→关闭，关闭不可省略
- 推荐使用with语句，自动安全关闭，避免资源泄漏
- 访问模式：r只读、w只写（覆盖）、a追加
