# P37 讲解过程 - 【进阶篇】文件操作
## 本P定位

本P是【进阶篇】文件操作的第一讲，属于入门基础部分。老师先从概念层解释文件操作是什么、为什么需要它，再铺垫三个核心基础概念（文件定义、文件类型、文件操作流程），最后进入第一个核心操作——打开和关闭文件。后续小节会依次讲解路径和目录操作，本P是整个文件操作章节的基础铺垫。

---

## 讲解过程

### 一、引入：文件操作的意义

课程一开始，老师先说明文件操作的价值：让程序与本地文件互动，实现数据的持久化存储。强调没有文件操作时，数据存在内存中，程序结束或电脑关机后数据就没了（"非常健忘"）。有了文件操作，数据可以保存到硬盘或U盘，关机后再打开程序数据依然存在。老师指出，无论是读取本地文档、Excel数据，还是保存程序运行结果、日志，都离不开文件操作。

### 二、课程大纲预告

老师预告本P会分三个小节讲解：
1. **文件操作**：学习打开、读取、写入、关闭等基础核心用法，搞定数据存储
2. **文件的路径**：掌握绝对路径和相对路径的使用，避免文件找不到的坑，精准定位文件
3. **目录的常用操作**：学习创建文件夹、查看目录内容、重命名或删除文件等，实现文件系统自动化管理

### 三、三个核心基础概念

在正式学习文件操作前，老师认为有必要先讲清楚三个概念，把地基打牢。

#### 3.1 什么是文件

老师给出简洁定义：文件就是存储在硬盘、U盘等外部设备上的二进制数据，目的是长期保存信息。例如电脑上的文档、图片、视频、音乐，本质上都是一堆二进制代码。程序通过文件操作可以读取这些二进制数据并转换成人类能看懂的内容，或者把程序里的数据转换成二进制写入文件。

#### 3.2 文件的两种类型

老师强调文件主要分为两类，操作方式略有不同，一定要区分清楚：

**文本文件**
- 人类可读的纯文字文件，存储字符信息
- 字符按照特定编码转换成二进制保存
- 常见例子：.txt文本文档、.py Python代码文件、.md文档
- 用记事本、VS Code、PyCharm等文本编辑器就能直接打开查看

**二进制文件**
- 计算机直接处理的文件，存储原始二进制数据（字节数据）
- 不是人类能看懂的文字
- 常见例子：.jpg图片、.mp3音乐、.exe可执行程序、.png图片等
- 必须用专用软件打开，比如图片要用图片查看器、音乐要用播放器
- 如果用文本编辑器打开，看到的会是一堆乱码

老师特别提醒一个关键注意点：操作文本文件时一定要关注文件编码，UTF-8是最常用的，不关注的话很容易出现乱码或报错。二进制文件则不需要管编码，直接操作原始数据即可。

老师最后补充说明：虽然日常功能上分成两种文件类型，但本质上它们都是二进制文件，因为计算机只能存储二进制数据。我们只是按照功能不同、查看方式不同做了分类。

#### 3.3 文件操作的三个固定步骤

老师强调不管读文件还是写文件，流程都是固定的，记住这三个步骤就不会乱：

1. **打开文件**：用open函数创建文件对象，相当于建立程序和文件之间的连接通道
2. **读写操作**：读取（把文件内容加载到程序内存）或写入（把程序内存中的数据写入文件）
3. **关闭文件**：使用close方法关闭文件对象，释放系统资源

老师特别强调关闭文件非常重要——不关闭的话会一直占用内存，还可能导致文件冲突，比如后面想修改文件时会提示被占用。

### 四、核心操作一：打开和关闭文件

这部分是本P的重点，老师用大量时间演示代码。

#### 4.1 打开文件：open函数

老师先介绍open函数的基本语法：

```python
f = open("文件路径", "访问模式", encoding="UTF-8")
```

open函数会生成一个文件对象，用变量f接收，之后所有读写操作都通过这个对象进行，相当于建立程序和文件的连接通道。

**三个参数逐个解释：**

- **文件路径**：文件在电脑上的具体位置，例如C盘的test.txt（绝对路径），或者test.txt（相对路径，表示当前项目下的文件）。老师预告路径如果写错会直接报错，所以一定要核对清楚。
- **访问模式**：控制文件的读写权限，r是只读，w是只写，a是追加，后面会详细讲。选对模式才能避免想读却不能读、想写却写不进去的报错问题。
- **encoding编码格式**：文本文件的编码格式，强烈推荐使用UTF-8，尤其是在处理中文时。如果不指定编码，很容易出现乱码或报错。注意：二进制文件操作时不需要加这个参数。

#### 4.2 演示：打开当前目录下的test.txt

老师开始演示操作流程：

1. 先在当前目录下新建一个test.txt文件，写入内容"Hello World"和"冰冰好"
2. 然后在Python中打开这个文件
3. 指定encoding为UTF-8，尤其是有中文时一定要指定
4. 使用只读模式（r），但这个参数可以不指定，默认就是只读模式
5. 用变量f接收文件对象
6. 最后调用f.close()关闭文件

```python
f = open("test.txt", "r", encoding="UTF-8")
# 读写操作省略
f.close()
```

老师演示了两种情况：
- 打开存在的test.txt文件：正常运行，不报错
- 打开不存在的test1.txt文件：报错FileNotFoundError

老师再次强调打开文件后一定要关闭，不然会一直占用系统资源，甚至导致文件无法被其他程序访问。

#### 4.3 推荐方式：with语句

老师指出手动写close很容易遗漏，所以推荐使用更安全的with语句。with语句的优势是代码块执行完后会自动关闭文件，不管中间有没有报错，不会遗漏关闭步骤，也不用手动写close方法。

**with语句语法：**

```python
with open("test.txt", "r", encoding="UTF-8") as f:
    # 在这里进行读写操作
    print(f.encoding)  # 可以访问文件对象的属性
```

老师演示了with语句的用法：
- 用as f把文件对象赋值给变量f
- f可以自定义命名
- 在with代码块内进行读写操作
- 代码块结束后自动关闭文件

老师演示通过f.closed属性检测文件关闭状态，代码块执行完后f.closed返回True，说明文件已自动关闭。

老师最后总结：with语句既简洁又安全，是实际开发中最常用的方式，大家一定要养成用with语句的习惯。

## 本P结论

本P完成了文件操作入门的所有概念铺垫和第一个核心操作的讲解。具体结论如下：

**文件操作的意义**：让程序实现数据持久化存储，把数据保存到硬盘/U盘，关机后数据不丢失

**文件类型区分**：
- 文本文件：人类可读，用特定编码转二进制，可用文本编辑器直接打开
- 二进制文件：计算机直接处理，用专用软件打开
- 操作文本文件必须关注编码（推荐UTF-8），二进制文件不需要管编码

**文件操作三步骤**：打开文件 → 读写操作 → 关闭文件，关闭文件不可省略

**open函数参数**：
- 文件路径：文件在电脑上的位置（绝对路径或相对路径）
- 访问模式：控制读写权限（r/w/a等）
- encoding：文本文件编码格式，二进制文件不需要此参数

**推荐操作方式**：使用with语句，代码块执行完后自动关闭文件，既简洁又安全，避免手动遗漏close
