# P37 资源清单 - 【进阶篇】文件操作
## 概述

本P为文件操作章节的第一讲，主要覆盖文件概念、文件类型区分、文件操作流程以及 open() / close() / with 语句的使用。以下资源清单涵盖程序、软件、库、文件类型、工具及配置参数，按来源和用途分类整理，供后续练习参考。

## 二、库 / 函数 / 方法

| 名称 | 类型 | 在本 P 中的作用 | 来源 | 后续可练习方向 |
|---|---|---|---|---|
| open() | 内置函数 | 文件操作的入口，根据路径、模式、编码三个参数打开文件，返回文件对象 | 字幕直接提到 | 练习 open() 的不同参数组合，测试 r/w/a 模式的行为差异；尝试读取不存在文件的报错处理 |
| close() | 文件对象方法 | 手动关闭文件对象，释放系统资源，防止文件被占用 | 字幕直接提到 | 对比关闭与不关闭文件时的系统资源占用情况，观察文件是否仍被占用 |
| with 语句 | 语法结构 / 上下文管理器 | 替代手动 close()，在代码块执行完毕后自动安全关闭文件，既简洁又安全 | 字幕直接提到 | 将手动 close() 的代码改写为 with 语句版本；模拟异常场景，验证 with 语句的自动关闭特性 |

## 四、配置参数 / 编码标准

| 名称 | 类型 | 在本 P 中的作用 | 来源 | 后续可练习方向 |
|---|---|---|---|---|
| UTF-8 | 编码标准 | 推荐使用的文本文件编码格式，处理中文时必须指定，避免乱码或 UnicodeDecodeError | 字幕直接提到 | 对比 GBK、UTF-8 等不同编码读取同一中文文件的效果，观察乱码现象；练习指定 encoding 参数 |
| encoding 参数 | 函数参数 | open() 函数的第三个参数，指定文本文件编码，二进制文件操作时禁止使用 | 字幕直接提到 | 测试省略 encoding 参数时读取中文文件的行为；区分文本模式和二进制模式下的参数差异 |
| r 模式（只读） | 访问模式 | open() 的默认访问模式，仅允许读取文件内容，不能修改或写入 | 字幕直接提到 | 测试 r 模式下尝试写入文件时的报错（PermissionError 或 io.UnsupportedOperation） |
| w 模式（只写） | 访问模式 | 打开文件用于写入，会覆盖已有内容，文件不存在则创建新文件 | 字幕直接提到 | 练习 w 模式写入文件，观察文件原有内容是否被覆盖 |
| a 模式（追加） | 访问模式 | 打开文件用于追加写入，不会覆盖原有内容，在文件末尾添加新内容 | 字幕直接提到 | 对比 w 和 a 模式的区别，练习向文件末尾追加内容 |

|---|---|---|---|---|
