# P39 知识框架 - 【进阶篇】文件路径
## 本P位置

- 课程定位：Python进阶篇，第39讲
- 前置内容：文件操作流程（open、read、write、close）
- 本P作用：解决文件操作中最常见的"file not found error"，专项讲解路径问题
- 后续关联：为本P的路径知识是文件读写、配置管理、项目部署的基础

## 核心问题

程序如何定位并访问指定文件——即如何正确表达"文件在哪里"

## 知识骨架

```
文件路径
├── 绝对路径（完整门牌号）
│   ├── 从根目录开始，含所有层级
│   ├── Windows示例：D:\旅游\三亚\照片.jpg
│   ├── 适用：系统固定文件、脚本独立运行
│   └── 特点：稳定可靠，但移植性差
│
├── 相对路径（当前位置参照）
│   ├── 参照点：脚本运行目录（当前工作目录）
│   ├── 符号：. = 当前目录（可省略）
│   ├── 符号：.. = 上一级目录（不可省略）
│   ├── 适用：项目内部文件操作
│   └── 特点：简洁可移植，但依赖当前目录
│
└── 转义问题（Windows特有问题）
    ├── 根因：Python反斜杠为转义符
    ├── 方案一：r前缀（原始字符串）——推荐
    └── 方案二：双反斜杠
```

## 关系与依赖

- **并列选择**：绝对路径 vs 相对路径，根据场景二选一
- **依赖关系**：相对路径依赖当前工作目录（由os.getcwd()决定）
- **转义冲突**：Windows反斜杠分隔符与Python转义字符的冲突，优先用r前缀解决
- **层级关系**：相对路径通过.和..构建目录层级关系

## 输入 / 输出 / 前置 / 支撑

| 类型 | 内容 |
|------|------|
| 输入 | 文件名字符串或路径字符串（如"test.txt"、r"D:\data\file.txt"） |
| 输出 | 文件对象（传给open函数的返回值，用于后续read/write操作） |
| 前置知识 | 文件操作基础（open函数）、字符串概念 |
| 支撑工具 | os.getcwd()（获取当前工作目录）、IDE路径复制功能 |

## 易错点

- **转义遗漏**：Windows路径漏加r前缀或双反斜杠，\t、\n被解析为转义字符
- **层级错位**：想访问上一级却写成./（当前目录），少一个点
- **路径硬编码**：项目移动后绝对路径全部失效，需逐一修改
- **cwd误解**：误以为脚本所在目录就是当前工作目录，实际是脚本启动时所在的目录

## 外部资源与对象

- **os.getcwd()**：标准库os模块方法，获取Python进程当前工作目录
- **IDE路径复制**：通过IDE右键菜单可快速获取文件的绝对路径
- **Windows文件资源管理器**：用于验证文件实际位置，辅助排查路径错误

## 核心心法

找准参考点——绝对路径的参考点是根目录，相对路径的参考点是脚本运行目录。想清楚"从哪里出发去找文件"，就能准确写出路径。

## 选型原则

项目内部文件操作优先相对路径（移植性强、团队协作部署方便）；访问系统固定位置文件用绝对路径（稳定性高、不受当前目录影响）。
