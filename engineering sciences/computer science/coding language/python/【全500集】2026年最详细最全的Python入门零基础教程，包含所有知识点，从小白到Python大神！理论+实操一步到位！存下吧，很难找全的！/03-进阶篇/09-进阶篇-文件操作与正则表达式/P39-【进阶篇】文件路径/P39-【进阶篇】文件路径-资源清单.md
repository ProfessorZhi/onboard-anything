## 一、Python 核心模块与方法

### os.getcwd()

- **类型**：标准库方法（Python 内置 os 模块）
- **在本 P 中的作用**：获取 Python 进程当前工作目录，用于验证相对路径的参照基准，避免因误判 cwd 与脚本所在目录不一致而导致的路径错误
- **来源**：字幕直接提到
- **后续可练习方向**：建议方向 —— 结合不同启动方式（IDE 运行 / 命令行 cd 到其他目录 / 双击脚本）观察 cwd 的差异，加深对"相对路径参照点"的理解

---

## 二、IDE 工具功能

### IDE 路径复制功能

- **类型**：IDE 功能（常见于 PyCharm、VS Code 等主流 Python IDE）
- **在本 P 中的作用**：通过右键菜单快速复制文件的绝对路径，避免手动输入长路径时出错（尤其是反斜杠转义问题），提升调试效率
- **来源**：字幕直接提到
- **后续可练习方向**：建议方向 —— 在 IDE 中创建测试项目，使用路径复制功能对比不同文件的绝对路径写法；尝试用 IDE 的终端或调试控制台验证 os.getcwd() 的返回值

## 三、操作系统组件

### Windows 文件资源管理器（Explorer）

- **类型**：操作系统内置工具
- **在本 P 中的作用**：用于手动验证文件实际存在的位置，协助排查"文件明明在却找不到"的路径问题；也用于快速复制文件路径
- **来源**：字幕直接提到（视频演示中使用）
- **后续可练习方向**：建议方向 —— 在文件资源管理器中创建不同层级的测试文件夹，手动对比"地址栏显示的路径格式"与"Python 中需要写的路径格式"的差异

## 四、字符串语法特性

### 原始字符串前缀（r""）

- **类型**：Python 语法特性（字符串字面量修饰符）
- **在本 P 中的作用**：取消反斜杠的转义功能，解决 Windows 路径中包含 \t、\n 等字符导致的路径识别错误，是处理 Windows 绝对路径的推荐写法
- **来源**：字幕直接提到
- **后续可练习方向**：建议方向 —— 对比普通字符串与原始字符串在处理路径、正则表达式时的行为差异；注意原始字符串不能以反斜杠结尾的规则

## 五、补充说明

本 P 以理论讲解为主，未涉及第三方库或外部网站。后续如需进一步练习路径相关的项目级应用（如路径拼接、跨平台路径处理），可关注以下方向：

- **pathlib 模块**（Python 3.4+）：提供面向对象的路径操作，比字符串拼接更安全
- **os.path 模块**：提供跨平台的路径判断与拼接方法
- **configparser / json**：将路径配置外置到配置文件，避免硬编码
