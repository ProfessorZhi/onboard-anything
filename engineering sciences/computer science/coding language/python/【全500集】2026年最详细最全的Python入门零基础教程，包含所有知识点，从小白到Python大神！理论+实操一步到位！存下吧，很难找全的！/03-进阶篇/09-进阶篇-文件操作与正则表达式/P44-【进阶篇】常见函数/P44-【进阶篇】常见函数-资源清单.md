# P44 资源清单 - 【进阶篇】常见函数
## 资源总览

本P涉及的外部资源较少，核心依赖为 Python 及其内置的 `re` 模块。以下为具体清单：

---

## 1. Python

- **类型**：编程语言 / 运行环境
- **在本P中的作用**：课程全程使用 Python 进行代码演示，所有正则函数均基于 Python 环境运行
- **来源**：字幕直接提到
- **后续练习方向**：在本地 IDE（如 PyCharm、VS Code）或 Jupyter Notebook 中编写和运行正则表达式代码；结合 Python 基础语法（字符串、列表、条件判断）做综合练习

---

## 2. re 模块（Python 标准库）

- **类型**：标准库
- **在本P中的作用**：提供 `re.search()`、`re.findall()`、`re.sub()`、`re.match()` 等正则操作函数，是本P所有代码演示的实现载体
- **来源**：字幕直接提到
- **后续练习方向**：用 re 模块实现一个简易的文本信息提取工具（如从一段文本中批量提取邮箱、手机号、日期等）；尝试用正则清理本地日志文件中的无效字符

---

## 3. re.I / re.IGNORECASE（忽略大小写标志）

- **类型**：标志常量
- **在本P中的作用**：作为 `re.search()`、`re.findall()`、`re.sub()` 的 `flags` 参数传入，控制匹配时忽略大小写，实现大小写不敏感的文本处理
- **来源**：字幕直接提到（在 re.sub() 的 flags 参数演示中）
- **后续练习方向**：在处理用户输入或网页文本时，使用 `re.I` 避免大小写差异导致漏匹配；对比加与不加 `flags` 时的匹配结果差异

---

## 4. match 对象

- **类型**：Python 数据类型（re 模块的返回对象）
- **在本P中的作用**：`re.search()` 返回 match 对象而非直接返回字符串，需调用 `.group()` 方法提取匹配内容；若匹配失败返回 `None`
- **来源**：字幕直接提到
- **后续练习方向**：探索 match 对象的更多方法（如 `.start()`、`.end()`、`.span()` 获得匹配位置信息）；对比 match 对象与 findall 返回的列表在实际项目中的使用差异

---

## 5. str.replace()（字符串替换方法，可作为 re.sub() 的对照）

- **类型**：字符串方法
- **在本P中的作用**：老师提到 `re.sub()` 类似 Word 的"查找替换"功能；`str.replace()` 是 Python 字符串的内置替换方法，虽然功能简单（不支持正则），但可作为对比学习对象
- **来源**：基于上下文推断（re.sub() 的类比说明）
- **后续练习方向**：对比 `str.replace()` 与 `re.sub()` 在处理含正则特殊字符（如 `.`、`*`、`(`）时的行为差异，理解为何正则替换需要专用函数

---

## 6. 正则表达式符号体系（\d、\w、\s、元字符等）

- **类型**：语法规则（不是代码库，但属于正则操作的必要支撑）
- **在本P中的作用**：本P调用 `re.findall(r'\d', ...)`、`re.sub(r'\s', '', ...)` 等代码，依赖正则表达式的模式规则；43P 已讲解的核心匹配规则是本P函数操作的前置基础
- **来源**：字幕直接使用（43P 已前置讲解）
- **后续练习方向**：系统复习正则元字符（`.`、`*`、`+`、`?`、`[]`、`{}`、`|`、`.`）和转义字符；设计一个包含多种正则规则的测试文本，验证各规则的匹配行为

---
