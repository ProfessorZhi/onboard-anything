# P44 讲解过程 - 【进阶篇】常见函数
## 本P定位

本P是"正则表达式"章节的第二个小节，属于进阶内容。前一小节（43P）已经讲解了正则表达式的核心匹配规则，本P在此基础上引入正则实战中最常用的三个高频函数：`re.search()`、`re.findall()` 和 `re.sub()`。老师强调这三个函数"用法简单，却能覆盖80%的正则实战场景"。

## 讲解过程

### 一、复习铺垫：re.match() 方法的局限性

老师先带大家回顾上一节学过的 `re.match()` 方法，目的是让学习者清晰看出新旧函数的差异与优势，避免混淆。

**核心特征讲解：**
- `re.match()` 的核心功能是"从字符串的**开头**开始匹配"
- 如果开头不符合规则，会直接返回 `None`
- 这种特性使它非常适合做**格式验证**，但不适合查找字符串中间的内容

**代码演示1——匹配成功的例子：**
```python
import re
res = re.match('TH', 'this is Python')
print(res.group())  # 输出：TH
```
解释：字符串开头就是"TH"，所以匹配成功，`group()` 可以提取结果。

**代码演示2——匹配失败的例子：**
```python
res = re.match('TH', 'Python is good')
print(res.group())  # 报错
```
解释：字符串开头是"PY"而不是"TH"，返回 `None`，`group()` 提取会报错。

**过渡：** 正是因为 `re.match()` 只看开头这一局限，需要引入新的函数来解决"查找字符串中间内容"的需求。

---

### 二、第一个核心函数：re.search()——全局找第一个匹配项

老师用"看书"的比喻来区分 `re.match()` 和 `re.search()`：
- `re.match()` 像只看**书的封面**，封面不符合就直接放弃
- `re.search()` 像从书的**第一页读到最后一页**，找到第一个目标就停下

**基本语法：**
```python
re.search(pattern, string, flags=0)
```
- `pattern`：正则匹配规则
- `string`：要搜索的原始字符串
- `flags`：可选参数，控制匹配方式（如忽略大小写）

**与 match 的对比演示：**
```python
# 同样的规则 "TH" 去匹配 "Python is good"
res = re.search('TH', 'Python is good')
print(res.group())  # 输出：TH（不会报错）
```
解释：`search()` 会遍历整个字符串，找到中间位置的"TH"就返回，不会像 `match()` 那样返回 `None`。

**演示多个匹配项的情况：**
```python
res = re.search('TH', 'Python I love python')
print(res.group())  # 只返回第一个 TH
```
结论：即使有多个匹配项，`search()` 也只返回第一个。

**演示没有匹配项的情况：**
```python
res = re.search('XYZ', 'Python')
print(res.group())  # 报错
```
解释：遍历完整个字符串都没有找到，返回 `None`，`group()` 会报错。

**对比表格总结：**
| 维度 | `re.match()` | `re.search()` |
|------|---------------|----------------|
| 匹配范围 | 只匹配字符串**开头** | 全局找**第一个** |
| 适用场景 | 验证字符串是否以特定规则开头（手机号、邮箱格式验证） | 从任意位置查找符合规则的内容（新闻中找第一个日期、从文本中找第一个邮箱） |

### 三、第二个核心函数：re.findall()——查找所有匹配项，返回列表

老师用"找苹果"的比喻说明：
- `re.search()` 是找**第一个**苹果就停下
- `re.findall()` 是把所有出现**苹果的地方**都标记出来，整理成清单

**核心特点：**
- 遍历整个字符串，找到所有符合规则的**非重叠子串**
- 以**列表**形式返回
- 即使没有找到任何匹配项，也返回**空列表 `[]`**（不会返回 `None`），这一点非常友好，不用像 `match()` 和 `search()` 那样判断是否为 `None`

**基本语法（与 search 完全相同，只是返回值不同）：**
```python
re.findall(pattern, string, flags=0)
```

**代码演示1——提取所有数字：**
```python
res = re.findall(r'\d', '123ABC456')
print(res)  # 输出：['1', '2', '3', '4', '5', '6']
```
每个列表元素就是匹配到的一个子串。

**代码演示2——匹配三个连续单词字符：**
```python
res = re.findall(r'\w{3}', '123ABC456XYZ.com')
print(res)  # 输出：['123', 'ABC', '456', 'XYZ', 'com']
```
解释：`\w` 包括数字、字母、下划线，`.` 不是单词字符，所以"com"匹配成功，"XYZ"后面的"."导致分隔。

**代码演示3——没有匹配项返回空列表：**
```python
res = re.findall(r'\d', 'ABCDEF')
print(res)  # 输出：[]
```
不会报错，直接返回空列表。

**应用场景：**
1. 提取文本中所有符合规则的内容：所有手机号、所有邮箱、所有日期
2. 批量收集关键信息：从系统日志提取所有错误码、从报表文本提取所有数值

### 四、第三个核心函数：re.sub()——替换匹配到的内容

**核心作用：**
- `sub` 是 `substitute`（替换）的缩写
- 遍历整个字符串，把所有符合规则的子串替换成指定内容
- 可控制替换次数，默认替换所有

**基本语法：**
```python
re.sub(pattern, repl, string, count=0, flags=0)
```
- `pattern`：正则匹配规则
- `repl`：替换的新内容（可以是字符串，也可以是函数。如果是函数，每个匹配项都会调用该函数，用返回值作为替换文本）
- `string`：原始字符串
- `count`：可选参数，默认 `0` 表示替换所有匹配项；设为 `1` 只替换第一个；设为 `2` 替换前两个，以此类推
- `flags`：控制匹配方式

老师强调这个函数就像 Word 的"查找替换"功能，可以只替换前两个，也可以替换所有，非常灵活。

**代码演示1——替换所有：**
```python
text = "I love python. Python is good. I love python"
res = re.sub('python', '冰冰', text)  # 未指定 count，默认替换所有
print(res)  # 输出：I love 冰冰. Python is good. I love 冰冰
```
解释：所有小写"python"都被替换，但大写"Python"没有被替换。

**代码演示2——只替换第一个（带警告处理）：**
```python
res = re.sub('python', '冰冰', text, count=1)  # 指定 count=1
print(res)  # 输出：I love 冰冰. Python is good. I love python
```
注意：老师演示时出现了 IDE 的语法警告提示（建议声明参数名），解决办法是显式写出 `count=1`。

**代码演示3——用正则规则替换数字：**
```python
text = "今天是第123天，温度26度"
res = re.sub(r'\d', 'X', text)  # 把所有单个数字替换成 X
print(res)  # 输出：今天是第XXX天，温度X度
```
```python
res = re.sub(r'\d{3}', 'XX', text)  # 把三个连续数字替换成 XX
print(res)  # 输出：今天是第XX天，温度26度（26不构成三个数字，所以没替换）
```

**代码演示4——flags 参数控制忽略大小写：**
```python
text = "123ABC123AAA456AAAAA789A"
res = re.sub('A', '*', text, flags=re.I)  # re.I 表示忽略大小写
print(res)  # 输出：123*BC123***456*****789*
```
解释：不加 `flags=re.I` 时只替换小写 `a`，加上后大写 `A` 也被替换。

**应用场景：**
1. **过滤敏感字符**：把脏话替换成星号（如"垃圾"→"***"）
2. **格式化文本**：统一日期格式（如"2025-11-18"→"2025/11/18"，规则是 `-`，替换为 `/`）
3. **清理无效字符**：删除特殊符号（如 `@`、`#`），或删除多余空白字符（用 `\s` 匹配空白字符，替换为空字符串）

**代码演示5——删除多余空白字符：**
```python
text = "Hello   Python   World"
res = re.sub(r'\s', '', text)  # 删除所有空白字符
print(res)  # 输出：HelloPythonWorld
```

### 五、核心总结与实用技巧

老师最后做了系统的总结，帮助学习者快速区分和使用三个函数。

**三个函数的核心用途对比：**
| 函数 | 核心功能 | 返回值 | 失败时返回 |
|------|---------|--------|-----------|
| `re.search()` | 全局找第一个匹配 | match 对象 | `None` |
| `re.findall()` | 全局找所有匹配 | 列表 | 空列表 `[]` |
| `re.sub()` | 替换匹配内容 | 替换后的新字符串 | 不适用 |

**快速选择函数的方法：**
- **想提取信息**（提取所有手机号、邮箱、日期）→ 用 `findall()`
- **只找第一个匹配**（找第一个日期、第一个链接）→ 用 `search()`
- **验证格式**（验证手机号、邮箱、身份证号是否符合规则）→ 用 `match()`
- **替换或过滤**（打码敏感词、清理特殊字符、统一文本格式）→ 用 `sub()`

**课后练习建议：**
1. 用 `findall()` 从一段文本中提取所有邮箱
2. 用 `sub()` 过滤评论中的敏感词
3. 用 `search()` 从本地日志中找第一个错误信息

## 本P结论

1. **`re.search()`** 会在整个字符串中遍历，找到第一个符合规则的子串就返回 match 对象，如果找不到返回 `None`。与只匹配开头的 `re.match()` 相比，更适合"从任意位置查找"的需求。

2. **`re.findall()`** 同样遍历整个字符串，但会找出所有匹配项，以列表形式返回。如果没有匹配项，返回空列表 `[]` 而不是 `None`，处理起来更省心，不需要额外判断。

3. **`re.sub()`** 用于替换匹配内容，默认替换所有匹配项，可通过 `count` 参数控制只替换前 n 个。可配合 `flags` 参数（如 `re.I` 忽略大小写）实现灵活匹配。典型应用包括敏感词过滤、日期格式统一、清理特殊字符或多余空白。

4. 三个函数的选用原则：提取多个用 `findall()`，只找一个用 `search()`，验证格式用 `match()`，替换内容用 `sub()`。
