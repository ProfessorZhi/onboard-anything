## 核心问题

如何用Python re模块完成"查找"和"替换"两种核心操作？

## 知识骨架

```
re模块函数体系
├── re.match() — 仅匹配开头，验证格式用
├── re.search() — 全局找第一个
├── re.findall() — 全局找所有，返回列表
└── re.sub() — 替换匹配内容
```

三个核心函数构成正则实战的主干：
- search → 定位第一个
- findall → 收集所有
- sub → 修改内容

## 关系与依赖

```
re.match() [已学] ──局限：只匹配开头──→ re.search() [新]
                                              │
                          ┌───────────────────┴───────────────────┐
                          ↓                                       ↓
                    re.findall()                            re.sub()
                    找所有返回列表                            替换内容
                          │                                       │
                          ↓                                       ↓
                  返回值：list                             返回值：str
                  失败：[]                               失败：N/A
```

函数选择决策树：
- 提取多个 → findall
- 找一个 → search
- 验证格式 → match
- 修改内容 → sub

## 输入 / 输出 / 前置 / 支撑

**re.search()**
- 输入：pattern, string, flags(可选)
- 输出：match对象 或 None
- 前置：正则匹配规则基础
- 支撑：re.I 忽略大小写

**re.findall()**
- 输入：pattern, string, flags(可选)
- 输出：list（元素为str子串）
- 前置：正则匹配规则基础
- 支撑：无匹配时返回[]，无需None判断

**re.sub()**
- 输入：pattern, repl, string, count(可选,默认0=全部), flags(可选)
- 输出：替换后的新字符串
- 前置：正则匹配规则基础
- 支撑：re.I忽略大小写、repl可为函数

## 易错点

- match()和search()失败返回None，调用.group()会报错；findall()失败返回[]不会报错
- sub()的count=1只替换第一个，不是替换一次后全替换
- search()只返回第一个匹配，即使字符串中有多个
- 提取match对象内容必须调用.group()方法
- flags参数需显式命名如flags=re.I，避免语法警告

## 外部资源与对象

| 对象 | 类型 | 作用 |
|------|------|------|
| Python re模块 | 标准库 | 全部函数的实现载体 |
| re.I / re.IGNORECASE | 标志常量 | 控制忽略大小写匹配 |
| match对象 | 返回类型 | search()的返回值，需.group()提取内容 |
