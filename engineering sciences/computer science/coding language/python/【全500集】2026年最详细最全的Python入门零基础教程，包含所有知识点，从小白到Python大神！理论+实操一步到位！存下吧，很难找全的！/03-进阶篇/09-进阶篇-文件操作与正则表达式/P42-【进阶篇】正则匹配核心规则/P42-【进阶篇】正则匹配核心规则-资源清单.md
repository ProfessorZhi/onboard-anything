# P42 资源清单 - 【进阶篇】正则匹配核心规则
## 一、库 / 模块

### re 模块
- **类型**：Python 标准库
- **在本 P 中的作用**：所有正则表达式操作的唯一入口，提供 `re.match()`、`re.search()`、`re.findall()` 等函数，本 P 通过 `re.match()` 演示单字符匹配、数量限定符、边界锚点等核心规则
- **来源**：字幕直接提到
- **建议方向**：查阅官方文档中本 P 未涉及的方法（如 `re.search()` 全局搜索、`re.findall()` 提取所有匹配、`re.sub()` 替换），了解其与 `re.match()` 头匹配的差异

---

## 二、命令 / 函数 / 方法

### re.match()
- **类型**：函数（re 模块）
- **在本 P 中的作用**：从字符串**开头**进行匹配，返回 Match 对象或 None，是本 P 所有演示的核心方法
- **来源**：字幕直接提到
- **建议方向**：对比 `re.search()` 在字符串任意位置搜索的能力，理解头匹配与全局搜索的区别

### .group()
- **类型**：方法（Match 对象）
- **在本 P 中的作用**：从 Match 对象中提取匹配到的字符串内容，配合 `re.match()` 返回值使用
- **来源**：字幕直接提到
- **建议方向**：了解 `group(0)` 与 `group(1)`、`group(2)` 等分组提取语法的区别，为后续分组捕获打基础

---

## 三、语法特性 / 概念

### r'' 原始字符串
- **类型**：Python 语法特性（字符串前缀）
- **在本 P 中的作用**：防止反斜杠被 Python 解释器转义，确保正则表达式中的 `\d`、`\w`、`\.` 等按字面意义传递给 re 模块
- **来源**：字幕直接提到
- **建议方向**：理解 Python 字符串转义与正则转义的差异，尝试不加 `r''` 时的行为差异
