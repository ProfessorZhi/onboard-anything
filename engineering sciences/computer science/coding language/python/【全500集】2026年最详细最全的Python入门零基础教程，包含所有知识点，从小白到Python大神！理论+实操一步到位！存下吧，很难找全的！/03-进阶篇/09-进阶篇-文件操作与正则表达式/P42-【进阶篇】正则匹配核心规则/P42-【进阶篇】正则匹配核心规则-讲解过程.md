# P42 讲解过程 - 【进阶篇】正则匹配核心规则
## 本P定位

本P是 Python 正则表达式（re模块）的核心内容，承接上一 P 对 re 模块基本流程（`re.match`、`group()`、`r''`原始字符串）的介绍，进入正则匹配规则本身。目标是通过代码演示让学习者掌握：① 单字符匹配规则；② 数量限定符；③ 边界锚点；④ 一个综合实战（手机号验证 + 邮箱格式验证）。

## 讲解过程

### 一、开场引入（0:01–13:00）

老师先提到已掌握 re 模块的基本使用流程（`re.match` → `group()` 提取），随后指出接下来要学习正则的核心——**匹配规则符号**。这些规则能够覆盖 80% 的基础使用场景，从单个字符的匹配规则开始讲起。

### 二、单个字符匹配规则（13:00–340:00）

#### 2.1 点号 `.`（13:00–91:00）

- **规则含义**：匹配除换行符 `\n` 以外的任意一个字符。
- **代码演示**：
  ```python
  r1 = re.match(r'.', 'A')
  ```
  - 测试字母 `A` → 匹配成功，通过 `group()` 提取。
  - 测试数字 → 匹配成功。
  - 测试转义字符 `\t` → 匹配成功。
  - 测试汉字 → 匹配成功。
  - 测试 `\n`（换行符）→ **匹配失败**，因为点号不包含换行符。
  - 测试特殊符号 `$` → 匹配成功。
- **结论**：点号可以匹配任何非换行符字符，所有结果均可用 `group()` 提取。

#### 2.2 方括号 `[]`（91:00–136:00）

- **规则含义**：方括号内放一组字符，匹配其中**任意一个**字符。
- **代码演示**：
  ```python
  r1 = re.match(r'[ABC]', 'A')   # 匹配成功
  r1 = re.match(r'[ABC]', 'B')   # 匹配成功
  r1 = re.match(r'[ABC]', 'D')   # 匹配失败
  ```
- **说明**：`match` 方法从头匹配，`[ABC]` 匹配字符串开头的第一个字符是否为 A/B/C 之一。

#### 2.3 方括号的范围写法（137:00–209:00）

- **`[0-9]`**：匹配任意一个数字字符，等价于后面要讲的 `\d`。
- **代码演示**：
  ```python
  r1 = re.match(r'[0-9]', '0')   # 匹配成功
  r1 = re.match(r'[0-9]', '9')   # 匹配成功
  r1 = re.match(r'[0-9]', '5')   # 匹配成功
  r1 = re.match(r'[0-9]', 'A')   # 匹配失败（非数字）
  ```
- **结论**：`[0-9]` 匹配 0 到 9 中的任意一个数字字符，其他字符均失败。
- **补充**：
  - `[a-z]` 匹配任意小写字母。
  - `[A-Z]` 匹配任意大写字母。
  - `[a-zA-Z]` 组合后可匹配任意字母，不限制大小写。

#### 2.4 方括号的取反 `[^...]`（211:00–268:00）

- **规则含义**：方括号内开头加 `^`（尖尖符号），表示**取反**，匹配除指定字符以外的任意字符。
- **代码演示**：
  ```python
  r1 = re.match(r'[^0-9]', '5')    # 不加 ^ 时成功，加 ^ 后失败
  r1 = re.match(r'[^0-9]', 'A')    # 加 ^ 后匹配成功（非数字）
  r1 = re.match(r'[^0-9]', '#')    # 加 ^ 后匹配成功
  ```
- **说明**：`[^0-9]` 等价于后面要讲的 `\D`，表示匹配任意一个非数字字符。
- **注意**：`^` 在方括号内（`[^...]`）表示取反，但不在方括号内时表示匹配字符串开头（见后文锚点部分）。

#### 2.5 常用特殊符号：反斜杠系列（278:00–334:00）

老师将带反斜杠的规则统一介绍，并强调使用原始字符串 `r''` 的必要性：

| 符号 | 含义 | 等价于 |
|------|------|--------|
| `\d` | 匹配任意一个**数字**字符 | `[0-9]` |
| `\D` | 匹配任意一个**非数字**字符 | `[^0-9]` |
| `\s` | 匹配任意一个**空白**字符（空格、制表符 `\t`、换行符 `\n` 等） | — |
| `\S` | 匹配任意一个**非空白**字符 | — |
| `\w` | 匹配任意一个**单词**字符（字母、数字、汉字、下划线 `_`） | — |
| `\W` | 匹配任意一个**非单词**字符（`@`、`#`、`\n`、`\t` 等特殊字符） | — |

**重要提示**：所有带反斜杠 `\` 的正则规则，建议在字符串前加 `r`（原始字符串），避免转义符冲突，更加安全。

#### 2.6 小结单字符匹配规则

- `.`：任意字符（不含 `\n`）
- `[]` / `[^...]`：字符集与取反
- `\d` / `\D`、`\s` / `\S`、`\w` / `\W`：预定义字符类

### 三、数量限定符（354:00–641:00）

老师指出：学会单个字符规则后，需要结合**数量限定符**来匹配多个字符。数量限定符跟在单个字符规则后面，限定前一个字符的出现次数。

#### 3.1 星号 `*`（372:00–443:00）

- **规则含义**：匹配前一个字符**零次或多次**。
- **代码演示**：
  ```python
  r1 = re.match(r'\d', '123ABC')       # 不加 *：只匹配到 "1"
  r1 = re.match(r'\d*', '123ABC')       # 加 *：匹配到 "123"
  r1 = re.match(r'\d*', 'ABC')          # 字符全为字母：匹配到空字符（零次）
  ```
- **关键特性**：最少可以是零次，最多次数不限。

#### 3.2 加号 `+`（445:00–501:00）

- **规则含义**：匹配前一个字符**一次或多次**。
- **与星号的区别**：至少要出现一次，不能为零次。
- **代码演示**：
  ```python
  r1 = re.match(r'\d+', 'ABC')   # 无数字 → **报错**（不满足"至少一次"）
  r1 = re.match(r'\d+', '1ABC')  # 有数字 → 匹配到 "1"
  r1 = re.match(r'\d+', '123ABC')  # 多个数字 → 匹配到 "123"
  ```

#### 3.3 问号 `?`（503:00–553:00）

- **规则含义**：匹配前一个字符**零次或一次**（最多一个）。
- **适用场景**：可选字符的场景。
- **代码演示**：
  ```python
  r1 = re.match(r'\d?', '123')   # 有多个数字，但只匹配到一个
  r1 = re.match(r'\d?', 'ABC')  # 无数字 → 匹配空字符
  ```

#### 3.4 `{M}` 精确次数（559:00–592:00）

- **规则含义**：匹配前一个字符恰好 **M 次**。
- **代码演示**：
  ```python
  r1 = re.match(r'\d{3}', '12345')  # 有5个数字，只匹配前3个 → "123"
  r1 = re.match(r'\d{3}', '12')     # 只有2个数字 → **匹配失败**
  ```

#### 3.5 `{M,N}` 范围次数（595:00–640:00）

- **规则含义**：匹配前一个字符至少 **M 次**，最多 **N 次**。
- **代码演示**：
  ```python
  r1 = re.match(r'\d{1,3}', '1')       # 1个 → 匹配 "1"
  r1 = re.match(r'\d{1,3}', '12')      # 2个 → 匹配 "12"
  r1 = re.match(r'\d{1,3}', '123')     # 3个 → 匹配 "123"
  r1 = re.match(r'\d{1,3}', '1234')    # 4个 → 只匹配前3个 → "123"
  r1 = re.match(r'\d{1,3}', 'ABC')    # 0个 → **匹配失败**（不满足最低1次）
  ```

### 四、边界锚点：开头 `^` 和结尾 `$`（641:00–866:00）

#### 4.1 锚点的作用与含义

- `^`（尖尖符号）：匹配**字符串开头**，要求字符串以指定内容开头。
- `$`（dollar 符号）：匹配**字符串结尾**，要求字符串以指定内容结尾。
- **两者结合**：确保**整个字符串**完全符合规则，避免"部分匹配"问题。

#### 4.2 实战：手机号验证（650:00–736:00）

- **需求**：验证手机号是否为 11 位数字。
- **第一版规则** `\d{11}`：
  ```python
  r1 = re.match(r'\d{11}', '13812345678')
  ```
  - 测试 11 位数字 → 匹配成功。
  - 测试 10 位数字 → 匹配失败。
  - 测试包含非数字字符 → 匹配失败。

#### 4.3 暴露第一版的问题：部分匹配（738:00–794:00）

老师演示了一个关键问题：
- 输入 `'ABC13812345678XYZ'` → `'\d{11}'` 仍然匹配成功！它匹配到了中间的 11 位数字，但整个字符串并不是有效手机号。
- **原因**：`match` 只从字符串开头匹配，但只要开头部分满足规则就成功，不会检查字符串后面是否有多余字符。

#### 4.4 解决：加锚点 `^` 和 `$`

- **最终规则** `^\d{11}$`：
  ```python
  r1 = re.match(r'^\d{11}$', 'ABC13812345678XYZ')  # 匹配失败（前后有多余字符）
  r1 = re.match(r'^\d{11}$', '138123456789')       # 匹配失败（12位）
  r1 = re.match(r'^\d{11}$', '13812345678')        # 匹配成功
  ```
- **结论**：加 `^` 和 `$` 后，只有整个字符串恰好是 11 位数字时才能匹配成功。

### 五、综合实战：邮箱格式验证（866:00–1246:00）

#### 5.1 分析邮箱规则

老师先引导分析邮箱的组成部分：
1. **用户名部分**：以单词字符（字母、数字、下划线）开头，**至少一个**字符 → `\w+`
2. **@ 符号**：固定为 `@`
3. **域名部分**：字母或数字，至少一个 → `[\w.]+`（后改为更严格写法）
4. **点号 `.`**：固定为 `.`，**需要转义** `\.`（因为 `.` 在正则中匹配任意字符）
5. **后缀**：2~4 位小写字母 → `[a-z]{2,4}`
6. **边界**：首尾加 `^` 和 `$`

**关于点号的坑**：老师特别强调，如果用 `\w+@\w+\.com` 这样的写法，点号 `.` 会匹配任意字符（包括字母），虽然表面上能匹配 `.com`，但逻辑上不正确。所以点号必须写成 `\.` 进行转义。

#### 5.2 分步构建正则规则

```python
# 开头：以至少一个单词字符开头
r'^\w+'

# @ 符号固定
r'^\w+@'

# 域名部分：字母或数字，至少一个
r'^\w+@[\w.]+'

# 点号转义
r'^\w+@[\w.]+\.'

# 后缀：2~4位小写字母
r'^\w+@[\w.]+\.[a-z]{2,4}$'
```

#### 5.3 完整规则代码演示

```python
# 规则定义
pattern = r'^\w+@[\w.]+\.[a-z]{2,4}$'

# 测试列表
li = [
    'ABC@123.CN',
    '冰冰@QQ.com',
    'invalid.email',
    'test@domain.com.cn'
]

for email in li:
    res = re.match(pattern, email)
    if res:
        print(res.group())
    else:
        print('邮箱格式错误')
```

**运行结果**：
- `ABC@123.CN` → 匹配成功
- `冰冰@QQ.com` → 匹配成功
- `invalid.email` → 匹配失败（没有 `@` 符号）
- `test@domain.com.cn` → 匹配失败（后缀超过4位，`.com.cn` 共5个字符）

#### 5.4 规则局限性说明

老师指出这是一个简化版，实际邮箱规则仍有不足：
- 后缀并不只是 2~4 位小写字母，任何字母组合都可以。
- 并没有限定只能支持某些常见邮箱（QQ、163、Gmail 等）。
- 等后续学到更多正则语法后，可以进一步细化完善。

### 六、代码规范提示：raw string

在演示过程中，老师多次提到使用 `r''` 原始字符串的必要性：
- 所有带反斜杠的正则（如 `\d`、`\w`、转义点号 `\.`），都应使用 `r'...'` 格式。
- 如果不使用 `r`，Python 会把反斜杠当作转义字符处理，产生语法警告（syntax warning），且逻辑可能出错。

## 提到的外部资源与对象

本 P 全部操作均基于 Python 标准库 `re` 模块，无外部资源调用。

## 本P结论

1. **单个字符匹配规则**：`.` 匹配任意非换行符字符；`[]` 匹配字符集中的任意一个字符；`[^...]` 取反；`\d/\D`、`\s/\S`、`\w/\W` 是常用的预定义字符类。所有反斜杠规则建议搭配 `r''` 原始字符串使用。
2. **数量限定符**：`*`（零次或多次）、`+`（一次或多次）、`?`（零次或一次）、`{M}`（恰好M次）、`{M,N}`（M到N次）。加号与星号的核心区别在于是否允许零次。
3. **边界锚点**：`^` 匹配字符串开头，`$` 匹配字符串结尾。两者组合 `^...$` 可以严格限定整个字符串完全符合规则，避免部分匹配问题。
4. **实战应用**：手机号验证规则 `^\d{11}$`、邮箱验证规则 `^\w+@[\w.]+\.[a-z]{2,4}$`，均通过 `re.match()` + `group()` 提取结果进行判断。
5. **点号转义**：正则中的 `.` 匹配任意字符，如需匹配字面意义的点号，必须写 `\.`。
