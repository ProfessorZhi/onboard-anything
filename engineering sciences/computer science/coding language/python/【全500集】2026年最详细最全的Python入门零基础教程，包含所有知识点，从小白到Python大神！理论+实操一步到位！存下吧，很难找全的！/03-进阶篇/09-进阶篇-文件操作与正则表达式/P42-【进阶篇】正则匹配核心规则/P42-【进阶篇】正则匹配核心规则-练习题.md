## 参考答案概览

> 提示：做完后再查看，重点是理解机制而非死记硬背。

| 题号 | 核心考察点 |
|------|------------|
| 1.1 | 符号分类：A-数量、B-数量+字符集、C-边界、D-字符集、E-数量、F-字符集(取反)、G-边界、H-字符集 |
| 1.2 | C（`\d+\.\d{3}` 要求整数部分至少一位小数部分恰好三位） |
| 2.1 | `re.match` 只从头匹配，缺少 `^...$` 导致部分匹配陷阱 |
| 2.2 | `a*`：是/是/是；`a+`：是/否/是；`a?`：是/是/否 |
| 2.3 | `r''` 避免 Python 解释层与正则解释层对反斜杠的双重转义 |
| 3.1 | `r'^1\d{10}$'` |
| 3.2 | `r'^\d{4}-\d{2}-\d{2}$'` → 输出 `2026-01-01` |
| 3.3 | `re.search(r'\d{11}', text).group()` |
| 4.1 | 基础正则无法表达“且”关系，需要 `(?=...)`  lookahead（后续课程内容） |
| 4.2 | 防止跨行匹配，增强正则的段落级控制能力 |
| 4.3 | `r'^(\d{3}-?\d{8}|\d{11})$'`（使用分组和管道符） |
