# P42 知识框架 - 【进阶篇】正则匹配核心规则
## 本P位置

Python 正则表达式系统学习的核心分P。位于「进阶篇」，承接 P41 的 re 模块基本流程（`re.match` → `group()` → `r''`），进入匹配规则本身。是后续更复杂正则场景（分组、替换、预编译）的前置基础。

## 核心问题

如何精确描述和提取字符串中的特定模式？

## 知识骨架

### 一、单字符匹配层

| 符号 | 含义 | 等价替身 |
|------|------|----------|
| `.` | 任意字符（不含 `\n`） | — |
| `\d` | 数字字符 | `[0-9]` |
| `\D` | 非数字字符 | `[^0-9]` |
| `\w` | 单词字符（字母/数字/汉字/下划线） | — |
| `\W` | 非单词字符 | — |
| `\s` | 空白字符（空格/制表符/换行符） | — |
| `\S` | 非空白字符 | — |
| `[abc]` | 字符集中任意一个 | — |
| `[^abc]` | 取反：非 a/b/c 的任意字符 | — |
| `[a-z]` `[A-Z]` | 范围写法 | — |

### 二、数量限定层（修饰前一个字符/规则）

| 符号 | 含义 | 关键特性 |
|------|------|----------|
| `*` | 零次或多次 | 允许零次 |
| `+` | 一次或多次 | 至少一次 |
| `?` | 零次或一次 | 可选 |
| `{M}` | 恰好 M 次 | 精确 |
| `{M,N}` | M 到 N 次 | 范围 |

### 三、边界锚点层

| 符号 | 含义 | 作用 |
|------|------|------|
| `^` | 字符串开头 | 左侧约束 |
| `$` | 字符串结尾 | 右侧约束 |
| `^...$` | 完整匹配 | 防止部分匹配 |

### 四、实战模板

```
^\d{11}$                          # 手机号（11位纯数字）
^\w+@[\w.]+\.[a-z]{2,4}$          # 简化邮箱
```

## 关系与依赖

```
re.match() ←─ 前置：P41 基础流程
    │
    ├── 单字符规则 ──（数量符）─→ 连续字符匹配
    │
    ├── 边界锚点 ──（组合使用）─→ 完整字符串验证
    │
    └── r'' 原始字符串 ──（必须配合）─→ 反斜杠规则（\d \w \.）
```

- 数量符必须跟在单字符规则后面才有意义
- `^` 和 `$` 可单独使用，但组合使用才能解决部分匹配问题
- `r''` 与所有反斜杠规则（`\d` `\w` `\.`）配合使用

## 输入 / 输出 / 前置 / 支撑

- **前置知识（P41）**：re 模块基本流程 → `re.match()` 头匹配、`group()` 提取、`r''` 原始字符串
- **本P输入**：待匹配的字符串
- **本P输出**：Match 对象（通过 `group()` 提取匹配内容）或 None
- **支撑工具**：Python 标准库 `re` 模块（无需第三方库）

## 易错点

1. **点号不匹配换行符**：`\n` 必须用显式规则匹配
2. **方括号内 `^` 是取反**，方括号外才是锚点，两者语义完全不同
3. **星号允许零次，加号至少一次**：`\d+` 对纯字母字符串直接失败
4. **部分匹配陷阱**：不加 `^...$`，`'ABC13812345678XYZ'` 会被 `'\d{11}'` 误判为有效手机号
5. **正则点号需转义**：`.` 在正则中匹配任意字符，匹配字面点号必须写 `\.`
6. **建议全程使用 `r''`**：避免 Python 转义与正则转义冲突

## 外部资源与对象

| 对象 | 类型 | 作用 |
|------|------|------|
| `re` 模块 | Python 标准库 | 所有正则操作的入口 |
| `re.match()` | 函数 | 从字符串开头执行匹配 |
| `.group()` | Match 对象方法 | 提取匹配内容 |
| `r''` 原始字符串 | Python 语法 | 保护反斜杠不被解释 |

本 P 无外部平台、仓库、第三方工具调用，全部基于 Python 内置标准库。
