# P100 讲解过程 - 【数据分析】pandas的常见操作1

本P是pandas常见操作系列的第一节，属于pandas进阶学习的前置内容。老师在回顾了pandas基本属性、读取、保存操作的基础上，引入对现有数据结构的修改与扩充操作。本P覆盖索引修改、列名修改、添加列、添加行、DataFrame对象拼接五种常见操作，但数据合并因内容复杂被拆分到下一节。

## 讲解过程

### 一、开场与内容预告

课程开始，老师回顾了之前的学习内容：pandas的基本属性、基本的读取和保存操作。然后预告本节的学习范围：

- 修改索引（行索引和列名）
- 数据的添加与合并
- 删除数据

关于数据合并，老师明确说明合并有三种方式，但本节只讲一种，因为合并操作较为复杂，一次讲太多学生容易遗忘，建议先理解一种再逐步深入。

### 二、修改行索引

#### 读取基础数据

老师打开上节课保存的数据文件（data1），确认数据已正常读取，数据结构规整。

#### 观察index属性

在数据框上调用`df.index`，输出显示这是一个`RangeIndex`（从0开始的整数序列）。老师强调index本身是一个索引对象，不是普通列表，直接通过下标赋值无法修改其值。

#### 错误尝试：直接下标修改

老师演示了错误的修改方式：

```python
df.index[1] = 1
```

运行后发现修改无效，索引值并未改变。通过`.values`属性可以查看索引底层的数组结构，但同样无法直接单元素修改。老师由此引入正确的修改方法。

#### 方法一：直接赋值（将索引改为从1开始）

老师演示利用`np.arange()`生成整数序列，直接赋值给`df.index`：

```python
df.index = np.arange(1, len(df) + 1)
```

老师特别解释了为什么终止值要`+1`：

- 原始索引是0到15共16个元素
- `np.arange(1, 16)` 生成的是1到15，只有15个元素
- 数量不匹配会导致报错
- 因此终止值必须是`len(df) + 1`，即1到16共16个元素

#### 方法二：rename方法（选择性修改）

老师介绍使用`df.rename()`方法，通过字典形式指定修改规则：

```python
df.rename(index={1: 10}, inplace=True)
```

其中`inplace=True`参数表示在内部直接修改，无需重新赋值（老师提示这是前两天刚讲过的知识点）。此方法适合只修改部分索引值，而不是全量替换。

### 三、修改列名

#### 单个列名修改

老师演示用`df.rename(columns={'时间': '日期'})`将列名从“时间”改为“日期”。老师提到这是个人偏好，认为“日期”更好听。

#### 批量列名修改

老师先尝试用`.columns`属性的方式单个修改，但发现比较麻烦。转而演示直接赋值方式：

```python
df.columns = ['A', 'B', 'C', 'D', 'E', 'F']
```

老师强调赋值列表的元素数量必须与实际列数完全一致，如果数量不匹配会报错。学生需要注意这个细节。

### 四、添加新列

老师关闭之前的代码窗口，开启新文件演示。

#### 创建随机数列

使用`numpy`的随机数功能添加一列分数：

```python
df['分数'] = np.random.randint(0, 101, len(df))
```

老师回顾随机数函数的三个参数：起始值、终止值、生成数量。本例中分数范围设定为0到100分，数量与DataFrame的行数保持一致，确保长度匹配。

运行后可以看到新增的“分数”列，包含随机生成的整数值，每次运行结果不同。

### 五、添加新行

#### 初次尝试：直接赋值方式的问题

老师定义了一个列表作为新行数据，包含了日期、姓名、学历、薪资范围、年龄、城市、分数等字段信息。使用`df['new_row'] = list_data`的方式添加。

运行后发现数据并未作为新行添加，而是变成了一列新字段，原有数据行保持不变。老师指出这是错误的方向，需要重新思考。

#### 重新分析DataFrame的结构

老师通过`df.loc[1]`打印第一行数据，观察返回结果的结构特点。发现打印出来的是一个Series对象，可以通过列名（如`df.loc[1]['A']`）访问对应值。

老师引导学生发现：DataFrame的一行数据类似字典结构，列名是键，值是键对应的值。这个观察为使用字典方式添加行奠定了基础。

#### 方法一：字典方式添加单行

老师将新行数据改写为字典形式，键为列名，值为对应数据：

```python
new_row = {'A': '2024-03-16', 'B': '李雷', 'C': '本科', 'D': '20K~35K', '年龄': 19, '城市': '北京', '分数': 78}
df = df.append(new_row, ignore_index=True)
```

运行后仍然报错。老师用画图方式解释报错原因：添加新行后数据有了，但索引没有来源指定。数据库中的insert语句需要指定主键ID，而这个ID通常是自动增长的；DataFrame的行索引同样需要指定来源。

#### 正确方法：设置ignore_index=True

老师将代码修改为：

```python
df = df.append(new_row, ignore_index=True)
```

再次运行，成功添加了新行数据。新增的“李白”记录（本科、薪资20K~35K、19岁、北京、78分）出现在表格末尾，索引自动增长为16。

老师评价这种添加方式较为繁琐，一般在实际工作中较少使用，除非在循环中批量处理或从其他表格中获取特定字段的数据时才考虑使用。

df2 = df.head()

提取原数据框的前5行作为df2。

## 本P结论

1. **修改行索引**有两种方式：直接赋值`df.index = np.arange()`适用于全量替换；`df.rename(index={旧值: 新值})`适用于选择性修改，配合`inplace=True`可内部修改。

2. **修改列名**同样有两种方式：单个修改用`df.rename(columns={'旧名': '新名'})`；批量修改直接赋值`df.columns = [列表]`，列表长度必须与列数完全一致。

3. **添加列**直接赋值新列名即可，数据来源可以是常量、列表或通过numpy生成的随机数列，长度需与DataFrame行数一致。

4. **添加行**需要使用`df.append(字典或DataFrame, ignore_index=True)`，字典的键对应列名，值对应单元格数据，`ignore_index=True`用于自动生成连续索引。注意`append`方法不修改原DataFrame，需接收返回值。

5. **DataFrame对象拼接**使用`df1.append(df2)`将两个表垂直拼接，保持原有索引值，如需连续索引需额外处理。
