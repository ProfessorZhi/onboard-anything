# P94 知识框架 - 【数据分析】数组的常用操作1
## 本P位置
Python入门 → 数据分析模块 → NumPy数组操作章节
**定位**：承上（数组基础）启下（数组高级操作），是数组处理从理论走向实践的关键一讲。

## 核心问题
- NumPy如何用矢量化替代显式循环？
- 轴（axis）如何控制多维数组的操作方向？
- 数组的排序、转置、添加操作的语法与语义是什么？

## 知识骨架

```
NumPy数组操作
├── 矢量化（核心思想：用数组方法替代循环）
├── 排序
│   ├── 冒泡排序（Python原生，双层for循环）
│   └── np.sort()（NumPy内置，axis控制方向）
│       ├── axis=1 → 按行排序（水平）
│       └── axis=0 → 按列排序（垂直）
├── 转置
│   └── .T 属性（行变列，列变行）
├── 添加
│   ├── np.append(arr, value) → 末尾追加
│   └── np.insert(arr, index, value, axis) → 指定位置插入
└── axis参数（贯穿所有多维操作的基础概念）
    ├── axis=0 → 垂直方向（列）
    └── axis=1 → 水平方向（行）
```

## 关系与依赖

- **矢量化** 是所有NumPy数组操作的底层思想
- **axis概念** 是理解排序、insert方向、多维操作的前提
- **reshape** 是转置操作的前置（需先确定矩阵形状）
- 本P → 下节（删除、去重、拼接、堆叠、拆分）

## 输入 / 输出 / 前置 / 支撑

| 知识点 | 输入 | 输出 | 前置 | 支撑 |
|--------|------|------|------|------|
| np.sort() | 多维数组 + axis | 排序后数组 | axis概念 | NumPy库 |
| .T | 多维数组 | 转置数组 | reshape确定形状 | NumPy库 |
| np.append() | 数组 + 值 | 扩展后数组 | — | NumPy库 |
| np.insert() | 数组 + 索引 + 值 + axis | 插入后数组 | axis概念 | NumPy库 |

**前置知识**：数组切片、布尔数组、花式索引、reshape变形

## 易错点

1. **元素交换顺序**：冒泡排序中先覆盖再赋值temp，导致原始值丢失；正确顺序是先temp保存，再赋值交换
2. **axis方向混淆**：axis=0是列方向（垂直），axis=1是行方向（水平），与直觉可能相反
3. **矢量化缺失**：能用NumPy内置方法时避免用Python循环，执行效率差距1-2个数量级
