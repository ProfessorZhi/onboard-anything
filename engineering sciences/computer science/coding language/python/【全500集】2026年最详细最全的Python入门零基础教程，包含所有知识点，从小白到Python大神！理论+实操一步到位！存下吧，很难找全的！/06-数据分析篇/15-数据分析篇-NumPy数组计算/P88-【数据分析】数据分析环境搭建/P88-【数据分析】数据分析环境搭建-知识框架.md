## 知识骨架

```
Python数据分析环境
├── Python解释器（内核层）
│   ├── 负责代码执行
│   ├── 版本≥3.0即可，3.7测试通过
│   └── 必须加入PATH环境变量
│
├── pip包管理器（工具层）
│   ├── 负责安装/管理第三方库
│   └── 验证命令：pip list
│
├── PyCharm IDE（开发层）
│   ├── 推荐Professional版，功能更强
│   └── 新建项目时需指定正确解释器路径
│
└── 核心库（应用层，按学习顺序）
    ├── NumPy（数组基础库）→ 先学，3-5课掌握
    ├── Pandas（数据处理核心）→ 最重要、最难，贯穿后续
    ├── Matplotlib（基础可视化）→ 数据展示用
    └── Pyecharts（交互图表）→ 更美观，Matplotlib替代选择
```

---

## 关系与依赖

| 层级关系 | 说明 |
|----------|------|
| 解释器 → 库 | Python解释器是底层运行环境，NumPy/Pandas等库需单独安装 |
| NumPy → Pandas | Pandas构建在NumPy之上，NumPy是基础 |
| 数据处理 → 可视化 | Matplotlib/Pyecharts用于处理完成后的结果展示 |
| pip → 所有库 | 所有第三方库通过pip install xxx安装 |
| PyCharm → 解释器 | 新建项目时必须指定已安装的Python解释器路径 |

## 易错点

| 易错点 | 正确做法 |
|--------|----------|
| 安装Python时忘记勾选"Add Python to PATH" | 必须在安装界面勾选，否则命令提示符无法识别python命令 |
| 混淆Python解释器与第三方库的关系 | 解释器仅提供执行环境，numpy/pandas等库需单独pip install |
| PyCharm新建项目时未指定正确解释器 | 需手动选择安装的Python解释器路径，否则项目无法运行 |
| 以为安装Python就包含了所有库 | 解释器只是一个内核，库需要独立安装 |
| 追求最新版本Python | 老师实测3.7版本与库兼容性良好，版本≥3.0即可 |
