# P95 知识框架 - 【数据分析】数组的常用操作2
## 本P位置

数据分析 → NumPy数组操作 → 数组的增删改查

前置：P94 数组的删除操作基础
后置：pandas 数据清洗与分析

本P是NumPy数组操作的能力扩展，承接元素级操作（删除），延伸到数组级操作（去重、拼接），为后续数据整合做准备。

## 核心问题

1. 如何删除指定位置的数组元素？
2. 如何去除重复数据并排序？
3. 如何将多个数组合并为一个？
4. axis参数在二维数组操作中的含义？

## 知识骨架

```
NumPy 数组操作
├── 删除：np.delete(arr, obj, axis=None)
│   ├── axis=None（默认）：展平后按索引删除
│   ├── axis=0：沿行方向删除行
│   └── axis=1：沿列方向删除列
├── 去重：np.unique(arr, return_index, return_counts)
│   └── 返回唯一值（自动排序）
└── 拼接：np.concatenate((arr1, arr2), axis=0)
    ├── axis=0：纵向拼接（行追加），要求列数相同
    └── axis=1：横向拼接（列追加），要求行数相同
```

## 关系与依赖

```
np.delete ← axis概念（从reshape/排序继承）
np.unique ← 依赖一维数组结构
np.concatenate ← 依赖形状匹配规则
          ↓
    数据分析阶段
          ↓
    pandas（数据清洗、分析、可视化）
    ↑
    依赖numpy数组作为数据源
```

axis参数是贯穿数组操作的核心概念：axis=0操作行，axis=1操作列。忘记时通过reshape+delete写小例子验证。

## 输入 / 输出 / 前置 / 支撑

| 函数 | 输入 | 输出 | 前置 | 支撑 |
|---|---|---|---|---|
| np.delete | 数组、索引、[axis] | 删除元素后的新数组 | axis概念 | - |
| np.unique | 数组 | 唯一值数组（排序） | - | - |
| np.concatenate | (arr1, arr2...), axis | 合并后的数组 | 形状匹配 | - |

前置知识：数组创建、reshape、axis基础

## 易错点

1. 二维数组删除时axis含义与直觉相反：axis=1删列，axis=0删行
2. np.unique主要适用于一维数组，二维数组去重效果有限（结构限制）
3. 拼接时形状必须匹配：纵向拼接需列数相同，横向拼接需行数相同
4. 形状不匹配时抛出ValueError
