# P89 知识框架 - 【数据分析】数据分析师与需求分析
## 本P位置
数据分析模块 → 章节一：数据分析背景与流程导论
作用：建立数据分析全局认知，为后续实操（环境配置、数据获取、清洗、分析、可视化）做铺垫
前置：无技术前置，但依赖Python基础语法认知
后续关联：爬虫技术、Pyecharts可视化、Excel进阶

## 核心问题
1. 大数据时代为什么要做数据分析？
2. 数据分析的完整流程是什么？
3. 数据从哪里来？哪些能取、哪些不能取？

## 知识骨架

```
数据分析
├── 背景：海量数据 → 肉眼无法处理 → 需要工具提取价值
├── 定义：从混杂数据中提取有价值信息
├── 类型
│   ├── 描述性：描述现状
│   ├── 探索性：发现规律
│   └── 验证性：检验假设
├── 流程（五步）
│   ├── 1. 明确目的和思路
│   ├── 2. 数据收集
│   ├── 3. 数据处理（清洗）
│   ├── 4. 数据分析
│   └── 5. 数据展现（可视化）
└── 数据获取途径
    ├── 公开数据集下载（阿里天池等）
    ├── 爬虫（需合法）
    └── 购买/企业积累
```

## 输入 / 输出 / 前置 / 支撑

| 类别 | 内容 |
|---|---|
| **输入** | 原始数据（杂乱、空值、错误值）、明确分析目的 |
| **输出** | 清洗后数据、分析结论、可视化图表、决策支撑 |
| **前置** | Python基础语法、Python环境配置 |
| **支撑** | Excel/数据透视表、阿里天池数据集、淘宝网页（演示场景） |

## 易错点
1. **爬虫违法边界**：手机号、个人隐私数据不可爬；需检查robots.txt
2. **数据填充谨慎**：金额类数据不能用平均值随意填充
3. **区分分析类型**：描述性≠探索性≠验证性，阶段不同目的不同
4. **数据清洗优先**：未经清洗的数据直接分析会得出错误结论
