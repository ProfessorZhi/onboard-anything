## 核心问题

- NumPy数组如何指定和转换数据类型？
- 如何在保持元素总数不变的前提下改变数组形状？
- 形状不同的数组能否进行算术运算？其匹配规则是什么？

---

## 知识骨架

```
dtype（数据类型）
    │
    ├── 三种核心类型：int / float / string
    ├── 指定方式：np.arange(..., dtype=int)
    └── 作用：决定数组底层存储格式

reshape（形状重塑）
    │
    ├── 核心约束：元素总数必须能整除目标维度
    ├── 合法示例：10元素 → (2,5)、(5,2)、(1,10)
    ├── 不合法示例：10元素 → (3,3) 不报错但结果不可预期
    └── 二维结构：外层"大框"包多个一维，每行长度必须统一

Broadcasting（广播机制）
    │
    ├── 规则：兼容维度必须相等，NumPy自动扩展不兼容维度
    ├── 典型场景：一维(5,) + 二维(3,5) → 一维广播到每行
    ├── 核心条件：一维长度 = 二维列数
    └── 本质：让形状不等的数组参与运算的自动对齐策略
```

---

## 关系与依赖

```
np.arange() ──→ np.array() ──→ .reshape()
                │
                ├── .dtype ← 指定数据类型
                ├── .shape ← 查看形状
                │
                └── Broadcasting ← 运算层面
                      ↑
                      依赖：数组已创建 + 形状可兼容
```

- **dtype** 是数组的元数据属性，影响存储但不改变结构
- **reshape** 改变结构（shape），为Broadcasting创造兼容形状
- **Broadcasting** 是运算规则，依赖形状匹配，dtype自动兼容

## 易错点

- **dtype指定**：`np.arange(1, 21, 2, dtype=int)` 中的dtype参数位置
- **reshape整除**：10个元素reshape成(3,3)新版NumPy不报错但结果错误，必须手动确保整除
- **Broadcasting误判**：以为维度不同就一定不能相加，实际上列数相等即可广播
- **广播失败**：(5,) + (4,5) 报错，因为一维5列与二维4行不兼容，误以为只要列数相等就行
- **报错信息**：`ValueError: operands could not be broadcast together with shapes (x, y) (m, n)`
