# P91 资源清单 - 【数据分析】索引和切片

> **说明**：本P实际内容为 dtype、reshape、广播机制（Broadcasting），与标题"索引和切片"不符。以下资源清单以实际讲解内容为准。来源标注规则：「字幕直接提到」表示原文明确出现；「基于上下文推断」表示课程演示中隐含依赖但未单独点名。

---

## 一、库与框架

### NumPy

- **类型**：第三方 Python 库
- **在本P中的作用**：全P的核心教学对象。提供 ndarray 数据结构、dtype 系统、reshape 方法及广播机制的全部底层实现；所有代码演示均基于此库运行。
- **来源**：字幕直接提到
- **建议方向**：
  - 阅读 NumPy 官方文档中 [Array creation](https://numpy.org/doc/stable/user/basics.creation.html) 与 [Broadcasting](https://numpy.org/doc/stable/user/basics.broadcasting.html) 两节
  - 对比 NumPy 与 Python 原生列表在运算速度上的差异（可用 `timeit` 模块测量）
  - 查阅 NumPy 版本更新说明，确认当前版本对非法 reshape 的处理行为（本P提到新旧版本行为有差异）

---

## 二、函数与方法

### np.arange()

- **类型**：NumPy 内置函数
- **在本P中的作用**：创建等差整数序列作为数组输入源；本P用其生成 1–20、1–10、11–25 等多段序列，是所有演示的起点。支持 `dtype` 参数在创建时直接指定类型。
- **来源**：字幕直接提到
- **建议方向**：
  - 对比 `np.arange()` 与 `np.linspace()` 的参数语义差异（步长 vs 元素个数）
  - 练习在 `np.arange()` 中分别指定 `dtype=int`、`dtype=float`、`dtype=str`，观察输出格式变化
  - 尝试步长为浮点数时的精度问题，理解为何官方推荐浮点步长时改用 `np.linspace()`

### np.array()

- **类型**：NumPy 内置函数
- **在本P中的作用**：将 Python 原生 `range` 对象或列表转换为 NumPy ndarray；本P用于构造参与广播运算的一维数组 `aa11`、`aa12`。
- **来源**：字幕直接提到
- **建议方向**：
  - 练习传入嵌套列表创建二维数组，观察自动推断的 dtype
  - 对比 `np.array(range(...))` 与 `np.arange()` 在语义和性能上的区别
  - 尝试传入混合类型列表（如 `[1, 2.0, '3']`），观察 NumPy 的类型提升（type casting）行为

### .reshape()

- **类型**：ndarray 实例方法
- **在本P中的作用**：将一维数组重塑为二维形状，为广播机制演示准备形状兼容的操作数；本P演示了合法（2×5、5×2、1×10）与不合法（3×3）两类情况，并指出新版 NumPy 对非法 reshape 不报错但结果不可预期。
- **来源**：字幕直接提到
- **建议方向**：
  - 练习使用 `-1` 作为占位维度（如 `.reshape(-1, 5)`），让 NumPy 自动推算缺失维度
  - 对比 `.reshape()` 返回视图（view）还是副本（copy）的条件，避免意外修改原数组
  - 结合下一集预告的 `.transpose()` 练习行列互换，理解两者的本质区别

## 三、属性

### .dtype

- **类型**：ndarray 实例属性（可在创建时作为参数传入）
- **在本P中的作用**：指定或查看数组的底层数据类型；本P将其作为知识点专项讲解，演示了 `dtype=int` 与 `dtype=string` 的效果，并说明实际使用中只需掌握 `int`、`float`、`string` 三种。
- **来源**：字幕直接提到
- **建议方向**：
  - 查阅 NumPy dtype 体系完整表（bool、int8/16/32/64、float32/64、complex、str 等），理解位宽对内存的影响
  - 练习使用 `.astype()` 方法在已有数组上转换类型，观察精度损失场景（如 float→int 截断）
  - 在大规模数据场景下对比 `int32` 与 `int64` 的内存占用差异，建立"按需选型"意识

### .shape

- **类型**：ndarray 实例属性
- **在本P中的作用**：查看数组的维度结构；本P用于验证 reshape 结果是否符合预期，并在广播机制错误演示中作为报错信息的组成部分（`shapes (2,5) (3,5)`）。
- **来源**：字幕直接提到
- **建议方向**：
  - 练习通过 `.shape` 在运算前做防御性检查，养成验证形状再运算的习惯
  - 理解 `(5,)` 与 `(1,5)` 与 `(5,1)` 三种形状在广播规则下的不同行为

## 四、语言内置对象

### Python range()

- **类型**：Python 内置函数 / 序列类型
- **在本P中的作用**：作为 `np.array()` 的输入序列，生成整数范围；本P中 `range(1, 6)`、`range(6, 11)` 等用于构造演示数组。
- **来源**：基于上下文推断（代码中直接出现，但未单独讲解）
- **建议方向**：
  - 理解 `range` 是惰性对象，传入 `np.array()` 时会被完整求值
  - 对比直接使用 `np.arange()` 替代 `np.array(range(...))` 的简洁性，形成选型习惯

---

## 五、概念与机制

### Broadcasting（广播机制）

- **类型**：NumPy 运算规则 / 核心概念
- **在本P中的作用**：本P的核心难点。解释了形状不完全相同的数组为何能参与算术运算：一维数组（长度等于二维数组列数）会被自动"广播"到每一行，实现逐行对位运算。本P同时演示了广播失败时的 `ValueError` 报错。
- **来源**：字幕直接提到
- **建议方向**：
  - 阅读 NumPy 官方文档 Broadcasting 一节，掌握完整的"从尾部对齐"规则（trailing dimensions rule）
  - 练习更复杂的广播场景：`(3,1)` + `(1,4)` → `(3,4)`，理解维度为1时的自动扩展逻辑
  - 对比广播与显式 `np.tile()` / `np.repeat()` 的效果差异，理解广播的零内存复制优势
  - 在实际数据分析场景中练习：对二维数据集的每一行减去均值向量（标准化操作）

## 六、报错信息

### ValueError: operands could not be broadcast together with shapes

- **类型**：Python 异常 / NumPy 运行时错误信息
- **在本P中的作用**：广播失败时的标准报错信号；本P多次复现此错误（形状不等的一维数组相加、行数不同的二维数组相加），帮助学生建立"形状不兼容→明确报错"的预期。
- **来源**：字幕直接提到
- **建议方向**：
  - 练习主动触发此报错，养成从报错信息中读出"实际形状"并推断原因的能力
  - 结合 `.shape` 属性，练习在运算前手动验证广播兼容性的防御性编程风格
