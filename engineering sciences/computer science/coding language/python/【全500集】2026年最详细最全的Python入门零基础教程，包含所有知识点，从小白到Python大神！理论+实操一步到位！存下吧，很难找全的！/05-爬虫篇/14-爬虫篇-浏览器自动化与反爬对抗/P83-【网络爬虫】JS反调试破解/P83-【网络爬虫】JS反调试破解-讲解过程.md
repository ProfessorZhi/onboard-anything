# P83 讲解过程 - 【网络爬虫】JS反调试破解
## 本P定位

本P是JS逆向系列的延伸，聚焦于反调试技术的识别与破解。承接前面JS逆向案例的讲解，引入网站针对调试行为的防护手段及其绕过方法，属于爬虫工程师进阶技能。

## 讲解过程

### 一、反调试技术概述

老师首先回顾JS逆向在爬虫中的地位，强调这是爬虫工程师必备能力。然后指出当前趋势：部分网站不再只是加密参数，还会对调试行为本身设置障碍，这称为"反调试技术"。

老师列举了常见的反调试手段：

- **快捷键与右键禁用**：禁止F12、右键打开检查
- **调试环境检测**：在JS中植入代码，判断页面是否处于浏览器调试状态，检测到后清空数据或执行恶意代码
- **浏览器窗口状态检测**：通过比较页面可视区域与浏览器总窗口的宽高差，判断是否打开了控制台
- **代码混淆**：使调试分析更加困难
- **定时循环检测**：持续检测调试器状态，发现后执行反调试代码

### 二、基础反调试演示案例

老师编写了一段简单的演示代码，展示两种基础干扰手段：

#### 演示1：debugger无限断点

```html
<script>
  function a() {
    debugger;
  }
  function b() {
    a();
    setTimeout(b, 1000);
  }
  b();
</script>
```

打开浏览器页面时一切正常。一旦打开控制台，代码会立即停在debugger断点位置，形成无限递归，只要尝试调试就会卡在这个位置无法继续。老师演示在断点处点击继续后，代码再次回到断点，无法跳过。老师指出这种手段作用有限，在控制台点击"不再于此处暂停"可解决，但实用价值不大。

#### 演示2：控制台清屏

```javascript
setInterval(function() {
  console.clear();
}, 100);
```

通过定时器每隔100毫秒执行console.clear()，用户输入任何内容到控制台都会立即被清空，无法进行调试观察。老师演示输入变量赋值时内容一闪即逝。老师指出可以在清屏代码处打断点阻止执行，但一旦放行又会持续干扰。

### 三、复杂反调试网站分析

老师打开一个真实的反爬网站（域名未明确说明），展示其多层反调试措施：

#### 第一层防护

- 禁用右键和F12快捷键，无法打开控制台
- 页面弹出提示："检测到非法调试，请关闭浏览器刷新页面"

老师演示绕过方法：在打开页面之前就打开控制台，然后切换到目标页面，这样可以在防护触发前切入。

#### 进入调试后的多重检测

网站进入后继续实施多层检测：

**无限debugger**：代码在内存中通过Function构造函数动态创建函数执行，无法直接修改源码。

**定时清屏**：通过setInterval定时清空控制台内容，干扰调试。

**检测逻辑分析**：

老师逐段分析HTML页面中的反调试代码：

1. 禁用CTRL+S快捷键，触发提示"检测到非法调试"
2. 禁用F12按键，触发相应提示
3. 禁用右键菜单检查功能
4. 检测window对象属性（如webDriver），判断是否通过Selenium等自动化工具打开，是则不执行后续数据加载
5. 检测Canvas指纹用于识别自动化环境
6. nd8函数（用于检测调试器是否开启）：通过定时器每500ms检查window调试状态
7. 定时器每200ms检查调试开关状态，若调试器已开则暂停相关操作

### 四、油猴脚本编写与破解

老师介绍使用油猴插件在浏览器中执行自定义脚本绕过反调试。

#### 油猴插件介绍

油猴（Tampermonkey）是一个浏览器扩展，用于执行用户脚本。安装方式：将下载的crx文件拖入浏览器扩展程序页面。

使用方法：安装后在浏览器右上角点击油猴图标，可添加新脚本。脚本结构包括元数据块（脚本名称、作者、匹配站点等）和执行代码。

#### 第一层脚本编写

针对外层页面的防护，老师编写油猴脚本：

```javascript
// 禁用鼠标右键
document.oncontextmenu = function() { return false; };

// 禁用鼠标左键点击（部分网站的检查绑定）
document.onmousedown = function() { return false; };

// 禁用F12按键
document.onkeydown = document.onkeyup = function(e) {
  if (e.key == 'F12') return false;
};
```

老师演示安装脚本后刷新页面，右键和F12都可以正常使用了。

#### 第二层脚本补充

继续编辑脚本，修改被禁用的全局函数：

```javascript
// 修改debug相关全局函数
window.nd8 = function() {};
window.ndebug = function() {};

// 关闭清屏定时器
clearInterval(定时器句柄);
```

老师解释需要先通过开发者工具找到定时器句柄的位置，然后用clearInterval清除。

### 五、iframe嵌套与第二层防护

老师发现真实数据不在外层页面，而是通过iframe嵌套加载。破解外层后，还需要处理iframe内的页面。

#### iframe内页面的检测手段

**宽高检测**：通过window.outerWidth - window.innerWidth判断差值是否超过300像素，outerHeight - innerHeight判断差值是否超过300像素，若超过则判定打开了控制台。

**时间差检测**：使用debugger断点结合时间戳计算，在断点处停留超过50毫秒即判定为调试状态。关键变量使用const声明防止修改。

```javascript
// 时间差检测逻辑示例
const start = new Date().getTime();
debugger;
const end = new Date().getTime();
if (end - start > 50) {
  // 判定为调试状态
}
```

老师指出由于const声明，脚本无法直接修改这些变量值。

#### 绕过方案

**宽高检测绕过**：将浏览器开发者工具设置为"停靠到窗口右侧"等模式，使页面可视区域保持正常大小，从而绕过宽高差检测。

**时间差检测绕过**：通过定时器在页面数据加载完成后再修改Date对象，使其返回的时间差保持在安全范围内。

```javascript
setTimeout(function() {
  // 延迟5秒后执行，确保数据已加载
  // 修改Date.prototype相关方法
}, 5000);
```

老师演示脚本效果，说明需要针对iframe内的页面单独编写脚本，匹配对应的域名URL。

## 本P结论

JS反调试技术是网站反爬的重要组成部分，常见手段包括快捷键禁用、调试环境检测、窗口状态检测、代码混淆和定时循环检测。绕过反调试需要综合使用多种技术：在页面加载前打开控制台、编写油猴脚本覆盖被禁用的函数和事件、使用外部工具模式避免宽高检测、通过延迟修改关键对象绕过时间差检测。对于iframe嵌套的页面，需要针对不同层级的域名分别编写脚本。本P建立了反调试破解的基本思路，后续可能还有更多进阶内容需要单独录制视频讲解。
