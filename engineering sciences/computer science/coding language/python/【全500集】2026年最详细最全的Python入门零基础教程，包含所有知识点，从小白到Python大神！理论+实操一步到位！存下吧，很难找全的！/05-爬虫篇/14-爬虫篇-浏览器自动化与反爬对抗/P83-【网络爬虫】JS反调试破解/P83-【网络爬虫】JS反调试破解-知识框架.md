# P83 知识框架 - 【网络爬虫】JS反调试破解
## 本P位置
- **课程定位**：网络爬虫 → JS逆向进阶 → 反调试破解
- **承接**：P82 JS逆向基础案例
- **后续**：更多进阶反爬手段（未覆盖）

## 核心问题
如何识别并绕过网站部署的JS反调试机制，从而获取被保护的数据。

## 知识骨架

```
反调试技术
├── 检测类：调试环境检测
│   ├── F12/右键禁用 → 快捷键拦截
│   ├── window属性检测 → 检测webDriver、Canvas指纹
│   ├── 窗口宽高差检测 → outerWidth - innerWidth > 300px
│   └── 时间差检测 → debugger停留 > 50ms
├── 干扰类：调试过程干扰
│   ├── 无限debugger断点 → Function构造函数动态生成
│   └── 定时console.clear() → setInterval清屏
└── 绕过方案
    ├── 先开控制台再切页面
    ├── 油猴脚本覆盖被禁用函数
    ├── 开发者工具停靠侧边 → 避免宽高差
    └── 延迟hook关键对象 → 绕过时间差检测
```

## 关系与依赖

```
浏览器开发者工具 ← 工作环境
       ↓
检测层（网站JS）
       ↓
油猴脚本 ← 绕过层（用户自定义）
       ↓
iframe嵌套页面 ← 需分域名单独处理
       ↓
Selenium检测 ← 自动化工具特征识别
```

反调试与JS逆向是攻防关系：逆向工程师需要先破解反调试，才能进入参数分析阶段。

## 输入 / 输出 / 前置 / 支撑

| 节点 | 说明 |
|------|------|
| **输入** | 目标网站页面，含反调试代码 |
| **输出** | 可正常调试的页面环境，或已提取的数据 |
| **前置** | 浏览器开发者工具使用基础、JavaScript基础、HTML iframe概念 |
| **支撑** | 油猴插件、Tampermonkey脚本语法 |

## 易错点

- debugger无限断点误以为代码卡死，实际只需取消"在此处暂停"
- 以为绕过外层即可，忽略iframe嵌套页面的独立防护
- const声明的变量无法直接修改，需通过原型链或延迟覆盖
- 全局定时器句柄需要先定位再clearInterval，不能凭空写句柄
- webDriver属性检测导致Selenium直接失效，无法通过简单参数伪装绕过

## 外部资源与对象

| 资源/对象 | 角色 | 关联 |
|----------|------|------|
| **油猴（Tampermonkey）** | 浏览器扩展，执行自定义JS脚本 | 核心绕过工具 |
| **开发者工具** | F12调试环境 | 既是工作环境，也是被检测对象 |
| **Selenium / webDriver** | 自动化控制浏览器 | 被网站检测的特征之一 |
| **iframe** | HTML嵌套框架 | 分层防护的载体，需分域名处理 |
| **Function构造函数** | 动态创建函数 | 网站用于生成不可见源码的debugger |
| **Date.prototype** | 时间对象原型 | 时间差检测的hook目标 |
