## 参考答案要点

**结构识别**
1. `歌曲ID` → `下载地址`
2. D → B → A → C

**机制理解**
3. 签名验证用于验证请求的合法性，防止请求被篡改或伪造。主要防御无头的自动化脚本和简单重放攻击。
4. IP代理池解决请求频率过高导致的IP封禁问题；Cookie池解决频繁请求被识别为机器人（同一会话大量请求）的问题。两者配合可在保持请求频率的同时伪装成不同用户。

**最小实践**
5. `https://example.com/api/download/?id=12345`
6. 会报“xxx未定义”错误（找不到md5或base64函数）。解决方法是完整追踪调用链，将所有被调用的子函数一并提取。

**扩展思考**
7. 可采用容错重试机制或记录失败项事后补抓的策略。涉及异常处理编程模式。
8. 直接提取：优点是省时、保证与原站一致；风险是依赖外部JS环境、可能违反网站服务条款。Python重写：优点是完全可控、不依赖JS环境；风险是逻辑复杂时重写困难、可能遗漏细节。
