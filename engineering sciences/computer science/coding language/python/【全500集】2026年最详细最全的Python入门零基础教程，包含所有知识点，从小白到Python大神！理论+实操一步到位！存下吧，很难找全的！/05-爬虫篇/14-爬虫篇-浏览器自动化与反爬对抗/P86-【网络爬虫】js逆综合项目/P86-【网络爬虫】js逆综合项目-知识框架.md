# P86 知识框架 - 【网络爬虫】js逆综合项目
## 本P位置
网络爬虫章节 · JS逆向综合项目实战
前承：单点反爬技术（IP代理、Cookie池、并发）已铺垫
后续：本P完成列表抓取+JS逆向分析，第三步下载及并发实现留待后续

## 核心问题
**如何在有签名验证的网站中完整抓取全站数据？**
拆解为三个子问题：
1. 数据接口在哪里（定位）
2. 请求参数如何构造（参数分析）
3. 签名如何生成（JS逆向）

## 知识骨架

### 一、数据采集三层架构

| 层级 | 任务 | 关键技术 |
|------|------|----------|
| L1 列表层 | 获取歌曲名称、ID、首页URL | requests + xpath |
| L2 详情层 | 获取下载地址 | requests + jsonpath |
| L3 下载层 | 保存文件到本地 | 文件IO |

### 二、JS逆向四步法（本章核心方法论）

```
1. 抓包定位 → 找到目标请求
2. 参数分析 → 对比多请求，筛出动态参数
3. 代码定位 → 搜索关键字，进入生成函数
4. 代码提取 → 不重写，直接把JS函数搬到HTML调用
```

### 三、反爬对抗技术栈

```
并发请求 ──→ 触发IP封禁 ──→ 动态IP代理池
     │
     └─→ 触发Cookie识别 ──→ Cookie池轮换
```

## 关系与依赖

```
歌曲列表页 ──[解析]──→ 歌曲ID ──[构造请求]──→ 下载地址接口
                                        │
                                   [JS逆向]──→ 签名 + 时间戳
                                        │
                                   [验证通过]──→ 获取下载地址

下载地址 ──[下载]──→ 本地文件
```

**依赖链**：L1输出是L2输入 → 签名破解是下载地址请求的前提 → 所有请求依赖反爬对抗措施

## 输入 / 输出 / 前置 / 支撑

### 输入
- 目标网站URL（歌曲列表首页）
- 页码范围（已知：50页）

### 输出
- L1：`[(歌名, 歌曲ID, 首页URL), ...]`
- L2：`{歌曲ID: 下载地址}`
- L3：本地音频文件

### 前置依赖
- HTTP协议基础（GET请求、URL构造）
- HTML结构理解（DOM树、属性定位）
- JavaScript基础（函数调用、字符串拼接）

### 支撑工具
- 浏览器开发者工具（Network面板抓包、Sources面板JS调试）
- Python开发环境（PyCharm）

## 易错点

| 场景 | 错误表现 | 正确做法 |
|------|----------|----------|
| URL拼接 | 多/少斜杠导致404 | 手动验证拼接后URL可访问 |
| 签名函数提取 | 报"xxx未定义" | 逐步补全所有被调用的子函数 |
| 动态参数识别 | 遗漏隐藏参数导致签名错误 | 对比多个不同请求，逐一确认 |
| JS函数依赖 | 只复制主函数 | 完整追踪调用链，复制全部依赖 |
