# P86 讲解过程 - 【网络爬虫】js逆综合项目
## 本P定位

本P是一个**JS逆向综合项目实战**，目标是对一个原创音乐交易平台进行整站数据抓取。课程从需求分析入手，逐步演示了如何通过抓包定位接口、分析请求参数、完成第一步歌曲列表页面的抓取，并开始深入分析第二个接口（获取下载地址）的签名生成逻辑。核心难点在于JS逆向，涉及到请求参数中签名的生成规则分析。

## 讲解过程

### 一、项目背景与需求说明

老师首先打开目标网站，这是一个原创音乐交易平台，页面中展示了大量原创音乐列表。目标很明确：把整个网站的原创音乐数据全部抓取下来，具体需要获取每首歌曲的名字、歌曲ID、以及最终的下载地址。

网站共有50页，每页约20首歌，总量约1000首。

### 二、整体抓取思路规划

老师分析了整个数据抓取的流程，将其拆解为三个步骤：

**第一步**，获取每一页的歌曲列表，从中提取歌曲名称、歌曲ID、以及歌曲首页地址。

**第二步**，拿着歌曲ID去请求一个专门获取下载地址的接口。

**第三步**，根据下载地址下载歌曲并保存到本地。

### 三、技术栈与技术点梳理

老师明确列出了本项目涉及的所有技术点：

基础技术方面，使用`requests`库发送HTTP请求，使用`xpath`解析HTML页面提取数据，使用`jsonpath`处理JSON响应数据，以及文件IO操作进行歌曲保存。

进阶反爬对抗技术方面，由于全站数据量大（2000+请求），需要引入**并发**来提升抓取效率。同时因为并发请求频繁，极易触发IP封禁，所以需要使用**动态IP代理池**。此外，在获取下载地址的接口中，请求需要携带Cookie，同一个Cookie一直请求会被识别为同一用户，所以还需要引入**Cookie池**轮换使用。

### 四、第一步实现：抓取歌曲列表

老师打开目标网站的歌曲列表页面，通过浏览器开发者工具分析请求参数。发现请求URL中有一个`p`参数代表页码，其他参数暂不关注。直接在浏览器地址栏修改`p=1`可以访问第一页。

老师创建了一个Python项目，编写脚本发送`requests.get`请求，直接请求列表页面，发现**不需要Cookie**就能直接拿到HTML数据，数据中包含了所有歌曲信息。

接下来使用xpath进行解析：

找到包含所有歌曲的父容器（`div`标签`class`属性值定位），其下有多个`dl`标签，每个`dl`代表一首歌。用xpath定位到所有的`dl`元素后进行遍历。

在每个`dl`标签下分别提取：歌曲名称——从`h3`下的`a`标签的`text`属性获取；歌曲首页地址——从同一个`a`标签的`href`属性获取，但由于是相对路径，需要拼接完整的域名才能变成可访问的URL；歌曲ID——从"收藏"按钮的`a`标签的`id`属性获取。

老师演示了代码编写和调试过程：初次运行时发现首页地址拼接后多了一个斜杠导致404，修正后成功获取到歌曲ID、名称和首页地址。第一步任务完成。

### 五、第二步分析：获取下载地址的接口与JS逆向

老师来到某首歌的首页，通过开发者工具抓包分析下载请求。找到了一个返回歌曲详细信息的接口，响应数据中包含了歌曲的下载地址。

分析这个GET请求的参数，发现参数很多。老师通过对比两首不同歌曲的请求参数，逐一排查哪些参数是固定不变的，哪些是动态变化的。

分析结论：大部分参数是固定不变的，只有三个参数是动态的，分别是歌曲ID（明显，随歌曲变化）、时间戳（形如`17xxxx`的数字串，一看就是时间戳格式）、以及签名（未知生成方式的加密字符串）。

### 六、JS逆向：定位签名生成函数

老师把不变的部分记录下来，然后将目标转向签名参数——这是整个逆向的核心。

在开发者工具中搜索签名关键字，找到了生成签名的JavaScript代码位置。老师打了断点，刷新页面让程序中断到签名生成的地方。

进入生成函数，发现签名生成的规则如下：先将所有请求参数转换成字符串格式，然后在参数字符串前面拼接一段固定前缀字符串，后面拼接一段固定后缀字符串，最后调用一个加密函数生成最终的签名。

老师验证了前缀和后缀字符串是固定不变的——换了不同浏览器、不同歌曲，这两段字符串始终不变，因此可以硬编码到代码中。

### 七、JS逆向：提取签名生成函数

进入加密函数内部查看，发现调用链路很长，嵌套了多个函数（R、E、S、M、A、C等），层层调用、循环处理，整体逻辑复杂。如果用Python逐一实现这些函数的等价逻辑，工作量巨大且容易出错。

老师的策略是：不重写，直接把JavaScript函数代码复制出来，然后在HTML文件中创建一个`demo.html`，将签名生成相关的函数（包括主函数和它内部调用的所有子函数）全部粘贴进去。

在`demo.html`中调用这个函数，传入待加密的参数字符串，验证能否成功生成签名。如果报"xxx未定义"的错误，就说明这个函数内部还调用了其他函数，需要继续把这些被调用的函数也复制进来。

老师演示了这个逐步补全函数依赖的过程，直到所有被调用的函数都齐全，能够成功执行并输出签名结果。

## 本P结论

1. 原创音乐交易平台的歌曲列表页面可以直接通过`requests`请求HTML获取，无需Cookie和复杂参数，用`xpath`解析即可提取歌曲名称、ID和首页地址。

2. 获取歌曲下载地址的接口请求参数中，大部分是固定值，只有三个动态参数：歌曲ID（已知）、时间戳（直接生成）、签名（需要通过JS逆向获取）。

3. 签名的生成规则是：固定前缀字符串 + 请求参数拼接字符串 + 固定后缀字符串 → 调用加密函数。由于加密函数调用链复杂，老师采用直接提取JavaScript代码到HTML文件中进行调用验证的方案。

4. 本P只完成了第一步（歌曲列表抓取）和第二步的JS逆向分析部分，第三步（下载歌曲）以及并发、动态IP、Cookie池的代码实现将在后续内容中继续。
