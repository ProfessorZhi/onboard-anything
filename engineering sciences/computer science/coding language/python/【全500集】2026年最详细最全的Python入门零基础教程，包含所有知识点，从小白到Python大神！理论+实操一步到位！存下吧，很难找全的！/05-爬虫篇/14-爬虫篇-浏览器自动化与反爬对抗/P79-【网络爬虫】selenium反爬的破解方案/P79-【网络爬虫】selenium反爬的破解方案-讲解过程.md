# P79 讲解过程 - 【网络爬虫】selenium反爬的破解方案
## 本P定位

本P是Python爬虫系列课程的一个技术专题，聚焦于selenium自动化工具被网站反爬检测后的破解方案。老师先讲解反爬的检测原理，再通过一个实际案例演示如何绕过检测并成功获取数据。

## 讲解过程

### 一、反爬检测的背景与原理

课程开场，老师指出selenium作为自动化测试工具，虽然能够模拟浏览器操作来爬取数据，但很多平台针对selenium已经部署了反爬措施。

老师逐一讲解了几种常见的检测方式：

**1. 浏览器窗口信息栏提示**

当使用selenium启动Chrome浏览器时，窗口顶部会显示"Chrome 正受到自动化测试软件的控制"这样的文字标识。这个标识本质上是Chrome的infobar机制，任何网站都可以通过JavaScript检测页面是否存在该特征。

**2. navigator.webdriver 属性检测**

正常的浏览器在控制台执行 `navigator.webdriver` 会返回 `undefined` 或 `false`，但通过selenium启动的浏览器会返回 `true`。网站通过JS读取这个属性值，就能判断访问者是否使用了selenium。

**3. 用户行为特征检测**

真人用户浏览网页时，滚动速度不均匀、点击操作有停顿、鼠标移动轨迹自然。而爬虫为了高效采集数据，通常会快速滚动、直接点击下一页。这种行为模式的差异也会被反爬系统捕获。

**4. JS代码植入的陷阱元素**

部分网站会在前端JS中埋设特殊元素或属性，监控浏览器的某些行为是否被触发，比如检测用户是否按下了F12打开开发者工具。

老师强调，这些检测手段本质都是"特征识别"，对应的破解思路就是"特征伪装"——它检测什么，你就改什么。

### 二、演示案例：空气质量网站反爬

老师选取了一个空气质量查询网站作为演示对象。

**第一步：正常访问**

老师直接在浏览器中打开该网站，可以看到数据正常加载，页面显示友好。

**第二步：使用selenium访问**

老师演示了基础selenium代码结构：导入WebDriver、创建Chrome对象、打开目标URL、设置等待时间。运行后发现，页面虽然能够打开，但数据区域为空，没有任何内容展示。网站已经识别出这是selenium启动的浏览器，因此拒绝加载真实数据。

### 三、破解方案一：Chrome配置参数

老师介绍通过ChromeOptions添加启动参数来实现初级伪装。

**参数一：`--disable-infobars`**

这个参数的作用是禁用浏览器上方的信息栏，执行后窗口顶部不会再显示"正在受到自动化测试软件的控制"的提示文字。老师在代码中演示添加这个参数后，重新运行程序，确认该提示文字确实消失了。

**参数二：`--disable-extensions`**

禁用浏览器扩展程序，防止扩展特征被用于识别自动化工具。

**参数三：`--disable-dev-shm-usage`**

禁用开发者模式相关功能，进一步减少可被检测的特征点。

老师演示这三个参数一起添加到ChromeOptions中时，infobar提示确实去掉了，但网站仍然不加载数据，说明还有其他检测机制在起作用。

### 四、破解方案二：执行防检测JS脚本

老师指出，网站很可能是通过JavaScript检测`navigator.webdriver`这个属性来判断是否使用selenium。因此需要用JS脚本来修改或删除这些属性值。

**JS脚本的作用机制**

老师在课程中提供了一个现成的防检测脚本，该脚本会在每次打开新页面之前执行，用于覆盖或清除selenium启动浏览器时生成的相关属性，让网站无法通过JS检测到自动化工具的特征。

**代码层面的实现**

老师演示了在selenium中使用`execute_script()`命令来执行JS脚本。关键点在于：这个脚本必须在WebDriver对象创建之后、访问目标页面之前执行。老师将脚本文件路径或脚本内容直接写入代码中，并强调这是固定的几个参数组合，不需要自己编写，直接复制使用即可。

**验证效果**

再次运行程序后，等待页面加载完成（老师提到这个网站网速较慢，加载需要一定时间），最终数据成功显示出来了，说明反爬检测已被成功绕过。

### 五、代码总结

课程最后，老师将完整的破解代码框架进行了梳理：

- 导入selenium相关模块
- 创建ChromeOptions配置对象
- 添加三个启动参数（disable-infobars、disable-extensions、disable-dev-shm-usage）
- 将配置传入WebDriver初始化
- 创建WebDriver对象后、get访问页面之前，执行防检测JS脚本
- 后续即可正常进行元素定位和数据提取

老师明确表示，这套代码逻辑是固定的，实际使用时直接复制模板即可，不需要每次都手动编写。

**防检测JS脚本**：老师提供的现成脚本文件，用于覆盖`navigator.webdriver`等selenium特征属性，实现浏览器伪装。该脚本在课程中直接展示代码内容，可直接复制使用。

## 本P结论

1. selenium启动的Chrome浏览器存在多个可被检测的特征，包括窗口infobar提示、`navigator.webdriver`属性、用户行为模式等。

2. 绕过检测需要两步走：第一步通过Chrome启动参数禁用infobars和扩展；第二步通过JS脚本在页面加载前修改或清除浏览器属性。

3. 这套破解方案的核心逻辑是固定的，配置参数和JS脚本都可以作为模板直接复用。

4. 实际爬虫项目中，如果遇到selenium被拒绝加载数据的情况，可以尝试本P介绍的方法进行排查和应对。
