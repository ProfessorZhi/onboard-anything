## 核心问题

如何避免高并发爬虫被目标网站识别与限流？

- **IP维度**：集中请求暴露单一出口IP
- **用户维度**：高频请求触发用户维度检测
- **解决路径**：IP池分散请求来源 + Cookie池轮换用户身份

## 知识骨架

### 一、IP池

**概念**：预先储备可用的代理IP池，请求时随机选取，通过代理服务器转发请求

**三层结构**：

```
请求 → 随机代理IP → 目标服务器 → 代理IP → 响应
```

**实现要点**：

- 从代理服务商API获取IP列表（HTTP/HTTPS代理）
- requests通过`proxies`参数注入代理
- 池化封装：初始化时批量获取，发送时随机选取

### 二、Cookie池

**概念**：预存多个Cookie值，请求时随机轮换，模拟多用户访问

**两类场景**：

- 需登录网站：多账号批量登录，从响应中提取Cookie
- 无需登录网站：手动多次访问采集不同Cookie

**实现要点**：

- 登录场景：`response.cookies`获取Cookie对象
- Cookie管理类：初始化时预加载，随机返回

### 三、组合方案

**思路**：每个Cookie与一个代理IP一一绑定，每次请求同时轮换，模拟不同地点、不同账号的真实访问

## 易错点

1. **白名单未配置**：调用代理API时报错"当前IP不在白名单内"，需在平台将本机IP加入白名单
2. **代理过期**：代理IP有有效期（分钟级），需在过期前重新获取
3. **Cookie时效**：Cookie会过期，需定期刷新或重新登录
4. **协议不匹配**：HTTP代理和HTTPS请求混用，需正确配置`proxies`字典的键
5. **池耗尽未重试**：IP池/Cookie池用空后无兜底机制，导致后续请求失败
