# P81 讲解过程 - 【网络爬虫】JS逆向滑动验证码轨迹破解
## 本P定位

本P是JS逆向实战系列的一部分，目标是**破解去哪网登录流程中的滑动验证码**，通过构造伪造的滑动轨迹来绕过前端验证，从而完成自动化登录流程中的验证码发送步骤。

本P属于**基础构建阶段**，重点在于分析滑动验证码的请求结构、加密方式，以及通过Python代码模拟生成符合要求的滑动轨迹参数。

## 讲解过程

### 1. 案例演示与需求分析

**开场引入**
- 老师说明今天继续讲解JS逆向
- 目标案例是去哪网（qunar.com），一个用户常用的旅行预订网站
- 演示登录场景：输入手机号后需要先通过滑动验证码才能获取短信验证码

**抓包演示**
- 打开浏览器开发者工具，切到Network标签
- 输入手机号，点击“获取验证码”
- 滑动验证码出现，需要将滑块拖动到最右边
- 拖动完成后，Network中捕获两个关键请求

**需求拆解**
老师边分析边在纸上写下接口分析：

1. **第一个接口：滑动验证请求**
   - 拖动滑块后触发
   - 验证滑块轨迹是否通过

2. **第二个接口：获取手机验证码请求**
   - 需要携带滑动验证通过的token
   - 携带手机号码参数
   - 请求成功后手机收到短信验证码

3. **第三个接口：登录请求**
   - 输入短信验证码后提交
   - 完成登录流程

老师强调：这三个接口需要分别分析，本P重点攻克第一个——**滑动验证请求的参数构造**。

### 3. 滑动轨迹结构解析

老师详细讲解轨迹信息的构成，以抓包获取的真实数据为例：

**轨迹数据格式**
每条轨迹记录包含**四个值**：

| 序号 | 含义 | 示例值 |
|------|------|--------|
| 第一个 | 时间戳后五位（精确到毫秒） | 58387 |
| 第二个 | 滑块X轴坐标（距浏览器左边距离） | 42 |
| 第三个 | 滑块Y轴坐标（距浏览器顶部距离） | 固定值 |
| 第四个 | 累计滑动的总距离 | 0 |

**数据解读**
- 时间戳后五位：鼠标按下滑块的精确时间
- X轴坐标：初始位置是42像素（滑块起始位置）
- Y轴坐标：固定值，与浏览器窗口大小有关
- 滑动距离：第一条记录为0，表示还未开始滑动

**完整轨迹示例**
老师展示一段真实的滑动轨迹数据，说明：
- 从起始位置开始
- 每次记录一个时间节点时的滑块位置
- 直到拖动到最右边（X轴距离达到约438像素）
- 整个滑动过程生成多条记录

### 4. AES加密方式分析

### 5. Python代码构造滑动轨迹

**新建文件**
老师创建Python文件，开始构造模拟轨迹的函数

**参数准备**
```python
import time
import random

# 获取开始时间（精确到毫秒）
start_time = int(time.time() * 1000)

# 生成随机总距离（422~438像素范围）
total_width = random.randint(422, 438)

# 定义X轴Y轴起始位置（可随意设定）
x_pos = 200
y_pos = 300
```

老师解释：
- 滑动总宽度是范围值，不需要拖到最顶部
- X轴Y轴起始位置与浏览器窗口有关，可以随机设定

**轨迹生成逻辑**
```python
track_list = []
current_x = 0
current_time = start_time

while current_x < total_width:
    # 生成随机滑动距离（2~50像素）
    step_width = random.randint(10, 50)
    
    # 如果超过总宽度，调整为总宽度
    if current_x + step_width > total_width:
        step_width = total_width - current_x
    
    # 每条记录四个值
    time_str = str(current_time)[-5:]
    track_list.append([time_str, current_x, y_pos, current_x])
    
    # 更新位置和时间
    current_x += step_width
    current_time += random.randint(20, 35)
```

**初始值处理**
老师调试时发现第一个位置应该是0，需要特殊处理：
```python
# 先添加初始位置
track_list.append([str(start_time)[-5:], 0, y_pos, 0])
```

### 7. AES加密实现

**提取密钥**
- 从JS代码中找到AES密钥
- 密钥是固定的字符串

**Python实现AES加密**
```python
from Crypto.Cipher import AES

def aes_encrypt(data, key):
    cipher = AES.new(key.encode(), AES.MODE_ECB)
    # PKCS7填充会自动处理
    encrypted = cipher.encrypt(data.encode())
    return encrypted.hex()
```

## 本P结论

1. **去哪网滑动验证码由三个接口组成**：滑动验证请求、获取验证码请求、登录请求，本P重点突破第一个

2. **滑动轨迹参数结构**：每条轨迹包含四个值——时间戳后五位、X轴坐标、Y轴坐标、累计滑动距离，完整轨迹是一个二维列表

3. **加密方式**：滑动轨迹数据经过JSON序列化后，使用AES-ECB-PKCS7进行加密，密钥固定可从JS中提取

4. **轨迹生成策略**：总滑动距离在422-438像素范围内随机，单次滑动距离2-50像素随机，时间间隔20-35毫秒随机
