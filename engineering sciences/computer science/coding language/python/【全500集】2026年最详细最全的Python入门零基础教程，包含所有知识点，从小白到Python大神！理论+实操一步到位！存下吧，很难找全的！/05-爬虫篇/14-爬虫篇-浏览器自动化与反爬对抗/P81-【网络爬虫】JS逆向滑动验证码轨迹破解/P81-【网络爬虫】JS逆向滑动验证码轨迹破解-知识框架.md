# P81 知识框架 - 【网络爬虫】JS逆向滑动验证码轨迹破解
## 本P位置

JS逆向实战系列，进阶爬虫技能节点。前置：HTTP抓包基础、Python基础、JS阅读能力。后续：浏览器自动化控制、接口串联完成完整登录。

## 核心问题

如何构造伪造的滑动轨迹参数，绕过去哪网滑动验证码的前端验证。

## 知识骨架

1. **接口分析**：登录流程三个接口——滑动验证请求（目标）、获取验证码请求（依赖token）、登录请求（最终目标）
2. **轨迹数据格式**：每条记录4个值——时间戳后5位、X坐标、Y坐标、累计滑动距离，二维列表结构
3. **轨迹生成策略**：总距离422-438px随机，单次步长2-50px随机，时间间隔20-35ms随机，首条记录为[时间戳后5位, 0, Y固定, 0]
4. **AES加密实现**：AES-ECB模式，PKCS7填充，16字节固定密钥，参数经JSON序列化后加密
5. **验证结果列表**：固定格式列表，10-100个随机数量

## 关系与依赖

抓包分析（Chrome DevTools Network） → 参数定位 → JS断点调试（Sources） → 加密逻辑还原 → Python轨迹构造 → AES加密 → 参数组装 → 提交验证请求

## 输入 / 输出 / 前置 / 支撑

**前置能力**：Python基础、HTTP协议理解、Chrome DevTools使用
**输入**：去哪网登录页面滑动验证码
**输出**：可绕过验证的滑动轨迹参数（加密后）
**支撑工具**：Chrome DevTools（Network抓包、Sources断点）、PyCryptodome（Python AES加密库）

## 易错点

- 时间戳截取错误：必须是str时间戳的**后五位**，不是前五位
- 初始位置遗漏：首条记录X坐标必须为0，表示滑块起始位置
- 滑动距离超限：总距离必须落在422-438px范围内，过大或过小都会被风控
- 密钥硬编码：密钥从JS中提取后需验证是否固定，不可用随机密钥
