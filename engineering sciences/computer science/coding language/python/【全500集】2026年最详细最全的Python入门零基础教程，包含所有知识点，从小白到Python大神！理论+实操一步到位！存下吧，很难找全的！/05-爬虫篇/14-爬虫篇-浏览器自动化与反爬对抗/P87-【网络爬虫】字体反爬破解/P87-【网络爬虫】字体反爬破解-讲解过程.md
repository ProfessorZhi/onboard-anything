# P87 讲解过程 - 【网络爬虫】字体反爬破解
## 本P定位

本P是字体反爬的专题课，属于网络爬虫进阶内容。老师采用“先实现后破解”的教学路径：先从开发者视角演示如何用自定义字体实现反爬，再从爬虫视角讲解对应的破解思路。核心目标是让学习者理解字体反爬的本质原理，掌握从字体文件中提取映射关系的技能。

## 讲解过程

### 一、问题引入：什么是字体反爬

老师首先指出一个常见的爬虫困境：网页上明明能看到正常的内容，但查看HTML源码时却是一堆乱码。这就是字体反爬的典型特征。老师强调，理解原理之后会发现这个东西其实并不复杂。

### 二、字体反爬的原理解析

老师开始讲解字体反爬的实现原理，这是本P的理论核心部分。

**字体文件的本质**

老师指出，字体文件本质上是一张张矢量图（称为“字形”）。在HTML页面中写的字符，当设置了自定义字体后，渲染过程是这样的：

1. 浏览器先获取HTML中的字符
2. 将字符转换为UNICODE字符码
3. 将UNICODE字符码转换为16进制数据
4. 在字体文件中查找该16进制字符码对应的字形
5. 将字形绘制到网页上

**自定义字体的映射原理**

老师用一个具体例子说明：假如在HTML中写了一个乱码字符，而在字体文件中，这个乱码对应的字形被设计成数字“1234”。那么页面显示的就是1234，但源码中看到的是乱码。这就是字体反爬的核心原理——通过修改UNICODE字符码与字形的对应关系，让源码与显示内容不一致。

### 三、开发视角：使用FontCreator实现字体反爬

老师切换到开发者视角，演示如何使用FontCreator工具创建自定义字体文件。

**第一步：安装并打开FontCreator**

老师提到需要使用FontCreator这个工具来创建字体文件。打开工具后，老师已经预先绘制了一些数字字体（0123456789）。

**第二步：建立字符映射关系**

老师演示了具体的映射配置过程。假设要将显示为“零”的位置映射到数字：
- 字符“0”对应数字“零”
- 字符“1”对应数字“一”
- 字符“2”对应数字“二”
- 以此类推...

老师强调，这个映射关系是通过UNICODE字符码来关联的。每个字形都有一个对应的字符码，格式类似于“0xXXXX”（16进制）。

**第三步：Python中获取文字的UNICODE字符码**

老师切换到Python环境，演示如何获取中文字符的UNICODE码：

```python
s = "网络爬虫的代码"
for i in s:
    u = ord(i)
    hex_code = hex(u)
    print(hex_code)
```

运行后会得到每个字符对应的16进制UNICODE码。老师提示前面的"0x"前缀可以忽略，只需要记后面的4位即可。

**第四步：在FontCreator中配置字形信息**

回到FontCreator工具，老师演示逐个修改字形信息：
- 选中要修改的字形
- 在字形信息中填写对应的UNICODE字符码（去掉0x前缀）
- 同时设置一个可识别的字形名称
- 保存修改

老师依次修改了数字0到9的字形信息，每个都对应不同的中文或符号。

**第五步：导出字体文件**

老师介绍了三种常见的字体文件格式：
- TTF格式：兼容性最好，各个浏览器和平台都能支持
- WOFF格式：相对TTF更轻量，体积更小，做了压缩
- WOFF2格式：WOFF的升级版本

老师演示了导出字体文件的过程。

### 四、前端实现：在网页中使用自定义字体

老师新建一个HTML网页文件，演示如何应用自定义字体实现反爬。

**引入字体文件**

在CSS中添加@font-face规则：

```css
@font-face {
    font-family: '自定义字体名称';
    src: url('字体文件路径');
}
```

**应用字体**

在CSS中设置页面使用这个自定义字体：

```css
body {
    font-family: '自定义字体名称';
}
```

**验证反爬效果**

老师在页面中写了一段数据：
- 年龄：18
- 出生年月日：2006-01-15

由于在HTML中写的实际是映射后的字符（根据之前配置的映射规则），使用自定义字体后，页面上显示的是正常数字，但查看HTML源码时看到的是中文或其他符号。这就是字体反爬的完整实现。

### 五、破解视角：解析字体文件获取映射关系

老师从爬虫工程师的角度，讲解如何破解字体反爬。

**破解的核心思路**

既然知道了渲染原理，破解的关键就是找到UNICODE字符码与字形的对应关系。具体步骤：

1. 获取网站使用的字体文件（通过抓包工具如Fiddler/Charles/BurpSuite）
2. 解析字体文件，分析其中的映射关系
3. 将HTML中的字符转换为UNICODE字符码
4. 根据映射关系找到真实值并替换

**使用fonttools库解析字体文件**

老师演示使用Python的fonttools库来解析字体文件。首先安装库：

```bash
pip install fonttools
```

然后编写解析代码：

```python
from fontTools.ttLib import TTFont

# 打开字体文件
font = TTFont('字体文件.woff')

# 转换为XML方便查看
font.saveXML('output.xml')
```

**分析XML文件中的映射关系**

老师打开生成的XML文件，重点讲解两个关键部分：

第一部分是cmap标签，记录了字符码与字形名称的对应关系：
- code属性表示字符的16进制UNICODE码
- name属性表示对应的字形名称

第二部分是字形数据，包含了矢量图的路径信息。

老师通过实例说明：
- 假如XML中显示code="0x6211"对应name="五"
- 说明在HTML中写“䔱”（0x6211对应的字符）时，使用该字体后会显示“五”

**手动映射方法**

对于简单的字体反爬，可以直接通过XML建立映射字典：

```python
# 建立映射关系
mapping = {
    '䔱': '五',
    '䜣': '一',
    # ... 其他映射
}

# 应用映射
def decrypt(text):
    return ''.join(mapping.get(char, char) for char in text)
```

## 本P结论

本P的核心结论有以下几点：

1. **字体反爬的本质**：通过自定义字体文件，修改UNICODE字符码与字形的对应关系，让HTML源码中的字符与页面显示内容不一致。渲染时浏览器会根据映射关系查找字形并绘制到页面。

2. **实现字体反爬的步骤**：创建或修改字体文件 → 配置字符码与字形的映射 → 在网页中引入并应用该字体。

3. **破解字体反爬的关键**：获取网站的字体文件 → 解析字体文件提取映射关系 → 将HTML中的字符根据映射关系转换为真实内容。

4. **fonttools是核心工具**：该库可以将字体文件转换为XML格式，通过分析XML中的cmap和字形数据，就能建立完整的映射关系字典。
