# P87 练习题 - 【网络爬虫】字体反爬破解
## 结构识别

### 1. 识别字体反爬特征

以下是一段网页的 HTML 片段和对应的显示效果：

```html
<p>本期价格： &#100844; &#100845; &#100846; &#100847; 元</p>
```

浏览器显示：`本期价格： 3999 元`

已知：字符 `&#100844;` 的十进制 UNICODE 码为 100844，对应十六进制 `0x186BC`。

请回答：
1. 该页面最可能使用了哪种反爬技术？
2. `&#100844;` 这种实体引用写法说明了什么？
3. 如果直接用 `requests` 库抓取此页面，`p` 标签的内容会是什么形式？

## 机制理解

### 2. 三层映射的对应关系

字体反爬涉及三层映射。请将左侧的技术要素与右侧的作用用字母对应起来：

**左侧（技术要素）**

| 序号 | 要素 |
|------|------|
| A | ord("6") |
| B | cmap 表中 0x186BC → glyph0 |
| C | glyph0 → 字形矢量数据 |
| D | HTML 文本中的字符 `䉸` |

**右侧（作用）**

| 序号 | 作用 |
|------|------|
| ① | 从浏览器渲染结果反推 HTML 中实际写的是什么 |
| ② | 告知字体文件"这个码要查 glyph0 这个字形" |
| ③ | 获取字符在内存中的 UNICODE 码值 |
| ④ | 字体文件实际绘制图形的数据来源 |

对应关系：`A → ③`、`B → ②`、`C → ④`、`D → ①`（仅示例，正确答案请自行填写）

## 参考答案要点

> **第1题**  
> 1. 字体反爬（自定义字体映射）  
> 2. 说明 HTML 源码中确实写的是乱码字符，而不是 CSS 隐藏  
> 3. 会是形如 `&#100844;` 的实体引用，或直接的乱码字符

> **第2题**  
> A→③、B→②、C→④、D→①  
> 解析：第一层映射（HTML字符→UNICODE码），第二层映射（ cmap表 ），第三层映射（字形数据）

> **第3题**  
> 1. 不是，只是字形名称  
> 2. `glyph1` 没有语义信息，不知道它对应数字几  
> 3. 需要从字体文件中获取每个 glyph 的实际字形数据（或者从已知的固定映射中推断）

> **第4题**（代码示例）  
> ```python
> from fonttools.ttLib import TTFont
> 
> font = TTFont("price.woff")
> cmap_table = font.getBestCmap()
> result = {int(k): v.split(".")[0] for k, v in cmap_table.items()}
> print(result)
> ```

> **第5题**  
> 1. 可以一次性解析获取完整映射表后缓存，不必每次请求都解析  
> 2. 每次使用随机码（打乱映射表）、定期轮换字体文件、使用 woff2 压缩使解析更复杂

> **第6题**  
> 1. 浏览器渲染时通过字体文件查找字形并绘制，映射对用户透明  
> 2. 复制的是源码中的乱码字符，而非渲染后的字形  
> 3. 针对自动化爬虫/无头浏览器，不针对有视觉需求的人类用户

> **第7题**  
> 1. 理论上可行，但工作量极大，正则需匹配 cmap 标签中的十六进制码和 name 属性  
> 2. 在线工具如 [FontForge](https://fontforge.org/) 或 [Transfonter](https://transfonter.org/)、其他库如 `fonteditorlib`、`opentype.js`
