# P87 知识框架 - 【网络爬虫】字体反爬破解
## 本P位置

网络爬虫进阶课·字体反爬专题。前承HTML解析与抓包基础，后接动态字体与验证码识别。是“攻守对抗”思维的训练：先学攻击（开发者如何设置反爬），再学防御（爬虫如何破解反爬）。

## 核心问题

网页显示正常数字/文字，但HTML源码中是一堆乱码。核心矛盾是：如何建立乱码字符与真实内容的映射关系？

## 知识骨架

字体反爬的本质是**字符码到字形的映射劫持**。

浏览器渲染时，字符并不直接显示，而是通过字体文件查找对应的矢量图字形。字体反爬的做法是：修改字体文件中的映射关系，让乱码字符指向正常字形。页面渲染时浏览器按映射查字形，显示正常；但HTML源码本身并未改变，看到的是乱码。

关键路径：HTML字符 → UNICODE码（16进制） → 字体文件 cmap 表 → 字形名称 → 字形矢量图 → 页面显示

破解的逆路径：获取字体文件 → 解析 cmap 表 → 建立字符码到字形的映射 → 用映射还原HTML中的乱码

## 关系与依赖

字体反爬涉及三层映射：
- 第一层：HTML中的字符文本 → 该字符的默认 UNICODE 码（ord()函数可查）
- 第二层：字体文件中定义的 UNICODE 码 → 字形名称（cmap 表记录）
- 第三层：字形名称 → 字形矢量图数据

破解时需要打通第一层和第二层。第三层通常不需要，因为字形本身已经画好了数字。

依赖关系：理解字体反爬需要掌握HTML/CSS（引入自定义字体）、字符编码基础（UNICODE）、Python文件操作与字符串处理。

## 输入 / 输出 / 前置 / 支撑

输入：包含乱码的 HTML 页面 + 页面引用的 woff/ttf 字体文件（通过抓包获取）

输出：根据映射关系还原后的真实文本

前置知识：HTML/CSS 基础、Python 基本语法、ord() 和 hex() 函数

支撑工具链：
- FontCreator：制作/编辑自定义字体文件（开发视角用）
- fonttools（Python库）：解析 woff/ttf 字体文件，提取 cmap 映射（破解视角用）
- Fiddler/BurpSuite/Charles：抓包获取网站使用的字体文件

## 易错点

cmap 表中 name 属性不是最终显示的字符，而是字形名称。字形名称需要再查字形数据或通过其他方式确认对应的真实值。

UNICODE 码需去掉 "0x" 前缀再在字体文件中查找，否则匹配失败。

woff 格式是压缩格式，fonttools 读取时会自动解压，不影响解析。

## 外部资源与对象

| 对象 | 角色 | 作用 |
|------|------|------|
| FontCreator | 字体文件编辑器 | 创建自定义字体，演示反爬实现过程 |
| fonttools | Python库（ttLib模块） | 解析 woff/ttf，转换为 XML 或直接读取 cmap 表 |
| Fiddler / BurpSuite / Charles | 抓包工具 | 拦截 HTTP 请求，获取网站返回的字体文件 |
| woff / ttf / woff2 | 字体文件格式 | 反爬载体，存储自定义映射关系 |
| XML（中间格式） | 可读格式 | fonttools.saveXML() 将二进制字体转为可分析的 XML |

核心依赖：fonttools 是破解的必用库，cmap 表是映射关系的核心数据源。
