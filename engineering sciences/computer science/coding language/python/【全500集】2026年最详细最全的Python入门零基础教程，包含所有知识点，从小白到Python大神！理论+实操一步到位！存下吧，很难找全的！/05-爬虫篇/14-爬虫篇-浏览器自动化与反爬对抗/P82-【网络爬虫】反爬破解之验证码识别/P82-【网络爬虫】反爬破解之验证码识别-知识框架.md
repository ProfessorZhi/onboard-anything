# P82 知识框架 - 【网络爬虫】反爬破解之验证码识别
## 本P位置

爬虫反爬实战·验证码处理模块。承接 P81 滑动轨迹验证码 JS 逆向破解，本 P 转向图文验证码（普通字符型）的识别与破解，为后续滑块、旋转等高阶验证码的处理奠定基础。

## 核心问题

如何用 Python 完成图文验证码的自动识别并成功登录目标网站。

## 知识骨架

**验证码识别三步法**
1. 请求获取验证码图片
2. OCR 识别图片文字
3. 携带结果发送登录请求

**核心依赖链**：Session（保持 Cookie）→ 获取验证码图片 → 识别 → 登录请求

## 关系与依赖

验证码识别三步之间存在严格依赖：第一步是第二步的前置，第二步是第三步的输入。Session 对象贯穿全流程，负责维持 Cookie 身份一致。免费 OCR 与打码平台是识别层的两种可选方案，互为替代。打码平台输出结果后，若验证码需要物理操作（如滑块、旋转），则依赖 Selenium 执行实际行为。

## 输入 / 输出 / 前置 / 支撑

**输入**：目标网站登录页面 URL、验证码图片地址

**输出**：验证码图片中识别出的字符（字符串）

**前置**：requests 库、ddddOCR 库 或 打码平台账号与 Token

**支撑**：requests.Session() 维持 Cookie 身份；打码平台 API 文档；Selenium（滑块类验证码场景）

## 易错点

- 使用不同 Session 对象获取验证码和发送登录请求，导致 Cookie 不一致，验证码无法关联用户身份，登录失败。
- 免费 OCR 识别精准度有限，存在误识别，需考虑容错或重试机制。
- 调用打码平台时验证码类型参数选错，返回结果无效。
- 误以为打码平台能直接完成滑块验证码自动化，实际只返回滑动距离，需配合 Selenium 模拟物理滑动。

## 外部资源与对象

**ddddOCR**：Python 免费 OCR 库，自带预训练模型，安装即用，精准度有限，适用于低准确率要求的场景。

**打码平台**：收费验证码识别服务，提供 HTTP API，接收图片返回识别结果，支持普通字符、滑块距离、旋转角度、点选坐标等多种类型，按识别次数计费（约 0.002 元/次）。

**requests.Session()**：requests 库的会话对象，在多次请求间自动保持 Cookie，是验证码与登录请求关联的关键。

**打码平台开发文档**：API 接口规范文档，包含 Token 获取方式、接口地址、验证码类型参数说明及 Python 调用示例。

**Selenium**：浏览器自动化工具，在打码平台返回滑块距离或旋转角度后，用于模拟真实鼠标滑动或旋转操作，完成物理交互。
