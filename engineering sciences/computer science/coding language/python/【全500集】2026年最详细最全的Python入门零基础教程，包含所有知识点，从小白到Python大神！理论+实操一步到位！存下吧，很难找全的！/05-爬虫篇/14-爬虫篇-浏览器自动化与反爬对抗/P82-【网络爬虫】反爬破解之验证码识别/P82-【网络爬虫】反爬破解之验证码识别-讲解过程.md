# P82 讲解过程 - 【网络爬虫】反爬破解之验证码识别
## 本P定位

本 P 承接上一 P 的滑动轨迹验证码 JS 逆向破解，转向讲解另一类常见验证码——图文验证码（普通字符型）的识别与破解。目标是完成使用图文验证码登录网站的全流程操作，核心是演示两步关键技术：如何用 Python 发送请求拿到验证码图片，以及如何将图片中的验证码文字识别出来。本 P 属于爬虫反爬实战中的验证码处理环节，为后续高阶验证码（如滑块、旋转）的处理做铺垫。

## 讲解过程

### 1. 场景引入与目标确认

老师先回顾上一 P 的内容——已经破解了滑动轨迹校验类验证码，然后自然过渡到本 P 的主题：常规的图文验证码怎么破解。以一个打码平台网站作为演示对象，指出该网站登录页面存在图文验证码。讲解目标是破解该网站的登录流程。

### 2. 抓包分析登录请求

老师演示了完整的抓包过程：随意填写账号、密码和验证码，提交登录请求，用浏览器开发者工具抓取网络请求。分析发现该登录请求不是 AJAX 请求，而是普通的表单提交，请求参数只有三个：账号、密码、验证码，没有加密，结构简单。老师明确指出这一步"毫无难度"，真正的难点在于验证码识别。

### 3. 验证码识别三步法

老师系统地提出了验证码识别的标准流程，将其拆分为三个步骤：

- **第一步**：发送请求获取验证码图片
- **第二步**：识别验证码图片中的文字内容
- **第三步**：携带识别结果发送登录请求

老师强调第三步（发请求登录）本身代码很简单，难的是第一步（拿验证码）和第二步（识别验证码）。

### 4. 第一步：获取验证码图片

老师演示如何获取验证码图片。在浏览器中右击验证码图片，选择"在新标签中打开图片"，获得验证码图片地址。关键操作点：

- 必须使用 `requests.Session()` 对象来发请求，原因在于验证码和 Cookie 是绑定用户身份的。
- 同一个 Session 发请求时，Cookie 会自动在多个请求之间保持一致，确保后续登录请求能关联到之前拿到的验证码。
- 用 Session 对象的 `.get()` 方法请求验证码图片地址，将响应内容以二进制写模式（`wb`）保存为 `.jpg` 文件。
- 老师特别提醒：拿验证码时必须带上 Cookie，后续登录请求也必须带同一个 Cookie，否则验证码无法与用户身份关联，会导致登录失败。

### 5. 第二步：OCR 识别验证码

老师介绍了第一种识别方式——使用 Python 的 OCR 第三方库（根据上下文推断为 ddddOCR）。操作流程极简，只需两行核心代码：

- 第一行：创建 OCR 识别对象。
- 第二行：调用识别方法，将图片的二进制内容传入，返回识别结果。

老师演示了多轮识别测试，部分结果准确（如识别出 "RRWW"、"M7RF"、"PKMR"），部分存在误差（如 "RRRW" 实际应为 "RRWW"，"PKTR" 实际应为 "PKMR"）。老师承认这个自带模型的 OCR 库精准度不是很高，存在识别错误的情况，但优点是免费、不花钱。

### 6. 登录验证演示

老师将三步串联起来演示完整的登录流程：先用 Session 拿验证码并识别出结果，再用同一个 Session 发登录请求。登录请求携带账号、密码、识别出的验证码以及保持的 Cookie。由于老师没有该网站的注册账号，登录请求被重定向到了注册页面，但验证码识别本身是成功的，证明整个流程跑通。

### 7. 引入打码平台作为高精准方案

老师指出如果对识别精准度要求高，免费 OCR 库不够用，需要使用打码平台。随即演示了打码平台的开发文档和接口使用方法：

- 注册账号后获得 Token，调用平台提供的接口地址。
- 将验证码图片通过接口发送给打码平台，平台返回识别结果。
- 根据验证码类型选择对应的识别类型参数（如普通中英文四位数的类型值、纯数字类型、纯字母类型等）。

老师列举了打码平台支持的多种验证码类型：普通字符型、滑块类、旋转类、鼠标轨迹类、文字点选类、数字计算题、汉字计算题等，并说明了各自的计费标准（如普通中英文验证码 1 元可识别 500 次）。

### 8. 滑块类验证码的补充说明

老师特别说明滑块类验证码的处理方式：打码平台只能识别出滑动的距离（目标缺口位置），但无法直接模拟真实的滑动轨迹。要实现自动化滑动，需要借助 Selenium（CD 流）打开页面，然后用 Selenium 控制鼠标按照平台返回的距离进行真实滑动操作。同理，旋转类验证码也需要 Selenium 操作鼠标进行物理旋转。

## 本P结论

本 P 完成了图文验证码识别与登录破解的完整流程，得出以下结论：

1. **验证码识别三步走**：获取验证码图片 → 识别图片文字 → 携带结果发送登录请求，第三步本身无难度，关键在第一步和第二步。

2. **Session 对象是关键**：获取验证码和后续登录请求必须使用同一个 Session 对象，保证 Cookie 身份一致，否则验证码无法生效。

3. **免费 OCR 库可用但不准**：ddddOCR 等自带模型的 OCR 库可以免费识别验证码，但精准度有限，适用于对准确率要求不高的场景。

4. **高精准需求用打码平台**：打码平台提供专业的验证码识别服务，收费便宜（通常几分钱一次），支持多种验证码类型，是商业级爬虫的首选方案。

5. **高级验证码需要 Selenium 配合**：滑块类、旋转类等需要物理操作的验证码，打码平台只能给出识别结果（如滑动距离），实际自动化操作仍需 Selenium 模拟鼠标行为。
