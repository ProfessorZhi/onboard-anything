# P82 资源清单 - 【网络爬虫】反爬破解之验证码识别
---

## 1. requests 库

- **名称**：requests（Python HTTP 请求库）
- **类型**：Python 标准库 / HTTP 客户端工具
- **在本 P 中的作用**：作为网络请求的基础工具，用于获取验证码图片和发送登录请求。本 P 中的 Session 对象即基于 requests 库实现。
- **来源**：基于上下文推断（Python 网络爬虫基础依赖，贯穿整个爬虫学习路径）
- **建议方向**：深入理解 requests 的 GET/POST 请求机制，掌握 response 对象的属性与方法（如 status_code、headers、content）

---

## 2. requests.Session()

- **名称**：requests.Session()（会话对象）
- **类型**：Python requests 库的 API / 对象
- **在本 P 中的作用**：核心依赖，用于在同一会话中自动维持 Cookie。获取验证码和发送登录请求必须使用同一个 Session，确保验证码能关联到用户身份，避免 Cookie 不一致导致的验证失败。
- **来源**：字幕直接提到
- **建议方向**：对比有 Session 和无 Session 的请求差异，理解 Cookie 在无状态 HTTP 协议中的作用；练习 Session 与 Cookie 的手动管理

---

## 3. ddddOCR（推断名称）

- **名称**：ddddOCR（基于上下文推断）
- **类型**：Python 第三方库 / OCR 识别工具
- **在本 P 中的作用**：免费开源的验证码识别库，自带预训练模型，安装即用。将验证码图片的二进制内容传入，返回识别的字符结果。本 P 演示了其基础用法（约两行代码），但也指出其识别精准度有限，存在误识别情况。
- **来源**：基于上下文推断（字幕提到"Python 的 OCR 第三方库"并展示极简调用方式，符合 ddddOCR 的特征）
- **建议方向**：pip install 后测试不同风格验证码的识别效果；对比本地部署与在线 API 的准确率差异；探索参数调优或模型替换的可能性

---

## 4. 打码平台

- **名称**：打码平台（本 P 演示的具体平台 + 通用的第三方识别服务平台）
- **类型**：第三方服务平台 / 商业 API 服务
- **在本 P 中的作用**：提供高精准度的验证码识别能力，作为免费 OCR 库的替代方案。支持多种验证码类型（普通字符、滑块距离、旋转角度、点选坐标、文字点选、数字计算、汉字计算等），按识别次数计费。本 P 演示了其注册流程、Token 获取和接口调用方式。
- **来源**：字幕直接提到
- **建议方向**：注册打码平台账号并体验免费额度；对比不同打码平台的识别准确率与价格；测试多种验证码类型的识别效果（字符、滑块、旋转、点选等）

---

## 5. 打码平台开发文档

- **名称**：打码平台开发文档（API 文档）
- **类型**：技术文档 / API 规范
- **在本 P 中的作用**：提供接口地址、Token 获取方式、验证码类型参数说明、Python 调用示例等技术规范。是接入打码平台的核心参考依据，决定了接口调用的正确性。
- **来源**：字幕直接提到（"演示了打码平台的开发文档"）
- **建议方向**：仔细阅读文档中的接口说明和参数列表；对照文档编写自己的封装函数；关注平台的错误码说明和请求频率限制

---

## 6. Selenium

- **名称**：Selenium（浏览器自动化框架）
- **类型**：自动化工具 / 测试框架
- **在本 P 中的作用**：用于配合打码平台处理需要物理操作的验证码。本 P 明确指出：打码平台只能识别出滑块滑动距离或旋转角度，但无法模拟真实鼠标轨迹。使用 Selenium 可以控制鼠标按平台返回的结果进行物理滑动或旋转，模拟真人操作。
- **来源**：字幕直接提到
- **建议方向**：回顾 Selenium 的基本操作（定位元素、模拟点击/输入、鼠标拖拽）；结合打码平台返回的滑块距离，实现完整的滑块验证码自动化；探索 ActionChains 的高级鼠标操作

---

## 7. 浏览器开发者工具（DevTools）

- **名称**：浏览器开发者工具（Chrome DevTools / Firefox Developer Tools）
- **类型**：浏览器内置工具 / 调试工具
- **在本 P 中的作用**：用于抓包分析登录请求。演示了如何通过开发者工具的 Network 面板抓取登录请求，分析请求参数（账号、密码、验证码）、请求方法（表单提交，非 AJAX）和 Cookie 情况，从而确定请求构造方式。
- **来源**：字幕直接提到
- **建议方向**：熟练使用 Network 面板的筛选、搜索、查看请求详情功能；尝试分析不同网站的登录请求结构；对比不同验证码类型的网络请求特征

---

## 8. Python 文件操作

- **名称**：Python 文件操作（内置 open / write）
- **类型**：Python 内置功能 / 工具
- **在本 P 中的作用**：将 requests 获取的验证码图片二进制内容（response.content）以二进制写模式（`wb`）保存为本地 `.jpg` 文件，供 OCR 识别使用。
- **来源**：基于上下文推断（"将响应内容以二进制写模式保存为 .jpg 文件"）
- **建议方向**：练习二进制文件的读写操作；理解 `wb` 写模式与 `w` 写模式的区别；尝试用 Pillow（PIL）库对下载的图片进行预处理以提升 OCR 识别率

---

## 9. 目标网站（本 P 演示站点）

- **名称**：打码平台网站（本 P 演示用的目标登录网站）
- **类型**：网站 / 演示对象
- **在本 P 中的作用**：作为破解目标，其登录页面存在图文验证码。老师通过该网站演示了完整的验证码识别与登录流程：抓包分析请求 → 获取验证码图片 → OCR 识别 → 携带结果登录。
- **来源**：字幕直接提到
- **建议方向**：寻找其他带有图文验证码的网站进行练习（如小型论坛、电商登录页）；尝试用不同方式（免费 OCR / 打码平台）完成同一目标的验证码识别
