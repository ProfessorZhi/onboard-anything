# P82 练习题 - 【网络爬虫】反爬破解之验证码识别
## 结构识别

**题目 1**：请将以下三个步骤按验证码自动识别的正确流程排序：

- A. 携带识别结果发送登录请求
- B. OCR 识别图片中的文字
- C. 请求获取验证码图片

**答案**：C → B → A

---

## 机制理解

**题目 2**：为什么在验证码识别流程中，必须使用同一个 Session 对象来完成「获取验证码图片」和「发送登录请求」这两个操作？

**答案要点**：
- Session 对象会在多次请求间自动保持 Cookie
- 验证码图片是与当前用户身份绑定的，服务器通过 Cookie 识别是哪位用户
- 如果用不同 Session，Cookie 不一致，服务器无法关联验证码与用户，登录必然失败

---

**题目 3**：ddddOCR（免费）和打码平台（收费）这两种验证码识别方案，分别适合什么场景？

**答案要点**：

| 方案 | 适用场景 |
|------|----------|
| ddddOCR | 低准确率要求、预算有限、识别量不大的场景 |
| 打码平台 | 高准确率要求、需要滑块距离/旋转角度/点选坐标等结构化结果、识别量大的场景 |

---

## 最小实践

**题目 4**：假设已通过 `requests.Session()` 获取到验证码图片并保存为 `captcha.png`，请写出使用 ddddOCR 识别该图片的最小代码片段。

**参考答案**：

```python
import ddddocr

ocr = ddddocr.DdddOcr()
with open('captcha.png', 'rb') as f:
    result = ocr.classification(f.read())
print(result)
```

---

**题目 5**：使用 `requests.Session()` 分别请求验证码图片并发送登录请求，写出核心代码结构（登录 URL、验证码图片 URL、请求头等用占位符表示）。

**参考答案**：

```python
session = requests.Session()

# 第一步：获取验证码图片
captcha_response = session.get('验证码图片URL')
with open('captcha.png', 'wb') as f:
    f.write(captcha_response.content)

# 第二步：OCR识别（省略）
# captcha_code = ...

# 第三步：发送登录请求（使用同一session保持Cookie一致）
login_data = {
    'username': '你的账号',
    'password': '你的密码',
    'captcha': captcha_code
}
login_response = session.post('登录URL', data=login_data)
```

## 扩展思考

**题目 6**：打码平台返回滑块距离（如 `dx=150`）后，为什么不能直接把距离发给服务器完成滑块验证，还需要 Selenium？

**答案要点**：
- 打码平台只返回「识别结果」（这里是滑动距离数字）
- 服务器验证的是「物理行为」——真实的鼠标滑动轨迹
- 滑块验证会检测滑动速度、加速度、轨迹曲线等行为特征
- Selenium 可以模拟真实浏览器的鼠标移动操作，生成合理的滑动轨迹，使验证通过

---

**题目 7**：如果使用 ddddOCR 识别验证码失败（识别错误），自动化流程应当如何容错？

**答案要点**：
- 识别错误是正常现象（免费OCR精准度有限）
- 常见容错策略：
  - 循环重试：识别失败则重新获取图片再识别，设置最大重试次数
  - 结果校验：对识别结果做格式校验（如纯数字、长度范围），不符合则重试
  - 备选方案：ddddOCR 连续失败 N 次后，切换到打码平台

## 最小替代练习

**题目 8**（无 ddddOCR 场景）：如果当前环境无法安装 ddddOCR，请简述如何改用打码平台 API 完成验证码识别，需要准备哪些信息？

**答案要点**：
需要准备：
- 打码平台账号与密码（或 API Token）
- 打码平台的 API 接口地址
- 验证码图片文件（二进制数据）
- 验证码类型参数（如 `type=1010` 表示普通字符型）

调用流程：调用平台接口 → 发送图片数据 → 接收识别结果字符串
