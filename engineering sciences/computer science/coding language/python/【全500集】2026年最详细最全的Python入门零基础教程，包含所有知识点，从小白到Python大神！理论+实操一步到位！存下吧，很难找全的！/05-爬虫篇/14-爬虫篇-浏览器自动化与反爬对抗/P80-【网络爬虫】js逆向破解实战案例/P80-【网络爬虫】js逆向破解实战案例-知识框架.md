# P80 知识框架 - 【网络爬虫】js逆向破解实战案例
## 本P位置
网络爬虫进阶 → JS逆向领域 → 实战案例驱动
前序：HTTP抓包基础、Python requests使用
后续：复杂混淆对抗、反调试绕过

## 核心问题
如何用Python模拟出浏览器中JS生成的加密参数（密码、签名等）？

## 知识骨架

### 1. JS逆向两条路径
| 路径 | 适用场景 | 核心操作 |
|-----|---------|---------|
| Python原生还原 | JS未混淆、算法已知（MD5/AES等） | 提取盐值/密钥，Python重写 |
| Python调用JS | JS混淆、算法复杂 | 提取函数+依赖，pyexecjs执行 |

### 2. 断点调试定位三步法
```
调用栈追踪 → 明文回溯 → 加密函数定位
```
- 调用栈：定位请求发起位置
- 回溯：找到明文密码出现位置
- 进入函数：观察明文→密文转换过程

### 3. pyexecjs执行流程
```
读取JS文件 → execjs.compile() → ctx.call(函数名, 参数)
```

### 4. 签名类参数还原要点
- 动态参数（sign/ts）每次变化
- 函数往往依赖外部变量（window.xxx、Cookie等）
- 提取时必须一并获取所有依赖

## 关系与依赖

```
Python爬虫端
    ├── requests（发请求）
    ├── hashlib（原生加密还原）
    └── pyexecjs（调用JS引擎）
            └── Node.js（运行时，必须安装配置）
                    └── npm（管理全局模块路径）

浏览器端
    └── DevTools
            ├── Network（抓包）
            └── Sources（断点调试）
                    └── 调用栈（追踪执行链路）

目标网站
    └── 加密函数
            ├── 算法逻辑
            └── 外部依赖（盐值/密钥/全局变量）
```

## 输入 / 输出 / 前置 / 支撑

| 类型 | 内容 |
|-----|------|
| **输入** | 明文密码、翻译内容、加密参数位置 |
| **输出** | 加密后的密文、正确的sign签名 |
| **前置知识** | Python requests使用、HTTP抓包基础、MD5哈希原理 |
| **支撑工具** | DevTools（断点调试）、Node.js（JS运行时）、pyexecjs（Python-JS桥接） |

## 易错点

1. **盐值遗漏**：Python还原MD5加盐时忘记拼接盐值，导致结果不一致
2. **Node.js未配置**：pyexecjs调用失败，提示找不到execjs引擎
3. **依赖缺失**：提取的JS函数引用了外部变量，本地执行时未定义
4. **时间戳精度**：ts要求13位毫秒级，直接用time.time()秒级会失败
5. **混淆代码硬啃**：花费大量时间分析混淆逻辑，应优先考虑直接提取函数
