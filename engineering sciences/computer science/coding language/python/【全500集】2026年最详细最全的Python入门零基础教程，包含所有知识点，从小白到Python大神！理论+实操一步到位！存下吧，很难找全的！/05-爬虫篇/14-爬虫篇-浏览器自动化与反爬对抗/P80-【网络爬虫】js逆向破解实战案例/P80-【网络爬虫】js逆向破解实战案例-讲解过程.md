# P80 讲解过程 - 【网络爬虫】js逆向破解实战案例
## 本P定位

本P是JS逆向破解的实战案例教学，属于网络爬虫进阶内容。课程从简单的未混淆JS加密分析入手，逐步引导至经过混淆的JS代码处理，最后演示通过`pyexecjs`库在Python中执行JS代码来完成逆向破解。核心目标是让学习者掌握两种JS逆向思路：**纯Python还原**与**Python调用JS代码**。

---

## 讲解过程

### 一、案例引入：登录接口的密码加密分析

**1.1 问题背景交代**

老师指出，在爬虫场景中，遇到登录接口时，密码通常以密文形式传输，直接明文传输无法完成模拟登录。需要分析密文是如何生成的，找出加密算法并通过Python进行还原。

**1.2 抓包演示**

老师打开目标网站，进入登录页面，在输入框填写账号密码后，点击登录按钮。通过开发者工具拦截请求，观察请求参数。确认密码字段（password）已经是加密后的密文形式，无法直接明文传输。

**1.3 断点调试定位加密位置**

老师点击开发者工具中的"调用栈"（Call Stack）面板，查看当前请求的调用来源。这个面板可以展示请求是从哪个JS文件、哪一行代码发起的，以及完整的调用链路。

点击进入调用栈，找到请求发送的具体位置。在请求参数附近打一个断点，重新触发登录操作。程序在断点处暂停，此时观察密码参数，发现已经是加密后的密文。

**1.4 回溯明文位置**

老师查看调用栈中的其他函数入口，发现在更早的位置可以找到明文密码。向上追溯调用栈，找到一个函数，该函数的参数中包含用户输入的原始密码（明文）。在明文位置重新打一个断点，再次触发登录操作。此时观察到密码参数还是用户输入的原始值。

**1.5 定位加密函数**

将断点移动到处理密码的函数调用处，观察函数如何处理明文密码。在函数调用前后分别设置断点，对比输入与输出的差异。

单步进入加密函数内部查看实现逻辑。老师发现该函数内部执行了以下操作：密码字符串拼接上一段固定盐值字符串（sait），然后对拼接后的结果做MD5哈希，得到最终的密文。

**1.6 提取盐值**

老师将断点停在函数执行后的位置，查看加密后的结果值。同时记录下拼接时使用的盐值字符串，这个盐值是固定的，需要在Python还原时使用。

### 二、Python实现MD5加盐加密还原

**2.1 环境准备**

老师准备编写Python代码实现相同的加密逻辑。代码开头导入`requests`模块用于后续发送HTTP请求。

**2.2 定义加密函数**

定义一个名为`md5_encrypt`的函数，接收明文密码作为参数。函数内部实现加密逻辑：首先将明文密码与提取到的盐值字符串拼接，然后使用Python内置的`hashlib`模块创建MD5对象，调用`update`方法传入拼接后的字符串（需要编码为UTF-8），最后通过`hexdigest`获取十六进制表示的密文并返回。

**2.3 测试验证**

老师以"123456"作为测试密码，调用加密函数，将返回值与浏览器断点处观察到的密文进行对比。确认两者完全一致，说明Python实现的加密逻辑与JS加密逻辑等价，逆向还原成功。

**2.4 结论引出**

老师总结：对于未混淆的JS代码，可以直接分析出加密算法，然后用Python还原实现。但对于经过混淆的复杂JS代码，这种方式效率很低，需要换一种思路。

### 三、混淆JS代码的处理方案：Python调用JS

**3.1 方案说明**

老师提出：当目标网站的JS代码经过混淆处理，逻辑复杂难以逐行分析时，没必要硬啃。可以直接找到完整的加密函数代码，复制到Python中执行。通过Python调用JS引擎来运行这段代码，传入明文参数即可获得密文结果。

**3.2 工具安装：pyexecjs**

老师演示安装Python库`pyexecjs`，该库的作用是在Python环境中执行JavaScript代码。安装命令为`pip install pyexecjs`。

**3.3 环境依赖：Node.js**

老师强调使用`pyexecjs`库需要先安装Node.js运行环境。该库底层调用的是`execjs`引擎，而`execjs`依赖Node.js。安装步骤：访问Node.js官网下载长期支持版（LTS），双击安装包按引导完成安装，安装完成后配置环境变量。验证安装成功的方法是在CMD命令行中输入`node -v`，若能显示版本号则说明Node.js安装正常。

老师还提到`execjs`模块路径的配置：通过CMD运行`npm root -g`可以查看全局模块存放路径，该路径需要加入环境变量，以便Python能够找到Node.js的模块。

**3.4 Python调用JS代码演示**

老师创建一个JS文件`demo.js`，编写两个简单的JS函数示例：

```javascript
function add(a, b) {
    return a + b;
}
function work() {
    return 100;
}
```

然后在Python中演示调用流程：

```python
import execjs

# 第一步：编译JS代码
with open('demo.js', 'r', encoding='utf-8') as f:
    js_code = f.read()
ctx = execjs.compile(js_code)

# 第二步：调用函数
result = ctx.call('add', 100, 200)
print(result)  # 输出300
```

老师解释：首先通过`execjs.compile()`方法编译JS代码得到上下文对象，然后调用上下文的`call`方法，传入函数名和参数，即可执行JS函数并获取返回值。

### 四、翻译网站JS逆向实战

**4.1 目标分析**

老师切换到翻译网站场景。打开开发者工具的Network面板，在网页翻译框输入"python3"，点击翻译按钮，拦截到翻译接口的请求。观察请求详情，发现这是一个POST请求，表单数据包含多个参数。

**4.2 参数分类**

老师逐个分析请求参数的性质：

- `from`：源语言，如"en"表示英语
- `to`：目标语言，如"zh"表示中文
- `query`：翻译内容，这里是"python3"
- `token`：固定值，每次请求不变
- `sign`：签名参数，每次请求都变化，是核心分析对象
- `ts`：时间戳参数，每次请求都变化

其中`sign`和`ts`是动态变化的参数，是请求成功的关键。

**4.3 构造请求框架**

老师开始编写Python代码构造请求。复制请求URL和请求头（Headers），包括User-Agent、Cookie、Referer等字段，确保请求头信息完整。

**4.4 动态参数的初步处理**

时间戳`ts`参数：观察发现是一个13位的时间戳（精确到毫秒），在Python中可以通过`int(time.time() * 1000)`生成。

签名`sign`参数：目前先留空或复制抓包获取的值，尝试发送请求验证问题所在。将构造好的请求发送出去，果然返回错误（error 998），说明签名参数缺失或错误导致请求被拒。

**4.5 定位签名生成代码**

老师回到开发者工具，准备定位`sign`参数的生成位置。复制参数名`sign`，在"Sources"面板的搜索功能中粘贴并搜索。由于JS可能是动态加载的，部分JS文件可能搜不到结果，需要逐个文件搜索。

**4.6 断点调试追踪**

搜索到结果后，在相关代码位置打断点。刷新页面再次触发翻译操作，程序在断点处暂停。观察调用栈，定位到签名构造的函数入口。进入该函数内部，单步执行，观察变量值的变化。

老师发现在函数执行前，变量是原始翻译内容"python3"；执行后，生成了签名值。通过鼠标悬停或控制台查看变量的具体值。

**4.7 提取签名生成函数**

找到生成`sign`参数的具体函数后，老师尝试直接提取该函数。右键点击函数名，选择"Reveal in Sources panel"或直接双击，跳转到函数定义的位置。复制整个函数代码，粘贴到本地JS文件或控制台进行测试。

**4.8 环境依赖补充**

复制过来的函数可能依赖外部变量（如`window`对象的某个属性、其他全局变量等）。老师通过在断点处查看变量，发现函数内部引用了一个名为`r`的变量，该变量来自`window`对象的某个属性（如`window.xxx`）。这个值可能是服务器端生成的固定值，需要记录下来。

调试过程中还发现函数引用了`n`、`m`等外部变量或函数，需要逐个在断点处查找并记录。老师的处理方式是：将这些依赖的外部变量也复制到本地JS文件中定义，确保函数能够独立执行。

**4.9 JS代码本地调试**

老师演示两种调试方式：

**方式一：HTML文件调试**

创建一个HTML文件，在`<body>`中添加`<script>`标签，将复制的JS代码粘贴进去。在代码末尾调用签名生成函数并打印结果，然后用浏览器打开HTML文件，在控制台查看输出。

**方式二：pyexecjs调试**

直接将JS代码放入Python中，通过`execjs.compile()`编译，然后调用函数获取结果，与浏览器中的值对比验证是否正确。

## 本P结论

**1. JS逆向两种路径的选择**

- **JS代码未混淆**：优先通过断点调试分析出加密算法，用Python原生实现还原（如MD5加盐、Base64等标准算法）。这种方式性能好、无外部依赖。
- **JS代码混淆**：直接提取完整加密函数代码，在Python中通过`pyexecjs`调用执行。无需理解复杂逻辑，只需确保函数依赖完整即可。

**2. 断点调试是核心技能**

无论是简单还是复杂的JS加密分析，开发者工具的断点调试能力都是关键。需要熟练掌握：调用栈追踪（定位调用链）、在关键位置设置断点、观察变量值变化、单步进入函数内部、提取函数源码。

**3. 签名类参数需要完整还原**

对于`sign`这类动态签名参数，往往依赖多个外部变量（全局变量、Cookie值、窗口属性等）。提取函数时必须一并获取这些依赖，否则函数无法独立运行。

**4. 环境准备不可忽视**

使用`pyexecjs`前需确保Node.js已正确安装并配置环境变量。验证方式是在命令行执行`node -v`查看版本号。
