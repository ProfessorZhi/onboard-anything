# P84 知识框架 - 【网络爬虫】多线程并发爬虫案例
## 本P位置

Python网络爬虫系列 · 入门到进阶的衔接点
- 前置：单线程爬虫基础（数据请求、JSON解析、文件存储）
- 本P核心：从串行到并发的范式转变
- 后置：代理IP池、异步协程（asyncio）

## 核心问题

**如何在I/O密集型爬虫任务中复用等待时间？**

串行模式下，线程在等待网络响应时处于阻塞状态，CPU和磁盘资源浪费。解决方案是将爬虫拆解为三个阶段，让各阶段并行执行，实现I/O等待时间的复用。

## 知识骨架

```
┌─────────────────────────────────────────────────────────────┐
│                      多线程并发爬虫模型                       │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  [请求线程] ──→ 队列A ──→ [解析线程] ──→ 队列B ──→ [存储线程] │
│       ↓                  ↓                     ↓            │
│   网络I/O等待          CPU计算            磁盘I/O写入       │
│   (最耗时)             (最快)              (较慢)           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**三层抽象**：
1. **生产者-消费者模式**：三阶段解耦，线程间无直接调用
2. **Queue队列**：线程安全的数据管道
3. **threading模块**：创建与管理工作线程

## 关系与依赖

```
requests ──→ 发起HTTP请求 ──→ 获取响应数据
    │
    └─→ response.json() ──→ 触发 json 解析
                                │
                                └─→ 字典结构 ──→ 提取字段
                                                │
os ──→ 目录操作 ──→ 创建歌手文件夹/歌曲子文件夹
    │
    └─→ 文件写入 ──→ 保存 .txt/.jpg/.mp3

Queue ──→ 线程间通信 ──→ 连接三个工作线程

threading ──→ 创建线程 ──→ 启动并发执行
```

## 输入 / 输出 / 前置 / 支撑

| 类型 | 内容 |
|------|------|
| **输入** | 歌手名称（字符串） |
| **输出** | 歌手文件夹 → 歌曲子文件夹 → {歌曲名.mp3, 歌曲名.jpg, 歌曲名.txt} |
| **前置依赖** | 单线程爬虫流程（requests + json + os） |
| **技术支撑** | threading.Thread（线程创建）、queue.Queue（队列通信）、requests（网络I/O） |

## 易错点

1. **请求头缺失X-Requested-With**：服务端返回HTML而非JSON，导致解析失败
2. **URL复制错误**：误用页面地址而非AJAX接口地址
3. **并发被封禁**：请求频率骤增触发反爬，未使用代理IP导致IP被封
4. **目录未判空**：重复创建已存在文件夹时抛出异常
5. **响应编码问题**：二进制文件（图片/音频）必须用`wb`模式写入
