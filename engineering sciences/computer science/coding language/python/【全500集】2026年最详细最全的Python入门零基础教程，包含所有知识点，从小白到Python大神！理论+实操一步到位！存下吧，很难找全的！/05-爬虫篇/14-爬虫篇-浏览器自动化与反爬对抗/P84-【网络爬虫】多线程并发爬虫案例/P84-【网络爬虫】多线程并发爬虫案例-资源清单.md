# P84 资源清单 - 【网络爬虫】多线程并发爬虫案例
---

## Python 库与模块

### requests

- **类型**：Python 第三方库
- **作用**：发送 HTTP/HTTPS POST 请求获取歌曲列表，下载封面图片和 MP3 文件。本 P 中承担全部网络 I/O 操作。
- **来源**：字幕直接提到（用于演示请求发送、响应获取、文件下载）
- **建议方向**：尝试使用 `requests.Session()` 保持会话；设置 `timeout` 参数防止请求无限等待；使用 `stream=True` 参数分块下载大文件

### json

- **类型**：Python 标准库
- **作用**：解析接口返回的 JSON 响应，从嵌套字典中提取歌曲标题、作者、音频 URL、图片 URL、歌词等字段。
- **来源**：字幕直接提到（解析 `response.json()` 的返回值）
- **建议方向**：结合 `json.dumps()` 和 `json.loads()` 处理复杂嵌套结构；处理响应数据中可能存在的空值或异常字段

### os

- **类型**：Python 标准库
- **作用**：创建歌手文件夹和歌曲子文件夹两级目录结构；使用 `os.path.exists()` 判断目录是否存在，避免重复创建报错。
- **来源**：字幕直接提到（文件组织结构实现）
- **建议方向**：使用 `os.path.join()` 确保跨平台路径兼容；使用 `shutil` 模块实现更复杂的文件操作；添加异常处理保护文件写入操作

### queue

- **类型**：Python 标准库（`queue.Queue`）
- **作用**：作为线程间数据传递的管道。队列 A 连接请求线程与解析线程，队列 B 连接解析线程与存储线程，实现生产者-消费者模式。
- **来源**：字幕直接提到（多线程并发模型讲解）
- **建议方向**：了解队列的线程安全特性；设置队列最大长度限制生产者速度；尝试使用 `task_done()` 和 `join()` 控制线程同步

### threading

- **类型**：Python 标准库（`threading.Thread`）
- **作用**：创建三个独立工作线程——请求线程、解析线程、存储线程，实现爬虫三个阶段的并行执行。
- **来源**：字幕直接提到（多线程并发原理讲解）
- **建议方向**：学习线程的 `start()` 和 `join()` 方法控制执行顺序；使用线程锁（`threading.Lock`）保护共享资源；考虑线程池（`concurrent.futures.ThreadPoolExecutor`）管理线程数量

## 工具与平台

### 浏览器开发者工具（Browser DevTools）

- **类型**：调试工具
- **作用**：通过 Network 面板捕获和分析 AJAX 请求。定位歌曲列表接口的 POST 地址、请求头参数（User-Agent、X-Requested-With）、请求体参数（input、filter、page、types），以及响应 JSON 结构。
- **来源**：字幕直接提到（演示如何分析网页请求）
- **建议方向**：使用 Elements 面板分析页面 DOM 结构；尝试过滤 XHR 类型的请求快速定位 API；使用 Console 面板执行 JavaScript 片段验证接口逻辑

### 目标网站（音乐 API 接口）

- **类型**：目标网站 / 第三方 API
- **作用**：提供歌手歌曲搜索接口（POST 请求），返回 JSON 格式的歌曲列表，包含每首歌的标题、作者、MP3 下载地址、封面图片地址、歌词文本。是本 P 爬虫的数据来源。
- **来源**：字幕直接提到（作为演示案例的目标站点）
- **建议方向**：尝试搜索不同歌手验证接口稳定性；分析是否存在签名验证或参数加密机制；观察接口是否支持批量查询或存在频率限制

### 代理 IP 池

- **类型**：外部服务
- **作用**：配合并发爬虫使用，每次请求轮换不同出口 IP，降低同一 IP 的请求频率，避免被目标网站反爬机制识别和封禁。
- **来源**：字幕直接提到（提示异步爬虫的注意事项，预告后续课程展开）
- **建议方向**：寻找可用的代理 IP 服务提供商；学习代理 IP 的验证与筛选方法；实现代理 IP 的自动轮换与失效重试机制

## 外部文档与协议

### HTTP 协议

- **类型**：网络协议 / 基础知识
- **作用**：本 P 中涉及的 POST 请求、请求头设置（User-Agent、X-Requested-With）、响应状态码、Content-Type 等均属于 HTTP 协议范畴。
- **来源**：基于上下文推断（贯穿整个网络爬虫内容）
- **建议方向**：系统学习 HTTP 请求方法、请求头、响应状态码的含义；了解 HTTPS 与 HTTP 的区别；学习 Cookie 和 Session 的工作机制

### JSON 数据格式

- **类型**：数据格式 / 基础知识
- **作用**：目标网站接口返回的数据格式。本 P 展示了如何解析 JSON 响应、提取嵌套字段、处理可能存在的空值。
- **来源**：字幕直接提到（响应数据分析）
- **建议方向**：学习 JSON Schema 验证数据结构；了解 JSON 与 Python 字典的互相转换；处理复杂嵌套结构和数组数据
