# P76 资源清单 - 【网络爬虫】视频抓取
---

## 一、核心依赖库

### moviepy

- **类型**：Python 第三方视频处理库
- **在本P中的作用**：将 B 站分离下载的 .m4s 视频流和音频流文件合成为完整的 .mp4 视频
- **来源**：字幕直接提到（未明确说出库名，基于上下文和常见工具链推断）
- **建议方向**：学习 moviepy 的其他功能，如视频剪辑、添加字幕、调整播放速度、批量处理多个视频文件

### requests

- **类型**：Python HTTP 请求库（内置库）
- **在本P中的作用**：发送 HTTP 请求下载 .m4s 视频流和音频流文件，是视频抓取的执行工具
- **来源**：基于上下文推断（核心抓取工具，本P实际编写代码使用）
- **建议方向**：深入学习 requests 的 Session 机制、异常处理、文件流下载进度显示、大文件分块下载

## 二、平台与网站

### B站（哔哩哔哩）

- **类型**：目标视频平台
- **在本P中的作用**：作为音视频分离存储架构的典型案例，展示如何抓取两个 .m4s 流文件并合成
- **来源**：字幕直接提到
- **建议方向**：尝试抓取不同分区的视频（纪录片、教程、番剧），对比不同视频的 URL 特征和请求头差异

### 抖音网页版

- **类型**：目标视频平台
- **在本P中的作用**：作为视频地址无规律平台的典型案例，展示单个视频直链下载方法
- **来源**：字幕直接提到
- **建议方向**：对比抖音网页版与移动端的视频地址获取方式差异，了解 API 接口抓取的可行性

## 三、开发工具

### 浏览器开发者工具（Network 面板）

- **类型**：浏览器内置调试工具
- **在本P中的作用**：抓取 B 站 Network 面板中的 .m4s 流地址，是发现视频/音频直链的必经步骤
- **来源**：字幕直接提到
- **建议方向**：熟练使用过滤器功能（如 domain、type）、分析 XHR 与 Fetch 请求的区别、导出请求为 cURL 命令

- **类型**：Python 包索引镜像站点

- **来源**：字幕直接提到

## 五、后续学习依赖

### Selenium

- **类型**：浏览器自动化测试框架
- **在本P中的作用**：老师预告可用于模拟抖音用户向上滑动刷视频的行为，实现批量视频地址的获取
- **来源**：字幕直接提到（发音为"水逆流"，为 Selenium 的谐音）
- **建议方向**：学习 Selenium 的元素定位方法、等待机制、模拟滑动操作、处理反爬机制（验证码、IP封锁）

### Appium

- **类型**：移动端自动化测试框架
- **在本P中的作用**：老师提及可用于配合手机模拟器实现抖音视频批量抓取
- **来源**：基于上下文推断（老师提到手机模拟器配合方案）
- **建议方向**：了解 Appium 环境配置、Android 模拟器设置、触摸操作模拟

## 六、文件格式

### .m4s 格式

- **类型**：流媒体分片文件格式
- **在本P中的作用**：B站视频和音频分别以 .m4s 格式存储，需要分别下载后合成
- **来源**：字幕直接提到
- **建议方向**：了解 m4s 与 mp4 的区别、MPEG-DASH 协议基础、为何流媒体采用分片存储

## 七、关键配置文件

### HTTP Headers

- **类型**：请求头配置
- **在本P中的作用**：B站请求必须携带 Referer（防盗链）和 Cookie（身份校验），抖音同样需要构造完整的请求头
- **来源**：字幕直接提到
- **建议方向**：深入理解各 Header 字段的作用（如 User-Agent、Accept、Accept-Encoding）、使用浏览器插件快速复制 Headers、练习自动维护 Cookie
