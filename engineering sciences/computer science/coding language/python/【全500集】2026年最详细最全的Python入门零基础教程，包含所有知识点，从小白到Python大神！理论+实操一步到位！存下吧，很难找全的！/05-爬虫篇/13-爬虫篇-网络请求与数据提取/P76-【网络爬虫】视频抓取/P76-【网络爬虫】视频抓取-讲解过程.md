# P76 讲解过程 - 【网络爬虫】视频抓取
## 本P定位

本P为网络爬虫章节中关于**视频抓取**的专项讲解，属于进阶内容。前置知识包括 requests 库基础用法、浏览器开发者工具（Network 面板抓包）、HTTP 请求头（Headers）的构造。

本P解决的核心问题：

- 如何针对**视频与音频分离**的视频网站（B站）进行抓取和合成
- 如何针对**单个视频地址无规律**的平台（抖音）进行简单下载
- 视频下载后的**音频+视频合并**问题

## 讲解过程

### 一、B站视频抓取的原理铺垫（13.5-149）

老师首先指出 B 站视频的特殊性：**视频和音频是分开存储的**，这意味着抓取时需要分别找到两个 URL（视频流地址和音频流地址），下载后还需要手动合成才能得到完整的视频文件。

老师随即打开浏览器，访问 B 站某视频页面（歌曲《龙族》相关），通过开发者工具（F12 → Network 面板）进行抓包演示。老师在面板中观察到两个以 `.m4s` 结尾的 URL 地址，这就是视频流和音频流的分片文件。老师指出，通过 URL 地址本身很难直接区分哪个是视频、哪个是音频，因为它们的参数结构几乎相同。

关于抓取思路，老师用一句话概括：**request 发请求 → 下载到本地**。核心难点不在爬虫逻辑，而在于请求头（Headers）的构造。老师提到该网站**必须携带 Referer 等多个请求头字段**，否则服务器会拒绝响应。

### 二、下载代码的编写与调试（255-256）

老师开始编写代码，首先演示在代码编辑器中粘贴视频地址。老师强调了两个关键点：

1. **必须携带 Cookie**：否则请求无法通过身份校验
2. **请求地址即 URL**：需要把从 Network 面板复制的两个 URL 分别作为两个请求的目标

（此处字幕显示大量歌曲歌词，属于视频播放过程中老师等待下载完成的背景音）

### 三、视频合成工具的安装与使用（537-699）

下载完成后，老师转而讲解如何将分离的视频和音频合成为完整视频。老师介绍 Python 中有一个专门的**视频剪辑库**（字幕中未明确说出库名，但根据上下文和常见工具推断为 moviepy），可以用来实现视频合并、叠加等操作。

老师演示安装该库时遭遇网络超时问题（多次安装失败）。随后老师尝试切换为**国内镜像源**（字幕提到清华源），使用 `-i` 参数指定镜像地址后，安装顺利完成。老师解释之所以选择国内源，是因为该库依赖较多、国内网络环境直接访问 PyPI 较慢。

安装完成后，老师演示两行核心合成代码：

- **导入模块**：导入 moviepy 的相关模块
- **调用合成方法**：将两个文件路径（视频 .mp4 和音频 .m4s）作为参数传入，指定输出文件名，即可自动叠加生成新视频

老师随后运行合成代码，验证结果：打开资源管理器确认文件生成成功，确认播放正常（歌曲《这江山我起笔》）。老师总结 B 站视频抓取的要点在于**视频音频分离抓取 + 事后合成**。

### 四、抖音视频抓取的思路对比（749-837）

老师接下来转向抖音视频抓取，首先说明抖音与 B 站的核心区别：**抖音视频地址没有规律**，无法像传统页面数据那样通过构造 URL 模板 + 分页遍历的方式批量获取。

老师解释道，批量采集抖音视频的关键在于**批量获取视频地址本身**——但这需要模拟用户"向上滑动刷视频"的行为，而 requests 作为静态请求库**无法实现页面滚动**，因此传统爬虫思路在此失效。老师预告该问题需要用到后续章节将要学习的**Selenium 自动化测试框架**（字幕中称为"水逆流"，为 Selenium 的谐音/误记）来实现模拟滑动，或者借助手机模拟器配合 Appium 等方案。

老师明确表示：用 requests 只能处理**单个视频地址的下载**，无法实现批量滚动抓取。

### 五、抖音单个视频下载代码演示（883-1042）

虽然无法批量抓取，老师仍然演示了单个抖音视频的下载方法。

老师打开抖音网页版，找到一个 1 分 15 秒的视频，复制其视频地址。随后编写代码：

```python
import requests

url = "刚才复制的抖音视频地址"
headers = {...}  # 包含 User-Agent、Referer、Cookie 等字段
response = requests.get(url, headers=headers)
with open("output.mp4", "wb") as f:
    f.write(response.content)
```

老师运行代码后成功下载视频，确认播放内容为数码发布会相关片段（华为小折叠屏手机发布会视频）。

老师最后总结：**单个视频下载逻辑本身很简单**，真正的难点在于如何批量获取视频地址，而这超出了本 P 使用 requests 能解决的范围。

## 本P结论

1. **B 站视频需要分离抓取**：视频和音频存储为两个独立的流文件（`.m4s` 格式），抓取后需使用 moviepy 等工具进行合成。

2. **B 站请求必须构造完整的 Headers**：至少包含 `Referer` 和 `Cookie`，否则请求会被拦截。

3. **moviepy 合成操作极简**：仅需两行代码（导入 + 调用合成方法），传入两个文件路径和输出文件名即可完成叠加。

4. **抖音视频地址无规律**：无法通过 URL 模板构造实现批量抓取，单靠 requests 无法模拟页面滚动获取更多视频。

5. **抖音批量抓取的正确思路**：需借助 Selenium 或 Appium 等自动化工具模拟用户滑动行为，先批量采集视频地址，再循环下载。老师预告后续章节会涉及。

6. **单个抖音视频下载与 B 站逻辑相同**：找到视频直链地址 → 构造 Headers → requests.get → 写入文件。
