# P76 知识框架 - 【网络爬虫】视频抓取
## 本P位置

网络爬虫章节 → 进阶专题 → 视频资源抓取
前置：requests 基础、开发者工具 Network 面板抓包、Headers 构造
后续：Selenium（抖音批量抓取依赖）

## 核心问题

1. 如何抓取音视频分离架构的视频（以 B 站为代表）
2. 如何处理视频地址无规律的平台（以抖音为代表）
3. 如何将分离的视频流和音频流合成为完整文件

## 知识骨架

```
视频抓取
├── 平台类型A：音视频分离存储
│   ├── 特征：两个 .m4s 流文件，URL 参数相似难以区分
│   ├── 抓取策略：分别下载视频流 + 音频流
│   ├── 请求要求：Referer + Cookie 必须携带
│   └── 后处理：使用 moviepy 合成
│
├── 平台类型B：地址无规律
│   ├── 特征：无法通过 URL 模板构造批量地址
│   ├── 单个下载：与普通文件下载相同
│   └── 批量获取：需 Selenium 模拟用户行为
│
└── 视频合成
    ├── 工具：moviepy
    ├── 安装：需使用国内镜像源（清华源）解决超时
    └── 操作：两行代码完成叠加
```

## 关系与依赖

```
开发者工具(Network面板) ──→ 发现 .m4s 流地址
         │
         ▼
requests ──→ 下载 .m4s 文件（必须携带 Referer + Cookie）
         │
         ▼
moviepy ──→ 视频 + 音频 合成输出 .mp4
         │
         │ (后续章节)
         ▼
Selenium ──→ 模拟页面滚动，批量获取抖音视频地址
```

## 输入 / 输出 / 前置 / 支撑

| 类型 | 内容 |
|---|---|
| **输入** | 视频流地址（从 Network 面板复制）、音频流地址（从 Network 面板复制）、Headers 构造参数 |
| **输出** | 完整视频文件（.mp4） |
| **前置** | requests.get 用法、开发者工具 Network 面板操作、Headers 构造（Referer/Cookie） |
| **支撑** | moviepy（合成）、PyPI 清华镜像源（安装）、Selenium（后续批量抓取） |

## 易错点

1. **B 站请求漏 Header**：未携带 Referer 或 Cookie → 403 拒绝
2. **B 站只下单个流**：只下载视频或只下载音频 → 播放无声音或无画面
3. **moviepy 安装超时**：直连 PyPI 慢 → 需切换国内镜像源
4. **抖音误用批量爬虫思维**：抖音视频地址无规律，requests 无法模拟滑动 → 需 Selenium
5. **混淆两个平台的抓取逻辑**：B 站难点在前端抓包 + 合成，抖音难点在批量地址获取

## 外部资源与对象

| 资源/对象 | 角色 | 关联 |
|---|---|---|
| **moviepy** | 视频合成工具 | 接收两个 .m4s 文件路径，输出 .mp4 |
| **PyPI 清华镜像源** | 包安装加速 | 解决 moviepy 安装网络超时 |
| **Selenium** | 后续依赖 | 模拟抖音滑动行为，批量获取视频地址 |
| **抖音网页版** | 目标平台 | 演示单个视频直链获取 |
| **B 站** | 目标平台 | 代表音视频分离架构的抓取场景 |
| **开发者工具 Network 面板** | 抓包工具 | 发现 .m4s 流地址的必经步骤 |
