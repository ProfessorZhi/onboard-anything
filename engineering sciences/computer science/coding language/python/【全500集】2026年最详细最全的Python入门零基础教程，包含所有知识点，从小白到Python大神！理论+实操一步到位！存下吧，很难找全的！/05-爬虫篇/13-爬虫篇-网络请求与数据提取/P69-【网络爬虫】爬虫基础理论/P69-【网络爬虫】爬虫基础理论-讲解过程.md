# P69 讲解过程 - 【网络爬虫】爬虫基础理论
## 本P定位

本P为爬虫入门的第一课，主要目标是让零基础学员建立对网络爬虫的初步认知。课程从“大数据时代数据值钱”这一现象切入，依次讲解爬虫的定义、学习爬虫的目的、爬虫的主要应用场景、爬虫的四大分类、爬虫的基本工作流程，最后介绍Robots协议及其法律边界。本P不涉及代码编写，属于纯理论铺垫。

曾老师进一步列举了类似的数据分析平台，如考古家、查妈妈等，这些平台年费高达900多元，说明这些数据具有很高的商业价值，从而引出核心问题：这些数据从哪里来？

### 四、数据来源的几种方式

**时间段：1:16 - 1:52**

1. 自己采集数据
2. 从第三方公司购买（如天眼查可以购买企业数据）
3. 公司自有的数据
4. 通过人工问卷调查收集数据
5. **通过爬虫抓取数据**

### 五、什么是爬虫

**时间段：1:53 - 4:09**

曾老师正式给出爬虫的定义：

- 爬虫也叫**网络蜘蛛**或**网络机器人**
- 将互联网比作一张**蜘蛛网**，每个网格交汇点是一个**节点**（对应一台电脑）
- 网络蜘蛛在蜘蛛网上从各个节点到处爬行，采集网络数据
- 本质上是一个**自动抓取互联网上数据的程序**

曾老师用蜘蛛网和蜘蛛爬行的直观比喻，帮助学员理解爬虫的工作形象。

### 六、为什么要学爬虫

**时间段：4:09 - 5:43**

曾老师阐述学习爬虫的理由和好处：

**场景举例——电商竞品分析：**
- 假设你做了一款产品，想了解同行同类产品的表现
- 传统方式：一个网站一个网站手动去点去看，效率低
- 爬虫方式：写一个爬虫程序，自动从各平台抓取竞品的**销量、价格、用户评论**等数据
- 抓取后进行数据分析，可以为产品**定价**和**产品优化**提供依据
- 从市场现有产品的用户反馈中分析出**用户高频提及的缺点**，指导产品改进

**核心结论：** 爬虫的价值在于让互联网数据为我所用，产生实际价值。

### 七、爬虫数据有什么用

**时间段：5:43 - 11:19**

曾老师进一步讲解爬虫数据的具体应用场景：

#### 1. 变现与副业
- 采集的数据可以**打包出售**
- 可以在网上**接单**，帮不会爬虫的人抓取数据
- 成为**爬虫工程师**：大公司和小公司都有需求，一线城市薪资1万多起步，高级2-3万，甚至有月薪4万的岗位
- 搞副业、找工作均可

#### 2. 搜索引擎（百度为例）
- 举例回到199几年李彦宏创办百度时的场景
- 搜索的本质就是一个爬虫
- 搜索引擎后台有一个**搜索引擎爬虫**，会事先在网上到处爬取网页，把数据保存到自己的数据库
- 用户在搜索框输入内容时，百度从自己数据库中检索并返回结果
- 当年会这个技术的人很少，所以百度能成功

#### 3. 今日头条的内容来源
- 今日头条本身不产生内容
- 内容是通过爬虫从各大新闻平台抓取来的
- 本质是一个内容分发平台

#### 4. 数据分析
- 根据自己所在行业，抓取对应数据
- 从数据中分析挖掘价值
- 为自己的产品做优化、辅助定价和升级决策

#### 5. 舆情分析
- 互联网上做实时的数据监控
- 监控到数据后给出相应预警提示
- 这一块也需要用爬虫抓取数据

#### 6. 数据聚合
- 将不同来源、不同网站的数据聚合到一起
- 给用户提供便捷的数据获取方式

#### 7. 应用开发
- 一些自己没有数据的软件（如股票分析软件、天气预报软件）
- 需要从其他平台抓取数据后嵌入自己的应用中

### 八、爬虫的四大分类

**时间段：11:22 - 19:00**

曾老师将爬虫分为四大类，并逐一讲解特征和区别。

#### 第一类：通用爬虫（General Purpose Web Crawler）

**定义：** 能够在网上抓取各种信息的爬虫，不针对特定网站，发现一个网站就抓一个网站，只要有新网页就会抓取。

**典型用途：** 构成搜索引擎（如Google、搜狗、Bing等）

**工作原理：** 先通过爬虫把数据采集下来放到数据库，用户搜索时从数据库检索结果，本质是互联网数据的整合商。

**四大特点：**

1. **广泛性**：抓取绝大多数互联网网站，覆盖性强
2. **自动化**：能够自动发现网页并抓取，遇到有链接的页面会把链接也放入抓取队列
3. **智能化**：具备分页抓取等智能功能
4. **持续性**：持续抓取互联网网站，实时更新，有固定周期
5. **去重处理**：对重复数据进行过滤处理
6. **性能优化**：会根据不同网站和网络环境优化性能，提高抓取效率

#### 第二类：聚焦爬虫（Focused Crawler）【重点】

**定义：** 针对某一类特定网站或特定类型数据进行抓取的程序，只抓特定数据，不是什么都抓。

**典型场景：** 抓取某个抖音直播间的弹幕信息、进出场人数、成交信息等针对性数据。

**核心特征：**

1. **定制性强**：可根据各种需求定制开发，抓取你想要的数据
2. **精准度高**：定位明确，知道要采什么数据、采哪些网站，不会满互联网到处爬
3. **效率高**：只抓目标数据，不做无用功
4. **隐蔽性较强**：由于每次访问都会被目标网站记录，如果持续高频访问容易被识别为爬虫并封禁，因此需要采取一定的**反爬虫策略**
5. **数据处理**：抓取到数据后还需对数据进行处理和清洗，提取有价值的数据
6. **定时更新**：通常需要定时运行更新数据

#### 第三类：增量式爬虫（Incremental Crawler）

**定义：** 在原有数据基础上，只抓取更新的数据。

**工作逻辑：**
- 第一次抓取了什么数据记录下来
- 第二次再抓时，发现已有数据就不重复抓取，只抓新增的部分
- 每次只抓最新、最新的数据

**典型场景：** 新闻网站、论坛等需要实时更新最新内容的场景。

#### 第四类：生成网络爬虫（Dynamic Web Crawler）

**定义：** 专门用来访问互联网生成层面的网页（即动态网页）。

**技术背景：**
- 有些网页直接请求时返回的页面没有数据
- 数据需要通过特定的请求（后续会讲AJAX请求）再次访问才能获取
- 不是一次请求就能拿下来的，需要多次请求配合

**说明：** 曾老师预告这个内容在后续课程中会详细讲解。

### 九、爬虫的工作流程

**时间段：19:00 - 22:35**

曾老师讲解爬虫的四个基本步骤，并配合浏览器演示。

#### 四步流程

| 步骤 | 操作 | 说明 |
|------|------|------|
| 第一步 | 发请求 | 指定要爬取的网站地址 |
| 第二步 | 获取响应 | 获取该网页的返回数据 |
| 第三步 | 提取数据 | 从响应内容中提取需要的目标数据 |
| 第四步 | 保存数据 | 把数据存储起来 |

#### 对比人工操作

人工手动操作的流程是：打开网站 → 手动复制数据 → 保存。爬虫则将这个过程自动化。

#### 批量抓取逻辑

曾老师指出，实际抓取时数据往往分布在多个网页中，需要循环处理：

1. **URL队列管理**：将要抓取的多个网页地址放入Python列表中进行排队
2. **循环处理**：从队列中取出一个URL → 发请求 → 获取响应 → 提取数据 → 保存数据 → 再取下一个URL，循环往复
3. **URL发现**：在解析当前页面时，如果发现新的待抓取网页链接，也要把它加入URL队列，继续循环抓取，直到把所有目标网页抓完

**核心代码逻辑总结：** 发请求 → 获取响应 → 提取数据 → 存储数据

### 十、Robots协议

**时间段：22:35 - 27:39**

#### 什么是Robots协议

- Robots协议也称为**robots.txt**
- 存放在网站的根目录下（域名/robots.txt即可访问）
- 文件内容包含**声明性语句**

#### 协议内容

1. **User-agent**：声明允许哪些爬虫名称可以采集数据
2. **Disallow**：声明不允许被访问的URL路径

#### 演示示例

曾老师打开**百度的robots.txt**文件进行演示：

- 百度spider（百度的爬虫）不允许访问根目录
- 各家搜索引擎的爬虫名称及其禁止访问的路径列表

#### Robots协议的法律效力

曾老师明确指出：

- **robots.txt协议本身没有法律强制力**，仅是道德层面的约束
- 作为一个爬虫工程师，robots协议可以直接忽视，不一定要遵守
- **原因：** 如果严格遵守robots协议，很多数据根本拿不到。例如淘宝在协议中明确禁止百度等爬虫访问，但实际仍有大量数据被采集

#### 数据采集的法律边界【重点】

曾老师特别强调爬虫的**法律合规问题**：

**绝对禁止采集的数据：**
- 用户**隐私数据**（名字、电话、地址、身份证号等）
- 涉及用户隐私的数据**绝对不能采集**，违法
- 曾老师警示：“爬虫写得好，牢饭吃得早”
- 采集用户隐私数据达到几十条、上百条就会**判刑**

**可采集的数据：**
- **公开数据**可以采集，没问题

**职业原则：**
- 无论是接单还是在公司工作，如果别人让你采集涉及用户隐私或商业机密的数据，**这种钱不能赚**
- 前几年有新闻报道：爬虫公司因为采集违法数据，整个公司的人都被抓进去
- 曾老师反复叮嘱：有原则、会技术，踩出去的时候要按规范来

## 本P结论

1. **爬虫的定义**：网络爬虫（网络蜘蛛/网络机器人）是自动抓取互联网上数据的程序，原理类似蜘蛛在蜘蛛网上爬行采集。

2. **学习爬虫的价值**：获取互联网数据用于竞品分析、数据分析、产品优化、舆情监控等，数据本身可以变现，也可以通过接单或成为爬虫工程师获得收入。

3. **爬虫四大分类**：
   - **通用爬虫**：广泛抓取，用于搜索引擎，覆盖性强、自动化、持续更新
   - **聚焦爬虫**：针对特定网站或数据类型定制抓取，精准度高、效率高，但需注意反爬虫策略
   - **增量式爬虫**：只抓取新增/更新的数据，用于新闻、论坛等实时更新场景
   - **生成网络爬虫**：用于抓取动态网页（非一次请求即可获取数据的页面），后续课程会详细讲解

4. **爬虫工作流程**：发请求 → 获取响应 → 提取数据 → 保存数据，循环处理多个URL。

5. **Robots协议**：网站根目录下的robots.txt文件，声明允许/禁止哪些爬虫采集哪些数据。该协议没有法律强制力，爬虫工程师可以忽视。

6. **法律边界**：用户隐私数据（姓名、电话、身份证号等）绝对不能采集，违法且会判刑；公开数据可以采集。
