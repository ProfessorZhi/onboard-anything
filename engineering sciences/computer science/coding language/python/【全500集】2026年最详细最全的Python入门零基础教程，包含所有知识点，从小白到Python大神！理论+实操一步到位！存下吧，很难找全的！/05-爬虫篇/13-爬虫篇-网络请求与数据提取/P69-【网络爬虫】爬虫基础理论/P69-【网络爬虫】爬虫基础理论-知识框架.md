## 知识骨架

**一级知识节点：**
- 爬虫定义
- 爬虫分类
- 爬虫工作流程
- Robots协议
- 法律合规

**二级知识节点：**

爬虫定义 →
- 网络蜘蛛 / 网络机器人
- 自动抓取互联网数据的程序
- 类比：蜘蛛在蛛网上爬行采集

爬虫分类 →
- 通用爬虫：覆盖广、自动发现、持续更新 → 构成搜索引擎
- 聚焦爬虫：定向抓取、精准高效、需反爬策略 → 最常用
- 增量式爬虫：只取新增数据 → 新闻/论坛场景
- 动态网页爬虫：处理AJAX等二次请求 → 后续课程详解

爬虫工作流程 →
- 发请求 → 获取响应 → 提取数据 → 保存数据
- 批量抓取：URL队列 + 循环处理

Robots协议 →
- robots.txt存放于网站根目录
- 声明允许/禁止的爬虫和路径
- 无法律强制力

法律合规 →
- 禁止：用户隐私数据（违法且可判刑）
- 允许：公开数据

**分类间关系：**

- 增量式爬虫是聚焦爬虫的优化变体
- 动态网页爬虫是针对特殊页面结构的聚焦爬虫

**流程内部依赖：**
- 发请求 → 依赖目标URL
- 获取响应 → 依赖网络通信
- 提取数据 → 依赖响应内容解析
- 保存数据 → 依赖存储方案

## 易错点

1. **误以为Robots协议必须遵守**：Robots协议无法律强制力，仅是道德约束；淘宝禁止百度爬虫但数据仍被采集。

2. **混淆爬虫分类的应用场景**：通用爬虫≠什么都能抓，聚焦爬虫才是日常开发主力。

3. **忽视隐私数据的法律风险**：用户姓名/电话/身份证等信息绝对不能采集，"爬虫写得好，牢饭吃得早"并非玩笑。

4. **认为一次请求就能拿到所有数据**：动态网页需多次请求（AJAX），增量式爬虫需记录已抓数据避免重复。

5. **低估反爬虫的影响**：聚焦爬虫高频访问会被封禁，需使用代理IP、设置访问间隔等策略。
