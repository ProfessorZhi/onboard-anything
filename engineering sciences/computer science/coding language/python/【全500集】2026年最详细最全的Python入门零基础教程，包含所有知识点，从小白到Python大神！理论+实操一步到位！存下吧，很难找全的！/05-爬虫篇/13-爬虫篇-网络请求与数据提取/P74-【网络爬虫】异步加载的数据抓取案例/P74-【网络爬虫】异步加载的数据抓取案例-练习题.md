# P74 练习题 - 【网络爬虫】异步加载的数据抓取案例
## 一、结构识别

**题目1**  
以下哪种现象最可能说明网页数据采用异步加载方式？

A. 网页加载速度很慢  
B. 直接请求页面URL，Response的HTML中搜索不到目标数据，但浏览器打开页面正常显示  
C. 网页需要登录才能访问  
D. 网页URL包含“?”参数

答案：B  
解析：前后端分离架构中，初始HTML仅包含骨架结构，业务数据由JavaScript通过AJAX单独请求并渲染到页面上，因此直接在HTML中搜索不到数据是异步加载的典型特征。

---

**题目2**  
关于静态网页抓取与异步接口抓取的区别，以下说法正确的是：

A. 异步抓取比静态抓取复杂，因为需要处理JavaScript渲染  
B. 异步接口返回JSON数据，不需要解析HTML，反而更简单  
C. 静态网页抓取只能使用浏览器，无法使用requests库  
D. 两者本质相同，都需要解析HTML提取数据

答案：B  
解析：异步接口直接返回结构化的JSON数据，可直接通过response.json()转为Python字典，无需XPath或正则解析DOM树。

## 二、机制理解

**题目3**  
某招聘网站接口URL如下：
```
https://careers.tencent.com/tencentcareer/api/post//query?pageIndex=1&pageSize=10&language=zh-cn&area=cn&timestamp=1742365123456
```

请指出该URL中的可变参数和固定参数分别是什么，并说明timestamp参数的作用。

答案：  
- 固定参数：`pageSize=10`、`language=zh-cn`、`area=cn`  
- 可变参数：`pageIndex`（页码）、`timestamp`（时间戳）  
- timestamp作用：每次请求生成新的时间戳，使URL不同，防止服务器返回缓存数据，确保获取最新数据

---

**题目4**  
使用`requests.get()`后，为什么应优先使用`response.json()`而不是`response.text`解析异步接口返回的数据？

答案：  
1. 异步接口返回的是标准JSON格式，`response.json()`可直接将JSON字符串转为Python字典/列表，无需手动解析  
2. `response.text`返回字符串，还需`json.loads()`二次转换  
3. `response.json()`会对响应Content-Type进行校验，确保是application/json，若类型不匹配会抛出明确异常，帮助及时发现问题

## 四、扩展思考

**题目7**  
某新闻网站采用异步加载，点击“加载更多”按钮后，通过开发者工具定位到接口URL。但连续请求几次后接口突然返回空数据，可能的原因是什么？请提出至少两种排查思路。

参考答案：  
1. **时间戳问题**：时间戳参数可能已过期或格式不对，需每次请求动态生成  
2. **频率限制**：触发了网站的反爬机制（IP限流、请求频率限制），需要添加延时或使用代理  
3. **请求头缺失**：缺少必要的Headers（如User-Agent、Referer、Cookie等）  
4. **参数变化**：除时间戳外还有其他隐藏参数（如签名sign、token）需要从页面JS中提取  
5. **会话过期**：需要先完成登录获取Session/Cookie，再携带认证信息请求

排查思路：对比成功请求与失败请求的差异（参数、Headers、Cookie），检查浏览器Network面板中该请求的完整Headers和Payload。

**题目8**  
如果要抓取100页数据，谈谈你会如何设计抓取策略以平衡效率与稳定性？

参考答案：  
1. **分页循环**：使用for循环遍历页码，注意处理最后一页的判断（数据量不固定时）  
2. **延时控制**：每页请求间隔0.5-2秒，避免过快触发反爬  
3. **异常处理**：添加try-except捕获网络错误和响应异常，对失败请求可重试2-3次  
4. **增量记录**：记录已抓取的最大页码或最新数据ID，支持断点续爬  
5. **数据去重**：存入数据库时使用唯一索引（如数据ID）防止重复写入  
6. **并发优化**（可选）：可使用aiohttp或concurrent.futures并发请求，但需更严格控制频率

## 五、最小替代练习

**题目9**（无需真实网站的替代练习）  
使用公开测试API `https://jsonplaceholder.typicode.com/comments` 完成以下任务：

1. 请求该接口，获取前10条评论数据
2. 提取每条评论的`email`和`body`字段
3. 将结果保存到`comments.txt`文件中，每行格式为：`邮箱: xxx | 内容: xxx`

参考代码：

```python
import requests

url = "https://jsonplaceholder.typicode.com/comments"
response = requests.get(url)
data = response.json()

with open("comments.txt", "w", encoding="utf-8") as f:
    for item in data[:10]:
        f.write(f"邮箱: {item['email']} | 内容: {item['body']}\n")

print("前10条评论已保存")
```

**题目10**（进阶替代练习）  
利用`jsonplaceholder`的分页接口 `https://jsonplaceholder.typicode.com/posts`，模拟分页抓取逻辑：

1. 使用URL参数`?_page`和`?_limit`实现分页（示例：`?_page=1&_limit=5`）
2. 抓取前3页数据，每页5条
3. 汇总所有数据后，按userId分组统计每人发布的文章数量

参考代码：

```python
import requests

all_posts = []

# 分页抓取前3页
for page in range(1, 4):
    url = f"https://jsonplaceholder.typicode.com/posts?_page={page}&_limit=5"
    response = requests.get(url)
    data = response.json()
    all_posts.extend(data)

# 按userId分组统计
from collections import defaultdict
user_count = defaultdict(int)
for post in all_posts:
    user_count[post['userId']] += 1

print("各用户文章数量统计：")
for user_id, count in sorted(user_count.items()):
    print(f"用户 {user_id}: {count} 篇")
```
