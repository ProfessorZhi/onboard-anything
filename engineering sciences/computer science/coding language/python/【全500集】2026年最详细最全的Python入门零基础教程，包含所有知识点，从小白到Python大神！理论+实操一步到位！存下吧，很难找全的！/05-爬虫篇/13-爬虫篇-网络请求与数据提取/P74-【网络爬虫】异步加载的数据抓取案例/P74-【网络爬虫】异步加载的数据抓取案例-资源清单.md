## 三、Python 库与模块

### 5. requests 库

| 字段 | 内容 |
|------|------|
| **名称** | requests |
| **类型** | Python 第三方 HTTP 库 |
| **在本P中的作用** | 发送 GET 请求直接访问 AJAX 接口 URL；接收并解析响应数据；本P中无需添加请求头即可获取腾讯招聘接口数据，说明该接口无强反爬限制 |
| **核心用法（本P涉及）** | `requests.get(url)` / `requests.get(url, params=params)` / `response.json()` / `response.text` |
| **来源** | 字幕直接提到 |
| **建议练习方向** | 对比 `response.text` 与 `response.json()` 的适用场景及异常情况；练习通过 `params` 字典传递 URL 参数，而非手动拼接字符串；尝试在请求中加入 `headers`（User-Agent、Referer、Cookie），应对有基础反爬的接口 |

---

### 6. time 模块

| 字段 | 内容 |
|------|------|
| **名称** | time |
| **类型** | Python 标准库模块 |
| **在本P中的作用** | 动态生成毫秒级时间戳，用于替换腾讯招聘接口 URL 中的 `timeStamp` 参数；老师讲解了 `time.time()` 返回秒级浮点数、乘以 1000 再取整得到毫秒时间戳的完整逻辑 |
| **核心用法（本P涉及）** | `int(time.time() * 1000)` |
| **来源** | 字幕直接提到 |
| **建议练习方向** | 理解 Unix 时间戳的起点（1970-01-01 00:00:00 UTC）；练习用 `datetime` 模块将时间戳与人类可读时间互转；遇到其他网站接口中含有时间戳参数时，能独立判断是秒级还是毫秒级并正确生成 |

---

## 四、核心方法与配置项

### 7. response.json()

| 字段 | 内容 |
|------|------|
| **名称** | `response.json()` |
| **类型** | requests 库响应对象方法 |
| **在本P中的作用** | 将接口返回的 JSON 字符串直接转换为 Python 字典，替代手动调用 `json.loads(response.text)`；老师明确强调这是处理 JSON 接口响应的首选方式 |
| **来源** | 字幕直接提到 |
| **建议练习方向** | 了解该方法在响应 Content-Type 不为 `application/json` 时会抛出 `JSONDecodeError` 的情况，练习用 `try/except` 捕获；对比 `json` 标准库中 `json.loads()` 与该方法的区别 |

---

### 8. 文件写入配置（open 追加模式）

| 字段 | 内容 |
|------|------|
| **名称** | `open(filename, "a", encoding="utf-8")` |
| **类型** | Python 内置文件操作配置 |
| **在本P中的作用** | 将循环抓取到的每页岗位数据以追加模式逐条写入本地 TXT 文件，避免覆盖已写入数据；`encoding="utf-8"` 处理中文字符 |
| **来源** | 字幕直接提到 |
| **建议练习方向** | 将存储格式升级为结构化文件（CSV / JSON Lines）；练习用 `with open(...) as f` 语法确保文件正确关闭；了解在循环外统一开关文件与在循环内每次开关的性能与安全性差异 |

---

### 9. pIndex 页码参数

| 字段 | 内容 |
|------|------|
| **名称** | `pIndex`（腾讯招聘接口翻页参数） |
| **类型** | 接口 URL 查询参数 |
| **在本P中的作用** | 控制请求第几页数据，第 1 页传 `1`，第 2 页传 `2`，以此类推；老师通过对比三次翻页请求 URL 识别出该参数为唯一变化的页码参数 |
| **来源** | 字幕直接提到 |
| **建议练习方向** | 练习识别不同网站接口中功能相同但命名不同的翻页参数（page、pageNum、offset、cursor 等）；掌握通过对比两次请求 URL 快速定位可变参数的方法 |

---

### 10. timeStamp 时间戳参数

| 字段 | 内容 |
|------|------|
| **名称** | `timeStamp`（腾讯招聘接口时间戳参数） |
| **类型** | 接口 URL 查询参数 |
| **在本P中的作用** | 接口要求每次请求携带当前毫秒级时间戳；老师分析该参数每次请求均不同，需用 `int(time.time() * 1000)` 动态生成，不能固定写死 |
| **来源** | 字幕直接提到 |
| **建议练习方向** | 探索时间戳参数在接口反爬中的实际作用（防缓存 vs 签名验证）；练习遇到含签名验证的时间戳参数时如何分析加密逻辑（进阶方向） |
