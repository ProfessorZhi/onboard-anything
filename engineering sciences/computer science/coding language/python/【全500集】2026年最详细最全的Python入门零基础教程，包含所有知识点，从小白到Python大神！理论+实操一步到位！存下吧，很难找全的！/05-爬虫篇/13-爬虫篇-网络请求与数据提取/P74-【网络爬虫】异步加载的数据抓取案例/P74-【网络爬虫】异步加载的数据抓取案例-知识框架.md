# P74 知识框架 - 【网络爬虫】异步加载的数据抓取案例
## 本P位置

网络爬虫章节·动态内容处理模块。前置知识为静态网页抓取（requests + XPath/正则），本P解决的是前后端分离网站的异步数据抓取问题。

## 核心问题

如何从采用前后端分离架构的网站中，抓取通过AJAX异步加载的数据。

## 知识骨架

### 1. AJAX异步加载识别
- 前后端分离模式：HTML不含业务数据，数据由JS单独请求后渲染
- 识别方法：请求页面URL后，在Response中搜索目标内容，若为空则数据异步加载

### 2. 开发者工具抓包定位
- Network面板 → 观察请求列表
- 清空面板 → 触发页面操作（翻页、刷新、加载更多）→ 捕获新增请求
- Filter过滤 → 按关键词缩小范围（GET请求输入get）
- 逐个查看Response → 定位包含目标数据的接口URL

### 3. 接口请求与数据解析
- 直接requests.get(接口URL)，无需解析HTML
- response.json() 将JSON响应直接转为Python字典
- 遍历字典提取目标字段

### 4. URL参数分析
- 对比不同请求的URL参数
- 识别固定参数（pageSize、language等）和可变参数（pIndex页码、timeStamp时间戳等）
- 动态参数需要动态生成或从响应中提取

### 5. 分页抓取循环
- 页码参数循环递增
- 时间戳每次请求重新生成
- 循环拼接URL或使用params参数

### 6. 时间戳生成
- `int(time.time() * 1000)` → 毫秒级时间戳
- `time.time()` 返回秒级浮点数，乘1000转毫秒后取整

### 7. 数据存储
- TXT文件追加写入：`open("file.txt", "a", encoding="utf-8")`
- 可扩展为Excel（pandas）或数据库存储

## 关系与依赖

```
静态抓取（requests + XPath）
      ↓ 遇到数据搜不到
AJAX异步加载识别（开发者工具）
      ↓ 定位到接口URL
直接请求接口（requests.get）
      ↓ 接口返回JSON
JSON解析（response.json()）
      ↓ 提取目标字段
参数分析（定位页码/时间戳）
      ↓
分页循环抓取 → 数据存储
```

依赖工具：浏览器开发者工具（Chrome/Firefox DevTools）、requests库

## 输入 / 输出 / 前置 / 支撑

**输入**：目标网站URL（如careers.tencent.com）

**输出**：结构化数据列表（如岗位名称、工作地点、岗位要求）

**前置**：requests库基本用法、Python字典/列表操作、HTTP协议基础认知

**支撑**：浏览器开发者工具Network面板、JSON数据格式、时间戳概念

## 易错点

1. 在HTML Response中搜不到数据 ≠ 一定异步加载，也可能是JavaScript渲染或数据加密
2. 时间戳参数必须每次请求动态生成，固定值会导致接口拒绝或返回脏数据
3. 误以为异步抓取比静态抓取复杂，实际上更简单——因为直接拿到JSON数据，不需要解析HTML
4. 翻页参数命名不固定（可能是pIndex、page、pageNum等），需实际分析
5. response.json() 要求响应Content-Type为application/json，否则抛出异常
