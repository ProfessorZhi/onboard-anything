# P74 讲解过程 - 【网络爬虫】异步加载的数据抓取案例
## 本P定位

本P是异步加载（AJAX）数据抓取的入门与实战课。老师先从概念入手，通过浏览器开发者工具演示如何发现AJAX请求，再以三个不同网站（东方财富、36Kr、腾讯招聘）作为实操案例，最终完成腾讯招聘数据的多页抓取与本地存储。

## 讲解过程

### 一、AJAX异步数据概念引入

老师首先解释什么是AJAX异步数据：**前后端分离模式下，网页HTML本身不包含数据，数据通过JS异步请求获取后再渲染到页面**。用一句话概括：请求网页URL拿不到数据，因为数据是分开加载的。

### 二、浏览器开发者工具演示（东方财富网站）

老师打开东方财富股票数据网站，现场演示抓包流程：

1. 右键点"检查"打开控制台，切换到Network（网络）面板
2. 刷新页面，观察捕获的第一个请求（整个网页HTML）
3. 在Response响应内容中按Ctrl+F搜索"沪能科技"，**搜索结果为空**
4. 由此证明：该页面的股票数据不在HTML里

**核心结论**：如果网页HTML里搜不到数据，说明数据是AJAX异步加载的，此时不能再直接请求网页地址，而要去抓包找真正的数据接口。

### 三、手动定位AJAX请求的方法

老师讲解如何在众多请求中找到目标数据接口：

1. 清空Network面板，点击页面上的翻页或刷新操作触发数据更新
2. 在Filter过滤框输入关键词（如"get"表示GET请求），缩小范围
3. 从上往下逐个查看请求的Response内容，判断是否包含目标数据
4. 找到目标请求后，复制其URL即为数据接口地址

### 四、简单验证：直接请求AJAX接口

老师演示用requests直接请求刚才抓到的数据接口URL，验证能否拿到数据。结果显示接口返回清晰的JSON结构数据。**核心要点**：无论是否异步加载，对Python爬虫来说没有区别——直接请求带数据的接口即可，不需要解析HTML。

### 五、多网站示例（36Kr网站）

老师切换到36Kr网站再次演示：

1. 打开页面刷新，观察Network请求
2. 尝试在HTML请求的Response中搜索内容，发现数据很少
3. 清空后点击"查看更多"，观察新产生的请求
4. 找到返回JSON数据的接口，复制URL，直接展示数据内容

**强调**：这类接口返回的是纯数据，不再需要写XPath去提取数据，直接用JSON解析即可。

### 六、正式案例：腾讯招聘数据抓取

#### 1. 确定目标

抓取腾讯招聘网站的所有招聘岗位数据，页面总数约280多页。先实现单页抓取，再扩展为分页抓取。

#### 2. 抓包定位接口

- 打开腾讯招聘页面，打开控制台Network面板
- 点击翻页，捕获到数据接口请求
- 复制该URL作为抓取目标

#### 3. 分析URL参数

老师对比三页请求的参数变化，发现：

- **pIndex**：页码参数，第1页=1，第2页=2，第3页=3，随翻页改变
- **timeStamp**：时间戳参数，每次请求都会变化
- **pageSize**：每页数量，固定为20
- **language**：语言参数，固定为"zh-cn"
- 其余参数均为固定值

#### 4. 单页抓取代码编写

老师开始编写代码：

```
# 直接用requests.get请求接口URL
# 先不加请求头测试能否拿到数据
response = requests.get(接口URL)
print(response.text)
```

运行后确认可以拿到数据，说明没有反爬限制，无需添加Cookie等请求头。

#### 5. JSON数据解析

老师讲解响应数据的结构：

- 返回的是一个字典
- 字典的`data`字段里包含岗位信息
- 通过`data`字段再拿到岗位列表

**关键方法**：对于返回JSON格式的响应，直接使用`response.json()`方法，会自动将JSON转换为Python字典，比用`response.text`再手动解析更方便。

#### 6. 遍历提取岗位信息

老师遍历岗位列表，查看每条数据的字段结构：

- 岗位名称（`RecruitPostName`）
- 岗位要求
- 工作地点
- 详细信息地址

由于字段较多，老师选择性打印了岗位名称字段进行验证。

## 本P结论

**AJAX异步数据抓取的核心流程**：

1. **判断**：打开网页后，在HTML的Response中搜索目标数据，如果搜不到，说明数据是异步加载的
2. **定位**：清空Network面板后触发页面操作（翻页、刷新等），从上往下逐个查看请求的Response，找到包含目标数据的接口URL
3. **请求**：直接用`requests.get()`请求该接口URL即可，无需解析HTML
4. **解析**：接口返回JSON格式时，使用`response.json()`直接转为Python字典进行提取
5. **分页**：通过分析参数变化，找出页码参数和时间戳参数，动态替换后循环请求实现批量抓取
