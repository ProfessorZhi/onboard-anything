# P70 知识框架 - 【网络爬虫】http协议
## 本P位置
- **章节**：70-【网络爬虫】http协议
- **定位**：网络爬虫章节的前置理论课
- **作用**：为后续编写爬虫代码奠定协议基础，核心目标是理解"请求里包含什么"，才能在代码中模拟浏览器发请求
- **前序**：无明确前序，属于独立概念模块
- **后续**：直接支撑Python爬虫代码编写（requests库、模拟登录等实操）

## 核心问题
1. HTTP和HTTPS的本质区别是什么
2. 一个请求/响应的完整报文结构是怎样的
3. 如何在代码层面模拟浏览器的请求行为

## 知识骨架

```
HTTP协议
├── 协议基础
│   ├── HTTP：超文本传输协议，发布和接收HTML的方法
│   └── HTTPS：HTTP+SSL，数据加密传输
│       └── 两者请求报文格式一致，爬虫无需纠结协议类型
│
├── URL结构
│   ├── 协议 + 域名 + 路径 + 参数（查询字符串）
│   └── 参数位置决定请求方法（GET在URL，POST在请求体）
│
├── 请求报文（爬虫模拟的核心对象）
│   ├── 请求首行：方法 + 地址 + 版本
│   ├── 请求头（Key-Value）：User-Agent、Cookie、Referer等
│   ├── 空白行
│   └── 请求体：POST方法时存放参数
│
├── 请求方法
│   ├── GET：参数在URL，有长度限制
│   └── POST：参数在请求体，无限制，用于登录
│
├── 响应报文
│   ├── 状态行：版本 + 状态码 + 描述
│   ├── 响应头
│   ├── 空白行
│   └── 响应体：HTML/JSON等实际内容
│
├── 状态码
│   ├── 2xx：成功
│   ├── 3xx：重定向
│   ├── 4xx：客户端错误（403爬虫常见、404资源不存在）
│   └── 5xx：服务器错误
│
└── Cookie机制
    ├── 服务器通过Set-Cookie设置
    ├── 浏览器自动存储并携带
    └── 爬虫必须正确处理Cookie才能访问登录态内容
```

## 关系与依赖

| 层级 | 依赖关系 | 说明 |
|------|----------|------|
| 协议层 | HTTP/HTTPS → 请求报文 | 协议定义报文的格式规范 |
| 寻址层 | URL → 请求首行 | URL提供请求地址，解析后填入首行 |
| 方法层 | GET/POST → 参数位置 | 方法决定参数放在URL还是请求体 |
| 身份层 | 请求头 → 服务器识别 | User-Agent标识浏览器身份，Cookie维持会话 |
| 追踪层 | Referer → 来源校验 | 防盗链依赖此字段 |
| 反馈层 | 状态码 → 错误处理 | 爬虫根据状态码判断请求是否成功 |

**核心链路**：浏览器输入URL → DNS解析域名 → 拼接请求报文 → 发送请求 → 接收响应 → 解析渲染

## 输入 / 输出 / 前置 / 支撑

| 类型 | 内容 |
|------|------|
| **输入** | URL地址、请求方法、请求头参数、请求体数据 |
| **输出** | 响应报文（状态码+响应头+响应体） |
| **前置知识** | 浏览器基本操作、域名/IP基础概念 |
| **支撑工具** | 浏览器开发者工具（F12）→ Network面板、Application面板 |

## 易错点

1. **误以为一次页面访问只发一个请求**：实际上浏览器会先请求HTML，再解析其中引用的CSS/JS/图片/字体并多次请求
2. **混淆请求头与请求体**：User-Agent、Cookie、Referer是请求头字段，不是请求体
3. **登录场景用GET**：账号密码绝不能放URL，必须用POST+请求体
4. **不设置User-Agent**：默认User-Agent是程序标识，会被服务器识别为爬虫
5. **忽略Referer防盗链**：某些资源验证Referer来源，爬虫请求缺少此字段会被拒绝
6. **Cookie未正确携带**：访问登录态内容时必须携带有效Cookie，否则返回401/重定向
