# P70 资源清单 - 【网络爬虫】http协议
---

## 一、开发工具

### 浏览器开发者工具（F12）
- **类型**：调试工具
- **在本P中的作用**：核心演示工具，用于查看网页请求/响应的原始报文结构。老师全程通过它讲解 Network 面板和 Application 面板，是理解 HTTP 协议的可视化入口
- **来源**：字幕直接提到
- **建议方向**：掌握 F12 快捷键调出、面板切换、元素审查等基础操作，建立对浏览器内部工作机制的形象认知

### Network 面板
- **类型**：开发者工具子功能
- **在本P中的作用**：展示页面加载过程中发出的所有网络请求，包括请求方法、URL、状态码、耗时等信息。用于查看请求报文（Headers）和响应报文（Response）
- **来源**：字幕直接提到
- **建议方向**：刷新页面后观察请求列表，点击单个请求查看 Details，了解各标签页（Headers、Payload、Preview、Response）的含义

### Raw Headers / Raw Payload
- **类型**：开发者工具子功能
- **在本P中的作用**：以原始文本格式展示请求头、响应头、请求体的完整内容，帮助学习者直观看到 HTTP 报文的四部分结构（首行→请求头→空白行→请求体）
- **来源**：字幕直接提到
- **建议方向**：找一个真实网站，切换到 Raw 格式，对照知识框架验证报文的四部分划分是否与理论一致

### Application 面板 → Cookies
- **类型**：开发者工具子功能
- **在本P中的作用**：展示浏览器本地存储的所有 Cookie 数据，用于理解 Cookie 的存储机制、查看 Cookie 的 Key-Value 结构及有效期
- **来源**：字幕直接提到
- **建议方向**：登录一个网站后查看 Cookies，记录 Cookie 中包含的字段，理解哪些字段与身份认证相关

## 二、协议与标准

### HTTP
- **类型**：应用层协议
- **在本P中的作用**：HyperText Transfer Protocol，超文本传输协议，定义了客户端与服务器之间请求-响应的通信格式。爬虫的本质就是“用代码模拟人发 HTTP 请求”
- **来源**：字幕直接提到
- **建议方向**：后续学习 Python 的 requests 库时，理解它底层封装的正是 HTTP 协议的请求/响应逻辑

### HTTPS
- **类型**：安全协议
- **在本P中的作用**：HTTP + SSL/TLS，对传输数据进行加密，防止中间人攻击。课程强调它与 HTTP 的请求报文格式一致，爬虫无需区分
- **来源**：字幕直接提到
- **建议方向**：了解 HTTPS 的加密过程（SSL/TLS 握手），但写爬虫时聚焦于请求报文内容本身而非传输层加密

### HTTP/0.9 / HTTP/1.0 / HTTP/1.1
- **类型**：协议版本
- **在本P中的作用**：按历史顺序介绍协议演进，1.1 是目前主流版本，新增了五种请求方法。老师讲解目的是让学习者理解 GET/POST 并非一开始就存在，而是随版本迭代加入的
- **来源**：基于上下文推断
- **建议方向**：了解 HTTP/2、HTTP/3 的主要特点（如多路复用、头部压缩），但初学阶段聚焦 HTTP/1.1 的请求-响应模型

## 三、请求方法

### GET
- **类型**：HTTP 请求方法
- **在本P中的作用**：用于从服务器获取资源，参数拼接在 URL 后面（查询字符串）。老师强调爬虫最常用，且参数有长度限制
- **来源**：字幕直接提到
- **建议方向**：在浏览器开发者工具中观察 GET 请求的 URL 结构，尝试构造带参数的查询请求

### POST
- **类型**：HTTP 请求方法
- **在本P中的作用**：用于向服务器提交数据（如登录表单），参数放在请求体中。老师强调登录场景必须用 POST，账号密码不能暴露在 URL
- **来源**：字幕直接提到
- **建议方向**：找一个登录表单，抓包观察 POST 请求的请求体数据格式（如 application/x-www-form-urlencoded 或 JSON）

### OPTIONS / PUT / DELETE / TRACE / CONNECT
- **类型**：HTTP 请求方法（次要）
- **在本P中的作用**：HTTP/1.1 新增的五种方法，老师一带而过，爬虫场景较少使用但需了解方法体系
- **来源**：基于上下文推断
- **建议方向**：后续学习 RESTful API 时再深入这些方法的具体使用场景

## 四、请求头字段

### User-Agent
- **类型**：HTTP 请求头字段
- **在本P中的作用**：浏览器的身份标识字符串，包含浏览器内核、版本、操作系统等信息。老师强调如果不设置，服务器会识别为爬虫程序
- **来源**：字幕直接提到
- **建议方向**：在 Network 面板查看自己的 User-Agent 字符串，尝试构造不同的 User-Agent 理解反爬机制

### Cookie
- **类型**：HTTP 请求头字段
- **在本P中的作用**：存储用户登录状态和会话信息。老师强调访问需要登录的页面时，爬虫必须携带正确 Cookie
- **来源**：字幕直接提到
- **建议方向**：登录网站后抓包观察 Cookie 字段，尝试在代码中手动构造 Cookie 发送请求

### Referer
- **类型**：HTTP 请求头字段
- **在本P中的作用**：标识当前请求的来源页面，用于防盗链校验。老师提醒某些资源会验证 Referer，爬虫缺少此字段可能被拒绝
- **来源**：字幕直接提到
- **建议方向**：抓包观察 Referer 的值，理解它与请求 URL 之间的关系，思考防盗链的逻辑

### Accept / Accept-Encoding / Accept-Language
- **类型**：HTTP 请求头字段
- **在本P中的作用**：分别声明期望返回的内容类型、支持的压缩方式、期望的语言。属于浏览器自动发送的常规字段
- **来源**：字幕直接提到
- **建议方向**：了解这些字段的常见值（如 zh-CN、gzip），但爬虫实操时通常不必手动设置

### Content-Length / Content-Type
- **类型**：HTTP 请求头字段
- **在本P中的作用**：Content-Length 标识请求体字符长度，Content-Type 标识请求体数据类型（如 application/json）。POST 请求时需关注
- **来源**：字幕直接提到
- **建议方向**：POST 请求时注意 Content-Type 与请求体格式的一致性（如 JSON 数据应配 application/json）

### Host / Connection / X-Requested-With
- **类型**：HTTP 请求头字段
- **在本P中的作用**：Host 标识目标域名；Connection 控制连接类型（长/短连接）；X-Requested-With 标识 AJAX 异步请求。老师简略带过，爬虫实操中次要
- **来源**：基于上下文推断
- **建议方向**：了解含义即可，requests 库会自动处理这些字段

## 五、响应相关

### Set-Cookie
- **类型**：HTTP 响应头字段
- **在本P中的作用**：服务器通知浏览器设置 Cookie 的指令。老师用百度搜索场景演示，Set-Cookie 是 Cookie 机制的起点
- **来源**：字幕直接提到
- **建议方向**：抓包观察首次访问某网站时响应头中的 Set-Cookie，理解它与后续请求中 Cookie 字段的关系

### HTTP 状态码（1xx / 2xx / 3xx / 4xx / 5xx）
- **类型**：协议元素
- **在本P中的作用**：服务器对请求处理结果的数字代码。老师分类讲解，爬虫最常遇到 200（成功）、403（被拒）、404（不存在）、302（重定向）
- **来源**：字幕直接提到
- **建议方向**：在抓包过程中收集不同状态码的实例，建立“状态码→问题判断→处理策略”的对应关系

## 六、核心概念与数据结构

### URL（统一资源定位符）
- **类型**：数据结构
- **在本P中的作用**：由协议+域名+路径+参数组成。老师以 `https://www.baidu.com/s?wd=美食` 为例拆解各部分含义
- **来源**：字幕直接提到
- **建议方向**：练习分析不同网站的 URL 结构，理解路径与参数的区别

### Query String Parameters（查询字符串参数）
- **类型**：URL 参数位置
- **在本P中的作用**：GET 请求参数在 URL 中的显示区域，以 `?key=value&key2=value2` 形式呈现。老师预告后续写爬虫时会在 URL 后加参数
- **来源**：字幕直接提到
- **建议方向**：在浏览器地址栏构造带参数的 URL，观察开发者工具中参数的解析方式

### 请求报文 / 响应报文
- **类型**：数据结构
- **在本P中的作用**：HTTP 协议规定的通信格式。请求报文包含首行、请求头、空白行、请求体；响应报文包含状态行、响应头、空白行、响应体
- **来源**：字幕直接提到
- **建议方向**：用 Raw Headers 模式对照查看一个真实请求的完整报文，验证四部分结构

### Cookie 机制
- **类型**：浏览器-服务器交互机制
- **在本P中的作用**：服务器通过 Set-Cookie 设置，浏览器自动存储并在后续请求中携带。用于维持用户会话、存储偏好，是爬虫登录态处理的核心
- **来源**：字幕直接提到
- **建议方向**：通过具体场景（如电商搜索后推荐、广告追踪）理解 Cookie 的持久化行为

### DNS（域名系统）
- **类型**：网络基础设施
- **在本P中的作用**：将人类可读的域名（如 www.baidu.com）转换为 IP 地址。老师在浏览器请求流程中提及，属于前置知识
- **来源**：字幕直接提到
- **建议方向**：了解 DNS 解析的基本流程，后续学习爬虫的 IP 代理时会涉及 DNS 缓存等概念
