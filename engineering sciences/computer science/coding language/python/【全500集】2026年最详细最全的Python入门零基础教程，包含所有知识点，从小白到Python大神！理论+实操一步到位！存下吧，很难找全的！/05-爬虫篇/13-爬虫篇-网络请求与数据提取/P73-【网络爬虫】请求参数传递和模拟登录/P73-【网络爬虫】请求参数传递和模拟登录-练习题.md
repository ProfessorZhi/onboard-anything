## 答题区

**结构识别答案**

1.1 查询参数：A、C、D；B 不是（是路径参数）

1.2 代码A→查询参数；代码B→表单参数；代码C→JSON参数

1.3 混合使用：Set-Cookie 暗示 Cookie+Session，响应体中的 token 字段暗示 Token。需结合实际网站逻辑判断。

**机制理解答案**

2.1 Session ID 存储在 Cookie 中，requests 库的 `Session` 对象会自动管理 Cookie（存储 → 后续请求自动携带）。

2.2 关键区别：Cookie 由浏览器自动携带，Token 需要代码手动添加到请求头。Token 存储在 LocalStorage/SessionStorage 中。爬虫需从响应中提取 Token，并在后续请求的 `headers` 中手动添加。

2.3 场景A用 `data`（传统表单）；场景B用 `json`（RESTful API）。

2.4 A、C、D

**最小实践参考答案**

```python
# 3.1
r = requests.get("https://httpbin.org/get", params={"name": "张三", "age": 25})
print(r.json()["args"])
```

```python
# 3.2
payload = {"course": "Python爬虫", "lesson": 73, "completed": True}
r = requests.post("https://httpbin.org/post", json=payload)
print(r.json()["json"])
```

```python
# 3.3
login_data = {"username": "test", "password": "test"}
r = requests.post(login_url, json=login_data)
token = r.json().get("token")
headers = {"Authorization": f"Bearer {token}"} if token else {}
user_r = requests.get(userinfo_url, headers=headers)
print(user_r.json())
```
