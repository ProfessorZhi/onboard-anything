# P73 知识框架 - 【网络爬虫】请求参数传递和模拟登录
## 本P位置

Python网络爬虫教程第73集，爬虫进阶内容的前置章节。前半段承接第72P豆瓣Top250案例，引入URL查询参数概念；后半段从GET请求参数传递过渡到POST请求，讲解Cookie+Session与Token两种鉴权机制。为后续模拟登录实战提供理论和编码基础。

## 核心问题

如何让爬虫正确传递请求参数并维持登录状态？

两个子问题：

- GET/POST请求中，参数如何编码和传递？
- 登录成功后，如何让后续请求携带认证信息？

## 知识骨架

```
请求参数传递
├── GET请求参数
│   ├── 路径参数：URL路径部分
│   └── 查询参数：URL问号后，键=值&键=值
│       ├── 方式一：直接拼接URL
│       └── 方式二：requests.get(params=字典)
│
└── POST请求参数
    ├── 表单参数（Form Data）
    │   └── requests.post(data=字典)
    └── JSON参数
        └── requests.post(json=字典)

鉴权机制
├── Cookie + Session
│   ├── 服务器生成Session，返回Session ID
│   ├── 通过Set-Cookie响应头传递
│   ├── 浏览器自动存储并携带
│   └── 适用：非前后端分离项目（90%以上）
│
└── Token
    ├── 服务器生成Token密文
    ├── 通过响应体返回
    ├── 存储于LocalStorage/SessionStorage
    ├── 前端代码手动附加到请求
    └── 适用：前后端分离项目
```

**输出：**

- 响应内容（HTML/JSON/二进制）

## 易错点

1. GET参数拼接时中文未URL编码，导致请求失败或结果乱码
2. 混淆data和json参数：表单参数用data，JSON参数用json
3. 登录请求发送前误填正确密码，导致页面跳转后无法捕获请求
4. Token鉴权时忘记在后续请求中手动携带Token
5. 将Session ID（Cookie方式）与Token混淆：前者自动携带，后者需前端代码附加
6. 未通过抓包确认参数格式就直接写代码
