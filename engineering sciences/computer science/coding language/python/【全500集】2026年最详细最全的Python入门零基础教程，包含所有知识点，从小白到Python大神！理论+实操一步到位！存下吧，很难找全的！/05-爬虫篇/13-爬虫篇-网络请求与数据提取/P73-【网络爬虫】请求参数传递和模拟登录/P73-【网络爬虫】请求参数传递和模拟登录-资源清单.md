# P73 资源清单 - 【网络爬虫】请求参数传递和模拟登录
---

## 一、程序 / 库 / 框架

### 1. requests 库

- **类型**：Python 第三方 HTTP 客户端库
- **在本 P 中的作用**：核心请求工具，用于发送 GET/POST 请求，传递查询参数（params）、表单参数（data）、JSON 参数（json）。课程中所有代码示例均基于此库实现。
- **来源**：字幕直接提到
- **建议方向**：系统学习 requests 库 API（Session 对象、重试机制、异常处理），对比其与 urllib 的差异与适用场景

---

## 二、浏览器工具

### 2. 浏览器开发者工具（Network 面板）

- **类型**：浏览器内置调试工具（Chrome / Firefox / Edge 等均支持）
- **在本 P 中的作用**：用于抓包分析，观察 URL 查询参数的格式、POST 请求的 Payload 类型（Form Data / Request Payload）、响应头中的 Set-Cookie 字段，以及请求自动携带 Cookie 的行为。是确认参数格式和鉴权方式的关键工具。
- **来源**：字幕直接提到
- **建议方向**：熟练掌握 Network 面板的过滤、搜索、条件断点等高级功能，用于快速定位目标请求

### 3. 浏览器开发者工具（Application 面板）

- **类型**：浏览器内置调试工具
- **在本 P 中的作用**：用于查看 Cookie 存储情况（Cookie 选项卡）和 Token 存储位置（Local Storage / Session Storage 选项卡），辅助判断网站的鉴权机制类型。
- **来源**：字幕直接提到
- **建议方向**：对比同一网站在登录前后的 Cookie/LocalStorage 变化，理解鉴权数据的生命周期

## 三、网站 / 平台

### 4. 百度搜索（https://www.baidu.com）

- **类型**：搜索引擎
- **在本 P 中的作用**：演示 GET 请求查询参数传递的载体。输入关键词后，URL 末尾携带 `wd` 参数，通过两种方式（拼接 URL / params 参数）均成功获取搜索结果页面。
- **来源**：字幕直接提到
- **建议方向**：尝试抓取百度搜索结果（如获取前 10 条标题和链接），注意遵守 robots.txt 和搜索服务条款

### 5. 豆瓣 Top250（https://movie.douban.com/top250）

- **类型**：电影评分网站
- **在本 P 中的作用**：承接第 72P 案例，用于展示翻页参数（`start=0/50`）在 URL 中的体现，引出查询字符串参数的概念。演示参数修改后页面数据的相应变化。
- **来源**：字幕直接提到
- **建议方向**：编写多页爬虫，抓取豆瓣 Top250 的电影名称、评分、评价人数等信息；注意处理反爬机制

### 6. 手机版百度翻译（https://fanyi.baidu.com）

- **类型**：在线翻译平台
- **在本 P 中的作用**：演示 POST 请求的表单参数传递。抓包展示其 POST 请求携带 `kw` 字段进行翻译查询，成功获取包含 `session_id` 的响应数据。
- **来源**：字幕直接提到
- **建议方向**：尝试调用百度翻译接口，实现多语言翻译功能；注意官方 API 可能有调用频率限制

### 7. 中华网（https://www.china.com）

- **类型**：综合性新闻门户网站
- **在本 P 中的作用**：演示 Cookie + Session 鉴权机制的完整流程。登录后 Application 面板中 Cookie 新增用户身份字段，验证后续请求自动携带 Cookie 的行为。未携带有效 Cookie 时请求被拒绝。
- **来源**：字幕直接提到
- **建议方向**：尝试模拟登录中华网，抓取需登录可见的内容（如评论、个人中心）；注意账号安全

- **来源**：字幕直接提到
- **建议方向**：尝试获取 Token 后访问需登录的课程资源；注意前后端分离架构下 Token 的获取和附加方式

## 四、核心概念与机制

### 9. 查询字符串参数（Query String Parameters）

- **类型**：HTTP 请求参数类型
- **在本 P 中的作用**：GET 请求中 URL 问号后面的键值对部分，格式为 `键=值&键=值`。课程讲解了两种传递方式：直接拼接 URL（中文需 URL 编码）和使用 `requests.get(params=字典)` 自动编码。
- **来源**：字幕直接提到
- **建议方向**：在实战中优先使用 params 参数方式处理查询参数，避免手动编码的繁琐和错误

### 10. 表单参数（Form Data）

- **类型**：POST 请求参数类型
- **在本 P 中的作用**：POST 请求中以键值对形式编码在请求体中的参数，对应浏览器 Network 面板 Payload 下的 Form Data 部分。使用 `requests.post(data=字典)` 传递。
- **来源**：字幕直接提到
- **建议方向**：结合登录场景练习，抓取登录请求的 Form Data 格式，模拟表单提交

### 11. JSON 参数（Request Payload）

- **类型**：POST 请求参数类型
- **在本 P 中的作用**：以 JSON 格式编码放在请求体中的参数，常见于前后端分离项目。使用 `requests.post(json=字典)` 传递，返回值通常也是 JSON 格式。
- **来源**：字幕直接提到
- **建议方向**：练习使用 `response.json()` 方法直接解析 JSON 响应体，对比 `response.text` 的差异

### 12. Cookie + Session 鉴权机制

- **类型**：服务端会话管理机制
- **在本 P 中的作用**：服务器生成 Session 并通过 Set-Cookie 响应头返回 Session ID，浏览器自动存储并在后续请求中携带。课程详细图解了完整流程，并通过中华网演示验证。
- **来源**：字幕直接提到
- **建议方向**：理解 Session 存储位置（服务端内存/数据库）、过期机制；练习从登录响应中提取 Cookie 并复用

### 13. Token 鉴权机制

- **类型**：无状态身份验证机制
- **在本 P 中的作用**：服务器生成加密 Token 放在响应体返回，客户端存储于 LocalStorage，由前端代码手动附加到后续请求中。课程通过课堂派演示 LocalStorage 存储和前后端分离域名特征。
- **来源**：字幕直接提到
- **建议方向**：理解 Token 的生成逻辑（JWT 等标准）、有效期验证机制；练习从响应体提取 Token 并构造请求头

### 14. LocalStorage / SessionStorage

- **类型**：浏览器客户端存储 API
- **在本 P 中的作用**：Token 的常见存储位置。与 Cookie 不同，不会自动随请求发送，需由前端 JavaScript 代码主动取出并附加。
- **来源**：基于上下文推断（Token 鉴权机制需要存储载体）
- **建议方向**：了解二者的生命周期差异（SessionStorage 标签页关闭后清除），以及 IndexedDB 等其他存储方案

## 五、配置文件 / 数据格式

### 15. 请求头（Headers）

- **类型**：HTTP 请求配置
- **在本 P 中的作用**：课程中多次强调从浏览器开发者工具复制请求头，特别是 User-Agent 字段用于模拟浏览器访问，防止被反爬机制拦截。
- **来源**：字幕直接提到
- **建议方向**：学习常用请求头字段（Referer、Accept、Authorization 等），理解其作用并合理设置

### 16. 响应头（Set-Cookie）

- **类型**：HTTP 响应配置
- **在本 P 中的作用**：Cookie + Session 鉴权机制中，服务器通过 Set-Cookie 响应头将 Session ID 返回给浏览器。这是维持登录状态的关键字段。
- **来源**：字幕直接提到
- **建议方向**：抓包观察多个网站的 Set-Cookie 字段，理解 Cookie 的属性（Domain、Path、Expires、HttpOnly、Secure 等）
