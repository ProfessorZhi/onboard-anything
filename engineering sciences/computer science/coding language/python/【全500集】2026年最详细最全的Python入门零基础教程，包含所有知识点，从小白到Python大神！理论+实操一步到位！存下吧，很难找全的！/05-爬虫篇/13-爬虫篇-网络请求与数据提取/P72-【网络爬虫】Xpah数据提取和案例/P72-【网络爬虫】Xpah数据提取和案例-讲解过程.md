# P72 讲解过程 - 【网络爬虫】Xpah数据提取和案例
## 本P定位

本P是网络爬虫章节中的核心知识点，承接上一P的 `requests` 请求内容，正式引入 **XPath** 作为从 HTML 源码中精准定位和提取目标数据的工具。本P以 **豆瓣 Top 250 电影排行榜** 为实战案例，完整演示从发起请求、保存页面、调试 XPath 表达式，到最终用 Python 的 `lxml` 库完成数据提取的全流程。目标数据为每部电影的**名称**和**评分**，共计 250 条。

## 讲解过程

### 一、案例需求与数据验证

课程首先明确本 P 的任务：抓取豆瓣 Top 250 中的电影名称和评分。老师打开豆瓣 Top 250 网站，直观展示页面中电影名称和评分所在的区域。随后按照上一 P 学到的思路，第一步发送请求获取整个页面的 HTML 源码，并将其保存为本地文件 `douban.html`，通过指定 UTF-8 编码确保中文内容正确存储。

保存完成后，老师打开该 HTML 文件，使用编辑器的搜索功能（Ctrl+F）搜索页面中已知的第一部电影名"肖申克的救赎"来验证数据是否被成功获取。搜索结果显示数据确实存在于本地文件中，页面包含完整的 250 部电影信息。老师还依次展示了"霸王别姬""阿甘正传"等电影数据，确认整页数据都在。

### 二、XPath 概念引入

验证数据存在后，老师指出问题所在：整个网页的 HTML 源码全部被拿下来了，但需要的数据（电影名称和评分）只是其中很小的一部分，必须从 HTML 中把这些目标数据**提取**出来。由此引出本 P 的核心知识点——**XPath**（XML Path Language）。

老师介绍 XPath 的本质：它是一种专门用于在 XML 文档中定位和提取数据的路径语言。由于 HTML 文档的结构与 XML 本质相同，节点关系和标签形式一致，因此 XPath 完全适用于 HTML 页面的数据提取。

### 三、浏览器辅助工具介绍

为了方便在浏览器中调试 XPath 表达式，老师推荐安装一款 Chrome 浏览器插件：**XPath Helper**。该插件的启动方式为 `Shift + Ctrl + X`。安装后可以在页面上直接书写 XPath 表达式并实时查看匹配结果，非常适合在编写爬虫代码之前做前期的路径调试。

老师同时说明，即使不安装插件，也可以通过以下方式辅助定位：按 F12 打开开发者工具，切换到 Elements 面板，点击左上角的箭头图标选中页面中的目标元素，即可定位到该元素在 HTML 源码中的位置。如果需要某个元素的 XPath 表达式，可以右击该元素，选择 Copy → Copy XPath，浏览器会自动复制该元素的 XPath。

但老师指出，浏览器自动复制的 XPath 往往路径较长且不够稳定，如果能够自己手写 XPath，可以写得更加简洁高效。

### 四、XPath 基础语法讲解

老师按以下顺序逐一讲解 XPath 的语法核心。

#### 1. 节点选择符号

- **`/`（单斜杠）**：从根节点开始选择，表示"下面的直接子节点"。例如 `/html` 表示选择根节点 html 下面的直接子元素。
- **`//`（双斜杠）**：从根节点往下，不考虑元素的具体位置，只要在其下方即可匹配，是一种全局查找方式。
- **`.`（点号）**：表示当前节点。
- **`..`（双点号）**：表示当前节点的父节点（负节点）。

老师用浏览器开发者工具的搜索框实际演示了这些符号的用法。例如输入 `//ul` 可以选中页面中所有 `ul` 标签，`//ol//li` 则表示在 `ol` 标签下（不管中间有多少层）找到所有 `li` 标签。演示中搜索 `//ol//li` 得到 25 个结果，对应豆瓣 Top 250 的 25 部电影所在的 `li` 标签。

#### 2. 标签名与通配符

XPath 中直接写标签名表示选中该标签下的所有匹配元素。例如 `//li` 选中所有 `li` 标签，`//ul//li` 选中 `ul` 标签下的所有 `li`。

- **`*`（星号）**：通配符，表示任意标签名。当不确定中间节点叫什么名字时，可以用 `*` 代替，然后通过条件过滤。例如 `//*[@class="title"]` 表示选中任意标签中 class 属性等于 title 的元素。老师演示用 `//*[@class="title"]` 同样能精准找到所有电影名称的 `span` 标签，数量为 49 个（因为包含中英文两个名称），说明通配符配合属性过滤可以大幅简化 XPath 表达式。

#### 3. 属性选取符号 `@`

- **`@`（艾特符）**：用于选取元素的属性。例如 `//li//span[@class="title"]` 表示在 `li` 下查找 `span` 标签，并且该 `span` 的 class 属性必须等于 title。

老师演示了在开发者工具中输入 `//*[@class="title"]` 后，匹配结果显示 49 个，其中包含中文名称和英文名称两种不同的 `span` 标签。

#### 4. 谓语条件（中括号 `[]`）

谓语条件是 XPath 中最重要的过滤机制，用中括号包裹，用于对节点进行进一步的筛选。

- **按属性值过滤**：`//span[@class="title"]`
- **按索引过滤**：XPath 使用 1-based 索引，类似 Python 列表。例如 `//li//span[@class="title"][1]` 表示找每个 `li` 下面的第一个 class 为 title 的 `span` 标签。老师演示输入 `//li//span[@class="title"][1]` 找到了所有电影的第一个名称标签（中文名），共 25 个。
- **按位置函数过滤**：常用 `last()` 函数表示最后一个节点。例如 `//li//span[@class="title"][last()]` 找最后一个对应的 `span`（在有英文名的情况下是英文名）。还可以用 `last()-1` 表示倒数第二个。
- **比较运算符**：支持 `>`、`<`、`>=`、`<=`、`=` 等，用于数值属性的比较。

#### 5. 文本获取与属性获取

- **获取标签文本内容**：在 XPath 表达式末尾加上 `/text()` 即可获取该标签包裹的文本。例如 `//li//span[@class="title"]/text()` 可以获取所有匹配 `span` 标签内的文字内容。
- **获取属性值**：在 XPath 表达式末尾加上 `/@属性名` 即可获取该标签的指定属性值。例如 `//span[@class="rating_num"]/@class` 可以获取 class 属性值。老师强调，**页面提取数据只存在两种情况**：要么数据在标签的文本里（用 `text()` 获取），要么数据在标签的属性里（用 `@属性名` 获取）。

#### 6. 模糊匹配 `contains()`

- **`contains()` 函数**：用于文本内容的模糊匹配。例如 `//*[contains(text(), "肖申")]/* 可以找到文本内容中包含"肖申"字符串的元素。老师演示用 `//*[contains(text(), "当幸福")]/* 精确找到了"当幸福来敲门"这一部电影。`contains()` 在只知道部分关键词时非常有用。

### 五、Python 中使用 XPath

#### 1. 安装 lxml 库

老师说明在 Python 中使用 XPath 需要借助第三方库 `lxml`，它是专门用来解析 XML 和 HTML 文档的高性能库。通过 `pip install lxml` 命令安装（老师演示时显示已安装，直接提示成功）。

#### 2. 基本使用流程

在 Python 中使用 XPath 提取数据的步骤为三步：

1. 导入 `lxml.etree`
2. 将获取到的 HTML 字符串通过 `etree.HTML()` 转换为可解析的文档对象
3. 使用 `对象.xpath("xpath表达式")` 提取数据

老师从本地文件读取之前保存的 `douban.html` 内容（字符串形式），然后调用 `HTML = etree.HTML(html_content)` 将其转换为树形文档对象。

#### 3. 提取电影名称

老师首先演示直接定位所有电影名称的 `span` 标签。写出的 XPath 为 `//*[@class="title"]/text()`。运行后返回的是一个包含 49 个节点的列表（25 个中文名 + 24 个英文名，因为最后一名的英文名位置可能不同）。通过上下箭头可以在结果列表中逐一查看，确认包含"肖申克的救赎""霸王别姬""阿甘正传""泰坦尼克号""千与千寻""这个杀手不太冷"等电影名称。

#### 4. 提取电影评分

评分数据同样在一个 `span` 标签中，通过检查页面元素确认其 class 属性为 `rating_num`。使用 XPath `//*[@class="rating_num"]/text()` 同样得到 25 个评分结果。老师演示了完整的提取代码并运行验证。

### 六、批量数据提取的最佳实践

老师指出了一个关键的提取原则——**先定位父元素，再逐层提取**，而不是直接分别定位两个独立的数据字段。

具体步骤如下：

1. **第一步**：定位包含所有数据的父级元素。老师分析豆瓣 Top 250 页面的 HTML 结构：所有电影数据都在一个 `ul` 标签中，每一部电影对应一个 `li` 标签。XPath 写为 `//ul//li`，得到 25 个 `li` 元素。
2. **第二步**：对定位到的元素列表进行遍历（`for` 循环）。
3. **第三步**：在每个 `li` 元素内部再使用 XPath 提取具体字段。写法为 `li.xpath("./span[@class='title']/text()")`（注意此时路径前面用 `.` 表示当前节点上下文，因为是相对路径）。

老师强调，这种方式的优点在于每条数据的名称和评分都来自同一个 `li` 节点，天然保证了一一对应的关系，避免了分别提取后数据错位的问题。如果直接分别提取所有名称和所有评分，两个列表的顺序和对应关系需要额外保证。

## 本P结论

1. **XPath 是从 HTML 中精准提取数据的标准工具**，它通过路径表达式定位 XML/HTML 文档中的节点，适用于结构化的网页数据提取。
2. **XPath 核心语法包括**：`/` 和 `//` 选择节点、`.` 和 `..` 操作上下文、`@` 选取属性、`[]` 谓语条件过滤、`text()` 获取文本、`*` 通配符、`contains()` 模糊匹配、`last()` 等位置函数。
3. **Python 中使用 XPath 的标准流程**：安装 `lxml` 库 → 通过 `etree.HTML()` 转换文档 → 调用 `.xpath()` 执行表达式。
4. **批量提取数据的最佳实践**：先定位包含所有数据的父级容器（如 `//ul//li`），再遍历每个父元素，在其内部用相对路径提取各字段，保证数据一一对应。
5. **两种数据获取方式**：数据在标签文本中用 `text()`，在标签属性中用 `@属性名`。
6. **推荐借助 XPath Helper 插件或浏览器开发者工具进行前期调试**，手写 XPath 比自动复制更简洁稳定。
