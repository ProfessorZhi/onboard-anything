## 一、Python 库与模块

### 1. lxml

| 字段 | 内容 |
|---|---|
| **名称** | lxml |
| **类型** | Python 第三方库 |
| **在本 P 中的作用** | Python 端执行 XPath 查询的核心依赖；负责将 HTML 字符串解析为可遍历的文档树，并提供 `.xpath()` 方法执行路径表达式 |
| **来源** | 字幕直接提到 |
| **安装命令** | `pip install lxml` |
| **官方文档** | https://lxml.de |

**建议方向**
- 练习用 `etree.HTML()` 解析不同来源的 HTML 字符串，观察树形结构
- 对比 `etree.HTML()`（容错解析）与 `etree.fromstring()`（严格解析）的差异
- 尝试用 `etree.tostring()` 将解析后的节点再序列化回字符串，理解往返过程
- 阅读官方文档中 XPath 1.0 支持范围的说明，了解 lxml 未支持的函数

---

### 2. lxml.etree（etree 子模块）

| 字段 | 内容 |
|---|---|
| **名称** | lxml.etree |
| **类型** | lxml 子模块 |
| **在本 P 中的作用** | 提供两个关键接口：`etree.HTML(html_str)` 将字符串转为 ElementTree 文档对象；文档对象的 `.xpath("表达式")` 方法执行查询并返回 Python 列表 |
| **来源** | 字幕直接提到 |
| **典型用法** | `from lxml import etree` → `tree = etree.HTML(text)` → `result = tree.xpath("//span/text()")` |

**建议方向**
- 练习在遍历父节点后使用相对路径 `./` 和 `.//` 提取子字段，理解上下文节点切换
- 对比 `.xpath()` 返回列表与 `find()` / `findall()` 返回 Element 对象的区别（⚠️ 推断）
- 用 `etree.HTML()` 解析带有缺失标签的不规范 HTML，观察容错修复结果

## 二、浏览器插件

### 3. XPath Helper

| 字段 | 内容 |
|---|---|
| **名称** | XPath Helper |
| **类型** | Chrome 浏览器扩展插件 |
| **在本 P 中的作用** | 在浏览器页面内实时输入 XPath 表达式并即时高亮显示匹配结果，用于爬虫开发前的表达式调试，无需运行 Python 代码即可验证路径是否正确 |
| **启动快捷键** | Shift + Ctrl + X |
| **来源** | 字幕直接提到 |
| **获取渠道** | Chrome 应用商店搜索"XPath Helper"（⚠️ 推断获取方式） |

**建议方向**
- 对任意网页练习分别用 `/`、`//`、`.//` 写路径，观察匹配数量差异
- 故意写出冗长路径与简洁路径，比对结果是否等价，培养化简意识
- 用 `contains()` 在多个页面中做模糊定位，积累常用过滤表达式
- 与"浏览器 Copy XPath"的自动结果对比，手动简化冗余层级

## 七、XPath 语法速查（本 P 涉及的表达式片段）

> 以下不是独立资源，而是本 P 使用的 XPath 语法要素汇总，供检索与复习。

| 表达式 / 函数 | 类型 | 说明 | 来源 |
|---|---|---|---|
| `/` | 路径符 | 根节点或直接子节点 | 字幕直接提到 |
| `//` | 路径符 | 全局后代查找（跨层） | 字幕直接提到 |
| `.` | 上下文符 | 当前节点 | 字幕直接提到 |
| `..` | 上下文符 | 父节点 | 字幕直接提到 |
| `@属性名` | 属性选取 | 匹配或获取属性 | 字幕直接提到 |
| `[]` | 谓语 | 条件过滤（属性值 / 索引 / 函数） | 字幕直接提到 |
| `text()` | 函数 | 获取标签内文本节点 | 字幕直接提到 |
| `*` | 通配符 | 匹配任意标签名 | 字幕直接提到 |
| `contains(text(), "关键词")` | 函数 | 文本模糊匹配 | 字幕直接提到 |
| `last()` | 位置函数 | 选取最后一个节点 | 字幕直接提到 |
| `last()-1` | 位置函数 | 倒数第二个节点 | 字幕直接提到 |

**建议方向**
