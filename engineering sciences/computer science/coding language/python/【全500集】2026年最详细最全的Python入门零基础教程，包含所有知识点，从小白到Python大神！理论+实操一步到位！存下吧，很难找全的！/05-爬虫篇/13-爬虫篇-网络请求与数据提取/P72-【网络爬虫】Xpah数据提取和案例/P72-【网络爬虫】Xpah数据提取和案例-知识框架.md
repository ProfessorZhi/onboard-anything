# P72 知识框架 - 【网络爬虫】Xpah数据提取和案例

## 核心问题

如何从已获取的 HTML 源码中精准定位并提取目标数据？

---

## 知识骨架

```
XPath 数据提取流程
├── 概念：XML Path Language，专用于 XML/HTML 节点定位
├── 语法要素
│   ├── 节点路径：/（子）//（后代）.（当前）..（父）
│   ├── 属性选取：@
│   ├── 谓语过滤：[]（属性值 / 索引 / 位置函数）
│   ├── 数据获取：text()（文本）/ @attr（属性值）
│   ├── 通配符：*
│   ├── 模糊匹配：contains()
│   └── 位置函数：last()、last()-1
├── 调试工具
│   ├── XPath Helper（Chrome 插件，Shift+Ctrl+X 启动）
│   └── 浏览器开发者工具（Elements / Copy XPath）
└── Python 实现
    ├── 库：lxml
    └── 模块：etree（HTML() 解析 / xpath() 查询）
```

---

## 易错点

1. **索引从 1 开始**：XPath 使用 1-based 索引，`[1]` 表示第一个，不是 `[0]`
2. **相对路径需加点**：遍历父元素后用 `.//` 或 `./` 表示当前上下文
3. **数据只分两类**：要么在标签文本（`text()`），要么在标签属性（`@attr`）
4. **浏览器复制的 XPath 不稳定**：路径冗长，手写更简洁高效
5. **批量提取要保对应**：先定位父元素再遍历内部提取，避免名称与评分错位
