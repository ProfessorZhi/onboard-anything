## 最小实践

**5.** 请在空白处补全代码，构建一个最小包结构：

```
project/
    main.py
    utils/
        __init__.py
        math.py
```

要求：
- `utils/__init__.py` 中使用 `__all__` 控制只导出 `double` 函数，不导出 `triple` 函数
- `utils/math.py` 中定义 `double(x)` 返回 x×2，`triple(x)` 返回 x×3
- `main.py` 使用 `from utils import *` 导入后，能调用 `double(5)` 返回 10，但 `triple` 不可用

```python
# utils/math.py
# 补全代码

# utils/__init__.py
# 补全代码

# main.py
from utils import *
print(double(5))   # 期望输出: 10
# print(triple(5))  # 期望: NameError
```

## 最小替代练习

**9.** 动手实践：使用 VS Code 或任何编辑器，创建一个包含子包的完整包结构，然后通过不同的导入方式验证 `__all__` 的控制效果。

```
mylib/                    ← 包根目录
    __init__.py           ← 包初始化（设置 __all__）
    core.py               ← 核心模块
    helper.py             ← 辅助模块
    tests/                ← 子包
        __init__.py
        test_core.py
        test_helper.py
```

完成以下验证任务：

| 导入方式 | 预期行为 |
|----------|----------|
| `import mylib` | 可访问哪些内容？ |
| `from mylib import core` | 成功/失败？ |
| `from mylib import *` | 只导入哪些？与 `__all__` 是否一致？ |
| `from mylib.core import func` | 是否正常工作？ |
| `import mylib.tests.test_core` | 嵌套导入是否可行？ |

**参考答案**

| 题号 | 答案要点 |
|------|----------|
| 1 | A、D（B缺少__init__.py，C后缀错误） |
| 2 | 不影响，`__all__` 只作用于 `from...import *` |
| 3 | C（仅 tool_func 被导入） |
| 4 | 各执行 1 次 |
| 5 | 答案见下方代码块 |
| 6 | `__all__ = []` 导致 `import *` 无法导入任何模块，但 `from mypkg.utils import helper` 直接指定路径所以不受 `__all__` 影响——这题本身不冲突，若 core.py 用 `from mypkg.utils import *` 才会报错 |
| 7 | 包名遮蔽了标准库；方案：改名 / 使用完整路径 `import unittest` 前先确认未被覆盖 |
| 8 | 每次 import 语句都会执行 `__init__.py`，复杂逻辑会导致每次导入都产生性能开销 |
| 9 | 动手验证，无标准答案 |

```python
# utils/math.py
def double(x):
    return x * 2

def triple(x):
    return x * 3

# utils/__init__.py
__all__ = ["double"]  # 只导出 double
```
