## 核心配置文件

### 1. `__init__.py`

- **类型**：配置文件
- **作用**：Python 识别目录为包的关键文件，具备三个核心功能——标志包身份、控制导入范围（通过 `__all__`）、执行轻量级初始化代码
- **来源**：字幕直接提到
- **后续可练习方向**：建议方向 — 在自定义包中添加初始化逻辑（如预加载模块、设置日志、注册插件等），观察导入行为

---

### 2. `__all__` 变量

- **类型**：Python 变量（列表类型）
- **作用**：控制 `from ... import *` 的导出范围，决定批量导入时哪些模块或函数可见；可写在包的 `__init__.py` 中控制模块导出，也可写在模块文件中控制功能导出
- **来源**：字幕直接提到
- **后续可练习方向**：建议方向 — 在同一模块中定义多个函数，通过 `__all__` 灵活控制哪些函数对外部可见，模拟标准库的做法

## 语法要素（语言特性）

### 11. 三种导入语法

- **类型**：Python 语法要素
- **作用**：
  - `import 包名.模块名`：精确路径导入，适合复杂项目
  - `from 包名 import 模块名`：简化书写，适合常用模块
  - `from 包名 import *`：批量导入，需配合 `__all__` 使用
- **来源**：字幕直接提到
- **后续可练习方向**：建议方向 — 在大型开源项目中搜索这三种导入方式的使用场景，对比各自的优缺点
