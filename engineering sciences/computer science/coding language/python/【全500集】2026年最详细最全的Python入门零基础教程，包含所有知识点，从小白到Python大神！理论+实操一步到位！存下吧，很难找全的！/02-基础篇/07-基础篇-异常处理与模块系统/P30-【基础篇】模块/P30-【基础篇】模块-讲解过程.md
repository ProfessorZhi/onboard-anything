# P30 讲解过程 - 【基础篇】模块
## 本P定位

本P是「基础篇·模块」章节的第一小节，围绕 Python 模块这一核心概念展开教学。通过问题驱动引出模块的价值，逐一讲解模块的定义、三种类型、导入方式，最后补充 `if __name__ == "__main__"` 这一关键技巧及注意事项。

---

## 讲解过程

### 一、问题引入：代码多了会怎样

**时间区间：0:00 - 0:57**

课程一开始，老师先抛出痛点：当代码功能增多后，零散的代码变得难维护、难以复用。就像一堆杂乱的工具扔在地上，要用时找半天。借此引出本节主题——**模块和包**，强调它们是 Python 中的「代码收纳神器」，能让代码有序整理、跨项目复用，是从写零散代码到构建规范程序的关键。

本节分为两个小节：第一个小节讲模块（如何将功能整理成 .py 文件，用 import 调用），第二个小节讲包（用文件夹组织多个模块）。本 P 聚焦第一小节「模块」。

### 二、概念建立：什么是模块

**时间区间：0:57 - 2:03**

老师先回顾之前学过的函数、异常处理等知识点，指出写简单代码没问题，但项目功能多了代码就又长又乱，重复定义的函数、零散分布的变量难以维护，也无法跨项目复用。

随后给出模块的定义：

- **在 Python 中，一个 .py 文件就是一个模块。** 之前写的所有 .py 文件都可以称为模块。
- 模块就像一个专门的收纳盒，里面可以装变量、函数、类等代码。
- 举例：写一个 calc.py 文件，里面放求和、求积等计算函数，就是一个计算模块；写一个 greet.py 文件，里面放各种打招呼的函数，就是一个问候模块。分类清晰，用的时候一目了然，不用在几百行代码里翻找功能。

### 三、模块的三种类型

**时间区间：2:53 - 9:11**

老师将模块分为三类，分别讲解概念和用法。

#### 1. 内置模块

Python 安装好后自带的功能工具箱，不需要额外下载安装，导入就能直接使用。举例：

- `random` 模块：生成随机数
- `time` 模块：处理时间
- `max` 模块：做数学计算

**演示：使用 random 模块生成随机整数。**

```python
import random
nm = random.randint(1, 3)  # 生成 1~3 之间的随机整数（包前包后）
print(nm)
```

老师演示运行了多次，分别生成了 1、3、2 等结果，说明 `randint` 是包前包后的。如果要生成 1~100 之间的随机整数，改成 `randint(1, 100)` 即可。强调可以用这个模块写猜数字游戏，不用自己写生成随机数的复杂代码。

#### 2. 第三方模块

其他程序员或公司开发的实用模块（也叫第三方库），功能强大，比如：

- `requests` 模块：处理网络请求，爬取网页数据、调用接口
- `numpy` 模块：做数据分析，处理海量数据
- `matplotlib` 模块：画图表，生成折线图、柱状图

这类模块不是 Python 自带的，需要通过 **pip 工具安装** 后才能使用。

**演示：安装并使用 requests 模块。**

- 基础安装命令：`pip install 模块名`，例如 `pip install requests`。
- 老师打开命令行（Windows+R → CMD），演示执行 `pip install requests`，由于已安装过，速度很快。
- 随后演示安装 pandas（数据分析模块），说明有时下载会慢，因为官网在国外可能访问慢甚至超时。
- **推荐使用国内镜像源加速安装**，比如清华镜像源、豆瓣镜像源：

```bash
pip install 模块名 -i https://pypi.tuna.tsinghua.edu.cn/simple
```

安装完成后，就可以像内置模块一样直接导入使用。

**演示：用 requests 爬取百度页面。**

```python
import requests
response = requests.get("https://www.baidu.com")
response.encoding = "utf-8"  # 设置编码解决乱码
print(response.text)
```

老师演示运行后可以看到百度页面的文本内容，提到有乱码时可以设置 `response.encoding = "utf-8"` 来解决。

#### 3. 自定义模块

根据自己的项目需求，把常用功能写在 .py 文件里，相当于自己定制的专属工具箱。比如多个项目都需要计算 BMI、格式化输出用户信息等功能，写一次做成自定义模块，后续直接导入使用，既节省时间又保证功能统一，修改时也只需改一次，可维护性好。

**两点重要注意事项（新手避坑）：**

1. **模块名要遵循变量命名规则**：只能用字母、数字、下划线，不能以数字开头，不能用关键字。
2. **不能和 Python 内置模块、已安装的第三方模块重名。** 比如不能把自己的模块也叫 `random`，然后又在里面使用 `random` 的功能，否则会和内置模块冲突，导致内置模块功能用不了。这是新手常犯的错误。

### 四、创建并演示自定义模块

**时间区间：10:27 - 11:43**

老师开始实际演示创建自定义模块：

**1. 新建一个 mytools.py 文件，写入以下内容：**

```python
# mytools.py
model_name = "工具模块"  # 定义一个变量

def add(a, b):
    return a + b  # 求和函数

def greet(name):
    print(f"你好呀，{name}好")  # 打招呼函数
```

### 五、三种导入方式详解

**时间区间：11:44 - 19:33**

老师强调：导入模块就像打开收纳盒，使用模块里的功能就像从收纳盒里拿东西。Python 提供了三种常用导入方式，各有优缺点。

#### 方式一：`import 模块名 [as 别名]`

**语法：** `import 模块名` 或 `import 模块名 as 别名`（别名可选，用于简化长模块名）

**特点：** 导入模块的所有内容。使用时必须加 `模块名.功能名` 或 `别名.功能名` 前缀。

**优点：** 不会和当前文件的变量、函数、类冲突，适合模块功能比较多的场景。

**演示：导入 mytools 模块**

```python
import mytools
result = mytools.add(2, 3)
print(result)  # 输出 5
```

如果模块名很长，可以取别名简化：

```python
import mytools as mt
result = mt.add(2, 3)
print(result)  # 输出 5
```

#### 方式二：`from 模块名 import 功能名 [, 功能名2, ...]`

**语法：** `from 模块名 import 功能1, 功能2, ...`

**特点：** 只导入指定的功能，不导入其他没用的内容，节省资源。使用时直接写功能名，不用加模块名前缀。如果有多个功能，用逗号分隔。

**演示：只导入 add 函数和 model_name 变量**

```python
from mytools import add, model_name
result = add(1, 2)
print(result)  # 输出 3
print(model_name)  # 输出 "工具模块"
```

老师强调：如果没有导入 `greet` 函数，直接调用会报错，因为 `greet` 没有被导入进来。这说明这种方式只会导入你指定的功能。

**适用场景：** 只用到模块部分功能时，代码更简洁清爽。

#### 方式三：`from 模块名 import *`

**语法：** `from 模块名 import *`

**特点：** `*` 表示所有，导入模块里所有的功能。使用时同样直接写功能名，不用加前缀。

**演示：导入 mytools 所有功能**

```python
from mytools import *
greet("冰冰")  # 输出 "你好呀，冰冰好"
print(add(3, 4))  # 输出 7
print(model_name)  # 输出 "工具模块"
```

#### ⚠️ 方式三的风险：命名冲突

老师重点强调：虽然 `from 模块名 import *` 看似方便，但容易命名冲突。如果当前模块和导入的模块有同名的变量、函数或类，**后导入的会覆盖先导入的**，导致功能异常。

**演示命名冲突：**

```python
from mytools import *
print(model_name)  # 输出 "工具模块"

model_name = "py_model"  # 在当前文件定义同名变量
print(model_name)  # 输出 "py_model"（被覆盖了）
```

如果先定义当前文件的变量，再导入模块，结果相反。这说明顺序不同结果不同，非常容易出错。

**解决方案：** 如果两个模块有同名内容，建议使用 `import 模块名` 的方式，通过 `模块名.功能名` 来区分，避免冲突。

```python
import mytools

model_name = "py_model"  # 使用本模块的
print(model_name)  # 输出 "py_model"

print(mytools.model_name)  # 使用 mytools 模块的，输出 "工具模块"
```

老师建议：**尽量少用 `from 模块名 import *`**，除非非常确定不会和当前文件冲突。

### 六、关键技巧：`if __name__ == "__main__"`

**时间区间：19:36 - 24:05**

老师在自定义模块 `mytools.py` 中添加了一段测试代码：

```python
# 在 mytools.py 中添加
print(add(3, 5))  # 测试 add 函数是否正常
```

发现问题：**当直接运行 mytools.py 时测试代码会执行，这是正常的。但如果把 mytools 作为模块导入到其他文件（比如 py_model.py）时，测试代码也会被执行**，但测试代码只是想在本模块调试用，不应该被其他模块执行。

**解决方案：** 把测试代码放在 `if __name__ == "__main__"` 判断里：

```python
if __name__ == "__main__":
    print(add(3, 5))  # 只在直接运行本文件时执行
```

**原理讲解：**

- 每个模块都有一个内置的全局变量 `__name__`。
- 当模块**直接运行时**，`__name__` 的值是 `"__main__"`，所以 `if` 条件成立，测试代码执行。
- 当模块**被当作模块导入时**，`__name__` 的值是**模块的名称**（比如 `"mytools"`），所以 `if` 条件不成立，测试代码不执行。

老师通过打印 `__name__` 的值来演示验证了这个原理。

**作用总结：**
- 在自定义模块中写测试代码时，用 `if __name__ == "__main__"` 包裹，既能调试功能，又不影响模块被其他文件导入。
- 相当于把「隐私代码」藏起来，只让自己看，别人用不了。

### 七、导入模块的注意事项

**时间区间：24:13 - 25:38**

老师补充了三个注意事项，帮助写出更规范的代码：

1. **导入语句写在文件开头。** 方便组织和阅读代码，别人一眼就能知道用了哪些模块，不用在代码中间找半天，符合 Python 编程习惯。
2. **模块只导入一次。** 同一个模块在程序中只需导入一次，第一次导入后会加载到内存，后续再写导入语句不会重复加载，节省系统资源，避免重复导入导致的冲突和冗余。
3. **尽量避免命名冲突。** 如果两个模块有同名功能（比如 A 模块和 B 模块都有 `add` 函数），用 `A.add()` 和 `B.add()` 来区分；如果模块名太长，可以给模块取别名。另外尽量不要用 `from 模块名 import *`，很容易覆盖当前文件的变量或函数。

### 八、本 P 总结

**时间区间：25:38 - 27:24**

老师带大家回顾本节重点：

1. **模块的本质就是一个 .py 文件**，可以用来封装变量、函数、类，核心目的是实现代码的复用和模块化管理。
2. **模块分成三类：**
   - 内置模块：导入后直接使用
   - 第三方模块：pip 安装后才可导入使用
   - 自定义模块：自己写的、自己用
3. **三种导入方式各有优缺点，按需选择：**
   - `import 模块名 [as 别名]`：层级清晰、无冲突，适合复杂项目
   - `from 模块名 import 功能名`：简洁高效，适合只使用部分功能
   - `from 模块名 import *`：虽然方便，但容易冲突，尽量慎用
4. **关键技巧：** 用 `if __name__ == "__main__"` 包裹测试代码，既能调试又不会影响模块导入。
5. **注意：** 模块名不能和内置模块、第三方模块重名，否则会导致功能失效。

老师最后鼓励大家课后自己写一个自定义模块，把之前学的判断成绩等级、计算圆的面积等函数放进去，用不同方式导入使用，多多练习。学会用模块能让代码从杂乱无章变得井然有序，效率和可维护性都会大大提升，也是后续学习大型项目开发的基础。

## 本P结论

1. **模块 = .py 文件**，封装变量、函数、类，实现代码复用和模块化管理。
2. **模块分三类：** 内置模块（Python 自带，导入即用）、第三方模块（pip 安装后使用）、自定义模块（自己写的）。
3. **三种导入方式：**
   - `import 模块名 [as 别名]`：导入全部，用 `模块名.功能` 调用，无冲突，适合复杂项目
   - `from 模块名 import 功能名`：导入指定功能，用 `功能名` 直接调用，简洁，适合部分功能场景
   - `from 模块名 import *`：导入所有功能，直接调用，但易冲突，尽量少用
4. **关键技巧：** 自定义模块中的测试代码用 `if __name__ == "__main__"` 包裹，确保直接运行时执行测试、被导入时不执行。
5. **避坑提醒：** 模块名不能与内置/第三方模块重名；导入语句建议写在文件开头；同一模块只需导入一次；有同名冲突时优先用 `import` 方式通过 `模块名.` 前缀区分。
