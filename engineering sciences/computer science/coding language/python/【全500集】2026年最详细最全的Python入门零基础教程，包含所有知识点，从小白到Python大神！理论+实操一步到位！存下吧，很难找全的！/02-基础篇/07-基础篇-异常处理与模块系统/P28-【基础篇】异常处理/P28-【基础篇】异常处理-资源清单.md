# P28 资源清单 - 【基础篇】异常处理
---

## 1. Python 解释器

- **类型**：运行时环境
- **在本P中的作用**：执行所有异常处理代码的底层环境，用于运行和验证 try/except/else/finally 语法
- **来源**：字幕直接提到
- **建议方向**：在命令行中分别触发不同异常类型，观察 Python 的原始报错输出，对比有异常处理和无异常处理的程序行为差异

---

## 2. Python 内置异常类型（Built-in Exceptions）

- **类型**：标准库 / 内置类
- **在本P中的作用**：课程列举的 NameError、ValueError、ZeroDivisionError、IndexError、KeyError、TypeError、IOError、AttributeError、SyntaxError、IndentationError 等均为 Python 内置异常类，是异常捕获的目标对象
- **来源**：字幕直接提到
- **建议方向**：在交互式解释器中逐一触发这些异常，观察各自的错误描述信息，建立异常类型与错误场景的对应关系

---

## 3. Python 异常回溯机制（Traceback）

- **类型**：内置功能
- **在本P中的作用**：Python 在异常发生时输出的错误堆栈信息，包含文件路径、行号、具体代码、异常类型和描述信息，是定位问题的核心依据
- **来源**：字幕直接提到
- **建议方向**：故意制造错误，阅读并分析 Traceback 的四个组成部分（most recent call last、file 路径、LN 行号、in module），练习通过 Traceback 快速定位代码问题

---

## 4. Python 官方文档 - 内置异常

- **类型**：官方文档
- **在本P中的作用**：Python 内置异常类型的完整参考，包含异常类的继承体系图（BaseException → Exception → 具体异常类型），可查询每个异常的具体触发条件和用法
- **来源**：基于上下文推断（Python 官方文档是标准参考）
- **建议方向**：对照课程列举的异常类型，查阅官方文档中对应的详细说明，了解异常类的继承关系和更多未提及的内置异常

---

## 5. Python 官方文档 - 异常处理指南

- **类型**：官方文档
- **在本P中的作用**：关于异常处理的官方最佳实践说明，包括 try/except/else/finally 各子句的语法规范和执行流程
- **来源**：基于上下文推断
- **建议方向**：阅读官方文档中关于异常处理的完整章节，对比课程内容，补充学习异常处理的更多细节（如异常链、上下文管理器等进阶话题）

---

## 6. IDE / 代码编辑器（显示行号的工具）

- **类型**：开发工具
- **在本P中的作用**：课程强调 Traceback 中的 LN 行号信息，IDE 可以快速定位到报错代码所在行，便于根据异常回溯信息修正错误
- **来源**：基于上下文推断
- **建议方向**：使用带有行号显示功能的编辑器（如 VS Code、PyCharm）编写异常处理代码，故意制造错误后练习根据行号快速定位问题代码

---

## 7. Python traceback 模块

- **类型**：标准库模块
- **在本P中的作用**：提供对异常回溯信息的编程访问能力，可以将 Traceback 格式化为字符串或打印输出，用于日志记录和自定义错误处理
- **来源**：基于上下文推断
- **建议方向**：结合课程中的 `except Exception as e` 用法，学习使用 traceback 模块的 `print_tb()`、`format_exception()` 函数，将异常信息记录到日志文件

---

## 8. Python logging 模块

- **类型**：标准库模块
- **在本P中的作用**：课程提到程序需要给出"友好提示信息"，logging 模块是将异常信息、结构化日志输出到控制台或文件的标准化方式，比 print 更适合生产环境
- **来源**：基于上下文推断
- **建议方向**：将异常处理的 except 块与 logging 模块结合使用，学习配置不同级别的日志输出（DEBUG、INFO、ERROR），实现异常信息的规范化记录

---

## 9. Python sys 模块的 exc_info() 函数

- **类型**：标准库函数
- **在本P中的作用**：返回当前正在处理的异常信息元组 (type, value, traceback)，是 `except Exception as e` 之外另一种获取异常对象的方式，常用于需要同时获取异常类型和 Traceback 对象的场景
- **来源**：基于上下文推断
- **建议方向**：对比 `except Exception as e` 和 `sys.exc_info()` 两种方式的使用场景，理解何时需要同时获取异常类型、异常对象和完整的堆栈信息

---

## 10. Python __debug__ 内置常量

- **类型**：内置常量
- **在本P中的作用**：课程强调语法错误（SyntaxError、IndentationError）不能被 except 捕获，`__debug__` 用于控制断言语句的执行，在生产环境调试时与异常处理形成互补
- **来源**：基于上下文推断
- **建议方向**：学习 Python 的调试体系（assert 语句、__debug__、pdb 调试器），理解异常处理不是万能的，语法错误和逻辑错误需要不同的处理方式
