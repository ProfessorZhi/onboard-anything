# P29 资源清单 - 【基础篇】抛出自定义异常
---

## 核心语法资源

### 1. Exception 类

- **类型**：Python 内置类
- **在本 P 中的作用**：作为自定义异常的基类，通过 `Exception("错误描述信息")` 语法创建异常对象。本P用它来创建“余额不足”、“账户已冻结”等业务错误对象，作为学习面向对象之前自定义异常的入门写法。
- **来源**：字幕直接提到
- **后续可练习方向**：建议方向：尝试封装更多业务信息到异常对象中，例如包含错误码、用户ID、取款金额等字段；后续学习类与对象后，可将其替换为自定义异常类。

---

### 2. raise 语句

- **类型**：Python 关键字 / 控制流语句
- **在本 P 中的作用**：用于主动抛出已创建的自定义异常对象，使程序中断并输出指定的错误信息。本P演示了 `raise my_error` 的用法，替代了 print 输出错误信息的做法。
- **来源**：字幕直接提到
- **后续可练习方向**：建议方向：在其他业务场景中主动抛出异常，如用户注册时验证密码强度、订单处理时检查库存数量等。

---

### 3. try-except 异常捕获结构

- **类型**：Python 语法结构
- **在本 P 中的作用**：用于捕获被 raise 抛出的自定义异常，避免程序因异常而终止。本P演示了 `try...except Exception as e: print(e)` 的基本用法，将异常信息转化为友好的提示输出。
- **来源**：字幕直接提到
- **后续可练习方向**：建议方向：在一个完整的业务流程中嵌入 try-except，例如模拟用户注册流程，同时捕获多种不同的自定义异常，并分别给出针对性处理。

---

### 4. global 声明

- **类型**：Python 关键字
- **在本 P 中的作用**：用于在函数局部作用域中声明将要修改的是全局变量。本P演示了在取款函数 `withdraw()` 中使用 `global balance`，使余额扣减能够生效保存到全局变量。
- **来源**：字幕直接提到
- **后续可练习方向**：建议方向：在包含多个函数的程序中练习全局变量的读取与修改，理解 global 声明的必要性及适用场景。

---

## 辅助知识点

### 5. str() 类型转换

- **类型**：Python 内置函数
- **在本 P 中的作用**：在拼接异常信息字符串时，将数值类型的余额转换为字符串。本P演示了 `"余额不足，你的当前余额是" + str(balance) + "元"` 的用法。
- **来源**：字幕直接提到
- **后续可练习方向**：建议方向：在需要拼接不同类型数据的场景中练习使用 str()，避免类型错误。

---

### 6. 布尔条件判断

- **类型**：Python 语法特性
- **在本 P 中的作用**：用于简化条件判断。本P演示了 `if is_frozen:` 替代 `if is_frozen == True:` 的写法，利用布尔值本身可直接作为条件的特性简化代码。
- **来源**：字幕直接提到
- **后续可练习方向**：建议方向：在涉及开关、状态标记的代码中练习布尔条件判断，养成简洁的编码习惯。
