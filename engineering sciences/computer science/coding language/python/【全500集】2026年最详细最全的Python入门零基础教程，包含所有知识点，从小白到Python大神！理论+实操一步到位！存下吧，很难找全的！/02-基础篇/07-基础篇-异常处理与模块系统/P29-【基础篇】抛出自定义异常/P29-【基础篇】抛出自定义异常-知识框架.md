# P29 知识框架 - 【基础篇】抛出自定义异常
## 本P位置
异常处理章节 → 进阶篇 → 承接内置异常基础（P28），为后续自定义异常类（P30+）铺路

## 核心问题
如何在业务代码中精准描述和处理特定错误，而不是依赖模糊的内置异常

## 知识骨架
```
自定义异常
├── 为什么需要 → 精准描述业务错误、针对性处理、附加调试信息
├── 怎么实现 → 创建异常对象 + raise抛出
└── 怎么用 → 配合try-except捕获
```

## 关系与依赖
- 前置：异常处理基础（try/except、内置异常）
- 前置：全局变量与局部作用域（global声明）
- 后续：面向对象 → 自定义异常类替代Exception直接使用

## 输入 / 输出 / 前置 / 支撑
| 类型 | 内容 |
|------|------|
| 输入 | 业务场景（如取款金额、账户状态） |
| 输出 | 精准的业务错误提示（如"余额不足，当前余额800元"） |
| 前置 | Exception类、raise语句、try-except语法 |
| 支撑 | global声明、布尔条件判断 |

## 易错点
1. 修改全局变量未声明global
2. 布尔条件写if is_frozen == True而非if is_frozen
3. 独立错误用两个if而非if-elif（冻结和余额不足无关联）
4. 异常信息拼接忘记str()转换

## 外部资源与对象
本P无外部依赖，纯Python内置Exception类和raise语句
