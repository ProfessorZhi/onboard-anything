# P29 练习题 - 【基础篇】抛出自定义异常

## 结构识别

**Q1｜** 下列代码片段中，哪一行是"创建异常对象"，哪一行是"抛出异常"？请分别标注并说明两者缺一不可的原因。

```
e = Exception("余额不足，当前余额800元")
raise e
```

> 提示：对照知识骨架"怎么实现"一节作答，不超过3句话。

---

**Q2｜** 以下两段代码的结构差异是什么？各自适用于什么场景？

```python
# 版本A
if is_frozen:
    raise Exception("账户已冻结")
elif balance < amount:
    raise Exception("余额不足")

# 版本B
if is_frozen:
    raise Exception("账户已冻结")
if balance < amount:
    raise Exception("余额不足")
```

> 核心考点：账户冻结与余额不足是否存在逻辑关联，决定用 if-elif 还是两个独立 if。

---

## 机制理解

**Q3｜** 阅读以下代码，指出两处错误并说明会导致什么后果。

```python
balance = 1000
is_frozen = False

def withdraw(amount):
    if is_frozen == True:
        raise Exception("账户已冻结")
    if balance < amount:
        raise Exception("余额不足，当前余额" + balance + "元")
    balance -= amount
```

> 对照易错点1、3、4作答，每处错误写明：错误描述 → 后果 → 修正方式。

## 最小实践

**Q5｜** 独立编写一个 `login(username, password)` 函数，满足以下要求：

- 预设正确用户名为 `"admin"`，正确密码为 `"123456"`
- 用户名错误时，抛出异常，信息为 `"用户名不存在：<实际输入值>"`
- 密码错误时，抛出异常，信息为 `"密码错误"`
- 两种错误用两个独立的 `if`（不用 elif）
- 在函数外用 `try-except` 捕获并打印异常信息

完成后，用以下三组输入分别测试并记录输出：

| 测试用例 | username | password | 预期结果 |
|----------|----------|----------|----------|
| 1 | `"root"` | `"123456"` | 抛出用户名异常 |
| 2 | `"admin"` | `"000000"` | 抛出密码异常 |
| 3 | `"admin"` | `"123456"` | 正常登录 |
