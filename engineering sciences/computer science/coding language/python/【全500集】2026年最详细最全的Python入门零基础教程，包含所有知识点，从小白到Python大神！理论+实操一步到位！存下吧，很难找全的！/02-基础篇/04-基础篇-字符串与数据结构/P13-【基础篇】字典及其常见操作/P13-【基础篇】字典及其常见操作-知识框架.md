# P13 知识框架 - 【基础篇】字典及其常见操作
## 本P位置

Python基础篇第4小节，数据类型核心知识体系的第三种容器类型（列表→元组→字典）。承接列表索引查找的效率痛点，引出键值对映射结构的查询优势。为后续文件操作、JSON处理、API返回数据解析、配置管理等场景提供数据结构基础。

## 核心问题

1. 字典如何通过键实现高效查找？
2. 键的唯一性约束与值的灵活性之间是什么关系？
3. 为什么字典比列表更适合存储关联数据？
4. 字典的不安全查找（dict[key]）与安全查找（dict.get()）差异在哪里？

## 知识骨架

**字典 = 有序的键值对容器**

```
dict = {key1: value1, key2: value2, ...}
```

**核心特征**
- 键：唯一、不可变（可用元组，不可使用列表/字典/集合）
- 值：任意类型，无限制

**操作体系**

| 操作 | 语法 | 副作用 |
|------|------|--------|
| 安全查找 | dict.get(key, default) | 不存在返回default或None，无异常 |
| 不安全查找 | dict[key] | 不存在抛出KeyError |
| 添加/修改 | dict[key] = value | 键存在改值，不存在新增 |
| 删除并取值 | dict.pop(key, default) | 返回被删值，键不存在返回default或抛异常 |
| 清空 | dict.clear() | 保留空字典对象 |
| 获取键 | dict.keys() | 返回dict_keys视图对象 |
| 获取值 | dict.values() | 返回dict_values视图对象 |
| 获取键值对 | dict.items() | 返回dict_items，元素为元组 |

**数据结构地位**
```
序列：列表（可变）、元组（不可变）
映射：字典（可变）
```

## 关系与依赖

**前置依赖**：列表的基本使用（列表是有序但无键的容器，字典是有键的容器）
**对比关系**：字典 vs 列表 → 键索引 vs 位置索引
**后续支撑**：JSON序列化（Python dict ↔ JSON object）、字典遍历（for循环 + items()）、字典推导式
**逻辑关联**：字典的键不可变约束 → 元组可作为键 → 列表不可作为键（哈希约束）

## 输入 / 输出 / 前置 / 支撑

**输入**：键值对数据 {key: value}
**输出**：dict类型对象；各类视图对象（dict_keys/dict_values/dict_items）
**前置知识**：列表定义与索引访问、元组不可变特性
**支撑知识**：Python内置hash机制（理解为什么键必须不可变）、对象引用概念（字典存储的是对象引用而非副本）

## 易错点

- dict[key] 查不到键时直接抛KeyError，线上环境易导致程序崩溃
- 键重复时后者覆盖前者，不会报错
- dict.clear() 是清空内容而非删除变量，变量仍存在
- dict_keys/dict_values/dict_items 不是列表，不可直接索引，需用list()转换
- 字典定义时键不能使用列表、字典、集合，但可以嵌套列表或字典作为值
- dict.pop() 若不设默认值且键不存在，同样抛KeyError
