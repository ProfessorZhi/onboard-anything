# P13 讲解过程 - 【基础篇】字典及其常见操作
## 本P定位

本P是Python基础篇的第四小节，属于**数据类型核心知识体系**中的重要组成部分。在列表、元组之后引入字典，旨在帮助学习者理解**键值对映射结构**的数据组织方式，并为后续学习数据处理、配置管理、JSON应用等场景奠定基础。

## 讲解过程

### 一、引入字典概念

**通过生活类比建立认知**（字幕1.2-51.9秒）

老师首先指出字典是Python中一个非常灵活的**键值对数据容器**。用现实中的字典做类比：现实中查字典不需要一页一页翻，只需要通过**关键词**就能快速找到对应的解释。这里的"关键词"就相当于字典里的**键（key）**，对应的"解释"就是字典里的**值（value）**。

老师强调这种查找效率特别高，适合存储**用户信息、配置参数、处理映射关系**等场景。学会运用字典能够显著提升关联数据的查询效率，规范数据存储的逻辑。

### 二、字典的定义格式与基本规则

**明确数据结构特征**（字幕55.4-106.0秒）

老师说明字典是专门存储**键值对**的有序数据结构。每一个键都有它唯一的标志，就像字典里的一个词条对应一个解释，字典里的键也对应一个值。而且**值非常灵活**，不管是数字、字符串、列表，甚至另一个字典，都能够作为值进行存储。

定义格式使用**大括号**把键值对包裹起来，**键和值之间使用冒号分隔**，不同的键值对之间使用**逗号分隔**。基本格式为：`{key1: value1, key2: value2, ...}`

### 三、实例对比：列表存储 vs 字典存储

**通过对比凸显字典优势**（字幕106.0-270.3秒）

老师先用列表存储用户信息：

```python
info = ['冰冰', '女', 98, 87, 96]
```

指出问题：冰冰知道是姓名，女知道是性别，但"98、87、96"不知道是什么意思——可能是语文数学英语成绩，也可能是Python、HTML、Mexico的科目成绩。**可读性不够好，数据存储的清晰度不够**。

然后改用字典定义同样的信息：

```python
info = {
    'name': '冰冰',
    'sex': '女',
    'python': 98,
    'html': 87,
    'mexico': 96
}
```

老师解释这里体现了**映射关系**——"映射"就是对应的意思。比如"name"这个键对应"冰冰"这个值，"python"这个键对应98这个值。这种对应关系使得数据存储逻辑更清晰。

运行代码验证数据类型为`dict`（dictionary的缩写，意为字典）。

### 四、值的灵活性演示

**演示值可以包含任意数据类型**（字幕290.9-339.9秒）

老师补充说明值可以是任意数据类型。为字典添加一个"爱好"字段，值为**列表**：

```python
info['hobby'] = ['写代码', '唱歌', '追剧', '刷微博', '刷抖音']
```

运行验证没有报错，说明值的类型完全没有限制。

### 五、字典定义的两个重要注意点

**强调键的唯一性与值的灵活性**（字幕345.4-484.9秒）

#### 注意点1：键必须唯一，不允许重复

老师演示创建重复键的字典：

```python
demo = {'name': '冰冰', 'name': '小明'}
```

运行后发现**后面的值会覆盖前面的值**，输出结果只有`{'name': '小明'}`。老师用变量重复定义做类比：

```python
name = '冰冰'
name = '小明'
print(name)  # 输出小明
```

变量重复定义会覆盖，字典键重复也是同样的道理。

#### 注意点2：值没有唯一性要求

老师演示不同键可以对应相同的值：

```python
info['nickname'] = '冰冰'
```

运行后没有问题，说明**值是允许重复的，但键必须具备唯一性**。同时再次强调值的类型没有限制，数值、字符串、列表等都可以。

### 六、字典的查找操作

**对比两种查找方式的优劣**（字幕495.4-694.7秒）

#### 方式1：直接通过键取值 `dict[key]`

老师演示：

```python
info = {'name': '冰冰', 'sex': '女', 'python': 98, 'html': 87, 'mexico': 96}
print(info['python'])  # 输出98
```

存在该键时正常返回对应的值。但如果查找不存在的键：

```python
print(info['chinese'])  # 报错：KeyError
```

老师指出这种方式**找不到时会报错**，因此**不推荐使用**。

#### 方式2：使用get方法 `dict.get(key, default)` 【推荐】

老师演示：

```python
print(info.get('python'))  # 输出98
print(info.get('chinese'))  # 输出None，不报错
```

**get方法的核心机制**：
- 如果能找到对应的键，返回键对应的值
- 如果没有找到，返回设置的默认值
- 如果没有设置默认值，**默认返回None而不会报错**

老师进一步演示自定义默认值：

```python
print(info.get('chinese', '该键名不存在'))  # 输出：该键名不存在
```

老师强调使用get方法取值**能够避免程序因找不到键而崩溃**，建议优先使用。

### 七、添加或修改元素

**同一个语法实现两种操作**（字幕698.7-831.6秒）

老师说明添加和修改使用**同一个语法**：`dict[key] = value`

具体是添加还是修改，取决于键在字典中是否已存在：
- 键已存在 → 执行修改操作
- 键不存在 → 在字典中添加新的键值对

#### 修改示例：

```python
info = {'name': '冰冰', 'age': 18}
info['age'] = 28
print(info)  # {'name': '冰冰', 'age': 28}
```

原本age是18，修改后变成28。

#### 添加示例：

```python
info['tel'] = '1301111222'
print(info)  # {'name': '冰冰', 'age': 28, 'tel': '1301111222'}
```

原本只有两个键值对，执行后变成了三个键值对。

### 八、删除元素

**演示两种删除方式的差异**（字幕835.5-1050.4秒）

#### pop方法：`dict.pop(key, default)`

老师解释pop方法**删除指定的键值对**，并返回被删除的值。如果指定的键不存在，则返回设定的默认值；如果没有设定默认值，会报错。

##### 删除存在的键：

```python
info = {'name': '冰冰', 'age': 28}
result = info.pop('age')
print(result)  # 输出28，即被删除的值
print(info)    # 输出{'name': '冰冰'}，键值对已删除
```

##### 删除不存在的键（不设默认值）：

```python
info.pop('tel')  # 报错：KeyError，程序崩溃
```

##### 删除不存在的键（设置默认值）：

```python
result = info.pop('tel', '该键不存在')
print(result)  # 输出：该键不存在，程序继续运行
```

#### clear方法：`dict.clear()`

老师说明clear方法用于**清空字典里所有的元素**：

```python
info = {'name': '冰冰', 'age': 28}
info.clear()
print(info)  # 输出{}，空字典
```

字典仍然保留，可以继续进行添加、修改、删除等操作。

### 九、获取字典信息的方法

**为后续遍历字典做准备**（字幕1052.5-1199.3秒）

老师介绍三个获取字典信息的方法，并强调**在后续学习循环时非常有用**。

#### keys()方法：获取所有键

```python
info = {
    'name': '冰冰',
    'sex': '女',
    'python': 98,
    'html': 87,
    'mexico': 96,
    'hobby': ['写代码', '唱歌', '追剧', '刷微博', '刷抖音']
}
print(info.keys())
# 输出：dict_keys(['name', 'sex', 'python', 'html', 'mexico', 'hobby'])
```

返回的是特殊的`dict_keys`类型，不是列表也不是元组。

#### values()方法：获取所有值

```python
print(info.values())
# 输出：dict_values(['冰冰', '女', 98, 87, 96, ['写代码', '唱歌', '追剧', '刷微博', '刷抖音']])
```

#### items()方法：获取所有键值对

```python
print(info.items())
# 输出：dict_items([('name', '冰冰'), ('sex', '女'), ('python', 98), ('html', 87), ('mexico', 96), ('hobby', ['写代码', '唱歌', '追剧', '刷微博', '刷抖音'])])
```

老师指出每个键值对以**元组形式**存在。元组中的第一个元素是键，第二个元素是对应的值。

### 十、课堂总结

**梳理核心知识点与实际应用**（字幕1204.1-1356.5秒）

老师进行系统性总结：

#### 字典的核心特征
- 本质是有序的键值对容器（key-value）
- **键必须唯一且不可变**（不能用列表、字典、集合作为键，但**元组可以作为键**）
- **值可以是任意类型**，没有限制

#### 核心操作速记
| 操作 | 方法 | 说明 |
|------|------|------|
| 查找 | `dict.get(key)` | **推荐**，安全不报错 |
| 查找 | `dict[key]` | 不存在时报错，不推荐 |
| 增改 | `dict[key] = value` | 键存在则修改，不存在则添加 |
| 删除 | `dict.pop(key)` | 删除指定键值对并返回值 |
| 清空 | `dict.clear()` | 清空所有元素，保留空字典 |
| 获取键 | `dict.keys()` | 返回所有键 |
| 获取值 | `dict.values()` | 返回所有值 |
| 获取键值对 | `dict.items()` | 返回所有键值对（元组形式） |

#### 字典的核心优势
- 通过键快速查找值，**查询效率非常高**
- 不用像列表那样按顺序依次查找
- **特别适合存储关联数据**（如用户姓名、年龄、爱好等有映射关系的信息）
- 数据逻辑更清晰，一看到键名就知道对应的值是什么

#### 学习建议
老师建议课后自己**用字典存储个人信息**，练习查找、增加、修改、删除等操作，多动手就能熟练掌握。

## 本P结论

1. **字典是Python中存储键值对的有序数据结构**，通过键可以快速查找对应的值，查询效率远高于列表的顺序遍历。

2. **字典的定义格式**为`{key1: value1, key2: value2, ...}`，键值对之间用逗号分隔，键与值之间用冒号分隔。

3. **键必须唯一且不可变**（不能使用列表、字典、集合作为键，但元组可以），**值可以是任意数据类型**，没有限制。

4. **字典的核心操作**包括：查找（推荐使用`.get()`方法）、增改（`dict[key] = value`）、删除（`.pop()`或`.clear()`）、以及获取键/值/键值对的辅助方法（`.keys()`、`.values()`、`.items()`）。

5. **字典适用于存储关联数据**（如用户信息、配置参数、映射关系），能够规范数据存储逻辑、提升程序可读性，建议在遇到此类场景时优先考虑使用字典。
