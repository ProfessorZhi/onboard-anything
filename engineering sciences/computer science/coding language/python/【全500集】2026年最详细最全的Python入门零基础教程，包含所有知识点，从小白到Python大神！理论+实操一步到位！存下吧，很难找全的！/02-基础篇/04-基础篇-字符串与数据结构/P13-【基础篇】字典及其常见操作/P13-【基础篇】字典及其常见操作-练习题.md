## 2. 机制理解

**题目 2.1**  
解释 `dict[key]` 与 `dict.get(key)` 的核心区别是什么？什么场景下必须使用后者？

**答案要点：**  
① `dict[key]` 查找失败时抛出 `KeyError`；`dict.get(key)` 查找失败时返回 `None` 或指定的默认值。  
② 在不确定键是否存在的场景（如处理外部输入、用户数据、API 返回），必须用 `get()` 避免程序崩溃。

## 3. 最小实践

**题目 3.1**  
完成以下操作：

```python
scores = {"语文": 85, "数学": 92, "英语": 78}

# (1) 安全地获取 "物理" 成绩，若不存在返回 0
score_phys = ___________

# (2) 将 "语文" 成绩修改为 90
_________

# (3) 删除 "英语" 键值对，并获取其值
eng_score = ___________

print(scores)
```

**参考答案：**

```python
scores = {"语文": 85, "数学": 92, "英语": 78}

# (1) 安全地获取 "物理" 成绩，若不存在返回 0
score_phys = scores.get("物理", 0)

# (2) 将 "语文" 成绩修改为 90
scores["语文"] = 90

# (3) 删除 "英语" 键值对，并获取其值
eng_score = scores.pop("英语")

print(scores)
```

**题目 4.2**  
如果需要存储多个人员的信息，每个人包含姓名和年龄，以下哪种结构更合适？为什么？

```python
# 结构 A
data_a = [["Alice", 20], ["Bob", 25], ["Charlie", 22]]

# 结构 B
data_b = {"Alice": 20, "Bob": 25, "Charlie": 22}
```

**参考答案：**  
结构 B 更合适。原因：  
1. 用姓名作为键，可以直接通过 `data_b["Alice"]` 快速获取某人的年龄，无需遍历。  
2. 代码可读性更好，`data_b["Alice"]` 比 `data_a[0][1]` 语义清晰。  
3. 若需要按姓名查找或去重，字典的键唯一性约束天然支持。

**题目 4.3**  
思考：为什么 Python 要求字典的键必须是**不可变**类型？

**答案要点：**  
字典底层使用哈希表（Hash Table）实现，哈希函数要求输入值在程序运行期间保持不变才能产生稳定的哈希值，从而保证查找路径一致。若键是可变的（如列表），其哈希值可能改变，导致无法再通过原键找到对应值，破坏字典的数据完整性。
