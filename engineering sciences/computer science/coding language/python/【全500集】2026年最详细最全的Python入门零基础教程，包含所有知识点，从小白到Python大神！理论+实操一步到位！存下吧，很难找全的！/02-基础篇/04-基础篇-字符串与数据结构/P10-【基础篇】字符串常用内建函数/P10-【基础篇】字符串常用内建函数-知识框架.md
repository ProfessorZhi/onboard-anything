# P10 知识框架 - 【基础篇】字符串常用内建函数
## 本P位置

Python入门课程 → 基础篇 → 字符串章节 → 字符串内建函数小节

字符串章节最后一P，承上启下：前置知识涵盖字符串定义、切片、格式化；本P聚焦字符串内建函数；后续延伸列表、字典等数据结构。

## 核心问题

如何高效操作字符串（查找、修改、判断）？字符串与字节串如何相互转换？

## 知识骨架

```
字符串内建函数
├── 查找类 → 返回位置或计数
│   ├── find()    → 下标 / -1
│   ├── index()   → 下标 / 报错
│   └── count()   → 整数
├── 修改类 → 返回新字符串（原字符串不变）
│   ├── replace() → 新字符串
│   ├── split()   → 列表
│   ├── strip()   → 新字符串
│   ├── lower()   → 新字符串
│   └── upper()   → 新字符串
├── 判断类 → 返回布尔值
│   ├── startswith() → bool
│   ├── endswith()   → bool
│   ├── isupper()    → bool
│   └── islower()    → bool
└── 编码解码 → 字符串 ↔ 字节串
    ├── encode() → bytes（默认UTF-8）
    └── decode() → str
```

## 关系与依赖

| 关系类型 | 说明 |
|---------|------|
| 横向并列 | 查找/修改/判断三大类函数相互独立，可组合使用 |
| 纵向依赖 | 修改类函数的返回值（新字符串）可继续调用其他内建函数，形成链式调用 |
| 功能对比 | find()与index()功能相同，区别仅在于找不到时的返回值（-1 vs 报错） |
| 互为逆操作 | encode() ↔ decode() 构成完整的编码解码闭环 |
| 类型约束 | split()返回列表；isupper()/islower()只检测字母字符，中文字符不参与判断 |

## 输入 / 输出 / 前置 / 支撑

**输入**：字符串对象（str类型）

**输出**：
- 查找类 → int（位置或计数）
- 修改类 → str（返回新字符串）
- 判断类 → bool
- encode/decode → bytes / str

**前置依赖**：
- 字符串定义与索引切片
- 转义字符（\n、\t）
- f-string格式化

**支撑概念**：
- 字符串不可变（immutable）→ 修改类函数均返回新字符串，不修改原对象
- UTF-8编码 → encode()默认格式
- bytes类型 → 字节串的表示形式，带`b`前缀
- 列表类型 → split()返回值，下一章节详述

## 易错点

1. **find()与index()混淆**：功能完全相同，唯一区别在于找不到时的行为——find返回-1，index抛出ValueError。

2. **修改类函数返回值未捕获**：调用replace()、split()等方法后，若不赋值给变量或直接print，结果不会显示。

3. **split()分隔符默认值**：不指定分隔符时按空白字符（空格、\n、\t）分割，不是按空格分割。

4. **strip()范围误判**：只去除首尾连续字符，中间字符不受影响。

5. **isupper()/islower()范围过窄**：只检测字母字符；中文不参与判断，"冰冰".islower()返回True是因为没有大写字母。

6. **中文编码字节量**：一个中文字符在UTF-8下占3字节，字符串长度不等于字节数。

| 对象 | 角色 | 关联 |
|------|------|------|
| UTF-8 | encode()默认编码格式 | 与encode()深度绑定 |
| bytes类型 | 字节串数据类型 | encode()输出、decode()输入 |

| f-string | 字符串格式化 | 前置知识，非本P外部资源 |
