# P14 知识框架 - 【基础篇】集合及其常见操作
## 本P位置

Python基础篇第14小节。位于「列表、元组、字典」之后，「类型转换」之前。起承上启下作用：用已学容器知识引出集合，预告后续类型转换。

## 核心问题

集合如何在「无序」约束下实现「自动去重」，以及如何利用这一特性简化去重和数据对比逻辑。

## 知识骨架

```
集合（Set）
├── 定义语法：{} 包裹元素，逗号分隔
├── 空集合创建：set()（非 {}）
├── 三大核心规则
│   ├── 元素唯一 → 自动去重
│   ├── 元素无序 → 无索引/下标
│   └── 元素不可变 → int/str/tuple 可入，list/dict/set 不可入
├── 增操作
│   ├── add(x)：整体添加一个元素
│   └── update(iterable)：拆分可迭代对象后逐一添加
├── 删操作
│   ├── remove(x)：不存在则报错
│   ├── discard(x)：不存在不报错（推荐）
│   └── clear()：清空，保留空集合对象
└── 集合运算
    ├── 交集：& / intersection()
    └── 并集：| / union()
```

## 关系与依赖

- **前置依赖**：列表、元组、字典的「容器」概念
- **后续支撑**：类型转换（list ↔ set 互转）
- **应用场景**：去重（核心）、数据对比（交集/并集）
- **与字典区分**：{} 默认创建字典，集合必须有 set()

## 输入 / 输出 / 前置 / 支撑

| 类型 | 内容 |
|------|------|
| 输入 | 可迭代对象（字符串、列表、元组、字典、集合） |
| 输出 | 集合类型；集合运算返回新集合 |
| 前置 | 列表/元组/字典基础、for 循环（对 update 理解有帮助） |
| 支撑 | set() 内建函数 |

## 易错点

- **空集合陷阱**：`{}` 创建的是空字典，必须用 `set()` 创建空集合
- **无索引访问**：`s[0]` 报错，集合不支持按位置查找
- **remove vs discard**：删除不存在元素时 remove 报错，discard 不报错
- **update 行为**：将可迭代对象拆分后逐一添加，不是整体添加
- **元素类型限制**：列表、字典、集合本身不能作为集合元素
