# P12 知识框架 - 【基础篇】元组及其常见操作
## 本P位置

Python基础篇 · 数据容器章节 · 第12节
前序：P11 列表及其常见操作
后续：P13 字典及其常见操作
角色：引入第二种有序数据容器，对比列表建立"可变 vs 不可变"的选型直觉

## 核心问题

元组与列表结构几乎相同，为什么要单独存在？
答案：不可变性 → 数据安全 + 执行效率。适合存储固定不变的数据。

## 知识骨架

```
元组（tuple）
├── 定义语法
│   ├── 圆括号 () 包裹，逗号分隔
│   └── 单元素必须加尾部逗号（坑点）
├── 索引与切片
│   ├── 与列表/字符串规则完全一致
│   └── 规则：包前不包后，步长控制方向
├── 核心特性：不可修改
│   ├── 增、删、改 → 报错（TypeError）
│   └── 只支持查询操作
└── 查询操作
    ├── in / not in → 成员判断（返回bool）
    ├── index() → 首次出现位置（不存在报错）
    └── count() → 出现次数（不存在返回0）
```

## 关系与依赖

依赖前置知识：列表的索引切片语法（规则完全复用）
被后续依赖：函数多返回值、字典键、配置常量等场景
对比关系：与列表构成"不可变 vs 可变"的二元选型

## 输入 / 输出 / 前置 / 支撑

前置：列表定义、索引切片基础（规则可迁移）
输入：任意Python对象组合
输出：tuple类型变量、查询结果（bool/int）
支撑：Python内置tuple类型，无需第三方库

## 易错点

单元素元组不加逗号 → 被识别为括号表达式，类型取决于元素本身
index()元素不存在 → 抛出ValueError，不是返回-1
count()元素不存在 → 返回0，不会报错
切片越界 → 不报错，返回空元组（与索引不同）
