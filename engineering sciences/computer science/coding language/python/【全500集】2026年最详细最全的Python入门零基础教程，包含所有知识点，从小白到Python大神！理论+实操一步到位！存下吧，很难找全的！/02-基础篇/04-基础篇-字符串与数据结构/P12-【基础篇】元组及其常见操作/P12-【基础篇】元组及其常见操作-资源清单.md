## 一、内置类型

### 1. tuple（元组类型）

| 字段 | 内容 |
|------|------|
| 名称 | `tuple` |
| 类型 | Python 内置数据类型 |
| 本P作用 | 本P的核心学习对象。以圆括号 `()` 定义，存储有序、不可修改的数据序列；与 `list` 形成"不可变 vs 可变"的对比选型关系 |
| 来源 | 基于上下文推断（Python 标准语言内置，无需导入） |
| 建议方向 | 用 `tuple` 存储程序中不应被修改的常量数据（如星期名称、方向枚举、坐标对），与 `list` 版本做对比，观察修改操作的报错差异；查阅官方文档 [docs.python.org/3/library/stdtypes.html#tuple](https://docs.python.org/3/library/stdtypes.html#tuple) 了解完整行为规范 |

## 三、内置类型方法

### 4. tuple.index()

| 字段 | 内容 |
|------|------|
| 名称 | `tuple.index(value)` |
| 类型 | tuple 内置实例方法 |
| 本P作用 | 查找指定元素在元组中第一次出现的索引位置；演示了存在时返回整数索引，不存在时抛出 `ValueError`（课堂以注释方式标注报错，未实际运行） |
| 来源 | 基于上下文推断（字幕代码演示直接使用） |
| 建议方向 | 练习 `index()` 的完整签名 `index(value, start, stop)`，掌握限定查找范围的用法；结合 `try/except ValueError` 实现安全查找，避免程序因元素不存在而崩溃 |

### 5. tuple.count()

| 字段 | 内容 |
|------|------|
| 名称 | `tuple.count(value)` |
| 类型 | tuple 内置实例方法 |
| 本P作用 | 统计指定元素在元组中出现的次数；演示了存在时返回正整数，不存在时返回 `0`（与 `index()` 不同，不报错） |
| 来源 | 基于上下文推断（字幕代码演示直接使用） |
| 建议方向 | 构造含有重复元素的元组（如成绩序列），使用 `count()` 统计某分段出现频次；对比 `list.count()` 的行为，确认接口一致性 |

## 四、语言语法特性

### 6. 索引操作（Index Operator）

| 字段 | 内容 |
|------|------|
| 名称 | 下标索引 `t[i]` |
| 类型 | Python 语言语法特性（序列通用） |
| 本P作用 | 通过正整数索引（从 `0` 起）和负整数索引（从 `-1` 起）访问元组中指定位置的元素；演示了越界索引触发 `IndexError` |
| 来源 | 基于上下文推断（与列表/字符串规则复用，未重新讲解原理） |
| 建议方向 | 练习正负索引的对称关系（`t[0]` 与 `t[-len(t)]` 等价）；刻意触发 `IndexError` 并用 `try/except` 捕获，理解边界处理模式 |

### 7. 切片操作（Slice Operator）

| 字段 | 内容 |
|------|------|
| 名称 | 切片语法 `t[start:stop:step]` |
| 类型 | Python 语言语法特性（序列通用） |
| 本P作用 | 从元组中截取子序列；演示了正步长切片 `t[3:9:3]` 和负步长切片 `t[-1:-10:3]`，结合"包前不包后"原则逐步拆解计算过程 |
| 来源 | 基于上下文推断（规则与列表、字符串完全一致，本P直接复用） |
| 建议方向 | 练习省略 `start`/`stop` 的缺省切片（`t[::-1]` 反转）；对比切片越界（`t[0:100]`）不报错与索引越界报错的行为差异；使用 `slice` 对象封装切片参数，了解切片的对象化表示 |

### 8. 成员运算符（Membership Operators）

| 字段 | 内容 |
|------|------|
| 名称 | `in` / `not in` |
| 类型 | Python 语言运算符（序列通用） |
| 本P作用 | 判断某值是否为元组的**直接成员元素**（整体匹配，非子串包含）；演示了 `'星期' in weekdays` 返回 `False`，澄清"包含"与"成员"的语义区别 |
| 来源 | 基于上下文推断（字幕代码演示直接使用） |
| 建议方向 | 对比 `in` 在字符串（子串匹配）和元组（成员匹配）中的不同语义，构造混淆案例加深理解；进一步了解 `__contains__` 协议，理解 `in` 的底层调用机制 |

### 9. 单元素元组尾部逗号语法

| 字段 | 内容 |
|------|------|
| 名称 | 单元素元组定义语法 `(value,)` |
| 类型 | Python 语言语法细节 |
| 本P作用 | 本P重点强调的高频坑点：`(1)` 被解析为带括号的整数表达式，`(1,)` 才是真正的单元素元组；通过 `type()` 对比演示直接暴露差异 |
| 来源 | 字幕直接提到（课堂重点强调，多次演示） |
| 建议方向 | 练习用 `type()` 验证 `(True,)`、`('hello',)`、`(None,)` 等各类型单元素元组；尝试不带括号的单元素元组 `1,`（也合法），理解逗号才是元组的本质标志而非括号 |

## 五、异常类型（作为学习对象）

### 10. TypeError

| 字段 | 内容 |
|------|------|
| 名称 | `TypeError: 'tuple' object does not support item assignment` |
| 类型 | Python 内置异常类 |
| 本P作用 | 尝试对元组执行赋值修改（`t[1] = 10`）时抛出，用于直接验证元组"不可修改"的核心特性 |
| 来源 | 基于上下文推断（演示中直接运行报错） |
| 建议方向 | 了解 `TypeError` 的通用触发场景（类型不支持某操作）；尝试对元组执行 `append`、`remove`、`del` 等操作，观察各自的报错信息，区分 `TypeError` 与 `AttributeError` |

### 11. IndexError

| 字段 | 内容 |
|------|------|
| 名称 | `IndexError: tuple index out of range` |
| 类型 | Python 内置异常类 |
| 本P作用 | 使用越界索引访问元组元素时抛出，用于演示索引边界限制 |
| 来源 | 基于上下文推断（演示中故意触发） |
| 建议方向 | 对比索引越界（报错）与切片越界（返回空元组，不报错）的行为差异，理解两者设计逻辑的不同 |

### 12. ValueError（index 方法相关）

| 字段 | 内容 |
|------|------|
| 名称 | `ValueError` |
| 类型 | Python 内置异常类 |
| 本P作用 | 调用 `tuple.index()` 查找不存在的元素时抛出；本P以注释形式提示会报错，未实际演示捕获处理 |
| 来源 | 基于上下文推断（课堂提及但未展开） |
| 建议方向 | 用 `try/except ValueError` 包裹 `index()` 调用，实现"找到返回索引，找不到返回 -1"的安全封装；对比 `list.index()` 的相同行为，确认异常类型一致 |
