# P24 知识框架 - 【基础篇】函数参数：给工具传“原材料”
## 本P位置
Python 函数进阶篇章。承接 P23 函数定义与调用，引入"参数"机制解决函数复用性不足的核心痛点。是后续高阶函数（返回值、嵌套函数、装饰器等）的基础铺垫。

## 核心问题
函数如何接收外部数据？不同传参方式解决什么问题？

## 知识骨架

**参数机制 = 形参（定义端）+ 实参（调用端）+ 传参（赋值过程）**

```
实参 ──传参──▶ 形参（函数体内使用）
```

**四种参数类型（按复杂度递增）**

| 类型 | 语法 | 打包结果 | 适用场景 |
|------|------|----------|----------|
| 位置参数 | 无特殊标记 | 不打包 | 参数个数和含义固定的场景 |
| 默认参数 | `param=value` | 不打包 | 大多数情况下有确定值的场景 |
| `*args` | `*args` | 元组 | 参数数量不确定的加法类场景 |
| `**kwargs` | `**kwargs` | 字典 | 可选键值对配置场景 |

## 关系与依赖

```
形参 ←───── 传参（赋值）
 │                    │
 │  ←──位置参数───│───┘
 │       │
 │  ←──默认参数──┘（必须置于位置参数右侧）
 │
 ├─── *args（接收剩余位置参数）
 │
 └─── **kwargs（接收关键字参数）
```

**参数优先级**：位置参数 > 默认参数 > `*args` > `**kwargs`

## 输入 / 输出 / 前置 / 支撑

| 概念 | 说明 |
|------|------|
| 输入 | 实参（调用时传入的具体值） |
| 输出 | 函数体基于形参计算的结果 |
| 前置知识 | P23 函数定义与调用（def、函数体、函数调用） |
| 支撑知识 | f-string 格式化、`for` 循环、字典与元组基础 |

## 易错点

1. **位置参数个数不匹配**：少传或超传均报 `TypeError`
2. **默认参数置于位置参数左侧**：语法错误 `SyntaxError`，因会产生歧义
3. **`*args` 与 `**kwargs` 的参数名可改，但星号不可省**：`*numbers` 和 `**config` 同样有效
4. **`**kwargs` 必须用 `key=value` 形式传参**：直接传值会报 `TypeError`
5. **可变参数函数的可选参数均为空时**：`*args` 生成空元组 `()`，`**kwargs` 生成空字典 `{}`

## 外部资源与对象

| 资源 | 关联方式 |
|------|----------|
| Python 内置 `sum` 函数 | 自定义函数命名时需避免冲突（本 P 取名 `sum_nums`） |
| `print()` 的 `sep`、`end` 参数 | 作为默认参数的实际应用参照物 |

---

本 P 结论：参数是函数与外部数据交互的接口，形参定义入口，实参提供数据，四种类型覆盖了从固定到灵活的不同传参需求。
