# P23 资源清单 - 【基础篇】函数基础

> **说明**：本P是纯概念与语法入门课，字幕及讲解过程均明确指出未涉及任何外部平台、第三方库或工具。以下清单仅列出课程实际使用的 Python 内置资源与语言机制，来源已逐条注明，不作额外编造。

---

## 一、语言关键字

### 1. `def`

| 字段 | 内容 |
|------|------|
| **名称** | `def` |
| **类型** | Python 语言关键字 |
| **在本P中的作用** | 函数定义的起始标志，语法为 `def 函数名(参数列表):` ，是本P讲解的核心语法元素 |
| **来源** | 字幕直接提到；课程说明其为 define 的简写 |
| **建议方向** | 熟练默写函数定义的完整骨架（含冒号、缩进、文档字符串、return 占位）；对照 Python 官方关键字列表确认 `def` 不可用作变量名 |

---

## 二、内置函数

### 2. `print()`

| 字段 | 内容 |
|------|------|
| **名称** | `print()` |
| **类型** | Python 内置函数（built-in function） |
| **在本P中的作用** | 作为函数体内的执行语句，用于演示函数被调用后确实产生输出；也是本P唯一的可观测副作用，用于验证函数是否被执行 |
| **来源** | 字幕直接提到 |
| **建议方向** | 在自定义函数体中替换不同的 `print` 内容，观察多次调用时输出次数与调用次数的对应关系；对比"只定义不调用"与"调用两次"的输出差异 |

---

## 三、语言运行环境

### 3. Python 解释器

| 字段 | 内容 |
|------|------|
| **名称** | Python 解释器（Python Interpreter） |
| **类型** | 运行环境 / 工具 |
| **在本P中的作用** | 执行函数定义与调用的演示代码；老师通过"运行程序"验证函数是否输出、复现 `NameError` 报错，解释器是所有演示的执行载体 |
| **来源** | 基于上下文推断（课程全程依赖本地解释器运行代码，未说明具体版本） |
| **建议方向** | 确认本机 Python 版本（`python --version`）；在交互式模式（REPL）中逐行输入函数定义与调用，观察解释器即时反馈；在脚本模式下复现 `NameError`，理解报错信息的结构 |

---

## 四、语言机制

### 4. 标识符命名规则（Identifier Rules）

| 字段 | 内容 |
|------|------|
| **名称** | Python 标识符命名规则 |
| **类型** | 语言规范 / 语法约束 |
| **在本P中的作用** | 约束函数名的合法写法：只能由字母、数字、下划线组成，不能以数字开头，不能与关键字重名，严格区分大小写；课程明确指出函数命名规则与变量命名规则相同 |
| **来源** | 字幕直接提到 |
| **建议方向** | 整理一份"合法 / 非法函数名"对照表，覆盖数字开头、含空格、与关键字冲突等典型错误；查阅 Python 官方文档中关于标识符的定义（[https://docs.python.org/3/reference/lexical_analysis.html#identifiers](https://docs.python.org/3/reference/lexical_analysis.html#identifiers)） |

### 5. 文档字符串（Docstring）

| 字段 | 内容 |
|------|------|
| **名称** | 文档字符串（docstring） |
| **类型** | Python 语言机制 / 代码规范 |
| **在本P中的作用** | 出现在函数体首行，用三引号包裹，用于说明函数的功能；本P以 `greet()` 函数为例演示了最基础的写法，标注为可选 |
| **来源** | 字幕直接提到 |
| **建议方向** | 为自定义函数补写 docstring，然后用 `help(函数名)` 或 `函数名.__doc__` 验证其可被读取；了解 PEP 257 对 docstring 格式的规范建议 |

### 6. `NameError` 异常

| 字段 | 内容 |
|------|------|
| **名称** | `NameError: name 'xxx' is not defined` |
| **类型** | Python 内置异常类型 |
| **在本P中的作用** | 课程通过故意在函数定义之前调用 `greet()` 来触发此报错，用于演示"先定义后调用"规则的必要性；是本P唯一出现的报错类型 |
| **来源** | 字幕直接提到（原文引用了完整报错信息） |
| **建议方向** | 主动复现此报错，阅读完整错误信息的三段结构（文件路径、出错行、错误类型与描述）；对比 `NameError` 与 `IndentationError` 在函数场景下的触发条件 |
