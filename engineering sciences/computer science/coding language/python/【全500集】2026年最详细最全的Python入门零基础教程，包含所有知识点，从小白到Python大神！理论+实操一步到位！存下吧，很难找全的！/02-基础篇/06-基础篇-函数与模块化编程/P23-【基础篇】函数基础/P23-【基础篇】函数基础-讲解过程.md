# P23 讲解过程 - 【基础篇】函数基础
## 本P定位

本P是Python函数基础的开篇，主要完成两项任务：
1. 从概念层面讲清楚"什么是函数"和"为什么需要函数"
2. 从操作层面讲清楚函数的定义与调用这两个核心步骤

本P不涉及参数传递和返回值的高级用法，留待后续小节详细展开。

### 三、函数概念铺垫：工具箱比喻

**时间区间：[46.0-123.5]**

#### 3.1 什么是函数

老师用"代码工具箱"来比喻函数：将重复执行的逻辑、实现特定功能的代码片段封装成一个独立的单元，之后要用的时候不用反复写冗余代码，只需要简单调用一下就可以了。

#### 3.2 函数的工作模型

老师将函数比作"自动化的工具"：
- **输入（原材料）** = 参数
- **内部流程** = 函数体
- **输出（结果）** = 返回值

#### 3.3 直观例子

老师给出两个直观例子帮助理解：
- **打招呼工具**：输入人名，自动输出完整问候语
- **求和工具**：不管输入多少个数字，都能自动算出总和

### 四、为什么非要用函数：四个核心好处

**时间区间：[123.5-218.0]**

老师系统阐述了使用函数的四个核心好处，每个好处都配有具体例子：

#### 好处一：减少重复代码

老师举例：想要写十遍相同的问候逻辑，不需要复制粘贴十遍，封装成一个函数后，一行调用代码就能搞定，而且能避免复制粘贴出错的问题。

#### 好处二：代码更易读更好理解

老师指出函数名能直观表达功能：
- `greet` —— 一看就知道是用来问候的
- `sum` —— 一看就知道是用来求和的

看代码就像看说明书一样清晰。

#### 好处三：方便维护

老师举例：想把问候语从"你好呀"改成"hello"，只需要修改函数内部的一处代码，所有调用这个函数的地方都会自动更新，不用手动一个一个去改。

#### 好处四：拆分复杂任务

老师指出可以把大功能拆分成多个小函数。以用户注册功能为例，可以拆成：
- 验证手机号
- 创建账号
- 发送验证码

这几个小函数，代码结构更清晰，调用方便，调试也方便。

### 五、函数的核心步骤：定义与调用

**时间区间：[218.0-541.9]**

老师明确指出函数有两个核心步骤：**定义函数是打造工具，调用函数是使用工具，这两步缺一不可。**

#### 5.1 函数的定义

**时间区间：[229.2-329.5]**

老师逐项讲解函数定义的标准语法格式：

```
def 函数名(参数列表):
    文档字符串（可选）
    函数体
    return 返回值（可选）
```

**逐项拆解：**

| 组成部分 | 说明 |
|---------|------|
| `def` | 关键字，def end / define 的简写，中文意思是"定义" |
| 函数名 | 遵循标识符规则（与变量命名规则相同）：不能是关键字，只能由数字、字母、下划线组成，严格区分大小写 |
| 参数列表 | 可选，可以省略，也可以设定不同参数（本P不展开） |
| 冒号 `:` | 末尾加上冒号，回车后自动缩进 |
| 缩进 | 必须严格控制，函数体代码必须保持一致的缩进 |
| 文档字符串 | 可选，用于解释函数功能，方便自己和别人看懂 |
| 函数体 | 实现功能的具体代码块 |
| `return` | 可选，用于指定返回值，工具的输出结果（本P不展开） |

#### 5.2 实战演示：定义一个无参数无返回值的问候函数

**时间区间：[332.3-407.0]**

**需求**：定义一个输出固定问候语的函数

**实现步骤**：

```
def greet():
    """
    输出固定的问候语
    """
    print("你好呀")
    print("今天是美好的一天")
```

老师边写边讲解：
1. 用 `def` 关键字开头
2. 函数名叫 `greet`（问候的意思）
3. 括号内没有参数（因为目前还没学到参数）
4. 末尾加冒号
5. 回车后写函数体，可以加多行注释说明功能
6. 函数体内写 `print` 语句输出问候语

#### 5.3 调用函数的演示与排错

**时间区间：[407.0-476.8]**

老师运行程序后发现**没有任何输出**，于是引出关键问题：明明使用了 print 函数，为什么没有打印？

**解答**：定义完函数后想要使用它，需要通过"**函数名 + 括号**"的方式去调用。

老师演示正确写法：

```
greet()  # 函数名加小括号表示调用，即执行函数里面的功能
```

再次运行后，程序输出了"你好呀"和"今天是美好的一天"这两句话，说明函数体内的代码被执行了。

#### 5.4 调用次数的演示

**时间区间：[504.8-538.6]**

老师演示多次调用：调用 `greet()` 两次，就会输出两次问候语，说明函数被调用几次，内部的代码就执行几次。

老师进一步演示函数体内有多行代码的效果：
- 在函数体内添加 `print("-" * 20)` 分隔线
- 只写一行 `greet()` 调用，就能执行函数内的多行代码
- 老师强调：这样能够提高代码的复用性

### 六、两个关键注意点

**时间区间：[541.9-662.5]**

老师特别强调函数使用中必须牢记的两个注意点：

#### 注意点一：先定义后调用

**规则**：必须先写函数定义的代码，再写调用的代码。如果在函数定义之前就写调用，程序会报错。

**演示排错**：
- 老师故意在函数定义之前调用 `greet()`
- 运行后发现报错：`NameError: name 'greet' is not defined`（命名错误，名称 greet 没有定义）

**通俗解释**：就像去超市买东西，得先付钱把东西买下来，买下来之后才能吃。如果还没结账就直接吃了，是不可以的。

**原理说明**：代码的执行顺序本身是从上往下逐行执行的，先执行了调用代码，发现没有 greet 函数，当然就会报错，程序卡在这里不会继续往下执行。

#### 注意点二：函数不调用不执行

**规则**：只定义函数，不写调用的代码，函数里面的逻辑永远不会被执行。

**通俗解释**：就像打造了一个工具但从来没用过，它就发挥不了相应作用。你要用它就得调用，而且调用几次就会执行几次。

## 本P结论

1. **函数的本质**：Python中的代码工具箱，将重复执行的逻辑封装成独立单元，用时调用即可，无需重复编写代码。

2. **使用函数的四个核心好处**：减少重复代码、代码更易读更好理解、方便维护、能够拆分复杂任务。

3. **函数的核心步骤**：
   - **定义** = 打造工具（使用 `def` 关键字）
   - **调用** = 使用工具（使用"函数名 + 括号"）

4. **函数定义的标准语法**：
   ```
   def 函数名(参数列表):
       文档字符串（可选）
       函数体
       return 返回值（可选）
   ```

5. **函数名的命名规则**：遵循标识符规则，与变量命名规则相同。

6. **两个必须遵守的注意点**：
   - **必须先定义后调用**：代码从上往下逐行执行，调用在定义之前会导致 `NameError`
   - **函数不调用不执行**：只定义不调用，函数内的代码永远不会被执行
