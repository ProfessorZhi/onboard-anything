## 机制理解

### 题目1

以下函数用于计算商品折扣价。销售人员发现某些商品明明原价 500 元，却显示"折后价：500"，请指出问题所在并解释原因。

```python
def apply_discount(price, discount):
    if discount <= 0:
        print("折扣无效")
        return
    discounted = price * (1 - discount / 100)
    print(f"折后价：{discounted}")

final_price = apply_discount(500, 20)
print(f"实际到手价：{final_price}")
```

### 题目2

小李写了两个函数比较数字大小，他想知道为什么 `compare_v1` 可以直接比较，而 `compare_v2` 必须加判断？

```python
# 版本1
def compare_v1(a, b):
    if a > b:
        return a, b
    return b, a

# 版本2
def compare_v2(a, b):
    if a > b:
        return a, b
    return b, a

# 调用
max_val, min_val = compare_v1(3, 8)
print(max_val > min_val)

result = compare_v2(3, 8)
print(result[0] > result[1])
```

## 扩展思考

### 题目1：返回值的设计选择

在 `return x, y` 这种写法中，返回的实际上是 **tuple** 而不是两个独立的值。请思考：

- 如果后续调用者只需要其中一个值（如 x），使用 tuple 返回是否合适？
- 有没有更优雅的替代方案？提示：可以考虑返回 `dict` 或 `namedtuple`

### 题目2：return 与 print 的边界

一个调试函数 `debug()` 暂时只需要打印信息，不需要返回值。请判断以下两种写法哪种更规范？为什么？

```python
# 写法A
def debug(message):
    print(f"[DEBUG] {message}")

# 写法B
def debug(message):
    return print(f"[DEBUG] {message}")
```
