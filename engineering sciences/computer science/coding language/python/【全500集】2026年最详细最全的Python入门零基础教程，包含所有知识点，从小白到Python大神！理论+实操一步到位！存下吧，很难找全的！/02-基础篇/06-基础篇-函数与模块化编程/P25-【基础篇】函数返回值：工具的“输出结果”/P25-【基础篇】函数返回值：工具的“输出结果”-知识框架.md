# P25 知识框架 - 【基础篇】函数返回值：工具的“输出结果”
## 本P位置

Python 基础 → 函数 → 函数三要素：参数（输入）→ **返回值（输出）** → 文档字符串。本P承接 P24 参数概念，完成函数"输入-处理-输出"的完整闭环。

## 核心问题

函数执行完后，如何把结果传递给调用者？

## 知识骨架

```
return 关键字
├── 语义：返回值给调用者
├── 副作用：立即终止函数执行
└── 三种情况
    ├── 无 return / return 空 → 返回 None
    ├── return 单值 → 返回该值
    └── return 多值 → 返回 tuple（元组）

返回值接收
├── 单变量接收 → 得到 tuple
└── 解包接收 → 多个变量分别取值（推荐）
```

## 关系与依赖

- **前置**：参数概念（P24）—— 参数是输入，return 是输出
- **对比**：print 仅打印，return 传值；print 无返回值，return 产生返回值
- **对比**：return vs return None —— 等价，但显式 return 更清晰
- **对比**：单变量接收 vs 解包 —— 解包更易用，元组元素数量必须与变量数匹配

## 输入 / 输出 / 前置 / 支撑

| 角色 | 内容 |
|------|------|
| 输入 | 参数（已讲）、函数调用时的实参 |
| 输出 | return 后跟的表达式，类型不限 |
| 前置依赖 | P24 参数（输入） |
| 语法支撑 | 变量赋值、元组概念（打包/解包） |

## 易错点

1. **print ≠ return**：print 只打印不返回值；用 print 的函数无法参与后续计算，result 为 None
2. **return 终止函数**：return 后的代码永不执行，容易漏掉后续逻辑
3. **多返回值本质**：实际返回的是 tuple，不是多个独立值
4. **解包数量匹配**：变量数必须等于返回值数，否则报错
