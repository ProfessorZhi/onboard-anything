# P26 知识框架 - 【基础篇】 变量作用域：变量的“生效范围”
## 本P位置
- **课程阶段**：Python基础篇，函数专题（第26P）
- **前置依赖**：P23-P25 函数定义、参数传递、返回值
- **后续延伸**：闭包、装饰器、类与对象中的属性作用域
- **课程作用**：建立变量生命周期与可见性的基础认知，是理解函数封装和后续面向对象的基础

## 核心问题
变量定义后在哪里可见？在哪里可修改？修改是否影响原始值？

## 知识骨架

```
变量作用域
├── 局部变量（Local Variable）
│   ├── 定义位置：函数内部（含形参）
│   ├── 生命周期：函数调用时创建，函数结束时销毁
│   └── 可见范围：仅函数内部
│
├── 全局变量（Global Variable）
│   ├── 定义位置：函数外部
│   ├── 生命周期：程序启动时创建，程序结束时销毁
│   ├── 可见范围：整个模块（除非被局部变量遮蔽）
│   └── 读取：函数内部可直接访问
│   └── 修改：需使用 global 声明
│
└── global 关键字
    ├── 作用：声明变量为全局变量
    ├── 位置：必须在函数内部第一行使用
    └── 效果：解除只读限制，允许修改全局变量
```

## 易错点

1. **局部变量遮蔽全局变量**：函数内部赋值同名变量会创建新局部变量，不修改全局变量
2. **global 遗漏**：在函数内直接修改全局变量不会生效，会被解释器当作创建新局部变量
3. **global 位置错误**：必须放在函数内部第一行，在任何赋值之前
4. **跨文件作用域混淆**：全局变量只在本文件有效，跨文件需要 import
5. **形参即局部变量**：函数参数等同于函数内部的局部变量

## 核心结论（可直接引用）

- 变量定义位置决定作用域：函数内=局部，函数外=全局
- 局部变量随函数而生，随函数而灭
- 全局变量随程序而生，随程序而灭
- 读取全局变量无需声明，修改全局变量必须 global 声明
