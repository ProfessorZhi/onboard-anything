# P27 讲解过程 - 【基础篇】案例：函数版学员管理系统
## 本P定位

本 P 是函数版学员管理系统的完整实战项目课。老师先整体说明系统的六个功能模块和数据存储结构，然后逐步搭建系统框架（菜单界面 + 循环结构），再逐一实现添加、删除、修改、查询、显示所有、退出六个功能。核心目标是让零基础学习者通过一个完整小项目，熟练掌握函数的封装与调用、列表与字典的增删改查操作，以及 while 死循环控制流程。

## 讲解过程

### 一、项目整体介绍

老师开门见山说明这是一个实战项目，零基础学员可以用之前学过的函数、循环、字典和列表打造一个完整的学员管理系统。系统能实现六项核心操作：**添加、删除、修改、查询（单个）、显示所有、退出**。

老师逐一讲解了六个功能的具体交互逻辑：

- **添加学员**：录入学号、姓名、手机号，其中**姓名不允许重复**，重复时提示"已经存在"，不重复才添加。
- **删除学员**：按姓名查找，找到即删除，找不到提示未找到。
- **修改学员**：按姓名查找，找到后更新学号或手机号，不存在则提示。
- **查询学员**：按姓名查询，找到按格式显示，找不到提示。
- **显示所有**：把所有学员信息整齐排列输出。
- **退出系统**：需要**二次确认**，输入 yes 才退出，输入 no 继续运行，防止误操作。

### 二、数据存储方式说明

老师强调数据存储采用**列表 + 字典**的组合结构：

- 定义一个**全局列表** `info = []`，所有学员数据放在这个列表里，供所有函数共用，无需反复传递数据。
- 每个学员的信息是一个**字典**，包含三个键值对：`id`（学号）、`name`（姓名）、`tel`（手机号）。
- 举例：`info = [{"id": 1, "name": "冰冰", "tel": "130111222"}]`，结构清晰，方便后续增删改查。

### 三、搭建系统框架（第一步）

#### 1. 显示功能界面函数

老师认为显示功能界面的代码会被**多次使用**（每执行完一个操作都要重新显示菜单），而且并非连续输出——是"需要时输出，不需要时就不输出"，因此不适合用 for 循环，而应该封装为函数 `info_print()`。

该函数打印的内容包括：
- 分隔线（`=` 乘以 30）
- 居中的标题"学员管理系统 V1.0"
- 六个功能选项的序号和名称（1. 添加学员，2. 删除学员，3. 修改学员，4. 查询单个学员，5. 显示所有学员信息，6. 退出系统）

#### 2. 接收用户输入并分支处理

在 `info_print()` 函数调用之后，用 `input()` 接收用户输入的功能序号，存入变量 `user_num`，提示"请输入功能序号（1~6）"。

由于用户输入不同序号会执行不同功能，属于**多条件选一**的场景，老师选用 `if ... elif ... elif ... else` 结构：

```python
if user_num == "1":    # 添加学员
elif user_num == "2":  # 删除学员
elif user_num == "3":  # 修改学员
elif user_num == "4":  # 查询学员
elif user_num == "5":  # 显示所有学员
elif user_num == "6":  # 退出系统
else:
    print("输入有误，请重新输入1~6之间的序号")
```

老师提到 `elif` 结构的好处是前面的条件满足后后面的就不用判断了，能提高代码效率。

#### 3. 退出系统的二次确认

对于选项 6（退出系统），老师认为不需要封装成函数，因为这个功能只会用一次。实现逻辑是：
- 用 `input()` 询问"是否确认退出？（yes or no）"
- 如果输入 `"yes"`，输出"退出系统成功"并用 `break` 终止循环
- 如果输入 `"no"` 或其他内容，则继续运行（不退出）

#### 4. 循环结构的决策

老师指出了一个关键问题：**用户输入 NO 之后，当前代码仍然直接结束了**。原因是整个"显示功能界面 → 接收用户输入 → 执行对应功能"的流程只执行了一次，并没有形成循环。

经过分析：
- 循环次数不确定（不知道用户要做多少次操作）
- 但知道循环结束条件（用户输入 6 并确认退出）
- 因此选择 **while True 死循环 + break** 的方案

```python
while True:
    info_print()          # 显示功能界面
    user_num = input(...) # 接收用户输入
    # if-elif 分支处理各功能
    # 输入6并确认后执行 break 终止循环
```

老师强调：每一次循环开始都先显示功能界面，告诉用户每个序号对应的功能，然后再接收用户输入，这样的逻辑非常直观。

### 四、添加学员功能（第二步）

#### 1. 定义添加学员函数

老师将添加学员功能封装为函数 `add_info()`，函数名中 `add` 是添加，`info` 是信息的意思。调用方式是在第一步的 `if user_num == "1"` 分支中写 `add_info()`。

#### 2. 接收用户输入

在函数内用三个 `input()` 分别接收用户输入的学号（new_id）、姓名（new_name）、手机号（new_tel）。

#### 3. 组装字典并追加到列表

```python
student_dict = {"id": new_id, "name": new_name, "tel": new_tel}
info.append(student_dict)
```

老师演示后加了一行打印 `print(f"添加成功，学员信息为{student_dict}")` 用于测试验证添加是否成功。

#### 4. 实现姓名去重判断

老师发现了一个问题：如果再次添加同名的"冰冰"，系统会允许重复添加，这是不允许的。解决思路是在添加到列表之前，先**遍历已有列表**，判断姓名是否重复：

```python
for student in info:
    if new_name == student["name"]:
        print(f"错误：姓名为 {new_name} 的学员已存在，无法重复添加")
        return  # 结束函数运行，不再执行后续添加逻辑
```

老师特别强调：
- 这里应该用 `return` 而不是 `break`，因为 `break` 只能退出当前循环，而我们要结束的是**整个函数**的运行。
- `return` 后面可以不写内容，默认返回 `None`。

老师演示验证：添加"冰冰"后再添加同名"冰冰"会提示已存在，无法重复添加；添加"美女"则可以正常添加成功。

### 五、删除学员功能（第三步）

#### 1. 定义删除学员函数

封装为函数 `del_info()`，函数内用 `input()` 接收用户要删除的学员姓名，存入变量 `del_name`。

#### 2. 遍历查找并删除

```python
for student in info:
    if del_name == student["name"]:
        info.remove(student)
        print(f"删除成功，已删除姓名为 {del_name} 的学员信息")
        print(f"当前剩余学员有：{info}")
        return  # 删除成功后立即结束函数，避免重复删除
```

#### 3. 未找到的处理

如果 for 循环正常结束（遍历完所有学员都没有匹配），说明学员不存在，执行：

```python
print(f"错误：未找到姓名为 {del_name} 的学员")
```

老师演示：删除不存在的"苏苏"会提示未找到；删除存在的"冰冰"会删除成功并显示剩余学员为空列表。

#### 4. 关于数据持久化的说明

老师在演示过程中提到，当前系统没有实现数据持久化保存，每次重新运行程序列表就被初始化为空了。如果学完了文件读写，可以进一步完善实现数据的永久保存。

### 六、修改学员功能（第四步）

老师指出修改的逻辑和删除是类似的——都是按姓名查找，找到后执行操作，找不到则提示。

#### 1. 定义修改学员函数

封装为函数 `modify_info()`，接收用户输入的要修改的学员姓名，保存到变量中。

#### 2. 遍历查找并修改

```python
for student in info:
    if modify_name == student["name"]:
        # 直接接收新的学号和手机号并赋值给字典
        student["id"] = input("请输入新的学号")
        student["tel"] = input("请输入新的手机号")
        print("修改成功，更新后的信息如下：")
        print(f"学号：{student['id']}，姓名：{student['name']}，手机号：{student['tel']}")
        return
```

老师演示了字典的**直接赋值修改**操作——不需要先保存到临时变量再赋值，可以直接把 `input()` 的结果赋值给字典的键。修改后打印更新后的学员信息。

#### 3. 未找到的处理

循环正常结束后输出：`f"错误：未找到姓名为 {modify_name} 的学员"`

## 本P结论

1. **系统框架核心**：采用 `while True` 死循环搭配 `break` 实现持续运行，配合 `if-elif-else` 多分支处理用户输入的序号，直到用户选择退出才终止循环。

2. **数据存储结构**：使用**全局列表 `info`** 存储所有学员，每个学员信息用一个**字典**（键为 `id`、`name`、`tel`），列表中嵌套字典的结构便于增删改查操作。

3. **功能模块化**：每个功能（添加、删除、修改）都封装为独立函数，函数内部完成接收输入、业务逻辑处理、结果反馈的全流程。

4. **姓名去重机制**：添加前遍历列表判断姓名是否已存在，重复则 `return` 终止函数；删除和修改同样通过遍历查找，找到后执行操作并 `return`，找不到则给出明确提示。

5. **字典修改方式**：对已找到的字典直接通过键名赋值新值即可完成修改，无需借助临时变量中转。

6. **return 与 break 的区别**：`return` 结束整个函数运行，`break` 仅退出当前循环。在功能函数内部检测到异常条件（如姓名重复、删除目标不存在）时应使用 `return` 直接终止函数。
