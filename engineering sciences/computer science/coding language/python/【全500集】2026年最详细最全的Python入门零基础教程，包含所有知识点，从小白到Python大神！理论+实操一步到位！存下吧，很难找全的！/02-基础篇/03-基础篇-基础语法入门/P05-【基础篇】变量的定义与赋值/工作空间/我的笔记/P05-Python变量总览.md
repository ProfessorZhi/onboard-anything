# P05 Python 变量与常见数据类型总览

## 先纠正一个容易误入的目标

Python 里不存在“所有变量清单”。

变量不是语言预先内置好的一张名单，而是我在代码里自己起的名字。

所以真正该掌握的不是“背变量表”，而是：

1. 变量是什么
2. 变量能绑定什么对象
3. 变量和类型、对象、内存、回收的关系是什么

## 变量到底是什么

变量不是盒子，更准确地说，变量是一个**名字**，或者说一个**标签**。

例如：

```python
x = 10
```

更准确的理解是：

- 右边先产生一个对象 `10`
- 左边变量名 `x` 绑定到这个对象

所以重点不是“x 里装了 10”，而是“x 指向了值为 10 的对象”。

## 变量本身有没有类型

严格说，变量名本身不重要，真正有类型的是它指向的对象。

```python
x = 10
print(type(x))   # int

x = "hello"
print(type(x))   # str
```

这说明同一个变量名 `x` 可以先绑定 `int` 对象，再绑定 `str` 对象。

所以 Python 是动态类型语言，但不是无类型语言。

## 常见数据类型清单

你原来想找的，准确说应该叫“常见数据类型”，不是“所有变量”。

入门阶段先记这几类就够了：

### 1. `int`

整数类型。

```python
age = 18
count = 100
```

特点：

- 表示整数
- 可以做加减乘除等运算

### 2. `float`

浮点数，也就是小数。

```python
price = 19.9
height = 175.5
```

特点：

- 表示带小数的数字
- 常用于价格、身高、成绩等场景

### 3. `bool`

布尔类型，只有两个值。

```python
is_ok = True
is_admin = False
```

特点：

- 只有 `True` 和 `False`
- 常用于条件判断

### 4. `str`

字符串类型，本质上是一段文本。

```python
name = "Tom"
city = "Shanghai"
age_text = "18"
```

特点：

- 文字、符号、数字字符都可以属于字符串
- `"18"` 看起来像数字，但它本质上还是字符串

### 5. `list`

列表类型，可以按顺序放多个元素。

```python
scores = [90, 85, 88]
names = ["Tom", "Jack", "Lily"]
```

特点：

- 有顺序
- 可修改
- 里面可以放多个值

### 6. `tuple`

元组类型，也是一组有顺序的数据。

```python
point = (3, 5)
rgb = (255, 255, 255)
```

特点：

- 有顺序
- 一般认为不修改

### 7. `dict`

字典类型，按“键:值”存数据。

```python
user = {"name": "Tom", "age": 18}
```

特点：

- 适合表示带字段的数据
- 通过键查值

### 8. `set`

集合类型，常用于去重。

```python
nums = {1, 2, 3}
```

特点：

- 元素不重复
- 无序

### 9. `NoneType`

表示“空”或“暂时没有值”，对应值是 `None`。

```python
result = None
```

特点：

- 表示没有对象、没有结果、尚未赋有效值

## 一眼区分这些类型

```python
18              # int
3.14            # float
True            # bool
"18"            # str
[1, 2, 3]       # list
(1, 2, 3)       # tuple
{"a": 1}        # dict
{1, 2, 3}       # set
None            # NoneType
```

## 和 P06 的直接连接

最容易混淆的是这两个：

```python
18      # int
"18"    # str
```

它们看起来像，但不是一回事。

- `18` 是数字，可以直接计算
- `"18"` 是文本，默认不能直接和数字混算

这就是为什么 `input()` 之后经常要做：

```python
age = int(input("请输入年龄："))
```

## 常见变量命名示例

这些不是固定变量，只是常见且有语义的名字：

- `name`
- `age`
- `score`
- `price`
- `is_admin`
- `items`
- `user_info`

变量名不是越短越好，而是越清楚越好。

## 赋值时到底发生什么

```python
a = 10
b = a
```

这时更准确的理解是：

- `a` 绑定到对象 `10`
- `b` 也绑定到同一个对象 `10`

所以赋值很多时候不是“复制对象”，而是“增加一个新的绑定关系”。

## 重新赋值时发生什么

```python
a = 10
a = 20
```

这不是把对象 `10` 改成了 `20`。

真正发生的是：

- `a` 原来绑定 `10`
- 后来 `a` 改为绑定 `20`
- 原来的 `10` 对象是否回收，取决于还有没有别的引用

## 变量与垃圾回收

对象是否被回收，看的是“还有没有引用”。

```python
a = "hello"
b = a
a = 123
```

此时：

- `a` 指向 `123`
- `b` 仍然指向 `"hello"`

所以 `"hello"` 不会因为 `a` 改绑就立刻消失。

## 入门阶段最值得记住的 5 句话

1. 变量是名字，不是盒子。
2. 对象在右边，变量在左边。
3. 真正带类型的是对象，不是变量名。
4. 重新赋值通常是重新绑定，不是修改原对象类型。
5. 对象是否回收，取决于还有没有引用。
