# P05 讲解过程 - 【基础篇】变量的定义与赋值
## 本P定位

本 P 是 Python 基础教程的第三小节，承接前两节内容，首次引入 Python 编程的核心概念——变量。教学目标是让零基础学员理解变量的本质（内存中的命名空间）、掌握变量的定义与赋值语法、记住变量命名的硬性规则、了解基本数据类型分类。整 P 围绕“什么是变量 → 怎么赋值 → 怎么命名 → 数据有哪些类型”这条主线推进，讲练结合，边写代码边演示错误。

## 讲解过程

### 一、引出变量概念：贴标签的储物盒

课程开头直接点题：变量是 Python 编程的基础核心，不复杂。接着用生活化的比喻讲解变量是什么——把变量类比为生活中贴了标签的储物盒。拿大型仓库举例：仓库里有很多储物箱，给每个箱子贴一个好记的标签（比如“零花钱”“考试分数”），不用记复杂的内存地址（即具体物理位置），通过标签就能快速找到箱子里存放的物品（具体数据）。

老师进一步从技术层面给出定义：变量就是计算机内存中用来存放数据的命名空间。然后将二者对应起来——大型仓库 = 计算机内存，储物箱 = 内存地址，标签 = 变量名。强调不想直接操作内存地址，就给数据起一个变量名来保存；后期想拿数据，通过变量名就可以拿到。

### 二、变量的三个核心用处

老师总结了变量非常实用的三个核心用处。

**第一，数据标注。** 给杂乱的数据起一个有意义的名称。例如数字 18，定义一个变量 age 保存，age 就是年龄的意思，一眼就知道存的是年龄数据；13011112222 这个数字，定义一个变量 phone 保存，phone 就是手机号的意思，就知道存的是手机号。老师强调这样不管是自己还是别人看代码，都能立刻知道变量里存的是什么数据。

**第二，数据复用。** 如果一个相同的数据要多次使用，比如手机号 13011112222 要在多处出现，每次都敲一长串数字既麻烦又容易出错。把它保存到变量里，后续直接通过变量名使用即可。

**第三，动态修改。** 数据可以随时切换。比如一开始给变量 score（分数）赋值为 80，后来成绩提高了改成 90，程序之后就会自动使用新的分数。老师用余额举例：余额不是一成不变的，会根据收支情况调整，可以把余额放到变量里存储，然后动态修改。老师总结：过程中数据不是一成不变的，可以动态修改，非常灵活。

### 三、变量赋值语法讲解

搞懂概念后，进入关键步骤：怎么给变量装数据，也就是怎么给变量赋值。

老师先给出基本语法：**变量名 = 值**。例如 `answer = 42`，就是通过等号（赋值运算符）把 42 这个值赋给了 answer 变量。

**强调一个关键点：等号不是数学中的等于号，而是赋值运算符。** 含义是把右边的值放到左边的变量里。执行顺序是先计算右边的内容，再把最终的计算结果放到左边的变量中。

老师还提了一个代码书写小技巧：等号左右两边加一个空格，能让代码看起来更规范、更舒服。

### 四、演示：三种赋值方式

老师切换到 PyCharm 进行实操演示。

**方式一：直接赋值。** 以手机号 13011112222 为例，定义变量 phone 保存它。先写 `phone=13011112222`（不加空格），运行不报错，但会有黄色波浪线（不符合代码规范 PEP8）。然后加上空格写成 `phone = 13011112222`，波浪线消失。老师指出此时控制台没有输出，因为没有打印，想输出就使用 print 函数。

**方式二：变量之间的赋值（把一个变量的值赋给另一个变量）。** 例如新建一个变量 phone2，把 phone 的值赋给它——`phone2 = phone`。运行后打印 phone2，输出的是 13011112222。

**方式三：表达式赋值（把计算结果赋给变量）。** 例如 `n = 3 * 4`，赋值号右边是一个算术表达式。执行顺序是先计算右边 `3 * 4 = 12`，再把结果 12 赋给变量 n。打印 n 得到 12。老师总结：变量赋值非常灵活，有不同的赋值方式。

### 五、重要注意事项：引号的作用与变量必须先赋值再使用

老师在这一步专门讲了一个容易出错的地方。

**核心问题：带引号和不带引号有什么区别？** 老师演示：定义变量 name，赋值为 `冰冰`（加引号），打印输出正常。再尝试直接打印不带引号的 `冰冰`（理解为变量名），运行后报 NameError（名称错误），提示 name '冰冰' is not defined。原因分析：加了引号，Python 理解为文字数据（字符串），可以直接使用；不加引号，Python 理解为变量名，而变量名必须先赋值再使用，否则报错。

老师用一个生活比喻：就像去超市买东西，还没结账就直接吃了，人家可以把你当小偷抓；但先结完账再吃就没问题。所以变量必须先定义赋值，直接使用会报错。

**正确的两种做法：** 要么给文字加上引号作为字符串输出；要么先把变量赋值，再用 print 输出该变量。

### 六、变量的两个重要特性

#### 特性一：动态类型

老师先说明 Python 中变量非常灵活，可以随时换成不同类型的数据，后面的值会覆盖前面的值。

**演示环节（使用 Debug 调试）：** 定义变量 variable = 10（整型），然后重新赋值 = 100.5（浮点型），再重新赋值 = "冰冰"（字符串，加引号）。打印 variable，输出的是最后的"冰冰"。

老师用 Debug 模式分步演示执行过程（期间因 PyCharm 配置文件问题进行了临时修复，属于操作花絮）。每点击一次“下一步”，观察变量值的变化：第一步 variable = 10（int 整型），第二步 100.5 覆盖了 10，此时 variable = 100.5（float 浮点型），第三步"冰冰"覆盖了 100.5，此时 variable = "冰冰"（str 字符串型）。老师强调：代码是按照从上往下的顺序逐行执行的，所以后面的值会覆盖前面的值，变量可以被反复赋值，并且可以是不同的类型。

#### 特性二：序列赋值（批量赋值）

**使用场景：** 需要依次给多个变量赋值的场景。例如要存三门科目的成绩。

**语法演示：** `python_score, mysql_score, html_score = 99, 87, 98`

老师解释：变量之间用逗号隔开（中间空格是代码规范性要求），值之间也用逗号隔开。执行时，第一个值 99 赋给第一个变量 python_score，第二个值 87 赋给第二个变量 mysql_score，第三个值 98 赋给第三个变量 html_score。打印各个变量验证结果。

**必须注意的报错情况：** 变量名和值的数量必须一一对应。如果三个变量只给两个值，如 `a, b, c = 1, 2`，程序会报错，提示 ValueError: not enough values to unpack (expected 3, got 2)，意思是期待得到三个值但只获得了两个。老师强调值的数量和变量数量一定要保持一致。

### 七、变量的命名规则

老师先把变量名比作储物盒的标签，起得好不好很关键——既不能让程序报错，还得让人一眼就能看懂。

#### 硬性要求（必须遵守，违反必报错）

**规则一：变量名只能用字母、数字和下划线。** 老师演示合法命名：`user_name`、`n1`、`gun_class` 等，验证可以正常运行。如果使用特殊符号（如 @）或空格，程序会报错。

**规则二：不能以数字开头。** 字母和下划线可以作为开头，但数字不行。老师演示用下划线开头 `_name = "冰冰"` 没问题，字母开头没问题，但数字开头（如 `2name = "冰冰"`）会报错。原因解释：如果第一个是数字，Python 分不清楚你到底是数据还是变量。

**规则三：不能使用 Python 的关键字。** 例如 `class`、`True`、`if` 等，这些是 Python 自带的有特殊功能的关键词，不能用作变量名。老师提到关键字有 30 多个记不住怎么办，并指出了一个实用技巧：在 PyCharm 中，关键字会显示不同的颜色（如橙色），普通变量名是另一个颜色，以此判断是否可以使用。老师演示：用 `class = 123` 会报错，因为 class 是关键字；如果改成 `class_ = 123` 或 `myclass = 123` 就可以使用。

**规则四：严格区分大小写。** 老师演示：定义小写 `a = 1`，用大写 `A` 去打印会报 NameError，提示名称 A 未定义；但用小写 `a` 打印就正常。说明 name、Name、NAME 是三个不同的变量。

#### 命名规范（软性建议，强烈推荐遵守）

**建议一：变量名要能说明用途。** 例如 `student_age`（学生年龄）、`book_name`（书名），翻译一下就知道是什么意思。避免使用 a、b、c、d 或 26 个字母随意命名，否则后期自己看代码都不知道存的什么数据。

**建议二：多单词组成的变量名有三种命名方式，推荐下划线分割法。**

老师逐一讲解三种方式：

- **下划线分割法（snake_case，最推荐）：** 全部字母小写，单词之间用下划线隔开。例如 `user_name`，user 是一个单词，name 是一个单词，中间加下划线，翻译就是“用户的名称”，非常清晰。
- **大驼峰命名法（PascalCase）：** 每个单词首字母大写。例如 `UserName`，适用于类名定义（后面学到定义类时会用到）。
- **小驼峰命名法（camelCase）：** 第一个单词小写，从第二个单词开始每个首字母大写。例如 `userName`。

老师强调：如果没有特殊规定，三种方式都可以用，都能提高代码规范性，建议尽量遵守。

### 八、变量的数据类型

老师先说明：等号右边的数据可以是不同类型的，就像现实生活有不同的数据类型（数字、文字等）。Python 中的值也有不同类型，不同类型有不同的用途。

**查看数据类型的方法：使用 `type()` 函数。** 如果放变量进去，就查看变量的类型；如果放具体数据，就查看数据的具体类型。

老师预告：今天只讲基本数据类型，复合数据结构（如列表、元组、字典、集合）会在第三章讲解。

#### 基本数据类型分类

**类型一：数值类型（用于存储数字，可以进行计算）**

包含四种：

- **整型（int）：** 即整数，例如 100、-50、0。老师演示：`money = 1000001234`，打印 money 输出 1000001234；用 `print(type(money))` 输出 `<class 'int'>`，说明是整型。
- **浮点型（float）：** 即小数，例如 3.14159。老师演示：`print(type(3.14159))` 输出 `<class 'float'>`。
- **布尔型（bool）：** 只有两个值 True（真）和 False（假），通常用于条件判断。老师演示：`print(type(True))` 输出 `<class 'bool'>`。强调 True 是关键字，在 PyCharm 中有颜色显示，如果加引号就变成了字符串；False 同理，只有这两个值属于布尔类型。
- **复数型（complex）：** 用于表示实数难以表示的数据，需要结合虚数。老师演示：`num = 23j`，`print(type(num))` 输出 `<class 'complex'>`。其中 j 是固定的虚数单位，可以大写也可以小写但不能改成其他字母。属于数学物理常用类型，零基础学员了解即可。

**类型二：字符串类型（str，String 的缩写）**

用于存储文字，能够用单引号、双引号甚至三引号包裹。单行字符串用单引号或双引号即可。老师演示：单引号 `'冰冰'`、`type('冰冰')` 输出 `<class 'str'>`；双引号 `"hello world"`、`type("hello world")` 输出 `<class 'str'>`。单引号双引号都可以使用。

## 本P结论

1. **变量的本质**：变量是计算机内存中用来存放数据的命名空间，可以理解为内存里贴了标签的储物盒，通过变量名可以快速找到和操作其中的数据，无需记忆复杂的内存地址。

2. **变量的三大核心用处**：数据标注（给数据起有意义的名字便于识别）、数据复用（避免重复输入相同数据）、动态修改（数据可以随时被新值覆盖）。

3. **赋值语法**：基本格式为 `变量名 = 值`，其中等号是赋值运算符而非数学中的等于号，执行顺序是先计算右边再赋给左边。赋值有三种方式：直接赋值、变量间赋值、表达式赋值。

4. **引号决定数据类型**：带引号的是字符串（文字数据），可以直接使用；不带引号的是变量名，必须先赋值再使用，否则报 NameError。

5. **变量两大特性**：动态类型（变量可以反复赋值且可以是不同类型，后值覆盖前值）和序列赋值（多个变量可同时赋多个值，数量必须一一对应）。

6. **命名硬性要求**：只能使用字母、数字、下划线；不能以数字开头；不能使用 Python 关键字；严格区分大小写。

7. **命名推荐规范**：变量名要能说明用途；多单词推荐使用下划线分割法（snake_case），也可使用大驼峰或小驼峰命名法。

8. **基本数据类型**：数值类型（int 整型、float 浮点型、bool 布尔型、complex 复数型）和字符串类型（str）；可使用 `type()` 函数查看数据类型。
