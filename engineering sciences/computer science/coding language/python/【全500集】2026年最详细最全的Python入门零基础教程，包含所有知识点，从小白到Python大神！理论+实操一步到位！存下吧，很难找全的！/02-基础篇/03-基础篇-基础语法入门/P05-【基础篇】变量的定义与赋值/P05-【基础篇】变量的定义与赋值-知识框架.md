# P05 知识框架 - 【基础篇】变量的定义与赋值
## 本P位置

Python 基础教程第三小节。首次引入编程核心概念——变量，承接前两节语法铺垫，为后续数据类型、控制流程、函数等所有内容奠定基础。

## 核心问题

变量是什么？变量怎么用？变量怎么命名？

## 知识骨架

```
变量
├── 概念：内存中存放数据的命名空间
├── 三用处：数据标注、数据复用、动态修改
├── 赋值：变量名 = 值（赋值运算符，非等于号）
│   ├── 直接赋值
│   ├── 变量间赋值
│   └── 表达式赋值
├── 特性
│   ├── 动态类型（后值覆盖前值）
│   └── 序列赋值（数量必须一一对应）
├── 命名
│   ├── 硬性要求：字母数字下划线、不能数字开头、不能关键字、区分大小写
│   └── 软性规范：见名知义、snake_case 推荐
└── 类型
    ├── 数值：int / float / bool / complex
    └── 字符串：str
```

## 关系与依赖

变量是 Python 最基础的存储单元，后续所有语法都以变量为基础。

- **变量 → 数据类型**：变量存储数据，数据有类型
- **变量 → type()**：type() 用来查看变量或数据的类型
- **命名规则 → 代码执行**：违反硬性规则直接报错，软性规范不报错但不符合 PEP 8
- **引号 → 数据类型**：带引号 = 字符串（直接数据），不带引号 = 变量名（需先定义）

## 输入 / 输出 / 前置 / 支撑

- **前置**：了解计算机内存基本概念（前两节内容）
- **输入**：赋值操作、右值表达式
- **输出**：通过变量名取出存储的数据
- **支撑工具**：PyCharm（IDE）、Debug 调试（观察变量值变化）、PEP 8（代码规范）

## 易错点

1. **等号误当等于号**：赋值运算符把右值放入左变量，执行顺序从右到左
2. **引号遗漏**：字符串不加引号会被当作未定义的变量名，报 NameError
3. **序列赋值数量不匹配**：变量数 ≠ 值数，报 ValueError
4. **关键字作变量名**：class、True 等是 Python 保留字，直接使用报语法错误
5. **大小写混淆**：name、Name、NAME 是三个不同变量
6. **数字开头**：1name 非法，name1 合法
