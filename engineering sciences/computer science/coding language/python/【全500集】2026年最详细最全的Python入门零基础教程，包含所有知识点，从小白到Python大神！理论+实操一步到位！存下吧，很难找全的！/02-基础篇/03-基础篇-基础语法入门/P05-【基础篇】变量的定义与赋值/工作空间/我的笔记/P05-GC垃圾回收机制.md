## 什么是 GC？为什么需要它？

**GC = Garbage Collection = 垃圾回收**

自动找出那些"**以后再也不用的对象**"，把它们的内存归还给系统。

### 为什么很多语言都有 GC？

| 语言 | 内存管理方式 | 代表问题 |
|------|-------------|---------|
| C / C++ | 手动 `malloc`/`free` | 内存泄漏、野指针 |
| Python / Java / JS / Go | 自动 GC | 无法精确控制回收时机 |
| Rust | 所有权系统（无GC） | 编译时保证内存安全 |

据研究，**70%的安全漏洞来自内存问题**（缓冲区溢出、空指针、use-after-free）。

### 什么是循环引用？

```python
a = []            # A.refcnt = 1
b = []            # B.refcnt = 1
a.append(b)       # B.refcnt = 2
b.append(a)       # A.refcnt = 2
del a             # A.refcnt = 1
del b             # B.refcnt = 1
```

```
外部没有任何标签指向A和B，但A和B互相指着对方
→ 引用计数都 ≠ 0，无法被自动回收
→ 内存泄漏！
```

## Python 的两层内存管理

```
┌─────────────────────────────────────────────────────┐
│ 第1层：引用计数  →  实时回收（非循环引用）          │
│         refcnt=0 立即释放，不经过GC                 │
│         优点：无延迟、零额外开销                     │
├─────────────────────────────────────────────────────┤
│ 第2层：分代 GC    →  定期回收（循环引用）           │
│         标记-清除算法                               │
│         优点：自动处理循环引用                       │
│         缺点：有延迟、有性能开销                     │
└─────────────────────────────────────────────────────┘
```

### 分代回收机制

```
┌─────────────────────────────────────────────────────┐
│ 第0代（年轻代）│ 新创建的对象           │ 回收最频繁│
│ 第1代（中年代）│ 第0代回收后存活的对象  │ 回收较少  │
│ 第2代（老年代）│ 第1代回收后存活的对象  │ 回收最少  │
└─────────────────────────────────────────────────────┘

触发阈值（默认）：第0代 ≥ 700个对象 → 触发第0代GC
```

**核心思想：** 大部分对象是"朝生夕死"的，年轻代对象更容易死亡，不需要反复检查老年代对象。

---

## GC 的三件事

```
1. 标记（Mark）— 从根对象（全局变量、栈）出发，标记所有可达对象
2. 清除（Sweep）— 未被标记的就是不可达的（循环引用），回收它们
3. 晋升（Promote）— 存活对象从年轻代晋升到中代/老年代
```

---

## Python GC 的工作时机

| 情况 | 时机 |
|------|------|
| 引用计数归零 | 立即回收（不经过GC） |
| 循环引用 | 等第0代积累到700个对象才触发GC |

```python
import gc
gc.get_count()      # 返回 (第0代数量, 第1代数量, 第2代数量)
gc.get_threshold()  # 返回 (700, 10, 10) 触发阈值
gc.collect()        # 强制立即全代回收
gc.disable()       # 禁用自动GC
gc.enable()        # 启用自动GC
```

## 为什么需要分代？

```
如果没有分代：需要检查所有对象，数量多了会非常慢

观察：大部分对象是"朝生夕死"的

解决：分代回收
- 年轻代对象多但大部分会死，只扫描年轻代
- 老年代对象少，扫描频率低
- 效果：GC开销从 O(n) 降为 O(n × 存活率)
```

**统计规律：** 大约90%的对象在创建后第一轮GC就被回收了。

---

## 一句话总结

> **Python 的内存管理 = 引用计数（实时回收非循环引用）+ 分代GC（定期回收循环引用）。引用计数归零立即回收；循环引用靠GC的"标记-清除"定期清理，分代是为了减少扫描开销。**
