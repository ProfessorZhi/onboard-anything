# P06 类型对象像函数一样调用来创建实例

## 这份笔记真正解决什么问题

你现在会看到这些写法：

```python
int("123")
str(123)
list("abc")
dict([("a", 1), ("b", 2)])
map(int, ["1", "2"])
```

它们看起来都像“函数调用”，但本质上不全是普通函数。

这份笔记只解决 4 件事：

1. 类型对象能不能被调用来创建实例
2. 为什么它们看起来像函数
3. 常见类型对象的“创建实例用法”是什么
4. 为什么有些类型对象拿不到完整签名

## 一、先给结论

### 1. 大多数普通类型对象都可以被调用来创建实例

例如：

```python
int("123")
str(123)
list("abc")
tuple([1, 2, 3])
set([1, 2, 2, 3])
dict([("a", 1), ("b", 2)])
map(int, ["1", "2"])
```

这些都属于：

- 调用一个类型对象
- 得到这个类型的实例对象

### 2. 但不要偷懒记成“所有类型对象都完全一样”

更准确的说法是：

- 很多类型对象都支持被调用
- 调用后会创建实例
- 但各自吃什么输入、返回什么结果、能不能直接取到完整签名，都可能不同

一句话：

**很多类型对象都能像函数一样被调用，但它们本质上仍然是类型对象。**

## 二、为什么看起来像函数

因为它们的用法形式是：

```python
名字(...)
```

这和函数调用长得一样。

但本质上：

- 普通函数：函数对象被调用
- 类型对象：类型对象被调用

例如：

```python
type(int)   # <class 'type'>
type(str)   # <class 'type'>
type(list)  # <class 'type'>
type(map)   # <class 'type'>
```

所以：

- `int` 不是普通函数
- `list` 不是普通函数
- `map` 也不是普通函数

它们都是类型对象。

## 三、这能不能叫“构造函数”

### 入门理解版

可以先这么理解：

- `list(...)` 像是在“构造”一个 list 对象
- `str(...)` 像是在“构造”一个 str 对象
- `map(...)` 像是在“构造”一个 map 对象

这个方向是对的。

### 更规范版

更规范地说：

- 调用类型对象
- 创建对应类型的实例

为什么不建议你现在直接把它们全都等同于“构造函数”？

因为在 Python 里，再往底层讲，还会涉及：

- `__new__`：更偏“创建对象”
- `__init__`：更偏“初始化对象”

所以更稳的说法是：

**这是“类型对象的构造式调用”，更严谨地说，是“调用类型对象来创建实例”。**

## 四、常见类型对象的创建实例用法

### 1. `int`

| 项目 | 内容 |
|---|---|
| 它本身是什么 | 类型对象 |
| `type(int)` | `<class 'type'>` |
| 常见写法 | `int("123")` |
| 常见输入 | 数字、像数字的字符串 |
| 常见输出 | `int` 实例对象 |
| 结果例子 | `123` |
| 完整签名 | 某些环境下 `inspect.signature(int)` 直接取不到 |

例子：

```python
x = int("123")
print(x)        # 123
print(type(x))  # <class 'int'>
```

```python

print(m)         # <map object at 0x...>
print(type(m))   # <class 'map'>
print(list(m))   # [1, 2, 3]
```

这里：

- `map(...)` 返回的不是列表
- 而是 `map` 对象
- 要用 `list(m)` 才能把结果展开出来

## 五、为什么有些类型对象拿不到完整签名

你可能会发现：

```python
inspect.signature(float)   # 能取到
inspect.signature(list)    # 能取到
inspect.signature(int)     # 可能取不到
inspect.signature(str)     # 可能取不到
inspect.signature(map)     # 可能取不到
```

这不代表：

- 它没有调用规则

而更常见的是：

- 当前 Python 实现没有把这份签名信息完整暴露给 `inspect.signature(...)`

所以要记住：

**取不到签名，不等于它没有签名规则；只是当前环境没把这份信息直接暴露出来。**

## 六、最短对照表

| 名字 | 它本身是什么 | 调用后得到什么 | 看起来像函数吗 |
|---|---|---|---|
| `int` | 类型对象 | `int` 实例对象 | 是 |
| `str` | 类型对象 | `str` 实例对象 | 是 |
| `list` | 类型对象 | `list` 实例对象 | 是 |
| `dict` | 类型对象 | `dict` 实例对象 | 是 |
| `map` | 类型对象 | `map` 实例对象 | 是 |

## 七、最该记住什么

1. 很多类型对象都能被调用来创建实例。
2. 这种写法看起来像函数调用，但本质上是“调用类型对象”。
3. 你可以先把它理解成构造式调用。
4. 更规范地说，是“调用类型对象创建实例”。
5. 不同类型对象的输入要求、输出结果、签名可见性都可能不同。
