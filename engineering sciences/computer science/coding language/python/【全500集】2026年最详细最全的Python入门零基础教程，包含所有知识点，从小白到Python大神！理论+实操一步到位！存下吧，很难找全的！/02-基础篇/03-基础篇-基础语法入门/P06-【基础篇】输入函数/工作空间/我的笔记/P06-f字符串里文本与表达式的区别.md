# P06 f 字符串里文本与表达式的区别

## 这份笔记真正解决什么问题

避免把下面这些混成一件事：

```python
"nums.__doc__"
nums.__doc__
f"{label}"
f"{nums.__doc__}"
```

它们看起来很像，但本质上不是同一类东西。

## 一、先记最短结论

### 1. 带引号

优先按**字符串文本**看。

```python
"nums.__doc__"
```

这里的内容只是普通文本，不会自动执行。

### 2. 不带引号

优先按**表达式求值**看。

```python
nums.__doc__
```

这里会真的访问对象 `nums` 的 `__doc__` 属性。

### 3. `f"..."` 里的 `{...}`

花括号里放的是：

- 变量
- 或表达式

Python 会先算出花括号里的结果，再把结果插入字符串。

## 二、最容易混淆的四种情况

### 情况 A

```python
label = "nums.__doc__"
```

意思：

- `label` 得到的是字符串
- 内容是 `nums.__doc__`
- 这只是文本，不是执行

### 情况 B

```python
label = nums.__doc__
```

意思：

- 先求值 `nums.__doc__`
- 再把求值结果赋给 `label`

这次不是文本，而是表达式结果。

### 情况 C

```python
label = "nums.__doc__"
print(f"{label}")
```

意思：

- 先取变量 `label` 的值
- 再把这个值插入字符串

插入进去的内容仍然只是：

```python
nums.__doc__
```

不是去执行 `nums.__doc__`

### 情况 D

```python
print(f"{nums.__doc__}")
```

意思：

- 花括号里直接就是表达式 `nums.__doc__`
- Python 会先求值
- 再把结果插入字符串

所以这次和 `f"{label}"` 不一样。

## 三、为什么这点特别容易错

因为下面这两句长得很像：

```python
f"{label}"
f"{nums.__doc__}"
```

但它们的本质不同：

- `f"{label}"`：插入变量 `label` 的值
- `f"{nums.__doc__}"`：直接执行表达式 `nums.__doc__`

也就是说，花括号里如果放的是“文本变量”，插进去的就是文本。  
花括号里如果放的是“代码表达式”，插进去的就是表达式结果。

## 四、最小对比例子

```python
nums = [1, 2, 3]

label = "nums.__doc__"
label2 = nums.__doc__

print(label)
print(label2[:20])
print(f"{label}")
print(f"{nums.__doc__[:20]}")
```

可以这样理解：

- `label`：文本
- `label2`：求值结果
- `f"{label}"`：插文本
- `f"{nums.__doc__[:20]}"`：插表达式结果

## 五、判断口诀

可以用这三步判断：

1. 先看有没有引号
2. 再看花括号里放的是变量名，还是表达式
3. 最后问自己：这里是“当文本保存”，还是“先求值再插入”

## 六、现在最值得记住的 4 句话

1. `"nums.__doc__"` 只是文本，不会自动执行。
2. `nums.__doc__` 是表达式，会真的求值。
3. `f"{label}"` 插入的是变量 `label` 的值。
4. `f"{nums.__doc__}"` 插入的是表达式 `nums.__doc__` 的结果。
