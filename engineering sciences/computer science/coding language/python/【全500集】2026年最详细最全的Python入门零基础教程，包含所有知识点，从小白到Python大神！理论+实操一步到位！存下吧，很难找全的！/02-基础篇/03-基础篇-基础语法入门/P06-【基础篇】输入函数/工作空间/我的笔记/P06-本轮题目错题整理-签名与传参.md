# P06 本轮题目错题整理 - 签名与传参

## 题目签名

```python
def f(a, /, b, *args, c, d=10, **kwargs):
    pass
```

先把规则固定下来：

- `a`
  - 仅位置参数
- `b`
  - 普通参数
- `*args`
  - 收多余位置参数
- `c`
  - 仅关键字参数，必传
- `d=10`
  - 仅关键字参数，可不传
- `**kwargs`
  - 收多余关键字参数

## 本轮真正错的点

这轮不是“整体不会”，而是有两个具体误区：

1. 忘了 `**kwargs` 会收多余关键字参数
2. 把“默认值存在”误解成“这个参数已经被传过了”

## 错题 4

### 原题

```python
f(1, b=2, c=3, x=9)
```

### 你当时的判断

不合法，理由是 `x` 没定义。

### 正确结论

合法。

### 为什么

因为签名里有：

```python
**kwargs
```

所以多出来的关键字参数不会直接报错，而是会被 `kwargs` 收进去。

### 参数绑定结果

```python
a = 1
b = 2
args = ()
c = 3
d = 10
kwargs = {'x': 9}
```

### 这题暴露的误区

不是“所有没在形参里写出来的关键字参数都报错”，而是：

- 没有 `**kwargs` 时，才会报错
- 有 `**kwargs` 时，会被收进去

## 错题 5

### 原题

```python
f(1, 2, 3, d=4)
```

### 你当时的判断

不合法，理由是 `d=10` 已经给过了。

### 正确结论

不合法，但真正原因不是这个。

### 真正原因

`c` 是必传的仅关键字参数，但这次没有传。

签名里：

```python
*args, c, d=10
```

表示：

- `c` 在 `*args` 后面，所以它只能关键字传参
- `c` 没有默认值，所以它是必传参数

这次调用里虽然传了：

```python
d=4
```

但没有传：

```python
c=...
```

所以报错。

### 参数绑定会走到哪一步

```python
a = 1
b = 2
args = (3,)
d = 4
```

然后解释器发现：

```python
c 缺失
```

于是报错。

### 这题暴露的误区

默认值不是“这个参数已经给过了”，而是：

如果你不传，就自动用这个值。

所以：

```python
d=10
```

只表示 `d` 可选，不表示你不能再写 `d=4`。

## 错题 6

### 原题

```python
f(1, 2, c=3, d=4, y=5)
```

### 你当时的判断

不合法，理由是 `y` 没出现过。

### 正确结论

合法。

### 为什么

因为：

```python
**kwargs
```

会把多余关键字参数收进去。

### 参数绑定结果

```python
a = 1
b = 2
args = ()
c = 3
d = 4
kwargs = {'y': 5}
```

### 这题暴露的误区

和错题 4 是同一个根因：

你漏用了 `**kwargs` 的兜底能力。

## 本轮额外收获

### 1. 多给参数会不会报错，要看有没有接收器

- 多余位置参数：
  - 没有 `*args` -> 通常报错
  - 有 `*args` -> 被收集成元组
- 多余关键字参数：
  - 没有 `**kwargs` -> 通常报错
  - 有 `**kwargs` -> 被收集成字典

### 2. 少给参数会不会报错，要看是不是必传

如果某个参数：

- 没传
- 没有默认值
- 又不是被别的规则兜住

那通常就会报错。

### 3. 默认值的真正含义

默认值表示：

“这个参数可以不传；如果不传，就使用默认值。”

不表示：

- 这个参数已经被传过了
- 这个参数不能再显式传新值

例如：

```python
def g(x=10):
    pass
```

这两种都合法：

```python
g()
g(20)
```

## 本轮最值得记住的 3 句话

1. `**kwargs` 存在时，多余关键字参数不一定报错，可能会被收进去。
2. `*args` 后面的普通参数，往往已经变成仅关键字参数。
3. 默认值表示“可不传”，不是“已经传过了”。
