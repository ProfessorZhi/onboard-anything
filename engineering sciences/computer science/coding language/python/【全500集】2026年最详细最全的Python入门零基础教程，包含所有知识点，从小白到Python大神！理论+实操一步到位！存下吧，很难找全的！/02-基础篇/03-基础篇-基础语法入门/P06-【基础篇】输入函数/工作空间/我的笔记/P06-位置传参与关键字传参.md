# P06 位置传参与关键字传参

## 这份笔记真正解决什么问题

看函数调用时，快速判断：

- 参数是按顺序传的，还是按名字传的
- 为什么有的调用合法，有的会报错
- 位置传参和关键字传参的核心区别是什么

## 一、位置传参是什么

位置传参就是：

**按实参出现的顺序，把值依次传给形参。**

例如：

```python
def f(a, b, c):
    pass

f(1, 2, 3)
```

对应关系：

```python
a = 1
b = 2
c = 3
```

### 一句话记忆

**位置传参靠顺序。**

## 二、关键字传参是什么

关键字传参就是：

**按参数名，把值传给对应形参。**

例如：

```python
def f(a, b, c):
    pass

f(a=1, b=2, c=3)
```

对应关系：

```python
a = 1
b = 2
c = 3
```

### 一句话记忆

**关键字传参靠名字。**

## 三、两者最大的区别

### 位置传参

- 靠顺序匹配
- 顺序错了，绑定结果就可能错

例如：

```python
def f(name, age):
    pass

f("Tom", 18)
```

### 关键字传参

- 靠参数名匹配
- 顺序相对自由

例如：

```python
def f(name, age):
    pass

f(age=18, name="Tom")
```

这也是合法的。

## 四、能不能混着传

可以。

例如：

```python
def f(a, b, c):
    pass

f(1, 2, c=3)
```

这里：

- `1`、`2` 是位置传参
- `c=3` 是关键字传参

## 五、混着传时最重要的规则

### 规则

**位置实参必须写在关键字实参前面。**

正确：

```python
f(1, 2, c=3)
```

错误：

```python
f(1, c=3, 2)
```

因为一旦开始写关键字实参，后面就不能再回去写普通位置实参。

## 六、什么时候适合位置传参

适合：

- 参数数量少
- 参数顺序很清楚
- 调用很短

例如：

```python
print("hello")
input("请输入名字：")
```

## 七、什么时候适合关键字传参

适合：

- 参数比较多
- 参数含义容易混
- 想让调用更清楚

例如：

```python
nums.sort(reverse=True)
"a,b,c".split(sep=",")
```

关键字传参的优点是可读性高。

## 八、位置传参与关键字传参的常见错误

### 错误 1

把只能位置传的参数写成关键字。

例如：

```python
input(prompt="请输入内容：")
```

这类情况就会出错，因为 `prompt` 在签名里是仅位置参数。

### 错误 2

位置实参写在关键字实参后面。

例如：

```python
f(a=1, 2)
```

这是语法错误。

### 错误 3

同一个参数被传了两次。

例如：

```python
def f(a, b):
    pass

f(1, a=2)
```

这里 `a` 已经通过位置传参拿到 `1` 了，又通过关键字传参给了 `2`，会报错。

## 九、和 `*args` / `**kwargs` 的关系

这两个不是“位置传参/关键字传参”的第三种。

它们是：

- `*args`：收多余位置参数
- `**kwargs`：收多余关键字参数

所以它们是对前两种传参方式的“接收机制”，不是新的传参类别。

## 十、最短总结

1. 位置传参靠顺序。
2. 关键字传参靠名字。
3. 关键字传参顺序相对自由。
4. 混合传参时，位置实参必须写在关键字实参前面。
5. `*args` 收多余位置参数，`**kwargs` 收多余关键字参数。
