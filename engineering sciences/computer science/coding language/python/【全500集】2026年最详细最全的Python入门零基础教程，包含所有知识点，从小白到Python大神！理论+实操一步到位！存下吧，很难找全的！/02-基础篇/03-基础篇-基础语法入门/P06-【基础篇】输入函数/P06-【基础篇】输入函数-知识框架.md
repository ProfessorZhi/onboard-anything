# P06 知识框架 - 【基础篇】输入函数
## 本P位置

Python基础语法系列 · 输入输出体系第2课
前置：P5 print函数（程序→用户单向输出）
本P：input函数（用户→程序单向输入）
后续：变量类型、类型转换综合应用

## 核心问题

程序如何从用户获取数据？获取的数据是什么类型？如何让程序正确处理输入的数字？

## 知识骨架

**input函数** → 获取控制台用户输入 → 返回**字符串类型** → 若需数值运算 → 需**类型转换**（int/float）

```
input(提示信息) → 返回 str 类型
int(input()) → 返回 int 类型
float(input()) → 返回 float 类型
```

## 关系与依赖

```
print函数 ←|→ input函数（输入输出互补，共同构成交互基础）

input函数依赖：
  ├── 变量赋值机制（右→左执行顺序）
  └── 控制台运行环境

input函数支撑后续：
  ├── 数据类型学习（str vs int vs float）
  └── 异常处理学习（类型转换失败场景）
```

## 输入 / 输出 / 前置 / 支撑

| 类别 | 内容 |
|------|------|
| **输入** | 用户在控制台输入的任意字符 |
| **输出** | 字符串类型的值（赋值给变量） |
| **前置知识** | print函数、变量定义、赋值语句执行顺序 |
| **支撑知识** | Python数据类型体系、类型转换函数int/float |

## 易错点

1. **input永远返回字符串**——即使输入"18"，类型也是str，不能直接做算术运算
2. **提示信息可省略但不应省略**——省略后用户看不到任何提示，不知道该输入什么
3. **类型转换必须匹配**——int("ABC")会报错，转换前需确认输入可转换
4. **赋值顺序陷阱**——`a = input()`先执行右边input再赋值，变量a获得的是字符串
