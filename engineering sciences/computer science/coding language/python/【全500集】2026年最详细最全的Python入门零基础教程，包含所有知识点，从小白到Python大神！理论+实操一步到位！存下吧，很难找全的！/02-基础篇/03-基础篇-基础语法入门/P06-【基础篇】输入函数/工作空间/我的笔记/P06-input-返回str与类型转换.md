# P06 input 返回 str 与类型转换

## 这节真正要抓住什么

`input()` 的本质不是“帮我拿到数字”，而是“从键盘读入一段文本”。

所以即使我输入的是 `18`，程序先拿到的也不是整数 `18`，而是字符串 `"18"`。

## 最核心结论

```python
age = input("请输入年龄：")
```

上面这行执行后：

- `age` 是一个变量名
- `age` 指向一个 `str` 对象
- 如果用户输入 `18`，那它指向的是字符串对象 `"18"`

也就是说，不是“变量没有类型”，而是“变量绑定到了一个类型为 `str` 的对象”。

## 为什么 Python 不自动帮我转成数字

因为键盘输入本来就是文本，Python 不应该替我猜业务含义。

比如用户可能输入：

- `18`
- `18.5`
- `0018`
- `18岁`
- `abc`

如果 Python 自动判断，很容易猜错。

所以更稳的策略是：

1. 先统一按字符串接收
2. 需要计算时，再由程序员明确转换成 `int` 或 `float`

## 为什么 `"18" + "1"` 可以，但 `"18" + 1` 不行

```python
print("18" + "1")   # 181
print("18" + 1)     # 报错
```

原因：

- 前者是字符串和字符串拼接
- 后者是字符串和整数混算，类型不一致

## `age = int(age)` 到底发生了什么

这行代码不是“把原来的字符串对象改成整数对象”。

真正发生的是：

1. 先读取 `age` 当前指向的字符串对象，比如 `"18"`
2. `int("18")` 创建一个新的整数对象 `18`
3. 变量 `age` 重新绑定到这个新的整数对象

所以这是**重新绑定**，不是**原地改类型**。

## 对象流转图

```python
age = input("请输入年龄：")
```

如果输入 `18`：

`age` -> `"18"` (`str`)

然后执行：

```python
age = int(age)
```

变成：

`age` -> `18` (`int`)

旧的 `"18"` 字符串对象如果没有别的引用，就会变成可回收对象。

## 旧对象什么时候会被回收

在 CPython 里，核心看“还有没有引用指向它”。

例如：

```python
age = "18"
other = age
age = int(age)
```

这时：

- `age` 指向新的 `int` 对象 `18`
- `other` 还指向旧的 `str` 对象 `"18"`

所以旧对象不会立刻回收。

只有当旧对象没有任何引用时，它才会被回收。

## 一句话总结

`input()` 先拿文本，`int()`/`float()` 再把文本变成数字，变量始终只是标签，对象才真正带着类型。
