# P06 map 对象讲解

## 一、先抓本质

`map` 不是列表。

它更像一个“按规则逐个产出结果”的对象：

- 规则：把某个函数作用到一组数据上
- 结果：不是立刻全部算好
- 取值时：才一个个算出来

最小例子：

```python
m = map(int, ["1", "2", "3"])

print(m)         # <map object at 0x...>
print(type(m))   # <class 'map'>
print(list(m))   # [1, 2, 3]
```

一句话：

`map(...)` 先得到一个 `map` 对象，不是直接得到列表。

## 二、`map` 本身是什么

`map` 本身是一个类型对象：

```python
print(map)
print(type(map))
```

结果大致是：

```python
<class 'map'>
<class 'type'>
```

所以：

- `map` 是类型对象
- `map(...)` 是调用这个类型对象创建实例

这和：

```python
list(...)
str(...)
int(...)
```

是同一种大方向。

## 三、`map(...)` 里面放了什么

从理解角度，可以把一个 `map` 对象想成大致保存了两样东西：

1. 要调用的函数
2. 要遍历的数据来源

例如：

```python
m = map(int, ["1", "2", "3"])
```

你可以粗略理解成：

- 函数是 `int`
- 数据来源是 `["1", "2", "3"]`

但它不会一开始就变成 `[1, 2, 3]`。

它只是先把“以后怎么计算”这件事记住了。

## 四、为什么 `print(m)` 不是 `[1, 2, 3]`

因为：

```python
print(m)
```

打印的是对象本身的表示，不是自动把里面的结果全部展开。

所以你看到的是：

```python
<map object at 0x...>
```

意思是：

- 这是一个 `map` 对象
- 地址每次可能不同
- 这不是结果列表

和下面这句不是一回事：

```python
print(list(m))
```

后者才是在真正把结果取出来。

## 五、为什么 `list(m)` 又可以

因为 `m` 是一个可迭代对象。

`list(...)` 的一个常见用法就是：

把可迭代对象里的内容一个个取出来，收集成新的列表。

例如：

```python
print(list("abc"))            # ['a', 'b', 'c']
print(list(range(3)))         # [0, 1, 2]
print(list(map(int, ["1"])))  # [1]
```

所以：

```python
list(m)
```

本质上是在做：

1. 从 `m` 里不断取下一个值
2. 把这些值装进一个新的 `list` 实例对象里

一句话：

`list(m)` 返回的是一个新的 `list` 实例对象。

## 六、为什么它像“一次性的”

看这个例子：

```python
m = map(int, ["1", "2", "3"])

print(list(m))   # [1, 2, 3]
print(list(m))   # []
```

原因是：

- 第一次 `list(m)` 已经把 `m` 里的值都取走了
- 第二次再取时，已经没有剩余结果了

所以你会觉得它像“一次性的”。

更准确地说：

`map` 常常是会被消费掉的对象。

## 七、它和列表到底差在哪

### `list`

更像一个已经装好结果的盒子：

```python
nums = [1, 2, 3]
print(nums)   # [1, 2, 3]
print(nums)   # [1, 2, 3]
```

### `map`

更像一条按规则往外吐结果的流水线：

```python
m = map(int, ["1", "2", "3"])
print(m)         # <map object at 0x...>
print(list(m))   # [1, 2, 3]
print(list(m))   # []
```

所以不要把 `map` 理解成“另一种列表”。

更好的理解是：

它是“能按规则产出一串结果的对象”。

## 八、在 P06 里最常见的用法

```python
a, b = map(int, input("请输入两个整数，用空格分隔：").split())
```

它拆开就是：

```python
text = input("请输入两个整数，用空格分隔：")
parts = text.split()
m = map(int, parts)
a, b = m
```

执行过程：

1. `input(...)` 得到一个字符串
2. `split()` 把它拆成字符串列表
3. `map(int, ...)` 把每个字符串准备转成整数
4. `a, b = ...` 取出两个整数

## 九、从 CPython 角度可以怎样理解

先说结论：

`map` 在 CPython 里不是“先算完再存好”的列表结构。

更像是一个对象，里面大致保存：

- 要调用的函数
- 一个或多个内部迭代器

每次要下一个结果时，底层大致做的是：

1. 从内部迭代器里取下一个元素
2. 用这些元素去调用那个函数
3. 把结果返回出去

所以它天然就是“边取边算”。

这也是为什么：

- `print(m)` 不会看到 `[1, 2, 3]`
- `list(m)` 会真正把结果一个个取出来
- 取完后通常就空了

这里先抓机制，不必死记 C 结构体细节。

## 十、最短对照表

| 写法 | 本质 | 结果类型 | 结果示例 |
| :--- | :--- | :--- | :--- |
| `map(int, ["1", "2"])` | 创建 `map` 对象 | `map` | `<map object at 0x...>` |
| `list(map(int, ["1", "2"]))` | 展开 `map` 结果并收集成列表 | `list` | `[1, 2]` |
| `map(str.upper, ["a", "b"])` | 让函数逐个作用到元素上 | `map` | `<map object at 0x...>` |

## 十一、最该记住的话

`map` 不是列表。

它是一个按规则、按需产出结果的对象。

你真正看到结果，通常是因为你把它展开了，例如：

```python
list(m)
```
