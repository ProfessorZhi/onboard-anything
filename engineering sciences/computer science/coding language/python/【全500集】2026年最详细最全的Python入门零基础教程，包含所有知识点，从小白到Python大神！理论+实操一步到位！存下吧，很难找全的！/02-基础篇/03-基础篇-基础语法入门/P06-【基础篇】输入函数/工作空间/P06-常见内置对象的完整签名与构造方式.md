# P06 补充笔记 - 常见内置对象的完整签名与构造方式

这份笔记不解决“怎么刷题”，而是解决你这轮反复追问的三个根问题：

1. 新名字到底是函数，还是类型对象
2. “构造”在 Python 里更准确是什么意思
3. 常见内置对象的完整签名该怎么看

## 先给最短结论

- `input` 更接近**内置函数对象**
- `map`、`int`、`float`、`str`、`list` 更接近**类型对象**
- 写 `map(...)`、`int(...)`、`list(...)` 时，更准确地说是“调用类型对象创建实例”
- 在入门阶段把它们叫“构造”可以帮助理解，但更严格地说，Python 更常讲：
  - 类型对象
  - 调用类型对象
  - 创建实例

## 一、完整签名怎么看

### 1. `input`

```python
input(prompt='', /)
```

含义：

- `prompt` 是提示词
- 默认值是空字符串
- `/` 表示前面的参数只能按位置传参

所以：

```python
input()
input("请输入年龄：")
```

合法，但：

```python
input(prompt="请输入年龄：")
```

不合法。

### 2. `str.split`

更适合你当前理解的一种写法：

```python
str.split(self, sep=None, maxsplit=-1)
```

这里的关键不是死记参数，而是看懂：

- 这是 `str` 类型上的方法
- `self` 是调用它的那个字符串实例
- 真正写代码时不会手写 `self`

例如：

```python
raw = "10 20"
parts = raw.split()
```

这里等价地理解为：

```python
str.split(raw)
```

只是平时我们写成点号调用。

### 3. `map`

常见可读写法：

```python
map(function, iterable, *iterables)
```

你现在最该抓住的是：

- 第一个参数是“要作用到每个元素上的函数”
- 后面参数是“提供元素的可迭代对象”

例如：

```python
map(int, ["1", "2"])
```

表示：

- 从 `["1", "2"]` 里依次拿元素
- 对每个元素调用 `int(...)`
- 返回一个 `map` 对象

### 4. `int`

入门层常用可读写法：

```python
int(x=0, /)
```

你当前最该抓住的是：

- `int("18")` 可以把整数字符串转成整数
- `int("18.0")` 不行，因为那不是“整数形式字符串”

### 5. `float`

```python
float(x=0.0, /)
```

你当前最该抓住的是：

- `float("18")` 可以
- `float("18.0")` 也可以

### 6. `list`

```python
list(iterable=(), /)
```

含义：

- 你可以把一个可迭代对象展开成列表

例如：

```python
list(map(int, ["1", "2"]))
```

会得到：

```python
[1, 2]
```

## 二、什么叫“构造”

你这轮一直在问“构造函数”“实例化方法”，这里要压成一个准确但不绕的版本。

### 教学近似

可以先把下面这些理解成“构造对象”：

```python
int("18")
list("abc")
map(int, ["1", "2"])
```

这对入门是有帮助的。

### 更严格一点

更准确地说：

- `int`、`list`、`map` 是类型对象
- 你写 `int(...)`、`list(...)`、`map(...)`，是在调用这些类型对象
- 调用后得到对应类型的实例

所以：

```python
type(map)   # <class 'type'>
```

而：

```python
m = map(int, ["1", "2"])
type(m)     # <class 'map'>
```

## 三、解包赋值到底是什么

你问的“等式左边两个元素这种写法是不是语法糖”，本质上是在问**解包赋值**。

例如：

```python
a, b = ["10", "20"]
```

要成立，右边必须能提供**正好两个值**。

所以这些都可以：

```python
a, b = ["10", "20"]
a, b = ("10", "20")
a, b = map(int, ["10", "20"])
```

因为它们都能按顺序提供两个元素。

但这会报错：

```python
a, b = ["10", "20", "30"]
```

因为右边给了三个值，左边只接两个变量。

## 四、为什么 `a, b = map(...)` 能成立

因为 `map(...)` 返回的虽然不是列表，但它是一个**可迭代对象**，能按顺序产出元素。

例如：

```python
a, b = map(int, ["10", "20"])
```

可以理解成：

1. `map` 对象先准备好一个“结果流”
2. 解包赋值从这个结果流里依次拿值
3. 第一个给 `a`
4. 第二个给 `b`

最终：

```python
a == 10
b == 20
```

## 五、你当前最该稳住的 5 句话

1. `input()` 返回的是 `str`。
2. `split()` 返回的是 `list[str]` 这种思路下的字符串列表。
3. `map(int, iterable)` 返回的不是列表，而是 `map` 对象。
4. `a, b = ...` 属于解包赋值，右边要能按顺序提供两个值。
5. `int`、`float`、`list`、`map` 更适合理解成“类型对象”，调用它们会得到实例。
