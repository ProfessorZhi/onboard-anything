# P06 字符串双引号的本质

## 这份笔记真正解决什么问题

避免把这两层混在一起：

1. 代码里写字符串时用的引号
2. 字符串对象真正存的内容

这是一个特别容易错的点，尤其在看 `f-string` 时最容易混。

## 一、先给结论

代码里的双引号，首先是 **语法符号**。

它的作用是告诉 Python：

“这里是一段字符串字面量。”

例如：

```python
"abc"
```

这里的双引号主要是代码语法的一部分，不是字符串内容本身的一部分。

## 二、字符串对象真正存的是什么

例如：

```python
s = "abc"
```

这时变量 `s` 指向的是一个字符串对象，它真正的内容是：

```python
abc
```

不是：

```python
"abc"
```

也就是说：

- 代码里你写了双引号
- 但对象里保存的是双引号包起来的那段内容

## 三、为什么 `print` 时看不到双引号

例如：

```python
s = "abc"
print(s)
```

输出是：

```python
abc
```

不是：

```python
"abc"
```

原因就是：

- 双引号只是代码里创建字符串时的语法边界
- 字符串对象真正存的是内容 `abc`
- `print()` 打印的是内容，不会把那层语法引号也打印出来

## 四、`f-string` 为什么更容易让人混

例如：

```python
label = "nums.__doc__"
print(f"{label}:")
```

这里容易错想成：

“替换后会不会变成两个双引号？”

不会。

真正发生的是：

- 外层 `"..."` 是代码里的字符串语法
- `{label}` 表示把变量 `label` 的值插进这个字符串里
- `label` 的值是 `nums.__doc__`
- 所以最后得到的是：

```python
nums.__doc__:
```

不是：

```python
"nums.__doc__":
```

## 五、把这件事拆成两层看

看这句：

```python
print(f"{label}:")
```

### 第一层：代码层

这是一个 `f-string`。

也就是：

```python
f"...{表达式}..."
```

外层双引号的作用是：

- 告诉 Python 这是字符串代码

### 第二层：运行时内容层

`{label}` 会被变量 `label` 的值替换。

如果：

```python
label = "abc"
```

那么最终得到的字符串内容是：

```python
abc:
```

不是：

```python
"abc":
```

## 六、最小实验

```python
label = "abc"

print(label)
print(f"{label}")
print(f"{label}:")
```

输出分别是：

```python
abc
abc
abc:
```

这三次都不会把代码里的双引号打印出来。

## 七、什么时候双引号才会真的变成内容的一部分

只有当你把双引号本身也写进字符串内容里时，它才会真的成为内容。

例如：

```python
s = '"abc"'
print(s)
```

这次输出才是：

```python
"abc"
```

因为这时双引号已经不是外层语法边界，而是字符串内容的一部分了。

## 八、最短总结

1. 代码里的双引号首先是语法，不一定是内容。
2. 字符串对象真正保存的是引号里面的内容。
3. `print()` 默认打印内容，不打印语法层那对引号。
4. `f"{label}"` 是把变量值插进字符串，不会把外层代码引号一起插进去。
