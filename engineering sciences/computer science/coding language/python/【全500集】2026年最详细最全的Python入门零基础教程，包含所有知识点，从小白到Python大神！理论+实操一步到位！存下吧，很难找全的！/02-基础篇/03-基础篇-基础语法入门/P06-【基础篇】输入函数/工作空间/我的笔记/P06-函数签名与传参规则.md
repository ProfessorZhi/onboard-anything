# P06 函数签名与传参规则

## 这份笔记真正解决什么问题

当你看到这种东西时：

```python
input(prompt='', /)
split(self, /, sep=None, maxsplit=-1)
sort(self, /, *, key=None, reverse=False)
```

你不再只会“看着像天书”，而是能读懂：

- 这个函数怎么调用
- 哪些参数能按位置传
- 哪些参数能按关键字传
- `self`、`/`、`*`、`*args`、`**kwargs` 分别是什么意思

## 一、签名是什么

### 定义

函数签名可以理解成：**调用规则说明书**。

例如：

```python
split(self, /, sep=None, maxsplit=-1)
```

它告诉你：

- 这个函数有哪些参数
- 参数顺序是什么
- 哪些参数有默认值
- 哪些参数只能按位置传
- 哪些参数可以按关键字传

一句话记忆：

**签名 = 调用规则**

## 二、参数传递的两种基本方式

### 1. 位置传参

按顺序传。

```python
"a,b,c".split(",")
```

这里 `","` 是按位置传给 `sep` 的。

### 2. 关键字传参

按参数名传。

```python
"a,b,c".split(sep=",")
```

这里是把 `","` 传给名字叫 `sep` 的参数。

一句话：

- 位置传参：靠顺序
- 关键字传参：靠名字

## 三、`self` 是什么

### 定义

`self` 表示：**当前这个对象自己**

比如：

```python
"abc".split()
```

这里调用 `split()` 时，当前对象就是：

```python
"abc"
```

所以可以把它理解成底层更接近：

```python
str.split("abc")
```

在签名里写成：

```python
split(self, /, sep=None, maxsplit=-1)
```

意思就是：

- 第一个位置其实是当前对象本身
- 这里的当前对象就是 `self`

### 一句话记忆

**self = 当前对象**

## 四、`/` 是什么意思

### 定义

`/` 前面的参数是**仅位置参数**。

也就是说：

- 可以按位置传
- 不能写成 `name=value`

例如：

```python
input(prompt='', /)
```

意思是：

`prompt` 这个参数只能按位置传。

正确：

```python
input("请输入内容：")
```

不推荐也通常不允许：

```python
input(prompt="请输入内容：")
```

再比如：

```python
split(self, /, sep=None, maxsplit=-1)
```

这里的 `self` 在 `/` 前面，所以它只能按位置传。

### 一句话记忆

**`/` = 前面的参数不能写成 `name=value`**

## 五、`*` 是什么意思

### 定义

`*` 后面的参数是**仅关键字参数**。

也就是说：

- 不能只靠位置传
- 必须写成 `name=value`

例如：

```python
sort(self, /, *, key=None, reverse=False)
```

这里：

- `key`
- `reverse`

都在 `*` 后面，所以它们必须按关键字传。

正确：

```python
nums.sort(reverse=True)
nums.sort(key=len)
```

不要理解成：

```python
nums.sort(True)
```

### 一句话记忆

**`*` = 后面的参数必须写成 `name=value`**

## 六、默认值是什么意思

例如：

```python
split(self, /, sep=None, maxsplit=-1)
```

这里：

- `sep=None`
- `maxsplit=-1`

表示这些参数可以不传。

如果不传，就使用默认值。

例如：

```python
"a b c".split()
```

等价于大致：

```python
"a b c".split(sep=None, maxsplit=-1)
```

## 七、`*args` 是什么

### 定义

`*args` 表示：**把多出来的位置参数，收集成一个元组。**

例如：

```python
def f(*args):
    print(args)

f(1, 2, 3)
```

输出：

```python
(1, 2, 3)
```

也就是说：

- 你传了 3 个位置参数
- Python 把它们打包成一个 tuple
- 交给 `args`

`args` 的本质就是个名字，常用叫法而已。你写成 `*numbers` 也完全可以。

### 为什么要用 `*args`

因为有时候不知道调用者会传多少个位置参数。

例如求和函数：

```python
def add(*args):
    return sum(args)

print(add(1, 2))
print(add(1, 2, 3, 4))
```

这样就能接收任意多个位置参数。

### 一句话记忆

**`*args` = 打包多余的位置参数**

## 八、`**kwargs` 是什么

### 定义

`**kwargs` 表示：**把多出来的关键字参数，收集成一个字典。**

例如：

```python
def f(**kwargs):
    print(kwargs)

f(name="Tom", age=18)
```

输出：

```python
{'name': 'Tom', 'age': 18}
```

所以 `name="Tom"`、`age=18` 这些关键字参数被打包成一个 dict，交给 `kwargs`。

同样，`kwargs` 也只是惯例名字，你写成 `**data` 也可以。

### 为什么要用 `**kwargs`

因为有时候不知道调用者会传哪些关键字参数。

例如：

```python
def print_info(**kwargs):
    for k, v in kwargs.items():
        print(k, v)

print_info(name="Alice", city="Beijing")
```

### 一句话记忆

**`**kwargs` = 打包多余的关键字参数**

## 九、`*` 单独出现，和 `*args` 不是一回事

这是最容易混的点。

### 情况 A：`*args`

```python
def f(*args):
    ...
```

这里的 `*args` 是"收集多余位置参数"。

### 情况 B：单独一个 `*`

```python
def f(a, *, b, c):
    ...
```

这里的 `*` 不是变量。它的作用只是：

**从这里往后的参数，必须关键字传参。**

例如：

```python
def f(a, *, b):
    print(a, b)
```

只能这样调用：

```python
f(1, b=2)
```

不能这样：

```python
f(1, 2)
```

所以：

- `*args`：收集
- 单独 `*`：分界线

## 十、把这些规则放到一张图里

看这个例子：

```python
def demo(a, b=10, /, c=20, *, d, e=30, **kwargs):
    pass
```

可以这样读：

- `a`
  - 仅位置参数
  - 必传
- `b=10`
  - 仅位置参数
  - 可不传
- `c=20`
  - 普通参数
  - 可位置传，也可关键字传
- `d`
  - 仅关键字参数
  - 必传
- `e=30`
  - 仅关键字参数
  - 可不传
- `**kwargs`
  - 接住多出来的关键字参数

## 十一、和你现在学的 `input` / `split` / `sort` 怎么连起来

### 1. `input(prompt='', /)`

```python
input("请输入内容：")
```

说明：

- `prompt` 是仅位置参数
- 不能按关键字名随便传

### 2. `split(self, /, sep=None, maxsplit=-1)`

```python
"a,b,c".split(",")
"a,b,c".split(sep=",")
"a b c".split(maxsplit=1)
```

说明：

- `self` 是当前字符串对象
- `sep` 和 `maxsplit` 可以不传
- `sep`、`maxsplit` 不在 `/` 前面，所以可以按关键字传

### 3. `sort(self, /, *, key=None, reverse=False)`

```python
nums.sort()
nums.sort(reverse=True)
nums.sort(key=len)
```

说明：

- `self` 是当前列表对象
- `key` 和 `reverse` 在 `*` 后面
- 所以它们必须按关键字传

## 十二、把四种规则放一起看

例子：

```python
def f(a, b, /, c, d=4, *args, e, f=6, **kwargs):
    print(a, b, c, d, args, e, f, kwargs)
```

这个签名拆开就是：

- `/` 左边：`a`, `b` → **只能位置传**
- `/` 右边到 `*args` 前：`c`, `d` → **位置或关键字都可以**
- `*args`：收集多余的位置参数
- `e`, `f`（在 `*args` 后面）：**只能关键字传**
- `**kwargs`：收集多余的关键字参数

调用例子：

```python
f(1, 2, 3, 5, 7, 8, e=9, x=10, y=11)
```

对应关系：

- `a = 1`
- `b = 2`
- `c = 3`
- `d = 5`
- `args = (7, 8)`
- `e = 9`
- `f = 6`（默认值）
- `kwargs = {'x': 10, 'y': 11}`

### 终极总纲图

```python
def func(pos_only, /, normal, *args, kw_only, **kwargs):
    pass
```

含义：

- `pos_only`：只能位置传
- `normal`：位置或关键字都行
- `*args`：收集额外位置参数
- `kw_only`：只能关键字传
- `**kwargs`：收集额外关键字参数

## 十三、调用时也能用 `*` 和 `**`

### 调用时的 `*`

把一个可迭代对象拆开成位置参数：

```python
def f(a, b, c):
    print(a, b, c)

nums = [1, 2, 3]
f(*nums)
```

等价于：

```python
f(1, 2, 3)
```

### 调用时的 `**`

把一个字典拆开成关键字参数：

```python
def f(name, age):
    print(name, age)

data = {"name": "Tom", "age": 18}
f(**data)
```

等价于：

```python
f(name="Tom", age=18)
```

## 十四、定义时和调用时别混了

| 场景 | 写法 | 作用 |
|------|------|------|
| 定义函数时 | `def f(*args, **kwargs)` | 收集多余位置/关键字参数 |
| 调用函数时 | `f(*lst, **dct)` | 把列表/字典拆开成参数 |

一个是**收集**，一个是**拆开**。

## 十五、为什么框架里老出现 `*args, **kwargs`

因为框架、装饰器、通用工具函数，经常不知道调用者会传什么参数。

例如装饰器：

```python
def wrapper(*args, **kwargs):
    print("before")
    return target(*args, **kwargs)
```

把任意参数原样转发给 `target`。

所以它们特别适合写：

- 装饰器
- 中间层
- 通用封装
- 框架代码

## 十六、参数之间用什么分隔

**本质：参数之间用逗号隔开。**

但有几层要分清：

### 定义时

```python
def f(a, b=10, *args, c, **kwargs):
    pass
```

逗号分隔各个参数项。项内部可能是：

- `a`（普通参数）
- `b=10`（带默认值）
- `*args`（收集多余位置）
- `c`（仅关键字）
- `**kwargs`（收集多余关键字）
- `/`（仅位置标记）
- `*`（仅关键字标记）

### 调用时

```python
f(1, 2, 3, x=10, y=20)
```

逗号分隔各个实参项。项内部可能是：

- `1`（位置传参值）
- `x=10`（关键字传参）
- `*nums`（拆可迭代对象）
- `**data`（拆字典）

### 什么不能代替逗号

空格、分号、冒号都不行。逗号是唯一的分隔符。

## 十七、最容易混淆的点

### 误区 1

“签名就是参数列表。”

不够准确。

更准确地说，签名不只告诉你“有哪些参数”，还告诉你“怎么传”。

### 误区 2

“`self` 是关键字。”

不是。

它本质上只是一个约定俗成的参数名，表示当前对象。

### 误区 3

“有 `/` 和 `*` 只是写法花哨。”

不是。

它们会直接改变调用规则。

### 误区 4

“`*args` 和 `**kwargs` 是固定魔法名字。”

不完全对。

星号本身才关键，名字可以改。

只是大家习惯写：

```python
*args
**kwargs
```

## 十八、现在最值得记住的 6 句话

1. 签名 = 调用规则。
2. 位置传参靠顺序，关键字传参靠名字。
3. `self = 当前对象`。
4. `/ = 前面的参数只能按位置传`。
5. `* = 后面的参数只能按关键字传`。
6. `*args` 收多余位置参数，`**kwargs` 收多余关键字参数。
