# P06 函数签名与传参规则

## 这份笔记真正解决什么问题

当你看到这种东西时：

```python
input(prompt='', /)
split(self, /, sep=None, maxsplit=-1)
sort(self, /, *, key=None, reverse=False)
```

你不再只会“看着像天书”，而是能读懂：

- 这个函数怎么调用
- 哪些参数能按位置传
- 哪些参数能按关键字传
- 为什么有时会看到 `self`
- 为什么实例方法签名里常常又看不到 `self`

一句话：

**签名 = 调用规则说明书。**

## 一、什么是签名

签名可以理解成：函数或方法的调用规则。

例如：

```python
split(self, /, sep=None, maxsplit=-1)
```

它告诉你：

- 有哪些参数
- 参数顺序是什么
- 哪些参数有默认值
- 哪些参数只能按位置传
- 哪些参数可以按关键字传

## 二、两种最基本的传参方式

### 1. 位置传参

靠顺序传。

```python
"a,b,c".split(",")
```

这里 `","` 是按位置传给 `sep` 的。

### 2. 关键字传参

靠名字传。

```python
"a,b,c".split(sep=",")
```

这里是把 `","` 传给名字叫 `sep` 的参数。

一句话：

- 位置传参：靠顺序
- 关键字传参：靠名字

## 三、`self` 是什么

`self` 表示：当前这个对象自己。

例如：

```python
"abc".split()
```

这里当前对象就是：

```python
"abc"
```

更接近底层的写法是：

```python
str.split("abc")
```

所以：

- `str.split(self, ...)`：是在类型对象视角下写方法定义
- `"abc".split(...)`：是在实例对象视角下调用绑定方法

一句话：

**self = 当前对象**

## 四、为什么实例方法签名里常常看不到 `self`

这是最容易混的一点。

看这两句：

```python
list.append(nums, 4)
nums.append(4)
```

它们本质上在做同一件事。

区别只在于：

- `list.append(nums, 4)`：你显式把 `nums` 当作 `self` 传进去
- `nums.append(4)`：Python 自动把 `nums` 绑定给 `self`

所以：

- 看类型对象上的方法定义时，经常会看到 `self`
- 看实例对象上的绑定方法时，`self` 经常不再显示

例如：

```python
list.append(self, object, /)
nums.append(object, /)   # 这里的 self 已经默认绑定
```

准确说，实例方法不是“没有 `self`”，而是：

**`self` 已经隐含传入了。**

## 五、`/` 是什么意思

`/` 前面的参数是仅位置参数。

也就是说：

- 可以按位置传
- 不能写成 `name=value`

例如：

```python
input(prompt='', /)
```

这里 `prompt` 只能按位置传。

正确：

```python
input("请输入内容：")
```

不允许：

```python
input(prompt="请输入内容：")
```

一句话：

**`/` = 前面的参数不能写成 `name=value`**

## 六、`*` 是什么意思

`*` 后面的参数是仅关键字参数。

也就是说：

- 不能再按位置传
- 必须写成 `name=value`

例如：

```python
sort(self, /, *, key=None, reverse=False)
```

这里：

- `self` 只能按位置
- `key` 和 `reverse` 只能按关键字

所以：

```python
nums.sort(reverse=True)
```

是正常的，而不是：

```python
nums.sort(True)
```

一句话：

**`*` = 后面的参数只能按关键字传**

## 七、`*args` 和裸 `*` 的区别

### 1. 裸 `*`

只负责“划分边界”。

例如：

```python
def f(a, *, b, c=10):
    pass
```

这里：

- `a` 可以按位置传
- `b`、`c` 只能按关键字传

### 2. `*args`

既负责“划分边界”，也负责“收集多余位置参数”。

例如：

```python
def f(a, *args, b):
    pass
```

这里：

- `a` 先吃第一个位置参数
- 多余位置参数进 `args`
- `b` 只能按关键字传

一句话：

- `*`：只划边界
- `*args`：划边界 + 收集多余位置参数

## 八、`**kwargs` 是什么

`**kwargs` 用来收集多余关键字参数。

例如：

```python
def f(**kwargs):
    print(kwargs)
```

调用：

```python
f(x=1, y=2)
```

得到：

```python
{'x': 1, 'y': 2}
```

一句话：

**`**kwargs` = 收集多余关键字参数，并打包成字典**

## 九、参数绑定时，解释器大致按什么顺序工作

可以粗略记成这条流程：

1. 先看签名分区
2. 先吃位置参数
3. 再吃关键字参数
4. 多余位置参数给 `*args`
5. 多余关键字参数给 `**kwargs`
6. 缺失必填参数就报错
7. 没传但有默认值的参数，用默认值

## 十、最短例子把这些点串起来

```python
def f(a, /, b, c=30, *args, d, e=50, **kwargs):
    pass
```

这行可以读成：

- `a`：仅位置参数
- `b`：普通参数，可位置可关键字
- `c=30`：普通参数，有默认值
- `*args`：收集多余位置参数
- `d`：仅关键字参数，必传
- `e=50`：仅关键字参数，有默认值
- `**kwargs`：收集多余关键字参数

例如：

```python
f(1, 2, 3, 4, 5, d=6, x=7, y=8)
```

大致绑定成：

```python
a = 1
b = 2
c = 3
args = (4, 5)
d = 6
e = 50
kwargs = {'x': 7, 'y': 8}
```

## 十一、这一页真正要记住什么

1. 签名就是调用规则。
2. 位置传参靠顺序，关键字传参靠名字。
3. `/` 前面只能位置传，`*` 后面只能关键字传。
4. `*args` 收多余位置参数，`**kwargs` 收多余关键字参数。
5. 实例方法不是没有 `self`，而是 `self` 已经默认绑定到当前对象了。
