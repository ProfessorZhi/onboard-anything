# P06 类型对象与构造式调用：map、list、str

## 这份笔记真正解决什么问题

你现在已经看到这些现象：

```python
map(int, ["1", "2", "3"])
list(m)
str(123)
```

它们看起来都像“函数调用”，但又不是普通函数。

这页就是把下面 4 个问题一次讲清：

1. `map`、`list`、`str` 这些到底是不是类型对象
2. `map(...)`、`list(...)`、`str(...)` 到底是在做什么
3. 这能不能叫“构造函数”
4. 为什么前面的 `list` / `str` 笔记没有把“构造式调用”当主线来讲

## 一、先给结论

### 1. `map`、`list`、`str` 都是类型对象

例如：

```python
type(map)   # <class 'type'>
type(list)  # <class 'type'>
type(str)   # <class 'type'>
```

说明：

- `map` 是类型对象
- `list` 是类型对象
- `str` 是类型对象

### 2. 调用这些名字，通常是在创建实例

例如：

```python
m = map(int, ["1", "2", "3"])
nums = list("abc")
s = str(123)
```

这些都可以理解成：

- 调用类型对象
- 创建对应类型的实例对象

所以：

- `map(...)` -> 创建 `map` 实例对象
- `list(...)` -> 创建 `list` 实例对象
- `str(...)` -> 创建 `str` 实例对象

## 二、`map` 这一条线怎么理解

### 1. `map` 本身是类型对象

```python
type(map)
```

结果：

```python
<class 'type'>
```

### 2. `map(...)` 是在创建实例

```python
m = map(int, ["1", "2", "3"])
```

这里发生的是：

- 调用 `map` 这个类型对象
- 创建一个 `map` 实例对象

所以：

```python
print(type(m))
```

结果：

```python
<class 'map'>
```

### 3. `print(m)` 为什么不是 `[1, 2, 3]`

因为这时你打印的是：

- `map` 对象本身

所以结果类似：

```python
<map object at 0x...>
```

这里的地址每次运行可能不同，不用背。

### 4. `list(m)` 为什么又可以

因为：

- `m` 是可迭代对象
- `list(...)` 会把可迭代对象里的内容一个个取出来
- 再收集进新的 `list` 实例对象里

例如：

```python
m = map(int, ["1", "2", "3"])
print(m)         # <map object at 0x...>
print(type(m))   # <class 'map'>
print(list(m))   # [1, 2, 3]
```

所以：

**`list(m)` 本质上是：调用 `list` 这个类型对象，创建一个新的 `list` 实例对象。**

## 三、`list` 和 `str` 也是一样的吗

方向上是一样的。

### 1. `list`

```python
type(list)   # <class 'type'>
```

调用：

```python
nums = list("abc")
```

结果：

```python
['a', 'b', 'c']
```

也就是说：

- `list` 是类型对象
- `list(...)` 是在创建 `list` 实例对象

### 2. `str`

```python
type(str)   # <class 'type'>
```

调用：

```python
s = str(123)
```

结果：

```python
'123'
```

也就是说：

- `str` 是类型对象
- `str(...)` 是在创建 `str` 实例对象

## 四、这能不能叫“构造函数”

### 1. 入门理解版

可以先这么理解：

- `map(...)` 像是在“构造”一个 `map` 对象
- `list(...)` 像是在“构造”一个 `list` 对象
- `str(...)` 像是在“构造”一个 `str` 对象

这个理解方向是对的，而且有助于入门。

### 2. 更规范一点的说法

更规范地说：

- `map` / `list` / `str` 是类型对象
- 调用它们，是在调用类型对象来创建实例

为什么我不建议你现在把它们直接全都叫“构造函数”？

因为在 Python 里，严格往下讲还会牵涉：

- `__new__`：更偏“创建对象”
- `__init__`：更偏“初始化对象”

所以：

- 入门阶段说“构造式调用”没问题
- 更严谨时，说“调用类型对象创建实例”更稳

一句话：

**可以先把 `list(...)`、`str(...)`、`map(...)` 理解成构造式调用，但别急着把它直接等同于 `__init__`。**

## 五、每个类型对象都有自己的“构造式调用”吗

大方向上，可以这么理解：

- 很多类型对象都可以被调用
- 调用后会创建对应类型的实例对象

例如：

```python
int("123")
float("3.14")
str(123)
list("abc")
tuple([1, 2, 3])
set([1, 2, 2, 3])
dict([("a", 1), ("b", 2)])
map(int, ["1", "2"])
```

但要注意：

- “能不能调用”
- “调用时吃什么输入”
- “返回什么类型”

这些规则因类型而异。

所以不是一句“都有构造函数”就能讲完。

更稳的说法是：

**很多类型对象都支持被调用，但各自的调用规则、输入要求、输出结果都不同。**

## 六、为什么前面的 `list` / `str` 笔记没有按这个角度展开

这是因为前面的主目标不是讲“怎么创建对象”，而是讲：

1. 实例对象和类型对象怎么区分
2. 方法为什么挂在类型对象上
3. 实例方法为什么常常看不到 `self`
4. `list` 和 `str` 的常见方法与数据属性

也就是说，前面那两份笔记的主线是：

- 对象模型视角

而不是：

- 构造式调用视角

所以没有重点去讲：

- `list(...)` 吃什么输入
- `str(...)` 吃什么输入
- `map(...)` 返回什么对象

不是漏了，而是那两份笔记解决的是另一个问题。

你现在追到这里，说明学习阶段已经往前推了一步，所以单开这一页是更合适的。

## 七、如果从“构造式调用”角度看，`map` / `list` / `str` 的最短对照

| 名字 | 它本身是什么 | 调用后得到什么 | 最小例子 |
|---|---|---|---|
| `map` | 类型对象 | `map` 实例对象 | `map(int, ["1", "2"])` |
| `list` | 类型对象 | `list` 实例对象 | `list("abc")` |
| `str` | 类型对象 | `str` 实例对象 | `str(123)` |

## 八、这三者的输入和输出也一起记

| 写法 | 常见输入 | 常见输出 | 说明 |
|---|---|---|---|
| `map(int, ["1", "2"])` | 一个函数 + 可迭代对象 | `map` 对象 | 结果通常要再展开 |
| `list("abc")` | 可迭代对象 | `list` 对象 | 会把内容收集成列表 |
| `str(123)` | 任意可转字符串对象 | `str` 对象 | 会得到字符串表示 |

## 九、最短记忆版

1. `map`、`list`、`str` 本身都是类型对象。
2. `map(...)`、`list(...)`、`str(...)` 都可以理解成“构造式调用”。
3. 更严谨地说，是“调用类型对象来创建实例”。
4. 前面的 `list` / `str` 笔记主要在讲对象模型，不是在讲构造式调用，所以没有把这条线当主线展开。
