# P06 进阶测试 - 输入函数

这组题不再考“`input()` 返回什么”这种单点记忆，
而是考你是否真的看清了下面这条数据流：

`input()` -> `str` -> `split()` -> 多个 `str` -> `int/float/map` -> 赋值/计算

默认要求：

- 不运行代码，先靠脑子判断
- 说结论时，必须说“哪一步发生了什么”
- 如果会报错，要说出报错发生在什么阶段

---

## 一、机制追踪题

### 1. 逐步追踪

用户输入：

```python
10 20
```

请分析下面代码每一步的结果：

```python
raw = input("请输入两个整数：")
parts = raw.split()
nums = map(int, parts)
a, b = nums
```

你要回答：

1. `raw` 的值和类型
2. `parts` 的值和类型
3. `nums` 打印出来大致长什么样
4. `a` 和 `b` 的值与类型

### 2. 同样输入，不同写法

用户输入：

```python
3 5
```

比较下面两段代码，说明它们在“中间过程”上有什么本质区别。

```python
# 写法 A
a, b = input().split()
print(a + b)
```

```python
# 写法 B
a, b = map(int, input().split())
print(a + b)
```

你要回答：

1. 哪一段会报错
2. 报错的根因是什么
3. 两段代码里 `a`、`b` 的类型分别是什么

### 3. 看起来像一样，其实不一样

判断下面三句代码的结果类型：

```python
a = input()
b = input().split()
c = map(int, input().split())
```

你要分别写出：

1. `type(a)`
2. `type(b)`
3. `type(c)`
4. 哪一个最适合直接做算术计算，为什么

## 二、报错定位题

### 4. 精确判断哪一步炸掉

用户输入：

```python
18 abc
```

代码：

```python
x, y = map(int, input("请输入两个值：").split())
print(x + y)
```

回答：

1. 这段代码会不会报错
2. 如果报错，错在 `input`、`split`、`map`、赋值、还是 `print`
3. 为什么不是其他步骤先报错

### 5. 个数不匹配

用户输入：

```python
1 2 3
```

代码：

```python
a, b = input().split()
```

回答：

1. 程序是否报错
2. `split()` 本身有没有错
3. 真正出错的是哪一步
4. 这类错误的本质是什么

### 6. 类型转换边界

分别判断下面哪些表达式会成功，哪些会报错，并说明原因：

```python
int("18")
int("18.0")
float("18")
float("18.0")
int("-3")
float("-3.5")
```

## 三、签名与参数规则题

### 7. `input` 的签名到底约束了什么


判断下面哪些合法，哪些不合法，并说明原因：

### 8. 你最近在研究的“对象类型”题

判断下面四个名字各自更接近哪一类对象：

```python
input
map
str
list
```

你至少要回答到这一层：

1. 哪个是内置函数对象
2. 哪些是类型对象

### 9. 参数绑定升级题

已知：

```python
def f(a, /, b, *args, c, d=10, **kwargs):
```

## 四、改写与等价题

### 10. 拆开写

把下面这句代码改写成“初学者最容易看懂”的 4 步版本：

```python
a, b = map(int, input("请输入两个整数：").split())
```

要求：

1. 每一步都落到一个变量上
2. 显示写出类型转换
3. 最终能解释每个变量里装的是什么

### 11. 压缩写

把下面代码压缩成一行：

```python
raw = input("请输入两个小数：")
parts = raw.split()
x = float(parts[0])
y = float(parts[1])
print(x + y)
```

同时回答：压缩之后，为什么“可读性可能变差但表达力更强”？

## 五、最小编程题

### 12. 三数平均值

要求用户一行输入 3 个数字，用空格分隔。程序输出它们的平均值。

要求：

1. 必须用一次 `input()`
2. 必须能处理小数
3. 不允许写三次 `input()`

### 13. 姓名与分数混合输入

用户一行输入：

```python
张三 95 88
```

请写代码得到：

- `name` 是字符串
- `math_score` 是整数
- `english_score` 是整数
- 最后输出总分

要求你自己决定是“分步写”还是“紧凑写”，但你要能解释为什么这么选。

### 14. 输入格式诊断题

下面这段代码本来想让用户输入两个整数并求和：

```python
a, b = map(int, input("请输入两个整数："))
print(a + b)
```

它在很多情况下都不对。

请回答：

1. 它错在哪里
2. 如果用户输入 `12 34`，程序会发生什么
3. 正确写法是什么
4. 根因是“`input` 不行”还是“少了中间步骤”

### 15. 真正难一点的综合题

不用异常处理，只在“用户按要求输入”的前提下，完成下面程序：

- 用户输入一行内容：姓名、年龄、身高
- 例如：

```python
Alice 18 165.5
```

- 你要得到：
  - `name` 是 `str`
  - `age` 是 `int`
  - `height` 是 `float`
- 最后输出一句完整的话：

```python
Alice 明年 19 岁，当前身高 165.5 cm
```

额外要求：

1. 只用一次 `input()`
2. 至少给出两种写法：
   - 分步写法
   - 紧凑写法
3. 比较两种写法谁更适合现在的你
