# P06 多个输入的两种方式

## 先抓核心

`input()` 一次只读取一行，并返回一个 `str`。

所以“输入多个值”不是 `input()` 突然变强了，而是我们用了两种不同策略：

1. 调用多次 `input()`
2. 调用一次 `input()`，再把拿到的一整行拆开

## 方式一：多次 `input()`

最适合入门，也最直观。

```python
name = input("请输入名字：")
age = int(input("请输入年龄："))
city = input("请输入城市：")
```

特点：

- 一次问一个
- 程序流程清楚
- 最适合刚学输入函数时使用

## 方式二：一次 `input()`，一行输入多个值

例如用户输入：

```python
10 20
```

`input()` 实际拿到的是：

```python
"10 20"
```

这时要靠 `split()` 来拆。

```python
a, b = input("请输入两个整数，用空格分隔：").split()
```

这一步之后：

- `a` 是 `"10"`
- `b` 是 `"20"`

它们仍然还是字符串。

如果要计算，还要继续转类型：

```python
a = int(a)
b = int(b)
print(a + b)
```

## 更紧凑的写法

```python
a, b = map(int, input("请输入两个整数，用空格分隔：").split())
```

这行的意思是：

1. `input()` 读取整行文本
2. `split()` 按空格切成多个字符串
3. `map(int, ...)` 把每个字符串转成整数
4. 最后分别赋给 `a` 和 `b`

这写法很常见，但对入门来说，先理解拆开的版本更重要。

## 什么时候用哪种

### 用多次 `input()`

适合：

- 刚入门
- 每个输入含义都不同
- 你希望程序提示更清楚

### 用一次 `input()` + `split()`

适合：

- 一行里输入多个同类值
- 竞赛题、刷题题目
- 想让输入更紧凑

## 最容易错的地方

### 错误 1

以为 `input()` 可以直接返回多个数字。

错。

它先返回的仍然是一整行字符串。

### 错误 2

`split()` 之后就以为已经是整数了。

错。

`split()` 只是拆字符串，不负责类型转换。

### 错误 3

输入个数和接收变量个数不一致。

例如：

```python
a, b = input().split()
```

如果用户输入了 3 个值，或者只输入了 1 个值，就会报错。

## 一句话总结

多个输入的本质不是“多参数 input”，而是：

- 要么多次读
- 要么一次读完再拆
