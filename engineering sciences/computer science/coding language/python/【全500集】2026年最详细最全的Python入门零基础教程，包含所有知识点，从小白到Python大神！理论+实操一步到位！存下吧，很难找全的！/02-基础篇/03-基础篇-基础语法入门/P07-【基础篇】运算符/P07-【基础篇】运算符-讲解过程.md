# P07 讲解过程 - 【基础篇】运算符
## 本P定位

本 P 属于 Python 基础语法的运算符专题。老师把运算符类比为 Python 中的"计算工具"和"赋值工具"，指出在实际编程中如果需要对数值进行计算或修改变量的值，都离不开这两类运算符。

## 讲解过程

### 一、引入：运算符是什么

老师开门见山说明本 P 要学习算术运算符和赋值运算符两类，分别负责数学计算和修改变量值的功能，直接对应日常编程中"算"和"改"两类核心操作。

### 二、算术运算符：七个计算工具

老师首先明确算术运算符的适用范围是整数和小数等数值类型，运算逻辑与小学数学基本一致。然后逐一讲解七个算术运算符：

**1. 加法（+）和减法（-）**
加号用来把两个数相加，举例 `6 + 3 = 9`。减号是前面数减去后面数。

**2. 乘法（*）和除法（/）**
乘法用星号 `*` 表示而非日常的叉号。除法用斜杠 `/` 而非日常的除号。老师打开一个名为 `PYOPERATOR` 的 Python 文件开始演示。先运行 `print(1 / 2)` 得到 `0.5`（即三分），运行 `100 - 3` 得到 `97`，运行 `2 * 3` 得到 `6`。接着演示除法：运行 `10 / 4` 得到 `2.5`，运行 `10 / 5` 得到 `2.0`。老师特别强调**除法结果必定是浮点数**，无论能否整除，结果都会是浮点数类型。

**3. 取整除（//）**
用两个斜杠 `//` 表示，只取整数部分且向下取整。老师用火柴实物演示：拿 7 根火柴，两根两根一捆，捆成三捆，剩下 1 根。整除结果为 3，余数为 1。在代码中运行 `7 // 2` 得到 3，`7 % 2` 得到 1。再演示 `10 // 3` 和 `10 % 3`，通过 10 根火柴三根三根一捆的方式，说明整除结果为 3、余数为 1。

**4. 取余数（%）**
用百分号表示，返回不够除的数。继续上面的火柴例子，`7 % 2` 得到 1，即剩余的火柴数量。

**5. 幂运算（**）**
双星号表示幂运算，前者为底数、后者为指数。运行 `2 ** 3` 即 2 的三次方得到 8，运行 `3 ** 2` 即 3 的二次方得到 9。

**6. 关键注意事项：除数不能为零**
老师演示三种情况都会报错：`10 / 0`、`10 // 0`、`10 % 0`，均报出 `ZeroDivisionError`（除数为零错误）。强调这是数学规定，在让用户执行计算时要考虑这种边界情况，避免程序崩溃。

**7. 浮点数参与的运算规则**
老师演示只要运算中有浮点数参与，结果通常也是浮点数。运行 `2 + 2.0` 得到 `4.0`，加法、减法、乘法、除法、取整除都是如此。老师用集合思想解释：浮点数的范围比整数广，整数可以看作浮点数 1.0，但浮点数 1.5 不能简单看作整数 1 或点一，因此精度更高的浮点数类型作为结果的默认类型。

**8. 算术运算符优先级**
老师说明优先级与数学完全一致：括号 > 幂运算 > 乘 > 除 > 取整除 > 取余 > 加 > 减，同级运算从左往右计算。举例 `3 + 4 * 2` 先算 `4 * 2 = 8`，再算 `3 + 8 = 11`；`(3 + 4) * 2` 则先算括号内 `3 + 4 = 7`，再算 `7 * 2 = 14`。老师建议如果记不清优先级，直接加括号最简单且不容易出错。

### 三、赋值运算符：从简单赋值到复合赋值

老师先回顾简单赋值运算符 `=` 的三种用法：直接赋值、把变量值赋给另一个变量、把表达式结果赋给变量。强调 `=` 不是数学中的"等于"，而是"赋值"的意思。

**重点讲复合赋值运算符**，它的作用是**先对变量里的数值做运算，再把新结果重新装回该变量**，写法更简洁且可维护性更高。老师通过余额场景逐步演示问题所在：

```python
money = 1000  # 初始余额
money + 500   # 仅计算，结果未保存
print(money)  # 仍为1000，因为没有重新赋值
```

老师解释代码执行顺序：先执行 `money = 1000`，然后 `money + 500` 只执行了一次计算，并未修改 `money` 的值。如果想修改，必须把计算结果重新赋值给 `money`：

```python
money = 1000
money = money + 500  # 先算右边1000+500=1500，再赋值给左边的money
print(money)         # 输出1500
```

老师指出这里 `money` 出现了两次，如果后续要修改变量名，需要改两处，可维护性差。**引出复合赋值运算符 `+=`**：

```python
money = 1000
money += 500  # 等价于 money = money + 500
print(money)  # 输出1500
```

老师明确说明 `money += 500` 和 `money = money + 500` 完全等价，但前者更简洁，只需修改一处，代码可维护性更高。

然后演示其他复合赋值运算符：
- `-=`：`money = 1000; money -= 200` 等价于 `money = money - 200`，结果为 800。
- `*=`、`/=`、`//=`、`%=`、`**=` 等逻辑一致，均为先计算再赋值。

老师强调一个重要前提：**复合赋值运算符必须先定义变量**，否则会报 `NameError`（名称未定义错误）。演示直接写 `num += 5` 而不先 `num = 某个值` 会报错，正确的做法是先给变量赋一个初始值。

### 四、实用技巧与练习建议

- 日常最常用的算术运算符是加减乘除和取余数。
- 取余数 `%` 可用于判断奇偶性：一个数对 2 取余，余数为 0 则是偶数，余数为 1 则是奇数。
- 复合赋值中 `+=` 和 `-=` 使用频率最高，`*=`、`/=`、`//=` 等较少使用。
- 老师建议课后多练习，比如计算周长、面积，或用复合赋值改变变量值。

## 本P结论

1. **算术运算符共七个**：`+` `-` `*` `/` `//` `%` `**`，运算逻辑与数学一致，但 `*` 用星号、`/` 用斜杠、`//` 用双斜杠。优先级遵循括号 > 幂 > 乘除取整取余 > 加减。
2. **除法结果必定是浮点数**，且除数不能为零，否则 `ZeroDivisionError`。
3. **运算中有浮点数参与，结果通常也是浮点数**，因为浮点数范围更广、精度更高。
4. **赋值运算符分两类**：简单赋值 `=` 用于装数据；复合赋值 `+=` `-=` `*=` 等用于修改数据——先计算再重新赋值，比重复写变量名更简洁、可维护性更高。
5. **使用复合赋值运算符前必须先定义变量**并赋予初始值，否则 `NameError`。
6. 分不清优先级时直接加括号，简单且不易出错。
7. 取余 `%` 可用于判断奇偶性，课后多练计算类题目。
