# P07 知识框架 - 【基础篇】运算符
## 本P位置

Python基础语法 → 运算符专题 → 下一P承接比较/逻辑运算符

本P是数据类型（int/float）的第一次实战应用，为后续条件判断、循环提供计算能力支撑。

## 核心问题

**数值如何计算？变量值如何修改？**

两个子问题：
- 对数值做运算得到新值
- 对已存在变量的值做运算并更新

## 知识骨架

```
运算符
├── 算术运算符（7个）→ 产生新值
│   ├── + - * /
│   ├── // 取整除
│   ├── % 取余
│   └── ** 幂
└── 赋值运算符（2类）→ 修改变量
    ├── = 简单赋值
    └── += -= *= /= //= %= **= 复合赋值
```

## 关系与依赖

- **依赖前置**：变量概念、数值类型（int/float）
- **被后续依赖**：表达式、控制流中的条件计算
- **运算符优先级**：括号 > 幂 > 乘除取整取余 > 加减（同级左到右）

## 输入 / 输出 / 前置 / 支撑

| 运算类型 | 输入 | 输出 | 前置条件 |
|---------|------|------|---------|
| 算术运算 | int/float | int/float | 无（自包含） |
| 简单赋值 | 任意值 | 无（返回None） | 无 |
| 复合赋值 | int/float | 无（返回None） | 变量已定义+赋初值 |

**两条隐性规则**：
- 只要有浮点数参与算术运算，结果通常是浮点数
- 除法运算（/ // %）的除数不能为零

## 易错点

1. **除法必返float**：`10 / 5` 输出 `2.0` 而非 `2`
2. **除数为零报错**：`ZeroDivisionError`，需在用户输入场景做防御
3. **复合赋值前未定义变量**：`NameError`，必须先赋初值
4. **`=` 是赋值不是等于**：比较用 `==`
5. **优先级记不住时**：直接加括号，最简单
