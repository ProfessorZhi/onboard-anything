# P16 知识框架 - 【基础篇】简单if语句
## 本P位置

- **课程定位**：Python基础教程 → 流程控制语句 → 条件语句首次引入
- **前置内容**：变量、数据类型、运算符等基础知识
- **后续内容**：P17 else语句，P18 elif语句 → 构建完整条件分支体系
- **本P作用**：建立「程序能做选择」的认知，理解分支执行的基本逻辑

## 核心问题

- 程序如何根据不同情况执行不同代码？
- 如何用代码表达「条件成立」与「条件不成立」？
- 如何判断一个整数是偶数？
- 为什么Python用缩进区分代码块？

## 知识骨架

```
流程控制语句
└── 条件语句
    └── 简单if语句（本P）
        ├── 语法格式：if 条件表达式:
        ├── 条件表达式 → 布尔值（True / False）
        ├── True → 执行代码块
        ├── False → 跳过代码块
        └── 缩进规则（代码块标识）
```

**关键知识点：**

- **if语句**：程序分支的入口，根据条件决定执行路径
- **条件表达式**：产生布尔值的表达式
- **取模运算符 %**：判断整除，余数为0则整除
- **缩进**：Python代码块的结构化标识

## 关系与依赖

- **依赖**：if语句依赖布尔表达式，布尔表达式依赖比较/算术运算符
- **支撑**：取模运算 % 是本P实例「偶数判断」的数学工具
- **扩展**：if → if-else → if-elif-else 形成完整分支体系

## 输入 / 输出 / 前置 / 支撑

- **输入**：一个变量值（如 number = 16）
- **输出**：条件成立时执行代码块，不成立时跳过
- **前置知识**：变量赋值、算术运算符（%、==）
- **支撑工具**：PyCharm IDE + Debug调试器 + 断点

## 易错点

- 条件表达式后忘记英文冒号 `:` → SyntaxError
- 条件表达式内写人类语言「是偶数」→ 需转化为代码逻辑
- 普通除法 `/` 产生浮点数，无法用于整除判断
- 整除 `//` 只返回商，无法判断是否整除
- 代码块缩进不一致 → IndentationError
- 误以为缩进代码块外的语句受if控制 → 实际同级语句必然执行

## 外部资源与对象

- **PyCharm**：本P演示用IDE，用于编写和运行Python代码
- **Debug调试器**：PyCharm内置工具，逐步执行观察变量值与分支走向
- **断点（Breakpoint）**：Debug时设置的暂停标记，用于控制单步执行位置
