# P18 知识框架 - 【基础篇】多分支判断与分支嵌套
## 本P位置

- 课程定位：Python基础入门第18集
- 所属模块：流程控制·条件判断
- 承接：P17单分支if语句
- 铺垫：后续循环、函数等复杂逻辑的前置基础
- 作用：建立分支结构选型意识，理解条件关联性与执行效率的关系

## 核心问题

如何在程序中根据不同条件执行不同代码？当条件互斥时如何选型才能兼顾正确性与效率？

## 知识骨架

```
条件判断体系
├── 单分支 if
├── 二选一 if else → 三元表达式（简化形式）
├── 多选一 if elif else
└── 嵌套 if（if嵌套、if else嵌套、if elif嵌套）
```

## 关系与依赖

```
if单分支 ──┬── if else ─── 三元表达式
           │
           └── if elif else（依赖if else语法）

分支嵌套（组合任意上述结构）：
  外层 if/if else/if elif
    └─ 内层 if/if else/if elif else
```

- if else 是所有分支结构的语法基础
- if elif else 由多个 if else 串联形成
- 分支嵌套是结构递归组合，不引入新语法关键字
- 所有分支结构共享同一套比较运算符和逻辑运算符

## 输入 / 输出 / 前置 / 支撑

**输入**：用户通过input()传入数据（字符串）
**类型转换**：数值判断需float()，会员判断需.lower()统一大小写
**输出**：print()打印判断结果或提示信息

**前置知识**：

- 布尔值（True/False）
- 比较运算符（==、!=、>、<、>=、<=）
- 逻辑运算符（and、or、not）
- Python缩进规则（每层4空格）

**支撑工具**：
IDE的debug调试功能——逐行追踪分支执行路径，可视化条件判断过程

## 易错点

**1. 赋值与比较混淆**

- 一个=是赋值
- 两个==是判断相等
- 两者在if条件中语义完全不同，绝对不能混用

**2. 边界条件遗漏**

- 条件区间要首尾衔接，如 score>=90 和 score<90 之间是否存在空白
- 浮点数场景注意精度，如89.5这类边界值是否被覆盖

**3. 独立if与分支结构混用**

- 互斥条件应使用if else或if elif，而非两个独立if
- 独立if会无条件执行全部判断，分支结构遇成立则跳过后续

**4. 嵌套层数失控**

- 超过三层时考虑重构，避免代码可读性崩溃

**5. 缩进不一致**

- Python以缩进区分代码块，缩进错误直接导致逻辑错误或语法错误
