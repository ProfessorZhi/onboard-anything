# P18 讲解过程 - 【基础篇】多分支判断与分支嵌套
## 本P定位

本P位于Python基础入门课程的第18集，承接前面对单分支if语句的讲解，系统性地引入多分支判断与分支嵌套的概念。教学目标分为三个层次：理解二选一逻辑（if else）、掌握多选一逻辑（if elif else）、学会在条件分支内部再嵌入条件判断（分支嵌套）。整P围绕“效率”这一核心主题展开，通过对比独立if语句与关联分支结构的执行差异，帮助学习者建立正确的分支结构选型意识。

## 讲解过程

### 一、引入：什么是多分支语句

课程从第4秒开始正式引入多分支语句的概念。教师首先指出接下来要学习的是多分支语句，但在深入讲解之前，需要先理解一个更基础的结构——if else。教师通过流程图辅助说明：条件表达式会返回一个布尔值（true或false），当条件为true时执行代码块一（即if下的缩进代码），条件为false时执行代码块二（即else下的缩进代码）。教师强调，if else的核心特征是“二选一”——两个代码块中必选其中一个执行。

在格式层面，教师指出if else的写法是在原有if语句的基础上追加else部分。else下面放置的是条件不成立时要执行的代码块，同样需要缩进，可以是一行也可以是多行代码。

### 二、实例演示：判断是否及格

从第67秒开始，教师以“判断成绩是否及格”为例进行实操演示。需求描述为：用户输入分数，系统判断是否及格。教师指出，因为分数可能存在小数（如99.5分、0.5分的题目），所以需要将输入转换为浮点数（float）而非整数，浮点数的表示范围更广。

初始代码采用单独if语句实现：

```
score = float(input("请输入分数"))
if score >= 60:
    print("及格")
if score < 60:
    print("不及格")
```

教师随后演示运行效果：当输入68分时输出“及格”，输入35分时输出“不及格”，确认代码能够满足需求。

### 三、效率对比：独立if语句 vs if else结构

第220秒进入本P的核心教学点。教师提出关键问题：既然单独的两个if语句也能满足需求，为什么还需要if else结构？为了说明这个问题，教师引入了debug调试。

以79分为例，教师演示了单独if语句的执行流程：首先判断第一个条件（score >= 60），条件成立，执行print("及格")。此时程序已经知道结果应该是“及格”，但由于第二个if是独立的语句，程序会继续判断第二个条件（score < 60）。教师指出，这个第二次判断是完全不必要的——既然已经确定及格，就没必要再去看“不及格”的条件，这会浪费一点执行时间。

教师总结：当条件具备关联性时（即一个条件成立另一个必然不成立，或一个条件的结果决定了另一个条件的状态），应该使用关联的分支结构（if else或if elif），而不是两个独立的if语句。独立if语句会不管前面条件是否成立都执行全部判断，而关联分支结构当前面条件成立时会直接跳过后面的判断，从而提升执行效率。

### 四、debug演示：if else的执行流程

第335秒开始，教师使用debug模式完整演示if else的执行流程。

**场景一：输入98分**

- 条件score >= 60成立
- 直接执行if下的print语句
- if else结构整体结束
- 程序结束（因为后面没有其他代码）

**场景二：输入48分**

- 条件score >= 60不成立
- 跳过if分支，直接执行else下的print语句
- 输出“不及格”
- if else结构整体结束

通过debug调试，学习者可以直观看到：if else结构中，只要前面某个条件成立，后面的所有条件都不会再被判断。

### 五、三元表达式：if else的简化写法

第398秒开始，教师介绍if else的简化形式——三元表达式。教师指出，三元表达式适合简单的二选一场景，能够将if else写成一行代码。

三元表达式的格式为：

```
结果1 if 条件 else 结果2
```

教师演示如何将四行if else代码简化为一行的过程：

**原始代码（4行）：**

```
if score >= 60:
    print("及格")
else:
    print("不及格")
```

**简化版本1（1行）：**

```
print("及格" if score >= 60 else "不及格")
```

教师强调，三元表达式只适用于if和else下各只有一行代码的情况。如果有多行代码，就不能使用三元表达式，应该老老实实使用标准的if else结构。

**简化版本2：进一步提取print**

教师还演示了另一种简化思路：观察发现条件为真和为假时的结果都有print，可以将print提到表达式外面。

```
print("及格" if score >= 60 else "不及格")
```

教师通过运行验证：当输入68分时输出“及格”，输入38分时输出“不及格”，确认一行代码可以替代原来的四行代码。

### 六、if elif else：多选一结构

第618秒开始进入第二种多分支结构——if elif else。教师指出，if elif else适合有多个条件的场景，按照顺序判断每一个条件，只要其中一个条件成立就执行对应的代码块，后面的条件不再判断。如果所有条件都不成立，就执行else下的代码块。

教师借助流程图详细讲解执行流程：

- 条件1成立 → 执行语句块1 → 程序结束（跳出整个if elif else结构）
- 条件1不成立 → 判断条件2
  - 条件2成立 → 执行语句块2 → 程序结束
  - 条件2不成立 → 判断条件3
    - 以此类推……
- 所有条件都不成立 → 执行else下的语句块 → 程序结束

教师特别强调，else是充当默认分支的角色，可以单独是if elif（没有else），但加上else会更加完善。

### 七、实例演示：分数评级

第701秒开始，教师以“根据分数评级”为例演示if elif else的用法。需求是：根据用户输入的分数，输出对应的等级（优秀、良好、及格、不及格）。

教师首先定义了分数区间：

- 90~100分：优秀
- 75~90分：良好
- 60~75分：及格
- 0~60分：不及格

教师指出，如果使用独立的if语句，前面条件满足后没有必要的判断还会继续执行。条件具备关联性，应该使用if elif结构。

完整代码如下：

```
score = float(input("请输入分数"))

if score >= 90:
    print("优秀")
elif score >= 75:
    print("良好")
elif score >= 60:
    print("及格")
elif score >= 0:
    print("不及格")
```

教师演示运行效果：输入60分输出“及格”，输入58分输出“不及格”，输入75分输出“良好”。

### 八、debug演示：if elif的执行效率

第926秒开始，教师使用debug模式演示if elif的执行效率。

**场景一：输入98分**

- 条件score >= 90成立
- 执行对应的print语句
- 后续所有elif和else都不再判断

**场景二：输入68分**

- 条件score >= 90不成立
- 判断score >= 75，不成立
- 判断score >= 60，成立
- 执行对应的print语句
- 后续else不再判断

教师总结：只有前面条件不成立时才会判断后面的条件，前面条件成立就直接结束整个结构，这样执行效率会更高。

### 九、添加else处理非法输入

第977秒开始，教师演示添加else处理边界情况。当用户输入一个超出正常范围的分数（如120分）时，什么也不会输出。教师指出可以用else来提示用户输入有误。

修改后的代码在最后添加else分支：

```
else:
    print("输入的分数是有误的")
```

教师通过debug演示：输入10分（所有条件都不满足），程序依次判断条件1、条件2、条件3、条件4，全部不成立后执行else下的代码，输出“输入的分数是有误的”。

### 十、边界条件的严谨性

第1047秒开始，教师强调编写条件判断时需要注意边界条件的严谨性。

以判断“良好”等级为例，教师提到有的同学可能写为“score <= 89”，但这样会漏掉89.5分的情况。正确写法应该是“score < 90”配合前面的“score >= 90”。教师总结：写代码要尽量写严谨一些，考虑所有可能的情况。

### 十一、分支嵌套：条件中再嵌套条件

第1090秒开始进入最后一种结构——分支嵌套。教师指出，分支嵌套是指在一个条件的代码块里面再写一个if语句，适合“满足大条件后再判断小条件”的场景。

教师强调，嵌套的灵活性很高：

- 外层可以是单独的if语句、if else结构，也可以是if elif else结构
- 内层可以放在if分支、else分支或elif分支中
- 内层可以是if else、if elif或if elif else

教师明确表示，不需要死记硬背嵌套的具体形式，重点是掌握前面的基础结构，然后根据具体业务逻辑灵活组合。

### 十二、实例演示：会员购物优惠判断

第1156秒开始，教师以“会员购物优惠判断”为例演示分支嵌套的实际应用。

**需求分析：**

- 首先判断用户是否是会员（会员直接享受八折优惠）
- 如果不是会员，再根据购物金额判断是否建议注册会员或提示下次光临

**代码实现步骤：**

第一步：让用户输入是否是会员（Y或N）

```
is_member = input("是否是会员（Y/N）：")
is_member = is_member.lower()
```

教师解释使用.lower()方法将输入转换为小写，这样用户输入大写Y或小写y都可以识别，提升用户体验。

第二步：外层使用if else判断是否是会员

```
if is_member == "y":
    print("会员你好，享受八折优惠")
else:
    # 非会员的处理逻辑
```

第三步：在else分支内嵌套if elif else处理非会员的购物金额

```
else:
    money = float(input("请输入本次购物金额"))
    
    if money >= 200:
        print("建议您注册会员，享受折扣")
    elif money >= 100:
        print("再买100元即可享受折扣")
    else:
        print("下次光临")
```

教师解释：如果是会员就不需要再判断购物金额，所以嵌套的if elif else放在else分支内。只有非会员才会执行这段代码。

### 十三、debug演示：分支嵌套的执行逻辑

第1438秒开始，教师使用debug模式演示分支嵌套的执行逻辑。

**场景一：输入Y（是会员）**

- 条件is_member == "Y"成立
- 直接输出“会员你好，享受八折优惠”
- else分支完全不执行
- 整个嵌套结构结束

**场景二：输入N（非会员）**

- 条件is_member == "Y"不成立
- 进入else分支
- 提示输入购物金额
- 假设输入300元，大于等于200
- 执行第一个print语句
- 整个嵌套结构结束

教师强调：分支嵌套的关键是理清逻辑——嵌套的语句应该放在哪个分支内（if、else还是elif），要根据业务逻辑来决定。

### 十四、分支嵌套的注意事项

第1550秒开始，教师总结分支嵌套的两个小技巧。

**技巧一：缩进要层层递进**

每嵌套一层就要多缩进四个空格。这是Python的规范要求，通过缩进区分代码块的归属。教师建议使用IDE的自动缩进功能，不要手动打空格。

**技巧二：嵌套层数不要超过三层**

如果嵌套层数太多，代码会变得很复杂，不容易看懂，后期维护成本会很高。教师建议如果确实需要处理非常复杂的逻辑，考虑重构代码结构。

### 十五、课程总结

第1576秒开始，教师对本P内容进行总结回顾。

**第一点：if语句的核心**

if语句的核心是根据条件的结果（true或false）来决定执行哪一段代码。缩进是关键，不能出错。Python严格控制缩进，缩进错误会导致逻辑错误或语法错误。

**第二点：运算符是写条件的基础**

条件判断需要掌握两类运算符：

- 比较运算符：==（等于）、!=（不等于）、>、<、>=、<=
- 逻辑运算符：and（所有条件都成立）、or（其中一个条件成立）、not（反转条件）

**第三点：分支结构要选对**

根据实际场景选择合适的分支结构：

- 单条件 → 使用if语句
- 两种情况二选一 → 使用if else结构
- 简单的if else → 可以使用三元表达式简化
- 多个条件多选一 → 使用if elif或if elif else结构
- 条件中再套条件 → 使用分支嵌套

**第四点：关键提醒**

- ==是判断相等，一个=是赋值，绝对不能混用
- 非零非空的数据为true，零或空为false
- 代码缩进要一致，推荐四个空格，尽量使用自动缩进而非手动空格

教师最后建议，课后可以结合实际场景多练习，比如判断年龄是否成年、判断数字的正负、做简单的优惠活动判断等。

## 本P结论

1. **if else结构**：适用于二选一场景，当条件成立时执行if下的代码，条件不成立时执行else下的代码。与两个独立的if语句相比，if else具备关联性——条件成立时会跳过后续判断，执行效率更高。

2. **三元表达式**：if else的简化写法，格式为“结果1 if 条件 else 结果2”。仅适用于if和else下各只有一行代码的简单场景，复杂逻辑仍应使用标准if else结构。

3. **if elif else结构**：适用于多选一场景，按顺序判断每一个条件，第一个满足的条件对应的代码块被执行，后续条件不再判断。else作为默认分支处理所有条件都不满足的情况。

4. **分支嵌套**：在一个条件的代码块内再嵌入if语句，适用于“满足大条件后再判断小条件”的场景。嵌套层级不宜超过三层，缩进要层层递进（每层四个空格）。

5. **效率意识**：条件具备关联性时（一个条件成立另一个必然不成立），应使用关联分支结构而非独立if语句，以避免不必要的条件判断。

6. **代码严谨性**：编写条件判断时需考虑所有边界情况，如分数区间的边界值（避免遗漏89.5分这类情况）。
