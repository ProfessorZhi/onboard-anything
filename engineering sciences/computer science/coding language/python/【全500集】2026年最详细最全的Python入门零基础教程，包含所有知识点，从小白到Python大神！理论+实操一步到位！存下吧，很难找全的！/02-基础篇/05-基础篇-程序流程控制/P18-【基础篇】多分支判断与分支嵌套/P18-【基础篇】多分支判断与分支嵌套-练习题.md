## 结构识别

**题目 1**  
阅读以下代码，指出它属于哪种分支结构类型。

```python
score = 85
if score >= 90:
    print("优秀")
elif score >= 70:
    print("良好")
else:
    print("及格")
```

- A. 单分支 if
- B. 二选一 if else
- C. 多选一 if elif else
- D. 嵌套 if

**题目 1**  
答案：C。多选一 `if elif else`。当条件互斥且有多个分支时，使用 `elif` 串联各条件。

**题目 2**  
输出结果：`A`  
原因：Python 从上往下判断条件，第一个条件 `x > 5` 为 True 后，执行对应代码块并跳出整个 `if` 结构，不再判断后续 `elif`。

**题目 3**  
问题：`age = 18` 使用了赋值运算符 `=`，而非比较运算符 `==`。  
后果：赋值操作返回赋值的值（18），在布尔上下文中非零数值被视为 True，代码会进入分支执行，但逻辑本身是错误的——这行代码的本意是判断是否等于18，而非赋值为18。

**题目 4**  
不等价。  
- 两个独立 `if` 会依次判断每个条件，互不干扰，可能执行多个分支  
- `if elif else` 是互斥选择，遇到 True 分支后跳过后续所有判断  
对于互斥条件，应使用分支结构；需要同时判断多个无关条件时，才使用独立 `if`。

**题目 6**  
```python
member_level = input("请输入会员等级：")
consumption = float(input("请输入消费金额："))

if member_level == "白金":
    print(f"折后价格：{consumption * 0.6}")
elif member_level == "金卡":
    if consumption >= 100:
        print(f"折后价格：{consumption * 0.7}")
    else:
        print(f"折后价格：{consumption}")
elif member_level == "银卡":
    if consumption >= 300:
        print(f"折后价格：{consumption * 0.8}")
    else:
        print(f"折后价格：{consumption}")
else:
    if consumption >= 500:
        print(f"折后价格：{consumption * 0.9}")
    else:
        print(f"折后价格：{consumption}")
```

**题目 7**  
不存在漏洞。  
因为条件使用 `elif` 且从上往下递减判断：`score >= 80` 只有在 `score < 90` 时才会被判断，同理 `score >= 70` 只有在 `score < 80` 时才会被判断。这种写法是正确的，关键在于条件的顺序和递减关系。

**题目 8**  
1. 深层嵌套降低可读性的原因：
   - 缩进层级过深，代码右移明显，屏幕可见范围受限
   - 每层嵌套增加一个"心智模型维度"，理解成本指数增长
   - 维护时容易迷失在层级中，修改容易引入错误

2. 替代优化方案：
   - 使用多个 `if` 提前 return（卫语句），减少嵌套
   - 使用字典或映射表替代多层判断
   - 将内层逻辑抽取为独立函数
   - 使用策略模式或查表法
