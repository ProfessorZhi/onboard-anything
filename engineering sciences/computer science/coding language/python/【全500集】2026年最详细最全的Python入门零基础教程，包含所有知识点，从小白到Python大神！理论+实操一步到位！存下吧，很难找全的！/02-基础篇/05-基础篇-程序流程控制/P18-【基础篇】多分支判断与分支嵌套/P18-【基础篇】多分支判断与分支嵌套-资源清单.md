# P18 资源清单 - 【基础篇】多分支判断与分支嵌套
---

## 一、编程语言

**Python**
- 类型：编程语言
- 在本 P 中的作用：课程教学的核心语言，所有条件判断语法（if、else、elif）、类型转换、字符串方法均基于 Python 实现
- 来源：字幕直接提到
- 建议方向：结合本 P 的分支结构语法，自己编写不同场景的判断程序，熟悉 Python 基本语法规范

---

## 二、开发工具

**IDE（集成开发环境）**
- 类型：开发工具
- 在本 P 中的作用：承载代码编写、运行和调试的完整工作流；本 P 大量使用 debug 模式逐行演示分支执行流程，帮助学习者理解条件判断的实际执行路径
- 来源：字幕直接提到（“debug调试”功能演示）
- 建议方向：选择一个顺手的 Python IDE（如 PyCharm、VS Code），熟练掌握其 debug 功能，学会设置断点、逐行执行、观察变量值的变化

---

## 三、Python 内置函数

**input()**
- 类型：内置函数
- 在本 P 中的作用：接收用户输入，获取分数、会员身份等外部数据，是分支判断的触发入口
- 来源：字幕直接提到
- 建议方向：练习设计交互式程序，用 input() 获取用户数据后结合条件判断输出不同结果

**float()**
- 类型：内置函数
- 在本 P 中的作用：将 input() 返回的字符串转换为浮点数，支持带小数的成绩（如 99.5、0.5），避免整数类型丢失精度
- 来源：字幕直接提到（“因为分数可能存在小数”）
- 建议方向：编写涉及小数计算的判断程序，体会类型转换在条件判断前的必要性

---

## 四、字符串方法

**.lower()**
- 类型：字符串方法
- 在本 P 中的作用：统一会员输入的大小写，将用户输入的 Y/N（大写或小写）全部转为小写，避免条件判断时因大小写不一致导致逻辑错误
- 来源：字幕直接提到
- 建议方向：编写涉及用户文本输入的判断程序，使用 .lower() 或 .strip() 等方法提升输入容错性

---

## 五、调试功能

**debug 模式**
- 类型：IDE 内置调试功能
- 在本 P 中的作用：教师使用 debug 模式逐步演示 if else、if elif else、分支嵌套的执行流程，直观展示“条件成立则跳过后续判断”的执行特性，帮助学习者建立动态的代码执行意识
- 来源：字幕直接提到（多处 debug 演示场景）
- 建议方向：在 IDE 中对分支代码设置断点，用 debug 模式单步执行，观察在不同输入条件下程序如何进入各个分支，验证执行路径是否符合预期

---

## 六、运算符体系（前置支撑）

**比较运算符（==、!=、>、<、>=、<=）**
- 类型：运算符
- 在本 P 中的作用：构成条件判断的核心元素，用于比较分数、区间等数值关系；== 尤其强调与赋值 = 的区别，避免混用
- 来源：字幕直接提到
- 建议方向：设计涉及数值区间的判断程序，练习用比较运算符组合条件，注意边界值的处理

**逻辑运算符（and、or、not）**
- 类型：运算符
- 在本 P 中的作用：组合多个条件，如“分数大于等于 60 且小于 75”需要用 and 连接；或运算用于判断多个可能情况；not 用于反转布尔值
- 来源：基于上下文推断（分支结构常需组合条件）
- 建议方向：编写需要同时满足多个条件的判断程序，如“年满 18 岁且持有身份证才可注册”，练习 and/or/not 的组合使用

---

## 七、Python 语法结构

**if else 结构**
- 类型：语法结构
- 在本 P 中的作用：实现二选一逻辑，条件成立执行 if 分支，不成立执行 else 分支；相比两个独立 if，具备关联性，可提升执行效率
- 来源：字幕直接提到
- 建议方向：编写简单的二选一判断程序，对比 if else 与两个独立 if 的执行效果，体会关联性带来的效率差异

**三元表达式**
- 类型：语法结构（if else 简化形式）
- 在本 P 中的作用：将四行 if else 代码压缩为一行的简洁写法，适用于 if 和 else 下各仅一行代码的简单场景
- 来源：字幕直接提到
- 建议方向：对简单的二选一场景尝试用三元表达式改写，对比标准写法与简化写法，培养代码简洁意识

**if elif else 结构**
- 类型：语法结构
- 在本 P 中的作用：实现多选一逻辑，按顺序判断多个条件，第一个满足的条件对应的代码块被执行，后续条件不再判断；else 作为兜底分支处理所有条件均不满足的情况
- 来源：字幕直接提到
- 建议方向：编写涉及多个等级或区间的判断程序，如成绩评级、BMI 分类、温度区间等，练习 elif 的顺序逻辑和 else 的兜底作用

**分支嵌套**
- 类型：语法结构（结构组合）
- 在本 P 中的作用：在 if、else 或 elif 分支内部再嵌入条件判断，适用于“满足大前提后再细分小条件”的业务场景
- 来源：字幕直接提到
- 建议方向：设计需要先判断一个维度再判断另一个维度的程序，如本 P 的会员购物优惠案例（先判断是否会员，再判断购物金额），练习嵌套层级的设计和缩进规范
