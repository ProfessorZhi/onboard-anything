**Q2：区分 `=` 与 `==`**



## 最小实践

**Q5：年龄段判断器**

编写一段程序，完成以下功能：

1. 用 `input()` 接收用户输入的年龄；
2. 进行必要的类型转换；
3. 用**一个复合条件**（需同时用到 `and` 或 `or` 中的至少一个）判断：
   - 年龄在 18 到 60 之间（含边界）→ 打印 `"劳动年龄段"`
   - 否则 → 打印 `"非劳动年龄段"`

验收标准：输入 `18`、`60`、`17`、`61` 时结果均正确。

## 扩展思考

**Q6：短路求值**

Python 的 `and` 和 `or` 存在"短路"行为：一旦结果已可确定，后续表达式不再执行。

```python
def check():
    print("check 被调用了")
    return True

result = False and check()
```

运行上面的代码，`check()` 会被调用吗？请推断并验证。
再把 `False and` 改为 `True or`，结果又如何？

用一句话总结短路求值对程序性能或安全性有何实际意义（可联系"除零"或"空对象访问"场景思考）。
