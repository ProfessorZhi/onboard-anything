# P20 知识框架 - 【基础篇】while循环嵌套
## 本P位置

Python 基础篇 → 控制流 → 循环结构 → while 循环进阶。位于单层 while 循环之后，是学员从简单循环走向复杂逻辑的过渡节点。前置：单层 while 循环、变量自增、print() 基础用法。

## 核心问题

**如何用 while 循环处理「大条件套小任务」的二维结构？**

典型场景：一件事由两个纬度的循环构成，先按外层维度走一步，再按内层维度走完一圈。如座位的排与列、时钟的时针与分针。

## 知识骨架

嵌套循环的本质是**层级控制**：外层循环控制外维度，内层循环控制内维度。

```
while 外层条件:
    内层循环 (完整执行多轮)
    外层变量自增
```

执行规律：
- 外层进入一次 → 内层完整跑完所有轮 → 内层结束 → 外层才自增 → 外层才可能进入下一次迭代
- 内层循环变量每次在外层新迭代时重新初始化

## 关系与依赖

- **外层循环** = 外维度计数器（控制大任务的总次数）
- **内层循环** = 内维度计数器（控制每个大任务内小任务的次数）
- 内层依赖外层：内层变量的生命周期和重新初始化由外层控制
- 外层不依赖内层：外层可独立存在，但嵌套后两者必须严格控制缩进层级

## 输入 / 输出 / 前置 / 支撑

**前置知识：**

- 单层 while 循环语法
- 变量自增 / 递减逻辑（row += 1）
- print() 默认 end="\n" 行为

**本P产出：**

- 输入：一个嵌套结构的需求（如三排六列座位表）
- 输出：符合格式的循环打印结果
- 能力迁移：可处理任意二维遍历问题（矩阵、网格、时间刻度等）

## 易错点

1. **内层循环未嵌套**：将内外层循环写成平级代码块，导致顺序执行而非嵌套执行。症状：先完整跑完外层，再跑内层，输出顺序错误。
2. **外层变量自增位置错误**：row += 1 放在内层循环内部或外层循环最前面，导致计数从错误值开始（如从 2 而非 1）。
3. **内层变量未重置**：内层循环变量（如 column）若写在外层循环外部，则第二次进入外层迭代时 column 值仍为上一轮终值，内层循环不会重新执行。
4. **输出换行与对齐混淆**：未正确使用 end 参数，导致所有座位挤在同一行或每列都换行。解决方案：内层循环用 end="\t" 对齐，内层结束后用 print() 换行。

| 资源 / 对象 | 角色 | 关键参数 |
| --- | --- | --- |
| print() 函数 | 控制输出格式的核心工具 | end 参数：end="\t" 实现同行对齐，end="\n"（默认）实现换行 |
| Python 运行环境 | 执行嵌套循环的载体 | — |

---

**一句话总结：** while 嵌套循环是「大条件套小任务」的代码表达，执行规律是外层进一次、内层完整跑完一轮、内外层缩进严格分层、自增语句放在正确层级结尾处。
