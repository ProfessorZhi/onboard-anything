# P20 讲解过程 - 【基础篇】while循环嵌套
## 本P定位

本P是Python基础篇中 **while循环嵌套** 的核心讲解。在学员已掌握单层while循环的基础上，引入多层循环嵌套的概念、适用场景、语法结构以及完整的实战演示。老师通过"电影院找座位"的具象化例子和时钟类比，帮助学员理解"大条件套小任务"的嵌套逻辑，并最终实现一个三排六列的座位表输出。

---

## 讲解过程

### 一、引出嵌套概念与适用场景

老师首先开门见山，告知学员接下来学习 **while循环嵌套**。直接给出定义：在一个while循环的代码块里面再写另一个while循环。接着点明它的适用场景——**满足大条件后再重复小任务**。

为了让学生快速建立直观印象，老师举了一个生活中常见的例子：**电影院找座位**。电影院座位分布是按"排"和"列"组织的，找座位时必须先确定是哪一排（大的条件），再在那一排里面找到具体的列（小的重复任务）。老师指出这正是嵌套逻辑的典型应用。

### 二、结合流程图讲解嵌套循环的执行流程

老师展示了一个配套的流程图，开始逐步拆解嵌套循环的执行机制：

- 当外层（父）循环的条件为 **true** 时，执行父循环的代码块。
- 进入父循环后，如果子循环的条件也成立，就执行子循环的循环体。
- 子循环执行完毕后，回到子循环的条件判断处继续检查。如果条件仍然成立，就继续执行循环体，如此往复。
- 直到子循环的条件不成立，才退出子循环。
- 退出子循环后，回到父循环继续判断。如果父循环条件还成立，则再次进入子循环，执行完子循环后再回到父循环。
- 直到父循环的条件也不成立，整个嵌套循环才彻底结束。

老师特别强调了 **嵌套层级之间的控制流转**：子循环结束 → 返回父循环 → 父循环继续 → 可能再次进入子循环，这个来回切换的过程是嵌套循环的核心。

### 三、时钟类比：强化大条件套小任务的理解

老师使用了一个非常生活化的类比——**时钟**。时针转动一下（对应大条件），分针就要转完整整一圈（对应小任务的重复）。当时针转动第二下时，分针又重新转一圈。如此往复，直到时针不再转动（所有大条件都不满足）。

这个类比帮助学员理解：在外层循环的一次执行过程中，内层循环要完整执行多轮；只有当内层循环彻底结束后，外层循环才可能进入下一次迭代。

### 四、实战需求发布：输出三排六列的座位表

老师开始演示实际代码编写。首先明确任务目标：**输出三排六列的座位表**。老师展示了最终效果的示意图，画面上呈现出整齐的三行六列排列。

老师一边在代码编辑器中模拟输出，一边描述每一格的标注方式：第一排第一列、第一排第二列……直到第一排第六列，然后是第二排六列、第三排六列。

### 五、分析需求结构：单循环为何不够用

老师开始思考如何实现这个输出，随即抛出一个关键问题：**用一个循环能否完成？**

答案是**不行**。老师分析了这个座位表的结构特征：

- 整个座位被分成了 **3排**，这本身是一个循环（外层循环，循环次数为3）。
- 而每一排内部，又有 **6列**，这又是一个循环（内层循环，循环次数为6）。
- 换句话说，6列的循环是**嵌套在**3排的循环**之内的**。

老师进一步用找座位的例子来说明这个嵌套关系：假设要找"第二排第三座"，必须先在3排中定位到第2排（外层循环），然后再在这一排的循环中找到第3列（内层循环）。所以"1、2、3"是一个外层循环，"1、2、3、4、5、6"是一个内层循环，且内层循环嵌套在外层循环之中。

### 六、编写代码的第一步：外层循环框架

老师开始编写代码。既然外层循环控制排数，那就让 `row` 从第1行开始，循环条件为 `row <= 3`，每次循环打印"这是第row排"，循环结束后执行 `row += 1`。

运行后可以看到输出了"第1排"、"第2排"、"第3排"，但每一排内部并没有列的输出。老师指出问题所在：目前还没有加入内层循环来控制列。

### 七、添加内层循环：六列的循环

老师开始添加内层循环来输出每一排的列。设置 `column = 1`，条件 `column <= 6`，每次打印"第column列"，然后 `column += 1`。

此时老师将外层循环的 `row += 1` 和内层循环写成两个独立的代码块（缩进相同），而不是嵌套关系。运行后发现输出顺序混乱：先输出了完整的123三排（外层循环先执行完），然后再输出123456六列（内层循环单独执行），这显然不符合需求。

老师分析原因：`row += 1` 被放在了外层循环里，但内层循环没有嵌套进去，导致执行顺序变成了：外层循环全部执行完毕 → 再执行内层循环。

### 八、关键调试：`row += 1` 的位置问题

老师开始排查问题，将 `print()` 语句放在内层循环中，以直观显示当前是第几排第几列。运行后发现输出的竟然是"第2排"而不是"第1排"，原因是代码先执行了 `row += 1`，导致从2开始计数。

老师通过这个错误演示引出了一个关键知识点：**循环变量的自增语句必须放在正确的位置**。老师指出，`row += 1` 应该放在外层循环**代码块的最末尾**，也就是内层循环完全结束**之后**再去让排数加一。

正确的结构应该是：

```
外层循环:
    内层循环 (输出每一排的列)
    row += 1
```

这样才能保证：先完整输出一排的所有列，再让排数加一进入下一排。

### 九、使用制表符对齐输出：end 参数

老师注意到当前的输出每一列都会自动换行，因为 `print()` 函数默认的 `end` 参数值是 `\n`（换行符）。为了实现同一排的列对齐显示，需要把 `end` 参数改为制表符 `\t`。

修改后运行，发现所有座位都挤在同一行了——这是因为还没有在每一排结束时换行。老师解释：内层循环（6列）每执行完一次就换一行，这样就能实现排与排之间的分隔。

### 十、在正确位置插入换行：内层循环结束后换行

老师将一个 `print()` 语句放在内层循环结束后的位置（但仍在父循环内）。由于 `print()` 默认 `end="\n"`，所以执行时会自动换行。运行后发现座位表已经呈现出整齐的三行六列格式。

### 十一、逐行 Debug 演示完整执行流程

为了彻底讲清楚嵌套循环的执行过程，老师进行了完整的 Debug 演示，逐步追踪变量的变化和输出结果：

1. `row = 1`，`1 <= 3` 成立，进入外层循环。
2. `column = 1`，`1 <= 6` 成立，进入内层循环。
3. 输出"第一排第一列"，`column` 自增变为2。
4. `2 <= 6` 成立，输出"第一排第二列"，`column` 自增变为3。
5. 重复执行直到 `column = 7`，`7 <= 6` 不成立，**内层循环结束**。
6. 执行内层循环之后的 `print()`（换行）。
7. 执行 `row += 1`，`row` 从1变为2。
8. 回到外层循环条件判断，`2 <= 3` 成立，再次进入外层循环。
9. **关键点**：`column` 在新的一排开始时被重新赋值为1（因为内层循环需要重新开始）。
10. `column` 从1到6，输出"第二排第一列"到"第二排第六列"。
11. `column` 变为7，内层循环结束，换行，`row += 1`（2变为3）。
12. 第三排同理，`column` 从1到6。
13. 第三排结束后，`row += 1`（3变为4），`4 <= 3` 不成立，**外层循环结束**。
14. 外层循环下面没有其他代码，整个程序结束。

老师特别强调了一个细节：**内层循环的变量每次在外层循环进入新一次迭代时会被重新初始化**，这是因为内层循环本身也是一个完整的循环结构，它的初始化语句在外层循环的代码块内部，所以每次外层循环迭代时都会执行一次。

### 十二、最终效果确认

老师运行最终代码，确认输出效果为整齐的三排六列表格，每一列用制表符对齐，每一排结束后自动换行。老师总结：通过这个例子，我们实现了按照格式输出三排六列的座位表。

## 本P结论

1. **while循环嵌套的定义**：在一个while循环的代码块内部再写一个while循环，形成多层循环结构。
2. **适用场景**：适用于"满足大条件后再重复小任务"的场景，如电影院找座位（先定排、再定列）、时钟（时针转一下对应分针转一圈）等。
3. **嵌套循环的执行规律**：外层循环进入一次，内层循环会完整执行多轮；内层循环完全结束后，才能执行外层循环的下一次迭代；内层循环的变量会在每次外层循环进入新一次迭代时重新初始化。
4. **变量自增语句的位置**：循环变量的自增语句必须放在正确的缩进层级，确保内层循环结束后再让外层循环变量自增，避免计数从错误的值开始。
5. **输出格式控制**：利用 `print()` 函数的 `end` 参数，`end="\t"` 实现同行对齐，`end="\n"`（默认值）实现换行，通过在内层循环结束后放置换行语句来分隔不同的外层循环迭代。
