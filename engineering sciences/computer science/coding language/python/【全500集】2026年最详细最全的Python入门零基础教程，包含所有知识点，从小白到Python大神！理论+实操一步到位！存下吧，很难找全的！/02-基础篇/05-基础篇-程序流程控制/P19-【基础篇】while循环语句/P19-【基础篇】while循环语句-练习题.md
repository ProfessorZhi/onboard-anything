# P19 练习题 - 【基础篇】while循环语句

## 结构识别

**Q1** 阅读下面四段代码，指出每段的问题类型（死循环 / 累加结果错误 / 输出位置错误 / 无问题），并说明原因。

```python
# 片段 A
count = 1
while count <= 5:
    print(count)

# 片段 B
rs = 0
i = 1
while i <= 100:
    rs = rs + i
    print(rs)
    i += 1

# 片段 C
rs = 0
i = 1
while i <= 10:
    rs = rs + i
    i += i

# 片段 D
count = 1
while count <= 3:
    print(count)
    count += 1
print("完成")
```

> 提示：对照易错点逐一检查"更新语句是否存在、print 是否在体外、累加变量是否线性推进"。

---

## 机制理解

**Q2** 用文字描述 while 循环一次完整执行周期的顺序（不需要写代码）。  
要求：明确说出"条件何时被判断""循环体何时执行""何时退出"三个时间点，以及"更新语句"在流程中承担的角色。

**Q3** 以下说法是否正确？如果错误，请改正。

> "只要把变量放在循环体里，循环就一定会终止，不会死循环。"

## 最小实践

**Q4** 用 while 循环完成以下两个任务，每个写独立的程序：

1. 打印 1 到 10 的所有整数，每行一个。
2. 计算 1 + 2 + 3 + … + 50 的和，只在循环结束后输出最终结果。

要求：两段代码都必须包含"初始化 → 条件判断 → 循环体（含更新）"三个组成部分，并在注释中标注各部分位置。

**Q5**（位置敏感练习）将下面代码中的 `print(rs)` 分别放在"循环体内"和"循环体外"各运行一次，记录两次的输出行数与最后一行的值，并解释差异的原因。

```python
rs = 0
i = 1
while i <= 5:
    rs = rs + i
    i += 1
# 将 print(rs) 放在此处或循环体内，分别测试
```

## 最小替代练习（工具：PyCharm Debug）

**Q7** 在 PyCharm 中对下面这段代码开启 debug 模式，在 `while` 行打一个断点，然后连续点击"Step Over"逐行执行：

```python
count = 1
while count <= 4:
    print(count)
    count += 1
print("循环结束")
```

观察并记录：
- 每次回到 `while` 行时，`count` 的当前值是多少？
- 第几次判断条件时，程序跳出循环体？
- `print("循环结束")` 是在哪一步之后执行的？

> 目的：通过 debug 逐行追踪，建立"条件判断 → 执行 → 更新 → 再判断"的完整执行顺序感知。
