# P19 知识框架 - 【基础篇】while循环语句
## 本P位置

Python 控制流程 → 循环结构 → while 循环（第一种循环语法入门）

前置：无（Python 基础语法前三章已覆盖变量、输入输出、条件判断）
后续：for 循环（另一种循环语法）→ break / continue（循环控制）→ 嵌套循环

## 核心问题

**如何让程序重复执行一段代码，直到某个条件不再满足？**

while 循环回答的是"按条件驱动重复"这一基本问题，是所有循环结构的思想源头。

## 知识骨架

1. **循环的动机**：消除重复代码，提升可维护性与效率
2. **while 循环原理**：条件判断 → 为真则执行循环体 → 回到条件判断 → 循环往复直至条件为假
3. **语法格式**：`while 条件: 循环体`（其中更新条件的代码必须属于循环体的一部分）
4. **计数循环模式**：计数器初始化 → 条件判断 → 循环体（包含输出 + 计数器更新）
5. **累加求和模式**：累加变量（初值 0）+ 循环变量 → 每次循环 `rs = rs + i` → 循环变量自增
6. **代码位置语义**：循环体内代码重复执行；循环同级且在体外的代码在循环结束后执行一次
7. **死循环**：条件永为 True → 无出口 → 无限消耗资源（需手动终止）

## 输入 / 输出 / 前置 / 支撑

| 类型 | 内容 |
|---|---|
| **输入** | 布尔条件表达式（控制循环何时继续） |
| **输出** | 重复执行的循环体；累加求和的最终结果 |
| **前置** | 变量赋值、条件表达式（if 语句基础） |
| **支撑** | 调试工具（PyCharm debug）用于逐行追踪执行流程，理解条件变化 |

## 易错点

- **忘记更新条件**：死循环最常见成因，count 始终不变化
- **累加变量初始值设错**：初始值非 0 则结果不正确（应等于加法单位元）
- **累加语句写错位置**：误用 `i += i`（导致指数增长）而非 `i += 1`（线性推进）
- **print 位置错误**：把 `print(rs)` 放在循环体内会每轮输出一次，而非只看最终结果
- **混淆循环体内外代码层级**：循环结束后同级的 print 只执行一次
