# P22 讲解过程 - 【基础篇】循环控制：break和continue
## 本P定位

本P是循环控制的关键字部分，承接前面循环结构（while、for）的基础知识，讲解如何通过`break`和`continue`来控制循环流程，并补充讲解循环与`else`搭配使用的特性，最后对比`while`和`for`的使用场景。

## 讲解过程

### 一、引入：循环控制的两个核心关键字

从上一段内容过渡，介绍本节要讲的内容是循环控制的两个核心关键字：`break`和`continue`。

### 二、强调使用位置：必须在循环体里

- 明确指出`break`和`continue`必须在循环体里面使用，不能在单独的`if`语句或函数里面使用。
- 强调它们经常和`if`语句搭配使用，能够灵活地控制循环流程，但只能在循环里面用。

### 三、演示错误：单独使用的报错

1. 先演示在单独的`if`语句中`print("123")`，说明这是没有问题的，会正常运行。

2. 然后演示在单独的`if`语句中使用`break`，运行后报错：
   - 错误类型：SyntaxError（语法错误）
   - 错误信息：`break outside loop`（break在循环之外）
   - 用translate翻译确认错误含义。

3. 再演示在单独的`if`语句中使用`continue`，运行后也报错：
   - 错误类型：SyntaxError
   - 错误信息：continue没有应用于循环（它不在循环里面）
   - 用translate翻译确认错误含义。

### 四、讲解break的作用

1. 给出定义：break的作用是**满足条件的时候直接终止当前循环**，不管后续条件是否成立，直接执行循环外面的代码。

2. 生活化比喻：把循环比作吃苹果。原计划吃五个苹果，吃完第三个的时候吃饱了，那还会吃第四个、第五个吗？不会对不对。就暂停吃苹果的动作，在第三次循环的时候就结束。**提前结束循环**就可以使用`break`语句。

3. 代码验证（吃苹果场景）：
   ```python
   for i in range(1, 6):
       print(f"吃第{i}个苹果")
   ```
   先演示正常运行：吃第一个、第二个、第三个、第四个、第五个。

   然后加入条件判断和break：
   ```python
   for i in range(1, 6):
       if i == 3:
           print("吃饱了")
           break
       print(f"吃第{i}个苹果")
   ```
   运行结果：只吃到第二个，第三个输出"吃饱了"后循环终止，第四个、第五个没有吃。

4. 强调：如果只写`print("吃饱了")`而没有`break`，程序会继续吃第四个、第五个。加上`break`才是直接退出循环。

### 五、debug调试演示break的执行流程

用debug调试功能逐步演示：
1. 取出第一个数字1，吃第一个苹果，if条件不成立，忽略if语句，循环体执行完，回到条件判断，1+1=2。
2. 吃第二个苹果，条件不成立，继续，2+1=3。
3. 吃第三个苹果，3==3条件成立，执行两行代码：输出"吃饱了"、break退出循环。
4. 循环结束，下面没有语句，整个程序执行结束。

结论：**直接结束当前所在的while或for循环**。

### 六、讲解continue的作用

1. 给出定义：continue的作用是**满足条件的时候跳过当前次的剩余代码，直接开始下一次循环**。

2. 生活化比喻：原计划吃五个苹果，吃完第三个发现第三个有虫子（或者有蟑螂），第三个还会吃吗？肯定就不吃了，跳过这个，继续吃第四个、第五个，继续去吃下一个。

3. 代码验证（吃苹果场景）：
   ```python
   for i in range(1, 6):
       print(f"吃第{i}个苹果")
   ```
   先演示正常运行：吃五个苹果。

   然后加入条件判断和continue：
   ```python
   for i in range(1, 6):
       if i == 3:
           print("第3个苹果被老鼠啃了")
           continue
       print(f"吃第{i}个苹果")
   ```
   运行结果：吃第一个、吃第二个、第三个被跳过（没有执行print语句）、继续吃第四个、吃第五个。

4. 强调：如果第三个被虫子啃了，你不执行print吃第三个的语句，但会继续吃后面的。

### 七、debug调试演示continue的执行流程

用debug调试功能逐步演示：
1. 取出数字1，if条件不成立，忽略if语句，执行输出"吃第一个苹果"。1+1=2。
2. 数字2，if条件不成立，忽略if语句，执行输出"吃第二个苹果"。2+1=3。
3. 数字3，3==3成立，执行if里面的代码：输出"第3个苹果被老鼠啃了"，然后continue直接回到条件判断处。
4. 剩下的`print(f"吃第{i}个苹果")`这一行没有执行，因为continue跳过了。
5. 循环条件判断处，3+1=4，4不等于3，跳过if语句，输出"吃第四个苹果"。4+1=5。
6. 输出"吃第五个苹果"。循环结束，下面没有语句，程序结束。

结论：**跳过当前次的print，直接开始下一次循环**。continue结束本次循环，但下一次循环继续执行（如果条件成立的话）。

### 八、补充：循环与else的搭配使用

1. 引入：一个小知识点，循环也可以和else搭配使用。

2. 讲解else块的特点：**else块的代码只有在循环正常结束的时候才会执行**。

3. 解释什么是"循环正常结束"：**中途没有执行过break语句**就叫正常结束。如果中途执行过break，就叫异常结束（提前退出），else块就不会执行。

4. 代码演示（正常结束）：
   ```python
   for i in range(1, 6):
       print(f"循环第{i}次")
   else:
       print("循环正常结束哦")
   ```
   运行结果：循环结束后会输出"循环正常结束哦"，因为中途没有执行过break。

5. 代码演示（异常结束）：
   ```python
   for i in range(1, 6):
       if i == 3:
           break
       print(f"循环第{i}次")
   else:
       print("循环正常结束哦")
   ```
   运行结果：输出"循环正常结束哦"不会执行，因为中途执行过break，循环被提前退出，else块不会执行。

6. 说明：这个用法在判断循环是否完整执行时很有用，可以判断到底有没有执行完成所有次数，有没有提前退出循环。比如找学员，遍历完所有学员都没有找到，就说明循环正常结束（没有执行break），此时可以输出"学员不存在"。

7. 扩展：`for`和`else`可以结合，`while`和`else`也是一样的用法。总结：中途执行过break就不执行else里面缩进的代码，中途没有执行过break语句，else里面的代码就会正常执行。

### 九、对比while循环和for循环的选择

1. 从核心逻辑来看：
   - **while循环**：条件成立就循环
   - **for循环**：遍历元素或者说固定次数就循环

2. 从适用场景来看：
   - **while循环适合不确定循环次数的情况**。比如当用户输入退出就结束循环，如果输入的是其他就一直循环下去。这种情况不知道要循环多少次，但是知道结束的条件，就可以使用while循环。
   - **for循环适合确定循环次数的情况**。比如要循环五次、要循环八次，这种循环次数已知的情况，或者说遍历列表、字符串等可迭代对象，依次取出里面的一个个元素的情况。
   - 补充：遍历就是依次拿出里面的一个个元素。

3. 从代码复杂度来看：
   - while循环需要手动去更新条件，不然的话容易死循环。
   - for循环不需要手动控制，代码会更加简洁。

4. 选择技巧总结：
   - **优先使用for循环**：如果知道循环要执行多少次，或者要遍历可迭代对象，就优先使用for循环，简单而且高效。
   - **使用while循环**：如果不知道循环到底执行多少次，可能是1次、2次、3次、4次甚至5次、10次，只知道停止条件，就用while循环。
   - 举例：让用户一直输入，直到输入正确为止，当输入的是退出就结束循环；或者爬虫请求，直到请求成功才结束。这些知道停止条件的情况，可以用while循环结合break语句（死循环结合break），更好写一些。

### 十、总结本节核心知识点

1. 循环的核心是重复执行代码块，直到停止条件满足。while看条件，for看次数或者说里面是否有元素。

2. while循环的格式：`while 条件` + 循环体 + 更新条件。更新条件属于循环体，单独拎出来是为了避免死循环。

3. for循环的格式：`for 变量 in 可迭代对象`。可以搭配range生成的数值序列去进行遍历，会更加方便。

4. 控制的关键字：
   - **break**：立即退出当前循环。循环中的else遇到break就不会执行。
   - **continue**：跳过当前本次循环，继续下一次循环。只结束这一次，下一次继续。

5. 嵌套循环要注意缩进，层层缩进，每嵌套一层要多四个空格。尽量不要手动打空格，让它自动缩进。

6. 实用技巧：
   - 批量处理数据用for循环。
   - 等待事件触发用while循环。
   - 不知道循环次数就用while循环结合break退出机制。

7. 循环是Python当中处理重复任务的核心工具，课后可以多练几个例子：比如用for循环遍历列表、统计数字个数，用while循环实现用户登录重试等功能。

## 提到的外部资源与对象

1. **translate（翻译工具）**：用于翻译错误信息，帮助学生理解报错含义。在演示break和continue单独使用的报错时，用translate翻译了`break outside loop`和`continue没有应用于循环`这两条错误信息。

2. **debug调试功能**：用于逐步演示代码的执行流程。在讲解break和continue时，都用debug调试功能逐步演示了循环的执行过程，帮助学生理解break是直接退出循环、continue是跳过本次循环继续下一次的区别。

## 本P结论

1. `break`和`continue`必须在循环体里面使用，不能单独在`if`语句或函数里使用。它们经常和`if`语句搭配使用来灵活控制循环流程。

2. `break`的作用：满足条件时**直接终止当前循环**，不管后续条件是否成立，直接执行循环外面的代码。相当于"吃苹果吃饱了就停止"。

3. `continue`的作用：满足条件时**跳过当前次的剩余代码**，**直接开始下一次循环**。相当于"吃到坏苹果就跳过，继续吃下一个"。

4. 循环可以和`else`搭配使用：`else`块的代码只在循环**正常结束**（中途没有执行过break）时执行；如果中途执行过break提前退出，else块就不会执行。这个特性常用于判断循环是否完整执行完毕。

5. 循环选择建议：
   - 知道循环次数或要遍历可迭代对象 → 优先使用`for`循环（简洁高效）
   - 不知道循环次数只知道停止条件 → 使用`while`循环结合break（如用户持续输入直到正确、爬虫请求直到成功）
