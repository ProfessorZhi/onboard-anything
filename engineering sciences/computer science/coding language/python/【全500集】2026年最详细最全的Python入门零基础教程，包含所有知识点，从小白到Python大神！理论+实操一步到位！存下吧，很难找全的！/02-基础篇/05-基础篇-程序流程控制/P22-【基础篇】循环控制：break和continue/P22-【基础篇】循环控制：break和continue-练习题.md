# P22 练习题 - 【基础篇】循环控制：break和continue
## 一、结构识别

**题目 1.1** 阅读以下代码，写出程序运行结束后的输出结果：

```python
for i in range(1, 6):
    if i == 3:
        break
    print(i)
print("循环结束")
```

**题目 1.2** 阅读以下代码，写出程序运行结束后的输出结果：

```python
for i in range(1, 6):
    if i == 3:
        continue
    print(i)
print("循环结束")
```

**题目 1.3** 阅读以下代码，写出程序运行结束后的输出结果：

```python
numbers = [2, 4, 6, 7, 9]
for num in numbers:
    if num % 2 != 0:
        print(f"找到奇数：{num}")
        break
else:
    print("未找到奇数")
print("程序结束")
```

break 语句可以终止 **任何** 层次的循环（即嵌套循环中 break 会一次性退出所有循环层）。

在 `for...else` 或 `while...else` 结构中，else 子句的代码会在______时执行，在______时不执行。

**题目 2.3** 以下代码为什么会报错？请说明原因。

## 三、最小实践

**题目 3.1** 编写程序：使用 `while` 循环从键盘持续接收用户输入的整数，当用户输入 0 时使用 `break` 提前退出循环，并输出用户共输入了多少个非零数字。

## 四、扩展思考

**题目 4.1** 假设有以下嵌套循环结构：

```python
for i in range(3):
    for j in range(3):
        if j == 1:
            break
        print(i, j)
```

请分析：当 `j == 1` 时执行 `break`，程序会退出内层循环还是外层循环？输出结果是什么？

**题目 4.2** `break` 和 `continue` 虽然能控制循环，但过度使用会导致代码逻辑不清晰。请思考：如果不使用 `break`，如何用其他方式实现相同的 "查找即退出" 逻辑？（提示：考虑使用标志变量）

**题目 4.3** 有人说 "所有使用 `continue` 的场景都可以用 `if...else` 替代"，请举例说明这一说法是否正确，并讨论两种方式的优劣。

## 参考答案（简要）

**1.1** 输出：1 2 循环结束（换行分隔）

**1.2** 输出：1 2 4 5 循环结束（换行分隔）

**1.3** 输出：找到奇数：7 程序结束

**2.1** 错误。break 只控制最近一层循环，嵌套循环中需配合标志变量或异常机制才能退出所有层。

**2.2** 循环正常结束 / 执行过 break 提前退出

**2.3** 报错 `SyntaxError: break outside loop`。break 和 continue 必须写在循环体内部。

**3.1** 参考实现：使用计数器累加非零输入，input() 获取输入，int() 转换，break 在输入为 0 时触发。

**3.2** 参考实现：遍历列表，if num < 50: continue，else 分支打印符合条件的数字。

**3.3** 参考实现：for...else 语法，未进入 break 时 else 执行。

**4.1** 只退出内层循环，输出结果为：0 0、1 0、2 0。

**4.2** 可使用 found = False 标志变量，循环内判断找到时设 found = True，循环结束后判断标志状态。

**4.3** 不完全正确。continue 在多层嵌套且需跳过内层后续代码时更简洁；if...else 在可读性上可能更优，需根据场景选择。
