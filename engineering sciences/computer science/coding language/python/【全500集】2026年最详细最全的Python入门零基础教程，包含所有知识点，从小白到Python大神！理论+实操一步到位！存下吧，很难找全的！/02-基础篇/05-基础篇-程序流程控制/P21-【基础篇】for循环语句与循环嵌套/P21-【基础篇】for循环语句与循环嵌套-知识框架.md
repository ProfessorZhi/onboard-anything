# P21 知识框架 - 【基础篇】for循环语句与循环嵌套
## 本P位置

Python基础语法 → 控制流程 → 循环结构（第2/2部分）
前置：P20 while循环
后续：二维数据处理、图案打印、算法基础

---

## 核心问题

如何用更简洁的语法实现「按次数执行」和「逐个访问」两种循环场景？
循环嵌套如何用更少的代码实现多层控制？

---

## 知识骨架

```
for循环
├── 适用场景：计次循环、遍历循环
├── 语法格式：for 变量 in 可迭代对象
│   └── 每次取出一个元素赋值给变量
├── 次数控制：range函数
│   ├── range(stop)           → 0 ~ stop-1
│   ├── range(start, stop)     → start ~ stop-1
│   └── range(start, stop, step) → 带步长的序列
│       └── 步长可为负数（倒序）
└── 嵌套用法：外层控行，内层控列
    └── 换行：print() 或 end参数
```

---

## 关系与依赖

| 关系类型 | 内容 |
|---|---|
| 依赖前置 | while循环原理、变量赋值、print函数 |
| 对比关系 | for循环 vs while循环（代码更简洁，无需手动更新计数器） |
| 组成关系 | for循环 + range函数 = 精确计次循环 |
| 包含关系 | 可迭代对象 ⊃ 列表、字符串、元组、字典、集合 |
| 扩展关系 | 单层for → for嵌套 → 二维结构输出 |

## 输入 / 输出 / 前置 / 支撑

**前置知识：**
- 变量与数据类型（列表、字符串）
- 条件判断与缩进规则
- while循环基本原理

**核心输入：**
- 可迭代对象（列表/字符串/元组/字典/集合/range）

**核心输出：**
- 遍历的元素值
- 累加结果

**支撑工具：**

- 无需第三方库

---

## 易错点

| 易错点 | 正确认知 |
|---|---|
| range边界 | range遵循「包前不包后」，`range(1, 6)`输出1-5而非1-6 |
| 整数不可遍历 | `for i in 123` 报错TypeError，整数不是可迭代对象 |
| 字符串数字 vs 整数 | `"123"`遍历得到字符「1」「2」「3」，非数字1、2、3 |
| 嵌套换行 | 内层循环结束后需`print()`手动换行 |
| 步长为负 | `range(5, 0, -1)`倒序输出5、4、3、2、1 |
| 缩进层级 | 内层循环体必须比外层多一级缩进 |

## 外部资源与对象

| 对象 | 类型 | 作用 |
|---|---|---|
| **range()** | 内置函数 | 生成连续整数序列，for循环的次数生成器 |
| **可迭代对象** | 概念统称 | 列表、字符串、元组、字典、集合的父类概念 |
| **列表** | 数据类型 | 最常用的可迭代对象之一 |
| **字符串** | 数据类型 | 可逐字符遍历 |
| **字典** | 数据类型 | 遍历默认获取键 |
| **集合/元组** | 数据类型 | 无序/有序可迭代对象 |

---

## 核心结论

1. for循环自动完成迭代器管理，无需手动更新条件变量
2. range函数是for循环实现精确计次的标配工具
3. 循环嵌套本质：外层控制行数，内层控制列数
4. for嵌套比while嵌套代码更简洁（无需手动维护行列计数器）
