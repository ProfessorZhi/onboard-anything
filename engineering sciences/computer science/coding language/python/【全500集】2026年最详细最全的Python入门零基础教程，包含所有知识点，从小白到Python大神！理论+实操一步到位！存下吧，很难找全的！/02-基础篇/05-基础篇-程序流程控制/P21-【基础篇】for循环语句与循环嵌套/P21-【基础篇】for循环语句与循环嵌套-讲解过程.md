# P21 讲解过程 - 【基础篇】for循环语句与循环嵌套
## 本P定位

本P是Python基础语法系列中关于循环结构的第二个重要知识点，属于「for循环」的核心入门内容。承接上一P的while循环，引出一种更简洁、更适合计数和遍历场景的循环语句。同时引入了「循环嵌套」的概念，为后续输出二维结构（如打印矩形、星号图案）做铺垫。本P属于循环结构的深化阶段。

## 讲解过程

### 一、for循环的概念引入

老师先从概念入手，指出for循环的两种典型用途：计次循环和遍历循环。计次循环即按照固定次数执行，遍历循环即逐一访问可迭代对象中的每个元素。

这里重点解释了什么叫「可迭代对象」——凡是能够使用for循环语法不报错的对象，就是可迭代对象。具体列举了列表、字符串、元组、字典、集合，这些数据类型都可以用for循环来遍历。老师特别强调：如果用for循环去遍历某个数据而报出「is not iterable」错误，那它就不是可迭代对象。

### 二、for循环的基本格式

给出标准语法格式：

```
for 变量 in 可迭代对象:
    循环体
```

老师指出，可迭代对象可以是列表、元组、字典、集合、字符串，也可以是「次数序列」。次数序列一般由range函数生成，range是for循环中专门用来生成连续数值序列的函数。

### 三、用流程图解释for循环的执行原理

老师没有直接写代码，而是先画流程图，帮助学生建立直观理解：

- 首先检查可迭代对象中是否有元素
- 如果有，取出当前元素赋值给变量，然后执行循环体
- 执行完循环体后，返回继续检查对象中是否还有剩余元素
- 如果有，继续取出下一个元素执行循环体
- 直到对象为空，循环结束

老师用了一个通俗比喻：就像一个盒子，里面有东西就继续拿，拿完就结束，不需要手动更新条件变量。

### 四、range函数的三种用法

老师将range函数作为for循环的「次数生成器」重点讲解，给出三种常用格式：

**第一种：range(stop)**

只指定结束位置，默认从0开始，到stop-1结束（包前不包后）。例如`range(5)`会生成0、1、2、3、4这五个数字。

**第二种：range(start, stop)**

同时指定起始位置和结束位置，从start开始到stop-1结束。例如`range(1, 6)`会生成1、2、3、4、5这五个数字。

**第三种：range(start, stop, step)**

指定起始位置、结束位置和步长（间隔）。例如`range(1, 13, 3)`会生成1、4、7、10这四个数字，老师手算演示：1+3=4，4+3=7，7+3=10，10+3=13（超出结束位置不包含），验证结果正确。

老师特别提醒步长可以是负数，例如`range(11, 1, -3)`会生成11、8、5、2这四个数字（倒序递减）。

最后强调：range函数严格遵循「包前不包后」的规则。

### 五、第一个实操演示：遍历字符串

老师开始写代码演示。定义字符串`s = "ABCDEFG"`，然后写for循环逐个取出字符：

```python
s = "ABCDEFG"
for i in s:
    print(i)
```

运行后输出A到G七个字母。老师结合流程图，详细解释了执行过程：第一次循环取出A给i并打印，第二次循环取出B，第三次取出C，以此类推，直到第七次取出G，第八次检查发现字符串已空，循环结束。

随后在循环体中加入字符串拼接操作：

```python
for i in s:
    print(f"你好呀{i}")
```

这样每个字符都会拼接上「你好呀」输出。老师通过这个例子说明：遍历可迭代对象的本质就是逐一取出每个元素，每次取出的值都会保存到变量i中供循环体使用。

### 六、错误演示：整数不可遍历

老师故意写了一段会报错的代码来加深印象：

```python
for i in 123:
    print(i)
```

运行后立即报错：`TypeError: int object is not iterable`。老师指出报错信息已经明确告诉我们整数类型不是可迭代对象，所以这种语法不合法。

### 七、如何正确遍历数字

针对「想要取出1到10这十个数字」的需求，老师演示了一个绕路方法：把数字放到引号里变成字符串。但随即指出这样做的问题——取出的不是数字类型，而是字符类型，十位数会变成「一」和「零」两个字符。

正确做法是使用range函数生成数值序列：

```python
for i in range(1, 11):
    print(i)
```

这样就精确输出了1到10这十个数字。老师进一步演示了三种range格式的实际效果对比：

- `range(10)` 输出0到9
- `range(1, 10)` 输出1到9
- `range(1, 11)` 输出1到10
- `range(1, 13, 3)` 输出1、4、7、10

### 八、综合案例：计算1到100的和

老师布置一个实际需求：用for循环计算1到100的和。思路是先用range取出1到100的每个数字，然后累加。

```python
sum_num = 0
for i in range(1, 101):
    sum_num += i
print(sum_num)
```

运行结果为5050。老师将范围缩小到1到6来验证逻辑是否正确：

```python
sum_num = 0
for i in range(1, 7):
    sum_num += i
print(sum_num)
```

结果为15。老师结合代码逐次演示执行过程：初始sum_num为0，第一次取出1，0+1=1；第二次取出2，1+2=3；第三次取出3，3+3=6；第四次取出4，6+4=10；第五次取出5，10+5=15；第六次检查发现下一个数字是6（不包含在range范围内），循环结束，最后打印的sum_num就是15。

老师总结for循环的优点：起始位置、结束位置、步长都在range参数中决定了，不需要像while循环那样手动定义和更新计数器变量，代码更简洁。

### 九、for循环嵌套

老师指出for循环同样支持嵌套，用法和while嵌套类似，但代码更简洁。任务目标是输出一个3行×6列的矩形。

**第一层循环：控制行数**

```python
for i in range(1, 4):
    print(f"第{i}排")
```

运行后输出「第一排」「第二排」「第三排」。

**第二层循环：嵌套进第一层，控制列数**

```python
for i in range(1, 4):
    for j in range(1, 7):
        print(f"{i}-{j}")
```

运行后依次输出「1-1」到「1-6」（第一排六列），然后「2-1」到「2-6」（第二排六列），最后「3-1」到「3-6」（第三排六列）。

**最终版：格式化输出并控制换行**

```python
for i in range(1, 4):
    for j in range(1, 7):
        print(f"{i}-{j}", end="\t")
    print()
```

老师使用debug调试模式逐步演示执行流程：

- 外层i=1时，内层j依次取1到6，每次输出后用`\t`（制表符）分隔不换行
- j=7时超出范围，内层循环结束，执行`print()`换行
- 回到外层，i变为2，内层j重新从1开始，重复上述过程
- i=3时同样执行六次后，j=7超出范围，内层循环结束
- 外层i变为4，超出范围，整个程序结束

老师将for嵌套和上一P的while嵌套对比，指出for循环嵌套的优势：不需要手动定义行数和列数的初始值和更新逻辑，由range自动控制循环次数，代码更高效。

## 本P结论

1. **for循环的定义**：计次循环或遍历循环，按固定次数执行或逐一访问可迭代对象中的每个元素，执行完毕自动停止，不需要手动更新条件。

2. **可迭代对象的范围**：列表、字符串、元组、字典、集合都是可迭代对象；整数、浮点数等数值类型不可迭代，试图遍历会报`TypeError: int object is not iterable`。

3. **range函数的三种形式**：`range(stop)`从0到stop-1；`range(start, stop)`从start到stop-1；`range(start, stop, step)`按指定步长生成序列，三种形式均遵循「包前不包后」规则，步长step可以是负数实现倒序。

4. **for循环的基本语法**：`for 变量 in 可迭代对象:`，每次循环将当前元素赋值给变量供循环体使用，循环次数由可迭代对象的长度或range参数决定。

5. **计算累加和的典型写法**：定义累加变量初始值为0，在循环体内执行`累加变量 += 当前元素`，循环结束后变量中保存的就是总和。

6. **for循环嵌套**：for循环支持多层嵌套，外层循环控制行数，内层循环控制列数，每层由各自的range控制循环次数，嵌套结束后记得用`print()`换行，整个嵌套结构比while循环更简洁，无需手动维护行号和列号变量。
