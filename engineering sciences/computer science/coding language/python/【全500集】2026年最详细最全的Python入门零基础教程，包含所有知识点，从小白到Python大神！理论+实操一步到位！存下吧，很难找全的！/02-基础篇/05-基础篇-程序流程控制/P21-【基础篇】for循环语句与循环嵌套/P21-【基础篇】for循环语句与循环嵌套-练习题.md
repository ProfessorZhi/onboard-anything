# P21 练习题 - 【基础篇】for循环语句与循环嵌套
## 一、结构识别

**题目 1.1**
以下代码的输出是什么？

```python
for i in range(3, 7):
    print(i, end=' ')
```

> **答案**：`3 4 5 6 `（注意 range 包前不包后）

---

**题目 1.2**
以下代码的执行次数是多少？

```python
for x in range(10, 0, -2):
    print(x)
```

> **答案**：5 次（输出 10、8、6、4、2）

---

**题目 1.3**
指出下面代码的错误：

```python
for num in 100:
    print(num)
```

> **答案**：TypeError。整数 100 不是可迭代对象，无法被遍历。

## 二、机制理解

**题目 2.1**
对比 for 循环和 while 循环，说明 for 循环的优势是什么？

> **答案要点**：
> - for 循环自动完成迭代器管理，无需手动更新计数器
> - 代码更简洁，语义更明确（适合"遍历"或"计次"场景）
> - while 循环需要手动维护循环变量，容易出错

---

**题目 2.2**
`range(1, 10, 2)` 生成的是一个什么样的序列？它和 `range(1, 11, 2)` 有什么区别？

> **答案要点**：
> - range(1, 10, 2) → 1, 3, 5, 7, 9（5 个数）
> - range(1, 11, 2) → 1, 3, 5, 7, 9（5 个数，结果相同）
> - 区别在于 stop 参数：10 包前不包后，所以最大是 9；11 包前不包后，最大也是 9

---

**题目 2.3**
在循环嵌套中，"外层控行，内层控列"是什么意思？请解释其原理。

> **答案要点**：
> - 外层循环每执行一次，内层循环完整执行一轮
> - 外层循环次数决定输出行数
> - 内层循环次数决定每行列数
> - 内层循环结束后需要换行（print() 或 end 参数）

**题目 3.2**
使用 for 循环打印以下输出：

```
*
**
***
****
*****
```

```python
# 你的代码
```


**题目 3.3**
遍历字符串 `"Python"`，逐字符输出，并统计其中元音字母（a/e/i/o/u）的个数。

```python
# 你的代码
```

> **参考答案**：
```python
s = "Python"
vowels = "aeiouAEIOU"
count = 0
for ch in s:
    print(ch)
    if ch in vowels:
        count += 1
print(f"元音字母个数: {count}")
```

## 四、扩展思考

**题目 4.1**
已知列表 `data = [3, 1, 4, 1, 5, 9, 2, 6]`，请思考：
（1）如果要用 for 循环找到最大值，最少需要几行代码？
（2）能否在一次遍历中同时找到最大值和最小值？

> **参考答案**：
```python
# (1)
data = [3, 1, 4, 1, 5, 9, 2, 6]
max_val = data[0]
for x in data:
    if x > max_val:
        max_val = x
print(max_val)

# (2) 可以同时找
max_val = data[0]
min_val = data[0]
for x in data:
    if x > max_val:
        max_val = x
    if x < min_val:
        min_val = x
print(max_val, min_val)
```

**题目 4.2**
考虑这样一个问题：如果要打印一个 9×9 的乘法表（九九表），循环嵌套应该如何设计？尝试写出代码框架。

> **参考答案**：
```python
for i in range(1, 10):
    for j in range(1, i + 1):
        print(f"{j}×{i}={i*j}", end='\t')
    print()
```

**题目 4.3**
有人说"所有 while 循环都可以用 for 循环改写"，也有人说"不是所有 for 循环都能用 while 循环改写"。你认同哪种观点？请说明理由。

> **答案要点**：
> - 第一句不完全正确：while 循环可用于"无限循环等待事件"等场景，for 循环不适合
> - 第二句也不正确：for 循环本质上也是一种迭代控制，任何有限次数的迭代都可以用 while 实现（只需手动维护迭代器）
> - 正确理解：两者表达能力等价（理论上），但适用场景不同。for 更适合已知遍历对象，while 更适合条件驱动。
