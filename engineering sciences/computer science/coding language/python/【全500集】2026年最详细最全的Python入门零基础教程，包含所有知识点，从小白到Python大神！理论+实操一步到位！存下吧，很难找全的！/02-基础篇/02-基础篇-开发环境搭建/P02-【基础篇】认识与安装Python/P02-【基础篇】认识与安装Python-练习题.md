# P02 练习题 - 【基础篇】认识与安装Python
## 结构识别

### 题目1
以下关于Python在编程语言体系中的定位，说法正确的是：

A. Python是一种编译型语言，需要先编译才能运行
B. Python是一种解释型语言，逐行翻译并执行
C. Python既能编译又能解释，取决于用户选择
D. Python是一种低级语言，接近机器码

正确答案：B
解析：Python是解释型语言，"逐行翻译、边翻边执行"是其核心特征。

---

### 题目2
编译型语言与解释型语言的核心差异在于：

A. 代码是否开源
B. 是否支持跨平台
C. 翻译时机与执行方式的区别
D. 是否有GUI界面

正确答案：C
解析：编译型是"整段翻译→生成可执行文件→执行"；解释型是"逐行翻译→边翻边执行→每次运行需重新翻译"。跨平台性是差异带来的结果表现，而非核心定义。

---

### 题目3
以下工具与Python解释器的关系，正确的一项是：

A. pip 依赖 Python 解释器，IDLE 独立于 Python 解释器
B. pip 依赖 Python 解释器，IDLE 依赖 Python 解释器
C. pip 独立于 Python 解释器，IDLE 依赖 Python 解释器
D. pip 和 IDLE 都独立于 Python 解释器

正确答案：B
解析：pip（包管理器）和IDLE（编程界面）都是Python解释器生态的组成部分，均依赖解释器运行。根据依赖链：pip → 外部库 → 应用场景，pip是解释器的内置工具。

## 机制理解

### 题目1
教程将Python比喻为"翻译官"。请解释这个比喻的含义，以及Python作为"翻译官"具体完成了什么工作。

答案要点：

- 人类使用自然语言风格的Python代码下达指令
- 计算机只能理解二进制机器语言（0和1）
- Python解释器充当翻译角色，将Python代码逐行转换为机器能理解的指令
- 翻译过程发生在运行时，每次运行都要重新翻译（区别于编译型语言一次性翻译）

### 题目2
在Python安装过程中，必须勾选"Add Python.exe to PATH"选项。请回答：

1. PATH是什么？
2. 如果安装时没有勾选这一项，会出现什么后果？
3. 应该如何补救？

答案要点：

1. PATH是操作系统的环境变量，用于告知系统去哪里查找可执行程序
2. 不勾选：命令行输入python后，系统无法找到python.exe，会提示"不是内部或外部命令"
3. 补救方法：手动将Python安装目录添加到系统PATH环境变量；或重新运行安装程序，选择Modify并勾选相应选项

### 题目3
Python的"配置简单"优势体现在哪？pip install xxx 这个命令完成了什么工作？

答案要点：

- 配置简单指的是Python拥有内置的包管理器pip，安装第三方库只需一条命令
- pip install xxx 从Python官方包仓库（PyPI）下载指定库并自动安装到本地
- 这是Python生态丰富的基础——任何人都可以发布自己的库供他人使用

## 最小替代练习

### 题目1
除了从 python.org 下载安装包，还有其他方式可以获取Python运行环境。请列举两种替代方案，并简要对比其优缺点。

参考方向：

- **Anaconda**：面向数据科学，预装大量科学计算库（如NumPy、pandas），适合机器学习/数据分析入门。优点是一站式配置，缺点是体积庞大（占用数GB空间）
- **Miniconda**：Anaconda的精简版，只包含conda和Python，其他库按需安装
- **在线编程环境**（如 Replit、Jupyter Notebook在线版）：无需本地安装，适合临时使用或设备受限场景。缺点是脱离本地开发环境，不适合学习完整的项目开发流程
