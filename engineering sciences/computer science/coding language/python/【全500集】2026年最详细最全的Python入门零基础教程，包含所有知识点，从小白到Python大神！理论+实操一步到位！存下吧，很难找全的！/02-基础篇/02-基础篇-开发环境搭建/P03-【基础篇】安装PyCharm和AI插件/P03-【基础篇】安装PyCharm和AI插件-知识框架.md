# P03 知识框架 - 【基础篇】安装PyCharm和AI插件
## 本P位置

Python入门课程 → 基础篇 → 开发环境搭建 → 承上启下环节

- 承接：Python解释器安装（P2）
- 开启：Python代码编写与运行

---

## 核心问题

如何从零搭建可写、可跑、可智能辅助的Python开发环境？

---

## 知识骨架

```
Python开发环境
├── 代码编辑器（PyCharm）
│   ├── 安装与配置
│   ├── 项目管理（项目 = 专属文件夹）
│   ├── 解释器关联（让编辑器知道动力源在哪）
│   └── 个性化设置（字体、主题）
└── 效率插件
    ├── Translation（翻译）
    └── 通义灵码（AI辅助）
```

---

## 关系与依赖

```
[Python解释器] ──执行──▶ [代码]
      ▲                     ▲
      │                     │
      │关联                 │运行
      │                     │
[PyCharm编辑器] ──编辑──▶ [项目] ──▶ [.py文件]
      ▲
      │
   插件增强
      ▲
   通义灵码 / Translation
```

关键依赖链：**解释器 → PyCharm → 项目 → 代码文件**

---

## 输入 / 输出 / 前置 / 支撑

| 类型 | 内容 |
|---|---|
| 前置 | Python解释器已安装（P2） |
| 输入 | PyCharm安装包、插件zip包（可选） |
| 核心输入 | 目标路径（建议纯英文、无空格） |
| 输出 | 可运行的Python项目 |
| 支撑工具 | PyCharm marketplace、通义灵码官网 |

---

## 易错点

1. **顺序颠倒**：先装PyCharm后装解释器 → 代码无法运行
2. **路径含中文/空格**：项目功能异常、代码报错
3. **解释器路径选错**：需定位到 `Python.exe`，不是文件夹
4. **插件装完不重启**：功能不生效
5. **版本不匹配**：PyCharm与Python版本差异过大可能兼容问题

---

## 外部资源与对象

| 对象 | 角色 | 关系 |
|---|---|---|
| PyCharm | 代码编辑器 | 核心工具，依赖解释器 |
| Python解释器 | 执行引擎 | 必要依赖，独立于PyCharm |
| PyCharm marketplace | 插件分发渠道 | 通过settings → Plugins → marketplace访问 |
| Translation插件 | 翻译辅助 | 安装后需重启，依赖PyCharm |
| 通义灵码 | AI编码助手 | 阿里云出品，支持代码生成/解释/问答，需登录 |
| 通义灵码官网 | 插件备用下载 | 当marketplace下载慢时使用zip包本地导入 |
| pygame | 第三方库 | 课程演示用，与本P无依赖关系，仅作说明区分 |
