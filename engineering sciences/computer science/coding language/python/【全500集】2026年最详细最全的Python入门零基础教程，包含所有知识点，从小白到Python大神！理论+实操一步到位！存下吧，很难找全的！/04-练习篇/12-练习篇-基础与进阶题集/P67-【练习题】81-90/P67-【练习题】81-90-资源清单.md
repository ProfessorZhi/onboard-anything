## 一、标准库与内置函数

### os 模块

- **名称**：os
- **类型**：标准库模块
- **在本P中的作用**：提供文件与目录操作的全部底层能力，包括路径遍历（`os.walk`、`os.listdir`）、属性获取（`os.path.getsize`、`os.path.isfile`）等，是第81-84题所有文件处理任务的基础依赖
- **来源**：字幕直接提到
- **建议方向**：系统梳理 os.path 模块的其他路径操作方法（如 join、abspath、exists），理解 os 模块与 pathlib 模块的优劣对比

---

### os.walk()

- **名称**：os.walk()
- **类型**：标准库方法
- **在本P中的作用**：递归遍历指定目录树，返回 (root, dirs, files) 三元组生成器，用于第81题实现目录的深度遍历
- **来源**：字幕直接提到
- **建议方向**：结合 pathlib 的 rglob() 方法实现等效功能，对比两种方式的代码简洁性与可读性

---

### os.listdir()

- **名称**：os.listdir()
- **类型**：标准库方法
- **在本P中的作用**：获取指定目录下的所有文件和文件夹名称列表（仅返回名称，不含路径），用于第84题列出目标目录内容
- **来源**：字幕直接提到
- **建议方向**：注意与 os.walk() 的区别使用场景；结合 os.path.join() 拼接完整路径

---

### os.path.getsize()

- **名称**：os.path.getsize()
- **类型**：标准库方法
- **在本P中的作用**：根据文件完整路径获取文件大小（字节），用于第81题获取每个 .txt 文件的大小以进行比较
- **来源**：字幕直接提到
- **建议方向**：结合 shutil 模块的磁盘使用统计功能，了解不同场景下的文件大小获取方式

---

### os.path.isfile()

- **名称**：os.path.isfile()
- **类型**：标准库方法
- **在本P中的作用**：判断给定路径是否为普通文件（而非目录或链接），用于第84题过滤出实际文件，排除子目录
- **来源**：字幕直接提到
- **建议方向**：结合 os.path.isdir()、os.path.islink() 全面理解路径类型判断

---

### sorted()

- **名称**：sorted()
- **类型**：内置函数
- **在本P中的作用**：对列表进行排序，支持 key 参数指定排序键、reverse 参数指定倒序，用于第81题按文件大小排序找出最大文件
- **来源**：字幕直接提到
- **建议方向**：对比列表的 .sort() 方法，理解返回值差异；练习多条件排序

---

### max() / min()

- **名称**：max() / min()
- **类型**：内置函数
- **在本P中的作用**：求取可迭代对象的最大/最小值，用于第82题计算各课程的考试成绩最值
- **来源**：字幕直接提到
- **建议方向**：结合 key 参数实现自定义规则的最值查找；理解空列表调用时的处理

---

### sum() / len()

- **名称**：sum() / len()
- **类型**：内置函数
- **在本P中的作用**：sum() 求可迭代对象总和，len() 统计元素个数，两者配合用于第82题计算课程平均分
- **来源**：字幕直接提到
- **建议方向**：使用统计学库（如 statistics 模块的 mean()）实现更精确的平均值计算

---

### dict.get()

- **名称**：dict.get()
- **类型**：字典方法
- **在本P中的作用**：以指定键查找字典值，不存在时返回默认值而非抛出异常，用于第83题安全获取课程对应的教师姓名
- **来源**：字幕直接提到
- **建议方向**：对比直接索引 [] 的差异；设置合理的默认值处理缺失数据

---

### str.endswith()

- **名称**：str.endswith()
- **类型**：字符串方法
- **在本P中的作用**：判断字符串是否以指定后缀结尾，用于第81、84题过滤 .txt 扩展名文件
- **来源**：字幕直接提到
- **建议方向**：结合 startswith() 实现更复杂的文件名模式匹配

---

### lambda 表达式

- **名称**：lambda
- **类型**：Python 语法特性
- **在本P中的作用**：定义匿名函数，作为 sorted() 的 key 参数指定排序依据，用于第81题 `key=lambda x: x[1]` 按元组第二个元素排序
- **来源**：字幕直接提到
- **建议方向**：在实际场景中区分何时用 lambda、何时定义普通函数

---

### f-string（格式化字符串）

- **名称**：f-string
- **类型**：Python 语法特性
- **在本P中的作用**：在字符串前加 f 前缀，通过 {} 插入变量值，用于构造完整文件路径如 `f"{root}\{file}"`
- **来源**：字幕直接提到
- **建议方向**：对比 % 格式化、str.format() 方法的差异与适用场景

---

### with open() 上下文管理器

- **名称**：with open()
- **类型**：文件IO语法
- **在本P中的作用**：自动管理文件资源的打开与关闭，用于第82、83题读取 CSV 格式文本数据
- **来源**：字幕直接提到
- **建议方向**：理解上下文管理器的原理；学习不同模式（r/w/a）的使用差异

### float() 类型转换

- **名称**：float()
- **类型**：内置函数
- **在本P中的作用**：将字符串转换为浮点数，用于第82题将文件读取的成绩字符串转换为数值以便统计计算

## 六、后续学习路径建议

本P重点围绕 **os 模块文件操作** 和 **数据结构组合应用** 展开，建议后续按以下方向深化：

1. **文件系统层面**：学习 pathlib 模块替代 os.path 的现代化路径操作；了解 shutil 模块的文件移动、复制、压缩功能
2. **数据处理层面**：引入 csv 模块或 pandas 库处理结构化数据；学习 JSON 文件的读写操作
3. **代码质量层面**：为文件操作添加异常处理；将重复逻辑封装为函数；编写单元测试验证正确性
4. **性能优化层面**：了解大文件读取的内存优化（分块读取）；探索生成器在文件遍历中的内存效率优势
