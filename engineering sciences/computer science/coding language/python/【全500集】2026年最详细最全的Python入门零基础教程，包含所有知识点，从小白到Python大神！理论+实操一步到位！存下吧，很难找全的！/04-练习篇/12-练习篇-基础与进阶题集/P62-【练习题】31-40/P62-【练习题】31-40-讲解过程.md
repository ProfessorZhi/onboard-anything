### 练习题31 — 计算字典中所有值的总和

**需求**：定义函数，接受一个字典参数，返回字典中所有值的总和。

**实现步骤**：

1. 在PyCharm中打开预先准备好的Py文件，需求和示例字典已写入代码中。
2. 定义函数 `sum_of_dict_values(dict1)`，强调参数不使用 `dict` 命名，因为Python内置类 `dict` 会引发命名冲突，建议加数字或下划线。
3. 分析函数三要素：
   - 作用：计算并返回字典中所有值的总和
   - 参数：接受一个字典（用 `dict1` 避免冲突）
   - 返回值：返回总和
4. 使用 `dict1.values()` 方法获取字典中所有值，返回一个 `dict_values` 类型对象。
5. 演示用 `for value in dict1.values()` 循环遍历每个值，并用 `print(value)` 验证。
6. 累加求和：定义变量 `result = 0` 作为初始总和（0加任何数等于其本身），循环内执行 `result += value`，将当前值累加到总和。
7. 循环结束后 `return result`，将结果返回给调用方。
8. 调用函数并接收返回值，用 `print` 输出提示信息"字典中所有值的总和为"，展示最终结果为600（100+200+300）。
9. 补充验证：给字典增加键值对 `d=400`，结果变为1000，验证函数逻辑正确。

**涉及技术点**：`dict.values()`、循环遍历、累加技巧、函数返回值。

### 练习题32 — 计算导数序列和

**需求**：定义函数，根据传入的整数n返回对应的导数序列和。若n为偶数，计算和 1/2 + 1/4 + … + 1/n；若n为奇数，计算和 1/1 + 1/3 + … + 1/n。

**实现步骤**：

1. 打开PyCharm中的准备文件，确认需求。
2. 定义函数 `sum_of_harmonic_series(n)`。
3. 分析函数三要素：
   - 作用：根据传入的整数n，返回对应的导数序列和
   - 参数：传入一个正整数n
   - 返回值：返回导数序列和；若是负数则返回None
4. 首先判断输入是否为正整数——若 `n <= 0`，打印警告信息"n必须是一个正整数"，然后 `return None` 结束函数执行。
5. 用 `if n % 2 == 0` 判断是否为偶数：
   - 偶数情况：用 `for i in range(2, n+1, 2)` 遍历，步长为2，依次对每一项执行 `1/i`，并累加到 `result` 中
   - 奇数情况：用 `for i in range(1, n+1, 2)` 遍历，步长为2，同样累加
6. `result += 1/i` 中无需括号，因为Python先计算除法再执行加法。
7. 两个分支的累加逻辑相同（区别仅在循环起始值和步长），可在条件分支外统一写 `return result`。
8. 调用函数并传入不同参数验证：
   - n=2：结果为 1/2 = 0.5
   - n=4：结果为 1/2 + 1/4 = 0.75
   - n=5：结果为 1/1 + 1/3 + 1/5
   - n=-1：返回None，输出"n必须是一个正整数"警告

**涉及技术点**：循环步长控制 (`range` 的第三个参数)、条件分支、累加求和、`return None` 提前结束函数、代码复用优化。

### 练习题33 — 计算给定日期是一年中的第几天

**需求**：定义函数 `day_of_year(year, month, day)`，接收年份、月份、日期三个参数，计算这一天是该年的第几天。

**实现步骤**：

2. 定义函数 `day_of_year(year, month, day)`，参数为年份、月份、日期三个。
3. 先定义平年每月天数的列表 `[31, 28, 31, 30, 31, 30, 31, 31, 30, 31, 30, 31]`，索引0对应1月，索引11对应12月。
4. 定义辅助函数 `is_leap_year(year)` 用于判断闰年，返回True或False：
   - 闰年条件：(year % 4 == 0 and year % 100 != 0) or (year % 400 == 0)
   - 该函数直接 `return` 逻辑表达式本身即可
5. 在主函数中调用 `is_leap_year(year)` 判断是否闰年。若为闰年，将 `days_in_months[1]`（2月对应的元素）修改为29。
6. 计算累计天数：`sum(days_in_months[:month-1])` 累加当前月份之前的所有月份天数，再加上当前天数 `day`，即得到该日期是当年的第几天。
7. 返回计算结果。
8. 调用函数时，用 `input()` 和 `int()` 接收用户输入的年份、月份、日期，并将其转换为整型后传入函数。

### 练习题34 — 斐波那契数列

**需求**：定义函数 `fibonacci(n)`，计算斐波那契数列中的第n项。前两项均为1，从第三项开始每一项都是其前两项的和。

**实现步骤**：

1. 打开PyCharm准备文件，确认需求。先讲解斐波那契数列的定义：1, 1, 2, 3, 5, 8, 13, 21 …（即每项等于前两项之和）。
2. 定义函数 `fibonacci(n)`。
3. 分析函数三要素并确认参数n应为正整数。
4. **边界条件验证1**：若 `n <= 0`，打印"项数必须是一个正整数"，然后 `return` 结束函数（不写返回值则默认返回None）。
5. **边界条件验证2**：若 `n == 1` 或 `n == 2`（即前两项），直接 `return 1`。
6. **递归计算**：其余情况使用递归 `return fibonacci(n-1) + fibonacci(n-2)`：
   - `fibonacci(n-1)` 代表前一项
   - `fibonacci(n-2)` 代表前前项
   - 递归的出口就是前面的 `n == 1 or n == 2` 分支
7. **递归过程演示**（老师手动展开讲解）：
   - 以 `n=5` 为例：展开为 `fibonacci(4) + fibonacci(3)` → `fibonacci(3)+fibonacci(2) + fibonacci(2)+fibonacci(1)` → 继续分解直到所有项都到达 `n==1` 或 `n==2` 的返回条件 → 最终得到 1+1+1+1+1 = 5
   - 以 `n=7` 为例：完整展开递归树，最终结果为13
8. 调用函数验证：
   - `fibonacci(3)` → 2
   - `fibonacci(5)` → 5
   - `fibonacci(7)` → 13
9. 老师提示递归的局限性：对于较大的n值，递归可能导致性能问题（重复计算过多），小范围内没有问题。

**涉及技术点**：递归函数定义、递归终止条件（出口）、递归调用自己、递归栈的展开与回溯、边界条件处理。

## 本P结论

通过本P十道练习题的练习，学习者掌握并巩固了以下核心技能：

1. **函数三要素思维**（作用、参数、返回值）—— 每道题都严格按此框架分析，是规范编程的习惯养成。
2. **字典遍历与聚合** —— `dict.values()` 结合循环和累加变量是求字典值总和的标准写法。
3. **循环步长控制** —— `range(start, end, step)` 控制奇偶数序列的遍历，是求导数序列和的关键。
4. **辅助函数的拆分** —— 将闰年判断独立为 `is_leap_year()` 函数，体现了代码模块化的思想。
5. **列表的存储与修改** —— 用列表保存每月天数，并根据闰年动态修改二月天数。
6. **递归算法思想** —— 斐波那契数列是递归的经典案例，理解递归的终止条件和调用展开过程是本P的难点。
7. **输入校验与健壮性** —— 对负数和非正整数的提前处理配合 `return None`，提升了代码的鲁棒性。
8. **代码简化技巧** —— 将相同逻辑放在条件分支之外（如 `return result` 统一写在 `if/else` 之后），减少了冗余代码。
