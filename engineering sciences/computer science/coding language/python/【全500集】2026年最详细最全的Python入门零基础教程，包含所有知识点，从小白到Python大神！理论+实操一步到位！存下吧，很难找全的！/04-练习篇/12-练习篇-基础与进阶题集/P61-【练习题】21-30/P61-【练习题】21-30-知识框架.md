# P61 知识框架 - 【练习题】21-30
## 本P位置

Python入门课程·练习题章节·P61。全课前期已完成基础语法铺垫，本P进入**专项练习阶段**，以"需求→实现→验证"的闭环模式强化函数设计能力与循环思维。

---

## 核心问题

1. 循环次数不确定时如何控制退出
2. 如何生成"AA、AAA"等连续重复数字串
3. 如何设计一个规范的函数（作用/参数/返回值三要素）
4. 如何将判断逻辑与业务逻辑拆分为独立函数
5. 累加/累乘模式的初始值陷阱

---

## 知识骨架

```
函数定义三要素
├── 作用：描述函数做什么
├── 参数：定义输入
└── 返回值：定义输出

循环控制模式
├── while True + break：次数不定、结束条件定
└── for + range：次数确定

累加/累乘模式
├── 累加：初始值=0，result += x
└── 累乘：初始值=1，result *= x

数字串生成
└── current_term = current_term * 10 + a

数学计算
├── math模块 → math.pi（高精度圆周率）
└── round(x, n) → 保留n位小数

函数拆分思想
└── is_prime(x) + print_primes(start, end)
```

---

## 关系与依赖

- **循环 → 累加/累乘**：所有遍历场景均依赖循环体执行累加或累乘操作
- **函数三要素 → 所有函数题**：第22至25练统一按三要素框架展开
- **布尔判断 → 函数拆分**：is_prime返回True/False，print_primes消费该判断结果
- **math模块 → round() → 高精度计算**：math.pi提供基准值，round控制输出精度

---

## 易错点

1. **累乘初始值设错**：result = 0 会导致乘积永远为0，必须设为1
2. **return误以为仅返回值**：return 执行后函数立即结束，后续代码不执行
3. **素数判断范围错**：需检查[2, number-1]，容易遗漏或越界
4. **硬编码圆周率**：3.14精度不足，应使用 math.pi
5. **布尔函数冗余判断**：if is_prime(x) == True 可简化为 if is_prime(x)
