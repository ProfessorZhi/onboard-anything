## 讲解过程

### 第 51 练 —— time 模块的常见方法

**引入任务**：今天要学习时间处理，使用 time 模块实现时间相关的功能。打开 PyCharm，预先准备好了一个包含需求的 Python 文件。

**导入模块**：

```
import time
```

说明该模块用于处理时间相关的任务。

**逐一讲解 time 模块的五个常见方法**：

**1. `time.time()`**——获取当前时间的时间戳

解释时间戳的概念：即自**纪元（Epoch）**以来的描述，纪元指 **1970 年 1 月 1 日 0:00:00** 至今的秒数。演示代码：

```
current_time_stamp = time.time()
print(current_time_stamp)
```

运行后得到一个浮点数，表示从 1970 年 1 月 1 日 0:00:00 以来的秒数。

**2. `time.ctime()`**——将时间戳转换为本地时间的字符串表示

默认不传参时使用当前时间戳，返回形如 `WEDNESDAY APRIL 2 11:29:52 2025` 的字符串。也可以传入指定时间戳进行转换：

```
current_time_str = time.ctime()
print(current_time_str)

specified_time_str = time.ctime(1717102524)
print(specified_time_str)
```

传入时间戳 `1717102524` 后得到 `FRIDAY MAY 31 04:55:24 2024`。

**3. `time.localtime()`**——将时间戳转换为本地时间的 struct_time 对象

struct_time 对象包含年、月、日、时、分、秒等字段，是一个结构化的时间表示。可以不传参（默认当前时间），也可以传入指定时间戳：

```
current_local_time = time.localtime()
print(current_local_time)
```

输出结果展示年、月、日、时、分、秒以及 `wday`（星期几）等字段。讲解中提示可从网站获取时间戳，然后演示传入指定时间戳同样可以获取对应的 struct_time 对象，输出结果为 `2024年5月31日 04:55:24`。

**4. `time.gmtime()`**——将时间戳转换为 UTC（世界标准）时间的 struct_time 对象

与 `localtime` 的区别在于返回的是 UTC 时间。默认使用当前时间戳，也可传入指定时间戳：

```
current_utc_time = time.gmtime()
print(current_utc_time)

specified_utc_time = time.gmtime(1717102524)
print(specified_utc_time)
```

运行结果分别为 `2025年4月2日 05:36:24`（UTC 时间，比本地时间早约 8 小时）和 `2024年5月30日 20:55:24`。

**5. `time.asctime()`**——将 struct_time 对象转换为特定格式的字符串

该方法的默认格式为“星期 月份 日 时:分:秒 年”。演示将本地 struct_time 对象和 UTC struct_time 对象分别转换：

```
local_formatted_time_string = time.asctime(time.localtime())
print(local_formatted_time_string)

utc_formatted_time_string = time.asctime(time.gmtime())
print(utc_formatted_time_string)
```

输出分别为 `WEDNESDAY APRIL 2 13:39:23 2025`（本地时间）和 `WEDNESDAY APRIL 2 05:40:27 2025`（UTC 时间）。

**总结**：通过第 51 练，学员掌握了 time 模块的基本用法，学会将时间戳转换为本地时间、UTC 时间和特定格式的时间字符串，强调这些技能在数据处理、日志记录、系统监控等场景中有广泛应用。

### 第 52 练 —— 遍历打印 0~2999 并计算耗时

**需求**：遍历并打印从 0 到 2999 的所有整数，同时计算并输出整个遍历过程所消耗的总时间。

**实现思路**：在循环开始前记录一个时间戳，循环结束后再记录一个时间戳，两者相减即为耗时。

**代码演示**：

导入 time 模块后，使用 `time.time()` 在循环前后分别获取时间戳：

```
import time

start = time.time()

for i in range(3000):
    print(i)

end = time.time()

print(f"遍历从零到2999的所有整数所消耗的总时间为 {end - start} 秒")
```

**关键细节**：讲解了 `range(3000)` 的用法——因为 range 包前不包后，所以 `range(3000)` 的实际范围是 0 到 2999，无需显式传入起始值 0。

**排错过程**：初次运行时报错，原因是少了括号，删除多余括号后重新运行，得到结果 `0.0159` 秒左右。说明在较快的循环中耗时数值较小。

**总结**：学会通过在代码关键部分前后记录时间戳来计算代码执行所消耗的时间。

### 第 53 练 —— 遍历打印 0~9999 并保留三位小数

**需求**：遍历 0~9999 并计算耗时，以保留三位小数的浮点数形式输出。

**新方法引入**：`time.perf_counter()`

教师指出如果要更精确地测量短时间间隔，推荐使用 `time.perf_counter()`，而非 `time.time()`。原因：该方法返回一个**高精度单调递增的时钟值**，表示从程序启动到当前时刻的经过时间，在测量短时间间隔时比系统时钟具有更高的精度和稳定性。

**代码演示**：

```
import time

start = time.perf_counter()

for i in range(10000):
    print(i)

end = time.perf_counter()

print(f"遍历从零到9999的所有整数所消耗的总时间为 {end - start:.3f} 秒")
```

**格式化技巧**：在 f-string 中使用 `{end - start:.3f}` 可以直接保留三位小数输出，避免手动调用 `round()` 或 `format()`。多次运行结果在 `0.054` 秒到 `0.057` 秒之间波动，说明每次执行效率不完全一致。

**总结**：`perf_counter()` 非常适合测量短时间间隔，提供了比系统时钟更高的精度和稳定性。

### 第 54 练 —— 时间字符串转时间戳

**需求**：给定两个格式不同的时间字符串（`2025-04-02 23:40:00` 和 `2025/04/02 23:40:00`），将其转换为以秒为单位的整数时间戳。

**导入模块**：需要使用 time 模块。

**转换步骤**：

**第一步**：使用 `time.strptime()` 将时间字符串转换为 struct_time（时间数组）对象。

`strptime` 接收两个参数：第一个是时间字符串，第二个是格式字符串。

讲解了格式字符串中常用占位符的含义和大小写区分：

- `%Y` —— 四位数年份（如 2025）
- `%m` —— 月份（小写 m）
- `%d` —— 日（小写 d）
- `%H` —— 小时（24 小时制，大写 H）
- `%M` —— 分钟（大写 M）
- `%S` —— 秒（大写 S）

强调大小写必须严格区分，因为 `%m` 是月份，`%M` 是分钟。

**处理第一种格式**（横杠分隔：`2025-04-02 23:40:00`）：

```
time_array = time.strptime("2025-04-02 23:40:00", "%Y-%m-%d %H:%M:%S")
print(time_array)
```

运行后输出 struct_time 对象，年、月、日、时、分、秒字段均正确解析。

**第二步**：使用 `time.mktime()` 将 struct_time 对象转换为时间戳（以秒为单位的浮点数）。

```
time_stamp = int(time.mktime(time_array))
print(time_stamp)
```

用 `int()` 将浮点数转换为整数时间戳。

**处理第二种格式**（斜杠分隔：`2025/04/02 23:40:00`）：

```
time_array2 = time.strptime("2025/04/02 23:40:00", "%Y/%m/%d %H:%M:%S")
time_stamp2 = int(time.mktime(time_array2))
print(time_stamp2)
```

**注意事项**：格式占位符必须与输入字符串的实际分隔符严格对应。如果把 `%m` 写成 `-` 的形式去解析 `/` 分隔的字符串，会导致报错。讲解中故意演示了错误的对应方式导致报错的情况，强调格式匹配的重要性。

**扩展方法**：`time.strftime()` —— 与 `strptime` 相对应，用于将 struct_time 对象转换回时间字符串。

```
time_string = time.strftime("%Y-%m-%d %H:%M:%S", time_array)
print(time_string)
```

`strptime` 和 `strftime` 是互为逆操作的一对方法：

- `strptime`：将时间字符串解析为 struct_time
- `strftime`：将 struct_time 格式化为时间字符串

**总结**：学会了 time 模块中处理时间字符串、时间数组（struct_time）和时间戳之间转换的方法。

### 第 55 练 —— 延时操作

**需求**：程序暂停一秒后获取当前系统时间，并将其转换为年月日时分秒格式的字符串进行打印。

**核心方法**：`time.sleep()`——将程序暂停执行指定的秒数。

```
import time

time.sleep(1)  # 暂停一秒
```

**完整实现**：

```
import time

time.sleep(1)

local_time = time.localtime()
result = time.strftime("%Y-%m-%d %H:%M:%S", local_time)
print(result)
```

讲解了 `strftime` 的用法：第一个参数为格式化字符串，第二个参数为 struct_time 对象。`%Y` 为四位数年份，`%m` 为月份，`%d` 为日，`%H` 为小时，`%M` 为分钟，`%S` 为秒。

**验证延时效果**：讲解中对比了延时前后的秒数（原本 34 秒，延时后输出 35 秒），直观展示了 `sleep` 方法的暂停效果。指出如果某个场景中需要程序暂停运行，就可以使用 `sleep` 方法。

**总结**：学会使用 `time.sleep()` 让程序暂停执行指定的秒数。

### 第 56 练 —— 实现简单秒表

**需求**：用户按下回车键后开始计时，在用户中断程序时停止计时。

**实现思路**：用户按下回车键之前程序处于等待状态，按下后开始计时，每次循环计算当前时间与开始时间的差值，每秒更新一次显示，用户手动中断（Ctrl+C）时停止。

**代码演示**：

```
import time

print("按一下回车键开始计时，中断程序则停止计时")
input("请准备，按下回车键开始计时：")

start_time = time.time()
print("计时已开始")
print("计时中...", flush=True)

while True:
    elapsed = time.time() - start_time
    print(f"\r计时: {elapsed:.0f} 秒", end="", flush=True)
    time.sleep(1)
```

**关键代码解析**：

- **等待用户准备**：`input()` 函数用于等待用户按下回车键，不需要将输入值保存到变量中。
- **记录开始时间**：`time.time()` 获取计时开始的瞬间时间戳。
- **无限循环计时**：`while True` 保证程序持续运行，直到用户手动中断。
- **计算已过时间**：`elapsed = time.time() - start_time`，每次循环计算当前时刻与开始时刻的差值。
- **实时覆盖打印**：`\r` 使光标回到行首，`end=""` 防止换行，实现同一行内数字不断刷新的效果。
- **每秒更新**：每次循环最后 `time.sleep(1)` 让程序暂停一秒，确保每秒只更新一次显示。
- **格式化输出**：`{elapsed:.0f}` 表示不保留小数，直接输出整数秒数。

**讲解补充**：指出也可以用 `int(elapsed)` 将浮点数转换为整数，或者使用 f-string 的格式化功能来控制小数位数。

**总结**：通过第 56 练，将 time 模块的 `time()` 和 `sleep()` 方法结合 input 函数和 while 循环，实现了完整的秒表功能。

## 提到的外部资源与对象

1. **PyCharm**：本 P 全程使用的 Python 集成开发环境（IDE），用于打开和编辑 Python 文件并运行代码。
2. **Epoch（纪元）**：1970 年 1 月 1 日 0:00:00（UTC 时间），是 Unix/Linux 系统中时间计量的起点。
3. **UTC 时间**：世界标准时间（Coordinated Universal Time），与本地时间（中国为 UTC+8）相差约 8 小时。
4. **struct_time 对象**：time 模块中的时间结构对象，包含年、月、日、时、分、秒、星期、一年中的第几天、夏令时标志等九个元素。
5. **perfb_counter 与 time.time 的区别**：前者提供更高精度和稳定性，适合测量短时间间隔；后者受系统时间调整影响。

## 本P结论

1. **`time` 模块的五个核心方法**：`time.time()` 获取时间戳、`time.ctime()` 时间戳转本地字符串、`time.localtime()` 时间戳转本地 struct_time、`time.gmtime()` 时间戳转 UTC struct_time、`time.asctime()` struct_time 转固定格式字符串。
2. **时间戳↔字符串↔struct_time 之间的转换链路**：字符串 → struct_time 用 `strptime`，struct_time → 字符串用 `strftime`，struct_time → 时间戳用 `mktime`，时间戳 → struct_time 用 `localtime` 或 `gmtime`。
3. **耗时测量的两种方法**：`time.time()` 适用于一般场景，`time.perf_counter()` 适用于需要高精度的短时间测量。
4. **程序延时**：`time.sleep(N)` 可以让程序暂停 N 秒。
5. **计时器实现**：结合 `time.time()` 计算时间差、`while True` 无限循环、`time.sleep(1)` 控制刷新频率、`\r` 实现行内覆盖打印，即可实现秒表功能。
6. **格式占位符严格区分大小写**：`%m` 是月份，`%M` 是分钟；`%H` 是 24 小时制小时，`%S` 是秒。
7. **格式化输出**：`f-string` 中使用 `:.3f` 可直接保留三位小数输出，`:.0f` 可保留零位小数（输出整数）。
