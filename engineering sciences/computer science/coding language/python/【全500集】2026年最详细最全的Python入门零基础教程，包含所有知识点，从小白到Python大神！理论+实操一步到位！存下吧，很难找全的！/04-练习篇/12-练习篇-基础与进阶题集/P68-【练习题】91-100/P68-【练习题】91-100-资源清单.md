**re 模块**
- 类型：Python 标准库

- 来源：字幕直接提到
- 后续可练习方向：深入学习 re 模块的高级用法，如 finditer、split、sub 等方法；了解预编译正则以提升性能的场景

## 三、函数与方法

**re.findall()**
- 类型：函数/方法
- 在本 P 中的作用：提取字符串中所有匹配项，返回列表。P91、P92、P94 用于提取手机号和邮箱
- 来源：字幕直接提到
- 后续可练习方向：设计不同正则模式，从日志文件、网页源码、JSON 数据中提取目标信息

**re.search()**
- 类型：函数/方法
- 在本 P 中的作用：查找第一个匹配项，返回 match 对象或 None。P93 密码验证和 P95 分组提取中使用
- 来源：字幕直接提到
- 后续可练习方向：结合条件判断实现复杂验证逻辑；与 findall 对比理解何时用 search 何时用 findall

**re.compile()**
- 类型：函数/方法
- 在本 P 中的作用：将正则表达式编译为 Pattern 对象，配合 re.VERBOSE 提升复杂正则的可读性
- 来源：字幕直接提到
- 后续可练习方向：对高频使用的正则表达式进行预编译，减少重复解析开销

**open()**
- 类型：内置函数
- 在本 P 中的作用：打开外部文件，配合 f.read() 读取 P92 中的手机号数据文件
- 来源：字幕直接提到
- 后续可练习方向：练习 with 语句的上下文管理、多种读取模式（r/w/a）、按行读取等场景

**match.groups()**
- 类型：方法
- 在本 P 中的作用：返回 match 对象中所有捕获组组成的元组，P95 中用于一次性获取数量、名称、金额
- 来源：字幕直接提到
- 后续可练习方向：处理具有多个分组的数据，练习元组解包和下标访问

**match.group(n)**
- 类型：方法
- 在本 P 中的作用：按编号返回指定捕获组的内容，n 从 1 开始。P95 中分别提取斤数、物品名、金额
- 来源：字幕直接提到
- 后续可练习方向：练习命名的捕获组（?P<name>），使用 group('name') 获取字段，代码更清晰

## 四、正则表达式修饰符

**re.VERBOSE**
- 类型：修饰符/参数
- 在本 P 中的作用：启用多行编写和注释功能，P94 中用于组织复杂的邮箱正则表达式
- 来源：字幕直接提到
- 后续可练习方向：编写超长正则时使用多行和注释提高可维护性；注意空格在 VERBOSE 模式下会被忽略

---

## 五、字符类与量词

**\d**
- 类型：正则表达式原子
- 在本 P 中的作用：匹配任意数字字符，P91 手机号、邮箱、密码验证中均使用
- 来源：字幕直接提到
- 后续可练习方向：区分 \d 与 [0-9] 在 Unicode 环境下的行为差异

**[3-9]**
- 类型：正则表达式字符类
- 在本 P 中的作用：匹配 3 到 9 之间的任意一个数字，P91 手机号正则中限定第二位数字范围
- 来源：字幕直接提到
- 后续可练习方向：设计其他中国号码格式的正则（如身份证、邮政编码）并练习字符类的灵活运用

**\W**
- 类型：正则表达式原子
- 在本 P 中的作用：匹配非单词字符（除数字、字母、汉字、下划线），P93 密码验证中用于检测特殊符号
- 来源：字幕直接提到
- 后续可练习方向：注意 \W 包含空格，需根据业务需求决定是否排除空格；学习 \w（大写 W 的反义）

**\.**
- 类型：正则表达式转义序列
- 在本 P 中的作用：匹配字面量句号，P94 邮箱正则中转义域名分隔符
- 来源：字幕直接提到
- 后续可练习方向：区分 .（任意字符）与 \.（字面点号）的使用场景

**+?（非贪婪匹配）**
- 类型：正则表达式量词
- 在本 P 中的作用：.+? 表示非贪婪匹配，P95 购物详情提取中避免吞掉过多内容
- 来源：字幕直接提到
- 后续可练习方向：在有固定分隔符的场景中使用非贪婪；对比贪婪与非贪婪的行为差异
