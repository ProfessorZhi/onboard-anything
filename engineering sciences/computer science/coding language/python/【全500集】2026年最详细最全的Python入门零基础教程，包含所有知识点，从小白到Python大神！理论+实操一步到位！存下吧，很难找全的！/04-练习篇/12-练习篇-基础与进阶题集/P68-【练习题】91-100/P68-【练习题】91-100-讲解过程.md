# P68 讲解过程 - 【练习题】91-100
## 本P定位

本P是Python正则表达式系列练习题的第68集，覆盖练习题91至95。老师在约37分钟的课程中，通过5个递进式的实战练习，系统讲解了正则表达式在信息提取和格式验证两大场景中的应用。前两道题聚焦手机号码提取，后三道题逐步增加复杂度，涉及密码验证、邮箱提取以及带分组的多字段信息提取。

## 讲解过程

### 第91练：文本中提取手机号码（0:00 - 4:05）

课程以一个明确的需求开场：给定一段包含多个中国手机号码的文本，要求提取出所有符合格式的手机号码。老师首先在IDE中打开预先准备的PY文件，向学生展示示例文本的内容。文本中混杂着多个数字串，其中既有符合规范的11位手机号码（如19989881888、15619292345），也有不符合规范的数字（如12345位数不足、1998和981888位数为4位或6位）。

老师强调这类问题必须使用正则表达式来解决。第一步是导入re模块，用于正则表达式的相关操作。第二步是定义正则表达式的匹配模式。老师逐一拆解中国手机号码的格式规则：首位必须是数字1，第二位只能是3到9之间的数字（因为没有12开头的手机号），后面再跟9位数字，总计11位。

据此写出的pattern为r"1[3-9]\d{9}"。其中1表示固定的首位字符，[3-9]表示第二位可以是3到9中的任意一个数字（也可写成3456789，但[3-9]更简洁），\d{9}表示后面跟着9个数字字符。使用原生字符串r前缀避免转义问题。

接下来使用re.findall()方法从文本中查找所有匹配项。findall会返回一个列表，包含所有满足条件的手机号码。调用方式为re.findall(pattern, content)，将content中所有符合pattern格式的内容提取出来，存入results列表。最后遍历results列表，逐条打印提取到的手机号码。

运行结果验证了正则表达式的准确性：19989881888和15619292345被成功提取，而1998、981888、12345等不符合格式的数字则未被提取。老师指出这些不符合的案例中，有的是位数不足，有的是第二位数字不在3到9之间（如2开头）。

### 第92练：文件中提取手机号码（4:05 - 8:24）

第92练在前一题基础上增加了一个环节：数据源从直接写入的文本字符串改为从外部TXT文件读取。需求是从web_page_phone_numbers.txt文件中提取并输出所有符合格式的中国手机号码。老师首先展示了这个TXT文件的内容，其中包含很多手机号码和杂七杂八的文字信息。

操作流程依然是先导入re模块，然后定义同样的正则表达式模式pattern = r"1[3-9]\d{9}"。关键区别在于这次需要先读取文件内容，再对文件内容进行正则匹配。使用with open()语句以只读模式打开文件，指定UTF-8编码，通过f.read()方法将文件全部内容读取到一个字符串变量file_content中。

由于文件内容相对较少，可以一次性读取全部。老师通过print(file_content)确认已成功读取到文件的所有文本内容，验证这是一个字符串数据类型。然后对这个字符串调用re.findall(pattern, file_content)，将匹配结果存入results列表。最后遍历results，打印出所有提取到的手机号码。

运行结果显示成功获取了17775806806、15799938625等多个手机号码。老师总结道，这种从文件提取特定格式信息的能力非常实用，可以过滤掉所有无关文字，只保留对自己有用的数据。

### 第93练：密码验证（8:24 - 20:22）

第93练引入了条件判断与正则表达式的结合应用。需求是定义一个函数verify_password()来验证密码是否符合以下规范：长度位于6到20之间、必须包含至少一个小写字母、必须包含至少一个大写字母、必须包含一个数字字符、必须包含一个特殊符号。

老师首先导入re模块，随后开始定义函数。函数名定义为verify_password，接收一个参数password用于传入待验证的密码。老师强调函数设计时要考虑三大要素：作用、参数和返回值。本函数的作用是验证密码规范性，参数是待验证的密码。

关于返回值，老师提出了一个提升用户体验的设计思路：返回两个数据构成元组——第一个是布尔值表示是否符合规范（True或False），第二个是字符串表示不规范的具体原因（如果规范则返回None）。这样做的好处是，当验证不通过时，用户能清楚地知道失败的原因是什么。

接下来逐一编写验证逻辑。老师采用"先写反"的技巧——先假设密码是规范的，默认返回(True, None)，然后逐一检查各项条件，如果不满足就返回(False, 原因)。这样可以减少代码量，因为只有在不满足条件时才需要处理，否则默认通过。

第一个条件是长度检查，使用if not (6 <= len(password) <= 20)判断。如果长度不在6到20之间（含6和20），就返回(False, "密码必须在6~20之间")。第二个条件是必须包含至少一个小写字母，使用re.search(r"[a-z]", password)在整个密码中查找小写字母字符a到z。如果没找到（即结果为None），就返回(False, "密码必须包含至少一个小写字母")。第三个条件是必须包含至少一个大写字母，逻辑相同，使用re.search(r"[A-Z]", password)查找，失败时返回(False, "密码必须包含至少一个大写字母")。

第四个条件是必须包含一个数字字符，可以使用re.search(r"\d", password)或re.search(r"[0-9]", password)。\d匹配任意一个数字字符。如果没找到，返回(False, "密码必须包含至少一个数字字符")。

第五个条件是必须包含至少一个特殊符号。老师讲解了两种实现思路：一是将可接受的字符全部列举出来（数字、大小写字母、下划线、减号），然后用[^...]表示除此之外的都是特殊符号；二是直接使用\W（大写W）表示非单词字符。老师选择了后者，\W可以匹配除数字、字母、汉字和下划线以外的字符，包括各种特殊符号如井号、星号等。

老师随后进行多轮测试验证函数正确性。第一轮测试密码"Hello#666"，包含小写字母、大写字母、数字和井号特殊符号，长度适中，运行结果显示符合规范，原因显示为None。第二轮去掉数字，改为"Hello#World"，运行结果显示返回False并提示"密码必须包含至少一个数字字符"。第三轮去掉大写字母，改为"hello#666"，提示"密码必须包含至少一个大写字母"。第四轮去掉特殊符号井号，改为"Hello666"，提示"密码必须包含至少一个特殊字符"。第五轮测试超长字符串，提示"密码必须在6~20之间"。

老师特别总结了"先写反"判断技巧的原理：假设默认规范可以减少条件分支，只有在发现问题时才明确指出具体原因，这样代码更简洁且用户体验更好。

### 第94练：提取电子邮箱（20:22 - 26:08）

第94练从信息提取转向更复杂的正则模式构建。需求是给定一段包含多个邮箱地址的文本，要求提取出所有标准的邮箱地址。老师打开预先准备的PY文件，展示示例文本中混杂着各种信息，其中部分符合标准邮箱格式，部分不符合。

同样首先导入re模块。老师指出邮箱地址的规则比手机号码稍微复杂一些，建议使用re.compile()方法配合re.VERBOSE参数进行编译，这样可以使用多行字符串编写正则表达式，提高代码清晰度和可维护性。

邮箱地址的正则模式构建过程如下：

前半部分（@符号之前的用户名部分）可以包含数字（0-9）、小写字母（a-z）、大写字母（A-Z）、下划线（_）和减号（-），数量至少一个，因此写作[0-9a-zA-Z_.-]+。加号表示前面的字符类至少出现一次，因为用户名不能为空。

中间是固定的@符号，直接写@即可。

后半部分（@符号之后的域名部分）可以包含数字、小写字母或大写字母，数量至少一个，写作[0-9a-zA-Z]+。然后是点号（.），由于单独的点号在正则中表示任意字符，需要用反斜杠转义写成\.。最后是顶级域名，通常是2到4个字母，如com、cn、net、org等，写作[a-zA-Z]{2,4}。

完整的邮箱正则模式为：
```python
pattern = re.compile(
    r"[0-9a-zA-Z_.-]+@[0-9a-zA-Z]+\.[a-zA-Z]{2,4}",
    re.VERBOSE
)
```

re.VERBOSE参数的作用是支持多行编写和添加注释，使复杂的正则表达式更易读。

使用pattern.findall(content)方法从文本中提取所有匹配的邮箱地址，返回结果存入results列表。遍历results逐条打印。

运行结果显示成功提取出符合规则的邮箱地址，如python666@163.com等。同时正确排除了缺少@符号或不合法格式的字符串。老师指出，re.VERBOSE参数对于编写复杂正则表达式非常有帮助，可以让代码结构更清晰。

### 第95练：提取购物详情信息（26:08 - 约37:00）

第95练是本P最具综合性的练习，涉及分组捕获和多行文本处理。需求是给定一段包含购物信息的文本，要求提取出每一样物品的购买数量、物品名称以及所花费的金额。老师打开PY文件展示示例文本，包含多行购物记录，如"小明上街买菜"、"买了一斤黄瓜花了八元"、"两斤葡萄13.5元"等。

老师首先导入re模块。由于文本中有多行不同内容，每行包含一个完整的购物信息，最佳策略是将文本按行分割后逐行处理。使用content.split("\n")将文本以换行符为分隔符切分成列表，每一项元素就是一行内容。

通过for循环遍历lines列表，可以逐行读取数据。老师先用print(line)验证分割效果，确认每行数据都正确获取。

接下来定义正则表达式的匹配模式，从一行文本（如"买了一斤黄瓜花了八元"）中提取三个目标数据：购买数量（斤数）、物品名称、花费金额（几元）。使用圆括号()创建三个捕获组：

第一组：\d+，匹配购买数量（至少一个数字），如一斤黄瓜中的一。

第二组：.+?，非贪婪匹配物品名称，匹配任意字符至少一次。由于物品名称后面跟着"花了"这样的固定文字，可以用非贪婪模式.+?尽可能少地匹配，直到遇到后续的固定内容。

第三组：(\d+\.?\d*)?，可选的金额部分。其中\d+匹配整数部分，\.?可选的小数点，\d*零个或多个小数位数。这个整体用圆括号包裹并加问号，表示整个金额部分可有可无，因为有些记录是整数（如八元），有些是小数（如13.5元）。

完整的模式为：r"(\d+).+?花了(\d+\.?\d*)?元"

定义pattern变量接收编译后的模式。使用re.search()方法在每一行文本中进行匹配（注意这里用search而非findall，因为每行只有一个目标，不需要查找所有）。search会从头到尾扫描，找到第一个匹配就返回match对象，如果没找到则返回None。

在for循环中，对每一行调用re.search(pattern, line)。然后通过if match判断是否匹配成功（即返回值不是None）。如果匹配成功，使用match.groups()方法获取所有捕获组的内容，返回一个元组。或者使用match.group(1)、match.group(2)、match.group(3)分别获取第一、第二、第三个分组的内容（注意group的编号从1开始，不是0）。

老师演示了两种获取分组数据的方式。第一种是用groups()获取全部分组，然后用元组下标访问各元素。第二种是直接使用group(1)、group(2)、group(3)分别获取指定序号的分组。第二种方式更直观，因为无需记住下标对应的含义。

运行结果验证了分组捕获的正确性。例如"买了一斤黄瓜花了八元"，groups()返回('一', '黄瓜', None)，group(1)返回'一'，group(2)返回'黄瓜'，group(3)返回None（因为八元是整数，没有小数部分）。而"两斤葡萄13.5元"返回('两', '葡萄', '13.5')，成功捕获了小数金额。老师使用f字符串格式化输出结果，展示为"一斤黄瓜 8块"和"两斤葡萄 13.5块"等形式。

老师还演示了分组嵌套的情况，说明groups()返回的元组中包含所有分组的内容，用户可以根据需要选择性使用。
