## 知识骨架

```
re模块
├── 基础方法
│   ├── findall()      → 提取所有匹配项，返回列表
│   └── search()       → 查找第一个匹配项，返回match对象或None
├── 编译方法
│   └── compile(..., re.VERBOSE)  → 处理复杂正则，支持多行和注释
├── match对象方法
│   ├── groups()       → 返回所有捕获组组成的元组
│   └── group(n)       → 返回第n个捕获组（n从1开始）
│
正则表达式原子
├── \d{9}              → 9个数字
├── [3-9]              → 字符类：3到9之间的任意一个数字
├── \W                 → 非单词字符（特殊符号）
├── \.                 → 转义点号（字面量句号）
├── +?                 → 非贪婪匹配（.+?）
│
正则表达式修饰符
└── re.VERBOSE         → 启用多行编写和注释，提升可读性
```

## 易错点

1. **手机号正则第二位**：必须写[3-9]，不能写\d。1[3-9]排除了12开头的号码，这是真实号码规则。

2. **点号转义**：邮箱域名中的句号必须写成\.，否则.表示任意字符。

3. **分组编号从1开始**：group(0)是整个匹配，group(1)才是第一个捕获组。写成group(0)会拿到完整匹配而非目标字段。

4. **search返回None**：re.search()没匹配到时返回None而非空列表，直接对None调用group()会报错。必须先用if match判断。

5. **\W的边界**：\W匹配非单词字符，包括空格。如果密码验证中用\W判断特殊符号，空格也会被认定为"特殊符号"，需根据业务需求决定是否在字符集中显式排除空格。

6. **非贪婪.+?**的使用场景：当目标字段前后有固定分隔文字时，用.+?可避免贪婪匹配吞掉过多内容。

7. **re.VERBOSE中的空格**：启用VERBOSE后，正则表达式里的空格会被忽略，如果需要匹配空格必须显式写出\s或把空格放入字符类[\ ]。
