## 核心问题
如何用Python标准库`datetime`模块完成**日期获取、转换、运算、生成**四类任务。

## 知识骨架

```
datetime模块
├── datetime.datetime类
│   ├── datetime.now()                    ← 获取当前时间
│   ├── strptime(字符串, 格式)            ← 字符串→datetime对象
│   ├── strftime(格式)                    ← datetime对象→字符串
│   ├── .year/.month/.day等属性           ← 访问日期组件
│   └── replace(year=?, month=?, day=?)   ← 返回新日期对象
│
├── datetime.date类
│   └── date(年, 月, 日)                  ← 创建纯日期对象（无时间）
│
└── datetime.timedelta类
    └── timedelta(days=?, seconds=?)      ← 表示时间差，用于加减运算
```

**五种核心操作**：

| 操作类型 | 方法/对象 | 输入 | 输出 |
|----------|-----------|------|------|
| 获取当前时间 | `datetime.now()` | 无 | datetime对象 |
| 字符串→日期 | `strptime(s, fmt)` | 字符串 | datetime对象 |
| 日期→字符串 | `strftime(fmt)` | datetime对象 | 字符串 |
| 日期±时间差 | `datetime ± timedelta` | datetime + timedelta | datetime |
| 计算时间差 | `datetime - datetime` | datetime | timedelta |

**格式化符号速查**：`%Y`(4位年) `%m`(月) `%d`(日) `%H`(24时) `%M`(分) `%S`(秒)

## 关系与依赖

```
strptime ←→ strftime          ← 互逆操作，共同依赖格式字符串
datetime对象 ──加减timedelta──→ 新datetime对象
datetime对象 ──相减──→ timedelta对象 ──.days属性──→ 天数int

replace()      ← 不修改原对象，返回新对象
date() vs datetime.now()  ← date只有日期，datetime包含时分秒
```

## 输入 / 输出 / 前置 / 支撑

| 函数/方法 | 输入 | 输出 | 依赖 |
|-----------|------|------|------|
| `datetime.now()` | 无 | `datetime` | `import datetime` |
| `strptime(s, fmt)` | 字符串 | `datetime` | 需已知原字符串格式 |
| `strftime(fmt)` | `datetime` | 字符串 | 需指定目标格式 |
| `date(y,m,d)` | (int, int, int) | `date` | 参数需合法 |
| `timedelta(days=n)` | int | `timedelta` | 构造时间差 |
| `replace(**kwargs)` | `datetime` | `datetime` | 新对象非原地修改 |
| `days`属性 | `timedelta` | int | 两datetime相减 |

**字符串格式约定**：本P统一使用`"%Y-%m-%d"`作为标准输入输出格式。

## 易错点

1. **边界条件用`<=`而非`<`**：日期范围包含首尾两端，循环或判断时必须用`<=`（第65练）
2. **类型不匹配**：字符串与datetime对象不能直接相减，必须先用`strptime()`转换
3. **replace()不修改原对象**：返回新对象，需接收返回值
4. **闰年不精确**：天数÷365估算年龄误差大，仅作粗略参考（第62练）
5. **timedelta参数名**：构造时用`days=`而非直接传位置参数

- **格式符号**：遵循C标准（`%Y` `%m` `%d`等），与`time.strftime()`通用
