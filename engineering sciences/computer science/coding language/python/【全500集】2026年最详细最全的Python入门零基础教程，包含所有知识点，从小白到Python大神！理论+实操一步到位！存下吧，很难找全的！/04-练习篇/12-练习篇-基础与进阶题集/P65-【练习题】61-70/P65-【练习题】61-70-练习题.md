# P65 练习题 - 【练习题】61-70
## 一、结构识别

**题目1**：指出以下代码中的错误并说明原因

```python
import datetime

d = datetime.date(2025, 13, 1)  # 创建日期对象
print(d - "2024-01-01")          # 尝试直接相减
```

**考察点**：date类的参数合法性、字符串与日期对象的类型差异

---

**题目2**：以下代码的输出是什么？

```python
import datetime

dt = datetime.datetime.now()
new_dt = dt.replace(day=1)
print(dt.day)
print(new_dt.day)
```

**考察点**：replace()方法的返回值特性与原地修改的区分

---

## 二、机制理解

**题目3**：填空题

strptime 和 strftime 是互逆操作：

- `strptime(字符串, 格式)`：将 **字符串 → datetime对象**，需要预先知道________。
- `strftime(格式)`：将 **datetime对象 → 字符串**，需要指定________。

**考察点**：两种转换的方向性与前提条件

---

**题目4**：判断正误并说明理由

使用 `datetime.datetime.now() - datetime.datetime(2020, 1, 1)` 可以直接得到两个日期之间的天数差。

**考察点**：datetime相减返回值的类型与获取天数的方法

---

## 三、最小实践

**题目5**：基础转换

将字符串 `"2025-06-15"` 转换为 datetime 对象，提取月份，然后输出格式为 `"6月15日"` 的字符串。

**考察点**：strptime解析、属性访问、strftime格式化

---

**题目6**：日期运算

计算 2025-01-01 往后 100 天是哪一天，并输出为 `"YYYY-MM-DD"` 格式。

**考察点**：datetime与timedelta的加法运算

---

## 四、扩展思考

**题目7**：边界条件处理

计算一个日期范围 `[2025-01-01, 2025-01-31]` 的总天数，循环遍历并打印每一天的日期。

**考察点**：日期边界使用 `<=` 而非 `<`、循环结构、timedelta实现日期递增

---

**题目8**：综合应用 — 年龄计算

用户输入出生日期（格式：`YYYY-MM-DD`），程序判断其是否已满18岁（包含当天）。

```
输入：2010-01-01 → 输出：已满18岁
输入：2010-06-01 → 输出：未满18岁
```

**考察点**：strptime解析、timedelta构造、日期比较、闰年自动处理

---

## 五、最小替代练习

**题目9**：time模块实现同样功能

使用 `time.strptime()` 和 `time.strftime()` 替代 datetime 模块，实现：
- 将 `"2025-03-20"` 解析为时间元组
- 将当前时间格式化为 `"%Y年%m月%d日"` 格式

**考察点**：time模块与datetime模块的格式符号通用性、结构体时间与datetime对象的区别

---

**参考答案要点**

| 题号 | 关键答案 |
|------|----------|
| 1 | 月份13非法；字符串不能与date直接相减，需转换后用timedelta |
| 2 | 输出原日期的day；1（原对象不变） |
| 3 | 原字符串格式；目标格式 |
| 4 | 错误；返回timedelta，需访问`.days`属性才得天数 |
| 5 | strptime→`.month`→strftime |
| 6 | datetime + timedelta(days=100) → strftime输出 |
| 7 | 循环条件用 `<=`；每天加 `timedelta(days=1)` |
| 8 | 用 `timedelta(days=18*365)` 或精确到日期对象比较；包含当天 |
| 9 | 格式符号通用；time模块无timedelta，需手动转换 |
