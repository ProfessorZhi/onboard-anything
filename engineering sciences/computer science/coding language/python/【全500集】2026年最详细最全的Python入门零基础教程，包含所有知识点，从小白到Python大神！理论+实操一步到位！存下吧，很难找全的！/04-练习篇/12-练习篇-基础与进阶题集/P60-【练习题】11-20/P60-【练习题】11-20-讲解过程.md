## 讲解过程

### 第11练 列表偶数求和（0:00-5:00）

**需求确认**：给定整数列表，计算并打印所有偶数的和。

**逐步实现**：

1. **遍历列表**：使用`for number in numbers`循环逐个取出列表元素，先打印验证是否成功取出每个元素。
2. **偶数判断**：强调不能使用整除符号`//`，因为`3//2`和`2//2`结果都是1，无法区分奇偶。正确方法是使用取模运算符`%`，偶数的特点是`number % 2 == 0`（除以2余数为0）。举例说明：2%2=0、4%2=0（偶数），3%2=1（奇数）。
3. **累加求和**：在循环外部定义`result = 0`作为累加变量（初始值为0因为0+任何数=任何数），循环内部执行`result += number`或`result = result + number`将满足条件的偶数累加。
4. **输出结果**：循环结束后打印`result`，验证结果正确性（列表[2,4,6,8,10]的偶数和为30）。

**知识点总结**：遍历列表元素、奇偶数判断（取模运算）、累加器模式。

### 第12练 字典中找年龄最大者（5:00-10:20）

**需求确认**：字典以姓名为键、年龄为值，找出并打印年龄最大者的姓名与年龄。

**逐步实现**：

1. **初始化变量**：定义`max_age`保存最大年龄，初始值设为`float('-inf')`（负无穷大），确保任何年龄都能替换它；定义`max_name`保存对应姓名，初始值为空字符串`""`。
2. **遍历字典**：使用`for name, age in dict.items()`解包键值对，获取每个姓名和对应的年龄。先打印验证是否正确取出所有键值对。
3. **比较更新**：用`if age > max_age`判断当前年龄是否大于已记录的最大年龄，若成立则更新`max_age = age`和`max_name = name`。
4. **输出结果**：循环结束后打印格式化字符串，显示年龄最大者的姓名和年龄。验证：字典`{"张三": 18, "李四": 50}`输出"李四 50"。

**知识点总结**：字典遍历（`items()`方法）、解包赋值、动态更新最大值的方法、负无穷作为初始值。

### 第13练 有序列表插入元素（10:20-18:30）

**需求确认**：给定已按从小到大排序的列表，用户输入一个数，按原有排序规律将其插入正确位置。

**逐步实现**：

1. **获取用户输入**：使用`input()`函数并配合提示信息"请输入一个数"，由于`input()`返回字符串，需要用`int()`进行类型转换。
2. **索引遍历**：使用`for i in range(len(li))`基于下标遍历列表，`range(len(li))`生成0到`len(li)-1`的下标序列。验证：`[21, 33, 57, 88, 95]`遍历得到下标0-4及对应元素。
3. **查找插入位置**：用`if li[i] > num`判断当前元素是否大于待插入数，若是则说明找到了插入位置，执行`li.insert(i, num)`在位置`i`插入。
4. **提前退出**：插入成功后使用`break`退出循环，提高程序执行效率。
5. **处理末尾情况**：若循环正常结束（未执行`break`），说明待插入数比所有元素都大，此时使用`for-else`结构在`else`分支中执行`li.append(num)`追加到列表末尾。
6. **多场景验证**：输入11插入到最前面、输入59插入到57之后88之前、输入120追加到末尾。

**知识点总结**：类型转换、`range()`与`len()`组合遍历、`insert()`方法插入指定位置、`append()`追加到末尾、`for-else`结构（循环正常结束时的兜底逻辑）、`break`提前退出。

### 第14练 三个数字排序（18:30-24:30）

**需求确认**：接收用户输入的三个数字，按从小到大顺序输出。

**方式一：条件判断与值交换**

1. **获取输入并类型转换**：三次使用`input()`获取用户输入，用`float()`转换为浮点型（选择浮点型是因为范围比整型更广），分别存入`num1`、`num2`、`num3`变量。
2. **两两比较交换**：按照"冒泡排序"思路进行三轮比较——先比较`num1`和`num2`若前者大则交换，再比较`num1`和`num3`若前者大则交换，最后比较`num2`和`num3`若前者大则交换。使用序列赋值`num1, num2 = num2, num1`同时交换两个变量的值，原理是右边的值先计算再同时赋值给左边。
3. **输出结果**：循环结束后打印`num1`（最小）、`num2`（中间）、`num3`（最大）。

**方式二：内置`sorted()`函数**

1. 将三个数放入列表`[num1, num2, num3]`。
2. 调用`sorted()`函数对该列表排序，返回排序后的新列表。
3. 用变量接收结果并打印。验证：`sorted([10.5, 8.8, 12.7])`输出`[8.8, 10.5, 12.7]`。

**知识点总结**：类型转换、序列赋值交换值、`sorted()`内置函数简化排序、浮点型与整型的选择。

### 第15练 找最小连续九整除数（24:30-30:40）

**需求确认**：用户输入基数，找出由连续N个"9"组成的、能被该基数整除的最小数，并输出该数及除法结果。

**示例**：输入3→输出"9 ÷ 3 = 3.0"；输入11→输出"99 ÷ 11 = 9.0"。

**逐步实现**：

1. **获取用户输入**：用`input()`获取基数并用`int()`转换为整型。
2. **初始化计数器**：定义`n = 1`表示"9"的个数，至少为1个9。
3. **死循环构建数字**：使用`while True`死循环（因为循环次数未知），循环内部每次构建由N个9组成的数字——使用字符串重复操作`"9" * n`，再转换为整型`int("9" * n)`。
4. **递增计数器**：每次循环结束后执行`n += 1`，确保下次构建更多个9。
5. **整除判断**：在循环内部用`if n_9 % num == 0`判断是否能被整除，若能整除则执行格式化输出并用`break`退出循环。
6. **输出格式**：使用f-string格式化输出，包含N的值（九的个数）、实际数字`n_9`、除法结果`n_9 / num`。

**知识点总结**：`while True`死循环配合`break`、`"9" * n`字符串重复构造连续字符、`n_9 % num == 0`整除判断、格式化输出。

### 第16练 按数量打印星号（30:40-35:30）

**需求确认**：用户连续输入最多7个整数（每个在1-50之间），对每个有效输入打印对应数量的星号。

**逐步实现**：

1. **输入验证循环**：由于用户可能输入无效数字（超出1-50范围），需要循环验证。使用`while True`死循环嵌套在外层，在内部获取用户输入并用`int()`转换为整型。
2. **范围判断**：用`if 1 <= num <= 50`判断是否为有效输入，若是则打印`"*" * num`（星号重复N次）并用`break`退出循环；若否，则打印提示"输入无效，请输入1~50之间的整数"重新循环。
3. **外层循环控制次数**：在外层用`for i in range(7)`循环7次，确保最多接受7个有效输入。内层的`while`循环负责每个输入的反复验证直到有效。
4. **多场景验证**：输入7打印7个星号、输入58或59因超出范围被拒绝需重新输入、输入35打印35个星号。

**知识点总结**：循环嵌套（`for`循环嵌套`while`循环）、输入有效性检查、`break`提前退出、死循环确保有效输入。

### 第17练 判断奇数还是偶数（35:30-约38:30）

**需求确认**：用户输入一个数，判断是奇数还是偶数并输出结果。

**逐步实现**：

1. **获取用户输入**：用`input()`获取输入并用`int()`转换为整型，赋值给`num`变量。
2. **偶数判断**：回顾第11题知识点，使用取模运算符`%`。偶数特点：`num % 2 == 0`（除以2余数为0）。举例：200%2=0、400%2=0。
3. **条件分支**：使用`if-else`结构——`if num % 2 == 0`时输出"num是一个偶数"，`else`时输出"num是一个奇数"。
4. **验证**：输入100输出偶数、输入51输出奇数。

**知识点总结**：取模运算符`%`判断整除、`if-else`二选一分支。
