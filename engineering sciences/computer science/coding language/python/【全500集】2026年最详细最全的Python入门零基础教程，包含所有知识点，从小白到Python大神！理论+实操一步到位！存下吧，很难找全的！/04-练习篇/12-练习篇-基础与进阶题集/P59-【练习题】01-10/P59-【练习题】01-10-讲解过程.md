### 第一题：变量赋值与加法运算（0:00 ~ 2:53）

**代码演示**：

### 第二题：比较两个数字的大小（2:59 ~ 7:30）

**需求引入**：定义两个整型变量，比较它们的值并打印比较结果。

**代码演示**：
1. 定义变量 `a = 10`，`b = 20`。
2. 分析三种比较情况：a > b、a == b、a < b，指出三者互斥且覆盖所有可能，涉及条件判断应想到 `if` 语句。
3. 选择 `if-elif-else` 结构而非简单的 `if-else`，因为 `else` 无需写条件，表示"上述均不满足"，能减少代码量。
4. 初版代码直接在字符串中写死"大于""等于""小于"文字，运行验证输出 `A小于B`（因为 a=10 < b=20）。
5. 发现输出不够直观（缺少具体数值），引出 **F-string 格式化字符串**：在字符串前加 `f`，用花括号 `{}` 包裹变量。
6. 将三个分支的输出改为 `f"A {">"} B"`、`f"A 等于 B"`、`f"A 小于 B"` 格式。

**运行验证**：a=10, b=20 时输出 `10小于20`；a=20 时输出 `10等于20`；a=30 时输出 `30大于20`。

**知识点收束**：总结本练习涉及 `if-elif-else` 多分支结构、比较运算符（`>`、`==`）、F-string 格式化输出。补充说明还有 `<`、`<=`、`>=` 等比较运算符可供使用。

### 第三题：计算1~100的整数和（7:30 ~ 14:22）

**需求引入**：编写程序计算从1到100（包含两端）的所有整数的和，并打印结果。

**代码演示**：
1. 说明需要依次取出 1~100 的每个数字，因此先要**生成一个序列**。
2. 使用 `range` 函数，指出它是一个"数字生产机"，用于生成一系列连续的整数。
3. 讲解 `range` 的**包前不包后**规则：`range(1, 100)` 生成 1~99，要包含100需写成 `range(1, 101)`。
4. 直接打印 `range` 对象时不会列出所有数字——它是为循环设计的可迭代对象，需要通过 `for` 循环遍历取出每个数字。
5. 写 `for i in range(1, 101): print(i)`，运行验证循环确实依次取出 1~100。
6. 引入**累加思想**：类比手算 1+2+3+4+5 的过程——每次用前一次的计算结果加上当前数字，再重新赋值给结果变量。
7. 在循环前定义 `result = 0` 作为累加器初始值，解释 0 加任何数等于任何数本身。
8. 循环体内写 `result += i`（复合赋值运算符，等价于 `result = result + i`），将当前数字累加到总和。
9. 将 `print(result)` 初始放在循环体内（缩进状态下），运行后发现在循环期间打印了100次——因为缩进的代码属于循环体，重复执行100次。
10. 将 `print(result)` 取消缩进，放在循环结束后，只输出最终结果 `5050`。

**运行验证**：逐步演示累加过程：0+1=1 → 1+2=3 → 3+3=6 → 6+4=10，最终结果 5050。

**知识点收束**：总结本练习涉及 `for` 循环遍历整数序列、复合赋值运算符（`+=`、`-=`、`*=` 等）实现累加操作，以及循环体缩进对执行次数的影响。

### 第四题：统计1~4组成的三位数（14:22 ~ 19:56）

**需求引入**：有四个数字 1、2、3、4，统计能组成多少个互不相同且无重复数字的三位数，并列出各是多少。

**代码演示**：
1. 思路：使用三层嵌套 `for` 循环分别遍历百位、十位、个位数字，生成所有可能的组合。
2. 三个循环均用 `range(1, 5)`（因为包前不包后，实际生成 1~4）。
3. 先不加条件直接打印 `i j k`，运行展示所有原始组合（包含大量重复数字的组合如111、112等）。
4. 引入筛选条件：三个数字**互不相同**。用 `if` 语句配合逻辑运算符 `and` 和不等号 `!=` 写出：`i != j and i != k and j != k`，确保三个条件同时满足。
5. 将满足条件的组合打印出来，验证重复数字的组合已被过滤掉。
6. 发现题目还要求统计**个数**，于是定义计数器变量 `count = 0`。
7. 条件成立时执行 `count += 1`，循环结束后用 F-string 打印总个数。

**运行验证**：运行后输出24个满足条件的三位数，并通过列举验证总数为24。

**知识点收束**：总结本练习涉及三层嵌套 `for` 循环遍历多维集合、`if` 语句结合逻辑运算符（`and`、`!=`）精确筛选，以及计数器变量统计满足条件的数量。

### 第五题：打印九九乘法表（19:56 ~ 26:29）

**需求引入**：编写程序打印9×9乘法口诀表。先展示一张九九乘法表的参考效果图。

**代码演示**：
1. 分析九九乘法表的结构：共9行，每行若干列。第1行1列、第2行2列……第i行i列（列数不超过行数），呈直角三角形。
2. 用嵌套 `for` 循环实现：外层控制行 `for i in range(1, 10)`，内层控制列 `for j in range(1, 10)`。
3. 先打印 `f"第{i}行 第{j}列"`，运行发现每列单独一行输出（每打印一个值就换行）。
4. 解释 `print` 函数默认参数 `end="\n"`（换行符）导致每列后自动换行。
5. 改为 `end="\t"`（制表符），运行后发现同一行所有列都挤在一起输出，不换行——说明 `end` 参数控制输出结束行为。
6. 在内层循环结束后添加一个单独的 `print()`（即 `print(end="\n")` 或直接 `print()`），实现"每行所有列输出完才换行"。
7. 此时得到 9×9 的完整矩形，但九九乘法表应为直角三角形——第几行就只有几列。
8. 修改内层循环范围为 `range(1, i + 1)`（列数跟随行数 i）。
9. 观察乘法表格式：第 i 行第 j 列为 `j × i = i*j`，改为 `print(f"{j}×{i}={i*j}", end="\t")`。
10. 运行验证，直角三角形乘法表成功输出，每行末尾自动换行。

**运行验证**：最终输出符合九九乘法表标准格式，第1行1×1=1，第2行1×2=2、2×2=4，以此类推。

**知识点收束**：总结本练习涉及嵌套循环生成二维结构、`print` 函数的 `end` 参数控制输出结束行为（`\n` 换行、`\t` 制表符对齐）、以及列数随行数动态变化的直角三角形结构。

### 第六题：打印所有的水仙花数（26:29 ~ 31:42）

**需求引入**：编写程序找出并打印所有的水仙花数。先解释什么是水仙花数：一个三位数，其各位数字的立方和等于该数本身（如 1³+5³+3³=153）。

**代码演示**：
1. 思路：遍历所有三位数（100~999），逐一判断是否为水仙花数。
2. 用 `for n in range(100, 1000)` 遍历，先打印验证获取到了 100~999。
3. 判断水仙花数需要先**分别提取百位、十位、个位数字**：
   - **百位**：整除 100（`n // 100`）。例如 123//100=1，223//100=2。
   - **十位**：先整除10再取余10（`n // 10 % 10` 或 `(n // 10) % 10`）。原理：整除10去掉个位（如123→12），再对10取余得到个位（即12的个位=2）。
   - **个位**：直接对10取余（`n % 10`）。例如 123%10=3。
4. 分别用变量 `i`（百位）、`j`（十位）、`k`（个位）保存。
5. 计算立方和：`sum_of_cubi = i**3 + j**3 + k**3`。
6. 用 `if` 判断立方和是否等于原数 `n`，若相等则打印 `n`。
7. 运行后输出 153、370、371、407 四个水仙花数，并逐一验证。

**运行验证**：逐个验证——153=1³+5³+3³=1+125+27=153；370=3³+7³+0³=27+343+0=370。

**知识点收束**：总结本练习涉及整数的数位提取（整除 `//` 和取余 `%` 运算符）、幂运算（`**` 表示乘方）、以及条件判断找出满足特定规则的数字。

### 第七题：猴子吃桃问题（31:42 ~ 37:54）

**需求引入**：猴子每天早上吃掉前一天剩下的一半零一个，到第10天早上只剩1个桃子了，问第一天共摘了多少个。

**代码演示**：
1. 从**已知条件**出发：第10天早上桃子数量为1。
2. 采用**逆向思考**的方法，从第10天往前推算到第1天。
3. 推导公式：当天早上桃子数 = （前一天早上桃子数 + 1）× 2。反向即为：`previous = (current + 1) * 2`。
4. 手工验证公式：
   - 第10天剩1个 → 第9天：(1+1)×2 = 4 个
   - 第9天剩4个 → 第8天：(4+1)×2 = 10 个
   - 第8天剩10个 → 第7天：(10+1)×2 = 22 个
   - ……依此类推
5. 代码实现：`for day in range(9, 0, -1):` 从第9天倒序遍历到第1天（`range(9, 0, -1)` 步长-1实现倒序）。
6. 循环体内：`peaches = (peaches + 1) * 2`。
7. 循环结束后打印 `peaches`，即为第一天摘的桃子总数。
8. 运行得出结果 **1534 个**。
9. 进一步正向验证：1534÷2-1=766 → 766÷2-1=382 → 382÷2-1=190 → 190÷2-1=94 → 94÷2-1=46 → 46÷2-1=22 → 22÷2-1=10 → 10÷2-1=4 → 4÷2-1=1，完整链路验证正确。

**运行验证**：输出第一天共摘了1534个桃子，并通过正向除法验算逐天递减数量完全吻合。

**知识点收束**：总结本练习涉及逆向思维的数学建模、倒序 `for` 循环（`range` 步长为负）、以及利用递推公式通过循环实现计算。这种从结果倒推的方法在许多算法问题中具有重要应用价值。

## 本P结论

本P完成了 Python 入门练习题 01~08（共10道中的前8道）的讲解，覆盖以下核心知识点：

1. **变量定义与赋值**：通过赋值运算符 `=` 实现变量创建。
2. **算术运算符**：加法 `+`、幂运算 `**`。
3. **比较运算符**：`<`、`<=`、`>`、`>=`、`==`、`!=`。
4. **print 函数**：基本打印输出、`end` 参数（`\n` 换行与 `\t` 制表符）对输出格式的控制。
5. **input 函数**：接收用户输入，返回值为字符串类型。
6. **F-string 格式化字符串**：`f"..."` 语法在字符串中嵌入变量值。
7. **range 函数**：生成整数序列，包前不包后；正向步长默认1，可通过负数步长实现倒序。
8. **for 循环**：遍历整数序列、遍历字符串字符、多层嵌套循环。
9. **if-elif-else 多分支结构**：处理三种及以上互斥条件的分支判断。
10. **逻辑运算符**：`and` 连接多条件、`!=` 判断不等。
11. **复合赋值运算符**：`+=` 实现累加，等价于 `x = x + n`。
12. **计数原理**：定义初始计数器变量，在条件成立时累加统计数量。
13. **数位提取**：通过整除 `//` 和取余 `%` 运算提取整数的个位、十位、百位数字。
14. **逆向思维与递推**：猴子吃桃问题中从已知结果倒推初始值的方法，体现了程序设计中从目标反推条件的思路。
