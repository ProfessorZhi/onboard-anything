# P66 讲解过程 - 【练习题】71-80
## 本P定位

本P包含两道练习题的讲解：

- 第71链：定义银行账户类（BankAccount），支持存款和取款操作
- 第72链：实现简单版学生成绩管理系统，包含添加学生、记录成绩、计算平均分

两题都是面向对象编程的实战练习，重点在于类的设计与方法的实现思路。

## 讲解过程

### 第71链：银行账户类（0:00-15:40）

**1. 需求确认与类框架搭建**

老师先明确今天的练习目标：创建一个简单的银行账户类，支持存款和取款操作。打开PyCharm后，确认已准备好相关文件。

使用class关键字定义BankAccount类，说明该类用于模拟一个简单的银行账户。

**2. 分析类需要的方法和属性**

老师引导学生思考：一个类无非就是属性和方法两类组成部分。

方法方面：存款和取款操作是明确需要的，分别命名为deposit和withdraw。

属性方面：开户时需要有账户持有人的信息，以及当前的账户余额。讨论属性应该定义为类属性还是实例属性——如果定义为类属性，所有账户的持有者和余额都一样，这显然不符合现实，所以必须定义为实例属性。

**3. 定义__init__初始化方法**

在__init__方法中定义两个实例属性：

- self.owner：账户持有人，作为形参，实例化时传入具体值
- self.balance：账户余额，初始值设为0

老师解释了为什么初始值设为0：新开户时账户本来就没钱。也可以让balance作为可选形参，默认值为0，这样开户时可以直接存一笔钱进去。

**4. 实现存款方法deposit**

存款方法的作用是执行存款操作。需要一个参数amount表示存入金额，因为每次存的钱可能不同。是否需要返回值？不需要，只要执行操作即可。

实现逻辑：
- 判断存入金额是否为正数
- 如果是正数：self.balance = self.balance + amount（累加到当前余额）
- 如果不是正数：提示存款失败，存款金额必须是正数

**5. 实现取款方法withdraw**

取款方法与存款相反，是在当前账户余额的基础上减少一定数目。同样需要amount参数，也不需要返回值。

实现逻辑（两个条件必须同时满足，用and连接）：
- 判断要取出的金额是否为正数
- 判断取出金额是否小于等于当前账户余额（不能超支）
- 如果满足条件：self.balance = self.balance - amount
- 如果不满足：提示取款失败，取款金额必须是正数且不能超过当前账户余额

老师特别强调：取款有两个限制条件，必须同时满足，用and连接。

**6. 实现获取余额方法get_balance**

为了方便查看余额，定义一个get_balance方法。该方法不需要参数，直接返回self.balance即可。

**7. 完整实例化与功能测试**

实例化对象：account = BankAccount("冰冰", 100)

测试步骤：
- 查看初始余额：print("当前余额:", account.get_balance())，结果为100
- 执行存款：account.deposit(500)，再次查看余额为600
- 执行取款：account.withdraw(200)，再次查看余额为400

异常情况测试：
- 尝试取款超过余额的金额（如取1000，只有400）：提示取款失败，余额保持400不变
- 尝试存入负数金额：提示存款金额必须是正数，余额保持400不变

### 第72链：学生成绩管理系统（15:40-结束）

**1. 需求确认**

练习目标：实现一个简单的学生成绩管理系统，该系统允许添加学生、记录学生成绩，并计算每个学生的平均成绩。

**2. 定义成绩管理类框架**

使用class关键字定义StudentGradeManager类，这个类用于管理学生及其成绩。

**3. 分析需要的属性**

老师提出：添加学生时，需要把学生存储到某个数据结构中。由于每个学生有不同的成绩，存在对应关系，列表无法很好地表达这种对应关系，因此选择字典来存储。字典的键为学生姓名，值为学生对象。

在__init__方法中定义self.students = {}，初始化为空字典。

**4. 实现添加学生方法add_student**

方法作用：添加学生。参数为name（要添加的学生名称），不需要返回值。

实现逻辑：
- 先判断学生是否已存在（name not in self.students）
- 如果不存在：添加到学生字典，值为Student(name)对象
- 如果已存在：提示该学生已存在

**5. 定义学生类Student**

既然字典的值是学生对象，就需要先定义一个Student类来表示单个学生。

Student类的__init__方法：
- self.name：学生姓名，作为形参传入
- self.grades：学生成绩列表，初始化为空列表（因为一开始没有成绩）

**6. 实现添加成绩方法add_grade**

在Student类中定义add_grade方法。参数为grade（要添加的成绩），不需要返回值。

实现逻辑：将grade添加到self.grades列表中，使用append方法。

**7. 实现计算平均成绩方法average_grade**

在Student类中定义average_grade方法。

方法作用：计算该学生的平均成绩。

参数：不需要，因为直接拿自己的成绩列表计算即可。

返回值：返回计算后的平均成绩，如果没有成绩则返回0。

实现逻辑：
- 先判断是否有成绩（列表是否为空）
- 如果没有成绩：返回0并结束函数（避免除以0的错误）
- 如果有成绩：计算sum(self.grades) / len(self.grades)并返回

老师详细讲解了为什么需要判断列表是否为空：如果直接计算，列表为空时长度为零，除以零会报错。

**8. 补充查找学生方法get_student（思路讲解）**

老师提到在成绩管理类中还可以定义查找学生的方法get_student，通过姓名在字典中查找对应的学生对象，返回该对象或None。

## 本P结论

**第71链结论：**
- 银行账户类包含两个实例属性（owner、balance）和三个方法（deposit、withdraw、get_balance）
- 存款和取款都需要对输入金额进行有效性检查
- 取款额外需要检查余额是否充足
- 通过实例化对象并进行存款、取款、查询操作验证了类的功能

**第72链结论：**
- 学生成绩管理系统包含两个类：StudentGradeManager（管理类）和Student（学生类）
- 使用字典存储学生，键为姓名，值为学生对象，便于维护对应关系
- 每个学生有自己的成绩列表，通过add_grade方法添加成绩
- 计算平均分前需要判断成绩列表是否为空，避免除零错误
