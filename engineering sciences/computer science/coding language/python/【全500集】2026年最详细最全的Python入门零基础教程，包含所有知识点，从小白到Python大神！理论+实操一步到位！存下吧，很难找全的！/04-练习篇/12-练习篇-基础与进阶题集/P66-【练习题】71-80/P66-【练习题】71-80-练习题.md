## 一、结构识别

**题目 1.1**  
BankAccount 类的以下属性中，哪项属于**实例属性**（每次创建对象时独立的属性）？

- A. account_number（银行账号）
- B. bank_name（银行名称，全分行共用）
- C. interest_rate（利率，全行统一）
- D. min_balance（最低存款额，全行统一）

**答案**：A  
**解析**：实例属性是与具体对象绑定的数据，每个账户持有人独立拥有自己的余额和户名。B/C/D 属于类属性，所有实例共享。

---

**题目 1.2**  
以下代码片段属于哪种方法类型？

```python
def add_student(self, name):
    if name not in self.students:
        self.students[name] = Student(name)
```

- A. 类方法（@classmethod）
- B. 静态方法（@staticmethod）
- C. 实例方法
- D. 构造方法

**答案**：C  
**解析**：方法第一个参数为 `self`，且操作 `self.students` 实例属性，属于实例方法。

## 二、机制理解

**题目 2.1**  
BankAccount.withdraw() 方法的取款条件为什么要用 **and** 连接两个判断？单独检查余额不行吗？

**答案要点**：  
不行。两个条件缺一不可：
- **正数检查**：防止取负数导致余额异常增加
- **余额检查**：防止取款超额导致余额为负

两者同时满足才能完成取款，体现了**数据安全校验**的编程思维。

---

**题目 2.2**  
StudentGradeManager 和 Student 的分工是什么？为什么不合并成一个类？

**答案要点**：

| 类 | 职责 | 设计原则 |
|---|---|---|
| Student | 存储单个学生的数据，执行成绩计算 | **实体类**：单一职责 |
| StudentGradeManager | 管理多个学生的增删查操作 | **管理器类**：封装集合操作 |

合并的弊端：违背单一职责原则，类会过于臃肿；后续扩展（如按班级管理、按科目分组）会很困难。

---

**题目 2.3**  
以下两种成绩查询方式，哪个更安全？为什么？

```python
# 方式一
student = manager.get_student("张三")
if student:
    student.add_grade(95)

# 方式二
if "张三" in manager.students:
    manager.students["张三"].add_grade(95)
```

**答案**：方式一更安全。  
**解析**：
- 方式二直接访问内部字典 `manager.students`，违反了**封装性**原则
- 方式一通过 `get_student()` 方法获取对象，将内部存储结构隐藏起来
- 如果未来改用列表或数据库存储，方式一无需修改外部调用代码

**题目 4.3**  
平均成绩返回 0 在业务上是否合理？如果学生还没有成绩，是否应该返回 None 更合适？请说明理由。

**答案要点**：

| 返回值 | 适用场景 |
|---|---|
| 返回 0 | 数学上合理（0-sum/0 未定义），且便于后续累加计算 |
| 返回 None | 区分“未参加考试”和“考试平均分为0”的业务含义 |

**建议**：根据实际业务需求决定。如果分数不可能为0（如满分100），返回 None 更严谨。

## 五、最小替代练习

**题目 5.1**  
不使用 PyCharm，在 Python 内置交互环境（IDLE）或在线 Python 解释器中，编写并运行以下代码：

1. 创建一个 BankAccount 实例，户主为"小明"，初始余额 100
2. 存钱 50 元
3. 取钱 80 元
4. 查询当前余额

```python
# 完整可运行代码
class BankAccount:
    def __init__(self, owner, balance=0):
        self.owner = owner
        self.balance = balance

    def deposit(self, amount):
        if amount > 0:
            self.balance += amount
            print(f"存款成功，余额：{self.balance}")

    def withdraw(self, amount):
        if amount > 0 and amount <= self.balance:
            self.balance -= amount
            print(f"取款成功，余额：{self.balance}")
        else:
            print("取款失败")

    def get_balance(self):
        return self.balance

# 测试
account = BankAccount("小明", 100)
account.deposit(50)
account.withdraw(80)
print(f"当前余额：{account.get_balance()}")
```

**预期输出**：

```
存款成功，余额：150
取款成功，余额：70
当前余额：70
```
