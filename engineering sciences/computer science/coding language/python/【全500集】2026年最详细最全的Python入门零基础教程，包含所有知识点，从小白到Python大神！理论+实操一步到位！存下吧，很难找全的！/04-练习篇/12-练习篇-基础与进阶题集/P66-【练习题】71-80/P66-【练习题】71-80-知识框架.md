# P66 知识框架 - 【练习题】71-80
## 本P位置

- **课程定位**：Python入门 → 练习题章节（第66页）
- **章节归属**：66-【练习题】71-80
- **作用**：面向对象编程的实战练习，训练类的设计与方法实现
- **前置知识**：类定义语法、__init__方法、实例属性、self关键字
- **后续关联**：为更复杂的OOP实战打基础

## 核心问题

- 如何将现实事物抽象为类（属性 + 方法）
- 实例属性 vs 类属性的选择依据
- 方法的参数校验逻辑（存款/取款的双重条件）
- 两个类如何协作（Manager类 + 实体类模式）
- 数据结构选型：字典存储键值对、列表存储序列

## 知识骨架

### BankAccount 类（单类设计）

```
BankAccount
├── 实例属性
│   ├── owner（账户持有人）
│   └── balance（账户余额，初始0）
├── __init__(self, owner, balance=0)
├── deposit(amount) → 无返回值，正数累加
├── withdraw(amount) → 无返回值，正数且≤余额
└── get_balance() → 返回 self.balance
```

### Student + StudentGradeManager（双类协作）

```
StudentGradeManager（管理器类）
├── 实例属性
│   └── students（字典，键=姓名，值=Student对象）
├── __init__(self)
├── add_student(name) → 判重后存入字典
└── get_student(name) → 字典查找，返回对象或None

Student（实体类）
├── 实例属性
│   ├── name（学生姓名）
│   └── grades（成绩列表，初始空）
├── __init__(self, name)
├── add_grade(grade) → 追加到列表
└── average_grade() → 返回平均分，空列表返回0
```

## 关系与依赖

```
BankAccount
└── 自包含，无外部依赖

StudentGradeManager ──持有──▶ Student
        │
        ├── 创建 Student 实例存入 self.students
        ├── add_student() 调用 Student()
        └── get_student() 返回 Student 对象供外部使用
```

- Manager类负责管理生命周期（增删查）
- 实体类负责承载数据（成绩计算）
- 依赖方向：Manager → Student

## 输入 / 输出 / 前置 / 支撑

### BankAccount

| 方法 | 输入 | 输出 | 前置 |
|------|------|------|------|
| BankAccount(owner, balance?) | 持有人名、初始金额（可选） | BankAccount实例 | class定义 |
| deposit(amount) | 存款金额（正数） | 余额累加 | __init__执行 |
| withdraw(amount) | 取款金额（正数、≤余额） | 余额减少或失败提示 | 有余额 |
| get_balance() | 无 | 当前余额 | __init__执行 |

### StudentGradeManager + Student

| 方法 | 输入 | 输出 | 前置 |
|------|------|------|------|
| add_student(name) | 学生姓名 | 字典添加新键值对 | __init__执行 |
| add_grade(grade) | 单个成绩 | 列表追加 | Student实例存在 |
| average_grade() | 无 | 平均分或0 | Student实例存在 |

**支撑工具**：PyCharm（IDE，编写和运行代码）

## 易错点

- 取款校验必须用 **and** 连接两个条件：金额正数 **且** 不超余额
- 平均分计算前必须判断列表非空，否则除以零报错
- 字典判重用 **name not in self.students**，不能用 ==
- 初始余额设为0是业务逻辑：新账户本来就没钱
- 字典存储时键为姓名（字符串），值为对象（Student实例）
